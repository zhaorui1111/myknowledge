# 栈与队列（Stack & Queue）

> 栈与队列是两种最基础的线性数据结构，它们对元素的访问施加了严格的顺序约束。栈遵循后进先出（LIFO, Last In First Out），队列遵循先进先出（FIFO, First In First Out）。理解它们的本质、变体与应用场景，是解决大量算法问题的基石。

---

## 一、栈（Stack）

### 1.1 定义与核心概念

栈（Stack）是一种受限的线性表，只允许在一端（称为**栈顶**, top）进行插入（push）和删除（pop）操作。这种约束使得最后被压入的元素最先被弹出，即 LIFO 特性。

**核心操作**：

| 操作 | 描述 | 时间复杂度 |
|------|------|-----------|
| `push(x)` | 将元素 x 压入栈顶 | O(1) |
| `pop()` | 弹出并返回栈顶元素 | O(1) |
| `peek()` / `top()` | 查看栈顶元素但不弹出 | O(1) |
| `isEmpty()` | 判断栈是否为空 | O(1) |
| `size()` | 返回栈中元素个数 | O(1) |

**物理类比**：一摞盘子——只能从最上面取盘子或放盘子。

### 1.2 栈的底层实现

栈可以基于数组或链表实现。两种方式各有优劣：

| 实现方式 | 优点 | 缺点 |
|---------|------|------|
| 数组（顺序栈） | 缓存友好、随机访问、无额外指针开销 | 需要预分配或动态扩容 |
| 链表（链式栈） | 无需预分配、天然动态大小 | 每个节点额外指针开销、缓存不友好 |

#### 1.2.1 基于数组的栈实现

**Swift 实现**：

```swift
/// 基于数组的泛型栈实现
struct ArrayStack<Element> {
    private var storage: [Element] = []
    
    /// 元素个数
    var count: Int { storage.count }
    
    /// 是否为空
    var isEmpty: Bool { storage.isEmpty }
    
    /// 查看栈顶元素（不弹出）
    var peek: Element? { storage.last }
    
    /// 压入元素 - O(1) 均摊
    mutating func push(_ element: Element) {
        storage.append(element)
    }
    
    /// 弹出栈顶元素 - O(1)
    @discardableResult
    mutating func pop() -> Element? {
        storage.popLast()
    }
}

// 使用示例
var stack = ArrayStack<Int>()
stack.push(10)
stack.push(20)
stack.push(30)
print(stack.peek ?? -1)  // 30
print(stack.pop() ?? -1) // 30
print(stack.pop() ?? -1) // 20
print(stack.isEmpty)      // false
print(stack.pop() ?? -1) // 10
print(stack.isEmpty)      // true
```

#### 1.2.2 基于链表的栈实现

链表栈以链表头部作为栈顶，push/pop 都在头部操作，保证 O(1)。

**Swift 实现**：

```swift
/// 链表节点
class StackNode<T> {
    var value: T
    var next: StackNode?
    init(_ value: T, next: StackNode? = nil) {
        self.value = value
        self.next = next
    }
}

/// 基于链表的栈
class LinkedStack<T> {
    private var head: StackNode<T>?
    private(set) var count: Int = 0
    
    var isEmpty: Bool { head == nil }
    var peek: T? { head?.value }
    
    func push(_ value: T) {
        let node = StackNode(value, next: head)
        head = node
        count += 1
    }
    
    @discardableResult
    func pop() -> T? {
        guard let top = head else { return nil }
        head = top.next
        count -= 1
        return top.value
    }
}

// 使用示例
let linkedStack = LinkedStack<String>()
linkedStack.push("A")
linkedStack.push("B")
linkedStack.push("C")
print(linkedStack.peek ?? "")  // "C"
print(linkedStack.pop() ?? "") // "C"
print(linkedStack.count)        // 2
```

**Objective-C 实现**：

```objectivec
    _count--;
    return val;
}
@end
```

### 1.3 动态扩容分析

基于数组的栈在容量不足时需要扩容。常见策略是**倍增扩容**（capacity *= 2）：

- 单次 push 最坏 O(n)（触发扩容时需要拷贝所有元素）
- **均摊分析**（Amortized Analysis）：n 次 push 总拷贝次数为 1 + 2 + 4 + ... + n ≈ 2n，均摊每次 O(1)
- Swift 的 Array 和 OC 的 NSMutableArray 内部都采用类似策略

**空间复杂度**：O(n)，其中 n 为栈中元素个数。数组实现可能有未使用的预留空间（最多 2n），链表实现每个节点额外一个指针（64 位系统 8 字节）。

### 1.4 系统中的栈

栈在计算机系统中无处不在：

1. **函数调用栈（Call Stack）**：每次函数调用压入一个栈帧（Stack Frame），包含局部变量、返回地址、参数等。递归本质上就是利用调用栈。
2. **表达式求值**：编译器用栈处理中缀表达式转后缀、括号匹配。
3. **撤销/重做（Undo/Redo）**：编辑器用两个栈分别记录操作历史和已撤销操作。
4. **浏览器前进/后退**：两个栈分别维护历史页面和前进页面。
5. **iOS 中的 UINavigationController**：视图控制器栈，push/pop 管理页面层级。
6. **DFS（深度优先搜索）**：递归 DFS 隐式使用调用栈，迭代 DFS 显式使用栈。

## 二、队列（Queue）

### 2.1 定义与核心概念

队列（Queue）是一种受限的线性表，只允许在一端（**队尾**, rear/tail）插入，在另一端（**队头**, front/head）删除。这种约束使得最先入队的元素最先出队，即 FIFO 特性。

**核心操作**：

| 操作 | 描述 | 时间复杂度 |
|------|------|-----------|
| `enqueue(x)` | 将元素 x 加入队尾 | O(1) |
| `dequeue()` | 移除并返回队头元素 | O(1)* |
| `front()` / `peek()` | 查看队头元素但不移除 | O(1) |
| `isEmpty()` | 判断队列是否为空 | O(1) |
| `size()` | 返回队列中元素个数 | O(1) |

*注：基于数组的简单队列 dequeue 为 O(n)（需要移动元素），循环队列为 O(1)。

**物理类比**：排队买票——先来的人先买到票离开。

### 2.2 队列的底层实现

#### 2.2.1 基于链表的队列（推荐）

链表队列维护 head 和 tail 两个指针，入队在 tail 后追加，出队从 head 移除，均为 O(1)。

**Swift 实现**：

```swift
/// 基于链表的泛型队列
class LinkedQueue<T> {
    private class Node {
        var value: T
        var next: Node?
        init(_ value: T) { self.value = value }
    }
    
    private var head: Node?
    private var tail: Node?
    private(set) var count: Int = 0
    
    var isEmpty: Bool { head == nil }
    var front: T? { head?.value }
    
    /// 入队 - O(1)
    func enqueue(_ value: T) {
        let node = Node(value)
        if let tail = tail {
            tail.next = node
        } else {
            head = node
        }
        tail = node
        count += 1
    }
    
    /// 出队 - O(1)
    @discardableResult
    func dequeue() -> T? {
        guard let front = head else { return nil }
        head = front.next
        if head == nil { tail = nil }
        count -= 1
        return front.value
    }
}

// 使用示例
let queue = LinkedQueue<Int>()
queue.enqueue(1)
queue.enqueue(2)
queue.enqueue(3)
print(queue.front ?? -1)    // 1
print(queue.dequeue() ?? -1) // 1
print(queue.dequeue() ?? -1) // 2
print(queue.front ?? -1)    // 3
```


#### 2.2.2 循环队列（Circular Queue / Ring Buffer）

循环队列用固定大小的数组模拟环形结构，通过 front 和 rear 两个索引实现 O(1) 的入队和出队，避免了普通数组队列出队时的元素搬移。

**关键设计**：
- 数组大小为 capacity + 1（牺牲一个位置区分空/满）
- 队空条件：`front == rear`
- 队满条件：`(rear + 1) % size == front`
- 入队：`rear = (rear + 1) % size`
- 出队：`front = (front + 1) % size`

**Swift 实现**：

```swift
/// 循环队列 - 固定容量（LeetCode 622）
struct CircularQueue<T> {
    private var buffer: [T?]
    private var head: Int = 0
    private var tail: Int = 0
    private let cap: Int  // buffer.count = cap + 1
    
    init(capacity: Int) {
        precondition(capacity > 0)
        self.cap = capacity + 1
        self.buffer = Array(repeating: nil, count: capacity + 1)
    }
    
    var isEmpty: Bool { head == tail }
    var isFull: Bool { (tail + 1) % cap == head }
    var count: Int { (tail - head + cap) % cap }
    var front: T? { isEmpty ? nil : buffer[head] }
    var rear: T? { isEmpty ? nil : buffer[(tail - 1 + cap) % cap] }
    
    mutating func enqueue(_ value: T) -> Bool {
        guard !isFull else { return false }
        buffer[tail] = value
        tail = (tail + 1) % cap
        return true
    }
    
    @discardableResult
    mutating func dequeue() -> T? {
        guard !isEmpty else { return nil }
        let value = buffer[head]
        buffer[head] = nil  // 释放引用
        head = (head + 1) % cap
        return value
    }
}

// 使用示例
var cq = CircularQueue<Int>(capacity: 3)
print(cq.enqueue(1)) // true
print(cq.enqueue(2)) // true
print(cq.enqueue(3)) // true
print(cq.enqueue(4)) // false (满)
print(cq.dequeue()!) // 1
print(cq.enqueue(4)) // true (腾出一个位置)
print(cq.front!)     // 2
```

**Objective-C 实现**：

```objectivec
@interface SAKCircularQueue : NSObject
@property (nonatomic, readonly) BOOL isEmpty;
@property (nonatomic, readonly) BOOL isFull;
@property (nonatomic, readonly) NSUInteger count;
- (instancetype)initWithCapacity:(NSUInteger)capacity;
- (BOOL)enqueue:(id)object;
- (nullable id)dequeue;
- (nullable id)front;
- (nullable id)rear;
@end

@implementation SAKCircularQueue {
    NSMutableArray *_buffer;
    NSUInteger _head;
    NSUInteger _tail;
    NSUInteger _cap; // buffer.count = _cap
}

- (instancetype)initWithCapacity:(NSUInteger)capacity {
    self = [super init];
    if (self) {
        _cap = capacity + 1;
        _buffer = [NSMutableArray arrayWithCapacity:_cap];
        for (NSUInteger i = 0; i < _cap; i++) {
            [_buffer addObject:[NSNull null]];
        }
        _head = 0;
        _tail = 0;
    }
    return self;
}

- (BOOL)isEmpty { return _head == _tail; }
- (BOOL)isFull { return (_tail + 1) % _cap == _head; }
- (NSUInteger)count { return (_tail - _head + _cap) % _cap; }

- (id)front {
    if (self.isEmpty) return nil;
    id obj = _buffer[_head];
    return obj == [NSNull null] ? nil : obj;
}

- (id)rear {
    if (self.isEmpty) return nil;
    NSUInteger idx = (_tail - 1 + _cap) % _cap;
    id obj = _buffer[idx];
    return obj == [NSNull null] ? nil : obj;
}

- (BOOL)enqueue:(id)object {
    if (self.isFull) return NO;
    _buffer[_tail] = object;
    _tail = (_tail + 1) % _cap;
    return YES;
}

- (id)dequeue {
    if (self.isEmpty) return nil;
    id obj = _buffer[_head];
    _buffer[_head] = [NSNull null];
    _head = (_head + 1) % _cap;
    return obj == [NSNull null] ? nil : obj;
}
@end
```

### 2.3 系统中的队列

1. **操作系统任务调度**：进程/线程就绪队列（FIFO 调度、多级反馈队列）
2. **消息队列**：iOS 的 RunLoop 事件队列、GCD 的 dispatch queue
3. **BFS（广度优先搜索）**：逐层遍历图/树
4. **缓冲区**：I/O 缓冲、网络数据包缓冲（Ring Buffer）
5. **生产者-消费者模型**：多线程间通过队列传递数据
6. **iOS GCD**：`DispatchQueue` 本质就是任务队列（串行队列 FIFO 执行，并发队列 FIFO 出队但可并行执行）

## 三、双端队列（Deque, Double-Ended Queue）

### 3.1 定义

双端队列（Deque, 读作 "deck"）允许在两端都进行插入和删除操作，是栈和队列的泛化：
- 只用一端 push/pop → 栈
- 一端 push 另一端 pop → 队列
- 两端都可操作 → 双端队列

**核心操作**（均为 O(1)）：

| 操作 | 描述 |
|------|------|
| `pushFront(x)` | 在队头插入 |
| `pushBack(x)` | 在队尾插入 |
| `popFront()` | 从队头删除 |
| `popBack()` | 从队尾删除 |
| `peekFront()` | 查看队头 |
| `peekBack()` | 查看队尾 |

### 3.2 实现

**Swift 实现**（基于双向链表）：

```swift
/// 双端队列 - 基于双向链表
class Deque<T> {
    private class Node {
        var value: T
        var prev: Node?
        var next: Node?
        init(_ value: T) { self.value = value }
    }
    
    private var head: Node?
    private var tail: Node?
    private(set) var count: Int = 0
    
    var isEmpty: Bool { count == 0 }
    var peekFront: T? { head?.value }
    var peekBack: T? { tail?.value }
    
    func pushFront(_ value: T) {
        let node = Node(value)
        node.next = head
        head?.prev = node
        head = node
        if tail == nil { tail = node }
        count += 1
    }
    
    func pushBack(_ value: T) {
        let node = Node(value)
        node.prev = tail
        tail?.next = node
        tail = node
        if head == nil { head = node }
        count += 1
    }
    
    @discardableResult
    func popFront() -> T? {
        guard let front = head else { return nil }
        head = front.next
        head?.prev = nil
        if head == nil { tail = nil }
        count -= 1
        return front.value
    }
    
    @discardableResult
    func popBack() -> T? {
        guard let back = tail else { return nil }
        tail = back.prev
        tail?.next = nil
        if tail == nil { head = nil }
        count -= 1
        return back.value
    }
}

// 使用示例
let deque = Deque<Int>()
deque.pushBack(1)   // [1]
deque.pushBack(2)   // [1, 2]
deque.pushFront(0)  // [0, 1, 2]
print(deque.popFront()!) // 0
print(deque.popBack()!)  // 2
print(deque.peekFront!)  // 1
```

### 3.3 应用场景

- **滑动窗口最大值**：单调双端队列（见第五节）
- **工作窃取调度（Work Stealing）**：每个线程有自己的双端队列，从自己队列的一端取任务，空闲时从其他线程队列的另一端"偷"任务
- **回文检测**：从两端同时弹出比较
- **Python collections.deque**：标准库双端队列，底层是双向链表块

## 四、单调栈（Monotone Stack）

### 4.1 概念与原理

单调栈是一种特殊的栈使用技巧，栈内元素始终保持单调递增或单调递减的顺序。它的核心用途是：**在 O(n) 时间内，为数组中每个元素找到左边/右边第一个比它大（或小）的元素**。

**工作原理**（以「找右边第一个更大元素」为例，维护单调递减栈）：
- 从左到右遍历数组，对于当前元素 x：
  - 若栈非空且栈顶元素 < x，则栈顶元素的「右边第一个更大元素」就是 x，弹出栈顶
  - 重复直到栈为空或栈顶 ≥ x
  - 将 x（的索引）入栈

**时间复杂度**：O(n)——每个元素最多入栈一次、出栈一次，总操作 2n 次。
**空间复杂度**：O(n)——栈最多存储 n 个元素。

**单调栈的四种变体**：

| 目标 | 栈的单调性 | 遍历方向 |
|------|-----------|---------|
| 右边第一个更大 | 栈底到栈顶递减 | 从左到右 |
| 右边第一个更小 | 栈底到栈顶递增 | 从左到右 |
| 左边第一个更大 | 栈底到栈顶递减 | 从右到左 |
| 左边第一个更小 | 栈底到栈顶递增 | 从右到左 |

### 4.2 经典问题：下一个更大元素（Next Greater Element）

给定数组 nums，对每个元素找到它右边第一个比它大的元素，不存在则为 -1。

**Swift 实现**：

```swift
/// 下一个更大元素 - 单调递减栈
/// - Parameter nums: 输入数组
/// - Returns: result[i] 为 nums[i] 右边第一个比它大的元素，不存在为 -1
func nextGreaterElement(_ nums: [Int]) -> [Int] {
    let n = nums.count
    var result = Array(repeating: -1, count: n)
    var stack: [Int] = []  // 存储索引，栈内索引对应的值从底到顶单调递减
    
    for i in 0..<n {
        // 当前元素比栈顶大，说明找到了栈顶的下一个更大元素
        while let top = stack.last, nums[top] < nums[i] {
            result[top] = nums[i]
            stack.removeLast()
        }
        stack.append(i)
    }
    
    return result
}

// 示例
let nums = [2, 1, 2, 4, 3]
print(nextGreaterElement(nums))  // [4, 2, 4, -1, -1]
// 解释：2→4, 1→2, 2→4, 4→无, 3→无
```

**Objective-C 实现**：

```objectivec

    NSMutableArray<NSNumber *> *result = [NSMutableArray arrayWithCapacity:n];
    for (NSUInteger i = 0; i < n; i++) {
        [result addObject:@(-1)];
    }

        NSInteger current = nums[i].integerValue;

            NSUInteger topIdx = stack.lastObject.unsignedIntegerValue;
            if (nums[topIdx].integerValue < current) {
                [stack removeLastObject];
                result[topIdx] = nums[i];
            } else {
                break;
            }
        }
        [stack addObject:@(i)];
    }
    
    return [result copy];
}
```

### 4.3 经典问题：每日温度（Daily Temperatures, LeetCode 739）

给定每日温度数组，返回每天需要等几天才能遇到更高温度。本质就是「下一个更大元素的索引距离」。

**Swift 实现**：

```swift
func dailyTemperatures(_ temperatures: [Int]) -> [Int] {
    let n = temperatures.count
    var result = Array(repeating: 0, count: n)
    var stack: [Int] = []  // 单调递减栈（存索引）
    
    for i in 0..<n {
        while let top = stack.last, temperatures[top] < temperatures[i] {
            stack.removeLast()
            result[top] = i - top  // 等待天数 = 索引差
        }
        stack.append(i)
    }
    
    return result
}

// 示例

// [1, 1, 4, 2, 1, 1, 0, 0]
```


### 4.4 经典问题：柱状图中最大矩形（Largest Rectangle in Histogram, LeetCode 84）

给定柱状图的高度数组，找出能勾勒出的最大矩形面积。

**思路**：对每根柱子，找到它左边和右边第一个比它**矮**的柱子，确定以该柱子高度为矩形高度时的最大宽度。使用**单调递增栈**。

**关键洞察**：当一根柱子被弹出栈时，当前元素就是它右边第一个更矮的，新的栈顶就是它左边第一个更矮的。

```swift
func largestRectangleArea(_ heights: [Int]) -> Int {
    let n = heights.count
    var stack: [Int] = []  // 单调递增栈（存索引）
    var maxArea = 0
    
    for i in 0...n {
        // 在末尾虚拟一个高度为 0 的柱子，确保所有柱子都能被弹出
        let currentHeight = (i == n) ? 0 : heights[i]
        
        while let top = stack.last, heights[top] > currentHeight {
            stack.removeLast()
            let height = heights[top]
            // 宽度：左边界是新栈顶+1，右边界是 i-1
            let width = stack.isEmpty ? i : (i - stack.last! - 1)
            maxArea = max(maxArea, height * width)
        }
        stack.append(i)
    }
    
    return maxArea
}

// 示例
print(largestRectangleArea([2, 1, 5, 6, 2, 3])) // 10
// 解释：高度5和6组成的矩形，宽2高5，面积10
```

**Objective-C 实现**：

```objectivec
- (NSInteger)largestRectangleArea:(NSArray<NSNumber *> *)heights {
    NSUInteger n = heights.count;
    NSMutableArray<NSNumber *> *stack = [NSMutableArray array];
    NSInteger maxArea = 0;
    
    for (NSUInteger i = 0; i <= n; i++) {
        NSInteger currentHeight = (i == n) ? 0 : heights[i].integerValue;
        
        while (stack.count > 0) {
            NSUInteger topIdx = stack.lastObject.unsignedIntegerValue;
            if (heights[topIdx].integerValue > currentHeight) {
                [stack removeLastObject];
                NSInteger height = heights[topIdx].integerValue;
                NSInteger width;
                if (stack.count == 0) {
                    width = (NSInteger)i;
                } else {
                    width = (NSInteger)(i - stack.lastObject.unsignedIntegerValue - 1);
                }
                maxArea = MAX(maxArea, height * width);
            } else {
                break;
            }
        }
        [stack addObject:@(i)];
    }
    
    return maxArea;
}
```

**复杂度分析**：
- 时间：O(n)，每个元素最多入栈出栈各一次
- 空间：O(n)，栈空间

### 4.5 经典问题：接雨水（Trapping Rain Water, LeetCode 42）

给定 n 个非负整数表示柱子高度，计算下雨后能接多少水。

**单调栈解法**（横向计算，逐层累加）：

```swift
func trap(_ height: [Int]) -> Int {
    var stack: [Int] = []  // 单调递减栈
    var water = 0
    
    for i in 0..<height.count {
        while let top = stack.last, height[top] < height[i] {
            stack.removeLast()
            guard let left = stack.last else { break }
            // 宽度：当前位置到左边界之间
            let width = i - left - 1
            // 高度：左右两边较矮的那个减去底部高度
            let h = min(height[left], height[i]) - height[top]
            water += width * h
        }
        stack.append(i)
    }
    
    return water
}

// 示例
print(trap([0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1])) // 6
```

**对比：双指针解法**（纵向计算，更直观）：

```swift
func trapTwoPointers(_ height: [Int]) -> Int {
    var left = 0, right = height.count - 1
    var leftMax = 0, rightMax = 0
    var water = 0
    
    while left < right {
        if height[left] < height[right] {
            if height[left] >= leftMax {
                leftMax = height[left]
            } else {
                water += leftMax - height[left]
            }
            left += 1
        } else {
            if height[right] >= rightMax {
                rightMax = height[right]
            } else {
                water += rightMax - height[right]
            }
            right -= 1
        }
    }
    
    return water
}
```

## 五、单调队列（Monotone Deque）

### 5.1 概念与原理

单调队列是双端队列的一种特殊使用方式，队列内元素保持单调递减（或递增）。它的核心用途是：**在 O(n) 时间内解决滑动窗口最值问题**。

**工作原理**（以滑动窗口最大值为例，维护单调递减队列）：
- 队列存储元素的索引
- 队头始终是当前窗口的最大值
- 新元素入队前，从队尾弹出所有比它小的元素（它们不可能再成为最大值）
- 检查队头是否已滑出窗口，若是则弹出

### 5.2 经典问题：滑动窗口最大值（Sliding Window Maximum, LeetCode 239）

给定数组和窗口大小 k，返回每个窗口位置的最大值。

**暴力法**：每个窗口遍历 k 个元素取最大值，O(nk)。
**单调队列**：O(n)。

**Swift 实现**：

```swift
func maxSlidingWindow(_ nums: [Int], _ k: Int) -> [Int] {
    guard nums.count >= k, k > 0 else { return [] }
    
    var deque: [Int] = []  // 存索引，对应值从队头到队尾单调递减
    var result: [Int] = []
    result.reserveCapacity(nums.count - k + 1)
    
    for i in 0..<nums.count {
        // 1. 移除队头已滑出窗口的元素
        if let front = deque.first, front <= i - k {
            deque.removeFirst()
        }
        
        // 2. 从队尾移除所有比当前元素小的（它们不可能再成为最大值）
        while let back = deque.last, nums[back] <= nums[i] {
            deque.removeLast()
        }
        
        // 3. 当前元素索引入队
        deque.append(i)
        
        // 4. 窗口形成后，队头就是最大值
        if i >= k - 1 {
            result.append(nums[deque.first!])
        }
    }
    
    return result
}

// 示例
print(maxSlidingWindow([1, 3, -1, -3, 5, 3, 6, 7], 3))
// [3, 3, 5, 5, 6, 7]
// 窗口位置:        最大值:
// [1  3  -1] -3  5  3  6  7    → 3
//  1 [3  -1  -3] 5  3  6  7    → 3
//  1  3 [-1  -3  5] 3  6  7    → 5
//  1  3  -1 [-3  5  3] 6  7    → 5
//  1  3  -1  -3 [5  3  6] 7    → 6
//  1  3  -1  -3  5 [3  6  7]   → 7
```

**Objective-C 实现**：

```objectivec
- (NSArray<NSNumber *> *)maxSlidingWindow:(NSArray<NSNumber *> *)nums k:(NSInteger)k {
    if (nums.count < (NSUInteger)k || k <= 0) return @[];
    
    NSMutableArray<NSNumber *> *deque = [NSMutableArray array]; // 存索引
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    
    for (NSUInteger i = 0; i < nums.count; i++) {
        // 移除滑出窗口的队头
        if (deque.count > 0 && deque.firstObject.integerValue <= (NSInteger)i - k) {
            [deque removeObjectAtIndex:0];
        }
        
        // 从队尾移除比当前小的
        while (deque.count > 0) {
            NSUInteger backIdx = deque.lastObject.unsignedIntegerValue;
            if (nums[backIdx].integerValue <= nums[i].integerValue) {
                [deque removeLastObject];
            } else {
                break;
            }
        }
        
        [deque addObject:@(i)];
        
        if (i >= (NSUInteger)(k - 1)) {
            NSUInteger frontIdx = deque.firstObject.unsignedIntegerValue;
            [result addObject:nums[frontIdx]];
        }
    }
    
    return [result copy];
}
```

**复杂度分析**：
- 时间：O(n)，每个元素最多入队出队各一次
- 空间：O(k)，队列最多存储 k 个元素

**注意**：上面 Swift 实现中 `deque.removeFirst()` 是 O(n) 操作。在实际工程中，可以用一个 head 指针代替真正的 removeFirst，或使用真正的双端队列数据结构。对于算法题，这个常数因子通常可以接受。

## 六、经典算法题精讲

### 6.1 用栈实现队列（LeetCode 232）

**思路**：使用两个栈，一个负责入队（pushStack），一个负责出队（popStack）。出队时若 popStack 为空，将 pushStack 全部倒入 popStack。

**均摊分析**：每个元素最多被搬移一次（从 pushStack 到 popStack），所以每个操作均摊 O(1)。

```swift
class MyQueue {
    private var pushStack: [Int] = []  // 入队栈
    private var popStack: [Int] = []   // 出队栈
    
    func push(_ x: Int) {
        pushStack.append(x)
    }
    
    func pop() -> Int {
        transferIfNeeded()
        return popStack.removeLast()
    }
    
    func peek() -> Int {
        transferIfNeeded()
        return popStack.last!
    }
    
    func empty() -> Bool {
        pushStack.isEmpty && popStack.isEmpty
    }
    
    private func transferIfNeeded() {
        if popStack.isEmpty {
            while let val = pushStack.popLast() {
                popStack.append(val)
            }
        }
    }
}

// 使用示例
let myQueue = MyQueue()
myQueue.push(1)
myQueue.push(2)
print(myQueue.peek())  // 1
print(myQueue.pop())   // 1
print(myQueue.empty()) // false
```


### 6.2 用队列实现栈（LeetCode 225）

**思路**：使用一个队列。每次 push 后，将队列中前面的元素依次出队再入队，使新元素到达队头。

```swift
class MyStack {
    private var queue: [Int] = []
    
    func push(_ x: Int) {

            queue.append(queue.removeFirst())
        }
    }
    
    func pop() -> Int {
        queue.removeFirst()
    }
    
    func top() -> Int {
        queue.first!
    }
    
    func empty() -> Bool {
        queue.isEmpty
    }
}

// push 的时间复杂度 O(n)，pop/top 为 O(1)
```

### 6.3 有效的括号（Valid Parentheses, LeetCode 20）

判断字符串中的括号是否有效匹配。

```swift
func isValid(_ s: String) -> Bool {
    var stack: [Character] = []
    let mapping: [Character: Character] = [")": "(", "]": "[", "}": "{"]
    
    for char in s {
        if let expected = mapping[char] {
            // 右括号：检查栈顶是否匹配
            guard let top = stack.popLast(), top == expected else {
                return false
            }
        } else {
            // 左括号：入栈
            stack.append(char)
        }
    }
    
    return stack.isEmpty
}

// 示例
print(isValid("()[]{}"))   // true
print(isValid("(]"))       // false
print(isValid("([)]"))     // false
print(isValid("{[]}"))     // true
```

### 6.4 最小栈（Min Stack, LeetCode 155）

设计一个支持 O(1) 获取最小值的栈。

**思路**：使用辅助栈同步记录每个状态下的最小值。

```swift
class MinStack {
    private var stack: [Int] = []
    private var minStack: [Int] = [Int.max]  // 辅助栈，栈顶为当前最小值
    
    func push(_ val: Int) {
        stack.append(val)
        minStack.append(min(val, minStack.last!))
    }
    
    func pop() {
        stack.removeLast()
        minStack.removeLast()
    }
    
    func top() -> Int {
        stack.last!
    }
    
    func getMin() -> Int {
        minStack.last!
    }
}

// 使用示例
let minStack = MinStack()
minStack.push(-2)
minStack.push(0)
minStack.push(-3)
print(minStack.getMin()) // -3
minStack.pop()
print(minStack.top())    // 0
print(minStack.getMin()) // -2
```

### 6.5 逆波兰表达式求值（Evaluate Reverse Polish Notation, LeetCode 150）

后缀表达式（逆波兰表达式）求值：遇到数字入栈，遇到运算符弹出两个操作数计算后将结果入栈。

```swift
func evalRPN(_ tokens: [String]) -> Int {
    var stack: [Int] = []
    
    for token in tokens {
        switch token {
        case "+":
            let b = stack.removeLast()
            let a = stack.removeLast()
            stack.append(a + b)
        case "-":
            let b = stack.removeLast()
            let a = stack.removeLast()
            stack.append(a - b)
        case "*":
            let b = stack.removeLast()
            let a = stack.removeLast()
            stack.append(a * b)
        case "/":
            let b = stack.removeLast()
            let a = stack.removeLast()
            stack.append(a / b)  // 整数除法，向零截断
        default:
            stack.append(Int(token)!)
        }
    }
    
    return stack.last!
}

// 示例
print(evalRPN(["2", "1", "+", "3", "*"])) // 9  即 (2+1)*3
print(evalRPN(["4", "13", "5", "/", "+"])) // 6  即 4+(13/5)
```

```objectivec
- (NSInteger)evalRPN:(NSArray<NSString *> *)tokens {
    NSMutableArray<NSNumber *> *stack = [NSMutableArray array];
    NSSet *operators = [NSSet setWithArray:@[@"+", @"-", @"*", @"/"]];
    
    for (NSString *token in tokens) {
        if ([operators containsObject:token]) {
            NSInteger b = stack.lastObject.integerValue; [stack removeLastObject];
            NSInteger a = stack.lastObject.integerValue; [stack removeLastObject];
            NSInteger result = 0;
            if ([token isEqualToString:@"+"]) result = a + b;
            else if ([token isEqualToString:@"-"]) result = a - b;
            else if ([token isEqualToString:@"*"]) result = a * b;
            else result = a / b; // 向零截断
            [stack addObject:@(result)];
        } else {
            [stack addObject:@(token.integerValue)];
        }
    }
    
    return stack.lastObject.integerValue;
}
```

### 6.6 字符串解码（Decode String, LeetCode 394）

给定编码字符串如 `"3[a2[c]]"`，解码为 `"accaccacc"`。

**思路**：用两个栈，一个存数字（重复次数），一个存字符串（外层已构建的部分）。

```swift
func decodeString(_ s: String) -> String {
    var numStack: [Int] = []
    var strStack: [String] = []
    var currentStr = ""
    var currentNum = 0
    
    for char in s {
        if char.isNumber {
            currentNum = currentNum * 10 + Int(String(char))!
        } else if char == "[" {
            // 保存当前状态，开始新的子串
            numStack.append(currentNum)
            strStack.append(currentStr)
            currentNum = 0
            currentStr = ""
        } else if char == "]" {
            // 弹出重复次数和外层字符串
            let repeatCount = numStack.removeLast()
            let outerStr = strStack.removeLast()
            currentStr = outerStr + String(repeating: currentStr, count: repeatCount)
        } else {
            currentStr.append(char)
        }
    }
    
    return currentStr
}

// 示例
print(decodeString("3[a]2[bc]"))   // "aaabcbc"
print(decodeString("3[a2[c]]"))    // "accaccacc"
print(decodeString("2[abc]3[cd]ef")) // "abcabccdcdcdef"
```

```objectivec
- (NSString *)decodeString:(NSString *)s {
    NSMutableArray<NSNumber *> *numStack = [NSMutableArray array];
    NSMutableArray<NSString *> *strStack = [NSMutableArray array];
    NSMutableString *currentStr = [NSMutableString string];
    NSInteger currentNum = 0;
    
    for (NSUInteger i = 0; i < s.length; i++) {
        unichar ch = [s characterAtIndex:i];

            currentNum = currentNum * 10 + (ch - '0');
        } else if (ch == '[') {
            [numStack addObject:@(currentNum)];
            [strStack addObject:[currentStr copy]];
            currentNum = 0;
            currentStr = [NSMutableString string];
        } else if (ch == ']') {
            NSInteger repeatCount = numStack.lastObject.integerValue;
            [numStack removeLastObject];
            NSString *outerStr = strStack.lastObject;
            [strStack removeLastObject];
            
            NSMutableString *repeated = [NSMutableString stringWithString:outerStr];
            for (NSInteger j = 0; j < repeatCount; j++) {
                [repeated appendString:currentStr];
            }
            currentStr = repeated;
        } else {
            [currentStr appendFormat:@"%c", ch];
        }
    }
    
    return [currentStr copy];
}
```

### 6.7 基本计算器 II（Basic Calculator II, LeetCode 227）

实现一个基本计算器，支持 `+`, `-`, `*`, `/` 和空格，输入为非负整数。

**思路**：用栈处理优先级。遇到 `*` `/` 立即计算（弹出栈顶与当前数运算后入栈），遇到 `+` `-` 将数字（带符号）入栈，最后栈内所有数求和。

```swift
func calculate(_ s: String) -> Int {
    var stack: [Int] = []
    var currentNum = 0
    var operation: Character = "+"
    let chars = Array(s) + ["+"]  // 末尾加一个运算符触发最后一次计算
    
    for char in chars {
        if char.isNumber {
            currentNum = currentNum * 10 + Int(String(char))!
        } else if char != " " {
            switch operation {
            case "+": stack.append(currentNum)
            case "-": stack.append(-currentNum)
            case "*": stack.append(stack.removeLast() * currentNum)
            case "/": stack.append(stack.removeLast() / currentNum)
            default: break
            }
            operation = char
            currentNum = 0
        }
    }
    
    return stack.reduce(0, +)
}

// 示例
print(calculate("3+2*2"))   // 7
print(calculate(" 3/2 "))   // 1
print(calculate(" 3+5 / 2 ")) // 5
```

### 6.8 队列的 BFS 应用：二叉树层序遍历（LeetCode 102）

```swift
class TreeNode {
    var val: Int
    var left: TreeNode?
    var right: TreeNode?
    init(_ val: Int) { self.val = val }
}

func levelOrder(_ root: TreeNode?) -> [[Int]] {
    guard let root = root else { return [] }
    
    var result: [[Int]] = []
    var queue: [TreeNode] = [root]
    
    while !queue.isEmpty {
        let levelSize = queue.count
        var level: [Int] = []
        
        for _ in 0..<levelSize {
            let node = queue.removeFirst()
            level.append(node.val)
            if let left = node.left { queue.append(left) }
            if let right = node.right { queue.append(right) }
        }
        
        result.append(level)
    }
    
    return result
}
```

### 6.9 优先队列应用：前 K 个高频元素（Top K Frequent Elements, LeetCode 347）

虽然优先队列（堆）将在后续章节详细讲解，这里展示队列思想的扩展。

```swift
func topKFrequent(_ nums: [Int], _ k: Int) -> [Int] {
    // 1. 统计频率
    var freq: [Int: Int] = [:]
    for num in nums { freq[num, default: 0] += 1 }
    
    // 2. 桶排序：索引为频率，值为该频率的元素列表
    var buckets = Array(repeating: [Int](), count: nums.count + 1)
    for (num, count) in freq {
        buckets[count].append(num)
    }
    
    // 3. 从高频到低频收集结果
    var result: [Int] = []
    for i in stride(from: buckets.count - 1, through: 0, by: -1) {
        for num in buckets[i] {
            result.append(num)
            if result.count == k { return result }
        }
    }
    
    return result
}

// 示例
print(topKFrequent([1, 1, 1, 2, 2, 3], 2)) // [1, 2]
```

## 七、栈与递归的关系

### 7.1 递归的本质是栈

每次递归调用，系统会在调用栈上压入一个新的栈帧（Stack Frame），包含：
- 函数参数
- 局部变量
- 返回地址

递归返回时，栈帧弹出，恢复上一层的执行状态。因此，**任何递归算法都可以用显式栈改写为迭代**。

### 7.2 递归转迭代的通用模板

```swift
// 递归版 DFS（以二叉树前序遍历为例）
func preorderRecursive(_ root: TreeNode?) -> [Int] {
    guard let node = root else { return [] }
    return [node.val] + preorderRecursive(node.left) + preorderRecursive(node.right)
}

// 迭代版 DFS（显式栈）
func preorderIterative(_ root: TreeNode?) -> [Int] {
    guard let root = root else { return [] }
    var result: [Int] = []
    var stack: [TreeNode] = [root]
    
    while let node = stack.popLast() {
        result.append(node.val)
        // 先压右再压左，这样左先出栈
        if let right = node.right { stack.append(right) }
        if let left = node.left { stack.append(left) }
    }
    
    return result
}
```

### 7.3 栈溢出与尾递归

**栈溢出（Stack Overflow）**：递归深度过大时，调用栈超出系统限制（通常 1~8 MB），导致程序崩溃。

**解决方案**：
1. **改为迭代 + 显式栈**：堆内存远大于栈内存
2. **尾递归优化（Tail Call Optimization, TCO）**：如果递归调用是函数的最后一个操作，编译器可以复用当前栈帧。Swift 在 Release 模式下支持 TCO，但不保证。
3. **增大栈空间**：iOS 主线程栈 1MB，子线程默认 512KB，可通过 `Thread(stackSize:)` 调整

```swift
// 非尾递归：递归调用后还有 + 操作
func factorial(_ n: Int) -> Int {
    if n <= 1 { return 1 }
    return n * factorial(n - 1)  // 不是尾递归
}

// 尾递归：递归调用是最后一个操作
func factorialTail(_ n: Int, accumulator: Int = 1) -> Int {
    if n <= 1 { return accumulator }
    return factorialTail(n - 1, accumulator: n * accumulator)  // 尾递归
}
```

## 八、优先队列简介（Priority Queue）

优先队列是队列的扩展，每个元素有一个优先级，出队时总是优先级最高（或最低）的元素先出。它通常用**堆（Heap）**实现，将在「10-堆与优先队列」章节详细讲解。

**与普通队列的对比**：

| 特性 | 普通队列 | 优先队列 |
|------|---------|---------|
| 出队顺序 | FIFO（先进先出） | 按优先级 |
| 入队 | O(1) | O(log n) |
| 出队 | O(1) | O(log n) |
| 查看队头 | O(1) | O(1) |
| 底层实现 | 链表/数组 | 二叉堆/斐波那契堆 |

**应用场景**：Dijkstra 最短路径、Huffman 编码、任务调度、合并 K 个有序链表。

### 10.2 线程安全的栈与队列

在多线程环境中，需要对栈/队列的操作加锁：

```swift
/// 线程安全的栈（使用 os_unfair_lock）
class ConcurrentStack<T> {
    private var storage: [T] = []
    private var lock = os_unfair_lock()
    
    func push(_ value: T) {
        os_unfair_lock_lock(&lock)
        defer { os_unfair_lock_unlock(&lock) }
        storage.append(value)
    }
    
    func pop() -> T? {
        os_unfair_lock_lock(&lock)
        defer { os_unfair_lock_unlock(&lock) }
        return storage.popLast()
    }
}
```

```objectivec
/// 线程安全的队列（使用 GCD barrier）
@interface SAKConcurrentQueue<ObjectType> : NSObject
- (void)enqueue:(ObjectType)object;
- (void)dequeueWithCompletion:(void(^)(ObjectType _Nullable))completion;
@end

@implementation SAKConcurrentQueue {
    NSMutableArray *_storage;
    dispatch_queue_t _isolationQueue;
}

- (instancetype)init {
    self = [super init];
    if (self) {
        _storage = [NSMutableArray array];
        _isolationQueue = dispatch_queue_create("com.sak.queue", DISPATCH_QUEUE_CONCURRENT);
    }
    return self;
}

- (void)enqueue:(id)object {
    dispatch_barrier_async(_isolationQueue, ^{
        [self->_storage addObject:object];
    });
}

- (void)dequeueWithCompletion:(void(^)(id))completion {
    dispatch_async(_isolationQueue, ^{
        id obj = self->_storage.firstObject;
        if (obj) {
            dispatch_barrier_async(self->_isolationQueue, ^{
                [self->_storage removeObjectAtIndex:0];
            });
        }
        if (completion) completion(obj);
    });
}
@end
```

### 10.3 无锁队列（Lock-Free Queue）

在高性能场景中，可以使用 CAS（Compare-And-Swap）实现无锁队列，避免锁的开销。iOS 中 `libdispatch` 内部就大量使用了无锁数据结构。

**Michael-Scott 无锁队列**的核心思想：
- 使用 `OSAtomicCompareAndSwapPtrBarrier` 或 Swift 的 `Atomic` 类型
- 入队：CAS 修改 tail.next，然后 CAS 推进 tail
- 出队：CAS 推进 head

这属于高级并发编程范畴，实际开发中通常直接使用 GCD 或 `os_unfair_lock`。

### 10.4 Ring Buffer 在音视频中的应用

音视频处理中，Ring Buffer（循环缓冲区）是最常用的数据结构：

```swift
/// 音频采样 Ring Buffer
struct AudioRingBuffer {
    private var buffer: [Float]
    private var writeIndex: Int = 0
    private var readIndex: Int = 0
    private let capacity: Int
    
    init(capacity: Int) {
        self.capacity = capacity
        self.buffer = Array(repeating: 0, count: capacity)
    }
    
    var availableToRead: Int {
        (writeIndex - readIndex + capacity) % capacity
    }
    
    var availableToWrite: Int {
        capacity - 1 - availableToRead
    }
    
    mutating func write(_ samples: [Float]) -> Int {
        let count = min(samples.count, availableToWrite)
        for i in 0..<count {
            buffer[writeIndex] = samples[i]
            writeIndex = (writeIndex + 1) % capacity
        }
        return count
    }
    
    mutating func read(count: Int) -> [Float] {
        let actualCount = min(count, availableToRead)
        var result = [Float](repeating: 0, count: actualCount)
        for i in 0..<actualCount {
            result[i] = buffer[readIndex]
            readIndex = (readIndex + 1) % capacity
        }
        return result
    }
}
```

## 十一、面试高频题

### 面试题 1：如何用两个栈实现一个队列？时间复杂度是多少？

**答案要点**：
- 使用 pushStack 和 popStack
- 入队：直接 push 到 pushStack，O(1)
- 出队：若 popStack 非空直接 pop；否则将 pushStack 全部倒入 popStack 再 pop
- 均摊时间复杂度：每个元素最多被搬移一次，所以 push 和 pop 均摊都是 O(1)
- 最坏单次 pop 为 O(n)（触发搬移时）

### 面试题 2：设计一个支持 O(1) push、pop、getMin 的栈

**答案要点**：
- 方法一：辅助栈同步记录每个状态的最小值（空间 O(n)）
- 方法二：辅助栈只在新最小值出现时压入（注意处理重复值，用 ≤ 判断）
- 方法三：不用辅助栈，存差值（val - min），空间 O(1)，但有溢出风险

### 面试题 3：如何判断一个入栈序列和出栈序列是否匹配？（LeetCode 946）

**答案要点**：
- 模拟法：按入栈序列依次入栈，每次入栈后检查栈顶是否等于出栈序列当前位置，若是则弹出并推进出栈指针
- 最终栈为空则匹配

```swift
func validateStackSequences(_ pushed: [Int], _ popped: [Int]) -> Bool {
    var stack: [Int] = []
    var j = 0
    
    for val in pushed {
        stack.append(val)
        while let top = stack.last, j < popped.count, top == popped[j] {
            stack.removeLast()
            j += 1
        }
    }
    
    return stack.isEmpty
}
```

### 面试题 4：实现一个 O(1) 时间复杂度的最大栈（支持 popMax）

**答案要点**：
- 双向链表 + TreeMap（平衡BST）：链表存元素顺序，TreeMap 按值索引到链表节点
- popMax：从 TreeMap 找最大值对应的链表节点，O(log n) 删除
- 严格 O(1) 的 popMax 在一般情况下不可能，面试中通常接受 O(log n)

### 面试题 5：用栈实现中缀表达式求值

**答案要点**：
- 两个栈：操作数栈 + 运算符栈
- 遇到数字入操作数栈
- 遇到运算符：若优先级 ≤ 栈顶运算符，先计算栈顶，再入栈
- 遇到 `(`：直接入运算符栈
- 遇到 `)`：计算直到遇到 `(`
- 最后清空运算符栈

### 面试题 6：单调栈的时间复杂度为什么是 O(n)？

**答案要点**：
- 虽然有嵌套循环（外层 for + 内层 while），但每个元素最多入栈一次、出栈一次
- 总的入栈次数 = n，总的出栈次数 ≤ n
- 所以总操作次数 ≤ 2n，时间复杂度 O(n)
- 这是**均摊分析**的经典案例

### 面试题 7：iOS 中 UINavigationController 的 viewControllers 为什么用数组而不是链表？

**答案要点**：
- 数组支持随机访问，方便 `popToViewController(_:animated:)` 直接定位
- 导航栈通常很浅（< 10 层），数组的 O(n) 插入删除在小规模下性能优于链表
- 数组内存连续，缓存友好
- 需要支持 `setViewControllers(_:animated:)` 批量替换

## 十二、易错点与最佳实践

### 12.1 常见错误

1. **空栈/空队列操作**：pop/dequeue 前必须检查 isEmpty，否则数组越界或空指针
2. **循环队列的满/空判断混淆**：牺牲一个位置法中，`(tail+1)%cap == head` 是满，`tail == head` 是空
3. **单调栈方向搞反**：找「更大」用递减栈，找「更小」用递增栈，容易记混
4. **栈溢出**：递归深度过大时忘记改迭代，iOS 子线程栈只有 512KB
5. **双栈队列的 transfer 时机**：只在 popStack 为空时才搬移，不能每次 pop 都搬
6. **OC 中 NSMutableArray 的 removeObjectAtIndex:0**：这是 O(n) 操作，频繁出队应使用链表或 head 指针

### 12.2 最佳实践

1. **选择合适的实现**：
   - 大多数场景用数组栈（Swift Array / OC NSMutableArray）即可
   - 需要严格 O(1) 出队且频繁操作时用链表队列
   - 固定大小缓冲区用循环队列

2. **Swift 中的栈/队列**：
   - Swift 标准库没有 Stack/Queue 类型，直接用 Array（append/popLast 模拟栈，append/removeFirst 模拟队列）
   - `removeFirst()` 是 O(n)，如果性能敏感可用 `Deque`（Swift Collections 包）

3. **OC 中的注意事项**：
   - `NSMutableArray` 的 `removeLastObject` 是 O(1)，适合做栈
   - `removeObjectAtIndex:0` 是 O(n)，不适合做队列的出队操作
   - 高性能场景考虑用 C 数组或 CFMutableArray

4. **算法题中的技巧**：
   - 单调栈题目先确定「找更大还是更小」「找左边还是右边」
   - 柱状图/接雨水类题目在数组末尾加哨兵值简化边界处理
   - BFS 用队列，DFS 用栈（或递归）

## 十三、对比辨析

### 13.1 栈 vs 队列 vs 双端队列

| 特性 | 栈 | 队列 | 双端队列 |
|------|---|------|---------|
| 访问约束 | 只能访问栈顶 | 只能访问队头 | 两端都可访问 |
| 插入位置 | 栈顶 | 队尾 | 两端 |
| 删除位置 | 栈顶 | 队头 | 两端 |
| 顺序特性 | LIFO | FIFO | 灵活 |
| 典型应用 | DFS、括号匹配、撤销 | BFS、任务调度 | 滑动窗口、工作窃取 |

### 13.2 数组实现 vs 链表实现

| 维度 | 数组 | 链表 |
|------|------|------|
| 缓存性能 | 优秀（连续内存） | 较差（节点分散） |
| 空间开销 | 可能有预留空间浪费 | 每节点额外指针 |
| 扩容 | 需要拷贝（均摊 O(1)） | 无需扩容 |
| 随机访问 | O(1) | O(n) |
| 适用场景 | 大多数场景 | 频繁头部操作、大小不确定 |

### 13.3 单调栈 vs 单调队列

| 维度 | 单调栈 | 单调队列 |
|------|--------|---------|
| 数据结构 | 栈（只操作一端） | 双端队列（两端操作） |
| 核心功能 | 找左/右第一个更大/更小 | 滑动窗口最值 |
| 窗口约束 | 无窗口限制 | 有固定窗口大小 |
| 弹出条件 | 只从栈顶弹出 | 队尾弹出（维护单调性）+ 队头弹出（过期元素） |

### 13.4 递归 vs 迭代（显式栈）

| 维度 | 递归 | 迭代+显式栈 |
|------|------|-----------|
| 代码简洁性 | 通常更简洁 | 较冗长 |
| 栈空间 | 受系统栈限制（1~8MB） | 堆内存，几乎无限制 |
| 性能 | 函数调用开销 | 无额外调用开销 |
| 调试 | 栈帧信息丰富 | 需要手动维护状态 |
| 适用场景 | 深度较浅、逻辑清晰 | 深度可能很大、需要精确控制 |

## 十四、进阶话题

### 14.1 表达式解析与 Shunting-Yard 算法

Dijkstra 的调度场算法（Shunting-Yard Algorithm）用于将中缀表达式转换为后缀表达式（逆波兰表达式），核心就是运算符栈：

```
输入: 3 + 4 * 2 / (1 - 5)
处理过程:
  Token  | 输出队列      | 运算符栈
  3      | 3             |
  +      | 3             | +
  4      | 3 4           | +
  *      | 3 4           | + *
  2      | 3 4 2         | + *
  /      | 3 4 2 *       | + /    (* 优先级≥/，弹出)
  (      | 3 4 2 *       | + / (
  1      | 3 4 2 * 1     | + / (
  -      | 3 4 2 * 1     | + / ( -
  5      | 3 4 2 * 1 5   | + / ( -
  )      | 3 4 2 * 1 5 - | + /    (弹出到左括号)
  END    | 3 4 2 * 1 5 - / + |    (清空栈)

后缀表达式: 3 4 2 * 1 5 - / +
计算结果: 3 + (4*2)/(1-5) = 3 + 8/(-4) = 3 + (-2) = 1
```

### 14.2 栈在编译器中的应用

1. **语法分析**：LR 分析器使用分析栈（状态栈 + 符号栈）
2. **语义分析**：属性计算使用属性栈
3. **代码生成**：栈式虚拟机（如 JVM、CPython）的字节码执行

### 14.3 持久化栈（Persistent Stack）

函数式编程中的不可变栈，每次操作返回新版本，旧版本仍可访问：

```swift
/// 持久化栈 - 基于链表的不可变实现
indirect enum PersistentStack<T> {
    case empty
    case node(T, PersistentStack<T>)
    
    var isEmpty: Bool {
        if case .empty = self { return true }
        return false
    }
    
    var peek: T? {
        if case .node(let value, _) = self { return value }
        return nil
    }
    
    func push(_ value: T) -> PersistentStack {
        .node(value, self)
    }
    
    func pop() -> (T, PersistentStack)? {
        if case .node(let value, let rest) = self {
            return (value, rest)
        }
        return nil
    }
}

// 使用：每次操作产生新栈，旧栈不变
let s0 = PersistentStack<Int>.empty
let s1 = s0.push(1)  // [1]
let s2 = s1.push(2)  // [2, 1]
let s3 = s1.push(3)  // [3, 1] - 从 s1 分支
// s1、s2、s3 共存，共享底层节点
```

## 十五、总结

栈与队列虽然是最简单的数据结构，但它们的应用极其广泛：

**栈的核心价值**：
- 管理「最近相关性」——最近打开的括号、最近的函数调用、最近的操作
- 将递归问题转化为迭代
- 单调栈将 O(n²) 的「找最近更大/更小」问题优化到 O(n)

**队列的核心价值**：
- 管理「公平性」——先来先服务
- BFS 的逐层扩展
- 生产者-消费者解耦
- 单调队列将滑动窗口最值从 O(nk) 优化到 O(n)

**选择指南**：
- 需要 LIFO → 栈
- 需要 FIFO → 队列
- 需要两端操作 → 双端队列
- 需要按优先级 → 优先队列（堆）
- 找最近更大/更小 → 单调栈
- 滑动窗口最值 → 单调队列

掌握栈与队列，不仅是理解更复杂数据结构（树、图）的基础，更是解决大量实际工程问题和算法面试题的利器。
