# 哈希表（Hash Table）详解

> **哈希表**（Hash Table / Hash Map）是一种通过**哈希函数**将键（Key）映射到存储位置（桶/Bucket）的数据结构，支持平均 O(1) 时间复杂度的插入、查找和删除操作。它是计算机科学中最重要、最常用的数据结构之一，几乎所有现代编程语言都内置了哈希表实现。

## 1. 基本概念与术语

### 1.1 什么是哈希表

哈希表（Hash Table），也称散列表，是一种根据键（Key）直接访问内存存储位置的数据结构。其核心思想是：通过一个**哈希函数**（Hash Function）将键转换为数组下标，从而实现近乎常数时间的数据访问。

```
Key → Hash Function → Index → Bucket（存储位置）
```

### 1.2 为什么需要哈希表

在没有哈希表的情况下：
- **无序数组**：查找需要 O(n) 线性扫描
- **有序数组**：二分查找 O(log n)，但插入/删除需要移动元素 O(n)
- **平衡 BST**：查找/插入/删除均为 O(log n)

哈希表的优势在于：**平均情况下，插入、查找、删除均为 O(1)**。这使得它成为需要频繁查找的场景的首选数据结构。

### 1.3 核心术语

| 术语 | 英文 | 含义 |
|------|------|------|
| 哈希函数 | Hash Function | 将键映射为数组下标的函数 |
| 哈希值 | Hash Value / Hash Code | 哈希函数的输出 |
| 桶 | Bucket / Slot | 数组中的一个存储位置 |
| 冲突 | Collision | 不同的键映射到同一个桶 |
| 负载因子 | Load Factor | 已存元素数 / 桶总数（α = n/m） |
| 探测 | Probing | 开放寻址法中寻找空桶的过程 |
| 再哈希 | Rehash | 扩容时重新计算所有元素的位置 |
| 链地址法 | Separate Chaining | 每个桶维护一个链表/红黑树 |
| 开放寻址法 | Open Addressing | 冲突时在数组内寻找下一个空位 |

### 1.4 哈希表的抽象接口

```
HashTable<K, V>:
  - put(key: K, value: V)      // 插入或更新
  - get(key: K) -> V?          // 查找
  - remove(key: K) -> V?       // 删除
  - contains(key: K) -> Bool   // 是否存在
  - count -> Int               // 元素数量
  - isEmpty -> Bool            // 是否为空
```

## 2. 哈希函数设计

### 2.1 好的哈希函数的特征

一个优秀的哈希函数应满足：

1. **确定性**（Deterministic）：相同的键必须产生相同的哈希值
2. **均匀性**（Uniform Distribution）：输出尽可能均匀分布在整个值域
3. **高效性**（Efficiency）：计算速度快，O(1) 或 O(key_length)
4. **雪崩效应**（Avalanche Effect）：输入的微小变化导致输出的巨大变化

### 2.2 整数哈希

最简单的哈希方法是**除留余数法**（Division Method）：

```
h(k) = k mod m
```

其中 m 是桶的数量。m 的选择很关键：
- **避免 2 的幂**：如果 m = 2^p，则 h(k) 只取决于 k 的低 p 位，高位信息丢失
- **推荐质数**：选择不接近 2 的幂的质数，如 53、97、193、389、769、1543...
- **Knuth 乘法散列**：h(k) = floor(m × (k × A mod 1))，其中 A ≈ (√5 - 1)/2 ≈ 0.6180339887

**乘法散列法**（Multiplication Method）：

```
h(k) = floor(m × frac(k × A))
```

其中 frac(x) 取小数部分，A 为常数（Knuth 建议 A = (√5 - 1)/2）。

**位运算优化**（当 m = 2^p 时）：

```
h(k) = (k * 2654435769) >> (32 - p)   // 32位整数
```

2654435769 = floor(2^32 × (√5 - 1)/2)，这是 Knuth 乘法散列的整数版本。

### 2.3 字符串哈希

字符串哈希需要将变长输入映射为固定大小的整数。常见方法：

**多项式滚动哈希**（Polynomial Rolling Hash）：

```
h(s) = s[0] × p^(n-1) + s[1] × p^(n-2) + ... + s[n-1] × p^0  (mod m)
```

其中 p 是一个质数基数（常用 31 或 37），m 是模数。

**DJB2 哈希**（Daniel J. Bernstein）：

```
hash = 5381
for c in string:
    hash = hash * 33 + c    // 等价于 (hash << 5) + hash + c
```

**FNV-1a 哈希**（Fowler-Noll-Vo）：

```
hash = FNV_offset_basis  // 2166136261 (32-bit)
for byte in data:
    hash = hash XOR byte
    hash = hash * FNV_prime  // 16777619 (32-bit)
```

### 2.4 Swift 中的 Hashable 协议

Swift 使用 `Hashable` 协议来定义类型的哈希行为：

```swift
protocol Hashable: Equatable {
    func hash(into hasher: inout Hasher)
}
```

Swift 标准库使用 **SipHash-1-3**（从 Swift 4.2 开始）作为默认哈希算法，具有抗哈希洪泛攻击（Hash-Flooding Attack）的特性。每次程序启动时使用随机种子，因此哈希值在不同运行之间不可复现。

```swift
// 自定义类型实现 Hashable
struct Point: Hashable {
    let x: Int
    let y: Int
    
    func hash(into hasher: inout Hasher) {
        hasher.combine(x)
        hasher.combine(y)
    }
}

// 使用
let p = Point(x: 3, y: 4)
let hashValue = p.hashValue  // 每次运行可能不同
```

### 2.5 Objective-C 中的 hash 方法

```objectivec
// NSObject 的 hash 方法
@interface MyKey : NSObject <NSCopying>
@property (nonatomic, assign) NSInteger x;
@property (nonatomic, assign) NSInteger y;
@end

@implementation MyKey

- (NSUInteger)hash {
    // 经典组合方式：位移 + 异或
    return (NSUInteger)self.x * 31 + (NSUInteger)self.y;
}

- (BOOL)isEqual:(id)object {
    if (self == object) return YES;
    if (![object isKindOfClass:[MyKey class]]) return NO;
    MyKey *other = (MyKey *)object;
    return self.x == other.x && self.y == other.y;
}

// 作为 NSDictionary 的 key 必须实现 NSCopying
- (id)copyWithZone:(NSZone *)zone {
    MyKey *copy = [[MyKey allocWithZone:zone] init];
    copy.x = self.x;
    copy.y = self.y;
    return copy;
}

@end
```

**重要规则**：如果两个对象 `isEqual:` 返回 YES，则它们的 `hash` 必须相同。反之不必成立（冲突是允许的）。

### 2.6 通用哈希（Universal Hashing）

为了防止恶意输入导致所有键都冲突（退化为 O(n)），可以使用**通用哈希族**：

从一族哈希函数中随机选择一个，使得对于任意两个不同的键 x ≠ y：

```
Pr[h(x) = h(y)] ≤ 1/m
```

Carter-Wegman 通用哈希族：

```
h_{a,b}(k) = ((a × k + b) mod p) mod m
```

其中 p 是大于所有键的质数，a ∈ {1, ..., p-1}，b ∈ {0, ..., p-1} 随机选取。

## 3. 冲突解决策略

### 3.1 为什么冲突不可避免

根据**鸽巢原理**（Pigeonhole Principle）：如果键的可能取值数量大于桶的数量，则必然存在至少两个键映射到同一个桶。

例如：将所有可能的字符串映射到大小为 m 的数组，字符串的数量是无限的，而桶只有 m 个，冲突不可避免。

**生日悖论**（Birthday Paradox）告诉我们：在 m 个桶中，只需约 √(π×m/2) 个元素就有 50% 的概率发生冲突。对于 m = 365（生日问题），只需 23 人就有 50% 概率有两人同一天生日。

### 3.2 链地址法（Separate Chaining）

每个桶维护一个链表（或其他容器），所有映射到同一桶的元素都存储在该链表中。

```
Bucket[0] → [K1:V1] → [K5:V5] → nil
Bucket[1] → [K2:V2] → nil
Bucket[2] → nil
Bucket[3] → [K3:V3] → [K7:V7] → [K9:V9] → nil
...
```

**优点**：
- 实现简单
- 删除操作方便
- 负载因子可以大于 1
- 对哈希函数质量不太敏感

**缺点**：
- 链表节点需要额外内存（指针开销）
- 缓存不友好（链表节点分散在内存中）
- 链表过长时退化为 O(n)

**优化**：当链表长度超过阈值时转为红黑树（Java 8 的 HashMap 在链表长度 > 8 时转红黑树）。

### 3.3 开放寻址法（Open Addressing）

所有元素都存储在数组本身中，冲突时按照某种**探测序列**寻找下一个空位。

#### 3.3.1 线性探测（Linear Probing）

```
h(k, i) = (h'(k) + i) mod m,  i = 0, 1, 2, ...
```

优点：缓存友好（连续内存访问）。
缺点：**一次聚集**（Primary Clustering）——连续被占用的槽形成"簇"，新元素更容易落入簇中，导致簇越来越大。

#### 3.3.2 二次探测（Quadratic Probing）

```
h(k, i) = (h'(k) + c1×i + c2×i²) mod m,  i = 0, 1, 2, ...
```

常用 c1 = 0, c2 = 1：h(k, i) = (h'(k) + i²) mod m

优点：减轻一次聚集。
缺点：**二次聚集**（Secondary Clustering）——相同初始位置的键有相同的探测序列。且不保证能遍历所有桶（m 为质数且负载因子 < 0.5 时可保证）。

#### 3.3.3 双重哈希（Double Hashing）

```
h(k, i) = (h1(k) + i × h2(k)) mod m,  i = 0, 1, 2, ...
```

其中 h2(k) 必须与 m 互质（如 m 为质数，h2(k) < m 即可）。

优点：探测序列依赖于键本身，几乎消除聚集。
缺点：需要计算两个哈希函数。

#### 3.3.4 Robin Hood 哈希

一种改进的线性探测：插入时，如果当前元素的"探测距离"（probe distance，即当前位置与理想位置的距离）小于待插入元素的探测距离，则交换两者。这使得所有元素的探测距离趋于均匀，减少最坏情况查找时间。

### 3.4 开放寻址法的删除问题

直接删除会破坏探测链。解决方案：

1. **墓碑标记**（Tombstone / Deleted Mark）：删除时不真正清空，而是标记为"已删除"。查找时跳过墓碑继续探测，插入时可以复用墓碑位置。缺点：墓碑积累会降低性能，需要定期重建。

2. **后移法**（Backward Shift Deletion）：删除后将后续元素前移填补空位。实现复杂但避免了墓碑问题。

## 4. 链地址法实现

### 4.1 Swift 实现

```swift
/// 基于链地址法的哈希表实现
class ChainingHashTable<Key: Hashable, Value> {
    
    /// 链表节点
    private class Node {
        var key: Key
        var value: Value
        var next: Node?
        
        init(key: Key, value: Value, next: Node? = nil) {
            self.key = key
            self.value = value
            self.next = next
        }
    }
    
    private var buckets: [Node?]
    private(set) var count: Int = 0
    private let maxLoadFactor: Double = 0.75
    
    var isEmpty: Bool { count == 0 }
    
    /// 初始化，capacity 为初始桶数量
    init(capacity: Int = 16) {
        let cap = max(1, capacity)
        self.buckets = Array(repeating: nil, count: cap)
    }
    
    /// 计算键对应的桶索引
    private func bucketIndex(for key: Key) -> Int {
        var hasher = Hasher()
        hasher.combine(key)
        let hashValue = hasher.finalize()
        // 使用位与代替取模（当容量为2的幂时）
        // 这里用通用取模方式
        let index = abs(hashValue) % buckets.count
        return index
    }
    
    /// 当前负载因子
    var loadFactor: Double {
        return Double(count) / Double(buckets.count)
    }
    
    // MARK: - 核心操作
    
    /// 插入或更新键值对，返回旧值（如果是更新）
    @discardableResult
    func put(_ key: Key, value: Value) -> Value? {
        if loadFactor >= maxLoadFactor {
            resize(newCapacity: buckets.count * 2)
        }
        
        let index = bucketIndex(for: key)
        
        // 遍历链表查找是否已存在
        var current = buckets[index]
        while let node = current {
            if node.key == key {
                let oldValue = node.value
                node.value = value
                return oldValue  // 返回旧值表示更新
            }
            current = node.next
        }
        
        // 不存在，头插法插入新节点（O(1)）
        let newNode = Node(key: key, value: value, next: buckets[index])
        buckets[index] = newNode
        count += 1
        return nil  // 返回 nil 表示新插入
    }
    
    /// 查找键对应的值
    func get(_ key: Key) -> Value? {
        let index = bucketIndex(for: key)
        
        var current = buckets[index]
        while let node = current {
            if node.key == key {
                return node.value
            }
            current = node.next
        }
        return nil
    }
    
    /// 删除键值对，返回被删除的值
    @discardableResult
    func remove(_ key: Key) -> Value? {
        let index = bucketIndex(for: key)
        
        guard let head = buckets[index] else { return nil }
        
        // 头节点匹配
        if head.key == key {
            buckets[index] = head.next
            count -= 1
            return head.value
        }
        
        // 遍历查找前驱节点
        var prev = head
        var current = head.next
        while let node = current {
            if node.key == key {
                prev.next = node.next
                count -= 1
                return node.value
            }
            prev = node
            current = node.next
        }
        
        return nil
    }
    
    /// 是否包含某个键
    func contains(_ key: Key) -> Bool {
        return get(key) != nil
    }
    
    // MARK: - 扩容
    
    private func resize(newCapacity: Int) {
        let oldBuckets = buckets
        buckets = Array(repeating: nil, count: newCapacity)
        count = 0
        
        for head in oldBuckets {
            var current = head
            while let node = current {
                put(node.key, value: node.value)
                current = node.next
            }
        }
    }
    
    // MARK: - 下标访问
    
    subscript(key: Key) -> Value? {
        get { return get(key) }
        set {
            if let value = newValue {
                put(key, value: value)
            } else {
                remove(key)
            }
        }
    }
    
    // MARK: - 遍历
    
    func forEach(_ body: (Key, Value) -> Void) {
        for head in buckets {
            var current = head
            while let node = current {
                body(node.key, node.value)
                current = node.next
            }
        }
    }
    
    /// 所有键
    var keys: [Key] {
        var result: [Key] = []
        forEach { key, _ in result.append(key) }
        return result
    }
    
    /// 所有值
    var values: [Value] {
        var result: [Value] = []
        forEach { _, value in result.append(value) }
        return result
    }
}

// MARK: - 使用示例

let table = ChainingHashTable<String, Int>()
table.put("apple", value: 5)
table.put("banana", value: 3)
table.put("cherry", value: 8)

print(table.get("apple")!)     // 5
print(table.contains("grape")) // false

table["banana"] = 10
print(table["banana"]!)        // 10

table.remove("apple")
print(table.contains("apple")) // false
print(table.count)             // 2
```

### 4.2 Objective-C 实现

```objectivec
#import <Foundation/Foundation.h>

// MARK: - 链表节点

@interface _SAKHTNode : NSObject
@property (nonatomic, strong) id<NSCopying> key;
@property (nonatomic, strong) id value;
@property (nonatomic, strong, nullable) _SAKHTNode *next;
- (instancetype)initWithKey:(id<NSCopying>)key value:(id)value next:(_SAKHTNode * _Nullable)next;
@end

@implementation _SAKHTNode
- (instancetype)initWithKey:(id<NSCopying>)key value:(id)value next:(_SAKHTNode *)next {
    self = [super init];
    if (self) {
        _key = key;
        _value = value;
        _next = next;
    }
    return self;
}
@end

// MARK: - 链地址法哈希表

@interface SAKChainingHashTable : NSObject

@property (nonatomic, readonly) NSUInteger count;
@property (nonatomic, readonly) BOOL isEmpty;

- (instancetype)initWithCapacity:(NSUInteger)capacity;
- (void)setObject:(id)value forKey:(id<NSCopying>)key;
- (nullable id)objectForKey:(id<NSCopying>)key;
- (nullable id)removeObjectForKey:(id<NSCopying>)key;
- (BOOL)containsKey:(id<NSCopying>)key;
- (void)enumerateKeysAndValuesUsingBlock:(void (^)(id key, id value, BOOL *stop))block;

@end

@interface SAKChainingHashTable ()
@property (nonatomic, strong) NSMutableArray *buckets;
@property (nonatomic, assign) NSUInteger count;
@property (nonatomic, assign) NSUInteger capacity;
@end

@implementation SAKChainingHashTable

static const double kSAKMaxLoadFactor = 0.75;

- (instancetype)initWithCapacity:(NSUInteger)capacity {
    self = [super init];
    if (self) {
        _capacity = MAX(1, capacity);
        _count = 0;
        _buckets = [NSMutableArray arrayWithCapacity:_capacity];
        for (NSUInteger i = 0; i < _capacity; i++) {
            [_buckets addObject:[NSNull null]];
        }
    }
    return self;
}

- (instancetype)init {
    return [self initWithCapacity:16];
}

- (BOOL)isEmpty {
    return _count == 0;
}

- (NSUInteger)_bucketIndexForKey:(id<NSCopying>)key {
    NSUInteger h = [(id)key hash];
    return h % _capacity;
}

- (double)_loadFactor {
    return (double)_count / (double)_capacity;
}

- (void)setObject:(id)value forKey:(id<NSCopying>)key {
    NSParameterAssert(key != nil);
    NSParameterAssert(value != nil);
    
    if ([self _loadFactor] >= kSAKMaxLoadFactor) {
        [self _resizeWithNewCapacity:_capacity * 2];
    }
    
    NSUInteger index = [self _bucketIndexForKey:key];
    id bucketHead = _buckets[index];
    
    // 遍历链表查找已有键
    if (bucketHead != [NSNull null]) {
        _SAKHTNode *current = (_SAKHTNode *)bucketHead;
        while (current) {
            if ([(id)current.key isEqual:(id)key]) {
                current.value = value;  // 更新
                return;
            }
            current = current.next;
        }
    }
    
    // 头插法
    _SAKHTNode *existingHead = (bucketHead == [NSNull null]) ? nil : (_SAKHTNode *)bucketHead;
    _SAKHTNode *newNode = [[_SAKHTNode alloc] initWithKey:key value:value next:existingHead];
    _buckets[index] = newNode;
    _count++;
}

- (nullable id)objectForKey:(id<NSCopying>)key {
    NSUInteger index = [self _bucketIndexForKey:key];
    id bucketHead = _buckets[index];
    
    if (bucketHead == [NSNull null]) return nil;
    
    _SAKHTNode *current = (_SAKHTNode *)bucketHead;
    while (current) {
        if ([(id)current.key isEqual:(id)key]) {
            return current.value;
        }
        current = current.next;
    }
    return nil;
}

- (nullable id)removeObjectForKey:(id<NSCopying>)key {
    NSUInteger index = [self _bucketIndexForKey:key];
    id bucketHead = _buckets[index];
    
    if (bucketHead == [NSNull null]) return nil;
    
    _SAKHTNode *head = (_SAKHTNode *)bucketHead;
    
    // 头节点匹配
    if ([(id)head.key isEqual:(id)key]) {
        id removedValue = head.value;
        _buckets[index] = head.next ? (id)head.next : [NSNull null];
        _count--;
        return removedValue;
    }
    
    // 遍历查找
    _SAKHTNode *prev = head;
    _SAKHTNode *current = head.next;
    while (current) {
        if ([(id)current.key isEqual:(id)key]) {
            id removedValue = current.value;
            prev.next = current.next;
            _count--;
            return removedValue;
        }
        prev = current;
        current = current.next;
    }
    return nil;
}

- (BOOL)containsKey:(id<NSCopying>)key {
    return [self objectForKey:key] != nil;
}

- (void)_resizeWithNewCapacity:(NSUInteger)newCapacity {
    NSMutableArray *oldBuckets = _buckets;
    NSUInteger oldCapacity = _capacity;
    
    _capacity = newCapacity;
    _count = 0;
    _buckets = [NSMutableArray arrayWithCapacity:newCapacity];
    for (NSUInteger i = 0; i < newCapacity; i++) {
        [_buckets addObject:[NSNull null]];
    }
    
    for (NSUInteger i = 0; i < oldCapacity; i++) {
        id bucketHead = oldBuckets[i];
        if (bucketHead != [NSNull null]) {
            _SAKHTNode *current = (_SAKHTNode *)bucketHead;
            while (current) {
                [self setObject:current.value forKey:current.key];
                current = current.next;
            }
        }
    }
}

- (void)enumerateKeysAndValuesUsingBlock:(void (^)(id, id, BOOL *))block {
    BOOL stop = NO;
    for (NSUInteger i = 0; i < _capacity && !stop; i++) {
        id bucketHead = _buckets[i];
        if (bucketHead != [NSNull null]) {
            _SAKHTNode *current = (_SAKHTNode *)bucketHead;
            while (current && !stop) {
                block((id)current.key, current.value, &stop);
                current = current.next;
            }
        }
    }
}
```

## 5. 开放寻址法实现

### 5.1 Swift 实现（线性探测）

```swift
/// 基于开放寻址法（线性探测）的哈希表
class LinearProbingHashTable<Key: Hashable, Value> {
    
    /// 槽的状态
    private enum SlotState {
        case empty       // 从未使用
        case occupied    // 已占用
        case deleted     // 已删除（墓碑）
    }
    
    private var keys: [Key?]
    private var values: [Value?]
    private var states: [SlotState]
    private(set) var count: Int = 0
    private var tombstoneCount: Int = 0
    private let maxLoadFactor: Double = 0.6  // 开放寻址法负载因子应更低
    
    var isEmpty: Bool { count == 0 }
    
    init(capacity: Int = 16) {
        let cap = max(4, capacity)
        self.keys = Array(repeating: nil, count: cap)
        self.values = Array(repeating: nil, count: cap)
        self.states = Array(repeating: .empty, count: cap)
    }
    
    private var capacity: Int { keys.count }
    
    /// 有效负载因子（包含墓碑）
    private var effectiveLoadFactor: Double {
        return Double(count + tombstoneCount) / Double(capacity)
    }
    
    /// 计算初始索引
    private func hash(for key: Key) -> Int {
        var hasher = Hasher()
        hasher.combine(key)
        let h = hasher.finalize()
        return abs(h) % capacity
    }
    
    // MARK: - 核心操作
    
    /// 插入或更新
    @discardableResult
    func put(_ key: Key, value: Value) -> Value? {
        if effectiveLoadFactor >= maxLoadFactor {
            resize(newCapacity: capacity * 2)
        }
        
        var index = hash(for: key)
        var firstTombstone: Int? = nil
        
        // 线性探测
        while true {
            switch states[index] {
            case .empty:
                // 找到空位，插入
                let insertIndex = firstTombstone ?? index
                keys[insertIndex] = key
                values[insertIndex] = value
                if firstTombstone != nil {
                    // 复用墓碑位置
                    states[insertIndex] = .occupied
                    tombstoneCount -= 1
                } else {
                    states[insertIndex] = .occupied
                }
                count += 1
                return nil
                
            case .occupied:
                if keys[index] == key {
                    // 找到已有键，更新
                    let oldValue = values[index]
                    values[index] = value
                    return oldValue
                }
                
            case .deleted:
                // 记录第一个墓碑位置
                if firstTombstone == nil {
                    firstTombstone = index
                }
            }
            
            index = (index + 1) % capacity
        }
    }
    
    /// 查找
    func get(_ key: Key) -> Value? {
        var index = hash(for: key)
        
        while true {
            switch states[index] {
            case .empty:
                return nil  // 遇到空位，键不存在
                
            case .occupied:
                if keys[index] == key {
                    return values[index]
                }
                
            case .deleted:
                break  // 跳过墓碑继续探测
            }
            
            index = (index + 1) % capacity
        }
    }
    
    /// 删除（墓碑标记法）
    @discardableResult
    func remove(_ key: Key) -> Value? {
        var index = hash(for: key)
        
        while true {
            switch states[index] {
            case .empty:
                return nil
                
            case .occupied:
                if keys[index] == key {
                    let removedValue = values[index]
                    keys[index] = nil
                    values[index] = nil
                    states[index] = .deleted
                    count -= 1
                    tombstoneCount += 1
                    return removedValue
                }
                
            case .deleted:
                break
            }
            
            index = (index + 1) % capacity
        }
    }
    
    func contains(_ key: Key) -> Bool {
        return get(key) != nil
    }
    
    // MARK: - 扩容（同时清除墓碑）
    
    private func resize(newCapacity: Int) {
        let oldKeys = keys
        let oldValues = values
        let oldStates = states
        let oldCapacity = capacity
        
        keys = Array(repeating: nil, count: newCapacity)
        values = Array(repeating: nil, count: newCapacity)
        states = Array(repeating: .empty, count: newCapacity)
        count = 0
        tombstoneCount = 0
        
        for i in 0..<oldCapacity {
            if oldStates[i] == .occupied, let key = oldKeys[i], let value = oldValues[i] {
                put(key, value: value)
            }
        }
    }
    
    subscript(key: Key) -> Value? {
        get { return get(key) }
        set {
            if let value = newValue {
                put(key, value: value)
            } else {
                remove(key)
            }
        }
    }
}
```

```objectivec
    NSInteger firstTombstone = -1;

            NSUInteger insertIndex = (firstTombstone >= 0) ? (NSUInteger)firstTombstone : index;

        if (state == SAKSlotStateEmpty) {
            return nil;
        } else if (state == SAKSlotStateOccupied) {
            if ([(id)_keySlots[index] isEqual:(id)key]) {
                return _valueSlots[index];

            return nil;

            if ([(id)_keySlots[index] isEqual:(id)key]) {

                _keySlots[index] = [NSNull null];
                _valueSlots[index] = [NSNull null];
                _slotStates[index] = SAKSlotStateDeleted;
                _count--;
                _tombstoneCount++;
                return removedValue;
            }
        }
        
        index = (index + 1) % _capacity;
    }

    _valueSlots = [NSMutableArray arrayWithCapacity:newCapacity];

        }
    }
```

## 6. 负载因子与动态扩容

### 6.1 负载因子的意义

负载因子 α = n/m（n 为元素数，m 为桶数）直接影响哈希表的性能：

**链地址法**：
- 平均查找长度（成功）：1 + α/2
- 平均查找长度（失败）：α
- α 可以大于 1，但建议 α ≤ 0.75

**线性探测**：
- 平均查找长度（成功）：≈ (1/2)(1 + 1/(1-α))
- 平均查找长度（失败）：≈ (1/2)(1 + 1/(1-α)²)
- α 接近 1 时性能急剧下降，建议 α ≤ 0.7

| 负载因子 α | 链地址法（成功） | 线性探测（成功） | 线性探测（失败） |
|-----------|----------------|----------------|----------------|
| 0.25 | 1.125 | 1.17 | 1.39 |
| 0.50 | 1.250 | 1.50 | 2.50 |
| 0.75 | 1.375 | 2.50 | 8.50 |
| 0.90 | 1.450 | 5.50 | 50.5 |

### 6.2 扩容策略

当负载因子超过阈值时，需要扩容（Resize / Rehash）：

1. 创建一个更大的新数组（通常是原来的 2 倍）
2. 遍历旧数组中的所有元素
3. 对每个元素重新计算哈希值，插入新数组
4. 释放旧数组

**扩容的时间复杂度**：单次扩容 O(n)，但使用**均摊分析**（Amortized Analysis），每次插入的均摊时间仍为 O(1)。

### 6.3 均摊分析

假设初始容量为 1，每次满时翻倍：

```
插入第 1 个元素：扩容代价 1（从 0 扩到 1）
插入第 2 个元素：扩容代价 1（从 1 扩到 2）
插入第 3 个元素：扩容代价 2（从 2 扩到 4）
插入第 5 个元素：扩容代价 4（从 4 扩到 8）
插入第 9 个元素：扩容代价 8（从 8 扩到 16）
...
插入第 2^k + 1 个元素：扩容代价 2^k
```

插入 n 个元素的总代价：n（普通插入）+ 1 + 2 + 4 + ... + n/2 ≈ n + n = 2n

均摊每次插入代价：2n / n = O(1)

### 6.4 缩容策略

为了避免内存浪费，当负载因子过低时可以缩容：
- 通常在 α < 0.25 时缩容为原来的 1/2
- 注意避免"抖动"：如果扩容阈值为 0.75，缩容阈值不应设为 0.375（0.75/2），否则在边界反复插入删除会频繁扩缩容

### 6.5 Swift Dictionary 的扩容实现

Swift 的 `Dictionary` 内部使用开放寻址法（变体），扩容策略：
- 初始容量为 0（空字典不分配内存）
- 第一次插入时分配最小容量（通常为 2 或 4）
- 负载因子超过约 3/4 时翻倍扩容
- 容量始终为 2 的幂（便于位运算取模）

### 6.6 NSDictionary 的实现细节

Apple 的 `NSDictionary`/`NSMutableDictionary` 底层使用 **CFBasicHash**：
- 采用线性探测的开放寻址法
- 容量选择为特定的质数序列（3, 7, 13, 23, 41, 71, ...）
- 负载因子阈值约为 0.75
- Key 必须实现 `NSCopying`（字典会 copy key 以防止外部修改）

## 8. 经典算法题精讲

### 8.1 两数之和（Two Sum）

**题目**：给定数组 nums 和目标值 target，找出和为 target 的两个数的下标。

**思路**：遍历数组，对每个元素 x，在哈希表中查找 target - x 是否存在。

```swift
// Swift 实现
func twoSum(_ nums: [Int], _ target: Int) -> [Int] {
    // key: 数值, value: 下标
    var map: [Int: Int] = [:]
    
    for (i, num) in nums.enumerated() {
        let complement = target - num
        if let j = map[complement] {
            return [j, i]
        }
        map[num] = i
    }
    
    return []  // 题目保证有解，不会走到这里
}

// 测试
let result = twoSum([2, 7, 11, 15], 9)
print(result)  // [0, 1]
```

```objectivec
// Objective-C 实现
- (NSArray<NSNumber *> *)twoSum:(NSArray<NSNumber *> *)nums target:(NSInteger)target {
    // key: 数值, value: 下标
    NSMutableDictionary<NSNumber *, NSNumber *> *map = [NSMutableDictionary dictionary];
    
    for (NSUInteger i = 0; i < nums.count; i++) {
        NSInteger num = nums[i].integerValue;
        NSInteger complement = target - num;
        NSNumber *j = map[@(complement)];
        if (j != nil) {
            return @[j, @(i)];
        }
        map[@(num)] = @(i);
    }
    
    return @[];
}
```

**复杂度**：时间 O(n)，空间 O(n)。

### 8.2 字母异位词分组（Group Anagrams）

**题目**：给定字符串数组，将字母异位词组合在一起。

**思路**：将每个字符串排序后作为 key，原字符串作为 value 加入对应的列表。

```swift
func groupAnagrams(_ strs: [String]) -> [[String]] {
    var map: [String: [String]] = [:]
    
    for str in strs {
        // 排序后的字符串作为 key
        let sortedKey = String(str.sorted())
        map[sortedKey, default: []].append(str)
    }
    
    return Array(map.values)
}

// 优化：用字符计数数组作为 key（避免排序的 O(k log k)）
func groupAnagramsOptimized(_ strs: [String]) -> [[String]] {
    var map: [[Int]: [String]] = [:]
    
    for str in strs {
        // 26 个字母的计数数组作为 key
        var count = Array(repeating: 0, count: 26)
        let aValue = Character("a").asciiValue!
        for char in str.utf8 {
            count[Int(char - aValue)] += 1
        }
        map[count, default: []].append(str)
    }
    
    return Array(map.values)
}

// 测试
let groups = groupAnagrams(["eat", "tea", "tan", "ate", "nat", "bat"])
// [["eat", "tea", "ate"], ["tan", "nat"], ["bat"]]
```

```objectivec
- (NSArray<NSArray<NSString *> *> *)groupAnagrams:(NSArray<NSString *> *)strs {
    NSMutableDictionary<NSString *, NSMutableArray<NSString *> *> *map = [NSMutableDictionary dictionary];
    
    for (NSString *str in strs) {
        // 排序字符串作为 key
        NSArray *chars = [self _sortedCharsOfString:str];
        NSString *sortedKey = [chars componentsJoinedByString:@""];
        
        if (!map[sortedKey]) {
            map[sortedKey] = [NSMutableArray array];
        }
        [map[sortedKey] addObject:str];
    }
    
    return [map allValues];
}

- (NSArray<NSString *> *)_sortedCharsOfString:(NSString *)str {
    NSMutableArray *chars = [NSMutableArray arrayWithCapacity:str.length];
    for (NSUInteger i = 0; i < str.length; i++) {
        [chars addObject:[NSString stringWithFormat:@"%C", [str characterAtIndex:i]]];
    }
    return [chars sortedArrayUsingSelector:@selector(compare:)];
}
```

**复杂度**：排序法 O(n × k log k)，计数法 O(n × k)，其中 n 为字符串数量，k 为最长字符串长度。

### 8.3 最长连续序列（Longest Consecutive Sequence）

**题目**：给定未排序数组，找出最长连续元素序列的长度，要求 O(n) 时间。

**思路**：将所有元素放入 HashSet，对每个可能是序列起点的元素（即 num-1 不在集合中），向后扩展计数。

```swift
func longestConsecutive(_ nums: [Int]) -> Int {
    guard !nums.isEmpty else { return 0 }
    
    let numSet = Set(nums)
    var maxLength = 0
    
    for num in numSet {
        // 只从序列起点开始计数（num-1 不在集合中）
        if !numSet.contains(num - 1) {
            var currentNum = num
            var currentLength = 1
            
            while numSet.contains(currentNum + 1) {
                currentNum += 1
                currentLength += 1
            }
            
            maxLength = max(maxLength, currentLength)
        }
    }
    
    return maxLength
}

// 测试
print(longestConsecutive([100, 4, 200, 1, 3, 2]))  // 4 (序列 [1,2,3,4])
```

```objectivec
- (NSInteger)longestConsecutive:(NSArray<NSNumber *> *)nums {
    if (nums.count == 0) return 0;
    
    NSSet<NSNumber *> *numSet = [NSSet setWithArray:nums];

            while ([numSet containsObject:@(currentNum + 1)]) {
                currentNum++;
                currentLength++;
            }
            
            maxLength = MAX(maxLength, currentLength);
        }
    }
    
    return maxLength;
}
```

**复杂度**：时间 O(n)（每个元素最多被访问两次），空间 O(n)。

### 8.4 LRU 缓存（LRU Cache）

**题目**：设计一个 LRU（Least Recently Used）缓存，支持 O(1) 的 get 和 put。

**思路**：哈希表 + 双向链表。哈希表提供 O(1) 查找，双向链表维护访问顺序。

```swift
class LRUCache {
    
    private class DLinkedNode {
        var key: Int
        var value: Int
        var prev: DLinkedNode?
        var next: DLinkedNode?
        
        init(key: Int = 0, value: Int = 0) {
            self.key = key
            self.value = value
        }
    }
    
    private var cache: [Int: DLinkedNode] = [:]  // key -> node
    private let capacity: Int
    private let head: DLinkedNode  // 哨兵头节点
    private let tail: DLinkedNode  // 哨兵尾节点
    
    init(_ capacity: Int) {
        self.capacity = capacity
        self.head = DLinkedNode()
        self.tail = DLinkedNode()
        head.next = tail
        tail.prev = head
    }
    
    func get(_ key: Int) -> Int {
        guard let node = cache[key] else { return -1 }
        // 移到头部（最近使用）
        moveToHead(node)
        return node.value
    }
    
    func put(_ key: Int, _ value: Int) {
        if let node = cache[key] {
            // 已存在，更新值并移到头部
            node.value = value
            moveToHead(node)
        } else {
            // 新节点
            let newNode = DLinkedNode(key: key, value: value)
            cache[key] = newNode
            addToHead(newNode)
            
            if cache.count > capacity {
                // 淘汰尾部节点（最久未使用）
                let removed = removeTail()
                cache.removeValue(forKey: removed.key)
            }
        }
    }
    
    // MARK: - 双向链表操作
    
    private func addToHead(_ node: DLinkedNode) {
        node.prev = head
        node.next = head.next
        head.next?.prev = node
        head.next = node
    }
    
    private func removeNode(_ node: DLinkedNode) {
        node.prev?.next = node.next
        node.next?.prev = node.prev
    }
    
    private func moveToHead(_ node: DLinkedNode) {
        removeNode(node)
        addToHead(node)
    }
    
    private func removeTail() -> DLinkedNode {
        let node = tail.prev!
        removeNode(node)
        return node
    }
}

// 测试

lru.put(1, 1)
lru.put(2, 2)
print(lru.get(1))   // 1
lru.put(3, 3)       // 淘汰 key=2
print(lru.get(2))   // -1
lru.put(4, 4)       // 淘汰 key=1
print(lru.get(1))   // -1
print(lru.get(3))   // 3
print(lru.get(4))   // 4
```

### 8.5 前 K 个高频元素（Top K Frequent Elements）

**题目**：给定数组，返回出现频率前 k 高的元素。

```swift
func topKFrequent(_ nums: [Int], _ k: Int) -> [Int] {
    // 1. 统计频率
    var freqMap: [Int: Int] = [:]
    for num in nums {
        freqMap[num, default: 0] += 1
    }
    
    // 2. 桶排序：下标为频率，值为该频率的元素列表
    var buckets: [[Int]] = Array(repeating: [], count: nums.count + 1)
    for (num, freq) in freqMap {
        buckets[freq].append(num)
    }
    
    // 3. 从高频到低频收集结果
    var result: [Int] = []
    for i in stride(from: buckets.count - 1, through: 0, by: -1) {
        for num in buckets[i] {
            result.append(num)
            if result.count == k {
                return result
            }
        }
    }
    
    return result
}
```

```objectivec
    // 1. 统计频率
    NSMutableDictionary<NSNumber *, NSNumber *> *freqMap = [NSMutableDictionary dictionary];
    for (NSNumber *num in nums) {
        NSInteger count = freqMap[num].integerValue;
        freqMap[num] = @(count + 1);
    }
    
    // 2. 桶排序
    NSMutableArray<NSMutableArray<NSNumber *> *> *buckets = [NSMutableArray arrayWithCapacity:nums.count + 1];
    for (NSUInteger i = 0; i <= nums.count; i++) {
        [buckets addObject:[NSMutableArray array]];
    }
    
    [freqMap enumerateKeysAndObjectsUsingBlock:^(NSNumber *num, NSNumber *freq, BOOL *stop) {
        [buckets[freq.integerValue] addObject:num];
    }];
    
    // 3. 从高频到低频收集
    NSMutableArray<NSNumber *> *result = [NSMutableArray arrayWithCapacity:k];
    for (NSInteger i = (NSInteger)nums.count; i >= 0 && (NSInteger)result.count < k; i--) {
        for (NSNumber *num in buckets[i]) {
            [result addObject:num];
            if ((NSInteger)result.count == k) break;
        }
    }
    
    return [result copy];
}
```

**复杂度**：时间 O(n)（桶排序法），空间 O(n)。

### 8.6 子数组和等于 K（Subarray Sum Equals K）

**题目**：给定数组和整数 k，找出和为 k 的连续子数组的个数。

**思路**：前缀和 + 哈希表。如果 prefixSum[j] - prefixSum[i] = k，则 nums[i+1..j] 的和为 k。

```swift
func subarraySum(_ nums: [Int], _ k: Int) -> Int {
    // key: 前缀和, value: 该前缀和出现的次数
    var prefixSumCount: [Int: Int] = [0: 1]  // 初始前缀和 0 出现 1 次
    var currentSum = 0
    var count = 0
    
    for num in nums {
        currentSum += num
        // 查找是否存在前缀和为 currentSum - k 的位置
        if let prevCount = prefixSumCount[currentSum - k] {
            count += prevCount
        }
        prefixSumCount[currentSum, default: 0] += 1
    }
    
    return count
}

// 测试
print(subarraySum([1, 1, 1], 2))      // 2
print(subarraySum([1, 2, 3], 3))      // 2 ([1,2] 和 [3])
print(subarraySum([1, -1, 0], 0))     // 3 ([1,-1], [-1,0], [1,-1,0])
```

```objectivec
- (NSInteger)subarraySum:(NSArray<NSNumber *> *)nums k:(NSInteger)k {
    NSMutableDictionary<NSNumber *, NSNumber *> *prefixSumCount = [NSMutableDictionary dictionary];
    prefixSumCount[@0] = @1;  // 初始前缀和 0 出现 1 次
    
    NSInteger currentSum = 0;
    NSInteger count = 0;
    
    for (NSNumber *num in nums) {
        currentSum += num.integerValue;
        
        NSNumber *target = @(currentSum - k);
        NSNumber *prevCount = prefixSumCount[target];
        if (prevCount) {
            count += prevCount.integerValue;
        }
        
        NSInteger existing = prefixSumCount[@(currentSum)].integerValue;
        prefixSumCount[@(currentSum)] = @(existing + 1);
    }
    
    return count;
}
```

**复杂度**：时间 O(n)，空间 O(n)。

### 8.7 无重复字符的最长子串（Longest Substring Without Repeating Characters）

**题目**：给定字符串，找出不含重复字符的最长子串的长度。

**思路**：滑动窗口 + 哈希表记录字符最后出现的位置。

```swift
func lengthOfLongestSubstring(_ s: String) -> Int {
    let chars = Array(s)
    // key: 字符, value: 最后出现的下标
    var lastIndex: [Character: Int] = [:]
    var maxLength = 0
    var left = 0  // 窗口左边界
    
    for (right, char) in chars.enumerated() {
        if let prevIndex = lastIndex[char], prevIndex >= left {
            // 字符重复且在当前窗口内，收缩左边界
            left = prevIndex + 1
        }
        lastIndex[char] = right
        maxLength = max(maxLength, right - left + 1)
    }
    
    return maxLength
}

// 测试
print(lengthOfLongestSubstring("abcabcbb"))  // 3 ("abc")
print(lengthOfLongestSubstring("bbbbb"))     // 1 ("b")
print(lengthOfLongestSubstring("pwwkew"))    // 3 ("wke")
```

```objectivec
- (NSInteger)lengthOfLongestSubstring:(NSString *)s {
    NSMutableDictionary<NSString *, NSNumber *> *lastIndex = [NSMutableDictionary dictionary];
    NSInteger maxLength = 0;
    NSInteger left = 0;
    
    for (NSInteger right = 0; right < (NSInteger)s.length; right++) {
        NSString *ch = [s substringWithRange:NSMakeRange(right, 1)];
        NSNumber *prevIdx = lastIndex[ch];
        
        if (prevIdx && prevIdx.integerValue >= left) {
            left = prevIdx.integerValue + 1;
        }
        
        lastIndex[ch] = @(right);
        maxLength = MAX(maxLength, right - left + 1);
    }
    
    return maxLength;
}
```

**复杂度**：时间 O(n)，空间 O(min(n, |Σ|))，其中 |Σ| 为字符集大小。

### 8.8 有效的数独（Valid Sudoku）

**题目**：判断 9×9 数独是否有效（已填入的数字是否满足规则）。

```swift
func isValidSudoku(_ board: [[Character]]) -> Bool {
    // 用三组哈希集合分别记录行、列、3×3 宫格中出现的数字
    var rows = Array(repeating: Set<Character>(), count: 9)
    var cols = Array(repeating: Set<Character>(), count: 9)
    var boxes = Array(repeating: Set<Character>(), count: 9)
    
    for i in 0..<9 {
        for j in 0..<9 {
            let ch = board[i][j]
            if ch == "." { continue }
            
            let boxIndex = (i / 3) * 3 + (j / 3)
            
            if rows[i].contains(ch) || cols[j].contains(ch) || boxes[boxIndex].contains(ch) {
                return false
            }
            
            rows[i].insert(ch)
            cols[j].insert(ch)
            boxes[boxIndex].insert(ch)
        }
    }
    
    return true
}
```

**复杂度**：时间 O(1)（固定 81 个格子），空间 O(1)。

### 8.9 四数相加 II（4Sum II）

**题目**：给定四个整数数组，计算有多少个元组 (i, j, k, l) 使得 A[i] + B[j] + C[k] + D[l] = 0。

```swift
func fourSumCount(_ nums1: [Int], _ nums2: [Int], _ nums3: [Int], _ nums4: [Int]) -> Int {
    // 将问题分为两组：(A+B) 和 (C+D)
    var sumAB: [Int: Int] = [:]
    
    // 枚举 A+B 的所有可能和
    for a in nums1 {
        for b in nums2 {
            sumAB[a + b, default: 0] += 1
        }
    }
    
    // 枚举 C+D，查找 -(C+D) 在 sumAB 中的计数
    var count = 0
    for c in nums3 {
        for d in nums4 {
            let target = -(c + d)
            if let cnt = sumAB[target] {
                count += cnt
            }
        }
    }
    
    return count
}
```

### 8.10 设计 RandomizedSet（O(1) 插入/删除/随机获取）

**题目**：设计一个数据结构，支持 O(1) 的插入、删除和随机获取元素。

**思路**：数组 + 哈希表。数组存储元素（支持 O(1) 随机访问），哈希表存储元素到下标的映射（支持 O(1) 查找）。删除时将待删除元素与数组末尾交换，然后删除末尾。

```swift
class RandomizedSet {
    private var nums: [Int] = []           // 存储元素
    private var indexMap: [Int: Int] = [:]  // 元素 -> 下标
    
    func insert(_ val: Int) -> Bool {
        if indexMap[val] != nil { return false }
        
        indexMap[val] = nums.count
        nums.append(val)
        return true
    }
    
    func remove(_ val: Int) -> Bool {
        guard let index = indexMap[val] else { return false }
        
        // 将待删除元素与末尾交换
        let lastVal = nums[nums.count - 1]
        nums[index] = lastVal
        indexMap[lastVal] = index
        
        // 删除末尾
        nums.removeLast()
        indexMap.removeValue(forKey: val)
        
        return true
    }
    
    func getRandom() -> Int {
        return nums[Int.random(in: 0..<nums.count)]
    }
}
```

### 9.1 一致性哈希（Consistent Hashing）

一致性哈希用于分布式系统中的数据分片，解决传统取模法在节点增减时大量数据迁移的问题。

**核心思想**：
1. 将哈希值空间组织成一个环（0 ~ 2^32 - 1）
2. 将服务器节点映射到环上的某些位置
3. 数据的 key 也映射到环上，顺时针找到的第一个节点就是其归属节点
4. 增减节点时，只影响相邻节点之间的数据

**虚拟节点**：为了解决节点分布不均匀的问题，每个物理节点映射多个虚拟节点到环上。

```swift
class ConsistentHash {
    private var ring: [(hash: UInt32, node: String)] = []  // 有序环
    private let virtualNodes: Int
    
    init(nodes: [String], virtualNodes: Int = 150) {
        self.virtualNodes = virtualNodes
        for node in nodes {
            addNode(node)
        }
    }
    
    func addNode(_ node: String) {
        for i in 0..<virtualNodes {
            let virtualKey = "\(node)#\(i)"
            let hash = fnv1aHash(virtualKey)
            ring.append((hash: hash, node: node))
        }
        ring.sort { $0.hash < $1.hash }
    }
    
    func removeNode(_ node: String) {
        ring.removeAll { $0.node == node }
    }
    
    func getNode(for key: String) -> String? {
        guard !ring.isEmpty else { return nil }
        
        let hash = fnv1aHash(key)
        
        // 二分查找第一个 >= hash 的位置
        var lo = 0, hi = ring.count
        while lo < hi {
            let mid = (lo + hi) / 2
            if ring[mid].hash < hash {
                lo = mid + 1
            } else {
                hi = mid
            }
        }
        
        // 如果超出末尾，回到环的起点
        let index = lo % ring.count
        return ring[index].node
    }
    
    private func fnv1aHash(_ str: String) -> UInt32 {
        var hash: UInt32 = 2166136261
        for byte in str.utf8 {
            hash ^= UInt32(byte)
            hash &*= 16777619
        }
        return hash
    }
}
```

### 9.2 布隆过滤器（Bloom Filter）

布隆过滤器是一种空间效率极高的概率数据结构，用于判断元素**是否可能存在**于集合中。

**特点**：
- 可能误判（False Positive）：判断"存在"时可能实际不存在
- 不会漏判（No False Negative）：判断"不存在"时一定不存在
- 不支持删除（标准版本）

**原理**：
1. 使用 m 位的位数组，初始全为 0
2. 使用 k 个独立的哈希函数
3. 插入时：对元素计算 k 个哈希值，将对应位设为 1
4. 查询时：计算 k 个哈希值，如果所有对应位都为 1 则"可能存在"，否则"一定不存在"

**误判率公式**：

```
p ≈ (1 - e^(-kn/m))^k
```

其中 n 为已插入元素数，m 为位数组大小，k 为哈希函数个数。

**最优 k 值**：k = (m/n) × ln2 ≈ 0.693 × (m/n)

```swift
struct BloomFilter {
    private var bits: [Bool]
    private let size: Int
    private let hashCount: Int
    
    /// 初始化
    /// - Parameters:
    ///   - expectedCount: 预期元素数量
    ///   - falsePositiveRate: 期望误判率（如 0.01 表示 1%）
    init(expectedCount: Int, falsePositiveRate: Double = 0.01) {
        // 计算最优位数组大小：m = -n * ln(p) / (ln2)^2
        let m = Int(ceil(-Double(expectedCount) * log(falsePositiveRate) / pow(log(2.0), 2)))
        self.size = max(m, 64)
        
        // 计算最优哈希函数个数：k = (m/n) * ln2
        let k = Int(round(Double(self.size) / Double(expectedCount) * log(2.0)))
        self.hashCount = max(k, 1)
        
        self.bits = Array(repeating: false, count: self.size)
    }
    
    /// 插入元素
    mutating func insert(_ element: String) {
        for i in 0..<hashCount {
            let index = hashIndex(element, seed: i)
            bits[index] = true
        }
    }
    
    /// 查询元素是否可能存在
    func mightContain(_ element: String) -> Bool {
        for i in 0..<hashCount {
            let index = hashIndex(element, seed: i)
            if !bits[index] {
                return false  // 一定不存在
            }
        }
        return true  // 可能存在（有误判概率）
    }
    
    /// 使用不同种子生成不同的哈希值
    private func hashIndex(_ element: String, seed: Int) -> Int {
        var hash: UInt64 = UInt64(seed) &* 6364136223846793005 &+ 1442695040888963407
        for byte in element.utf8 {
            hash ^= UInt64(byte)
            hash &*= 1099511628211
        }
        return Int(hash % UInt64(size))
    }
}

// 使用示例
var bloom = BloomFilter(expectedCount: 1000, falsePositiveRate: 0.01)
bloom.insert("hello")
bloom.insert("world")

print(bloom.mightContain("hello"))  // true
print(bloom.mightContain("world"))  // true
print(bloom.mightContain("foo"))    // false（大概率）
```

**应用场景**：
- 网页爬虫 URL 去重
- 垃圾邮件过滤
- 数据库查询优化（避免磁盘 IO）
- CDN 缓存穿透防护
- iOS 中 Safari 的恶意网站检测

### 9.3 计数布隆过滤器（Counting Bloom Filter）

标准布隆过滤器不支持删除，计数布隆过滤器将每个位扩展为计数器（通常 4 位），支持删除操作。

### 9.4 布谷鸟过滤器（Cuckoo Filter）

布谷鸟过滤器基于布谷鸟哈希（Cuckoo Hashing），相比布隆过滤器：
- 支持删除
- 查找性能更好（最多查看 2 个位置）
- 空间效率在低误判率时更优

### 9.5 布谷鸟哈希（Cuckoo Hashing）

使用两个哈希函数和两个表，每个键只能在两个位置之一。插入时如果两个位置都被占用，则"踢走"其中一个元素到它的另一个位置，被踢走的元素再踢走下一个...直到找到空位或达到最大踢出次数（此时需要扩容或更换哈希函数）。

**优点**：查找最坏情况 O(1)（最多查看 2 个位置）。
**缺点**：插入可能触发连锁踢出，最坏情况 O(n)。

### 9.6 跳房子哈希（Hopscotch Hashing）

结合了链地址法和开放寻址法的优点：每个桶有一个固定大小的"邻域"（如 32 个连续槽），元素只能存储在其理想位置的邻域内。查找时只需检查邻域内的槽（通过位图快速定位），保证了缓存友好性和 O(1) 最坏查找。

### 10.2 线程安全的哈希表

#### 10.2.1 读写锁方案

```swift
import Foundation

/// 线程安全的哈希表（读写锁）
class ConcurrentHashTable<Key: Hashable, Value> {
    private var storage: [Key: Value] = [:]
    private let lock = NSLock()  // 简单互斥锁
    // 或使用 pthread_rwlock 实现读写锁以提高并发读性能
    
    func get(_ key: Key) -> Value? {
        lock.lock()
        defer { lock.unlock() }
        return storage[key]
    }
    
    func put(_ key: Key, value: Value) {
        lock.lock()
        defer { lock.unlock() }
        storage[key] = value
    }
    
    func remove(_ key: Key) {
        lock.lock()
        defer { lock.unlock() }
        storage.removeValue(forKey: key)
    }
}
```

#### 10.2.3 分段锁方案（Striped Lock）

```swift
/// 分段锁哈希表（减少锁竞争）
class StripedHashTable<Key: Hashable, Value> {
    private let segmentCount: Int
    private var segments: [Segment]
    
    private class Segment {
        var storage: [Key: Value] = [:]
        let lock = NSLock()
    }
    
    init(segmentCount: Int = 16) {
        self.segmentCount = segmentCount
        self.segments = (0..<segmentCount).map { _ in Segment() }
    }
    
    private func segment(for key: Key) -> Segment {
        var hasher = Hasher()
        hasher.combine(key)
        let hash = abs(hasher.finalize())
        return segments[hash % segmentCount]
    }
    
    func get(_ key: Key) -> Value? {
        let seg = segment(for: key)
        seg.lock.lock()
        defer { seg.lock.unlock() }
        return seg.storage[key]
    }
    
    func put(_ key: Key, value: Value) {
        let seg = segment(for: key)
        seg.lock.lock()
        defer { seg.lock.unlock() }
        seg.storage[key] = value
    }
    
    func remove(_ key: Key) {
        let seg = segment(for: key)
        seg.lock.lock()
        defer { seg.lock.unlock() }
        seg.storage.removeValue(forKey: key)
    }
}
```

## 11. 复杂度总览

### 11.1 各实现方式的复杂度对比

| 操作 | 链地址法（平均） | 链地址法（最坏） | 开放寻址（平均） | 开放寻址（最坏） |
|------|----------------|----------------|----------------|----------------|
| 查找 | O(1) | O(n) | O(1) | O(n) |
| 插入 | O(1) | O(n) | O(1) | O(n) |
| 删除 | O(1) | O(n) | O(1) | O(n) |
| 空间 | O(n + m) | - | O(m) | - |

注：m 为桶数量，n 为元素数量。最坏情况发生在所有元素哈希到同一个桶时。

### 11.2 与其他数据结构的对比

| 数据结构 | 查找 | 插入 | 删除 | 有序遍历 | 空间 |
|---------|------|------|------|---------|------|
| 哈希表 | O(1) avg | O(1) avg | O(1) avg | 不支持 | O(n) |
| 平衡 BST | O(log n) | O(log n) | O(log n) | O(n) | O(n) |
| 有序数组 | O(log n) | O(n) | O(n) | O(n) | O(n) |
| 跳表 | O(log n) | O(log n) | O(log n) | O(n) | O(n) |
| B 树 | O(log n) | O(log n) | O(log n) | O(n) | O(n) |

### 11.3 扩容的均摊复杂度

使用翻倍扩容策略时，n 次插入的总代价为 O(n)，均摊每次插入 O(1)。

证明（聚合分析）：
- 第 i 次扩容的代价为 2^i（复制 2^i 个元素）
- n 次插入中，扩容次数为 ⌊log₂n⌋
- 总扩容代价 = 1 + 2 + 4 + ... + 2^⌊log₂n⌋ < 2n
- 总代价 = n（普通插入）+ 2n（扩容）= 3n = O(n)
- 均摊代价 = O(n) / n = O(1)

## 12. 面试题

### 面试题 1：HashMap 和 HashTable 的区别（Java 视角，但原理通用）

**答案要点**：
- HashMap 允许 null key 和 null value；Hashtable 不允许
- HashMap 非线程安全；Hashtable 所有方法 synchronized（线程安全但性能差）
- HashMap 初始容量 16，扩容为 2 倍；Hashtable 初始容量 11，扩容为 2n+1
- HashMap 链表长度 > 8 时转红黑树（Java 8+）
- 现代推荐：ConcurrentHashMap（分段锁/CAS）替代 Hashtable

### 面试题 2：如何设计一个好的 hashCode？

**答案要点**：
- 一致性：equals 相等的对象必须有相同的 hashCode
- 均匀分布：不同对象的 hashCode 尽量分散
- 高效计算：避免复杂运算
- 常用技巧：选择质数（如 31）作为乘数，逐字段组合
- Swift 中使用 Hasher.combine() 自动处理
- 避免只用部分字段（可能导致大量冲突）

### 面试题 3：哈希冲突的解决方法有哪些？各自优缺点？

**答案要点**：
- 链地址法：实现简单、删除方便、负载因子可 > 1；缺点是缓存不友好、额外指针开销
- 开放寻址-线性探测：缓存友好；缺点是一次聚集
- 开放寻址-二次探测：减轻聚集；缺点是二次聚集、不保证遍历所有桶
- 开放寻址-双重哈希：几乎消除聚集；缺点是需要两个哈希函数
- Robin Hood 哈希：均匀化探测距离，减少最坏情况
- 布谷鸟哈希：查找 O(1) 最坏；缺点是插入可能连锁踢出

### 面试题 4：为什么 HashMap 的容量要求是 2 的幂？

**答案要点**：
- 取模优化：`hash % capacity` 等价于 `hash & (capacity - 1)`，位运算比取模快
- 扩容时的优化：元素要么在原位置，要么在原位置 + 旧容量的位置（只需看新增的一位是 0 还是 1）
- 缺点：如果哈希函数质量差，低位相同的键会聚集。Java 通过扰动函数（高位异或低位）缓解

### 面试题 5：什么是哈希洪泛攻击？如何防御？

**答案要点**：
- 攻击原理：攻击者构造大量哈希值相同的键，使哈希表退化为链表 O(n)，导致 DoS
- 防御措施：
  - 随机化哈希种子（Swift 的 Hasher 每次启动使用随机种子）
  - 使用密码学安全的哈希函数（如 SipHash）
  - 链表过长时转红黑树（Java 8 的 HashMap）
  - 限制单个桶的最大长度

### 面试题 6：NSDictionary 的 key 为什么要实现 NSCopying？

**答案要点**：
- NSDictionary 会 copy key 以防止外部修改 key 导致哈希值变化
- 如果 key 被修改，其哈希值改变，就无法在原来的桶中找到它
- 这是"不可变键"原则的体现
- Swift 的 Dictionary 通过值语义（struct）天然保证 key 不会被外部修改

### 面试题 7：如何实现一个 O(1) 的 LRU 缓存？

**答案要点**：
- 哈希表 + 双向链表
- 哈希表：key → 链表节点，O(1) 查找
- 双向链表：维护访问顺序，头部最近使用，尾部最久未使用
- get：查哈希表找到节点，移到链表头部
- put：新节点插入头部；超容量时删除尾部节点并从哈希表移除
- 双向链表使用哨兵节点简化边界处理

## 13. 易错点与最佳实践

### 13.1 常见错误

#### 错误 1：可变对象作为 key

```swift
// 错误！可变对象的哈希值可能改变
class MutableKey: Hashable {
    var name: String  // 可变属性
    
    func hash(into hasher: inout Hasher) {
        hasher.combine(name)
    }
    
    static func == (lhs: MutableKey, rhs: MutableKey) -> Bool {
        return lhs.name == rhs.name
    }
}

let key = MutableKey()
key.name = "hello"
var dict: [MutableKey: Int] = [key: 42]

key.name = "world"  // 修改了 key！
print(dict[key])    // nil！因为哈希值变了，找不到原来的桶

// 正确做法：使用不可变类型（struct 或 let 属性）作为 key
```

#### 错误 2：hash 和 isEqual 不一致

```objectivec
// 错误！isEqual 返回 YES 但 hash 不同
- (NSUInteger)hash {
    return arc4random();  // 每次返回不同值！
}

- (BOOL)isEqual:(id)object {
    // 即使 isEqual 正确，hash 不一致也会导致字典行为异常
    return [self.name isEqualToString:((MyObj *)object).name];
}

// 规则：isEqual == YES → hash 必须相同
// 反之不必：hash 相同不要求 isEqual == YES（冲突是允许的）
```

#### 错误 3：在遍历时修改哈希表

```swift
// 错误！遍历时修改会导致未定义行为或崩溃
var dict = ["a": 1, "b": 2, "c": 3]
for (key, value) in dict {
    if value < 2 {
        dict.removeValue(forKey: key)  // 运行时错误！
    }
}

// 正确做法 1：收集要删除的 key，遍历后统一删除
let keysToRemove = dict.filter { $0.value < 2 }.map { $0.key }
for key in keysToRemove {
    dict.removeValue(forKey: key)
}

// 正确做法 2：使用 filter 创建新字典
dict = dict.filter { $0.value >= 2 }
```

```objectivec
// Objective-C 中遍历时修改
NSMutableDictionary *dict = [@{@"a": @1, @"b": @2, @"c": @3} mutableCopy];

// 错误！

    if ([dict[key] integerValue] < 2) {

[dict enumerateKeysAndObjectsUsingBlock:^(id key, id obj, BOOL *stop) {

        [keysToRemove addObject:key];
    }
}];
[dict removeObjectsForKeys:keysToRemove];
```

#### 错误 4：忽略哈希表的无序性

```swift
// 哈希表的遍历顺序是不确定的！
let dict = ["b": 2, "a": 1, "c": 3]
for (key, _) in dict {
    print(key)  // 顺序不确定，不一定是 "b", "a", "c"
}

// 如果需要有序遍历，应该排序后遍历
for key in dict.keys.sorted() {
    print(key, dict[key]!)  // a:1, b:2, c:3
```

### 13.2 最佳实践

1. **选择合适的初始容量**：如果能预估元素数量，设置合适的初始容量避免多次扩容
2. **使用值类型作为 key**：Swift 中优先使用 struct（值语义天然不可变）
3. **实现高质量的 hash 函数**：使用 Hasher.combine() 组合所有参与 isEqual 判断的字段
4. **注意线程安全**：多线程环境下使用锁或并发容器
5. **避免过度使用哈希表**：小规模数据（< 50）用数组可能更快
6. **利用 default 值简化代码**：`dict[key, default: 0] += 1`
7. **考虑内存开销**：哈希表的内存开销比数组大（指针、空桶、负载因子限制）

## 14. 对比辨析

### 14.1 链地址法 vs 开放寻址法

| 维度 | 链地址法 | 开放寻址法 |
|------|---------|-----------|
| 空间利用 | 额外链表指针开销 | 无额外指针，但需要空桶 |
| 缓存性能 | 差（链表节点分散） | 好（连续内存） |
| 负载因子 | 可以 > 1 | 必须 < 1（建议 < 0.7） |
| 删除操作 | 简单（链表删除） | 复杂（需要墓碑或后移） |
| 聚集问题 | 无 | 有（线性/二次聚集） |
| 实现复杂度 | 简单 | 中等 |
| 适用场景 | 通用、删除频繁 | 内存敏感、缓存友好 |
| 典型实现 | Java HashMap | Python dict, Swift Dictionary |

### 14.2 Dictionary vs Set

| 维度 | Dictionary | Set |
|------|-----------|-----|
| 存储内容 | 键值对 (Key-Value) | 仅键 (Key) |
| 用途 | 映射关系 | 去重、集合运算 |
| 底层实现 | 相同（哈希表） | 相同（哈希表，value 为空） |
| 集合运算 | 不直接支持 | 支持（交/并/差/对称差） |

### 14.3 哈希表 vs 平衡 BST（如红黑树）

| 维度 | 哈希表 | 平衡 BST |
|------|-------|----------|
| 查找 | O(1) 平均 | O(log n) |
| 有序遍历 | 不支持 | 支持（中序遍历） |
| 范围查询 | 不支持 | 支持 |
| 最坏情况 | O(n)（退化） | O(log n)（保证） |
| 空间 | 有空桶浪费 | 每节点 2-3 个指针 |
| 适用场景 | 精确查找 | 需要有序性/范围查询 |

### 14.4 NSDictionary vs NSMapTable vs NSCache

| 维度 | NSDictionary | NSMapTable | NSCache |
|------|-------------|-----------|---------|
| Key 引用 | 强引用（copy） | 可配置（强/弱） | 强引用（不 copy） |
| Value 引用 | 强引用 | 可配置（强/弱） | 强引用 |
| 线程安全 | 不可变版本安全 | 否 | 是 |
| 自动淘汰 | 否 | 弱引用自动清理 | 是（内存压力） |
| Key 要求 | NSCopying | 无 | 无 |
| 适用场景 | 通用键值存储 | 弱引用关联 | 缓存 |

## 15. 进阶话题

### 15.1 完美哈希（Perfect Hashing）

如果键的集合是静态的（已知且不变），可以构造**完美哈希函数**：没有任何冲突，查找保证 O(1) 最坏情况。

**最小完美哈希**（Minimal Perfect Hash, MPH）：n 个键恰好映射到 0..n-1，无空桶。

工具：gperf（GNU Perfect Hash Function Generator）、CMPH 库。

应用：编译器关键字查找、只读数据库索引。

### 15.2 可扩展哈希（Extendible Hashing）

用于数据库和文件系统，支持动态增长而无需全量 rehash：
- 使用目录（directory）指向数据桶
- 桶满时只分裂该桶，目录按需翻倍
- 避免了传统哈希表扩容时的全量数据迁移

### 15.3 线性哈希（Linear Hashing）

另一种动态哈希方案：
- 按顺序逐个分裂桶（不是全量 rehash）
- 使用两个哈希函数交替
- 适用于磁盘存储场景

### 15.4 Swiss Table（abseil-cpp）

Google 开源的高性能哈希表实现（用于 C++ 的 absl::flat_hash_map）：
- 使用 SIMD 指令并行探测多个槽
- 每个槽有 1 字节的元数据（control byte）：空/已删除/哈希值高 7 位
- 探测时先用 SIMD 比较 16 个 control byte，快速过滤
- 比 std::unordered_map 快 2-3 倍

### 15.5 哈希表在数据库中的应用

1. **Hash Join**：将小表建立哈希表，扫描大表时在哈希表中查找匹配行
2. **Hash Index**：MySQL MEMORY 引擎支持哈希索引，精确查找 O(1)
3. **Hash Partition**：按哈希值分区，均匀分布数据
4. **Group By / Distinct**：使用哈希表进行分组和去重

### 15.6 分布式哈希表（DHT）

分布式系统中的哈希表，数据分布在多个节点上：
- **Chord**：环形拓扑，O(log n) 跳数查找
- **Kademlia**：基于 XOR 距离，用于 BitTorrent DHT
- **Amazon Dynamo**：一致性哈希 + 虚拟节点 + 向量时钟

### 15.7 哈希表的持久化

将哈希表持久化到磁盘的挑战：
- 扩容时的原子性（崩溃恢复）
- 磁盘 IO 优化（减少随机读写）
- 解决方案：LMDB（内存映射 B+ 树）、RocksDB（LSM-Tree）、LevelDB

### 15.8 Lock-Free 哈希表

无锁哈希表使用 CAS（Compare-And-Swap）原子操作实现并发安全：

```swift
// 概念示意（实际实现非常复杂）
// 使用 CAS 实现无锁插入
func lockFreePut(key: Key, value: Value) {
    let index = hash(key)
    while true {
        let current = buckets[index].load()  // 原子读
        let newNode = Node(key: key, value: value, next: current)
        if buckets[index].compareAndSwap(expected: current, desired: newNode) {
            return  // CAS 成功
        }
        // CAS 失败，重试
    }
}
```

著名实现：Java 的 ConcurrentHashMap（Java 8 使用 CAS + synchronized）、Cliff Click 的 NonBlockingHashMap。

## 总结

哈希表是计算机科学中最基础、最重要的数据结构之一。掌握哈希表需要理解：

1. **哈希函数**：如何将键均匀映射到有限空间
2. **冲突解决**：链地址法与开放寻址法的原理与权衡
3. **动态扩容**：负载因子控制与均摊分析
4. **工程实践**：线程安全、性能优化、系统级应用
5. **高级变体**：一致性哈希、布隆过滤器、布谷鸟哈希等

在 iOS 开发中，哈希表无处不在：从 NSDictionary/NSSet 到 Runtime 方法缓存，从 URL 路由到网络请求去重。深入理解哈希表的原理，能帮助我们写出更高效、更健壮的代码。
