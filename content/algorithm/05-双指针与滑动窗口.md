# 双指针与滑动窗口（Two Pointers & Sliding Window）详解

> **双指针**（Two Pointers）是一种使用两个（或多个）索引/引用在数据结构（数组、字符串、链表）上协同移动来求解问题的算法技巧；**滑动窗口**（Sliding Window）是双指针的一种重要特例，通过维护一个可伸缩的连续区间 `[left, right]` 来高效处理「连续子数组 / 子串」类问题。二者的共同精髓是：**用指针移动代替嵌套循环，把 O(n²) 的暴力解法优化到 O(n)**。

## 1. 基本概念与分类

### 1.1 什么是双指针

**双指针**（Two Pointers）并不是某一种具体的数据结构，而是一类**算法技巧 / 思想范式**。它的核心是：在遍历数据时，维护**两个索引变量**（在链表中是两个节点引用），通过控制这两个指针的移动方向、移动速度和移动条件，把原本需要嵌套循环才能解决的问题，压缩到一次或常数次线性扫描内完成。

```
数组：  [ a0  a1  a2  a3  a4  a5  a6  a7 ]
          ↑                            ↑
        left                        right     ← 对撞型：从两端向中间靠拢

数组：  [ a0  a1  a2  a3  a4  a5  a6  a7 ]
          ↑   ↑
        slow fast                          ← 快慢型：同向，速度不同

数组：  [ a0  a1 | a2  a3  a4 | a5  a6  a7 ]
                 ↑           ↑
               left        right            ← 滑动窗口：维护一段连续区间
```

### 1.2 为什么需要双指针

考虑一个最经典的问题：**在有序数组中找两个数，使它们的和等于目标值 target**。

- **暴力解法**：两层 for 循环枚举所有数对，时间复杂度 O(n²)。
- **哈希表解法**：用空间换时间，O(n) 时间 + O(n) 空间。
- **双指针解法**：利用数组**有序**这一性质，左右指针对撞，O(n) 时间 + **O(1) 空间**。

双指针的价值在于：**在满足特定前提（有序、单调性、连续性）时，用极低的空间开销换取线性时间复杂度**。它是面试和工程中处理数组/字符串/链表问题的「第一性工具」。

### 1.3 双指针的三大分类

| 分类 | 英文 | 指针移动特征 | 典型适用场景 |
|------|------|------------|------------|
| 对撞双指针 | Opposite Direction / Collision | 一个从头、一个从尾，相向而行直至相遇 | 有序数组求和、回文判断、盛水容器、三数之和 |
| 快慢双指针 | Fast & Slow | 同方向，速度（步长）不同 | 链表找中点、判断环、删除重复项、原地移除元素 |
| 滑动窗口 | Sliding Window | 同方向，右指针扩张、左指针收缩，维护连续区间 | 最长/最短连续子数组/子串、定长窗口统计 |

> **关键认知**：滑动窗口本质上是「**同向双指针**」的一个特例，只是它额外维护了一个「窗口状态」（如窗口内字符计数、窗口和）。因此本篇把滑动窗口作为双指针的重点子类单独深入展开。

### 1.4 使用双指针的前提条件

双指针不是万能的，它的正确性依赖于问题本身具备某种**单调性 / 有序性 / 连续性**：

1. **对撞双指针**：通常要求数组**有序**，或问题本身从两端推进不会遗漏解（如回文、盛水容器）。其正确性依赖「移动某个指针后，被排除的解一定不是最优/不可能成立」。
2. **快慢双指针**：依赖两个指针的**相对速度差**会产生确定性的相遇/落点关系（如环检测、找中点）。
3. **滑动窗口**：要求当窗口扩大时某个性质**单调变化**（如窗口和单调增、窗口内不同字符数单调增），这样收缩窗口才有意义。**若不具备单调性（如数组含负数求「和为定值的最长子数组」），滑动窗口会失效，需改用前缀和 + 哈希表**。


### 1.5 图解：三类指针的运动轨迹

为建立直觉，下面用 ASCII 图解三类双指针在同一数组上的典型运动过程。

**对撞双指针**（以两数之和 target=9，数组 [2,3,5,8,11] 为例）：

```
step1: [2, 3, 5, 8, 11]   2+11=13>9  -> right--
        L           R
step2: [2, 3, 5, 8, 11]   2+8=10>9   -> right--
        L        R
step3: [2, 3, 5, 8, 11]   2+5=7<9    -> left++
        L     R
step4: [2, 3, 5, 8, 11]   3+5=8<9    -> left++
           L  R
step5: [2, 3, 5, 8, 11]   5+5? L==R 停止；本例无解（演示移动逻辑）
              LR
```

**快慢双指针**（链表找中点 1→2→3→4→5）：

```
init:  [1]→2→3→4→5    slow=1, fast=1
        S
        F
step1: 1→[2]→3→4→5    slow=2, fast=3
           S
              F
step2: 1→2→[3]→4→5    slow=3, fast=5
              S
                    F
end:   fast.next==nil 停止，slow=3 即中点
```

**滑动窗口**（最长无重复子串 "abcab"）：

```
[a]bcab        window={a}      len=1
 LR
[ab]cab        window={a,b}    len=2
 L R
[abc]ab        window={a,b,c}  len=3  ← 当前最长
 L  R
a[bca]b        遇到第二个a,收缩L  window={b,c,a} len=3

```

### 1.6 决策树：遇到题目如何选择指针策略

```
问题涉及"连续子数组/子串"的最优化/计数？
├─ 是 → 元素全正 或 约束随窗口单调？
│        ├─ 是 → 滑动窗口（定长用固定窗口，变长用可变窗口）
│        └─ 否（含负数等）→ 前缀和 + 哈希表
└─ 否 → 涉及链表的环/中点/倒数第k个？

```

## 2. 为什么双指针能优化复杂度

### 2.1 从暴力到双指针的本质

以「有序数组两数之和」为例，理解双指针为什么正确且高效。

设有序数组 `nums`（升序），要找 `i < j` 使 `nums[i] + nums[j] == target`。

暴力法枚举所有 `(i, j)` 共 C(n,2) ≈ n²/2 个数对。双指针法令 `left = 0, right = n-1`：

- 若 `nums[left] + nums[right] == target`：找到答案。
- 若 `nums[left] + nums[right] < target`：当前和偏小。由于数组升序，**固定 right，任何更小的 left' < left 都会让和更小**，故 `left` 与任何 `right' ≤ right` 配对都不可能等于 target（都偏小）。因此可以安全地 `left += 1`，排除了一整列候选，绝不会漏解。
- 若 `nums[left] + nums[right] > target`：对称地，`right -= 1`。

**关键不变式**：每一步移动都**排除掉一行或一列**的候选解，且被排除的解一定不是答案。指针总移动次数 ≤ n，故时间复杂度 O(n)。

### 2.2 摊还分析（Amortized Analysis）

滑动窗口的 O(n) 复杂度尤其需要摊还分析来理解。考虑可变滑动窗口模板：右指针 `right` 从 0 走到 n-1（共 n 次），左指针 `left` 也只会从 0 单调走到 n-1（最多 n 次）。

虽然代码里有「外层 right 循环 + 内层 left 收缩 while 循环」的嵌套结构，**看起来像 O(n²)**，但 `left` 在**整个算法生命周期内总共**只前进 n 步（它从不回退）。因此内层 while 的总执行次数（跨所有外层迭代累加）≤ n。

总操作数 = right 移动 n 次 + left 移动 ≤ n 次 = **O(n)**。这就是摊还分析：单次外层迭代内层可能跑多次，但均摊到每次外层迭代是 O(1)。

### 2.3 空间复杂度优势

双指针通常只用常数个额外变量（指针 + 少量计数器），空间复杂度 **O(1)**（滑动窗口若需记录窗口内字符计数则为 O(k)，k 为字符集大小，通常视为常数）。相比哈希表解法的 O(n) 空间，双指针在空间敏感场景（如嵌入式、大数据流）优势明显。

## 3. 对撞双指针（左右指针）

### 3.1 核心思想

对撞双指针（Opposite-Direction Two Pointers）令 `left` 从数组头部、`right` 从数组尾部出发，根据当前状态决定移动哪个指针，二者相向而行直到 `left >= right`。

**通用模板**：

```
left = 0, right = n - 1
while left < right:
    根据 nums[left] 与 nums[right] 的关系做判断
    if 满足条件: 记录答案 / 同时收缩
    elif 偏小: left += 1
    else: right -= 1
```

### 3.2 例题：两数之和 II（有序数组）

> 给定升序数组 `numbers` 和目标 `target`，返回两个数的下标（1-based），使其和为 target。保证有唯一解。

**Swift 实现**：

```swift
func twoSum(_ numbers: [Int], _ target: Int) -> [Int] {
    var left = 0
    var right = numbers.count - 1
    while left < right {
        let sum = numbers[left] + numbers[right]
        if sum == target {
            return [left + 1, right + 1]   // 题目要求 1-based 下标
        } else if sum < target {
            left += 1                       // 和偏小，左指针右移增大和
        } else {
            right -= 1                      // 和偏大，右指针左移减小和
        }
    }
    return []                               // 无解（题目保证有解，兜底）
}

// 边界测试
// twoSum([2,7,11,15], 9)  -> [1,2]
// twoSum([2,3,4], 6)      -> [1,3]
// twoSum([-1,0], -1)      -> [1,2]  含负数也成立（只要有序）
```

**Objective-C 实现**：

```objc
- (NSArray<NSNumber *> *)twoSum:(NSArray<NSNumber *> *)numbers target:(NSInteger)target {
    NSInteger left = 0;
    NSInteger right = (NSInteger)numbers.count - 1;
    while (left < right) {
        NSInteger sum = numbers[left].integerValue + numbers[right].integerValue;
        if (sum == target) {
            return @[@(left + 1), @(right + 1)];
        } else if (sum < target) {
            left += 1;
        } else {
            right -= 1;
        }
    }
    return @[];
}

// 行为与 Swift 完全一致：相同输入返回相同 1-based 下标
```

**复杂度**：时间 O(n)（每个元素最多被访问一次），空间 O(1)。

### 3.3 例题：验证回文串

> 判断字符串是否为回文（只考虑字母和数字，忽略大小写）。

**Swift 实现**：

```swift
func isPalindrome(_ s: String) -> Bool {
    // 转为字符数组便于按索引访问（Swift String 不支持 O(1) 随机访问）
    let chars = Array(s.lowercased())
    var left = 0
    var right = chars.count - 1
    while left < right {
        // 跳过非字母数字字符
        while left < right && !chars[left].isLetter && !chars[left].isNumber {
            left += 1
        }
        while left < right && !chars[right].isLetter && !chars[right].isNumber {
            right -= 1
        }
        if chars[left] != chars[right] {
            return false
        }
        left += 1
        right -= 1
    }
    return true
}

// "A man, a plan, a canal: Panama" -> true
// "race a car"                     -> false
// ""                               -> true（空串视为回文）
// ".,"                             -> true（无有效字符）
```

**Objective-C 实现**：

```objc
- (BOOL)isPalindrome:(NSString *)s {
    NSString *lower = [s lowercaseString];
    NSCharacterSet *alnum = [NSCharacterSet alphanumericCharacterSet];
    // 预处理为 unichar 数组
    NSUInteger len = lower.length;
    unichar *buf = (unichar *)malloc(sizeof(unichar) * len);
    [lower getCharacters:buf range:NSMakeRange(0, len)];

    NSInteger left = 0;
    NSInteger right = (NSInteger)len - 1;
    BOOL result = YES;
    while (left < right) {
        while (left < right && ![alnum characterIsMember:buf[left]]) {
            left += 1;
        }
        while (left < right && ![alnum characterIsMember:buf[right]]) {
            right -= 1;
        }
        if (buf[left] != buf[right]) {
            result = NO;
            break;
        }
        left += 1;
        right -= 1;
    }
    free(buf);   // 别忘了释放手动分配的内存
    return result;
}
```

> **易错点**：内层「跳过非法字符」的 while 循环里必须再次判断 `left < right`，否则当字符串全是标点时指针会越界。Swift 与 OC 实现均已正确处理。

**复杂度**：时间 O(n)，空间 O(n)（为支持随机访问转成了字符数组；若用 OC 的 `characterAtIndex:` 直接访问可降到 O(1) 额外空间，但 Swift String 因 Unicode 设计不支持 O(1) 下标）。

### 3.4 例题：盛最多水的容器

> 给定 n 个非负整数 `height[i]`，每个表示坐标 i 处垂线的高度。找出两条线，使它们与 x 轴构成的容器能盛最多的水。容量 = `min(height[i], height[j]) * (j - i)`。

**贪心 + 对撞双指针**：左右指针从两端开始，每次移动**较矮的那条线**。

**为什么移动矮的一边？** 容量受限于较矮的边。若移动较高的边，宽度必然减小，而高度上限仍受那条矮边限制，容量不可能增大；只有移动矮边，才有机会遇到更高的线从而增大容量。这保证了不会错过最优解。

**Swift 实现**：

```swift
func maxArea(_ height: [Int]) -> Int {
    var left = 0
    var right = height.count - 1
    var maxWater = 0
    while left < right {
        let h = min(height[left], height[right])
        let area = h * (right - left)
        maxWater = max(maxWater, area)
        // 移动较矮的一边
        if height[left] < height[right] {
            left += 1
        } else {
            right -= 1
        }
    }
    return maxWater
}

// maxArea([1,8,6,2,5,4,8,3,7]) -> 49
// maxArea([1,1])               -> 1
```

**Objective-C 实现**：

```objc
- (NSInteger)maxArea:(NSArray<NSNumber *> *)height {
    NSInteger left = 0;
    NSInteger right = (NSInteger)height.count - 1;
    NSInteger maxWater = 0;
    while (left < right) {
        NSInteger hl = height[left].integerValue;
        NSInteger hr = height[right].integerValue;
        NSInteger h = MIN(hl, hr);
        NSInteger area = h * (right - left);
        maxWater = MAX(maxWater, area);
        if (hl < hr) {
            left += 1;
        } else {
            right -= 1;
        }
    }
    return maxWater;
}
```

**复杂度**：时间 O(n)，空间 O(1)。

### 3.5 例题：三数之和（双指针经典进阶）

> 给定数组 `nums`，找出所有和为 0 且不重复的三元组。

**思路**：先排序，固定一个数 `nums[i]`，在 `[i+1, n-1]` 区间用对撞双指针找两数之和为 `-nums[i]`。难点在**去重**。

**Swift 实现**：

```swift
func threeSum(_ nums: [Int]) -> [[Int]] {
    let sorted = nums.sorted()
    let n = sorted.count
    var result: [[Int]] = []
    var i = 0
    while i < n - 2 {
        // 剪枝：最小的数已大于 0，后面不可能和为 0
        if sorted[i] > 0 { break }
        // 跳过重复的固定数
        if i > 0 && sorted[i] == sorted[i - 1] {
            i += 1
            continue
        }
        var left = i + 1
        var right = n - 1
        let target = -sorted[i]
        while left < right {
            let sum = sorted[left] + sorted[right]
            if sum == target {
                result.append([sorted[i], sorted[left], sorted[right]])
                // 跳过重复的 left 和 right
                while left < right && sorted[left] == sorted[left + 1] { left += 1 }
                while left < right && sorted[right] == sorted[right - 1] { right -= 1 }
                left += 1
                right -= 1
            } else if sum < target {
                left += 1
            } else {
                right -= 1
            }
        }
        i += 1
    }
    return result
}

// threeSum([-1,0,1,2,-1,-4]) -> [[-1,-1,2],[-1,0,1]]
// threeSum([0,0,0])          -> [[0,0,0]]
// threeSum([1,2,3])          -> []
```

**Objective-C 实现**：

```objc
- (NSArray<NSArray<NSNumber *> *> *)threeSum:(NSArray<NSNumber *> *)nums {
    NSArray<NSNumber *> *sorted = [nums sortedArrayUsingSelector:@selector(compare:)];
    NSInteger n = (NSInteger)sorted.count;
    NSMutableArray<NSArray<NSNumber *> *> *result = [NSMutableArray array];
    NSInteger i = 0;
    while (i < n - 2) {
        if (sorted[i].integerValue > 0) break;
        if (i > 0 && sorted[i].integerValue == sorted[i - 1].integerValue) {
            i += 1;
            continue;
        }
        NSInteger left = i + 1;
        NSInteger right = n - 1;
        NSInteger target = -sorted[i].integerValue;
        while (left < right) {
            NSInteger sum = sorted[left].integerValue + sorted[right].integerValue;
            if (sum == target) {
                [result addObject:@[sorted[i], sorted[left], sorted[right]]];
                while (left < right && sorted[left].integerValue == sorted[left + 1].integerValue) left += 1;
                while (left < right && sorted[right].integerValue == sorted[right - 1].integerValue) right -= 1;
                left += 1;
                right -= 1;
            } else if (sum < target) {
                left += 1;
            } else {
                right -= 1;
            }
        }
        i += 1;
    }
    return result;
}
```

**复杂度**：时间 O(n²)（外层固定 O(n)，内层双指针 O(n)），空间 O(log n)~O(n)（排序所需）。

## 4. 快慢双指针

### 4.1 核心思想

快慢双指针（Fast & Slow Pointers）让两个指针**同方向**移动，但**步长（速度）不同**。最经典的形式是「快指针每次走 2 步，慢指针每次走 1 步」。它在链表问题中威力巨大，也用于数组的原地操作。

两大核心应用：

1. **链表找中点 / 找环**：利用速度差，快指针到尾时慢指针恰好在中点；若有环则快慢必相遇（Floyd 判圈算法）。
2. **数组原地去重 / 移除**：慢指针标记「下一个有效元素应放的位置」，快指针扫描所有元素。

### 4.2 例题：链表找中点

> 返回单链表的中间节点。若有两个中点，返回第二个。

**Swift 实现**：

```swift
class ListNode {
    var val: Int
    var next: ListNode?
    init(_ val: Int) { self.val = val; self.next = nil }
}

func middleNode(_ head: ListNode?) -> ListNode? {
    var slow = head
    var fast = head
    // fast 每次走两步，slow 走一步
    while fast != nil && fast?.next != nil {
        slow = slow?.next
        fast = fast?.next?.next
    }
    return slow   // fast 到末尾时，slow 在中点
}

// 1->2->3->4->5      中点为 3
// 1->2->3->4->5->6   中点为 4（两个中点取第二个）
```

**Objective-C 实现**：

```objc
@interface ListNode : NSObject
@property (nonatomic, assign) NSInteger val;
@property (nonatomic, strong, nullable) ListNode *next;
@end
@implementation ListNode @end

- (ListNode *)middleNode:(ListNode *)head {
    ListNode *slow = head;
    ListNode *fast = head;
    while (fast != nil && fast.next != nil) {
        slow = slow.next;
        fast = fast.next.next;
    }
    return slow;
}
```

> **找第一个中点的变体**：若要在偶数长度时返回前一个中点，把循环条件改为 `while fast?.next != nil && fast?.next?.next != nil`（即初始让 fast 先看一步），或令 fast 从 `head.next` 起步。

**复杂度**：时间 O(n)，空间 O(1)。

### 4.3 例题：判断链表是否有环（Floyd 判圈）

> 给定链表，判断其中是否存在环。

**原理**：若有环，快指针会先进入环并不断绕圈，慢指针随后进入环。由于每轮迭代二者距离缩短 1，快指针一定会「追上」慢指针（相遇）。若无环，快指针会先到达 `nil`。

**Swift 实现**：

```swift
func hasCycle(_ head: ListNode?) -> Bool {
    var slow = head
    var fast = head
    while fast != nil && fast?.next != nil {
        slow = slow?.next
        fast = fast?.next?.next
        if slow === fast {     // 引用相等，说明相遇 -> 有环
            return true
        }
    }
    return false               // fast 到达 nil -> 无环
}
```

**Objective-C 实现**：

```objc
- (BOOL)hasCycle:(ListNode *)head {
    ListNode *slow = head;
    ListNode *fast = head;
    while (fast != nil && fast.next != nil) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) {    // OC 中对象指针 == 比较地址，即引用相等
            return YES;
        }
    }
    return NO;
}
```

**复杂度**：时间 O(n)，空间 O(1)。

### 4.4 例题：寻找环的入口节点（Floyd 判圈第二阶段）

> 若链表有环，返回环的起始节点；否则返回 nil。

**数学证明**：设头到环入口距离为 `a`，环入口到首次相遇点距离为 `b`，相遇点回到环入口距离为 `c`（环长 = b + c）。

相遇时：慢指针走了 `a + b`，快指针走了 `a + b + k(b + c)`（绕了 k 圈）。由快指针速度是慢指针 2 倍：

```
2(a + b) = a + b + k(b + c)
=> a + b = k(b + c)
=> a = k(b + c) - b = (k-1)(b + c) + c
```

即 `a = c + (k-1)·环长`。这意味着：**从头节点和从相遇点各放一个指针，每次走一步，它们必在环入口相遇**（头指针走 a 步到入口，相遇点指针走 c + 整数圈也到入口）。

**Swift 实现**：

```swift
func detectCycle(_ head: ListNode?) -> ListNode? {
    var slow = head
    var fast = head
    while fast != nil && fast?.next != nil {
        slow = slow?.next
        fast = fast?.next?.next
        if slow === fast {
            // 第二阶段：一个指针回到头，同速前进
            var ptr = head
            while ptr !== slow {
                ptr = ptr?.next
                slow = slow?.next
            }
            return ptr      // 相遇点即环入口
        }
    }
    return nil
}
```

**Objective-C 实现**：

```objc
- (ListNode *)detectCycle:(ListNode *)head {
    ListNode *slow = head;
    ListNode *fast = head;
    while (fast != nil && fast.next != nil) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) {
            ListNode *ptr = head;
            while (ptr != slow) {
                ptr = ptr.next;
                slow = slow.next;
            }
            return ptr;
        }
    }
    return nil;
}
```

**复杂度**：时间 O(n)，空间 O(1)。

### 4.5 例题：删除有序数组中的重复项（原地）

> 原地删除升序数组的重复元素，每个元素只保留一个，返回新长度。不能用额外数组。

**思路**：`slow` 指向「下一个唯一元素应写入的位置」，`fast` 扫描全部。当 `nums[fast] != nums[slow]` 时，说明遇到新值，写入 `slow+1`。

**Swift 实现**：

```swift
func removeDuplicates(_ nums: inout [Int]) -> Int {
    if nums.isEmpty { return 0 }
    var slow = 0
    for fast in 1..<nums.count {
        if nums[fast] != nums[slow] {
            slow += 1
            nums[slow] = nums[fast]   // 把新值搬到 slow 处
        }
    }
    return slow + 1                    // 唯一元素个数
}

// nums = [1,1,2]          -> 2, nums 前两位 [1,2]
// nums = [0,0,1,1,1,2,2]  -> 3, 前三位 [0,1,2]
// nums = []               -> 0
```

**Objective-C 实现**（用 NSMutableArray）：

```objc
- (NSInteger)removeDuplicates:(NSMutableArray<NSNumber *> *)nums {
    if (nums.count == 0) return 0;
    NSInteger slow = 0;
    for (NSInteger fast = 1; fast < (NSInteger)nums.count; fast++) {
        if (![nums[fast] isEqualToNumber:nums[slow]]) {
            slow += 1;
            nums[slow] = nums[fast];
        }
    }
    return slow + 1;
}
```

**复杂度**：时间 O(n)，空间 O(1)。

### 4.6 例题：移动零（原地）

> 把数组中所有 0 移到末尾，保持非零元素的相对顺序。

**Swift 实现**：

```swift
func moveZeroes(_ nums: inout [Int]) {
    var slow = 0   // slow 指向下一个非零元素应放的位置
    for fast in 0..<nums.count {
        if nums[fast] != 0 {
            nums.swapAt(slow, fast)   // 把非零元素交换到前面
            slow += 1
        }
    }
}

// nums = [0,1,0,3,12] -> [1,3,12,0,0]
// nums = [0]          -> [0]
// nums = [1,2,3]      -> [1,2,3]（无零，原样）
```

**Objective-C 实现**：

```objc
- (void)moveZeroes:(NSMutableArray<NSNumber *> *)nums {
    NSInteger slow = 0;
    for (NSInteger fast = 0; fast < (NSInteger)nums.count; fast++) {
        if (nums[fast].integerValue != 0) {
            [nums exchangeObjectAtIndex:slow withObjectAtIndex:fast];
            slow += 1;
        }
    }
}
```

**复杂度**：时间 O(n)，空间 O(1)。交换写法保证非零元素相对顺序不变。

## 5. 滑动窗口原理与模板

### 5.1 什么是滑动窗口

**滑动窗口**（Sliding Window）维护一个数据结构上的**连续区间** `[left, right]`，随着算法推进，窗口的右边界不断向右扩张（纳入新元素），当窗口违反某个约束时，左边界向右收缩（移除旧元素）。整个过程窗口像「在数组上滑动」，故得名。

它专门解决「**连续子数组 / 子串**」相关的最优化或计数问题，例如：

- 最长不含重复字符的子串
- 和 ≥ target 的最短连续子数组
- 包含所有目标字符的最小覆盖子串
- 长度为 k 的子数组最大平均值

### 5.2 滑动窗口为什么有效

滑动窗口能把 O(n²)/O(n³) 的暴力枚举（枚举所有子区间）优化为 O(n)，前提是问题满足**单调性**：

> 当窗口 `[left, right]` 满足某条件时，**收缩 left 不会破坏「已扫描部分的最优性」**；当窗口不满足时，**扩张 right 是唯一可能恢复/改善的方向**。

例如「最长无重复子串」：当窗口内出现重复字符，继续扩张 right 只会更糟，必须收缩 left 直到去掉重复——这种「扩张-收缩」的单调推进保证每个元素最多被 left、right 各访问一次。

> **反例警示**：求「和恰好等于 k 的最长子数组」，若数组**含负数**，窗口和不再随窗口扩大而单调增，滑动窗口失效，应改用**前缀和 + 哈希表**（见第 9 章相关题目对比）。

### 5.3 通用模板（可变窗口）

这是面试中最高频的滑动窗口模板，强烈建议背熟：

```
// 伪代码
初始化 window 状态（如计数字典 / 窗口和）
left = 0
for right in 0..<n:
    c = s[right]
    将 c 加入窗口，更新 window 状态

    while (窗口需要收缩的条件):   // 关键：何时收缩
        d = s[left]
        将 d 移出窗口，更新 window 状态
        left += 1

    // 在此处更新答案（位置取决于求最长还是最短）
    更新 answer
```

**两个关键决策点**：

1. **收缩条件（while 的判断）**：
   - 求**最长**满足条件的窗口 → 当窗口**违反**约束时收缩（收缩到重新合法）。
   - 求**最短**满足条件的窗口 → 当窗口**满足**约束时持续收缩（在满足的前提下尽量缩小）。
2. **答案更新位置**：
   - 求**最长** → 在收缩 while **之后**更新（此时窗口合法且尽可能大）。
   - 求**最短** → 在收缩 while **之内**更新（每次满足时记录，再尝试缩更小）。

### 5.4 滑动窗口的状态维护

窗口状态根据问题不同而不同，常见有：

| 问题类型 | 窗口状态 | 数据结构 |
|---------|---------|---------|
| 无重复字符 | 字符 → 出现次数 | 哈希表 / 定长数组（ASCII） |
| 子数组和 | 当前窗口元素和 | 单个整数变量 |
| 覆盖子串 | 缺失/满足的目标字符数 | 哈希表 + need/valid 计数器 |
| 定长窗口统计 | 当前窗口聚合值 | 整数 / 计数数组 |

## 6. 固定大小滑动窗口

### 6.1 特征

固定窗口（Fixed-size Sliding Window）的窗口长度 `k` 恒定。右指针每前进一步，左指针随之前进一步，窗口大小始终为 k。核心操作是「**进一个、出一个**」地增量更新窗口聚合值，避免每次重新计算整段。

### 6.2 例题：长度为 k 的子数组最大和

> 求长度恰为 k 的连续子数组的最大和。

**Swift 实现**：

```swift
func maxSumSubarray(_ nums: [Int], _ k: Int) -> Int {
    guard nums.count >= k, k > 0 else { return 0 }
    // 先算第一个窗口的和
    var windowSum = 0
    for i in 0..<k { windowSum += nums[i] }
    var maxSum = windowSum
    // 滑动：每步加入右边新元素、移除左边旧元素
    for right in k..<nums.count {
        windowSum += nums[right] - nums[right - k]
        maxSum = max(maxSum, windowSum)
    }
    return maxSum
}

// maxSumSubarray([2,1,5,1,3,2], 3) -> 9 (子数组 [5,1,3])
// maxSumSubarray([2,3], 3)         -> 0 (长度不足)
```

**Objective-C 实现**：

```objc
- (NSInteger)maxSumSubarray:(NSArray<NSNumber *> *)nums k:(NSInteger)k {
    if ((NSInteger)nums.count < k || k <= 0) return 0;
    NSInteger windowSum = 0;
    for (NSInteger i = 0; i < k; i++) windowSum += nums[i].integerValue;
    NSInteger maxSum = windowSum;
    for (NSInteger right = k; right < (NSInteger)nums.count; right++) {
        windowSum += nums[right].integerValue - nums[right - k].integerValue;
        maxSum = MAX(maxSum, windowSum);
    }
    return maxSum;
}
```

**复杂度**：时间 O(n)，空间 O(1)。增量更新 `windowSum += nums[right] - nums[right-k]` 是固定窗口的精髓，避免 O(nk) 重算。

### 6.3 例题：长度为 k 的子数组最大平均值

> 返回长度为 k 的连续子数组的最大平均值。

**Swift 实现**：

```swift
func findMaxAverage(_ nums: [Int], _ k: Int) -> Double {
    var windowSum = 0
    for i in 0..<k { windowSum += nums[i] }
    var maxSum = windowSum
    for right in k..<nums.count {
        windowSum += nums[right] - nums[right - k]
        maxSum = max(maxSum, windowSum)
    }
    return Double(maxSum) / Double(k)
}

// findMaxAverage([1,12,-5,-6,50,3], 4) -> 12.75
```

**复杂度**：时间 O(n)，空间 O(1)。注意只需求最大和，最后再除以 k 即可，避免浮点累加误差。

### 6.4 例题：找到字符串中所有字母异位词

> 给定 s 和 p，找出 s 中所有 p 的异位词（字母组成相同）的起始索引。这是「定长窗口 + 计数比较」的典型。

**Swift 实现**：

```swift
func findAnagrams(_ s: String, _ p: String) -> [Int] {
    let sArr = Array(s.unicodeScalars).map { Int($0.value) }
    let pArr = Array(p.unicodeScalars).map { Int($0.value) }
    let n = sArr.count, m = pArr.count
    guard n >= m else { return [] }

    var need = [Int](repeating: 0, count: 128)   // p 的字符计数
    var window = [Int](repeating: 0, count: 128) // 当前窗口字符计数
    for c in pArr { need[c] += 1 }

    var result: [Int] = []
    for right in 0..<n {
        window[sArr[right]] += 1          // 右边进
        if right >= m {
            window[sArr[right - m]] -= 1  // 左边出（维持窗口长度 m）
        }
        if right >= m - 1 && window == need {
            result.append(right - m + 1)  // 窗口与 need 完全相同 -> 异位词
        }
    }
    return result
}

// findAnagrams("cbaebabacd", "abc") -> [0, 6]
// findAnagrams("abab", "ab")        -> [0, 1, 2]
```

**Objective-C 实现**：

```objc
- (NSArray<NSNumber *> *)findAnagrams:(NSString *)s p:(NSString *)p {
    NSUInteger n = s.length, m = p.length;
    if (n < m) return @[];
    NSInteger need[128] = {0};
    NSInteger window[128] = {0};
    for (NSUInteger i = 0; i < m; i++) {
        need[[p characterAtIndex:i]] += 1;
    }
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    for (NSUInteger right = 0; right < n; right++) {
        window[[s characterAtIndex:right]] += 1;
        if (right >= m) {
            window[[s characterAtIndex:right - m]] -= 1;
        }
        if (right >= m - 1) {
            BOOL equal = YES;
            for (int k = 0; k < 128; k++) {
                if (window[k] != need[k]) { equal = NO; break; }
            }
            if (equal) [result addObject:@(right - m + 1)];
        }
    }
    return result;
}
```

**复杂度**：时间 O(n · Σ)，Σ 为字符集大小（128，视为常数）→ O(n)；空间 O(Σ) = O(1)。

## 7. 可变大小滑动窗口

### 7.1 特征

可变窗口（Variable-size Sliding Window）的窗口长度动态变化。右指针主动扩张，当窗口违反/满足条件时左指针收缩。这是滑动窗口最强大、最常考的形式。

### 7.2 例题：无重复字符的最长子串（求最长）

> 给定字符串 s，找出不含重复字符的最长子串的长度。

**思路**：维护窗口内字符计数。右指针纳入字符；若该字符导致重复，收缩左指针直到无重复；每步更新最大长度。

**Swift 实现**：

```swift
func lengthOfLongestSubstring(_ s: String) -> Int {
    let chars = Array(s)
    var window: [Character: Int] = [:]   // 字符 -> 窗口内出现次数
    var left = 0
    var maxLen = 0
    for right in 0..<chars.count {
        let c = chars[right]
        window[c, default: 0] += 1
        // 收缩条件：c 出现超过一次（即有重复）
        while window[c]! > 1 {
            let d = chars[left]
            window[d]! -= 1
            left += 1
        }
        // 收缩后窗口合法，更新最长长度
        maxLen = max(maxLen, right - left + 1)
    }
    return maxLen
}

// lengthOfLongestSubstring("abcabcbb") -> 3 ("abc")
// lengthOfLongestSubstring("bbbbb")    -> 1 ("b")
// lengthOfLongestSubstring("pwwkew")   -> 3 ("wke")
// lengthOfLongestSubstring("")         -> 0
```

**Objective-C 实现**：

```objc
- (NSInteger)lengthOfLongestSubstring:(NSString *)s {
    NSInteger window[128] = {0};
    NSInteger left = 0, maxLen = 0;
    NSUInteger n = s.length;
    for (NSUInteger right = 0; right < n; right++) {
        unichar c = [s characterAtIndex:right];
        window[c] += 1;
        while (window[c] > 1) {
            unichar d = [s characterAtIndex:left];
            window[d] -= 1;
            left += 1;
        }
        maxLen = MAX(maxLen, (NSInteger)right - left + 1);
    }
    return maxLen;
}
```

**复杂度**：时间 O(n)（left、right 各最多移动 n 次），空间 O(Σ)。

### 7.3 例题：长度最小的子数组（求最短）

> 给定正整数数组 nums 和目标 target，找出和 ≥ target 的最短连续子数组长度。不存在返回 0。

**思路**：右指针扩张累加窗口和；当窗口和 ≥ target，**在满足条件时收缩**并记录最短长度。

**Swift 实现**：

```swift
func minSubArrayLen(_ target: Int, _ nums: [Int]) -> Int {
    var left = 0
    var windowSum = 0
    var minLen = Int.max
    for right in 0..<nums.count {
        windowSum += nums[right]
        // 满足条件时持续收缩，求最短
        while windowSum >= target {
            minLen = min(minLen, right - left + 1)  // 在 while 内更新
            windowSum -= nums[left]
            left += 1
        }
    }
    return minLen == Int.max ? 0 : minLen
}

// minSubArrayLen(7, [2,3,1,2,4,3]) -> 2 ([4,3])
// minSubArrayLen(4, [1,4,4])       -> 1 ([4])
// minSubArrayLen(11, [1,1,1,1,1])  -> 0 (总和不足)
```

**Objective-C 实现**：

```objc
- (NSInteger)minSubArrayLen:(NSInteger)target nums:(NSArray<NSNumber *> *)nums {
    NSInteger left = 0, windowSum = 0, minLen = NSIntegerMax;
    for (NSInteger right = 0; right < (NSInteger)nums.count; right++) {
        windowSum += nums[right].integerValue;
        while (windowSum >= target) {
            minLen = MIN(minLen, right - left + 1);
            windowSum -= nums[left].integerValue;
            left += 1;
        }
    }
    return minLen == NSIntegerMax ? 0 : minLen;
}
```

> **对比 7.2 与 7.3**：求最长在 while 外更新答案、while 判断「违反」；求最短在 while 内更新答案、while 判断「满足」。这是滑动窗口两大范式的分水岭，务必牢记。

**复杂度**：时间 O(n)，空间 O(1)。

### 7.4 例题：最小覆盖子串（滑动窗口之王）

> 给定 s 和 t，返回 s 中涵盖 t 所有字符（含重复）的最小子串。不存在返回空串。

**思路**：用 `need` 记录 t 中每个字符所需数量，`window` 记录窗口内字符数。`valid` 记录「已满足数量要求的字符种类数」。当 `valid == need.count` 时窗口覆盖了 t，尝试收缩求最短。

**Swift 实现**：

```swift
func minWindow(_ s: String, _ t: String) -> String {
    let sArr = Array(s)
    var need: [Character: Int] = [:]
    for c in t { need[c, default: 0] += 1 }

    var window: [Character: Int] = [:]
    var left = 0, valid = 0
    var start = 0, minLen = Int.max   // 记录最优窗口的起点与长度

    for right in 0..<sArr.count {
        let c = sArr[right]
        if need[c] != nil {
            window[c, default: 0] += 1
            if window[c]! == need[c]! { valid += 1 }
        }
        // 当所有需要的字符都已满足，收缩左边界求最短
        while valid == need.count {
            if right - left + 1 < minLen {
                start = left
                minLen = right - left + 1
            }
            let d = sArr[left]
            left += 1
            if need[d] != nil {
                if window[d]! == need[d]! { valid -= 1 }
                window[d]! -= 1
            }
        }
    }
    return minLen == Int.max ? "" : String(sArr[start..<start + minLen])
}

// minWindow("ADOBECODEBANC", "ABC") -> "BANC"
// minWindow("a", "a")               -> "a"
// minWindow("a", "aa")              -> ""  (t 需要两个 a，s 只有一个)
```

**Objective-C 实现**：

```objc
- (NSString *)minWindow:(NSString *)s t:(NSString *)t {
    NSInteger need[128] = {0};
    NSInteger window[128] = {0};
    NSInteger required = 0;   // 不同字符种类数
    for (NSUInteger i = 0; i < t.length; i++) {
        unichar c = [t characterAtIndex:i];
        if (need[c] == 0) required += 1;
        need[c] += 1;
    }
    NSInteger left = 0, valid = 0;
    NSInteger start = 0, minLen = NSIntegerMax;
    for (NSUInteger right = 0; right < s.length; right++) {
        unichar c = [s characterAtIndex:right];
        if (need[c] > 0) {
            window[c] += 1;
            if (window[c] == need[c]) valid += 1;
        }
        while (valid == required) {
            if ((NSInteger)right - left + 1 < minLen) {
                start = left;
                minLen = (NSInteger)right - left + 1;
            }
            unichar d = [s characterAtIndex:left];
            left += 1;
            if (need[d] > 0) {
                if (window[d] == need[d]) valid -= 1;
                window[d] -= 1;
            }
        }
    }
    return minLen == NSIntegerMax ? @"" : [s substringWithRange:NSMakeRange(start, minLen)];
}
```

**复杂度**：时间 O(|s| + |t|)，空间 O(Σ)。

### 7.5 例题：至多包含 K 个不同字符的最长子串

> 找出至多包含 k 个不同字符的最长子串长度。

**Swift 实现**：

```swift
func lengthOfLongestSubstringKDistinct(_ s: String, _ k: Int) -> Int {
    if k == 0 { return 0 }
    let chars = Array(s)
    var window: [Character: Int] = [:]
    var left = 0, maxLen = 0
    for right in 0..<chars.count {
        window[chars[right], default: 0] += 1
        // 不同字符数超过 k 时收缩
        while window.count > k {
            let d = chars[left]
            window[d]! -= 1
            if window[d]! == 0 { window.removeValue(forKey: d) }  // 计数归零要删除键
            left += 1
        }
        maxLen = max(maxLen, right - left + 1)
    }
    return maxLen
}

// lengthOfLongestSubstringKDistinct("eceba", 2) -> 3 ("ece")
// lengthOfLongestSubstringKDistinct("aa", 1)    -> 2 ("aa")
```

> **易错点**：当某字符计数减到 0 时，**必须从字典移除该键**，否则 `window.count` 仍把它算作一个不同字符，导致结果错误。

**复杂度**：时间 O(n)，空间 O(k)。

## 8. 同向双指针与原地操作

### 8.1 核心思想

同向双指针（Same-Direction Two Pointers）让 `slow` 与 `fast` 朝同一方向移动，`fast` 负责「探索」，`slow` 负责「构建结果」。它是数组**原地修改**（in-place）的利器，空间复杂度 O(1)。快慢双指针（第 4 章）是其特例（速度差固定）；此处更广义——slow 仅在满足条件时前进。

### 8.2 例题：移除元素（原地）

> 原地移除数组中所有等于 val 的元素，返回新长度。元素顺序可变。

**Swift 实现**：

```swift
func removeElement(_ nums: inout [Int], _ val: Int) -> Int {
    var slow = 0
    for fast in 0..<nums.count {
        if nums[fast] != val {
            nums[slow] = nums[fast]   // 保留非 val 元素
            slow += 1
        }
    }
    return slow
}

// nums = [3,2,2,3], val = 3 -> 2, 前两位 [2,2]
// nums = [0,1,2,2,3,0,4,2], val = 2 -> 5
```

**Objective-C 实现**：

```objc
- (NSInteger)removeElement:(NSMutableArray<NSNumber *> *)nums val:(NSInteger)val {
    NSInteger slow = 0;
    for (NSInteger fast = 0; fast < (NSInteger)nums.count; fast++) {
        if (nums[fast].integerValue != val) {
            nums[slow] = nums[fast];
            slow += 1;
        }
    }
    return slow;
}
```

**复杂度**：时间 O(n)，空间 O(1)。

### 8.3 例题：有序数组的平方（双指针归并）

> 给定升序数组（含负数），返回每个元素平方后组成的升序数组。

**思路**：平方后最大值一定出现在数组**两端**（绝对值最大处）。用对撞双指针从两端取较大平方填入结果数组末尾。

**Swift 实现**：

```swift
func sortedSquares(_ nums: [Int]) -> [Int] {
    let n = nums.count
    var result = [Int](repeating: 0, count: n)
    var left = 0, right = n - 1
    var pos = n - 1               // 从结果末尾往前填
    while left <= right {
        let leftSq = nums[left] * nums[left]
        let rightSq = nums[right] * nums[right]
        if leftSq > rightSq {
            result[pos] = leftSq
            left += 1
        } else {
            result[pos] = rightSq
            right -= 1
        }
        pos -= 1
    }
    return result
}

// sortedSquares([-4,-1,0,3,10]) -> [0,1,9,16,100]
// sortedSquares([-7,-3,2,3,11]) -> [4,9,9,49,121]
```

**Objective-C 实现**：

```objc
- (NSArray<NSNumber *> *)sortedSquares:(NSArray<NSNumber *> *)nums {
    NSInteger n = (NSInteger)nums.count;
    NSMutableArray<NSNumber *> *result = [NSMutableArray arrayWithCapacity:n];
    for (NSInteger i = 0; i < n; i++) [result addObject:@0];
    NSInteger left = 0, right = n - 1, pos = n - 1;
    while (left <= right) {
        NSInteger l = nums[left].integerValue;
        NSInteger r = nums[right].integerValue;
        NSInteger leftSq = l * l, rightSq = r * r;
        if (leftSq > rightSq) {
            result[pos] = @(leftSq);
            left += 1;
        } else {
            result[pos] = @(rightSq);
            right -= 1;
        }
        pos -= 1;
    }
    return result;
}
```

**复杂度**：时间 O(n)，空间 O(n)（结果数组，原地不可行因符号交错）。

## 9. 经典算法题精讲

本章汇总双指针与滑动窗口的高频经典题，每题给出思路、双语实现与复杂度。

### 9.1 反转字符串（对撞）

> 原地反转字符数组。

**Swift 实现**：

```swift
func reverseString(_ s: inout [Character]) {
    var left = 0, right = s.count - 1
    while left < right {
        s.swapAt(left, right)
        left += 1
        right -= 1
    }
}
// ["h","e","l","l","o"] -> ["o","l","l","e","h"]
```

**Objective-C 实现**：

```objc
- (void)reverseString:(NSMutableArray<NSString *> *)s {
    NSInteger left = 0, right = (NSInteger)s.count - 1;
    while (left < right) {
        [s exchangeObjectAtIndex:left withObjectAtIndex:right];
        left += 1;
        right -= 1;
    }
}
```

**复杂度**：时间 O(n)，空间 O(1)。

### 9.2 两数之和的有序变体与四数之和

> 四数之和：找出所有和为 target 的不重复四元组。

**思路**：排序后固定两个数（两层循环），内层用对撞双指针，整体 O(n³)。去重逻辑与三数之和一致。

**Swift 实现**：

```swift
func fourSum(_ nums: [Int], _ target: Int) -> [[Int]] {
    let a = nums.sorted()
    let n = a.count
    var res: [[Int]] = []
    if n < 4 { return res }
    var i = 0
    while i < n - 3 {
        if i > 0 && a[i] == a[i - 1] { i += 1; continue }
        var j = i + 1
        while j < n - 2 {
            if j > i + 1 && a[j] == a[j - 1] { j += 1; continue }
            var left = j + 1, right = n - 1
            while left < right {
                let sum = a[i] + a[j] + a[left] + a[right]
                if sum == target {
                    res.append([a[i], a[j], a[left], a[right]])
                    while left < right && a[left] == a[left + 1] { left += 1 }
                    while left < right && a[right] == a[right - 1] { right -= 1 }
                    left += 1; right -= 1
                } else if sum < target {
                    left += 1
                } else {
                    right -= 1
                }
            }
            j += 1
        }
        i += 1
    }
    return res
}
// fourSum([1,0,-1,0,-2,2], 0) -> [[-2,-1,1,2],[-2,0,0,2],[-1,0,0,1]]
```

> **溢出提示**：四数相加在 C/OC 中可能溢出 int，应使用 `NSInteger`（64 位）或在 Swift 中用 `Int`（平台位宽）；若数值极大需考虑 `Int64`/overflow 检查。

**复杂度**：时间 O(n³)，空间 O(log n)（排序）。

### 9.3 接雨水（双指针经典难题）

> 给定柱状高度数组，计算下雨后能接多少雨水。

**思路（对撞双指针）**：位置 i 能接的水 = `min(左侧最高, 右侧最高) - height[i]`。用 `leftMax`、`rightMax` 配合左右指针，每次处理较矮一侧（因为该侧的水量已由较矮的最大值确定）。

**Swift 实现**：

```swift
func trap(_ height: [Int]) -> Int {
    var left = 0, right = height.count - 1
    var leftMax = 0, rightMax = 0
    var water = 0
    while left < right {
        if height[left] < height[right] {
            // 左侧较矮：左侧水量由 leftMax 决定
            if height[left] >= leftMax {
                leftMax = height[left]
            } else {
                water += leftMax - height[left]
            }
            left += 1
        } else {
            if height[right] >= rightMax {
                rightMax = height[right]
            } else {
                water += rightMax - height[right]
            }
            right -= 1
        }
    }
    return water
}
// trap([0,1,0,2,1,0,1,3,2,1,2,1]) -> 6
// trap([4,2,0,3,2,5])             -> 9
```

**Objective-C 实现**：

```objc
- (NSInteger)trap:(NSArray<NSNumber *> *)height {
    NSInteger left = 0, right = (NSInteger)height.count - 1;
    NSInteger leftMax = 0, rightMax = 0, water = 0;
    while (left < right) {
        NSInteger hl = height[left].integerValue;
        NSInteger hr = height[right].integerValue;
        if (hl < hr) {
            if (hl >= leftMax) leftMax = hl;
            else water += leftMax - hl;
            left += 1;
        } else {
            if (hr >= rightMax) rightMax = hr;
            else water += rightMax - hr;
            right -= 1;
        }
    }
    return water;
}
```

**复杂度**：时间 O(n)，空间 O(1)。

### 9.4 替换后的最长重复字符

> 字符串 s，可替换至多 k 个字符，求替换后最长的相同字符连续子串长度。

**思路（可变窗口）**：维护窗口内出现次数最多的字符数 `maxCount`。当 `窗口长度 - maxCount > k`（需替换的字符数超过 k）时收缩。

**Swift 实现**：

```swift
func characterReplacement(_ s: String, _ k: Int) -> Int {
    let chars = Array(s)
    var count = [Int](repeating: 0, count: 26)
    var left = 0, maxCount = 0, result = 0
    let aValue = Int(Character("A").asciiValue!)
    for right in 0..<chars.count {
        let idx = Int(chars[right].asciiValue!) - aValue
        count[idx] += 1
        maxCount = max(maxCount, count[idx])
        // 窗口内非最多字符的数量 > k，需收缩
        if (right - left + 1) - maxCount > k {
            let lidx = Int(chars[left].asciiValue!) - aValue
            count[lidx] -= 1
            left += 1
        }
        result = max(result, right - left + 1)
    }
    return result
}
// characterReplacement("ABAB", 2) -> 4
// characterReplacement("AABABBA", 1) -> 4
```

> **微妙点**：这里用 `if` 而非 `while` 收缩，且 `maxCount` 不回退也不影响正确性——因为我们只关心**历史最大窗口**，maxCount 偏大只会让窗口暂时不收缩，但窗口长度只增不减，最终 result 正确。这是该题的经典「不回退」技巧。

**复杂度**：时间 O(n)，空间 O(26) = O(1)。

### 9.5 和为定值最长子数组的对比（含负数 → 滑窗失效）

> 给定含负数的数组，求和恰为 k 的最长连续子数组长度。

**为什么不能用滑动窗口**：含负数时，扩大窗口和可能减小、收缩窗口和可能增大，**失去单调性**，滑动窗口的「收缩决策」无依据。正确解法是**前缀和 + 哈希表**：

**Swift 实现**：

```swift
func maxSubArrayLen(_ nums: [Int], _ k: Int) -> Int {
    var prefixSum = 0
    var firstIndex: [Int: Int] = [0: -1]  // 前缀和 -> 最早出现的下标
    var maxLen = 0
    for (i, num) in nums.enumerated() {
        prefixSum += num
        // 若存在 prefix[j] = prefixSum - k，则 (j, i] 区间和为 k
        if let j = firstIndex[prefixSum - k] {
            maxLen = max(maxLen, i - j)
        }
        if firstIndex[prefixSum] == nil {
            firstIndex[prefixSum] = i      // 只记最早，求最长
        }
    }
    return maxLen
}
// maxSubArrayLen([1,-1,5,-2,3], 3) -> 4 ([1,-1,5,-2])
// maxSubArrayLen([-2,-1,2,1], 1)   -> 2 ([-1,2])
```

> **结论**：这是双指针知识体系里极其重要的「**边界辨析**」——滑动窗口仅在「全正数」或具备单调性时适用；含负数的子数组和问题，前缀和 + 哈希才是正解。面试常借此考查对滑窗适用条件的理解。

**复杂度**：时间 O(n)，空间 O(n)。

### 9.6 颜色分类（荷兰国旗问题，三指针）

> 数组只含 0、1、2，原地排序使相同颜色相邻。

**思路（三指针）**：`low` 指向下一个 0 应放的位置，`high` 指向下一个 2 应放的位置，`cur` 扫描。

**Swift 实现**：

```swift
func sortColors(_ nums: inout [Int]) {
    var low = 0, cur = 0, high = nums.count - 1
    while cur <= high {
        if nums[cur] == 0 {
            nums.swapAt(low, cur)
            low += 1
            cur += 1
        } else if nums[cur] == 2 {
            nums.swapAt(cur, high)
            high -= 1
            // 注意：cur 不前进，因换来的元素还需判断
        } else {
            cur += 1
        }
    }
}
// nums = [2,0,2,1,1,0] -> [0,0,1,1,2,2]
```

**Objective-C 实现**：

```objc
- (void)sortColors:(NSMutableArray<NSNumber *> *)nums {
    NSInteger low = 0, cur = 0, high = (NSInteger)nums.count - 1;
    while (cur <= high) {
        NSInteger v = nums[cur].integerValue;
        if (v == 0) {
            [nums exchangeObjectAtIndex:low withObjectAtIndex:cur];
            low += 1; cur += 1;
        } else if (v == 2) {
            [nums exchangeObjectAtIndex:cur withObjectAtIndex:high];
            high -= 1;
        } else {
            cur += 1;
        }
    }
}
```

> **易错点**：交换到 high 时 `cur` **不能前进**，因为从 high 换过来的值尚未检查；而交换到 low 时 cur 可前进，因 low ≤ cur，换来的值必是已扫描过的 1。

**复杂度**：时间 O(n)，空间 O(1)。

### 9.7 删除链表的倒数第 N 个节点（快慢指针 + 间隔）

> 一次遍历删除链表倒数第 n 个节点。

**思路**：快指针先走 n 步，然后快慢同步前进，快指针到尾时慢指针恰在待删节点的前驱。用哑节点（dummy）统一处理删除头节点的边界。

**Swift 实现**：

```swift
func removeNthFromEnd(_ head: ListNode?, _ n: Int) -> ListNode? {
    let dummy = ListNode(0)
    dummy.next = head
    var fast: ListNode? = dummy
    var slow: ListNode? = dummy
    // 快指针先走 n+1 步，使 fast 与 slow 间隔 n+1
    for _ in 0...n {
        fast = fast?.next
    }
    // 同步前进，fast 到 nil 时 slow 在待删节点前驱
    while fast != nil {
        fast = fast?.next
        slow = slow?.next
    }
    slow?.next = slow?.next?.next   // 删除
    return dummy.next
}
// 1->2->3->4->5, n=2 -> 1->2->3->5
// 1, n=1 -> nil（哑节点保证安全）
```

**Objective-C 实现**：

```objc
- (ListNode *)removeNthFromEnd:(ListNode *)head n:(NSInteger)n {
    ListNode *dummy = [ListNode new];
    dummy.next = head;
    ListNode *fast = dummy, *slow = dummy;
    for (NSInteger i = 0; i <= n; i++) {
        fast = fast.next;
    }
    while (fast != nil) {
        fast = fast.next;
        slow = slow.next;
    }
    slow.next = slow.next.next;
    return dummy.next;
}
```

> **哑节点（dummy node）的价值**：当待删的是头节点时，没有前驱节点，dummy 充当虚拟前驱，统一了删除逻辑，避免特判。这是链表双指针的经典工程技巧。

**复杂度**：时间 O(n)（一次遍历），空间 O(1)。

### 9.8 相交链表（双指针走两遍）

> 找两个单链表的相交起始节点，不相交返回 nil。

**思路**：两指针 pA、pB 分别从 headA、headB 出发，到尾后切换到对方链表头。若相交，二者走过 `lenA + lenB` 后必在交点相遇；若不相交，同时到达 nil。

**Swift 实现**：

```swift
func getIntersectionNode(_ headA: ListNode?, _ headB: ListNode?) -> ListNode? {
    if headA == nil || headB == nil { return nil }
    var pA = headA
    var pB = headB
    while pA !== pB {
        pA = (pA == nil) ? headB : pA?.next   // 走完 A 换到 B
        pB = (pB == nil) ? headA : pB?.next   // 走完 B 换到 A
    }
    return pA   // 相遇点（交点或同时为 nil）
}
```

**Objective-C 实现**：

```objc
- (ListNode *)getIntersectionNode:(ListNode *)headA headB:(ListNode *)headB {
    if (headA == nil || headB == nil) return nil;
    ListNode *pA = headA, *pB = headB;
    while (pA != pB) {
        pA = (pA == nil) ? headB : pA.next;
        pB = (pB == nil) ? headA : pB.next;
    }
    return pA;
}
```

> **数学原理**：设 A 独有长度 a、B 独有长度 b、公共长度 c。pA 走 a+c+b 到交点，pB 走 b+c+a 到交点，路程相等故同步到达。不相交时 a+c+b == b+c+a 步后同时为 nil，循环终止。

**复杂度**：时间 O(m+n)，空间 O(1)。

### 9.9 回文链表（快慢找中点 + 反转后半）

> 判断单链表是否回文，要求 O(1) 空间。

**思路**：快慢指针找中点 → 反转后半部分 → 双指针前后比较 → （可选）恢复链表。

**Swift 实现**：

```swift
func isPalindromeList(_ head: ListNode?) -> Bool {
    if head == nil || head?.next == nil { return true }
    // 1. 快慢找中点
    var slow = head, fast = head
    while fast?.next != nil && fast?.next?.next != nil {
        slow = slow?.next
        fast = fast?.next?.next
    }
    // 2. 反转后半部分（从 slow.next 开始）
    var prev: ListNode? = nil
    var cur = slow?.next
    while cur != nil {
        let next = cur?.next
        cur?.next = prev
        prev = cur
        cur = next
    }
    // 3. 双指针比较前半与反转后的后半
    var p1 = head
    var p2 = prev
    var result = true
    while p2 != nil {
        if p1?.val != p2?.val { result = false; break }
        p1 = p1?.next
        p2 = p2?.next
    }
    return result
}
```

**Objective-C 实现**：

```objc
- (BOOL)isPalindromeList:(ListNode *)head {
    if (head == nil || head.next == nil) return YES;
    ListNode *slow = head, *fast = head;
    while (fast.next != nil && fast.next.next != nil) {
        slow = slow.next;
        fast = fast.next.next;
    }
    ListNode *prev = nil, *cur = slow.next;
    while (cur != nil) {
        ListNode *next = cur.next;
        cur.next = prev;
        prev = cur;
        cur = next;
    }
    ListNode *p1 = head, *p2 = prev;
    BOOL result = YES;
    while (p2 != nil) {
        if (p1.val != p2.val) { result = NO; break; }
        p1 = p1.next;
        p2 = p2.next;
    }
    return result;
}
```

**复杂度**：时间 O(n)，空间 O(1)。综合运用了快慢指针、链表反转、对撞比较三种技巧。

### 9.10 水果成篮（至多两种类型的最长窗口）

> 数组表示一排树的水果种类，篮子至多装两种水果，求能连续采摘的最多水果数。本质是「至多包含 2 个不同元素的最长子数组」。

**Swift 实现**：

```swift
func totalFruit(_ fruits: [Int]) -> Int {
    var basket: [Int: Int] = [:]   // 水果种类 -> 窗口内数量
    var left = 0, maxFruit = 0
    for right in 0..<fruits.count {
        basket[fruits[right], default: 0] += 1
        while basket.count > 2 {     // 超过两种就收缩
            let f = fruits[left]
            basket[f]! -= 1
            if basket[f]! == 0 { basket.removeValue(forKey: f) }
            left += 1
        }
        maxFruit = max(maxFruit, right - left + 1)
    }
    return maxFruit
}
// totalFruit([1,2,1])     -> 3
// totalFruit([0,1,2,2])   -> 3 ([1,2,2])
// totalFruit([1,2,3,2,2]) -> 4 ([2,3,2,2])
```

**复杂度**：时间 O(n)，空间 O(1)（至多 3 个键）。这是 7.5「至多 K 个不同字符」的 K=2 实例化，展示模板复用。

### 9.11 字符串的排列（定长窗口 + 计数匹配）

> 判断 s2 是否包含 s1 的某个排列作为子串。

**Swift 实现**：

```swift
func checkInclusion(_ s1: String, _ s2: String) -> Bool {
    let m = s1.count, n = s2.count
    if m > n { return false }
    let a = Array(s1.unicodeScalars).map { Int($0.value) - 97 }
    let b = Array(s2.unicodeScalars).map { Int($0.value) - 97 }
    var need = [Int](repeating: 0, count: 26)
    var window = [Int](repeating: 0, count: 26)
    for c in a { need[c] += 1 }
    for right in 0..<n {
        window[b[right]] += 1
        if right >= m { window[b[right - m]] -= 1 }   // 维持定长
        if right >= m - 1 && window == need { return true }
    }
    return false
}
// checkInclusion("ab", "eidbaooo") -> true ("ba")
// checkInclusion("ab", "eidboaoo") -> false
```

**复杂度**：时间 O(n)，空间 O(1)。与 6.4「找异位词」同构，区别仅是返回 bool。

### 9.12 最长重复子数组的对比（双指针 vs DP）

> 两个数组的最长公共连续子数组长度。

这是一个**双指针思想不直接适用、需 DP** 的辨析案例。朴素双指针无法处理「对齐偏移」，需用动态规划 `dp[i][j] = (A[i]==B[j]) ? dp[i-1][j-1]+1 : 0`。

**Swift 实现**：

```swift
func findLength(_ nums1: [Int], _ nums2: [Int]) -> Int {
    let m = nums1.count, n = nums2.count
    var dp = [[Int]](repeating: [Int](repeating: 0, count: n + 1), count: m + 1)
    var maxLen = 0
    for i in 1...m {
        for j in 1...n {
            if nums1[i-1] == nums2[j-1] {
                dp[i][j] = dp[i-1][j-1] + 1
                maxLen = max(maxLen, dp[i][j])
            }
        }
    }
    return maxLen
}
// findLength([1,2,3,2,1], [3,2,1,4,7]) -> 3 ([3,2,1])
```

> **辨析意义**：并非所有「子数组」问题都能用双指针/滑窗。当涉及**两个序列的对齐匹配**时，通常需要二维 DP。识别「何时不该用双指针」与「何时该用」同等重要。

**复杂度**：时间 O(m·n)，空间 O(m·n)（可滚动数组优化到 O(n)）。
## 10. 复杂度总览

| 技巧 / 题型 | 时间复杂度 | 空间复杂度 | 关键前提 |
|------------|----------|----------|---------|
| 对撞双指针（两数之和、回文） | O(n) | O(1) | 有序 / 两端推进无遗漏 |
| 三数之和 / 四数之和 | O(n²) / O(n³) | O(log n)（排序） | 排序 + 去重 |
| 盛水容器 / 接雨水 | O(n) | O(1) | 贪心移动较矮边 |
| 快慢双指针（链表中点/环） | O(n) | O(1) | 速度差产生确定相遇 |
| 数组原地去重 / 移除 | O(n) | O(1) | 同向扫描 + 构建 |
| 固定窗口（定长子数组） | O(n) | O(1) | 增量更新窗口聚合 |
| 可变窗口（最长/最短子串） | O(n) | O(Σ) | 性质单调可收缩 |
| 最小覆盖子串 | O(\|s\|+\|t\|) | O(Σ) | need/valid 计数 |
| 含负数子数组和（对比） | O(n) | O(n) | 前缀和+哈希（非滑窗） |

> **共性结论**：双指针/滑动窗口把暴力的 O(n²)~O(n³) 降到 O(n)~O(n²)，核心代价是「前提条件」（有序、单调、连续）。摊还分析是理解其线性复杂度的关键。

## 11. 面试题

**Q1：滑动窗口和双指针是什么关系？**

A：滑动窗口是双指针的一个**子类**（同向双指针 + 维护窗口状态）。双指针更宽泛，包含对撞、快慢、同向三大类。滑动窗口特指用左右边界维护一段**连续区间**来解决子数组/子串问题。

**Q2：什么情况下滑动窗口会失效？**

A：当问题失去**单调性**时。典型例子：数组含**负数**时求「和为定值/不超过定值的最长子数组」——扩大窗口和不再单调递增，收缩决策失去依据。此时应改用**前缀和 + 哈希表**。滑动窗口通常要求元素全正（窗口和单调）或约束随窗口扩张单调变化（如不同字符数）。

**Q3：求「最长」和求「最短」滑动窗口模板有何区别？**

A：核心两点不同——（1）**收缩条件**：求最长在窗口「违反约束」时收缩（恢复合法），求最短在窗口「满足约束」时收缩（缩到极限）。（2）**答案更新位置**：求最长在收缩 while **之后**更新（窗口合法且最大），求最短在收缩 while **之内**更新（每次满足都记录）。

**Q4：Floyd 判圈算法为什么快指针速度是 2 而不是 3？**

A：速度差为 1 保证快慢指针在环内每轮**接近 1 个节点**，必然恰好相遇而不会「跳过」。若速度差大于 1（如快走 3、慢走 1，差 2），在奇数长度环中可能两指针「错过」永不在同一节点相遇（虽然增大环遍历轮次仍能相遇，但分析更复杂）。速度比 2:1 是最简洁且数学性质最优雅的选择，且第二阶段找环入口的推导依赖于这个 2:1 比例。

**Q5：为什么接雨水可以用双指针做到 O(1) 空间？**

A：位置 i 的水量 = `min(leftMax, rightMax) - height[i]`。双指针法每次处理**较矮的一侧**：当 `height[left] < height[right]` 时，左侧的水量**只取决于 leftMax**（因为右边必有比 height[right] 不低的边界保证 rightMax ≥ height[right] > height[left]，min 由左侧决定）。因此无需预先用数组存所有 leftMax/rightMax，只用两个变量即可，空间从 O(n) 降到 O(1)。

**Q6：三数之和的去重为什么要分三处？**

A：三处去重分别针对三个数：（1）固定数 `i` 去重——`nums[i] == nums[i-1]` 时跳过，避免相同起点重复枚举；（2）找到解后 `left` 去重——跳过相邻相同值；（3）`right` 去重——同理。只去固定数无法避免内层产生重复三元组，三处缺一不可。

**Q7：滑动窗口里用哈希表统计字符 vs 用定长数组，怎么选？**

A：若字符集已知且小（如 ASCII 128、小写字母 26），用**定长数组**（`Int[128]`）更快、缓存友好、无哈希开销，空间也是 O(1)。若字符集大或未知（Unicode、任意对象），用哈希表更通用。本篇 OC 实现多用定长数组（性能好），Swift 示例兼有两种以演示通用写法。

**Q8：删除有序数组重复项（每个元素保留至多 2 个）怎么改？**

A：把判断条件从 `nums[fast] != nums[slow]` 改为 `slow < 2 || nums[fast] != nums[slow-2]`：即只要当前元素与「慢指针往前数第 2 个」不同就保留。这是同向双指针的通用扩展模板（保留至多 k 个）。

## 12. 易错点与最佳实践

### 12.1 常见易错点

1. **循环终止条件 `<` vs `<=`**：对撞双指针求「两个不同元素配对」用 `left < right`；快慢/同向遍历用 `<= high` 或 `< n`，需根据是否允许 left==right 仔细区分。荷兰国旗用 `cur <= high`。

2. **收缩内层 while 忘记边界检查**：回文跳过非法字符时，内层 while 必须再判 `left < right`，否则全标点字符串会越界。

3. **计数归零未删键**：用哈希表统计「不同字符数」时，字符计数减到 0 必须 `removeValue`，否则 `count` 虚高导致逻辑错误（见 7.5）。

4. **求最短窗口答案更新位置写错**：求最短必须在收缩 while **内部**更新答案，写到外面会漏掉最优解。

5. **滑动窗口误用于含负数**：见 9.5，含负数子数组和问题不可用滑窗。

6. **整数溢出**：四数之和、子数组和累加在 C/OC 中用 `int` 可能溢出，应用 `NSInteger`/`Int64`。Swift 默认 `Int` 在 64 位平台为 64 位，相对安全，但极端值仍需 `addingReportingOverflow`。

7. **链表快指针空判断顺序**：`while fast != nil && fast.next != nil`，两个条件顺序不能反（短路求值，先判 fast 非空再访问 fast.next），否则空指针崩溃。

8. **荷兰国旗换 high 后 cur 误前进**：换来的元素未检查，cur 必须停留。

### 12.2 最佳实践

1. **背熟两套模板**：可变窗口「求最长」与「求最短」模板，区分收缩条件与答案位置。
2. **优先定长数组**：字符集已知时用定长计数数组替代哈希表，性能更好。
3. **先判前提**：用滑窗前先确认单调性/有序性是否成立，含负数立即警惕。
4. **指针命名清晰**：用 `left/right`、`slow/fast`、`low/high` 等语义化命名，避免 `i/j` 混淆。
5. **Swift 字符串先转数组**：Swift String 不支持 O(1) 下标，涉及随机访问先 `Array(s)`，否则 `s[s.index(...)]` 是 O(n)。
6. **OC 用 characterAtIndex 高效**：NSString 的 `characterAtIndex:` 是 O(1)，大量访问可先 `getCharacters:range:` 拷到 buffer。

## 13. 对比辨析

### 13.1 对撞 vs 快慢 vs 滑动窗口

| 维度 | 对撞双指针 | 快慢双指针 | 滑动窗口 |
|------|----------|----------|---------|
| 方向 | 相向（头↔尾） | 同向（速度差） | 同向（右扩左缩） |
| 前提 | 有序 / 两端单调 | 速度差产生相遇 | 区间性质单调 |
| 典型 | 两数和、回文、接雨水 | 链表环、找中点、去重 | 最长/最短子串子数组 |
| 状态 | 通常无额外状态 | 通常无额外状态 | 维护窗口状态（计数/和） |

### 13.2 滑动窗口 vs 前缀和

| 维度 | 滑动窗口 | 前缀和 + 哈希 |
|------|---------|-------------|
| 适用 | 全正数 / 单调约束 | 含负数 / 任意区间和 |
| 时间 | O(n) | O(n) |
| 空间 | O(1)~O(Σ) | O(n) |
| 思路 | 动态伸缩区间 | 预处理累加和差值查找 |

### 13.3 双指针 vs 哈希表（两数之和场景）

| 维度 | 双指针 | 哈希表 |
|------|-------|-------|
| 前提 | 数组需有序 | 无需有序 |
| 时间 | O(n)（已序） / O(n log n)（需排序） | O(n) |
| 空间 | O(1) | O(n) |
| 适用 | 有序 + 返回值/多解 | 无序 + 返回下标 |

## 14. 进阶话题

### 14.1 多指针与 N 数之和的递归泛化

三数之和、四数之和可统一为「kSum」递归框架：排序后，外层递归固定一个数将问题降为 (k-1)Sum，递归基为 2Sum（双指针）。时间复杂度 O(n^(k-1))。这体现了双指针作为「最内层 2Sum 求解器」的基础地位。

### 14.2 单调队列优化的滑动窗口（窗口最值）

「滑动窗口最大值」问题（求每个长度 k 窗口的最大值）不能用简单计数，需用**单调双端队列**（Monotonic Deque）维护窗口内的候选最大值，队首始终是当前窗口最大值，整体 O(n)。这是滑动窗口与单调栈/队列结合的高级应用，常作为困难题出现。

### 14.3 双指针在字符串匹配中的延伸

双指针思想延伸到 KMP、Manacher（最长回文子串，中心扩展是对撞双指针的变体）、Z 函数等字符串算法。中心扩展法求最长回文，本质是对每个中心用对撞双指针向两侧扩展，O(n²)。

### 14.4 在流式数据 / 大数据中的应用

滑动窗口是**流处理**（Stream Processing）的核心概念：在 Flink、Kafka Streams 中，「时间窗口」「计数窗口」与算法滑动窗口同源。处理无法全部载入内存的数据流时，O(1) 空间的双指针/窗口技巧尤为关键。

### 14.5 iOS 工程中的实际应用

1. **列表去重/合并**：分页加载时合并新旧数据，用双指针归并去重，避免 O(n²) 的 `contains` 检查。
2. **手势滑动检测**：连续触摸点序列用滑动窗口平滑/采样。
3. **限流/节流**：滑动时间窗口统计单位时间内请求数（与算法滑窗同构），用于网络请求限流。
4. **音视频缓冲区管理**：环形缓冲区配合读写双指针（生产者-消费者），与快慢指针思想相通。

## 小结

双指针与滑动窗口是处理数组、字符串、链表问题的「第一性工具」。掌握三大分类（对撞、快慢、同向/滑窗）、两套滑窗模板（求最长/求最短）、以及关键的适用前提（有序、单调、连续），就能把大量看似复杂的 O(n²) 问题优雅地压缩到 O(n)。务必牢记其失效边界（含负数 → 前缀和），并通过摊还分析真正理解其线性复杂度的来源。
