## 1. 图的基本概念

### 1.1 什么是图

图 G = (V, E) 由两个集合组成：
- **顶点集 V（Vertex Set）**：图中所有节点的集合，|V| 通常记为 n
- **边集 E（Edge Set）**：图中所有连接关系的集合，|E| 通常记为 m

每条边连接两个顶点，表示它们之间存在某种关系。

### 1.2 为什么需要图

线性结构（数组、链表）只能表达一对一关系；树能表达一对多的层级关系；而图能表达**多对多**的任意关系：

| 场景 | 顶点 | 边 |
|------|------|------|
| 社交网络 | 用户 | 好友关系 |
| 地图导航 | 路口/城市 | 道路 |
| 课程依赖 | 课程 | 先修关系 |
| 网页链接 | 页面 | 超链接 |
| 状态机 | 状态 | 转换 |
| 进程调度 | 任务 | 依赖 |

### 1.3 基本术语

- **相邻（Adjacent）**：若边 (u, v) ∈ E，则 u 和 v 相邻
- **度（Degree）**：与顶点关联的边数。有向图分入度（in-degree）和出度（out-degree）
- **路径（Path）**：顶点序列 v₁→v₂→...→vₖ，其中相邻顶点间有边
- **路径长度**：路径上边的数目（无权图）或边权之和（带权图）
- **环/回路（Cycle）**：起点等于终点的路径
- **简单路径/简单环**：不重复经过顶点的路径/环
- **连通（Connected）**：无向图中 u 到 v 存在路径
- **强连通（Strongly Connected）**：有向图中 u 到 v 且 v 到 u 都存在路径
- **连通分量（Connected Component）**：极大连通子图
- **稠密图（Dense Graph）**：m ≈ n²，边很多
- **稀疏图（Sparse Graph）**：m ≈ n 或 m << n²

## 2. 图的分类

### 2.1 按方向分

| 类型 | 定义 | 边表示 | 例子 |
|------|------|--------|------|
| 无向图（Undirected） | 边没有方向，(u,v) = (v,u) | {u, v} | 好友关系、道路 |
| 有向图（Directed/Digraph） | 边有方向，(u,v) ≠ (v,u) | (u, v) | 关注关系、任务依赖 |

### 2.2 按权重分

| 类型 | 定义 | 应用 |
|------|------|------|
| 无权图（Unweighted） | 所有边等价 | 社交关系、可达性 |
| 带权图（Weighted） | 边附带数值权重 | 距离、费用、容量 |

### 2.3 特殊图

- **完全图（Complete Graph）**：任意两个顶点间都有边，n 个顶点有 n(n-1)/2 条边（无向）
- **二分图（Bipartite Graph）**：顶点可分为两个不相交集合，每条边的两端分属不同集合
- **DAG（Directed Acyclic Graph）**：有向无环图，是拓扑排序、动态规划的基础
- **树（Tree）**：连通无环的无向图，n 个顶点恰好 n-1 条边
- **森林（Forest）**：多棵不相交的树的集合

## 3. 图的存储结构

### 3.1 邻接矩阵（Adjacency Matrix）

用 n×n 的二维数组表示，matrix[i][j] = 1（或权重）表示 i 到 j 有边。

**优点**：O(1) 查询两顶点是否相邻；实现简单

**缺点**：空间 O(n²)，稀疏图浪费严重；遍历邻居需 O(n)

#### Swift 实现

```swift
/// 邻接矩阵表示的图
struct AdjacencyMatrixGraph {
    private var matrix: [[Int]]  // 0 表示无边，非零表示边权
    let vertexCount: Int
    let isDirected: Bool
    
    init(vertexCount: Int, isDirected: Bool = false) {
        self.vertexCount = vertexCount
        self.isDirected = isDirected
        self.matrix = Array(repeating: Array(repeating: 0, count: vertexCount),
                           count: vertexCount)
    }
    
    /// 添加边，weight 默认为 1（无权图）
    mutating func addEdge(from u: Int, to v: Int, weight: Int = 1) {
        guard u >= 0, u < vertexCount, v >= 0, v < vertexCount else { return }
        matrix[u][v] = weight
        if !isDirected {
            matrix[v][u] = weight
        }
    }
    
    /// 查询 u 到 v 是否有边
    func hasEdge(from u: Int, to v: Int) -> Bool {
        guard u >= 0, u < vertexCount, v >= 0, v < vertexCount else { return false }
        return matrix[u][v] != 0
    }
    
    /// 获取顶点 u 的所有邻居
    func neighbors(of u: Int) -> [Int] {
        guard u >= 0, u < vertexCount else { return [] }
        var result: [Int] = []
        for v in 0..<vertexCount {
            if matrix[u][v] != 0 {
                result.append(v)
            }
        }
        return result
    }
    
    /// 获取顶点的度（无向图）/ 出度（有向图）
    func degree(of u: Int) -> Int {
        return neighbors(of: u).count
    }
    
    /// 获取入度（有向图）
    func inDegree(of v: Int) -> Int {
        guard v >= 0, v < vertexCount else { return 0 }
        var count = 0
        for u in 0..<vertexCount {
            if matrix[u][v] != 0 {
                count += 1
            }
        }
        return count
    }
}
```

```objectivec
- (BOOL)hasEdgeFrom:(NSInteger)u to:(NSInteger)v;
- (NSArray<NSNumber *> *)neighborsOf:(NSInteger)u;
- (NSInteger)degreeOf:(NSInteger)u;
- (NSInteger)inDegreeOf:(NSInteger)v;

@end

@implementation AdjacencyMatrixGraph {
    NSInteger **_matrix;  // C 二维数组以获得 O(1) 访问性能
}

- (instancetype)initWithVertexCount:(NSInteger)count isDirected:(BOOL)directed {
    self = [super init];
    if (self) {
        _vertexCount = count;
        _isDirected = directed;
        _matrix = (NSInteger **)calloc(count, sizeof(NSInteger *));
        for (NSInteger i = 0; i < count; i++) {
            _matrix[i] = (NSInteger *)calloc(count, sizeof(NSInteger));
        }
    }
    return self;
}

- (void)dealloc {
    for (NSInteger i = 0; i < _vertexCount; i++) {
        free(_matrix[i]);
    }
    free(_matrix);
}

- (void)addEdgeFrom:(NSInteger)u to:(NSInteger)v weight:(NSInteger)weight {
    if (u < 0 || u >= _vertexCount || v < 0 || v >= _vertexCount) return;
    _matrix[u][v] = weight;
    if (!_isDirected) {
        _matrix[v][u] = weight;
    }
}

- (BOOL)hasEdgeFrom:(NSInteger)u to:(NSInteger)v {
    if (u < 0 || u >= _vertexCount || v < 0 || v >= _vertexCount) return NO;
    return _matrix[u][v] != 0;
}

- (NSArray<NSNumber *> *)neighborsOf:(NSInteger)u {
    if (u < 0 || u >= _vertexCount) return @[];
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    for (NSInteger v = 0; v < _vertexCount; v++) {
        if (_matrix[u][v] != 0) {
            [result addObject:@(v)];
        }
    }
    return [result copy];
}

- (NSInteger)degreeOf:(NSInteger)u {
    return [self neighborsOf:u].count;
}

- (NSInteger)inDegreeOf:(NSInteger)v {
    if (v < 0 || v >= _vertexCount) return 0;
    NSInteger count = 0;
    for (NSInteger u = 0; u < _vertexCount; u++) {
        if (_matrix[u][v] != 0) {
            count++;
        }
    }
    return count;
}

@end
```

### 3.2 邻接表（Adjacency List）

每个顶点维护一个链表/数组，存储其所有邻居。这是**算法面试和工程实践中最常用**的表示方式。

**优点**：空间 O(V+E)，稀疏图高效；遍历邻居只需 O(degree(v))

**缺点**：查询特定边 O(degree(v))；不如矩阵直观

#### Swift 实现

```swift
/// 邻接表表示的图（最常用）
struct AdjacencyListGraph {
    /// 邻居信息：目标顶点 + 边权
    struct Edge {
        let to: Int
        let weight: Int
        
        init(to: Int, weight: Int = 1) {
            self.to = to
            self.weight = weight
        }
    }
    
    private var adjacencyList: [[Edge]]
    let vertexCount: Int
    let isDirected: Bool
    
    init(vertexCount: Int, isDirected: Bool = false) {
        self.vertexCount = vertexCount
        self.isDirected = isDirected
        self.adjacencyList = Array(repeating: [], count: vertexCount)
    }
    
    /// 添加边
    mutating func addEdge(from u: Int, to v: Int, weight: Int = 1) {
        guard u >= 0, u < vertexCount, v >= 0, v < vertexCount else { return }
        adjacencyList[u].append(Edge(to: v, weight: weight))
        if !isDirected {
            adjacencyList[v].append(Edge(to: u, weight: weight))
        }
    }
    
    /// 获取顶点的所有出边
    func edges(from u: Int) -> [Edge] {
        guard u >= 0, u < vertexCount else { return [] }
        return adjacencyList[u]
    }
    
    /// 获取顶点的所有邻居顶点编号
    func neighbors(of u: Int) -> [Int] {
        return edges(from: u).map { $0.to }
    }
    
    /// 出度
    func outDegree(of u: Int) -> Int {
        guard u >= 0, u < vertexCount else { return 0 }
        return adjacencyList[u].count
    }
    
    /// 边数（有向图直接统计，无向图除2）
    var edgeCount: Int {
        let total = adjacencyList.reduce(0) { $0 + $1.count }
        return isDirected ? total : total / 2
    }
}
```

```objectivec
}

}

}
```

### 3.3 边列表（Edge List）

直接存储所有边的列表 [(u, v, w), ...]。适用于 Kruskal 算法等需要按权排序边的场景。

```swift
/// 边列表表示
struct EdgeListGraph {
    struct Edge: Comparable {
        let from: Int
        let to: Int
        let weight: Int
        
        static func < (lhs: Edge, rhs: Edge) -> Bool {
            return lhs.weight < rhs.weight
        }
    }
    
    var edges: [Edge] = []
    let vertexCount: Int
    
    mutating func addEdge(from u: Int, to v: Int, weight: Int = 1) {
        edges.append(Edge(from: u, to: v, weight: weight))
    }
    
    /// 按权重排序（用于 Kruskal）
    func sortedEdges() -> [Edge] {
        return edges.sorted()
    }
}
```

### 3.4 存储结构对比

| 操作 | 邻接矩阵 | 邻接表 | 边列表 |
|------|----------|--------|--------|
| 空间复杂度 | O(V²) | O(V+E) | O(E) |
| 添加边 | O(1) | O(1) | O(1) |
| 查询边 (u,v) | O(1) | O(degree(u)) | O(E) |
| 遍历 u 的邻居 | O(V) | O(degree(u)) | O(E) |
| 遍历所有边 | O(V²) | O(V+E) | O(E) |
| 适用场景 | 稠密图、频繁查边 | 稀疏图（绝大多数场景） | 边排序相关算法 |

**面试建议**：默认使用邻接表，除非题目明确是稠密图。

## 4. 广度优先搜索 BFS

### 4.1 核心思想

BFS（Breadth-First Search）从源顶点出发，按「层次」逐步向外扩展——先访问距离为 1 的所有顶点，再访问距离为 2 的，以此类推。使用**队列（Queue）**作为辅助数据结构。

### 4.2 为什么需要 BFS

- **最短路径**：无权图中，BFS 首次到达某顶点时的路径就是最短路径
- **层次遍历**：需要按距离/层级处理的问题（如社交网络中的「几度好友」）
- **可达性判定**：从 s 能否到达 t

### 4.3 算法流程

```
BFS(graph, source):
    创建队列 queue，将 source 入队
    创建 visited 集合，标记 source 已访问
    while queue 非空:
        u = queue.dequeue()
        处理 u
        for each neighbor v of u:
            if v 未被访问:
                标记 v 已访问
                queue.enqueue(v)
```

**关键细节**：visited 在**入队时**标记，而非出队时，否则同一顶点可能被多次入队造成重复访问。

### 4.4 Swift 完整实现

```swift
/// BFS 遍历 - 返回遍历顺序
func bfs(graph: AdjacencyListGraph, source: Int) -> [Int] {
    var visited = Array(repeating: false, count: graph.vertexCount)
    var queue: [Int] = []  // 用数组模拟队列（面试简写），生产环境用 Deque
    var queueFront = 0
    var order: [Int] = []
    
    visited[source] = true
    queue.append(source)
    
    while queueFront < queue.count {
        let u = queue[queueFront]
        queueFront += 1
        order.append(u)
        
        for edge in graph.edges(from: u) {
            let v = edge.to
            if !visited[v] {
                visited[v] = true  // 入队时标记！
                queue.append(v)
            }
        }
    }
    
    return order
}

/// BFS 最短路径 - 返回源到各顶点的距离（-1 表示不可达）
func bfsShortestPath(graph: AdjacencyListGraph, source: Int) -> [Int] {
    var distance = Array(repeating: -1, count: graph.vertexCount)
    var queue: [Int] = []
    var queueFront = 0
    
    distance[source] = 0
    queue.append(source)
    
    while queueFront < queue.count {
        let u = queue[queueFront]
        queueFront += 1
        
        for edge in graph.edges(from: u) {
            let v = edge.to
            if distance[v] == -1 {  // 未访问
                distance[v] = distance[u] + 1
                queue.append(v)
            }
        }
    }
    
    return distance
}

/// BFS 路径恢复 - 返回源到目标的具体路径
func bfsPath(graph: AdjacencyListGraph, source: Int, target: Int) -> [Int]? {
    var visited = Array(repeating: false, count: graph.vertexCount)
    var parent = Array(repeating: -1, count: graph.vertexCount)
    var queue: [Int] = []
    var queueFront = 0
    
    visited[source] = true
    queue.append(source)
    
    while queueFront < queue.count {
        let u = queue[queueFront]
        queueFront += 1
        
        if u == target {
            // 回溯路径
            var path: [Int] = []
            var curr = target
            while curr != -1 {
                path.append(curr)
                curr = parent[curr]
            }
            return path.reversed()
        }
        
        for edge in graph.edges(from: u) {
            let v = edge.to
            if !visited[v] {
                visited[v] = true
                parent[v] = u
                queue.append(v)
            }
        }
    }
    
    return nil  // 不可达
}
```

### 4.5 Objective-C 完整实现

```objectivec
/// BFS 遍历 - 返回遍历顺序
- (NSArray<NSNumber *> *)bfsFrom:(NSInteger)source inGraph:(AdjacencyListGraph *)graph {
    NSInteger n = graph.vertexCount;
    BOOL *visited = (BOOL *)calloc(n, sizeof(BOOL));
    NSMutableArray<NSNumber *> *order = [NSMutableArray array];
    NSMutableArray<NSNumber *> *queue = [NSMutableArray array];
    
    visited[source] = YES;
    [queue addObject:@(source)];
    NSInteger front = 0;
    
    while (front < queue.count) {
        NSInteger u = queue[front].integerValue;
        front++;
        [order addObject:@(u)];
        
        for (GraphEdge *edge in [graph edgesFrom:u]) {
            NSInteger v = edge.to;
            if (!visited[v]) {
                visited[v] = YES;  // 入队时标记
                [queue addObject:@(v)];
            }
        }
    }
    
    free(visited);
    return [order copy];
}

/// BFS 最短路径 - 返回距离数组
- (NSArray<NSNumber *> *)bfsDistanceFrom:(NSInteger)source
                                 inGraph:(AdjacencyListGraph *)graph {
    NSInteger n = graph.vertexCount;
    NSMutableArray<NSNumber *> *distance = [NSMutableArray arrayWithCapacity:n];
    for (NSInteger i = 0; i < n; i++) {
        [distance addObject:@(-1)];
    }
    
    NSMutableArray<NSNumber *> *queue = [NSMutableArray array];
    distance[source] = @(0);
    [queue addObject:@(source)];
    NSInteger front = 0;
    
    while (front < queue.count) {
        NSInteger u = queue[front].integerValue;
        front++;
        
        for (GraphEdge *edge in [graph edgesFrom:u]) {
            NSInteger v = edge.to;
            if (distance[v].integerValue == -1) {
                distance[v] = @(distance[u].integerValue + 1);
                [queue addObject:@(v)];
            }
        }
    }
    
    return [distance copy];
}

/// BFS 路径恢复 - 返回从 source 到 target 的最短路径
- (NSArray<NSNumber *> *)bfsPathFrom:(NSInteger)source
                                  to:(NSInteger)target
                             inGraph:(AdjacencyListGraph *)graph {
    NSInteger n = graph.vertexCount;
    BOOL *visited = (BOOL *)calloc(n, sizeof(BOOL));
    NSInteger *parent = (NSInteger *)malloc(n * sizeof(NSInteger));
    for (NSInteger i = 0; i < n; i++) parent[i] = -1;
    
    NSMutableArray<NSNumber *> *queue = [NSMutableArray array];
    visited[source] = YES;
    [queue addObject:@(source)];
    NSInteger front = 0;
    BOOL found = NO;
    
    while (front < queue.count && !found) {
        NSInteger u = queue[front].integerValue;
        front++;
        
        if (u == target) { found = YES; break; }
        
        for (GraphEdge *edge in [graph edgesFrom:u]) {
            NSInteger v = edge.to;
            if (!visited[v]) {
                visited[v] = YES;
                parent[v] = u;
                [queue addObject:@(v)];
            }
        }
    }
    
    NSArray<NSNumber *> *result = nil;
    if (found) {
        NSMutableArray<NSNumber *> *path = [NSMutableArray array];
        NSInteger curr = target;
        while (curr != -1) {
            [path insertObject:@(curr) atIndex:0];
            curr = parent[curr];
        }
        result = [path copy];
    }
    
    free(visited);
    free(parent);
    return result;
}
```

### 4.6 BFS 的复杂度

- **时间复杂度**：O(V + E) — 每个顶点入队/出队一次，每条边检查一次
- **空间复杂度**：O(V) — visited 数组 + 队列最大容量（最坏情况下某一层包含所有顶点）

### 4.7 BFS 的变体与应用

#### 多源 BFS

从多个源点同时开始 BFS，用于求「所有点到最近源点的距离」。典型题目：LeetCode 542「01矩阵」。

```swift
/// 多源 BFS - 所有源点同时入队
/// 典型应用：求每个位置到最近 0 的距离
func multiSourceBFS(grid: [[Int]], sources: [(Int, Int)]) -> [[Int]] {
    let rows = grid.count, cols = grid[0].count
    var distance = Array(repeating: Array(repeating: -1, count: cols), count: rows)
    var queue: [(Int, Int)] = []
    var queueFront = 0
    
    // 所有源点同时入队，距离为 0
    for (r, c) in sources {
        distance[r][c] = 0
        queue.append((r, c))
    }
    
    let directions = [(0, 1), (0, -1), (1, 0), (-1, 0)]
    
    while queueFront < queue.count {
        let (r, c) = queue[queueFront]
        queueFront += 1
        
        for (dr, dc) in directions {
            let nr = r + dr, nc = c + dc
            if nr >= 0, nr < rows, nc >= 0, nc < cols, distance[nr][nc] == -1 {
                distance[nr][nc] = distance[r][c] + 1
                queue.append((nr, nc))
            }
        }
    }
    
    return distance
}
```

#### 双向 BFS

从起点和终点同时 BFS，相遇时即找到最短路径。适用于起点终点明确且图很大的场景，可将搜索空间从 O(b^d) 降至 O(b^(d/2))，其中 b 是分支因子，d 是最短路径长度。

```swift
/// 双向 BFS - 起点终点同时搜索
func bidirectionalBFS(graph: AdjacencyListGraph, source: Int, target: Int) -> Int {
    if source == target { return 0 }
    
    var frontVisited: Set<Int> = [source]
    var backVisited: Set<Int> = [target]
    var frontQueue: [Int] = [source]
    var backQueue: [Int] = [target]
    var steps = 0
    
    while !frontQueue.isEmpty && !backQueue.isEmpty {
        steps += 1
        
        // 总是扩展较小的一侧（优化策略）
        if frontQueue.count > backQueue.count {
            swap(&frontQueue, &backQueue)
            swap(&frontVisited, &backVisited)
        }
        
        var nextLevel: [Int] = []
        for u in frontQueue {
            for edge in graph.edges(from: u) {
                let v = edge.to
                if backVisited.contains(v) {
                    return steps  // 相遇，找到最短路径
                }
                if !frontVisited.contains(v) {
                    frontVisited.insert(v)
                    nextLevel.append(v)
                }
            }
        }
        frontQueue = nextLevel
    }
    
    return -1  // 不可达
}
```

#### 层级 BFS（按层处理）

很多 BFS 问题需要「一层一层」处理，每层代表一个距离增量。模板：

```swift
/// 层级 BFS 模板 - 逐层处理
func levelBFS(graph: AdjacencyListGraph, source: Int) -> [[Int]] {
    var visited = Array(repeating: false, count: graph.vertexCount)
    var currentLevel: [Int] = [source]
    var levels: [[Int]] = []
    visited[source] = true
    
    while !currentLevel.isEmpty {
        levels.append(currentLevel)
        var nextLevel: [Int] = []
        
        for u in currentLevel {
            for edge in graph.edges(from: u) {
                let v = edge.to
                if !visited[v] {
                    visited[v] = true
                    nextLevel.append(v)
                }
            }
        }
        
        currentLevel = nextLevel
    }
    
    return levels
}
```

## 5. 深度优先搜索 DFS

### 5.1 核心思想

DFS（Depth-First Search）沿着一条路径尽可能深入，直到无法继续才回溯（Backtrack）。使用**栈（Stack）**或**递归调用栈**作为辅助。

### 5.2 为什么需要 DFS

- **路径探索**：判断从 s 到 t 是否存在路径
- **连通性分析**：找出所有连通分量
- **环检测**：判断图中是否有环
- **拓扑排序**：对 DAG 进行线性排序
- **强连通分量**：Tarjan / Kosaraju 算法的基础
- **回溯搜索**：组合、排列等问题的状态空间探索

### 5.3 算法流程（递归版）

```
DFS(graph, u, visited):
    标记 u 已访问
    处理 u（前序位置）
    for each neighbor v of u:
        if v 未被访问:
            DFS(graph, v, visited)
    处理 u 的后续（后序位置）
```

### 5.4 Swift 完整实现

```swift
/// DFS 递归版 - 返回遍历顺序
func dfsRecursive(graph: AdjacencyListGraph, source: Int) -> [Int] {
    var visited = Array(repeating: false, count: graph.vertexCount)
    var order: [Int] = []
    
    func dfs(_ u: Int) {
        visited[u] = true
        order.append(u)  // 前序：进入时处理
        
        for edge in graph.edges(from: u) {
            let v = edge.to
            if !visited[v] {
                dfs(v)
            }
        }
        // 后序位置：所有子节点已处理完
    }
    
    dfs(source)
    return order
}

/// DFS 迭代版（用显式栈，避免大图递归栈溢出）
func dfsIterative(graph: AdjacencyListGraph, source: Int) -> [Int] {
    var visited = Array(repeating: false, count: graph.vertexCount)
    var stack: [Int] = [source]
    var order: [Int] = []
    
    while let u = stack.popLast() {
        if visited[u] { continue }
        visited[u] = true
        order.append(u)
        
        // 逆序入栈，保证访问顺序与递归版一致
        for edge in graph.edges(from: u).reversed() {
            let v = edge.to
            if !visited[v] {
                stack.append(v)
            }
        }
    }
    
    return order
}

/// DFS 求所有连通分量（无向图）
func connectedComponents(graph: AdjacencyListGraph) -> [[Int]] {
    var visited = Array(repeating: false, count: graph.vertexCount)
    var components: [[Int]] = []
    
    func dfs(_ u: Int, component: inout [Int]) {
        visited[u] = true
        component.append(u)
        for edge in graph.edges(from: u) {
            if !visited[edge.to] {
                dfs(edge.to, component: &component)
            }
        }
    }
    
    for v in 0..<graph.vertexCount {
        if !visited[v] {
            var component: [Int] = []
            dfs(v, component: &component)
            components.append(component)
        }
    }
    
    return components
}

/// DFS 记录时间戳（发现时间 + 完成时间）
/// 时间戳是 Tarjan、SCC、桥/割点等高级算法的基础
func dfsWithTimestamps(graph: AdjacencyListGraph)
    -> (discovery: [Int], finish: [Int]) {
    var visited = Array(repeating: false, count: graph.vertexCount)
    var discovery = Array(repeating: 0, count: graph.vertexCount)
    var finish = Array(repeating: 0, count: graph.vertexCount)
    var time = 0
    
    func dfs(_ u: Int) {
        visited[u] = true
        time += 1
        discovery[u] = time  // 发现时间
        
        for edge in graph.edges(from: u) {
            if !visited[edge.to] {
                dfs(edge.to)
            }
        }
        
        time += 1
        finish[u] = time  // 完成时间
    }
    
    for v in 0..<graph.vertexCount {
        if !visited[v] {
            dfs(v)
        }
    }
    
    return (discovery, finish)
}
```

```objectivec
             graph:(AdjacencyListGraph *)graph
           visited:(BOOL *)visited
             order:(NSMutableArray<NSNumber *> *)order {
    visited[u] = YES;
    [order addObject:@(u)];
    
    for (GraphEdge *edge in [graph edgesFrom:u]) {
        if (!visited[edge.to]) {
            [self _dfsHelper:edge.to graph:graph visited:visited order:order];
        }
    }

/// DFS 迭代版（显式栈）
- (NSArray<NSNumber *> *)dfsIterativeFrom:(NSInteger)source
                                  inGraph:(AdjacencyListGraph *)graph {
    NSInteger n = graph.vertexCount;
    BOOL *visited = (BOOL *)calloc(n, sizeof(BOOL));
    NSMutableArray<NSNumber *> *order = [NSMutableArray array];
    NSMutableArray<NSNumber *> *stack = [NSMutableArray arrayWithObject:@(source)];
    
    while (stack.count > 0) {
        NSInteger u = stack.lastObject.integerValue;
        [stack removeLastObject];
        
        if (visited[u]) continue;
        visited[u] = YES;
        [order addObject:@(u)];
        
        // 逆序入栈保证顺序一致
        NSArray<GraphEdge *> *edges = [graph edgesFrom:u];
        for (NSInteger i = edges.count - 1; i >= 0; i--) {
            GraphEdge *edge = edges[i];
            if (!visited[edge.to]) {
                [stack addObject:@(edge.to)];
            }
        }
    }
    
    free(visited);
    return [order copy];
}

/// 求连通分量
```

### 5.6 DFS 的复杂度

- **时间复杂度**：O(V + E) — 每个顶点访问一次，每条边检查一次
- **空间复杂度**：O(V) — visited 数组 + 递归栈深度（最坏 O(V)，如链式图）

### 5.7 前序与后序的意义

DFS 有两个关键处理位置：
- **前序（Pre-order）**：进入顶点时 → 适合做「发现」操作（标记、计数、记录路径）
- **后序（Post-order）**：离开顶点时（所有后代已处理完）→ 适合做「汇总」操作（拓扑排序、SCC、子树大小）

```swift
/// 前序与后序同时记录
func dfsPrePost(graph: AdjacencyListGraph) -> (preOrder: [Int], postOrder: [Int]) {
    var visited = Array(repeating: false, count: graph.vertexCount)
    var preOrder: [Int] = []
    var postOrder: [Int] = []
    
    func dfs(_ u: Int) {
        visited[u] = true
        preOrder.append(u)   // 前序位置
        
        for edge in graph.edges(from: u) {
            if !visited[edge.to] {
                dfs(edge.to)
            }
        }
        
        postOrder.append(u)  // 后序位置
    }
    
    for v in 0..<graph.vertexCount {
        if !visited[v] {
            dfs(v)
        }
    }
    
    return (preOrder, postOrder)
}
```

**重要性质**：DFS 后序的逆序就是拓扑排序（对于 DAG）。

## 6. 拓扑排序

### 6.1 什么是拓扑排序

对有向无环图（DAG）的所有顶点进行线性排序，使得对于图中每条有向边 (u, v)，u 在排序中出现在 v 之前。

### 6.2 应用场景

- 课程选修顺序（先修课 → 后修课）
- 编译依赖解析（先编译被依赖的模块）
- 任务调度（先执行前置任务）
- 事件处理顺序

### 6.3 方法一：Kahn 算法（BFS 方式）

基于入度的贪心策略：每次取出入度为 0 的顶点（没有前置依赖），将其加入结果，并将其所有后继的入度减 1。

```swift
/// Kahn 算法 - BFS 拓扑排序
/// 返回排序结果；若图有环则返回 nil
func topologicalSortKahn(graph: AdjacencyListGraph) -> [Int]? {
    let n = graph.vertexCount
    var inDegree = Array(repeating: 0, count: n)
    
    // 计算所有顶点的入度
    for u in 0..<n {
        for edge in graph.edges(from: u) {
            inDegree[edge.to] += 1
        }
    }
    
    // 入度为 0 的顶点入队
    var queue: [Int] = []
    var queueFront = 0
    for v in 0..<n {
        if inDegree[v] == 0 {
            queue.append(v)
        }
    }
    
    var result: [Int] = []
    
    while queueFront < queue.count {
        let u = queue[queueFront]
        queueFront += 1
        result.append(u)
        
        for edge in graph.edges(from: u) {
            inDegree[edge.to] -= 1
            if inDegree[edge.to] == 0 {
                queue.append(edge.to)
            }
        }
    }
    
    // 若结果数量 < n，说明图中有环
    return result.count == n ? result : nil
}
```

```objectivec
/// Kahn 算法 - BFS 拓扑排序
- (NSArray<NSNumber *> *)topologicalSortKahnInGraph:(AdjacencyListGraph *)graph {
    NSInteger n = graph.vertexCount;
    NSInteger *inDegree = (NSInteger *)calloc(n, sizeof(NSInteger));
    
    // 计算入度
    for (NSInteger u = 0; u < n; u++) {
        for (GraphEdge *edge in [graph edgesFrom:u]) {
            inDegree[edge.to]++;
        }
    }
    
    // 入度为 0 的入队
    NSMutableArray<NSNumber *> *queue = [NSMutableArray array];
    for (NSInteger v = 0; v < n; v++) {
        if (inDegree[v] == 0) {
            [queue addObject:@(v)];
        }
    }
    
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    NSInteger front = 0;
    
    while (front < queue.count) {
        NSInteger u = queue[front].integerValue;
        front++;
        [result addObject:@(u)];
        
        for (GraphEdge *edge in [graph edgesFrom:u]) {
            inDegree[edge.to]--;
            if (inDegree[edge.to] == 0) {
                [queue addObject:@(edge.to)];
            }
        }
    }
    
    free(inDegree);
    
    // 若不能排完所有顶点，说明有环
    return (result.count == n) ? [result copy] : nil;
}
```

### 6.4 方法二：DFS 后序逆序

对图进行 DFS，将顶点按完成时间逆序排列即为拓扑排序。

```swift
/// DFS 拓扑排序
/// 返回排序结果；若图有环则返回 nil
func topologicalSortDFS(graph: AdjacencyListGraph) -> [Int]? {
    let n = graph.vertexCount
    // 0: 未访问, 1: 正在访问（在递归栈中）, 2: 已完成
    var state = Array(repeating: 0, count: n)
    var postOrder: [Int] = []
    var hasCycle = false
    
    func dfs(_ u: Int) {
        if hasCycle { return }
        state[u] = 1  // 标记为正在访问
        
        for edge in graph.edges(from: u) {
            let v = edge.to
            if state[v] == 1 {
                // 遇到正在访问的顶点 → 有环！
                hasCycle = true
                return
            }
            if state[v] == 0 {
                dfs(v)
            }
        }
        
        state[u] = 2  // 标记为已完成
        postOrder.append(u)  // 后序记录
    }
    
    for v in 0..<n {
        if state[v] == 0 {
            dfs(v)
            if hasCycle { return nil }
        }
    }
    
    return postOrder.reversed()  // 后序逆序 = 拓扑序
}
```

```objectivec
/// DFS 拓扑排序
- (NSArray<NSNumber *> *)topologicalSortDFSInGraph:(AdjacencyListGraph *)graph {
    NSInteger n = graph.vertexCount;
    // 0: 未访问, 1: 正在访问, 2: 已完成
    NSInteger *state = (NSInteger *)calloc(n, sizeof(NSInteger));
    NSMutableArray<NSNumber *> *postOrder = [NSMutableArray array];
    __block BOOL hasCycle = NO;
    
    // 使用 block 实现递归
    __block void (^dfs)(NSInteger) = ^(NSInteger u) {
        if (hasCycle) return;
        state[u] = 1;
        
        for (GraphEdge *edge in [graph edgesFrom:u]) {
            NSInteger v = edge.to;
            if (state[v] == 1) {
                hasCycle = YES;
                return;
            }
            if (state[v] == 0) {
                dfs(v);
            }
        }
        
        state[u] = 2;
        [postOrder addObject:@(u)];
    };
    
    for (NSInteger v = 0; v < n; v++) {
        if (state[v] == 0) {
            dfs(v);
            if (hasCycle) {
                free(state);
                return nil;
            }
        }
    }
    
    free(state);
    
    // 后序逆序
    NSMutableArray<NSNumber *> *result = [NSMutableArray arrayWithCapacity:n];
    for (NSInteger i = postOrder.count - 1; i >= 0; i--) {
        [result addObject:postOrder[i]];
    }
    return [result copy];
}
```

### 6.5 两种方法对比

| 特性 | Kahn (BFS) | DFS 后序逆序 |
|------|-----------|-------------|
| 环检测 | 结果数 < n | 遇到 state=1 的邻居 |
| 是否唯一 | 可用优先队列得到字典序最小 | 难以控制顺序 |
| 空间 | O(V) 入度数组 + 队列 | O(V) state 数组 + 递归栈 |
| 适用场景 | 需要字典序、层级信息 | 需要同时做其他 DFS 操作 |

## 7. 连通性分析

### 7.1 无向图连通分量

使用 DFS/BFS 从每个未访问顶点出发，遍历到的所有顶点构成一个连通分量。

```swift
/// 无向图连通分量 - 返回每个顶点所属的分量编号
func labelComponents(graph: AdjacencyListGraph) -> (labels: [Int], count: Int) {

            componentCount += 1
        }
    }

}
```

### 7.2 有向图强连通分量（SCC）

强连通分量（Strongly Connected Component）：有向图中的极大顶点子集，其中任意两顶点可互相到达。

#### Kosaraju 算法（两遍 DFS）

1. 第一遍 DFS：记录完成时间（后序）
2. 构建反向图
3. 按完成时间逆序，对反向图做第二遍 DFS，每次 DFS 得到一个 SCC

```swift
/// Kosaraju 算法 - 求有向图强连通分量
func kosaraju(graph: AdjacencyListGraph) -> [[Int]] {
    let n = graph.vertexCount
    var visited = Array(repeating: false, count: n)
    var finishOrder: [Int] = []  // 后序
    
    // 第一遍 DFS，记录完成顺序
    func dfs1(_ u: Int) {
        visited[u] = true
        for edge in graph.edges(from: u) {
            if !visited[edge.to] {
                dfs1(edge.to)
            }
        }
        finishOrder.append(u)
    }
    
    for v in 0..<n {
        if !visited[v] { dfs1(v) }
    }
    
    // 构建反向图
    var reverseGraph = AdjacencyListGraph(vertexCount: n, isDirected: true)
    for u in 0..<n {
        for edge in graph.edges(from: u) {
            reverseGraph.addEdge(from: edge.to, to: u)
        }
    }
    
    // 第二遍 DFS，按完成时间逆序遍历反向图
    visited = Array(repeating: false, count: n)
    var components: [[Int]] = []
    
    func dfs2(_ u: Int, component: inout [Int]) {
        visited[u] = true
        component.append(u)
        for edge in reverseGraph.edges(from: u) {
            if !visited[edge.to] {
                dfs2(edge.to, component: &component)
            }
        }
    }
    
    for v in finishOrder.reversed() {
        if !visited[v] {
            var component: [Int] = []
            dfs2(v, component: &component)
            components.append(component)
        }
    }
    
    return components
}
```

```objectivec
    // 构建反向图
    AdjacencyListGraph *reverseGraph =
        [[AdjacencyListGraph alloc] initWithVertexCount:n isDirected:YES];
    for (NSInteger u = 0; u < n; u++) {
        for (GraphEdge *edge in [graph edgesFrom:u]) {
            [reverseGraph addEdgeFrom:edge.to to:u weight:1];
        }
    }
    
    // 第二遍 DFS
    memset(visited, 0, n * sizeof(BOOL));
    NSMutableArray<NSArray<NSNumber *> *> *components = [NSMutableArray array];
    
    __block void (^dfs2)(NSInteger, NSMutableArray *);
    dfs2 = ^(NSInteger u, NSMutableArray *component) {
        visited[u] = YES;
        [component addObject:@(u)];
        for (GraphEdge *edge in [reverseGraph edgesFrom:u]) {
            if (!visited[edge.to]) {
                dfs2(edge.to, component);
            }
        }
    };
    
    for (NSInteger i = finishOrder.count - 1; i >= 0; i--) {
        NSInteger v = finishOrder[i].integerValue;
        if (!visited[v]) {
            NSMutableArray<NSNumber *> *component = [NSMutableArray array];
            dfs2(v, component);
            [components addObject:[component copy]];
        }
    }
    
    free(visited);
    return [components copy];
}
```

### 7.3 Tarjan 算法（一遍 DFS 求 SCC）

Tarjan 算法只需一遍 DFS，用 dfn（发现时间）和 low（能到达的最小发现时间）两个数组配合栈来识别 SCC。

```swift
/// Tarjan 算法 - 一遍 DFS 求强连通分量
func tarjanSCC(graph: AdjacencyListGraph) -> [[Int]] {
    let n = graph.vertexCount
    var dfn = Array(repeating: 0, count: n)    // 发现时间
    var low = Array(repeating: 0, count: n)    // 能到达的最小 dfn
    var onStack = Array(repeating: false, count: n)
    var visited = Array(repeating: false, count: n)
    var stack: [Int] = []
    var timer = 0
    var components: [[Int]] = []
    
    func dfs(_ u: Int) {
        timer += 1
        dfn[u] = timer
        low[u] = timer
        visited[u] = true
        onStack[u] = true
        stack.append(u)
        
        for edge in graph.edges(from: u) {
            let v = edge.to
            if !visited[v] {
                dfs(v)
                low[u] = min(low[u], low[v])
            } else if onStack[v] {
                // v 在栈中，说明 u 能通过回边到达 v
                low[u] = min(low[u], dfn[v])
            }
        }
        
        // 若 u 是 SCC 的根（low[u] == dfn[u]）
        if low[u] == dfn[u] {
            var component: [Int] = []
            while true {
                let v = stack.removeLast()
                onStack[v] = false
                component.append(v)
                if v == u { break }
            }
            components.append(component)
        }
    }
    
    for v in 0..<n {
        if !visited[v] {
            dfs(v)
        }
    }
    
    return components
}
```

## 8. 二分图判定

### 8.1 什么是二分图

二分图（Bipartite Graph）：图的顶点可以被分为两个不相交集合 U 和 V，使得每条边的两个端点分别属于 U 和 V。

**等价条件**：图中不存在奇数长度的环。

### 8.2 判定方法：染色法

用 BFS 或 DFS 进行二着色。若能成功给所有顶点着两种颜色（相邻顶点颜色不同），则是二分图。

```swift
/// 二分图判定 - BFS 染色法
/// 返回 (是否为二分图, 各顶点颜色)
func isBipartite(graph: AdjacencyListGraph) -> (Bool, [Int]) {
    let n = graph.vertexCount
    var color = Array(repeating: -1, count: n)  // -1: 未着色, 0/1: 两种颜色
    
    for start in 0..<n {
        if color[start] != -1 { continue }  // 已着色
        
        // BFS 染色
        var queue: [Int] = [start]
        var front = 0
        color[start] = 0
        
        while front < queue.count {
            let u = queue[front]
            front += 1
            
            for edge in graph.edges(from: u) {
                let v = edge.to
                if color[v] == -1 {
                    color[v] = 1 - color[u]  // 染对立色
                    queue.append(v)
                } else if color[v] == color[u] {
                    return (false, color)  // 相邻同色 → 非二分图
                }
            }
        }
    }
    
    return (true, color)
}

/// 二分图判定 - DFS 染色法
func isBipartiteDFS(graph: AdjacencyListGraph) -> Bool {
    let n = graph.vertexCount
    var color = Array(repeating: -1, count: n)
    
    func dfs(_ u: Int, c: Int) -> Bool {
        color[u] = c
        for edge in graph.edges(from: u) {
            let v = edge.to
            if color[v] == -1 {
                if !dfs(v, c: 1 - c) { return false }
            } else if color[v] == c {
                return false  // 冲突
            }
        }
        return true
    }
    
    for v in 0..<n {
        if color[v] == -1 {
            if !dfs(v, c: 0) { return false }
        }
    }
    return true
}
```

```objectivec

        color[start] = 0;

                    color[v] = 1 - color[u];
                    [queue addObject:@(v)];
                } else if (color[v] == color[u]) {
                    result = NO;  // 相邻同色
                }
            }
        }
    }
    
    free(color);
    return result;
}
```

## 9. 环检测

### 9.1 无向图环检测

无向图中，DFS 遇到已访问且不是当前节点父节点的邻居，即存在环。

```swift
/// 无向图环检测 - DFS
func hasCycleUndirected(graph: AdjacencyListGraph) -> Bool {
    let n = graph.vertexCount
    var visited = Array(repeating: false, count: n)
    
    func dfs(_ u: Int, parent: Int) -> Bool {
        visited[u] = true
        
        for edge in graph.edges(from: u) {
            let v = edge.to
            if !visited[v] {
                if dfs(v, parent: u) { return true }
            } else if v != parent {
                // 访问到已访问的非父顶点 → 有环
                return true
            }
        }
        return false
    }
    
    for v in 0..<n {
        if !visited[v] {
            if dfs(v, parent: -1) { return true }
        }
    }
    return false
}
```

### 9.2 有向图环检测

有向图中使用三色标记法：白色（未访问）、灰色（正在访问/在递归栈中）、黑色（已完成）。遇到灰色节点说明有环。

```swift
/// 有向图环检测 - 三色标记 DFS
/// 返回 (是否有环, 环路径)
func detectCycleDirected(graph: AdjacencyListGraph) -> (hasCycle: Bool, cycle: [Int]) {
    let n = graph.vertexCount
    // 0: 白色(未访问), 1: 灰色(正在访问), 2: 黑色(已完成)
    var state = Array(repeating: 0, count: n)
    var parent = Array(repeating: -1, count: n)
    var cycleStart = -1
    var cycleEnd = -1
    
    func dfs(_ u: Int) -> Bool {
        state[u] = 1  // 灰色
        
        for edge in graph.edges(from: u) {
            let v = edge.to
            if state[v] == 1 {
                // 遇到灰色 → 有环
                cycleStart = v
                cycleEnd = u
                return true
            }
            if state[v] == 0 {
                parent[v] = u
                if dfs(v) { return true }
            }
        }
        
        state[u] = 2  // 黑色
        return false
    }
    
    for v in 0..<n {
        if state[v] == 0 {
            if dfs(v) {
                // 恢复环路径
                var cycle: [Int] = [cycleStart]
                var curr = cycleEnd
                while curr != cycleStart {
                    cycle.append(curr)
                    curr = parent[curr]
                }
                cycle.append(cycleStart)
                return (true, cycle.reversed())
            }
        }
    }
    
    return (false, [])
}
```

```objectivec
/// 有向图环检测
- (BOOL)hasCycleInDirectedGraph:(AdjacencyListGraph *)graph {
    NSInteger n = graph.vertexCount;
    NSInteger *state = (NSInteger *)calloc(n, sizeof(NSInteger));
    // 0: 白色, 1: 灰色, 2: 黑色

    __block void (^dfs)(NSInteger);
    dfs = ^(NSInteger u) {
        if (hasCycle) return;
        state[u] = 1;
        
        for (GraphEdge *edge in [graph edgesFrom:u]) {
            NSInteger v = edge.to;
            if (state[v] == 1) {
                hasCycle = YES;
                return;
            }
            if (state[v] == 0) {
                dfs(v);
            }
        }
        
        state[u] = 2;
    };
    
    for (NSInteger v = 0; v < n; v++) {
        if (state[v] == 0 && !hasCycle) {
            dfs(v);
        }
    }
    
    free(state);
    return hasCycle;
}
```

### 9.3 并查集检测环（无向图）

逐条添加边，若两端点已在同一集合中，则添加该边会形成环。

```swift
/// 并查集（Union-Find）
struct UnionFind {
    private var parent: [Int]
    private var rank: [Int]
    
    init(_ n: Int) {
        parent = Array(0..<n)
        rank = Array(repeating: 0, count: n)
    }
    
    mutating func find(_ x: Int) -> Int {
        if parent[x] != x {
            parent[x] = find(parent[x])  // 路径压缩
        }
        return parent[x]
    }
    
    mutating func union(_ x: Int, _ y: Int) -> Bool {
        let px = find(x), py = find(y)
        if px == py { return false }  // 已在同一集合
        
        // 按秩合并
        if rank[px] < rank[py] {
            parent[px] = py
        } else if rank[px] > rank[py] {
            parent[py] = px
        } else {
            parent[py] = px
            rank[px] += 1
        }
        return true
    }
}

/// 并查集检测无向图是否有环
func hasCycleUnionFind(edges: [(Int, Int)], vertexCount: Int) -> Bool {
    var uf = UnionFind(vertexCount)
    for (u, v) in edges {
        if !uf.union(u, v) {
            return true  // u 和 v 已连通，再连会成环
        }
    }
    return false
}
```

## 10. 复杂度分析总结

| 算法/操作 | 时间复杂度 | 空间复杂度 | 适用图类型 |
|-----------|-----------|-----------|-----------|
| BFS | O(V+E) | O(V) | 所有图 |
| DFS | O(V+E) | O(V) | 所有图 |
| 拓扑排序 (Kahn) | O(V+E) | O(V) | DAG |
| 拓扑排序 (DFS) | O(V+E) | O(V) | DAG |
| 连通分量 (DFS/BFS) | O(V+E) | O(V) | 无向图 |
| SCC - Kosaraju | O(V+E) | O(V+E) | 有向图 |
| SCC - Tarjan | O(V+E) | O(V) | 有向图 |
| 二分图判定 | O(V+E) | O(V) | 无向图 |
| 环检测 (DFS) | O(V+E) | O(V) | 有向/无向 |
| 环检测 (并查集) | O(E·α(V)) ≈ O(E) | O(V) | 无向图 |

**注意**：
- V 和 E 分别表示顶点数和边数
- 邻接表存储下上述复杂度成立；邻接矩阵存储则遍历操作变为 O(V²)
- α(V) 是反阿克曼函数，增长极慢，可视为常数

## 11. 典型例题与实战

### 例题 1：岛屿数量（LeetCode 200）

**题目**：给定一个由 '1'（陆地）和 '0'（水）组成的二维网格，计算岛屿数量。

**思路**：将网格视为图，每个格子是顶点，相邻陆地格子间有边。岛屿数 = 陆地连通分量数。

```swift
/// LeetCode 200 - 岛屿数量
func numIslands(_ grid: [[Character]]) -> Int {
    guard !grid.isEmpty else { return 0 }
    let rows = grid.count, cols = grid[0].count
    var visited = Array(repeating: Array(repeating: false, count: cols), count: rows)
    var count = 0
    
    let directions = [(0, 1), (0, -1), (1, 0), (-1, 0)]
    
    func dfs(_ r: Int, _ c: Int) {
        visited[r][c] = true
        for (dr, dc) in directions {
            let nr = r + dr, nc = c + dc
            if nr >= 0, nr < rows, nc >= 0, nc < cols,
               !visited[nr][nc], grid[nr][nc] == "1" {
                dfs(nr, nc)
            }
        }
    }
    
    for r in 0..<rows {
        for c in 0..<cols {
            if grid[r][c] == "1" && !visited[r][c] {
                dfs(r, c)
                count += 1
            }
        }
    }
    
    return count
}
```

```objectivec
/// LeetCode 200 - 岛屿数量（OC 版）
- (NSInteger)numIslands:(NSArray<NSArray<NSString *> *> *)grid {
    if (grid.count == 0) return 0;
    NSInteger rows = grid.count;
    NSInteger cols = grid[0].count;
    BOOL **visited = (BOOL **)calloc(rows, sizeof(BOOL *));
    for (NSInteger i = 0; i < rows; i++) {
        visited[i] = (BOOL *)calloc(cols, sizeof(BOOL));
    }
    
    NSInteger count = 0;
    NSInteger dx[] = {0, 0, 1, -1};
    NSInteger dy[] = {1, -1, 0, 0};
    
    // 使用 BFS 避免深度递归
    for (NSInteger r = 0; r < rows; r++) {
        for (NSInteger c = 0; c < cols; c++) {
            if ([grid[r][c] isEqualToString:@"1"] && !visited[r][c]) {
                count++;
                // BFS
                NSMutableArray *queue = [NSMutableArray array];
                [queue addObject:@[@(r), @(c)]];
                visited[r][c] = YES;
                NSInteger front = 0;
                
                while (front < queue.count) {
                    NSArray *pos = queue[front];
                    front++;
                    NSInteger cr = [pos[0] integerValue];
                    NSInteger cc = [pos[1] integerValue];
                    
                    for (NSInteger d = 0; d < 4; d++) {
                        NSInteger nr = cr + dx[d];
                        NSInteger nc = cc + dy[d];
                        if (nr >= 0 && nr < rows && nc >= 0 && nc < cols
                            && !visited[nr][nc]
                            && [grid[nr][nc] isEqualToString:@"1"]) {
                            visited[nr][nc] = YES;
                            [queue addObject:@[@(nr), @(nc)]];
                        }
                    }
                }
            }
        }
    }
    
    for (NSInteger i = 0; i < rows; i++) free(visited[i]);
    free(visited);
    return count;
}
```

### 例题 2：课程表（LeetCode 207）

**题目**：有 n 门课程，prerequisites[i] = [a, b] 表示修 a 前必须先修 b。判断是否可能完成所有课程。

**思路**：构建有向图，判断是否有环。无环（DAG）则可以完成，有环则不可以。

```swift
/// LeetCode 207 - 课程表（能否完成所有课程）
func canFinish(_ numCourses: Int, _ prerequisites: [[Int]]) -> Bool {
    // 建图
    var graph = AdjacencyListGraph(vertexCount: numCourses, isDirected: true)
    for pre in prerequisites {
        graph.addEdge(from: pre[1], to: pre[0])  // b → a
    }
    
    // 拓扑排序（Kahn），看能否排完所有课程
    var inDegree = Array(repeating: 0, count: numCourses)
    for u in 0..<numCourses {
        for edge in graph.edges(from: u) {
            inDegree[edge.to] += 1
        }
    }
    
    var queue: [Int] = []
    var front = 0
    for v in 0..<numCourses {
        if inDegree[v] == 0 { queue.append(v) }
    }
    
    var count = 0
    while front < queue.count {
        let u = queue[front]
        front += 1
        count += 1
        
        for edge in graph.edges(from: u) {
            inDegree[edge.to] -= 1
            if inDegree[edge.to] == 0 {
                queue.append(edge.to)
            }
        }
    }
    
    return count == numCourses
}
```

### 例题 3：单词接龙（LeetCode 127）

**题目**：给定 beginWord、endWord 和 wordList，找从 beginWord 到 endWord 的最短转换序列长度，每次只能改一个字母。

**思路**：每个单词是顶点，相差一个字母的单词间有边。求 BFS 最短路径。

```swift
/// LeetCode 127 - 单词接龙
func ladderLength(_ beginWord: String, _ endWord: String,
                  _ wordList: [String]) -> Int {
    var wordSet = Set(wordList)
    if !wordSet.contains(endWord) { return 0 }
    
    var queue: [String] = [beginWord]
    var front = 0
    var visited: Set<String> = [beginWord]
    var steps = 1
    
    while front < queue.count {
        let levelSize = queue.count - front
        
        for _ in 0..<levelSize {
            let word = queue[front]
            front += 1
            
            if word == endWord { return steps }
            
            // 尝试改变每个位置的每个字母
            var chars = Array(word)
            for i in 0..<chars.count {
                let original = chars[i]
                for c in "abcdefghijklmnopqrstuvwxyz" {
                    if c == original { continue }
                    chars[i] = c
                    let newWord = String(chars)
                    if wordSet.contains(newWord) && !visited.contains(newWord) {
                        visited.insert(newWord)
                        queue.append(newWord)
                    }
                }
                chars[i] = original
            }
        }
        steps += 1
    }
    
    return 0
}
```

### 例题 4：克隆图（LeetCode 133）

**题目**：给定一个无向连通图中的一个节点引用，返回该图的深拷贝。

```swift
/// LeetCode 133 - 克隆图
class GraphNode {
    var val: Int
    var neighbors: [GraphNode?]
    init(_ val: Int) {
        self.val = val
        self.neighbors = []
    }
}

func cloneGraph(_ node: GraphNode?) -> GraphNode? {
    guard let node = node else { return nil }
    
    var cloned: [Int: GraphNode] = [:]  // val → 克隆节点
    
    func dfs(_ original: GraphNode) -> GraphNode {
        if let existing = cloned[original.val] {
            return existing
        }
        
        let copy = GraphNode(original.val)
        cloned[original.val] = copy
        
        for neighbor in original.neighbors {
            if let neighbor = neighbor {
                copy.neighbors.append(dfs(neighbor))
            }
        }
        
        return copy
    }
    
    return dfs(node)
}
```

### 例题 5：网格中的最短路径（BFS + 状态压缩）

**题目**：在 m×n 网格中，从左上角到右下角，可以消除最多 k 个障碍物，求最短路径。

```swift
/// LeetCode 1293 - 网格中的最短路径（带状态的 BFS）
func shortestPath(_ grid: [[Int]], _ k: Int) -> Int {
    let rows = grid.count, cols = grid[0].count
    if rows == 1 && cols == 1 { return 0 }
    
    // 状态：(行, 列, 已消除障碍数)
    var visited = Array(repeating: Array(repeating: Array(repeating: false, count: k + 1),
                                        count: cols), count: rows)
    var queue: [(Int, Int, Int)] = [(0, 0, 0)]  // (row, col, obstacles)
    var front = 0
    visited[0][0][0] = true
    var steps = 0
    
    let directions = [(0, 1), (0, -1), (1, 0), (-1, 0)]
    
    while front < queue.count {
        let levelSize = queue.count - front
        steps += 1
        
        for _ in 0..<levelSize {
            let (r, c, obs) = queue[front]
            front += 1
            
            for (dr, dc) in directions {
                let nr = r + dr, nc = c + dc
                guard nr >= 0, nr < rows, nc >= 0, nc < cols else { continue }
                
                let newObs = obs + grid[nr][nc]
                if newObs > k { continue }  // 超过消除限制
                
                if nr == rows - 1 && nc == cols - 1 { return steps }
                
                if !visited[nr][nc][newObs] {
                    visited[nr][nc][newObs] = true
                    queue.append((nr, nc, newObs))
                }
            }
        }
    }
    
    return -1
}
```

## 12. 常见面试题

### 面试题 1：判断图中是否存在从 A 到 B 的路径

**答案要点**：从 A 出发做 BFS 或 DFS，如果能访问到 B 则存在路径。时间 O(V+E)。

### 面试题 2：BFS 和 DFS 的区别？什么时候用哪个？

**答案要点**：

| 维度 | BFS | DFS |
|------|-----|-----|
| 数据结构 | 队列 | 栈/递归 |
| 搜索策略 | 逐层扩展 | 深入回溯 |
| 最短路径 | 天然保证（无权图） | 不保证 |
| 空间 | O(分支因子^深度) 最坏情况 | O(深度) |
| 适用场景 | 最短路径、层序遍历 | 连通性、环检测、拓扑排序 |

### 面试题 3：如何检测有向图中的环？

**答案要点**：
1. DFS 三色标记法：白(未访问)、灰(正在递归栈中)、黑(已完成)。遇到灰色说明有环。
2. 拓扑排序（Kahn）：如果无法排完所有顶点则有环。
3. 时间复杂度均为 O(V+E)。

### 面试题 4：拓扑排序的两种实现及区别？

**答案要点**：
- Kahn（BFS）：基于入度，直观，方便获取字典序最小拓扑序（用最小堆）
- DFS 后序逆序：一遍搞定，代码简短，方便同时做 SCC 检测
- 两者时间复杂度相同 O(V+E)

### 面试题 5：什么是强连通分量？如何求？

**答案要点**：
- SCC 是有向图中任意两点可互达的极大子图
- Kosaraju：两遍 DFS（正向记录完成序、反向图按逆序 DFS）
- Tarjan：一遍 DFS + dfn/low 数组 + 栈，更高效
- 应用：2-SAT 问题、死锁检测、缩点

### 面试题 6：如何判断二分图？

**答案要点**：BFS/DFS 染色法。用两种颜色交替给顶点着色，若出现相邻同色则不是二分图。等价于判断图中是否有奇数环。

### 面试题 7：图的遍历为什么需要 visited 数组？能否省略？

**答案要点**：
- 图可能有环，不标记 visited 会死循环
- 即使无环（DAG），不标记也会重复访问（一个顶点可能有多个入边）
- 树是唯一可以不用 visited 的特殊图（无环且路径唯一），但通常用 parent 变量替代

### 面试题 8：网格/矩阵类问题为什么能用图算法？

**答案要点**：
- 网格本质是隐式图：每个格子是顶点，上下左右相邻且满足条件的格子间有边
- 无需显式建图，直接用 directions 数组枚举邻居
- 常见问题：岛屿数量、最短路径、连通区域面积

## 13. 易错点与最佳实践

### 13.1 常见陷阱

**陷阱 1：BFS visited 标记时机错误**

```swift
// ❌ 错误：出队时标记
while !queue.isEmpty {
    let u = queue.removeFirst()
    visited[u] = true  // 太晚了！可能已被重复入队
    ...
}

// ✅ 正确：入队时标记
if !visited[v] {
    visited[v] = true  // 入队即标记
    queue.append(v)
}
```

**陷阱 2：无向图 DFS 环检测忘排除父节点**

```swift
// ❌ 错误：把到达父节点误判为有环
if visited[v] { return true }  // 无向图中 v 可能就是 parent

// ✅ 正确：排除 parent
if visited[v] && v != parent { return true }
```

**陷阱 3：有向图环检测只用 visited（二值）**

```swift
// ❌ 错误：有向图中 visited=true 不一定代表有环（可能从其他路径到达）
if visited[v] { hasCycle = true }

// ✅ 正确：用三色（白/灰/黑），只有灰色才代表有环
if state[v] == 1 { hasCycle = true }  // 灰色 = 正在当前递归路径上
```

**陷阱 4：拓扑排序忘记处理非连通图**

```swift
// ❌ 错误：只从一个顶点开始 DFS
dfs(0)

// ✅ 正确：遍历所有未访问顶点
for v in 0..<n {
    if !visited[v] { dfs(v) }
}
```

**陷阱 5：邻接表重复添加无向边**

```swift
// ❌ 错误：无向图 addEdge 已经添加了双向边，输入中又当作两条有向边添加
for edge in edges {
    graph.addEdge(from: edge[0], to: edge[1])  // 内部已加了反向边
    graph.addEdge(from: edge[1], to: edge[0])  // 多加了一次！
}

// ✅ 正确：确认 addEdge 的语义
graph.addEdge(from: edge[0], to: edge[1])  // 内部自动处理双向
```

### 13.2 最佳实践

1. **默认用邻接表**：除非题目是稠密图或需要 O(1) 查边，否则邻接表空间更优
2. **BFS 用于求最短路**：无权图最短路 = BFS，这是核心直觉
3. **DFS 用于连通性/环/拓扑**：凡是需要「走到底再回来」的问题用 DFS
4. **大图用迭代而非递归**：避免栈溢出（Python 默认递归限制 1000，Swift/OC 系统栈约 1MB）
5. **网格问题用 directions 数组**：`[(0,1),(0,-1),(1,0),(-1,0)]` 优雅处理四方向
6. **visited 可用 distance 数组替代**：`distance[v] == -1` 等价于 `!visited[v]`
7. **图的输入要考虑**：顶点编号是否从 0 开始、是否有自环、是否有重边

## 14. 对比辨析

### 14.1 BFS vs DFS

| 维度 | BFS | DFS |
|------|-----|-----|
| 时间复杂度 | O(V+E) | O(V+E) |
| 空间复杂度 | O(V)（队列宽度） | O(V)（栈深度） |
| 最短路径保证 | 是（无权图） | 否 |
| 实现方式 | 队列（迭代） | 递归/栈（迭代） |
| 搜索顺序 | 逐层 | 尽深 |
| 典型应用 | 最短路径、层序、传染模型 | 连通性、环、拓扑、回溯 |
| 完备性 | 总能找到最短路 | 找到的路径不一定最短 |
| 内存消耗特点 | 层宽越大消耗越多 | 路径越深消耗越多 |

### 14.2 图 vs 树

| 维度 | 树 | 图 |
|------|-----|------|
| 环 | 无环 | 可能有环 |
| 连通性 | 连通 | 不一定连通 |
| 边数 | 恰好 n-1 | 任意 |
| 父子关系 | 唯一父节点 | 无固定父子 |
| 遍历 | 不需要 visited | 必须 visited |
| 路径唯一性 | 任意两点路径唯一 | 可能多条路径 |

### 14.3 邻接矩阵 vs 邻接表

| 维度 | 邻接矩阵 | 邻接表 |
|------|----------|--------|
| 空间 | O(V²) | O(V+E) |
| 查边 | O(1) | O(degree) |
| 遍历邻居 | O(V) | O(degree) |
| 适用 | 稠密图、Floyd | 稀疏图（大多数场景） |
| 缓存友好性 | 矩阵行连续存储 | 链表/动态数组 |

### 14.4 Kahn vs DFS 拓扑排序

| 维度 | Kahn (BFS) | DFS 后序逆序 |
|------|-----------|-------------|
| 思想 | 贪心剥离入度=0的层 | 后序 = 依赖都处理完了 |
| 环检测 | 结果数 < n | 遇到灰色节点 |
| 字典序控制 | 用优先队列可获取最小字典序 | 难以控制 |
| 代码复杂度 | 略长（需算入度） | 简短 |
| 并行度信息 | 天然获得（同一层可并行） | 不容易获取 |

## 附录：完整的图算法模板（面试速查）

### BFS 模板

```swift
func bfsTemplate(n: Int, adj: [[Int]], source: Int) -> [Int] {
    var dist = Array(repeating: -1, count: n)
    var queue: [Int] = [source]
    var front = 0
    dist[source] = 0
    
    while front < queue.count {
        let u = queue[front]; front += 1
        for v in adj[u] where dist[v] == -1 {
            dist[v] = dist[u] + 1
            queue.append(v)
        }
    }
    return dist
}
```

### DFS 模板

```swift
func dfsTemplate(n: Int, adj: [[Int]]) -> [Int] {
    var visited = Array(repeating: false, count: n)
    var result: [Int] = []
    
    func dfs(_ u: Int) {
        visited[u] = true
        result.append(u)
        for v in adj[u] where !visited[v] {
            dfs(v)
        }
    }
    
    for v in 0..<n where !visited[v] { dfs(v) }
    return result
}
```

### 拓扑排序模板

```swift
func topoSort(n: Int, adj: [[Int]]) -> [Int]? {
    var inDeg = Array(repeating: 0, count: n)
    for u in 0..<n { for v in adj[u] { inDeg[v] += 1 } }
    
    var queue = (0..<n).filter { inDeg[$0] == 0 }
    var front = 0, result: [Int] = []
    
    while front < queue.count {
        let u = queue[front]; front += 1
        result.append(u)
        for v in adj[u] {
            inDeg[v] -= 1
            if inDeg[v] == 0 { queue.append(v) }
        }
    }
    return result.count == n ? result : nil
}
```

## 总结

图论基础与遍历是算法面试的重中之重，几乎所有高阶图算法（最短路径、最小生成树、网络流）都建立在 BFS/DFS 的基础之上。核心要记住：

1. **存储用邻接表**，除非明确是稠密图
2. **BFS 求最短路**（无权图），DFS 求连通性/环/拓扑排序
3. **BFS 入队时标记 visited**，不是出队时
4. **DFS 有向图环检测用三色**，无向图排除 parent
5. **拓扑排序 = BFS Kahn 或 DFS 后序逆序**，有环则无法拓扑排序
6. **网格问题是隐式图**，directions 数组 + BFS/DFS 即可

掌握了这些，图论面试题将迎刃而解。
