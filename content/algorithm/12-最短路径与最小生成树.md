## 一、概述与知识图谱

### 1.1 什么是最短路径（Shortest Path）

在加权图（Weighted Graph）中，从顶点 u 到顶点 v 的**最短路径**是指所有 u→v 路径中，边权之和最小的那一条。形式化定义：

设 G = (V, E, w)，其中 w: E → ℝ 为权函数。u 到 v 的最短路径距离为：

```
δ(u, v) = min{ w(p) | p 是从 u 到 v 的路径 }
```

其中 w(p) = Σ w(e)，对路径 p 上的每条边 e 求和。若不存在 u→v 路径，则 δ(u, v) = +∞。

**路径的概念补充**：

- **路径（Path）**：从 u 到 v 的一个顶点序列 v₀, v₁, ..., vₖ，其中 v₀=u, vₖ=v，且每对相邻顶点之间有边
- **简单路径（Simple Path）**：路径中没有重复顶点
- **环/回路（Cycle）**：起点和终点相同的路径
- **负权环（Negative Cycle）**：边权之和为负的环

### 1.2 什么是最小生成树（Minimum Spanning Tree, MST）

对于连通无向加权图 G = (V, E, w)，其**生成树**（Spanning Tree）是包含所有顶点 V 且恰好有 |V|-1 条边的无环连通子图。**最小生成树**是所有生成树中边权之和最小的那棵。

```
MST(G) = argmin_{T ⊆ E, |T|=|V|-1, T连通} Σ w(e), e ∈ T
```

**MST 的存在条件**：图必须是连通的。非连通图可以求每个连通分量的 MST（称为最小生成森林）。

**MST 的唯一性**：如果所有边权互不相同，则 MST 唯一。如果存在相同边权，可能存在多棵不同的 MST（但权值之和相同）。

### 1.3 为什么重要

**最短路径的应用**：

- 导航系统（Google Maps / Apple Maps）：实时路径规划
- 网络路由协议（OSPF 基于 Dijkstra，RIP 基于 Bellman-Ford）
- 社交网络：计算用户间最短社交链
- 游戏 AI：NPC 寻路（通常结合 A* 算法）
- 航班/铁路调度：最少中转或最短耗时
- 差分约束系统：利用最短路径求解不等式组

**最小生成树的应用**：

- 通信网络铺设：最少光纤/电缆连接所有城市
- 电路板布线（PCB Layout）：最短连线覆盖所有引脚
- 聚类分析：删除 MST 中最长的 k-1 条边得到 k 个聚类
- 近似算法：TSP 的 2-近似算法基于 MST
- 图像分割：基于 MST 的区域合并

## 二、核心理论基础

### 2.1 松弛操作（Relaxation）

所有最短路径算法的核心都基于**松弛**（Relaxation）操作：

```
RELAX(u, v, w):
    if dist[u] + w(u,v) < dist[v]:
        dist[v] = dist[u] + w(u,v)
        prev[v] = u
```

**直觉**：当前通过 u 到达 v 的路径比已知最短路径更短，就更新。这一操作名称来源于将"紧绷的弦松开"——dist[v] 从一个较大的上界"松弛"到一个更小的值。

### 2.2 最优子结构性质

**定理**：最短路径的子路径也是最短路径。

**证明**（反证法）：设 p = s → ... → u → ... → v 是 s 到 v 的最短路径，其中 s → ... → u 是 p 的前缀子路径。假设存在从 s 到 u 的更短路径 p'，则将 p' 与 u → ... → v 拼接得到的路径比 p 更短，与 p 是最短路径矛盾。

### 2.3 三角不等式

**定理**：对任意边 (u, v)，有 δ(s, v) ≤ δ(s, u) + w(u, v)。

**证明**：s 到 v 的最短路径不会比"s 到 u 的最短路径 + 边 (u,v)"更长（因为后者也是 s 到 v 的一条合法路径）。

### 2.4 MST 的切割性质（Cut Property）

**定理（Cut Property）**：对于图的任意一个切割（将顶点分为两个非空集合 S 和 V\S），横跨切割的最小权边一定属于某棵 MST。

**证明**：设 e = (u, v) 是横跨切割的最小权边（u ∈ S, v ∈ V\S）。设 T 是一棵不包含 e 的 MST。T 中存在一条从 u 到 v 的路径（因为 T 是生成树），该路径必然经过另一条横跨切割的边 f。将 f 替换为 e 得到新树 T'，权值 w(T') = w(T) - w(f) + w(e) ≤ w(T)（因为 w(e) ≤ w(f)）。所以 T' 也是 MST，且包含 e。

### 2.5 MST 的环性质（Cycle Property）

**定理（Cycle Property）**：对于图中任意环 C，环中权值最大的边 e（若唯一最大）不属于任何 MST。

**证明**：假设 e 属于某棵 MST T。删除 e 后 T 分为两个连通分量，形成一个切割。环 C 中必有另一条横跨此切割的边 f（f ≠ e），且 w(f) < w(e)。将 e 替换为 f 得到权值更小的生成树，矛盾。

## 三、Dijkstra 算法

### 3.1 算法原理

Dijkstra（戴克斯特拉）算法是解决**非负权图单源最短路径**的贪心算法，由 Edsger W. Dijkstra 于 1956 年提出（论文发表于 1959 年）。

**核心思想**：维护一个已确定最短路径的顶点集合 S（称为"已解决集"），每次从未确定的顶点中选出 dist 最小的加入 S，然后用它去松弛相邻顶点。

**贪心选择性质的证明**：

设当前选出的顶点为 u（dist 最小的未确定顶点），需证 dist[u] = δ(s, u)。

反证：假设存在更短的路径 P: s → ... → x → ... → u，其中 x 是 P 上第一个不在 S 中的顶点。设 y 是 P 上 x 的前驱（y ∈ S）。

1. 因为 y ∈ S，dist[y] = δ(s, y) 已正确计算
2. 对边 (y, x) 做过松弛，所以 dist[x] ≤ dist[y] + w(y, x) = δ(s, x)
3. 又因为所有边权非负：δ(s, x) ≤ δ(s, u) < dist[u]（假设 P 更短）
4. 所以 dist[x] ≤ δ(s, x) ≤ δ(s, u) < dist[u]
5. 但 u 是 dist 最小的未确定顶点，而 x 也未确定且 dist[x] < dist[u]，矛盾！

### 3.2 图解执行过程

```
示例图（无向，非负权）：

    2       1
A ----→ B ----→ D
|       ↑       ↑
|3      |1      |3
↓       |       |
C ------+-------+
    4       2

边：A-B(2), A-C(3), B-D(1), C-B(1), C-D(2)

Dijkstra(source=A):

Step 0: dist = [A:0, B:∞, C:∞, D:∞], S = {}
Step 1: 取 A(0), S={A}
        松弛 A→B: dist[B] = min(∞, 0+2) = 2
        松弛 A→C: dist[C] = min(∞, 0+3) = 3
        dist = [A:0, B:2, C:3, D:∞]
Step 2: 取 B(2), S={A,B}
        松弛 B→D: dist[D] = min(∞, 2+1) = 3
        dist = [A:0, B:2, C:3, D:3]
Step 3: 取 C(3) 或 D(3), 取 C, S={A,B,C}
        松弛 C→B: dist[B] = min(2, 3+1) = 2（不更新）
        松弛 C→D: dist[D] = min(3, 3+2) = 3（不更新）
Step 4: 取 D(3), S={A,B,C,D}
        无出边

最终：dist = [A:0, B:2, C:3, D:3]
路径：A→D 最短路为 A→B→D (权=3)
```

### 3.3 算法伪代码

```
DIJKSTRA(G, w, s):
    for each v ∈ V:
        dist[v] ← ∞
        prev[v] ← NIL
    dist[s] ← 0
    Q ← MinPriorityQueue containing all vertices
    while Q ≠ ∅:
        u ← Q.extractMin()
        for each neighbor v of u:
            if dist[u] + w(u,v) < dist[v]:
                dist[v] ← dist[u] + w(u,v)
                prev[v] ← u
                Q.decreaseKey(v, dist[v])
    return (dist, prev)

// 懒删除版本（更常用的实现）：
DIJKSTRA-LAZY(G, w, s):
    for each v ∈ V:
        dist[v] ← ∞
        prev[v] ← NIL
    dist[s] ← 0
    Q ← MinHeap
    Q.insert((0, s))
    while Q ≠ ∅:
        (d, u) ← Q.extractMin()
        if d > dist[u]: continue    // 过时条目，跳过
        for each (v, w) in adj[u]:
            if dist[u] + w < dist[v]:
                dist[v] ← dist[u] + w
                prev[v] ← u
                Q.insert((dist[v], v))  // 允许重复入堆
    return (dist, prev)
```

### 3.4 复杂度分析

**二叉堆（懒删除）实现**：

- 每条边最多导致一次 insert：O(E) 次 insert
- 每次 insert/extractMin 为 O(log(堆大小))，堆大小最大 O(E)
- extractMin 共 O(E) 次（包括被跳过的过时条目）
- 总时间：O(E log E) = O(E log V)（因为 E ≤ V²，log E ≤ 2 log V）

**标准形式**：O((V + E) log V)

**斐波那契堆实现**：

- extractMin: O(log V) 摊还，共 V 次有效提取 → O(V log V)
- decreaseKey: O(1) 摊还，共 E 次 → O(E)
- 总时间：O(V log V + E)
- 理论最优，但常数因子大，实际中通常二叉堆更快

**朴素数组实现**：

- 每次线性扫描找最小值：O(V)
- 共 V 次提取：O(V²)
- 松弛操作：O(E)
- 总时间：O(V² + E) = O(V²)
- 对于稠密图（E ≈ V²）反而比堆实现更快（常数更小）

**空间复杂度**：O(V + E)（邻接表 + dist/prev 数组 + 堆）

### 3.5 Swift 完整实现

```swift
import Foundation

// MARK: - 通用最小堆

struct MinHeap<T> {
    private var elements: [T] = []
    private let comparator: (T, T) -> Bool
    
    init(comparator: @escaping (T, T) -> Bool) {
        self.comparator = comparator
    }
    
    var isEmpty: Bool { elements.isEmpty }
    var count: Int { elements.count }
    
    mutating func insert(_ element: T) {
        elements.append(element)
        siftUp(from: elements.count - 1)
    }
    
    mutating func extractMin() -> T? {
        guard !elements.isEmpty else { return nil }
        if elements.count == 1 { return elements.removeLast() }
        let min = elements[0]
        elements[0] = elements.removeLast()
        siftDown(from: 0)
        return min
    }
    
    func peek() -> T? { elements.first }
    
    private mutating func siftUp(from index: Int) {
        var child = index
        while child > 0 {
            let parent = (child - 1) / 2
            if comparator(elements[child], elements[parent]) {
                elements.swapAt(child, parent)
                child = parent
            } else { break }
        }
    }
    
    private mutating func siftDown(from index: Int) {
        var parent = index
        let n = elements.count
        while true {
            var smallest = parent
            let left = 2 * parent + 1
            let right = 2 * parent + 2
            if left < n && comparator(elements[left], elements[smallest]) { smallest = left }
            if right < n && comparator(elements[right], elements[smallest]) { smallest = right }
            if smallest == parent { break }
            elements.swapAt(parent, smallest)
            parent = smallest
        }
    }
}

// MARK: - 带权图

struct WeightedGraph {
    let n: Int
    var adj: [[(v: Int, w: Int)]]
    
    init(_ n: Int) {
        self.n = n
        self.adj = Array(repeating: [], count: n)
    }
    
    mutating func addDirectedEdge(_ u: Int, _ v: Int, _ w: Int) {
        adj[u].append((v: v, w: w))
    }
    
    mutating func addUndirectedEdge(_ u: Int, _ v: Int, _ w: Int) {
        adj[u].append((v: v, w: w))
        adj[v].append((v: u, w: w))
    }
}

// MARK: - Dijkstra 算法

struct ShortestPathResult {
    let dist: [Int]
    let prev: [Int]
    let source: Int
    
    /// 重建从 source 到 target 的路径
    func path(to target: Int) -> [Int] {
        guard dist[target] < Int.max / 2 else { return [] }
        var result: [Int] = []
        var cur = target
        while cur != -1 {
            result.append(cur)
            cur = prev[cur]
        }
        return result.reversed()
    }
    
    /// 源到 target 的最短距离（不可达返回 nil）
    func distance(to target: Int) -> Int? {
        let d = dist[target]
        return d < Int.max / 2 ? d : nil
    }
}

/// Dijkstra 算法 - 二叉堆 + 懒删除
/// - 要求：所有边权 >= 0
/// - 时间复杂度：O((V+E) log V)
/// - 空间复杂度：O(V + E)
func dijkstra(_ graph: WeightedGraph, source: Int) -> ShortestPathResult {
    let INF = Int.max / 2
    var dist = Array(repeating: INF, count: graph.n)
    var prev = Array(repeating: -1, count: graph.n)
    dist[source] = 0
    
    var heap = MinHeap<(d: Int, v: Int)>(comparator: { $0.d < $1.d })
    heap.insert((d: 0, v: source))
    
    while let top = heap.extractMin() {
        let (d, u) = (top.d, top.v)
        if d > dist[u] { continue }  // 懒删除：过时条目
        
        for edge in graph.adj[u] {
            let newDist = dist[u] + edge.w
            if newDist < dist[edge.v] {
                dist[edge.v] = newDist
                prev[edge.v] = u
                heap.insert((d: newDist, v: edge.v))
            }
        }
    }
    
    return ShortestPathResult(dist: dist, prev: prev, source: source)
}

/// Dijkstra 朴素版 - O(V²)，适合稠密图
func dijkstraNaive(_ graph: WeightedGraph, source: Int) -> ShortestPathResult {
    let INF = Int.max / 2
    let n = graph.n
    var dist = Array(repeating: INF, count: n)
    var prev = Array(repeating: -1, count: n)
    var visited = Array(repeating: false, count: n)
    dist[source] = 0
    
    for _ in 0..<n {
        var u = -1
        for v in 0..<n {
            if !visited[v] && (u == -1 || dist[v] < dist[u]) {
                u = v
            }
        }
        if u == -1 || dist[u] == INF { break }
        visited[u] = true
        
        for edge in graph.adj[u] {
            let newDist = dist[u] + edge.w
            if newDist < dist[edge.v] {
                dist[edge.v] = newDist
                prev[edge.v] = u
            }
        }
    }
    
    return ShortestPathResult(dist: dist, prev: prev, source: source)
}
```

```objectivec
#import <Foundation/Foundation.h>

// ==================== 最小堆 ====================

@interface SPHeapNode : NSObject
@property (nonatomic, assign) NSInteger vertex;
@property (nonatomic, assign) NSInteger dist;
+ (instancetype)node:(NSInteger)v dist:(NSInteger)d;
@end

@implementation SPHeapNode
+ (instancetype)node:(NSInteger)v dist:(NSInteger)d {
    SPHeapNode *n = [SPHeapNode new];
    n.vertex = v; n.dist = d;
    return n;
}
@end

@interface SPMinHeap : NSObject
- (void)push:(SPHeapNode *)node;
- (SPHeapNode *)pop;
- (BOOL)isEmpty;
@end

@implementation SPMinHeap {
    NSMutableArray<SPHeapNode *> *_h;
}
- (instancetype)init {
    if (self = [super init]) _h = [NSMutableArray array];
    return self;
}
- (BOOL)isEmpty { return _h.count == 0; }
- (void)push:(SPHeapNode *)node {
    [_h addObject:node];
    NSInteger i = _h.count - 1;
    while (i > 0) {
        NSInteger p = (i-1)/2;
        if (_h[i].dist < _h[p].dist) {
            [_h exchangeObjectAtIndex:i withObjectAtIndex:p]; i = p;
        } else break;
    }
}
- (SPHeapNode *)pop {
    if (_h.count == 0) return nil;
    SPHeapNode *top = _h[0];
    _h[0] = _h.lastObject;
    [_h removeLastObject];
    NSInteger i = 0, n = _h.count;
    while (YES) {
        NSInteger l = 2*i+1, r = 2*i+2, s = i;
        if (l < n && _h[l].dist < _h[s].dist) s = l;
        if (r < n && _h[r].dist < _h[s].dist) s = r;
        if (s == i) break;
        [_h exchangeObjectAtIndex:i withObjectAtIndex:s];
        i = s;
    }
    return top;
}
@end

// ==================== Dijkstra ====================

@interface DijkstraResult : NSObject
@property (nonatomic, strong) NSArray<NSNumber *> *dist;
@property (nonatomic, strong) NSArray<NSNumber *> *prev;
- (NSArray<NSNumber *> *)pathTo:(NSInteger)target;
@end

@implementation DijkstraResult
- (NSArray<NSNumber *> *)pathTo:(NSInteger)target {
    if (self.dist[target].integerValue >= NSIntegerMax/2) return @[];
    NSMutableArray *path = [NSMutableArray array];
    NSInteger cur = target;
    while (cur != -1) {
        [path insertObject:@(cur) atIndex:0];
        cur = self.prev[cur].integerValue;
    }
    return [path copy];
}
@end

/// Dijkstra 算法 (OC版)
/// @param adj 邻接表 adj[u] = @[@[@(v), @(w)], ...]
/// @param n 顶点数
/// @param src 源顶点
DijkstraResult *dijkstraOC(NSArray<NSArray<NSArray<NSNumber *> *> *> *adj,
                           NSInteger n, NSInteger src) {
    const NSInteger INF = NSIntegerMax / 2;
    NSMutableArray<NSNumber *> *dist = [NSMutableArray arrayWithCapacity:n];
    NSMutableArray<NSNumber *> *prev = [NSMutableArray arrayWithCapacity:n];
    for (NSInteger i = 0; i < n; i++) {
        dist[i] = @(INF);
        prev[i] = @(-1);
    }
    dist[src] = @(0);
    
    SPMinHeap *pq = [SPMinHeap new];
    [pq push:[SPHeapNode node:src dist:0]];
    
    while (![pq isEmpty]) {
        SPHeapNode *top = [pq pop];
        NSInteger u = top.vertex, d = top.dist;
        if (d > dist[u].integerValue) continue;  // 懒删除
        
        for (NSArray<NSNumber *> *edge in adj[u]) {
            NSInteger v = edge[0].integerValue;
            NSInteger w = edge[1].integerValue;
            NSInteger nd = dist[u].integerValue + w;
            if (nd < dist[v].integerValue) {
                dist[v] = @(nd);
                prev[v] = @(u);
                [pq push:[SPHeapNode node:v dist:nd]];
            }
        }
    }
    
    DijkstraResult *r = [DijkstraResult new];
    r.dist = [dist copy];
    r.prev = [prev copy];
    return r;
}
```

### 3.7 Dijkstra 的局限性

**不能处理负权边**：

```
反例：
    A --1--> B
    A --5--> C
    B -(-10)-> C

Dijkstra 执行：
1. 取 A(0), 松弛: dist[B]=1, dist[C]=5
2. 取 B(1), 松弛: dist[C]=min(5, 1-10)=-9
   但若在某些情况下 C 已被取出并标记为已确定（dist=5），则不会再更新

关键：贪心假设"当前最小dist即最终最短距离"在负权边下不成立
```

```swift
    var dist = Array(repeating: INF, count: graph.n)
    dist[source] = 0
    var heap = MinHeap<(d: Int, v: Int)>(comparator: { $0.d < $1.d })
    heap.insert((d: 0, v: source))
    
    while let top = heap.extractMin() {
        let (d, u) = (top.d, top.v)
        if u == target { return d }  // 早停
        if d > dist[u] { continue }
        for edge in graph.adj[u] {
            let nd = dist[u] + edge.w
            if nd < dist[edge.v] {
                dist[edge.v] = nd
                heap.insert((d: nd, v: edge.v))
            }
        }
    }
    return nil
}
```

## 四、Bellman-Ford 算法

### 4.1 算法原理

Bellman-Ford 算法可以处理**含负权边**的图，并能检测**负权环**。由 Richard Bellman 和 Lester Ford Jr. 分别独立提出（1958 年）。

**核心思想**：对所有边进行 |V|-1 轮松弛。

**为什么是 V-1 轮**：在无负环图中，从源到任意顶点的最短路径最多经过 V-1 条边（经过 V 个顶点各至多一次）。第 k 轮结束后，所有经过至多 k 条边的最短路径都已正确计算。

**负环检测**：如果第 V 轮仍有松弛发生，说明存在从源可达的负权环。

### 4.2 数学证明

**定理**：设图 G 无从源 s 可达的负权环。经过 k 轮全边松弛后，对所有顶点 v，若 s→v 最短路径经过至多 k 条边，则 dist[v] = δ(s, v)。

**证明**（对 k 归纳）：

- k=0：只有 dist[s]=0=δ(s,s)，正确。
- 归纳步：设第 k-1 轮后正确。设最短路 s=v₀→v₁→...→vₖ=v（恰 k 条边）。由归纳假设 dist[vₖ₋₁]=δ(s,vₖ₋₁)。第 k 轮松弛 (vₖ₋₁,vₖ)：dist[v] ≤ dist[vₖ₋₁]+w(vₖ₋₁,v)=δ(s,v)。又 dist[v]≥δ(s,v)（松弛不低于真实值）。故 dist[v]=δ(s,v)。

### 4.3 复杂度分析

- **时间复杂度**：O(V × E)
- **空间复杂度**：O(V)
- **提前退出优化**：若某轮无更新可 break

### 4.4 Swift 实现

```swift
struct Edge {
    let u: Int, v: Int, w: Int

struct BellmanFordResult {
    let dist: [Int]
    let prev: [Int]
    let hasNegativeCycle: Bool
    
    func path(to target: Int) -> [Int] {
        guard !hasNegativeCycle, dist[target] < Int.max / 2 else { return [] }
        var result: [Int] = []
        var cur = target
        while cur != -1 { result.append(cur); cur = prev[cur] }
        return result.reversed()
    }
}

/// Bellman-Ford 算法
/// - 支持负权边，可检测负权环
/// - 时间 O(VE)，空间 O(V)
func bellmanFord(edges: [Edge], n: Int, source: Int) -> BellmanFordResult {
    let INF = Int.max / 2
    var dist = Array(repeating: INF, count: n)
    var prev = Array(repeating: -1, count: n)
    dist[source] = 0
    
    // V-1 轮松弛
    for _ in 1...(n - 1) {
        var updated = false
        for edge in edges {
            guard dist[edge.u] < INF else { continue }
            let nd = dist[edge.u] + edge.w
            if nd < dist[edge.v] {
                dist[edge.v] = nd
                prev[edge.v] = edge.u
                updated = true
            }
        }
        if !updated { break }  // 提前退出
    }
    
    // 第 V 轮检测负环
    var hasNegCycle = false
    for edge in edges {
        guard dist[edge.u] < INF else { continue }
        if dist[edge.u] + edge.w < dist[edge.v] {
            hasNegCycle = true
            break
        }
    }
    
    return BellmanFordResult(dist: dist, prev: prev, hasNegativeCycle: hasNegCycle)
}

/// 找到负权环上的所有顶点
func findNegativeCycle(edges: [Edge], n: Int, source: Int) -> [Int] {
    let INF = Int.max / 2
    var dist = Array(repeating: INF, count: n)
    var prev = Array(repeating: -1, count: n)
    dist[source] = 0
    var cycleVertex = -1
    
    for _ in 0..<n {
        cycleVertex = -1
        for edge in edges {
            guard dist[edge.u] < INF else { continue }
            if dist[edge.u] + edge.w < dist[edge.v] {
                dist[edge.v] = dist[edge.u] + edge.w
                prev[edge.v] = edge.u
                cycleVertex = edge.v
            }
        }
    }
    
    guard cycleVertex != -1 else { return [] }
    
    var v = cycleVertex
    for _ in 0..<n { v = prev[v] }
    
    var cycle: [Int] = []
    var cur = v
    repeat { cycle.append(cur); cur = prev[cur] } while cur != v
    cycle.append(v)
    cycle.reverse()
    return cycle
}
```

## 五、SPFA 算法（Shortest Path Faster Algorithm）

### 5.1 算法原理

SPFA 是 Bellman-Ford 的**队列优化**。核心观察：只有上一轮被更新的顶点才可能松弛其邻居。用队列维护"已更新顶点"，避免无效遍历。

### 5.2 复杂度

- **平均**：O(kE)，k ≈ 2
- **最坏**：O(VE)（可被精心构造的图卡到）
- 竞赛中慎用，可能被"卡 SPFA"

### 5.3 负环检测

记录入队次数 count[v]，若 count[v] >= V 则存在负环。

### 5.4 Swift 实现

```swift
/// SPFA - Bellman-Ford 队列优化
/// 平均 O(kE)，最坏 O(VE)
func spfa(_ graph: WeightedGraph, source: Int) -> BellmanFordResult {
    let INF = Int.max / 2
    let n = graph.n
    var dist = Array(repeating: INF, count: n)
    var prev = Array(repeating: -1, count: n)
    var inQueue = Array(repeating: false, count: n)
    var count = Array(repeating: 0, count: n)
    
    dist[source] = 0
    var queue: [Int] = [source]
    inQueue[source] = true
    count[source] = 1
    var head = 0
    
    while head < queue.count {
        let u = queue[head]; head += 1
        inQueue[u] = false
        
        for edge in graph.adj[u] {
            let nd = dist[u] + edge.w
            if nd < dist[edge.v] {
                dist[edge.v] = nd
                prev[edge.v] = u
                if !inQueue[edge.v] {
                    queue.append(edge.v)
                    inQueue[edge.v] = true
                    count[edge.v] += 1
                    if count[edge.v] >= n {
                        return BellmanFordResult(dist: dist, prev: prev, hasNegativeCycle: true)
                    }
                }
            }
        }
    }
    
    return BellmanFordResult(dist: dist, prev: prev, hasNegativeCycle: false)
}

    }
```

## 六、Floyd-Warshall 算法

### 6.1 算法原理

Floyd-Warshall 求解**全源最短路径**，基于动态规划。

**状态**：dp[k][i][j] = 从 i 到 j，中间顶点编号 ≤ k 的最短路径长度。

**转移**：dp[k][i][j] = min(dp[k-1][i][j], dp[k-1][i][k] + dp[k-1][k][j])

**空间优化**：原地更新，去掉第一维。

**关键**：k 必须在最外层循环。

### 6.2 复杂度

- **时间**：O(V³)
- **空间**：O(V²)
- **适用**：V ≤ 500 的稠密图

### 6.3 Swift 实现

```swift
struct FloydResult {
    let dist: [[Int]]
    let next: [[Int]]
    let hasNegativeCycle: Bool
    
    func path(from i: Int, to j: Int) -> [Int] {
        guard dist[i][j] < Int.max / 2 else { return [] }
        var result = [i]
        var cur = i
        while cur != j {
            cur = next[cur][j]
            if cur == -1 { return [] }
            result.append(cur)
        }
        return result
    }
}

/// Floyd-Warshall 全源最短路径
/// 时间 O(V³)，空间 O(V²)
func floydWarshall(_ graph: WeightedGraph) -> FloydResult {
    let n = graph.n
    let INF = Int.max / 2
    
    var dist = Array(repeating: Array(repeating: INF, count: n), count: n)
    var next = Array(repeating: Array(repeating: -1, count: n), count: n)
    
    for i in 0..<n { dist[i][i] = 0; next[i][i] = i }
    for u in 0..<n {
        for edge in graph.adj[u] {
            if edge.w < dist[u][edge.v] {
                dist[u][edge.v] = edge.w
                next[u][edge.v] = edge.v
            }
        }
    }
    
    // k 最外层
    for k in 0..<n {
        for i in 0..<n {
            guard dist[i][k] < INF else { continue }
            for j in 0..<n {
                guard dist[k][j] < INF else { continue }
                let through = dist[i][k] + dist[k][j]
                if through < dist[i][j] {
                    dist[i][j] = through
                    next[i][j] = next[i][k]
                }
            }
        }
    }
    
    var hasNegCycle = false
    for i in 0..<n { if dist[i][i] < 0 { hasNegCycle = true; break } }
    
    return FloydResult(dist: dist, next: next, hasNegativeCycle: hasNegCycle)
}

/// 传递闭包
func transitiveClosure(_ graph: WeightedGraph) -> [[Bool]] {
    let n = graph.n
    var reach = Array(repeating: Array(repeating: false, count: n), count: n)
    for i in 0..<n { reach[i][i] = true }
    for u in 0..<n { for edge in graph.adj[u] { reach[u][edge.v] = true } }
    for k in 0..<n {
        for i in 0..<n {
            for j in 0..<n {
                reach[i][j] = reach[i][j] || (reach[i][k] && reach[k][j])
            }
        }
    }
    return reach
}
```

## 七、Johnson 算法

### 7.1 算法原理

Johnson 算法求含负权边稀疏图的全源最短路。通过"重新赋权"消除负权边，再对每个顶点跑 Dijkstra。

**步骤**：

1. 添加超级源 s，s 到所有顶点连权 0 的边
2. 从 s 跑 Bellman-Ford 得 h[v] = δ(s, v)
3. 重赋权：w'(u,v) = w(u,v) + h[u] - h[v] ≥ 0
4. 对每个顶点跑 Dijkstra，结果还原：真实距离 = Dijkstra距离 - h[u] + h[v]

**时间**：O(V²logV + VE)

### 7.2 Swift 实现

```swift
/// Johnson 全源最短路径
func johnson(_ graph: WeightedGraph) -> [[Int]]? {
    let n = graph.n
    let INF = Int.max / 2
    
    // 添加超级源
    var edges: [Edge] = []
    for u in 0..<n {
        for edge in graph.adj[u] {
            edges.append(Edge(u: u, v: edge.v, w: edge.w))
        }
    }
    for v in 0..<n { edges.append(Edge(u: n, v: v, w: 0)) }
    
    let bf = bellmanFord(edges: edges, n: n + 1, source: n)
    if bf.hasNegativeCycle { return nil }
    let h = Array(bf.dist[0..<n])
    
    // 重赋权
    var reweighted = WeightedGraph(n)
    for u in 0..<n {
        for edge in graph.adj[u] {
            reweighted.addDirectedEdge(u, edge.v, edge.w + h[u] - h[edge.v])
        }
    }
    
    // 每个顶点跑 Dijkstra
    var result = Array(repeating: Array(repeating: INF, count: n), count: n)
    for u in 0..<n {
        let dj = dijkstra(reweighted, source: u)
        for v in 0..<n {
            if dj.dist[v] < INF {
                result[u][v] = dj.dist[v] - h[u] + h[v]
            }
        }
    }
    return result
}
```

## 八、Kruskal 算法（最小生成树）

### 8.1 算法原理

将所有边按权升序排列，依次尝试加入。用并查集判断是否形成环——两端在不同连通分量则加入 MST。

**正确性**：基于切割性质。

**时间**：O(E log E) = O(E log V)

### 8.2 Swift 实现

```swift
// MARK: - 并查集

struct UnionFind {
    private var parent: [Int]
    private var rank: [Int]
    private(set) var componentCount: Int
    
    init(_ n: Int) {
        parent = Array(0..<n)
        rank = Array(repeating: 0, count: n)
        componentCount = n
    }
    
    mutating func find(_ x: Int) -> Int {
        if parent[x] != x { parent[x] = find(parent[x]) }
        return parent[x]
    }
    
    @discardableResult
    mutating func union(_ x: Int, _ y: Int) -> Bool {
        let rx = find(x), ry = find(y)
        if rx == ry { return false }
        if rank[rx] < rank[ry] { parent[rx] = ry }
        else if rank[rx] > rank[ry] { parent[ry] = rx }
        else { parent[ry] = rx; rank[rx] += 1 }
        componentCount -= 1
        return true
    }
    
    mutating func connected(_ x: Int, _ y: Int) -> Bool {
        return find(x) == find(y)
    }
}

// MARK: - Kruskal

struct MSTEdge: Comparable {
    let u: Int, v: Int, w: Int
    static func < (lhs: MSTEdge, rhs: MSTEdge) -> Bool { lhs.w < rhs.w }
}

struct MSTResult {
    let edges: [MSTEdge]
    let totalWeight: Int
    let isConnected: Bool
}

/// Kruskal 最小生成树
/// 时间 O(ElogE)，空间 O(V+E)
func kruskal(edges: [MSTEdge], vertexCount: Int) -> MSTResult {
    let sorted = edges.sorted()
    var uf = UnionFind(vertexCount)
    var mstEdges: [MSTEdge] = []
    var totalWeight = 0
    
    for edge in sorted {
        if uf.union(edge.u, edge.v) {
            mstEdges.append(edge)
            totalWeight += edge.w
            if mstEdges.count == vertexCount - 1 { break }
        }
    }
    
    return MSTResult(edges: mstEdges, totalWeight: totalWeight,
                     isConnected: mstEdges.count == vertexCount - 1)
}
```

```objectivec
    NSArray *sorted = [edges sortedArrayUsingComparator:^(KruskalEdge *a, KruskalEdge *b) {
        return a.w < b.w ? NSOrderedAscending : (a.w > b.w ? NSOrderedDescending : NSOrderedSame);
    }];
    UnionFindOC *uf = [[UnionFindOC alloc] initWithSize:n];
    NSMutableArray<KruskalEdge *> *mst = [NSMutableArray array];
    for (KruskalEdge *e in sorted) {
        if ([uf unionX:e.u y:e.v]) {
            [mst addObject:e];
            if (mst.count == n - 1) break;
        }
    }
    return [mst copy];
}
```

## 九、Prim 算法（最小生成树）

### 9.1 算法原理

从单个顶点出发，不断选择"距离当前 MST 最近的"顶点加入。类似 Dijkstra 但 key[v] 是到 MST 集合的最小边权（非总路径长度）。

**正确性**：切割性质。

### 9.2 复杂度

- 二叉堆：O((V+E)logV)
- 朴素：O(V²)——稠密图首选

### 9.3 Swift 实现

```swift
/// Prim MST - 二叉堆版
func prim(_ graph: WeightedGraph, start: Int = 0) -> MSTResult {
    let n = graph.n
    let INF = Int.max / 2
    var key = Array(repeating: INF, count: n)
    var parent = Array(repeating: -1, count: n)
    var inMST = Array(repeating: false, count: n)
    key[start] = 0
    
    var heap = MinHeap<(key: Int, v: Int)>(comparator: { $0.key < $1.key })
    heap.insert((key: 0, v: start))
    var mstEdges: [MSTEdge] = []
    var totalWeight = 0
    
    while let top = heap.extractMin() {
        let (k, u) = (top.key, top.v)
        if inMST[u] { continue }
        inMST[u] = true
        totalWeight += k
        if parent[u] != -1 {
            mstEdges.append(MSTEdge(u: parent[u], v: u, w: k))
        }
        for edge in graph.adj[u] {
            if !inMST[edge.v] && edge.w < key[edge.v] {
                key[edge.v] = edge.w
                parent[edge.v] = u
                heap.insert((key: edge.w, v: edge.v))
            }
        }
    }
    
    return MSTResult(edges: mstEdges, totalWeight: totalWeight,
                     isConnected: mstEdges.count == n - 1)
}

/// Prim 朴素版 O(V²)
func primNaive(_ graph: WeightedGraph) -> MSTResult {
    let n = graph.n
    let INF = Int.max / 2
    var key = Array(repeating: INF, count: n)
    var parent = Array(repeating: -1, count: n)
    var inMST = Array(repeating: false, count: n)
    key[0] = 0
    var mstEdges: [MSTEdge] = []
    var totalWeight = 0
    
    for _ in 0..<n {
        var u = -1
        for v in 0..<n {
            if !inMST[v] && (u == -1 || key[v] < key[u]) { u = v }
        }
        guard u != -1 && key[u] < INF else { break }
        inMST[u] = true
        totalWeight += key[u]
        if parent[u] != -1 { mstEdges.append(MSTEdge(u: parent[u], v: u, w: key[u])) }
        for edge in graph.adj[u] {
            if !inMST[edge.v] && edge.w < key[edge.v] {
                key[edge.v] = edge.w; parent[edge.v] = u
            }
        }
    }
    return MSTResult(edges: mstEdges, totalWeight: totalWeight,
                     isConnected: mstEdges.count == n - 1)
}
```

```objectivec
/// Prim MST (OC) - 朴素 O(V²)
NSInteger primTotalWeightOC(NSArray<NSArray<NSArray<NSNumber *> *> *> *adj, NSInteger n) {
    const NSInteger INF = NSIntegerMax / 2;
    NSMutableArray<NSNumber *> *key = [NSMutableArray arrayWithCapacity:n];
    NSMutableArray<NSNumber *> *inMST = [NSMutableArray arrayWithCapacity:n];
    for (NSInteger i = 0; i < n; i++) { key[i] = @(INF); inMST[i] = @NO; }
    key[0] = @0;
    NSInteger totalW = 0;
    
    for (NSInteger iter = 0; iter < n; iter++) {
        NSInteger u = -1;
        for (NSInteger v = 0; v < n; v++) {
            if (!inMST[v].boolValue && (u == -1 || key[v].integerValue < key[u].integerValue))
                u = v;
        }
        if (u == -1 || key[u].integerValue >= INF) break;
        inMST[u] = @YES;
        totalW += key[u].integerValue;
        for (NSArray<NSNumber *> *edge in adj[u]) {
            NSInteger v = edge[0].integerValue, w = edge[1].integerValue;
            if (!inMST[v].boolValue && w < key[v].integerValue) key[v] = @(w);
        }
    }
    return totalW;
}
```

### 9.5 Kruskal vs Prim

| 特性 | Kruskal | Prim |
|------|---------|------|
| 思想 | 按边权排序逐条加入 | 从一点逐步扩展 |
| 数据结构 | 并查集 | 优先队列 |
| 适合 | 稀疏图、边列表 | 稠密图、邻接矩阵 |
| 多连通分量 | 自然得到生成森林 | 需每个分量起始 |

### 10.2 K 站中转最便宜航班（LeetCode 787）

最多 k 次中转的最便宜路径——BF 变体只松弛 k+1 轮。

```swift
func findCheapestPrice(_ n: Int, _ flights: [[Int]], _ src: Int,
                       _ dst: Int, _ k: Int) -> Int {
    let INF = Int.max / 2
    var dist = Array(repeating: INF, count: n)
    dist[src] = 0
    for _ in 0...k {
        let prev = dist  // 用上轮副本避免同轮传递
        for f in flights {
            if prev[f[0]] < INF {
                dist[f[1]] = min(dist[f[1]], prev[f[0]] + f[2])
            }
        }
    }
    return dist[dst] < INF ? dist[dst] : -1
}
```

### 10.4 最小体力消耗路径（LeetCode 1631）

最小化路径最大边权——变形 Dijkstra，dist 用 max 而非 sum。

```swift
func minimumEffortPath(_ heights: [[Int]]) -> Int {
    let m = heights.count, n = heights[0].count, INF = Int.max / 2
    var dist = Array(repeating: Array(repeating: INF, count: n), count: m)
    dist[0][0] = 0
    var heap = MinHeap<(e: Int, r: Int, c: Int)>(comparator: { $0.e < $1.e })
    heap.insert((e: 0, r: 0, c: 0))
    let dirs = [(0,1),(0,-1),(1,0),(-1,0)]
    while let top = heap.extractMin() {
        let (e, r, c) = (top.e, top.r, top.c)
        if r == m-1 && c == n-1 { return e }
        if e > dist[r][c] { continue }
        for (dr, dc) in dirs {
            let nr = r+dr, nc = c+dc
            guard nr >= 0 && nr < m && nc >= 0 && nc < n else { continue }
            let ne = max(dist[r][c], abs(heights[nr][nc] - heights[r][c]))
            if ne < dist[nr][nc] { dist[nr][nc] = ne; heap.insert((e: ne, r: nr, c: nc)) }
        }
    }
    return 0
}
```

### 10.5 阈值距离内邻居最少的城市（LeetCode 1334）

全源最短路 + 统计。

```swift
func findTheCity(_ n: Int, _ edges: [[Int]], _ distanceThreshold: Int) -> Int {
    let INF = Int.max / 2
    var dist = Array(repeating: Array(repeating: INF, count: n), count: n)
    for i in 0..<n { dist[i][i] = 0 }
    for e in edges { dist[e[0]][e[1]] = e[2]; dist[e[1]][e[0]] = e[2] }
    for k in 0..<n {
        for i in 0..<n { guard dist[i][k] < INF else { continue }
            for j in 0..<n { guard dist[k][j] < INF else { continue }
                dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
            }
        }
    }
    var minCnt = n, result = 0
    for i in 0..<n {
        let cnt = (0..<n).filter { $0 != i && dist[i][$0] <= distanceThreshold }.count
        if cnt <= minCnt { minCnt = cnt; result = i }
    }
    return result
}
```

## 十一、面试高频题目

### 11.1 基础概念

**Q1：Dijkstra 为什么不能处理负权边？**

A：贪心假设"当前 dist 最小的未确定顶点已是最终最短距离"在负权边下不成立。通过更远的顶点经过负权边可能得到更短路径。

**Q2：Floyd 中 k 为什么必须在最外层？**

A：dp[k][i][j] 的语义是"i→j 经过编号≤k 的中间顶点的最短路"。k 在外层确保计算时 dp[i][k] 和 dp[k][j] 已包含前 k-1 个顶点。

**Q3：Bellman-Ford 如何检测负环？**

A：第 V 轮（无负环 V-1 轮应收敛）若仍有松弛发生，说明存在负环——距离可以无限减小。

### 11.2 设计题

**Q4：如何求次短路径？**

```swift
func secondShortest(_ graph: WeightedGraph, source: Int, target: Int) -> Int {
    let INF = Int.max / 2, n = graph.n
    var dist1 = Array(repeating: INF, count: n)
    var dist2 = Array(repeating: INF, count: n)
    dist1[source] = 0
    var heap = MinHeap<(d: Int, v: Int)>(comparator: { $0.d < $1.d })
    heap.insert((d: 0, v: source))
    while let top = heap.extractMin() {
        let (d, u) = (top.d, top.v)
        if d > dist2[u] { continue }
        for edge in graph.adj[u] {
            let nd = d + edge.w
            if nd < dist1[edge.v] {
                dist2[edge.v] = dist1[edge.v]
                dist1[edge.v] = nd
                heap.insert((d: nd, v: edge.v))
            } else if nd > dist1[edge.v] && nd < dist2[edge.v] {
                dist2[edge.v] = nd
                heap.insert((d: nd, v: edge.v))
            }
        }
    }
    return dist2[target]
}
```

**Q5：差分约束系统怎么用最短路解？**

A：xᵢ - xⱼ ≤ c 转化为边 j→i 权为 c。添加超级源到所有顶点权 0，跑 Bellman-Ford。dist[v] 即 xᵥ 的一组解。有负环则无解。

**Q6：Kruskal 和 Prim 怎么选？**

A：稀疏图/边列表 → Kruskal；稠密图/邻接矩阵 → Prim 朴素版；需要生成森林 → Kruskal。

### 12.1 决策树

```
需要最短路径？
├── 单源
│   ├── 非负权 → Dijkstra（堆/朴素看稀疏/稠密）
│   ├── 边权0/1 → 0-1 BFS O(V+E)
│   └── 有负权 → Bellman-Ford / SPFA
├── 全源
│   ├── V ≤ 500 → Floyd O(V³)
│   ├── 稀疏+负权 → Johnson
│   └── 非负稀疏 → V 次 Dijkstra
└── 无权图 → BFS O(V+E)

需要 MST？
├── 稀疏/边列表 → Kruskal O(ElogE)
├── 稠密/邻接矩阵 → Prim朴素 O(V²)
└── 通用 → Prim堆 O((V+E)logV)
```

## 十三、易错点与最佳实践

### 13.1 常见错误

1. **负权图用 Dijkstra**：会得到错误结果
2. **BF 限步问题用当前轮值**：必须用上轮副本防止同轮传递
3. **Floyd 的 k 不在最外层**：结果错误
4. **整数溢出**：使用 Int.max/2 而非 Int.max
5. **非连通图忘记判断**：MST 可能不存在

### 13.2 最佳实践

1. 非负权首选 Dijkstra，懒删除比 decrease-key 实现更简单
2. INF 用 `Int.max / 2` 防溢出
3. 稠密图用朴素版（O(V²) 常数小）
4. 记得提前退出优化（Dijkstra 到目标早停，BF 无更新 break）
5. 路径重建通过 prev 数组逆向

## 十四、进阶拓展

### 14.1 A* 算法

带启发式的 Dijkstra，f(v) = g(v) + h(v)，h 为估计距离（需可采纳）。

```swift
func aStar(_ graph: WeightedGraph, source: Int, target: Int,
           h: (Int) -> Int) -> Int? {
    let INF = Int.max / 2
    var g = Array(repeating: INF, count: graph.n)
    g[source] = 0
    var heap = MinHeap<(f: Int, v: Int)>(comparator: { $0.f < $1.f })
    heap.insert((f: h(source), v: source))
    while let top = heap.extractMin() {
        let u = top.v
        if u == target { return g[target] }
        if top.f - h(u) > g[u] { continue }
        for edge in graph.adj[u] {
            let ng = g[u] + edge.w
            if ng < g[edge.v] {
                g[edge.v] = ng
                heap.insert((f: ng + h(edge.v), v: edge.v))
            }
        }
    }
    return nil
}
```

### 14.2 Borůvka 算法

```swift
func boruvka(edges: [MSTEdge], n: Int) -> Int {
    var uf = UnionFind(n)
    var totalWeight = 0, numComp = n
    while numComp > 1 {
        var cheapest = Array(repeating: -1, count: n)
        for (idx, edge) in edges.enumerated() {
            let cu = uf.find(edge.u), cv = uf.find(edge.v)
            if cu == cv { continue }
            if cheapest[cu] == -1 || edges[cheapest[cu]].w > edge.w { cheapest[cu] = idx }
            if cheapest[cv] == -1 || edges[cheapest[cv]].w > edge.w { cheapest[cv] = idx }
        }
        var merged = false
        for i in 0..<n {
            guard cheapest[i] != -1 else { continue }
            let e = edges[cheapest[i]]
            if uf.union(e.u, e.v) { totalWeight += e.w; numComp -= 1; merged = true }
        }
        if !merged { break }
    }
    return totalWeight
}
```

## 十五、总结

### 核心要点

1. **松弛操作**是所有最短路径算法的统一基础
2. **Dijkstra** 非负权图首选，贪心+堆，O((V+E)logV)
3. **Bellman-Ford** 处理负权边、检测负环，O(VE)
4. **SPFA** 是 BF 队列优化，平均快但最坏不变
5. **Floyd-Warshall** 全源最短路 DP，O(V³)
6. **Johnson** 重赋权+Dijkstra，稀疏图全源最优
7. **Kruskal** 贪心+并查集，适合稀疏图
8. **Prim** 类 Dijkstra 的 MST，稠密图用朴素版
9. MST 正确性基于**切割性质**和**环性质**
10. A*、双向搜索、分层图是工程优化方向

### 面试速记

- Dijkstra 不处理负权（贪心假设失效）
- Floyd 的 k 必须最外层（DP 状态含义）
- BF 第 V 轮检负环
- Kruskal 用并查集判环，Prim 用堆选最小切边
- MST 权值和唯一，形态可能不唯一
- 差分约束 ↔ 最短路径转化
