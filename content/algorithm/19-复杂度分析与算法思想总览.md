## 一、复杂度分析基础

### 1.1 为什么需要复杂度分析

在工程实践中，我们不可能对每个算法都在所有可能的输入上运行并计时。复杂度分析提供了一种**与具体硬件、编程语言无关**的抽象工具，让我们能够：

- 在编码之前就预判算法的可行性（对于 n = 10^6，O(n²) 约需 10^12 次操作，通常不可接受）
- 在多种方案之间做出理性选择
- 识别性能瓶颈的根本原因

### 1.2 基本概念

**输入规模（Input Size）**：问题中影响运行时间/空间的关键参数。例如数组长度 n、图的顶点数 V 和边数 E、字符串长度 m 等。

**基本操作（Elementary Operation）**：算法执行的最小计算步骤——比较、赋值、算术运算、指针/引用操作等。我们假设每个基本操作花费常数时间 O(1)。

**运行时间函数 T(n)**：算法在输入规模为 n 时执行的基本操作总数。

### 1.3 最好、最坏、平均情况

| 情况 | 含义 | 用途 |
|------|------|------|
| 最好情况（Best Case） | 所有规模为 n 的输入中，T(n) 最小的那个 | 通常参考价值不大，除非能证明大部分输入接近最好 |
| 最坏情况（Worst Case） | 所有规模为 n 的输入中，T(n) 最大的那个 | 最常用，给出性能的上界保证 |
| 平均情况（Average Case） | 在某种概率分布下，T(n) 的期望值 | 更接近实际，但分析通常更困难 |

**示例：线性搜索**

在长度为 n 的数组中搜索目标元素：
- 最好情况：目标在第一个位置，T(n) = 1 → O(1)
- 最坏情况：目标不存在或在最后，T(n) = n → O(n)
- 平均情况：假设目标等概率在任意位置或不存在，T(n) ≈ n/2 → O(n)

### 1.4 复杂度分析的意义层次

| 层次 | 关注点 | 典型应用 |
|------|--------|---------|
| 算法层 | 选择最优算法范式 | O(n²) vs O(n log n) |
| 数据结构层 | 选择支撑数据结构 | Array vs HashSet 判重 |
| 实现层 | 常数因子优化 | 缓存友好、SIMD、减少分支 |
| 系统层 | I/O、内存、并发 | 外部排序、内存映射、多线程 |

## 二、渐进记号与数学基础

### 2.1 大 O 记号（Big-O Notation）——渐进上界

**定义**：f(n) = O(g(n)) 当且仅当存在正常数 c 和 n₀，使得对所有 n ≥ n₀，有 f(n) ≤ c · g(n)。

直观含义：f(n) 的增长速度**不超过** g(n)（忽略常数因子和低阶项）。

**严格示例**：证明 3n² + 5n + 2 = O(n²)

取 c = 4, n₀ = 6：当 n ≥ 6 时，3n² + 5n + 2 ≤ 3n² + 5n² + 2n² = 10n²... 实际上取 c = 10, n₀ = 1 即可：3(1) + 5(1) + 2 = 10 ≤ 10(1) ✓

### 2.2 大 Ω 记号（Big-Omega）——渐进下界

**定义**：f(n) = Ω(g(n)) 当且仅当存在正常数 c 和 n₀，使得对所有 n ≥ n₀，有 f(n) ≥ c · g(n)。

直观含义：f(n) 的增长速度**不低于** g(n)。

**用途**：证明问题的复杂度下界。例如，基于比较的排序下界为 Ω(n log n)。

### 2.3 大 Θ 记号（Big-Theta）——渐进紧确界

**定义**：f(n) = Θ(g(n)) 当且仅当 f(n) = O(g(n)) 且 f(n) = Ω(g(n))。

即存在 c₁, c₂ > 0 和 n₀，使得对所有 n ≥ n₀：c₁·g(n) ≤ f(n) ≤ c₂·g(n)。

### 2.4 小 o 和小 ω 记号

- f(n) = o(g(n))：f(n) 的增长**严格慢于** g(n)，即 lim(n→∞) f(n)/g(n) = 0
- f(n) = ω(g(n))：f(n) 的增长**严格快于** g(n)，即 lim(n→∞) f(n)/g(n) = ∞

区别：O 允许 f 和 g 同阶（≤），o 要求严格小于（<）。

### 2.5 常见增长阶比较

从慢到快排列：

```
O(1) < O(log log n) < O(log n) < O(√n) < O(n) < O(n log n) < O(n√n) < O(n²) < O(n³) < O(2ⁿ) < O(n!) < O(nⁿ)
```

**实际感受**（n = 10^6 时，假设 1GHz 处理器每秒 10^9 次操作）：

| 复杂度 | 操作次数约 | 耗时约 |
|--------|-----------|--------|
| O(1) | 1 | 1 ns |
| O(log n) | 20 | 20 ns |
| O(√n) | 1000 | 1 μs |
| O(n) | 10^6 | 1 ms |
| O(n log n) | 2 × 10^7 | 20 ms |
| O(n²) | 10^12 | 16.7 min |
| O(n³) | 10^18 | 31.7 年 |
| O(2ⁿ) | 10^301029 | 宇宙毁灭 × 10^301020 |

### 2.6 渐进分析的运算规则

**规则 1 — 加法规则**：T(n) = T₁(n) + T₂(n) = O(max(f(n), g(n)))

若 T₁(n) = O(f(n))，T₂(n) = O(g(n))，则 T₁ + T₂ = O(max(f, g))。顺序执行的代码段，总复杂度取最慢的那段。

**规则 2 — 乘法规则**：T(n) = T₁(n) × T₂(n) = O(f(n) × g(n))

嵌套循环的典型规则。外层 O(f)，内层 O(g)，总计 O(f × g)。

**规则 3 — 常数规则**：O(c · f(n)) = O(f(n))

常数系数在渐进分析中可忽略。

**规则 4 — 多项式求和**：O(n^a + n^b) = O(n^max(a,b))

低阶项在 n 足够大时可忽略。

**规则 5 — 对数底数**：O(log_a(n)) = O(log_b(n))

因为 log_a(n) = log_b(n) / log_b(a)，底数之差只是常数因子。

## 三、时间复杂度详解

### 3.1 常见复杂度级别及典型算法

| 复杂度 | 名称 | 典型算法/操作 |
|--------|------|--------------|
| O(1) | 常数 | 数组随机访问、哈希表查找（平均）、栈 push/pop |
| O(log n) | 对数 | 二分查找、平衡 BST 操作、堆操作 |
| O(√n) | 根号 | 分块算法、质因数分解试除法 |
| O(n) | 线性 | 遍历、线性搜索、计数排序、KMP |
| O(n log n) | 线性对数 | 归并排序、快排（平均）、堆排序 |
| O(n²) | 平方 | 冒泡/插入/选择排序、朴素 DP（LIS） |
| O(n³) | 立方 | Floyd-Warshall、矩阵乘法（朴素）、区间 DP |
| O(2ⁿ) | 指数 | 子集枚举、暴力 TSP、递归斐波那契 |
| O(n!) | 阶乘 | 全排列生成、暴力旅行商 |

### 3.2 循环分析技巧

#### 单层循环

```swift
// T(n) = n → O(n)
for i in 0..<n {
    // O(1) 操作
}
```

#### 嵌套循环（独立范围）

```swift
// T(n) = n × n = n² → O(n²)
for i in 0..<n {
    for j in 0..<n {
        // O(1) 操作
    }
}
```

#### 内层依赖外层

```swift
// T(n) = Σ(i=0..n-1) i = n(n-1)/2 → O(n²)
for i in 0..<n {
    for j in 0..<i {
        // O(1) 操作
    }
}
```

#### 对数级循环

```swift
// T(n) = log₂(n) → O(log n)
var i = 1
while i < n {
    // O(1) 操作
    i *= 2  // 每次翻倍 → log₂(n) 次迭代
}
```

#### 嵌套对数循环

```swift
// T(n) = n × log(n) → O(n log n)
for i in 0..<n {        // O(n)
    var j = 1
    while j < n {       // O(log n)
        // O(1)
        j *= 2
    }
}
```

#### 调和级数循环

```swift
// T(n) = n/1 + n/2 + n/3 + ... + n/n = n·H(n) = O(n log n)
for i in 1...n {
    for j in stride(from: i, through: n, by: i) {
        // O(1) 操作
    }
}
// 典型应用：埃拉托斯特尼筛法
```

#### 根号级循环

```swift
// T(n) = √n → O(√n)
var i = 1
while i * i <= n {
    // O(1) 操作
    i += 1
}
```

取最坏情况分支：

```swift
    // O(n²) 操作
} else {
    // O(n) 操作
```

```swift
for i in 0..<n {
```

### 3.5 综合分析示例

```swift
/// 示例：计算排序数组中所有数对差的绝对值之和
/// 分析从暴力 O(n²) 优化到 O(n log n)
///
/// 暴力法：双重循环枚举所有 (i,j) 对
/// for i in 0..<n { for j in (i+1)..<n { total += abs(nums[i]-nums[j]) } }
/// → O(n²)
///
/// 优化：排序后利用前缀和
/// 排序后 nums[i] ≥ nums[j] (j < i)
/// nums[i] 与前 i 个数的差之和 = nums[i]*i - prefix[i]
func sumOfAbsoluteDifferences(_ nums: [Int]) -> Int {
    let sorted = nums.sorted()  // O(n log n)
    let n = sorted.count
    var total = 0
    var prefixSum = 0
    
    for i in 0..<n {            // O(n)
        total += sorted[i] * i - prefixSum
        prefixSum += sorted[i]
    }
    // 总时间：O(n log n)，瓶颈在排序
    // 空间：O(n)（排序用的临时空间）
    return total * 2
}
```

```objc
/// Objective-C 对照
- (NSInteger)sumOfAbsoluteDifferences:(NSArray<NSNumber *> *)nums {
    NSArray *sorted = [nums sortedArrayUsingSelector:@selector(compare:)];
    NSInteger n = sorted.count;
    NSInteger total = 0, prefixSum = 0;
    for (NSInteger i = 0; i < n; i++) {
        NSInteger val = sorted[i].integerValue;
        total += val * i - prefixSum;
        prefixSum += val;
    }
    return total * 2;
}
```

## 四、空间复杂度详解

### 4.1 定义

空间复杂度 S(n) 衡量算法运行过程中**额外**占用的存储空间（不含输入本身）。有些定义包含输入空间，需注意语境。

### 4.2 常见空间复杂度

| 复杂度 | 说明 | 示例 |
|--------|------|------|
| O(1) | 原地算法，只用常数个变量 | 冒泡排序、双指针、荷兰国旗 |
| O(log n) | 递归栈深度 log n | 二分查找递归版、快排最优情况 |
| O(n) | 线性辅助空间 | 归并排序、哈希表、BFS 队列 |
| O(n²) | 二维表 | Floyd 距离矩阵、朴素 LCS |
| O(2ⁿ) | 存储所有状态 | 状态压缩 DP |

### 4.3 递归的空间消耗

递归调用在系统栈上分配帧（存储局部变量、参数、返回地址）。递归深度 d 时栈空间 O(d)。

```swift
/// 递归阶乘：时间 O(n)，空间 O(n)（栈深度 n）
func factorial(_ n: Int) -> Int {
    guard n > 1 else { return 1 }
    return n * factorial(n - 1)
}

/// 迭代版：时间 O(n)，空间 O(1)
func factorialIterative(_ n: Int) -> Int {
    var result = 1
    for i in 2...n { result *= i }
    return result
}
```

```objc
- (NSInteger)factorial:(NSInteger)n {
    if (n <= 1) return 1;
    return n * [self factorial:n - 1];
}

- (NSInteger)factorialIterative:(NSInteger)n {
    NSInteger result = 1;
    for (NSInteger i = 2; i <= n; i++) result *= i;
    return result;
}
```

### 4.4 空间优化技巧

#### 4.4.1 滚动数组（Rolling Array）

DP 中当前行只依赖上一行时，只保留两行即可：

```swift
/// 0-1 背包：空间从 O(n×W) 优化到 O(W)
/// 关键：逆序遍历容量，保证每个物品只用一次
func knapsack(weights: [Int], values: [Int], capacity: Int) -> Int {
    let n = weights.count
    var dp = [Int](repeating: 0, count: capacity + 1)
    for i in 0..<n {
        for w in stride(from: capacity, through: weights[i], by: -1) {
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
        }
    }
    return dp[capacity]
}
```

```objc
- (NSInteger)knapsackWithWeights:(NSArray<NSNumber *> *)weights
                          values:(NSArray<NSNumber *> *)values
                        capacity:(NSInteger)capacity {
    NSInteger n = weights.count;
    NSMutableArray<NSNumber *> *dp = [NSMutableArray arrayWithCapacity:capacity + 1];
    for (NSInteger i = 0; i <= capacity; i++) [dp addObject:@0];
    
    for (NSInteger i = 0; i < n; i++) {
        NSInteger wi = weights[i].integerValue;
        NSInteger vi = values[i].integerValue;
        for (NSInteger w = capacity; w >= wi; w--) {
            NSInteger newVal = dp[w - wi].integerValue + vi;
            if (newVal > dp[w].integerValue) dp[w] = @(newVal);
        }
    }
    return dp[capacity].integerValue;
}
```

#### 4.4.2 原地算法

```swift
/// 原地反转数组：O(1) 空间
func reverseInPlace(_ arr: inout [Int]) {
    var left = 0, right = arr.count - 1
    while left < right {
        arr.swapAt(left, right)
        left += 1; right -= 1
    }
}

/// 原地移除所有零到末尾
func moveZeroes(_ arr: inout [Int]) {
    var insertPos = 0
    for i in 0..<arr.count {
        if arr[i] != 0 {
            arr.swapAt(insertPos, i)
            insertPos += 1
        }
    }
}
```

#### 4.4.3 位运算节省空间

```swift
/// 用位图代替布尔数组（空间节省 8 倍）
/// 标记 0~n-1 中哪些数出现过
struct BitSet {
    private var storage: [UInt64]
    
    init(size: Int) {
        storage = [UInt64](repeating: 0, count: (size + 63) / 64)
    }
    
    mutating func set(_ index: Int) {
        storage[index / 64] |= (1 << (index % 64))
    }
    
    func test(_ index: Int) -> Bool {
        return (storage[index / 64] & (1 << (index % 64))) != 0
    }
}
```

## 五、递归复杂度分析——主定理及其拓展

### 5.1 递归关系式

分治算法的时间复杂度通常可表示为：

```
T(n) = a · T(n/b) + f(n)
```

- a ≥ 1：子问题数量
- b > 1：规模缩小因子
- f(n)：分解 + 合并的开销

### 5.2 主定理（Master Theorem）

设 T(n) = a · T(n/b) + Θ(n^d)，令临界指数 c_crit = log_b(a)：

| 条件 | 结论 | 直觉 |
|------|------|------|
| d < c_crit（递归主导） | T(n) = Θ(n^(log_b(a))) | 叶节点总工作量主导 |
| d = c_crit（平衡） | T(n) = Θ(n^d · log n) | 每层工作量相同，log n 层 |
| d > c_crit（合并主导） | T(n) = Θ(n^d) | 根节点工作量主导 |

**更一般的形式**（带 log 因子）：

若 f(n) = Θ(n^(log_b(a)) · (log n)^k)，则 T(n) = Θ(n^(log_b(a)) · (log n)^(k+1))

### 5.3 主定理应用实例

| 算法 | 递推式 | a | b | d | c_crit | 情况 | 结果 |
|------|--------|---|---|---|--------|------|------|
| 二分查找 | T(n) = T(n/2) + O(1) | 1 | 2 | 0 | 0 | d = c_crit | O(log n) |
| 归并排序 | T(n) = 2T(n/2) + O(n) | 2 | 2 | 1 | 1 | d = c_crit | O(n log n) |
| 快速选择 | T(n) = T(n/2) + O(n) | 1 | 2 | 1 | 0 | d > c_crit | O(n) |
| Strassen | T(n) = 7T(n/2) + O(n²) | 7 | 2 | 2 | 2.807 | d < c_crit | O(n^2.807) |
| Karatsuba | T(n) = 3T(n/2) + O(n) | 3 | 2 | 1 | 1.585 | d < c_crit | O(n^1.585) |
| 二叉树遍历 | T(n) = 2T(n/2) + O(1) | 2 | 2 | 0 | 1 | d < c_crit | O(n) |

### 5.4 递归树法（Recursion Tree Method）

当主定理不适用时（如子问题大小不均），用递归树：

1. 画出调用树
2. 计算每层总工作量
3. 求所有层之和

**示例：T(n) = T(n/3) + T(2n/3) + cn**

```
                cn                    → cn
              /    \
         cn/3      2cn/3              → cn
        /   \      /    \
    cn/9  2cn/9 2cn/9  4cn/9          → cn
    ...
```

- 每层总工作量 = cn（子问题规模之和始终为 n）
- 最长路径：(2/3)^k · n = 1 → k = log_(3/2)(n)
- 最短路径：(1/3)^k · n = 1 → k = log_3(n)
- 总计介于 cn·log_3(n) 和 cn·log_(3/2)(n) 之间
- **T(n) = Θ(n log n)**

### 5.5 代换法（Substitution Method）

猜测 + 数学归纳法验证。适用于对结果有直觉但需要严格证明的情况。

**示例**：证明 T(n) = 2T(⌊n/2⌋) + n 的解为 O(n log n)

猜测 T(n) ≤ cn log n

归纳：
```
T(n) = 2T(⌊n/2⌋) + n
     ≤ 2 · c(n/2) · log(n/2) + n
     = cn · (log n - log 2) + n
     = cn·log n - cn + n
     ≤ cn·log n     （当 c ≥ 1）  ✓
```

### 5.6 非标准递推的处理

#### 线性递归

```
T(n) = T(n-1) + f(n)
```

解法：展开得 T(n) = T(0) + Σ f(i)，直接求和。

例：T(n) = T(n-1) + n → T(n) = n(n+1)/2 = O(n²)

#### 指数级递归

```
T(n) = 2T(n-1) + O(1)
```

解法：T(n) = O(2^n)。典型例子：朴素递归求斐波那契。

### 5.7 代码示例：快速幂的递归分析

```swift
/// 快速幂：T(n) = T(n/2) + O(1) → O(log n)
/// 其中 n 是指数
func power(_ base: Int, _ exp: Int, _ mod: Int) -> Int {
    if exp == 0 { return 1 }
    var result = power(base, exp / 2, mod)  // T(n/2)
    result = result * result % mod           // O(1)
    if exp % 2 == 1 {
        result = result * base % mod         // O(1)
    }
    return result
}

/// 迭代版快速幂：同样 O(log n) 时间，O(1) 空间
func powerIterative(_ base: Int, _ exp: Int, _ mod: Int) -> Int {
    var result = 1
    var b = base % mod
    var e = exp
    while e > 0 {
        if e & 1 == 1 {
            result = result * b % mod
        }
        b = b * b % mod
        e >>= 1
    }
    return result
}
```

```objc
// Objective-C 对照
```

## 六、均摊分析

### 6.1 概念

均摊分析（Amortized Analysis）研究的是**一系列操作**的平均代价。即使单次操作的最坏情况代价很高，只要高代价操作出现频率足够低，均摊代价仍然很小。

**与平均情况分析的区别**：均摊分析不涉及概率假设，它给出的是确定性保证——任意 n 次操作的总代价不超过某个上界。

### 6.2 三种方法

| 方法 | 核心思想 | 适用场景 |
|------|---------|---------|
| 聚合分析（Aggregate） | 计算 n 次操作总代价 T(n)，均摊 = T(n)/n | 操作模式规律 |
| 记账法（Accounting） | 给操作分配均摊代价，多付存银行，少付取银行 | 需要直觉指导 |
| 势能法（Potential） | 定义势能函数 Φ，均摊代价 = 实际代价 + ΔΦ | 最通用、最严谨 |

### 6.3 经典案例：动态数组扩容

Swift 的 Array 和 OC 的 NSMutableArray 在容量不够时以 2 倍扩容。

```swift
/// 动态数组模拟
struct DynamicArray<T> {
    private var storage: [T?]
    private(set) var count: Int = 0
    private var capacity: Int
    
    init(initialCapacity: Int = 1) {
        self.capacity = max(1, initialCapacity)
        self.storage = [T?](repeating: nil, count: capacity)
    }
    
    /// append 均摊分析：
    /// - 不扩容：代价 1
    /// - 扩容（发生在 n = 1,2,4,8,...,2^k）：代价 n+1（复制n个+插入1个）
    /// - n 次 append 总代价 ≤ n + (1+2+4+...+n) ≤ 3n
    /// - 均摊代价 = 3n/n = O(1)
    mutating func append(_ element: T) {
        if count == capacity {
            capacity *= 2
            var newStorage = [T?](repeating: nil, count: capacity)
            for i in 0..<count { newStorage[i] = storage[i] }
            storage = newStorage
        }
        storage[count] = element
        count += 1
    }
}
```

**势能法分析**：

定义势能 Φ = 2 × count - capacity（要求 count ≥ capacity/2 时 Φ ≥ 0）

- 普通 append：实际代价 = 1，ΔΦ = 2（count 增 1），均摊 = 1 + 2 = 3
- 扩容 append：实际代价 = count + 1，扩容前 Φ_old = 2n - n = n，扩容后 Φ_new = 2(n+1) - 2n = 2，ΔΦ = 2 - n，均摊 = (n+1) + (2-n) = 3

均摊代价恒为 3 = **O(1)**。

### 6.4 案例：多重出栈（Multipop）

```swift
/// 支持 multipop 的栈
struct AmortizedStack<T> {
    private var items: [T] = []
    
    /// push: O(1)
    mutating func push(_ item: T) {
        items.append(item)
    }
    
    /// multipop(k): 弹出 min(k, count) 个元素，最坏 O(n)
    /// 但 n 次操作（混合 push 和 multipop）总代价 O(n)
    /// 因为每个元素最多被 push 一次、pop 一次
    mutating func multipop(_ k: Int) -> [T] {
        let popCount = min(k, items.count)
        let popped = Array(items.suffix(popCount).reversed())
        items.removeLast(popCount)
        return popped
    }
}
```

**记账法分析**：给 push 分配均摊代价 2（1 用于实际 push，1 存入银行）。multipop 的均摊代价为 0（每次弹出从银行取 1）。总均摊代价 ≤ 2n = O(n)，均摊每次 O(1)。

### 6.5 案例：二进制计数器递增

```swift
/// 二进制计数器：increment 的均摊分析
/// 单次最坏 O(log n)（所有位翻转），均摊 O(1)
struct BinaryCounter {
    var bits: [Int]  // bits[i] 表示第 i 位
    
    init(size: Int) {
        bits = [Int](repeating: 0, count: size)
    }
    
    /// 聚合分析：n 次 increment
    /// 第 0 位：每次都翻，共 n 次
    /// 第 1 位：每 2 次翻一次，共 n/2 次
    /// 第 i 位：每 2^i 次翻一次，共 n/2^i 次
    /// 总翻转次数 = n + n/2 + n/4 + ... < 2n
    /// 均摊代价 = 2n/n = O(1)
    mutating func increment() {
        var i = 0
        while i < bits.count && bits[i] == 1 {
            bits[i] = 0  // 进位：1→0
            i += 1
        }
        if i < bits.count {
            bits[i] = 1  // 0→1
        }
    }
}
```

| 栈 multipop | O(n) | O(1) | 每元素最多 pop 一次 |

## 八、枚举与暴力搜索

### 8.1 思想与适用场景

枚举是最朴素的策略：系统遍历所有可能解，逐一检验。

**优点**：
- 正确性易保证
- 实现简单，不易出错
- 可作为正确性验证的基准（对拍）

**适用场景**：
- 问题规模极小（n ≤ 10）
- 快速验证思路
- 无法找到多项式算法的 NP-hard 问题（小规模实例）

### 8.2 枚举的系统化

| 枚举对象 | 空间大小 | 常用方法 |
|---------|---------|---------|
| 所有子集 | 2^n | 位掩码 / 递归 |
| 所有排列 | n! | next_permutation / 回溯 |
| 所有组合 C(n,k) | C(n,k) | 回溯 / 递推 |
| 所有数对 | n² | 双重循环 |
| 所有子串 | n² | 起点+终点双循环 |

### 8.3 示例：子集枚举

```swift
/// 位掩码枚举所有子集：O(2^n × n)
func subsets(_ nums: [Int]) -> [[Int]] {
    let n = nums.count
    var result: [[Int]] = []
    // 从 0 到 2^n - 1，每个数代表一个子集
    for mask in 0..<(1 << n) {
        var subset: [Int] = []
        for i in 0..<n {
            if mask & (1 << i) != 0 {
                subset.append(nums[i])
            }
        }
        result.append(subset)
    }
    return result
}
```

```objc
// Objective-C 对照
- (NSArray<NSArray<NSNumber *> *> *)subsets:(NSArray<NSNumber *> *)nums {
    NSInteger n = nums.count;
    NSMutableArray *result = [NSMutableArray array];
    for (NSInteger mask = 0; mask < (1 << n); mask++) {
        NSMutableArray *subset = [NSMutableArray array];
        for (NSInteger i = 0; i < n; i++) {
            if (mask & (1 << i)) {
                [subset addObject:nums[i]];
            }
        }
        [result addObject:[subset copy]];
    }
    return [result copy];
}
```

## 九、分治法

### 9.1 三步范式

1. **分解（Divide）**：将问题分为若干规模更小的子问题
2. **解决（Conquer）**：递归求解；规模足够小时直接解决
3. **合并（Combine）**：将子问题的解组合为原问题的解

### 9.2 适用条件

- 问题可分解为**结构相似**的子问题
- 子问题之间**相互独立**
- 存在高效的合并方法

### 9.3 经典分治算法

| 算法 | 分解方式 | 合并代价 | 总复杂度 |
|------|---------|---------|---------|
| 归并排序 | 二等分 | O(n) | O(n log n) |
| 快速排序 | 按 pivot 分区 | O(1) | O(n log n) avg |
| 最近点对 | 按 x 坐标分 | O(n) | O(n log n) |
| Karatsuba 乘法 | 高低位拆分 | O(n) | O(n^1.585) |
| 最大子数组和 | 二等分 | O(n) 跨界扫描 | O(n log n) |
| 逆序对计数 | 归并时计数 | O(n) | O(n log n) |

### 9.4 示例：逆序对计数

```swift
/// 利用归并排序计数逆序对
/// T(n) = 2T(n/2) + O(n) → O(n log n)
func countInversions(_ arr: [Int]) -> Int {
    var temp = arr
    return mergeSortCount(&temp, 0, arr.count - 1)
}

private func mergeSortCount(_ arr: inout [Int], _ left: Int, _ right: Int) -> Int {
    guard left < right else { return 0 }
    let mid = (left + right) / 2
    var count = 0
    count += mergeSortCount(&arr, left, mid)
    count += mergeSortCount(&arr, mid + 1, right)
    count += mergeCount(&arr, left, mid, right)
    return count
}

private func mergeCount(_ arr: inout [Int], _ left: Int, _ mid: Int, _ right: Int) -> Int {
    var temp = Array(arr[left...right])
    var i = 0, j = mid - left + 1, k = left
    var count = 0
    let leftSize = mid - left + 1
    
    while i < leftSize && j < temp.count {
        if temp[i] <= temp[j] {
            arr[k] = temp[i]; i += 1
        } else {
            // temp[i] > temp[j]：左半部分 i 及之后的元素都与 temp[j] 构成逆序对
            count += leftSize - i
            arr[k] = temp[j]; j += 1
        }
        k += 1
    }
    while i < leftSize { arr[k] = temp[i]; i += 1; k += 1 }
    while j < temp.count { arr[k] = temp[j]; j += 1; k += 1 }
    return count
}
```

```objc
// Objective-C 对照（核心逻辑）
- (NSInteger)mergeCountWithArray:(NSMutableArray<NSNumber *> *)arr
                            left:(NSInteger)left
                             mid:(NSInteger)mid
                           right:(NSInteger)right {

    NSInteger i = 0, j = mid - left + 1, k = left;
    NSInteger count = 0;
    NSInteger leftSize = mid - left + 1;
    
    while (i < leftSize && j < (NSInteger)temp.count) {
        if (temp[i].integerValue <= temp[j].integerValue) {
            arr[k] = temp[i]; i++;
        } else {
            count += leftSize - i;
            arr[k] = temp[j]; j++;
        }
        k++;
    }
    while (i < leftSize) { arr[k] = temp[i]; i++; k++; }
    while (j < (NSInteger)temp.count) { arr[k] = temp[j]; j++; k++; }
    return count;
}
```

## 十、贪心算法

### 10.1 核心思想

每一步选择中做出**当前看来最优**的选择，期望局部最优导出全局最优。

### 10.2 正确性证明——交换论证法

1. 假设存在一个最优解 OPT 不包含我们的贪心选择 g
2. 将 OPT 中的某个选择替换为 g，得到新解 OPT'
3. 证明 OPT' 不比 OPT 差
4. 结论：存在包含 g 的最优解 → 贪心选择安全

### 10.3 贪心 vs DP 对比

| 特征 | 贪心适用 | DP 适用 |
|------|---------|---------|
| 0-1 背包 | ✗（反例容易构造） | ✓ |
| 分数背包 | ✓（按单价排序） | — |
| 活动选择 | ✓（按结束时间） | ✓（但更慢） |
| 最长递增子序列 | ✓（贪心+二分） | ✓（朴素O(n²)） |
| 硬币找零（任意面额） | ✗（反例：面额1,3,4找6） | ✓ |
| 硬币找零（特殊面额如1,5,10,25） | ✓ | ✓ |

### 10.4 经典贪心问题

```swift
/// Huffman 编码：贪心构建最优前缀码
/// 策略：每次取频率最小的两个节点合并
/// 时间 O(n log n)（优先队列），空间 O(n)
class HuffmanNode: Comparable {
    let char: Character?
    let freq: Int
    var left: HuffmanNode?
    var right: HuffmanNode?
    
    init(char: Character?, freq: Int) {
        self.char = char
        self.freq = freq
    }
    
    static func < (lhs: HuffmanNode, rhs: HuffmanNode) -> Bool {
        return lhs.freq < rhs.freq
    }
    static func == (lhs: HuffmanNode, rhs: HuffmanNode) -> Bool {
        return lhs.freq == rhs.freq
    }
}

func buildHuffmanTree(_ frequencies: [(Character, Int)]) -> HuffmanNode? {
    // 用数组模拟最小堆（实际应用中用优先队列）
    var heap = frequencies.map { HuffmanNode(char: $0.0, freq: $0.1) }
    heap.sort()  // 简化：排序代替堆
    
    while heap.count > 1 {
        let left = heap.removeFirst()   // 频率最小
        let right = heap.removeFirst()  // 频率次小
        let parent = HuffmanNode(char: nil, freq: left.freq + right.freq)
        parent.left = left
        parent.right = right
        // 插入并保持有序
        let insertIdx = heap.firstIndex { $0.freq >= parent.freq } ?? heap.count
        heap.insert(parent, at: insertIdx)
    }
    return heap.first
}

/// 区间调度最大化（活动选择）
/// 贪心策略：按结束时间排序，选不冲突的
func maxActivities(start: [Int], end: [Int]) -> Int {
    let n = start.count
    let indices = (0..<n).sorted { end[$0] < end[$1] }
    var count = 1
    var lastEnd = end[indices[0]]
    for i in 1..<n {
        if start[indices[i]] >= lastEnd {
            count += 1
            lastEnd = end[indices[i]]
        }
    }
    return count
}
```

## 十一、动态规划

### 11.1 核心思想

将问题分解为重叠子问题，通过**记忆化/制表法**避免重复计算。

### 11.2 DP 设计五步法

1. **定义状态**：dp[i] 或 dp[i][j] 代表什么含义
2. **状态转移方程**：dp[i] 由哪些更小的状态推出
3. **初始条件/边界**
4. **计算顺序**：确保依赖的子状态已计算
5. **答案位置**

### 11.3 DP 的两种实现方式

| 方式 | 特点 | 优势 | 劣势 |
|------|------|------|------|
| 自顶向下（记忆化递归） | 递归+缓存 | 只计算需要的状态 | 递归开销、栈深度限制 |
| 自底向上（制表法） | 迭代填表 | 无递归开销、可空间优化 | 可能计算不需要的状态 |

### 11.4 经典 DP 问题全景

| 类型 | 代表问题 | 状态定义 | 复杂度 |
|------|---------|---------|--------|
| 线性 DP | LIS、打家劫舍 | dp[i] = 以 i 结尾的最优值 | O(n) 或 O(n²) |
| 区间 DP | 矩阵链乘、戳气球 | dp[i][j] = 区间[i,j]的最优值 | O(n³) |
| 背包 DP | 0-1/完全/多重背包 | dp[i][w] = 前i物品容量w的最优 | O(n×W) |
| 树形 DP | 二叉树最大路径和 | dp[node] = 以node为根的最优 | O(n) |
| 状压 DP | TSP、集合覆盖 | dp[mask] = 已访问集合的最优 | O(2^n × n) |
| 数位 DP | 计数特殊数字 | dp[pos][state][tight] | O(digits × states) |

### 11.5 示例：最长公共子序列（LCS）

```swift
/// LCS：dp[i][j] = s[0..<i] 与 t[0..<j] 的 LCS 长度
/// 时间 O(mn)，空间 O(mn)（可优化到 O(min(m,n))）
func longestCommonSubsequence(_ text1: String, _ text2: String) -> Int {
    let s = Array(text1), t = Array(text2)
    let m = s.count, n = t.count
    var dp = [[Int]](repeating: [Int](repeating: 0, count: n + 1), count: m + 1)
    
    for i in 1...m {
        for j in 1...n {
            if s[i-1] == t[j-1] {
                dp[i][j] = dp[i-1][j-1] + 1
            } else {
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
            }
        }
    }
    return dp[m][n]
}

/// 空间优化版：O(min(m,n))
func lcsOptimized(_ text1: String, _ text2: String) -> Int {
    let s = Array(text1), t = Array(text2)
    let m = s.count, n = t.count
    // 确保 t 是较短的，节省空间
    var dp = [Int](repeating: 0, count: n + 1)
    
    for i in 1...m {
        var prev = 0  // dp[i-1][j-1]
        for j in 1...n {
            let temp = dp[j]
            if s[i-1] == t[j-1] {
                dp[j] = prev + 1
            } else {
                dp[j] = max(dp[j], dp[j-1])
            }
            prev = temp
        }
    }
    return dp[n]
}
```

```objc
        }
    }
```

### 11.6 DP 优化技巧

| 技巧 | 原理 | 效果 |
|------|------|------|
| 滚动数组 | 只保留相邻行 | 空间 O(n²) → O(n) |

| 矩阵快速幂 | 线性递推加速 | O(n) → O(log n) |

## 十二、回溯与搜索

### 12.1 回溯 = DFS + 剪枝

系统探索解空间树，发现当前路径不可能产生有效解时回退。

### 12.2 通用模板

```swift
func backtrack(path: inout [Int], choices: [Int], result: inout [[Int]]) {
    if isComplete(path) {
        result.append(path)
        return
    }
    for choice in choices {
        if !isValid(path, choice) { continue }  // 剪枝
        path.append(choice)                      // 做选择
        backtrack(path: &path, choices: choices, result: &result)
        path.removeLast()                        // 撤销选择
    }
}
```

### 12.3 剪枝策略详解

| 策略 | 原理 | 示例 |
|------|------|------|
| 可行性剪枝 | 当前已违反约束 | 数独冲突 |
| 最优性剪枝 | 当前路径不可能优于已知最优 | 分支限界 |
| 对称性剪枝 | 跳过对称等价解 | 排列中去重 |
| 排序剪枝 | 排序后提前终止 | 组合求和超过 target 就 break |

### 12.4 示例：组合总和（去重版）

```swift
/// 从 candidates（含重复）中找出所有和为 target 的组合（每个数只能用一次）
/// 关键剪枝：排序 + 相邻去重
func combinationSum2(_ candidates: [Int], _ target: Int) -> [[Int]] {
    var result: [[Int]] = []
    var path: [Int] = []
    let sorted = candidates.sorted()  // 排序以便剪枝
    
    func backtrack(_ start: Int, _ remaining: Int) {
        if remaining == 0 {
            result.append(path)
            return
        }
        for i in start..<sorted.count {
            // 剪枝 1：超过目标，后面更大，全部跳过
            if sorted[i] > remaining { break }
            // 剪枝 2：同层去重（相同值只在第一次使用）
            if i > start && sorted[i] == sorted[i-1] { continue }
            
            path.append(sorted[i])
            backtrack(i + 1, remaining - sorted[i])
            path.removeLast()
        }
    }
    
    backtrack(0, target)
    return result
}
```

```objc
// Objective-C 对照
- (NSArray<NSArray<NSNumber *> *> *)combinationSum2:(NSArray<NSNumber *> *)candidates
                                            target:(NSInteger)target {
    NSArray *sorted = [candidates sortedArrayUsingSelector:@selector(compare:)];
    NSMutableArray *result = [NSMutableArray array];
    NSMutableArray *path = [NSMutableArray array];
    
    void (^__block backtrack)(NSInteger, NSInteger) = ^(NSInteger start, NSInteger remaining) {
        if (remaining == 0) {
            [result addObject:[path copy]];
            return;
        }
        for (NSInteger i = start; i < (NSInteger)sorted.count; i++) {
            NSInteger val = sorted[i].integerValue;
            if (val > remaining) break;
            if (i > start && sorted[i].integerValue == sorted[i-1].integerValue) continue;
            [path addObject:sorted[i]];
            backtrack(i + 1, remaining - val);
            [path removeLastObject];
        }
    };
    backtrack(0, target);
    return [result copy];
}
```

### 12.5 回溯的复杂度估算

| 问题 | 搜索空间 | 剪枝后实际 |
|------|---------|-----------|
| 全排列（n个不同元素） | O(n!) | O(n!) |
| 子集 | O(2^n) | O(2^n) |
| N 皇后 | O(n^n) | O(n!)（列+对角线约束） |
| 数独 | O(9^81) | 远小于（约束传播） |
| 组合求和 | O(2^n) | 排序剪枝后大幅减少 |

## 十三、双指针/滑动窗口/二分——复杂度优化利器

### 13.1 本质

它们不是独立的「算法思想」，而是**降低复杂度的优化手段**：

| 技巧 | 优化效果 | 利用的性质 |
|------|---------|-----------|
| 双指针 | O(n²) → O(n) | 有序性/单调性 |
| 滑动窗口 | O(n²) → O(n) | 窗口的连续性和单调扩缩 |
| 二分查找 | O(n) → O(log n) | 单调性/分界点 |
| 二分答案 | 枚举 → O(n log V) | 答案的单调性 |

### 13.2 双指针分类

**对撞指针**（两端向中间）：有序数组两数之和、接雨水、盛最多水的容器

**快慢指针**：链表环检测、删除倒数第K个、找中点

**同向指针**：滑动窗口、归并

```swift
/// 盛最多水的容器：对撞双指针 O(n)
func maxArea(_ height: [Int]) -> Int {
    var left = 0, right = height.count - 1
    var maxWater = 0
    while left < right {
        let water = min(height[left], height[right]) * (right - left)
        maxWater = max(maxWater, water)
        // 移动较短的一边（因为移动较长的不可能增加面积）
        if height[left] < height[right] {
            left += 1
        } else {
            right -= 1
        }
    }
    return maxWater
}
```

### 13.3 滑动窗口模板

```swift
/// 滑动窗口通用模板
func slidingWindow(_ s: String) -> Int {
    let arr = Array(s)
    var window: [Character: Int] = [:]
    var left = 0, result = 0
    
    for right in 0..<arr.count {
        // 1. 扩大窗口：加入 arr[right]
        window[arr[right], default: 0] += 1
        
        // 2. 收缩窗口：当窗口不满足条件时
        while !windowIsValid(window) {
            window[arr[left]]! -= 1
            if window[arr[left]] == 0 { window.removeValue(forKey: arr[left]) }
            left += 1
        }
        
        // 3. 更新答案
        result = max(result, right - left + 1)
    }
    return result
}
```

### 13.4 二分答案

当答案具有单调性时（小于某个值不满足，大于等于某个值满足），可以对答案空间二分：

```swift
/// 木头切割：将 n 根木头切成 k 段等长的，求最大长度
/// 答案具有单调性：长度越小越容易满足 k 段要求
func maxCutLength(_ woods: [Int], _ k: Int) -> Int {
    var lo = 1, hi = woods.max()!
    
    while lo <= hi {
        let mid = (lo + hi) / 2
        let pieces = woods.reduce(0) { $0 + $1 / mid }
        if pieces >= k {
            lo = mid + 1  // 可以更长
        } else {
            hi = mid - 1  // 太长了
        }
    }
    return hi  // 最大满足条件的长度
}
```

## 十五、综合实战：从暴力到最优的思维路径

### 15.1 实战一：最长无重复字符子串

**暴力 O(n³)**：枚举所有子串 O(n²)，对每个检查重复 O(n)。

**优化到 O(n)**：滑动窗口 + 哈希表记录字符最近出现位置。

```swift
func lengthOfLongestSubstring(_ s: String) -> Int {
    let arr = Array(s)
    var charIndex: [Character: Int] = [:]
    var left = 0, maxLen = 0
    
    for right in 0..<arr.count {
        if let prev = charIndex[arr[right]], prev >= left {
            left = prev + 1
        }
        charIndex[arr[right]] = right
        maxLen = max(maxLen, right - left + 1)
    }
    return maxLen
}
```

### 15.2 实战二：接雨水

**暴力 O(n²)** → **前缀最大值 O(n) 空间** → **双指针 O(1) 空间**

```swift
/// 双指针：O(n) 时间 O(1) 空间
func trap(_ height: [Int]) -> Int {
    var left = 0, right = height.count - 1
    var leftMax = 0, rightMax = 0, water = 0
    
    while left < right {
        if height[left] < height[right] {
            leftMax = max(leftMax, height[left])
            water += leftMax - height[left]
            left += 1
        } else {
            rightMax = max(rightMax, height[right])
            water += rightMax - height[right]
            right -= 1
        }
    }
    return water
}
```

### 15.3 实战三：编辑距离（经典二维 DP）

```swift
/// dp[i][j] = word1[0..<i] → word2[0..<j] 的最小编辑距离
func minDistance(_ word1: String, _ word2: String) -> Int {
    let s = Array(word1), t = Array(word2)
    let m = s.count, n = t.count
    var dp = [[Int]](repeating: [Int](repeating: 0, count: n + 1), count: m + 1)
    
    for i in 0...m { dp[i][0] = i }
    for j in 0...n { dp[0][j] = j }
    
    for i in 1...m {
        for j in 1...n {
            if s[i-1] == t[j-1] {
                dp[i][j] = dp[i-1][j-1]
            } else {
                dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])
            }
        }
    }
    return dp[m][n]
}
```

### 15.4 实战四：从暴力递归到 DP——硬币找零

展示完整的优化路径：

```swift
/// Step 1: 暴力递归（指数级复杂度）
func coinChange_Brute(_ coins: [Int], _ amount: Int) -> Int {
    if amount == 0 { return 0 }
    if amount < 0 { return -1 }
    var minCoins = Int.max
    for coin in coins {
        let sub = coinChange_Brute(coins, amount - coin)
        if sub >= 0 {
            minCoins = min(minCoins, sub + 1)
        }
    }
    return minCoins == Int.max ? -1 : minCoins
}

/// Step 2: 记忆化搜索（自顶向下 DP）
func coinChange_Memo(_ coins: [Int], _ amount: Int) -> Int {
    var memo = [Int: Int]()
    func dp(_ rem: Int) -> Int {
        if rem == 0 { return 0 }
        if rem < 0 { return -1 }
        if let cached = memo[rem] { return cached }
        var minCoins = Int.max
        for coin in coins {
            let sub = dp(rem - coin)
            if sub >= 0 { minCoins = min(minCoins, sub + 1) }
        }
        memo[rem] = minCoins == Int.max ? -1 : minCoins
        return memo[rem]!
    }
    return dp(amount)
}

/// Step 3: 自底向上 DP（最终版）
/// 时间 O(amount × coins.count)，空间 O(amount)
func coinChange(_ coins: [Int], _ amount: Int) -> Int {
    var dp = [Int](repeating: amount + 1, count: amount + 1)
    dp[0] = 0
    for i in 1...amount {
        for coin in coins {
            if coin <= i {
                dp[i] = min(dp[i], dp[i - coin] + 1)
            }
        }
    }
    return dp[amount] > amount ? -1 : dp[amount]
}
```

```objc

    dp[0] = @0;

        for (NSNumber *coin in coins) {

                if (newVal < dp[i].integerValue) dp[i] = @(newVal);
            }
        }
    }
    return dp[amount].integerValue > amount ? -1 : dp[amount].integerValue;
}
```

## 十六、常见面试题与答案要点

### Q1：O(log n) 和 O(n) 的实际差距？

当 n = 10^6 时，log₂n ≈ 20，差距 5 万倍。n = 10^9 时差 3000 万倍。这就是二分查找在大数据集上远优于线性搜索的原因。

### Q2：如何判断贪心 vs DP？

关键看**选择的可逆性**。贪心做了选择不回头（需证明正确），DP 探索所有选择取最优。方法：先试贪心，找反例，有反例就用 DP。

### Q3：均摊 O(1) vs 最坏 O(1) 的区别？

- 最坏 O(1)：每次都保证常数时间（如数组索引访问）
- 均摊 O(1)：n 次总代价 O(n)，单次可能 O(n)（如动态数组扩容）
- 实时系统需要最坏 O(1)，一般应用均摊足够

### Q4：DP 时间复杂度如何分析？

**状态数 × 每状态转移代价**。如编辑距离：mn 个状态 × O(1) 转移 = O(mn)。

### Q5：为什么比较排序下界是 Ω(n log n)？

n 个元素有 n! 种排列。每次比较将可能性分成两半。需要区分 n! 种排列，至少需要 log₂(n!) = Θ(n log n) 次比较（由 Stirling 近似 log(n!) ≈ n log n - n）。

### Q6：什么时候空间比时间更重要？

嵌入式/移动端内存受限、处理超大数据集、缓存友好性影响性能、递归深度可能栈溢出。

### Q7：如何从 O(n²) 优化到 O(n log n) 或 O(n)？

| 问题 | 原始 | 优化后 | 关键技巧 |
|------|------|--------|---------|
| 逆序对 | O(n²) | O(n log n) | 归并排序中计数 |
| LIS | O(n²) | O(n log n) | 贪心+二分 |
| 两数之和 | O(n²) | O(n) | 哈希表 |
| 接雨水 | O(n²) | O(n) | 双指针 |
| 最大矩形 | O(n²m) | O(nm) | 单调栈 |

### Q8：回溯的时间复杂度怎么估？

画搜索树前几层，估计分支因子 b 和深度 d → O(b^d)。N皇后粗略 O(n!)，子集 O(2^n)，全排列 O(n!)。

### Q9：哈希表为什么是 O(1)？什么时候退化？

依赖好的哈希函数均匀分布。最坏 O(n)（所有键碰撞到同一桶）。通过负载因子控制（< 0.75 扩容）和好的碰撞处理（链表→红黑树）保证近乎 O(1)。

### Q10：时间复杂度相同时如何选择算法？

考虑：常数因子、空间、缓存友好性、实现难度、最坏情况保证、实际数据特征（如几乎有序时插入排序优于快排）。

## 十七、易错点与最佳实践

### 17.1 常见易错点

| 错误 | 说明 | 正确做法 |
|------|------|---------|
| 漏标最坏情况 | "快排是 O(n log n)" | 应标注"平均 O(n log n)，最坏 O(n²)" |
| log 底数纠结 | "O(log₂n) ≠ O(log₁₀n)" | 底数差是常数因子，都写 O(log n) |
| 遗漏递归栈空间 | "二分查找空间 O(1)" | 递归版是 O(log n)，迭代版才是 O(1) |
| 字符串拼接低估 | "str + str 是 O(1)" | 大部分语言中是 O(n)，循环拼接 → O(n²) |
| 均摊与平均混淆 | "平均 O(1)" | 应说"均摊 O(1)"——确定性保证，无概率 |
| 忽略输入读取 | "O(1) 算法" | 读 n 个输入已是 O(n) |
| 低阶项未化简 | "O(n² + n log n)" | 应简化为 O(n²) |
| 主定理误用 | 不等分割套公式 | 需用递归树法 |
| 排序代价遗漏 | "双指针 O(n)" | 若需先排序，总复杂度是 O(n log n) |

### 17.2 最佳实践

**1. 先暴力，再优化**

暴力解保证正确性，是对拍基准。面试中先说暴力思路，再优化，展示思维过程。

**2. 分析清楚再编码**

在纸上推导复杂度和边界条件，确认可行再动手。面试中口述思路比直接敲代码更高效。

**3. 注意常数因子**

O(n) 但每步 100 次操作，可能不如 O(n log n) 但每步 1 次。实际要 profile。

**4. 数据规模敏感**

看到 n 的范围立刻判断需要什么级别复杂度。n ≤ 1000 → O(n²)，n ≤ 10^5 → O(n log n)。

**5. 空间换时间的权衡**

| 场景 | 策略 |
|------|------|
| 内存充裕、追求速度 | 哈希表、前缀和、DP 表 |
| 内存受限 | 原地算法、流式处理 |
| 递归深度过大 | 改迭代 + 显式栈 |

**6. 利用语言特性**

```swift
// Swift 标准库复杂度保证
// Array:  append O(1) 均摊, insert(at:0) O(n), contains O(n)
// Set:    insert/contains/remove O(1) 平均
// Dictionary: subscript O(1) 平均
// sorted(): O(n log n)，使用 Introsort（快排+堆排混合）

// 选择正确的数据结构
func hasDuplicate(_ nums: [Int]) -> Bool {
    // ❌ Array.contains → O(n²)
    // ✓ Set → O(n)
    var seen = Set<Int>()
    for num in nums {
        if !seen.insert(num).inserted { return true }
    }
    return false
}
```

```objc
// Objective-C 数据结构选择
// NSArray: containsObject O(n), objectAtIndex O(1)
// NSSet: containsObject O(1), 无序
// NSDictionary: objectForKey O(1)
// NSOrderedSet: containsObject O(1), 有序

- (BOOL)hasDuplicate:(NSArray<NSNumber *> *)nums {
    NSMutableSet *seen = [NSMutableSet setWithCapacity:nums.count];
    for (NSNumber *num in nums) {
        if ([seen containsObject:num]) return YES;
        [seen addObject:num];
    }
    return NO;
}
```

**7. 测试边界条件**

- 空输入（n = 0）
- 单元素（n = 1）
- 全相同元素
- 已排序/逆序
- 最大值（溢出检查）
