## 一、UINavigationController 导航体系

### 1.1 导航控制器的本质

```
UINavigationController 是一个容器控制器（Container VC），
用栈（stack）的方式管理子控制器：

┌────────────────────────────────┐
│      UINavigationController     │
│                                │
│   viewControllers（栈）：        │
│                                │
│   ┌──────────────────────────┐ │
│   │  VC-C  ← topViewController│ │  ← 当前可见
│   ├──────────────────────────┤ │
│   │  VC-B                    │ │
│   ├──────────────────────────┤ │
│   │  VC-A  ← rootViewController│ │  ← 栈底
│   └──────────────────────────┘ │
│                                │
│   NavigationBar（导航栏）       │
│   Toolbar（可选工具栏）         │
└────────────────────────────────┘

核心概念：
  - push：新 VC 入栈，向右滑入
  - pop：栈顶 VC 出栈，向左滑出
  - topViewController：栈顶控制器（当前显示）
  - viewControllers：整个栈数组
  - 每个子 VC 通过 self.navigationController 访问导航控制器
```

### 1.2 创建与配置

```objc
// 创建导航控制器
UIViewController *rootVC = [[HomeViewController alloc] init];
UINavigationController *nav = [[UINavigationController alloc]
                                initWithRootViewController:rootVC];

// 设为窗口的 rootViewController
self.window.rootViewController = nav;


// ===== 导航栏外观配置（iOS 13+ UINavigationBarAppearance）=====

UINavigationBarAppearance *appearance = [[UINavigationBarAppearance alloc] init];
[appearance configureWithOpaqueBackground];  // 不透明背景

// 背景色
appearance.backgroundColor = [UIColor systemBlueColor];

// 标题样式
appearance.titleTextAttributes = @{
    NSForegroundColorAttributeName: [UIColor whiteColor],
    NSFontAttributeName: [UIFont boldSystemFontOfSize:18]
};

// 大标题样式
appearance.largeTitleTextAttributes = @{
    NSForegroundColorAttributeName: [UIColor whiteColor]
};

// 返回按钮图标
appearance.backButtonAppearance.normal.titleTextAttributes = @{
    NSForegroundColorAttributeName: [UIColor whiteColor]
};

// 应用到导航栏
nav.navigationBar.standardAppearance = appearance;    // 标准模式
nav.navigationBar.scrollEdgeAppearance = appearance;  // 滚动到顶部时
nav.navigationBar.compactAppearance = appearance;     // 横屏紧凑模式


// ===== iOS 15+ 变化 =====
// iOS 15 开始 scrollEdgeAppearance 默认透明
// 如果页面有 ScrollView，滚动到顶部时导航栏会变透明
// 解决方案：显式设置 scrollEdgeAppearance
nav.navigationBar.scrollEdgeAppearance = nav.navigationBar.standardAppearance;
```

### 1.3 Push 与 Pop

```objc
// ===== Push =====
DetailViewController *detailVC = [[DetailViewController alloc] init];
[self.navigationController pushViewController:detailVC animated:YES];


// ===== Pop =====
// 返回上一页
[self.navigationController popViewControllerAnimated:YES];

// 返回到指定控制器
for (UIViewController *vc in self.navigationController.viewControllers) {
    if ([vc isKindOfClass:[HomeViewController class]]) {
        [self.navigationController popToViewController:vc animated:YES];
        break;
    }
}

// 返回到根控制器
[self.navigationController popToRootViewControllerAnimated:YES];


// ===== 直接替换栈 =====
// 场景：登录成功后替换整个导航栈，不允许返回到登录页
UIViewController *mainVC = [[MainViewController alloc] init];
[self.navigationController setViewControllers:@[mainVC] animated:YES];

// 插入中间页面（用户从深链接进入详情页，但 pop 时要经过列表页）
UIViewController *listVC = [[ListViewController alloc] init];
UIViewController *detailVC2 = [[DetailViewController alloc] init];
[self.navigationController setViewControllers:@[rootVC, listVC, detailVC2]
                                     animated:YES];


// ===== Push 传值 =====
DetailViewController *detail = [[DetailViewController alloc] init];
detail.itemId = @"12345";      // 正向传值：直接设置属性
detail.delegate = self;        // 反向传值：代理
[self.navigationController pushViewController:detail animated:YES];


// ===== Pop 传值方案 =====
// 方案1：Delegate
// 方案2：Block 回调
// 方案3：NotificationCenter
// 方案4：闭包（Swift）
```

### 1.4 UINavigationItem 配置

```objc
// 每个 ViewController 都有自己的 navigationItem
// 导航栏的内容由当前 topViewController 的 navigationItem 决定

self.navigationItem.rightBarButtonItem =

                                                  target:self

// 多个右侧按钮
UIBarButtonItem *editBtn = [[UIBarButtonItem alloc] initWithTitle:@"编辑"
                                                             style:UIBarButtonItemStylePlain
                                                            target:self
                                                            action:@selector(editTapped)];

                                                              style:UIBarButtonItemStylePlain
                                                             target:self
                                                             action:@selector(shareTapped)];
self.navigationItem.rightBarButtonItems = @[editBtn, shareBtn];


// ===== 自定义返回按钮 =====
// 方式1：自定义返回按钮文字（在前一个 VC 中设置）
self.navigationItem.backBarButtonItem =
    [[UIBarButtonItem alloc] initWithTitle:@"返回" style:UIBarButtonItemStylePlain
                                    target:nil action:nil];

// 方式2：完全替换返回按钮（在当前 VC 中设置 leftBarButtonItem）
self.navigationItem.leftBarButtonItem =
    [[UIBarButtonItem alloc] initWithImage:[UIImage systemImageNamed:@"chevron.left"]
                                     style:UIBarButtonItemStylePlain
                                    target:self
                                    action:@selector(customBack)];
// ⚠️ 设置 leftBarButtonItem 会同时禁用右滑返回手势
// 需要手动恢复：
self.navigationController.interactivePopGestureRecognizer.delegate = self;


// ===== 隐藏返回按钮 =====
self.navigationItem.hidesBackButton = YES;
```

### 1.5 导航栏隐藏与透明

```objc
// ===== 隐藏导航栏 =====
// 方式1：整个导航控制器隐藏导航栏
[self.navigationController setNavigationBarHidden:YES animated:YES];

// 方式2：在特定页面隐藏（在 viewWillAppear / viewWillDisappear 中切换）
- (void)viewWillAppear:(BOOL)animated {
    [super viewWillAppear:animated];
    [self.navigationController setNavigationBarHidden:YES animated:animated];
}
- (void)viewWillDisappear:(BOOL)animated {
    [super viewWillDisappear:animated];
    [self.navigationController setNavigationBarHidden:NO animated:animated];
}


// ===== 透明导航栏 =====
UINavigationBarAppearance *transparent = [[UINavigationBarAppearance alloc] init];
[transparent configureWithTransparentBackground];
transparent.backgroundColor = [UIColor clearColor];
transparent.shadowColor = [UIColor clearColor];  // 去掉底部分割线

self.navigationController.navigationBar.standardAppearance = transparent;
self.navigationController.navigationBar.scrollEdgeAppearance = transparent;


// ===== 滚动时渐变导航栏 =====
// 常见需求：页面顶部导航栏透明，下滑后逐渐变为不透明
- (void)scrollViewDidScroll:(UIScrollView *)scrollView {
    CGFloat offsetY = scrollView.contentOffset.y;
    CGFloat alpha = MIN(1.0, MAX(0, offsetY / 200.0));

    UINavigationBarAppearance *appearance = [[UINavigationBarAppearance alloc] init];
    if (alpha < 0.01) {
        [appearance configureWithTransparentBackground];
    } else {
        [appearance configureWithOpaqueBackground];
        appearance.backgroundColor = [[UIColor systemBlueColor] colorWithAlphaComponent:alpha];
    }
    appearance.shadowColor = alpha > 0.5 ? [UIColor separatorColor] : [UIColor clearColor];

    self.navigationController.navigationBar.standardAppearance = appearance;
    self.navigationController.navigationBar.scrollEdgeAppearance = appearance;
}
```

## 二、右滑返回手势

### 2.1 系统右滑返回

```objc
// UINavigationController 内置了交互式右滑返回手势：
// interactivePopGestureRecognizer（UIScreenEdgePanGestureRecognizer）

// 默认行为：
// - 从屏幕左边缘向右滑动 → 触发 pop 动画
// - 只在栈中有多个 VC 时有效
// - 自定义 leftBarButtonItem 后会失效


// ===== 恢复右滑返回 =====
// 自定义返回按钮后，需要手动设置 delegate
@interface DetailViewController () <UIGestureRecognizerDelegate>
@end

- (void)viewDidAppear:(BOOL)animated {
    [super viewDidAppear:animated];
    self.navigationController.interactivePopGestureRecognizer.delegate = self;
}

// ⚠️ 防止在根控制器启用右滑手势导致卡死
- (BOOL)gestureRecognizerShouldBegin:(UIGestureRecognizer *)gestureRecognizer {
    return self.navigationController.viewControllers.count > 1;
}
```

### 2.2 全屏右滑返回

```objc
// 系统右滑返回只能从屏幕左边缘触发
// 很多 App 实现了全屏右滑返回

// 原理：
// interactivePopGestureRecognizer 的 target 和 action 是私有的
// 可以取出其 target，绑定到自定义的 UIPanGestureRecognizer 上

@interface FullScreenPopNavigationController : UINavigationController
@end

@implementation FullScreenPopNavigationController

- (void)viewDidLoad {
    [super viewDidLoad];

    // 获取系统手势的 target
    id target = self.interactivePopGestureRecognizer.delegate;

    // 创建全屏 Pan 手势，绑定到相同的 target 和 action
    UIPanGestureRecognizer *pan = [[UIPanGestureRecognizer alloc]
        initWithTarget:target
                action:NSSelectorFromString(@"handleNavigationTransition:")];
    pan.delegate = self;

    [self.view addGestureRecognizer:pan];

    // 禁用系统的边缘手势
    self.interactivePopGestureRecognizer.enabled = NO;
}

- (BOOL)gestureRecognizerShouldBegin:(UIPanGestureRecognizer *)gestureRecognizer {
    // 只有栈中有多个 VC 才响应
    if (self.viewControllers.count <= 1) return NO;

    // 只响应向右滑动
    CGPoint velocity = [gestureRecognizer velocityInView:self.view];
    return velocity.x > 0 && fabs(velocity.x) > fabs(velocity.y);
}

@end

// ⚠️ 这种方式使用了私有 API（handleNavigationTransition:）
// 如果担心审核风险，可以使用自定义转场动画实现全屏右滑（见第六章）
```

## 三、UITabBarController

### 3.1 标签栏控制器

```
UITabBarController 也是容器控制器，管理多个平级的子控制器：

┌────────────────────────────────────────┐
│         UITabBarController              │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │                                  │  │
│  │    当前选中的 ViewController      │  │
│  │    （每个 tab 可以是 NavVC）       │  │
│  │                                  │  │
│  └──────────────────────────────────┘  │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │  [首页]  [发现]  [消息]  [我的]    │  │
│  │            UITabBar              │  │
│  └──────────────────────────────────┘  │
└────────────────────────────────────────┘

典型架构：
  UITabBarController
    ├── UINavigationController → HomeVC
    ├── UINavigationController → DiscoverVC
    ├── UINavigationController → MessageVC
    └── UINavigationController → ProfileVC
```

### 3.2 创建与配置

```objc
UITabBarController *tabBarController = [[UITabBarController alloc] init];

// 创建子控制器（每个 Tab 包裹一个 NavigationController）
UINavigationController *homeNav = [[UINavigationController alloc]
    initWithRootViewController:[[HomeViewController alloc] init]];
homeNav.tabBarItem = [[UITabBarItem alloc] initWithTitle:@"首页"
    image:[UIImage systemImageNamed:@"house"]
    selectedImage:[UIImage systemImageNamed:@"house.fill"]];

UINavigationController *discoverNav = [[UINavigationController alloc]
    initWithRootViewController:[[DiscoverViewController alloc] init]];
discoverNav.tabBarItem = [[UITabBarItem alloc] initWithTitle:@"发现"
    image:[UIImage systemImageNamed:@"safari"]
    selectedImage:[UIImage systemImageNamed:@"safari.fill"]];

UINavigationController *messageNav = [[UINavigationController alloc]
    initWithRootViewController:[[MessageViewController alloc] init]];
messageNav.tabBarItem = [[UITabBarItem alloc] initWithTitle:@"消息"
    image:[UIImage systemImageNamed:@"message"]
    selectedImage:[UIImage systemImageNamed:@"message.fill"]];
messageNav.tabBarItem.badgeValue = @"3";  // 角标

UINavigationController *profileNav = [[UINavigationController alloc]
    initWithRootViewController:[[ProfileViewController alloc] init]];
profileNav.tabBarItem = [[UITabBarItem alloc] initWithTitle:@"我的"
    image:[UIImage systemImageNamed:@"person"]
    selectedImage:[UIImage systemImageNamed:@"person.fill"]];

tabBarController.viewControllers = @[homeNav, discoverNav, messageNav, profileNav];
tabBarController.selectedIndex = 0;


// ===== TabBar 外观配置（iOS 13+ UITabBarAppearance）=====
UITabBarAppearance *tabAppearance = [[UITabBarAppearance alloc] init];
[tabAppearance configureWithOpaqueBackground];
tabAppearance.backgroundColor = [UIColor systemBackgroundColor];

// 选中/未选中状态
UITabBarItemAppearance *itemAppearance = tabAppearance.stackedLayoutAppearance;
itemAppearance.selected.iconColor = [UIColor systemBlueColor];
itemAppearance.selected.titleTextAttributes = @{
    NSForegroundColorAttributeName: [UIColor systemBlueColor]
};
itemAppearance.normal.iconColor = [UIColor systemGrayColor];
itemAppearance.normal.titleTextAttributes = @{
    NSForegroundColorAttributeName: [UIColor systemGrayColor]
};

tabBarController.tabBar.standardAppearance = tabAppearance;
if (@available(iOS 15.0, *)) {
    tabBarController.tabBar.scrollEdgeAppearance = tabAppearance;
}
```

### 3.4 隐藏 TabBar

```objc
// Push 到下一级页面时隐藏 TabBar
DetailViewController *detail = [[DetailViewController alloc] init];
detail.hidesBottomBarWhenPushed = YES;  // 在 push 之前设置
[self.navigationController pushViewController:detail animated:YES];

// ⚠️ hidesBottomBarWhenPushed 只对该 VC 及其之后 push 的所有 VC 生效
// 通常在第二级页面设置为 YES，后续页面自动继承隐藏状态


// ===== 动画隐藏 TabBar =====
- (void)setTabBarHidden:(BOOL)hidden animated:(BOOL)animated {
    CGRect frame = self.tabBarController.tabBar.frame;
    CGFloat offset = hidden ? frame.size.height : -frame.size.height;

    if (animated) {
        [UIView animateWithDuration:0.3 animations:^{
            self.tabBarController.tabBar.frame =
                CGRectOffset(self.tabBarController.tabBar.frame, 0, offset);
        }];
    } else {
        self.tabBarController.tabBar.frame =
            CGRectOffset(self.tabBarController.tabBar.frame, 0, offset);
    }
}
```

### 3.5 中间凸起按钮

```objc
// 很多 App 的 TabBar 中间有一个凸起的大按钮（如发布/拍照）

// 方案：在 TabBar 上添加自定义按钮
- (void)setupCenterButton {
    UIButton *centerBtn = [UIButton buttonWithType:UIButtonTypeCustom];
    centerBtn.frame = CGRectMake(0, 0, 56, 56);

    // 设置圆形背景
    centerBtn.backgroundColor = [UIColor systemBlueColor];
    centerBtn.layer.cornerRadius = 28;

    // 设置图标
    [centerBtn setImage:[UIImage systemImageNamed:@"plus"] forState:UIControlStateNormal];
    centerBtn.tintColor = [UIColor whiteColor];

    // 添加到 TabBar，位置在中间偏上
    CGFloat tabBarHeight = self.tabBarController.tabBar.frame.size.height;
    centerBtn.center = CGPointMake(self.tabBarController.tabBar.bounds.size.width / 2.0,
                                    tabBarHeight / 2.0 - 10);
    [self.tabBarController.tabBar addSubview:centerBtn];

    [centerBtn addTarget:self action:@selector(centerButtonTapped)
        forControlEvents:UIControlEventTouchUpInside];

    // ⚠️ 凸起部分超出 TabBar 边界，默认无法响应点击
    // 需要重写 TabBar 的 hitTest 方法
}

// 自定义 TabBar 子类
@implementation CustomTabBar

- (UIView *)hitTest:(CGPoint)point withEvent:(UIEvent *)event {
    // 如果点击在 TabBar 外但在凸起按钮范围内，让按钮响应
    if (!self.clipsToBounds && !self.hidden && self.alpha > 0) {
        for (UIView *subview in [self.subviews reverseObjectEnumerator]) {
            CGPoint subPoint = [subview convertPoint:point fromView:self];
            UIView *result = [subview hitTest:subPoint withEvent:event];
            if (result) return result;
        }
    }
    return [super hitTest:point withEvent:event];
}

@end
```

## 四、Modal Presentation（模态展示）

### 4.1 模态展示基础

```objc
// 模态展示：在当前 VC 上方弹出一个新的 VC
// 常用于登录、设置、编辑等独立流程

DetailViewController *detailVC = [[DetailViewController alloc] init];

// 设置展示样式
detailVC.modalPresentationStyle = UIModalPresentationFullScreen;

// 设置转场动画样式
detailVC.modalTransitionStyle = UIModalTransitionStyleCoverVertical;

// 展示
[self presentViewController:detailVC animated:YES completion:^{
    NSLog(@"展示完成");
}];

// 关闭
[self dismissViewControllerAnimated:YES completion:nil];
// 或者被展示的 VC 自己调用：
[self.presentingViewController dismissViewControllerAnimated:YES completion:nil];
// 实际开发中直接用 [self dismissViewControllerAnimated:YES completion:nil] 也可以
// 系统会自动找到正确的 presentingViewController 来执行 dismiss
```

### 4.2 modalPresentationStyle 详解

```objc
// ===== iOS 提供的模态展示样式 =====

// 1. UIModalPresentationFullScreen
//    全屏展示，覆盖整个屏幕
//    前一个 VC 的 viewWillDisappear / viewDidDisappear 会被调用
//    ⚠️ iOS 12 及之前的默认样式

// 2. UIModalPresentationPageSheet（iOS 13+ 默认）
//    类似卡片效果，不完全覆盖（iPad 上更明显）
//    iPhone 竖屏下几乎全屏，但顶部有圆角且可以下拉关闭
//    前一个 VC 的 viewWillDisappear 不会被调用
//    ⚠️ iOS 13 开始成为默认样式，很多 App 升级后出现适配问题

// 3. UIModalPresentationFormSheet
//    iPad 上居中显示较小的窗口
//    iPhone 上与 PageSheet 表现类似

// 4. UIModalPresentationOverFullScreen
//    全屏展示，但前一个 VC 不会从视图层级中移除
//    适合需要透明/半透明背景的场景（如自定义弹窗）
//    前一个 VC 的 viewWillDisappear 不会被调用

// 5. UIModalPresentationOverCurrentContext
//    覆盖当前 VC 的上下文
//    前一个 VC 不会从视图层级中移除
//    与 OverFullScreen 类似，但考虑容器控制器的上下文

// 6. UIModalPresentationPopover
//    iPad 上以气泡弹窗形式展示
//    iPhone 上回退为 FullScreen

// 7. UIModalPresentationAutomatic（iOS 13+）
//    系统自动选择最合适的样式
//    大部分情况下等同于 PageSheet
//    ⚠️ iOS 13+ 的默认值


// ===== 样式对比 =====
//
//  FullScreen          PageSheet           OverFullScreen
//  ┌──────────┐       ┌──────────┐        ┌──────────┐
//  │          │       │ ╭──────╮ │        │▒▒▒▒▒▒▒▒▒▒│
//  │  新 VC   │       │ │ 新VC │ │        │▒ 新 VC  ▒│
//  │（全屏）  │       │ │(卡片)│ │        │▒(透明bg)▒│
//  │          │       │ ╰──────╯ │        │▒▒▒▒▒▒▒▒▒▒│
//  │ 前VC移除 │       │ 前VC可见  │        │ 前VC保留  │
//  └──────────┘       └──────────┘        └──────────┘
```

### 4.3 modalTransitionStyle 转场动画

```objc
// ===== 内置转场动画 =====

// 1. UIModalTransitionStyleCoverVertical（默认）
//    从底部滑入
detailVC.modalTransitionStyle = UIModalTransitionStyleCoverVertical;

// 2. UIModalTransitionStyleFlipHorizontal
//    水平翻转
detailVC.modalTransitionStyle = UIModalTransitionStyleFlipHorizontal;

// 3. UIModalTransitionStyleCrossDissolve
//    淡入淡出
detailVC.modalTransitionStyle = UIModalTransitionStyleCrossDissolve;

// 4. UIModalTransitionStylePartialCurl
//    翻页效果（只支持 FullScreen）
detailVC.modalTransitionStyle = UIModalTransitionStylePartialCurl;
```

### 4.4 iOS 13 下拉关闭（Sheet）

```objc
// iOS 13+ PageSheet 样式默认支持下拉关闭

// 禁止下拉关闭
detailVC.isModalInPresentation = YES;

// 监听下拉关闭（通过 UIAdaptivePresentationControllerDelegate）
@interface DetailViewController () <UIAdaptivePresentationControllerDelegate>
@end

- (void)viewDidLoad {
    [super viewDidLoad];
    self.presentationController.delegate = self;
}

// 是否允许下拉关闭
- (BOOL)presentationControllerShouldDismiss:(UIPresentationController *)presentationController {
    // 有未保存的修改时禁止关闭
    return !self.hasUnsavedChanges;
}

// 用户尝试下拉但被阻止时调用
- (void)presentationControllerDidAttemptToDismiss:(UIPresentationController *)presentationController {
    // 弹出确认弹窗
```

## 五、Presentation 机制原理

### 5.1 presentingViewController 与 presentedViewController

```
Present 形成一条链式关系：

  VC-A  ─── present ───→  VC-B  ─── present ───→  VC-C

  VC-A.presentedViewController  == VC-B
  VC-B.presentingViewController == VC-A
  VC-B.presentedViewController  == VC-C
  VC-C.presentingViewController == VC-B

Dismiss 规则：
  - 调用中间 VC-B 的 dismiss → 先 dismiss VC-C，再 dismiss VC-B
  - 调用 VC-A 的 dismiss → 整条链都会被 dismiss
  - 系统会自动处理链式 dismiss 的顺序
```

### 5.2 definesPresentationContext

```objc
// 默认情况下 present 是从最近的全屏 VC（或根 VC）发起的
// definesPresentationContext 允许非全屏 VC 定义自己为 present 的上下文

// 场景：在一个 Child VC 中 present，让弹出的 VC 只覆盖 Child 区域

// 父 VC 中嵌入的 Child VC：

// popup 只会覆盖 childVC 的区域，而不是整个屏幕
```

```objc
@end
```

```objc
                                sourceViewController:(UIViewController *)source {

        initWithPresentedViewController:presented presentingViewController:presenting];

// ===== Step 2：实现 AnimatedTransitioning =====

@interface SlideUpAnimator : NSObject <UIViewControllerAnimatedTransitioning>
@property (nonatomic, assign) BOOL isPresenting;
@end

@implementation SlideUpAnimator

- (NSTimeInterval)transitionDuration:(id<UIViewControllerContextTransitioning>)context {
    return 0.35;
}

- (void)animateTransition:(id<UIViewControllerContextTransitioning>)transitionContext {
    UIView *containerView = transitionContext.containerView;

    if (self.isPresenting) {
        // Present：从下方滑入
        UIView *toView = [transitionContext viewForKey:UITransitionContextToViewKey];
        toView.frame = containerView.bounds;
        toView.transform = CGAffineTransformMakeTranslation(0, containerView.bounds.size.height);
        [containerView addSubview:toView];

        [UIView animateWithDuration:[self transitionDuration:transitionContext]
                              delay:0
             usingSpringWithDamping:0.85
              initialSpringVelocity:0.5
                            options:UIViewAnimationOptionCurveEaseOut
                         animations:^{
            toView.transform = CGAffineTransformIdentity;
        } completion:^(BOOL finished) {
            [transitionContext completeTransition:!transitionContext.transitionWasCancelled];
        }];

    } else {
        // Dismiss：向下滑出
        UIView *fromView = [transitionContext viewForKey:UITransitionContextFromViewKey];

        [UIView animateWithDuration:[self transitionDuration:transitionContext]
                              delay:0
                            options:UIViewAnimationOptionCurveEaseIn
                         animations:^{
            fromView.transform =
                CGAffineTransformMakeTranslation(0, containerView.bounds.size.height);
        } completion:^(BOOL finished) {
            [fromView removeFromSuperview];
            [transitionContext completeTransition:!transitionContext.transitionWasCancelled];
        }];
    }
}

@end


// ===== Step 3：使用 =====
@interface MyViewController ()
@property (nonatomic, strong) ModalTransitionDelegate *transitionDelegate;
@end

- (void)showDetail {
    DetailViewController *detailVC = [[DetailViewController alloc] init];
    self.transitionDelegate = [[ModalTransitionDelegate alloc] init];
    detailVC.transitioningDelegate = self.transitionDelegate;
    detailVC.modalPresentationStyle = UIModalPresentationCustom;  // 必须设为 Custom
    [self presentViewController:detailVC animated:YES completion:nil];
}
// ⚠️ transitioningDelegate 是 weak 的，必须用属性强引用 delegate 对象
```

### 6.4 交互式转场（手势驱动）

```objc
// UIPercentDrivenInteractiveTransition 可以将手势进度映射到转场动画进度

@interface InteractivePopTransition : UIPercentDrivenInteractiveTransition
@property (nonatomic, assign) BOOL isInteracting;
@property (nonatomic, weak) UINavigationController *navigationController;
@end

@implementation InteractivePopTransition

- (void)attachToViewController:(UIViewController *)viewController {
    self.navigationController = viewController.navigationController;

    UIPanGestureRecognizer *pan = [[UIPanGestureRecognizer alloc]
        initWithTarget:self action:@selector(handlePan:)];
    [viewController.view addGestureRecognizer:pan];
}

- (void)handlePan:(UIPanGestureRecognizer *)gesture {
    CGFloat progress = [gesture translationInView:gesture.view].x / gesture.view.bounds.size.width;
    progress = MIN(1.0, MAX(0, progress));

    switch (gesture.state) {
        case UIGestureRecognizerStateBegan:
            self.isInteracting = YES;
            [self.navigationController popViewControllerAnimated:YES];
            break;

        case UIGestureRecognizerStateChanged:
            [self updateInteractiveTransition:progress];
            break;

        case UIGestureRecognizerStateEnded:
        case UIGestureRecognizerStateCancelled: {
            self.isInteracting = NO;
            CGFloat velocity = [gesture velocityInView:gesture.view].x;
            // 进度超过 40% 或速度较快则完成转场
            if (progress > 0.4 || velocity > 500) {
                [self finishInteractiveTransition];
            } else {
                [self cancelInteractiveTransition];
            }
            break;
        }
        default:
            break;
    }
}

@end
```

## 七、ViewController 生命周期与容器关系

### 7.1 生命周期回调顺序

```
Push VC-B 到 VC-A 之上：
  VC-B  viewDidLoad
  VC-A  viewWillDisappear:
  VC-B  viewWillAppear:
  VC-B  viewDidAppear:
  VC-A  viewDidDisappear:

Pop VC-B 回到 VC-A：
  VC-B  viewWillDisappear:
  VC-A  viewWillAppear:
  VC-A  viewDidAppear:
  VC-B  viewDidDisappear:
  VC-B  dealloc（如果无其他强引用）

Present VC-B（FullScreen）：
  VC-B  viewDidLoad
  VC-A  viewWillDisappear:
  VC-B  viewWillAppear:
  VC-B  viewDidAppear:
  VC-A  viewDidDisappear:

Present VC-B（PageSheet / OverFullScreen）：
  VC-B  viewDidLoad
  VC-B  viewWillAppear:
  VC-B  viewDidAppear:
  ⚠️ VC-A 的 viewWillDisappear / viewDidDisappear 不会调用！

Dismiss VC-B（FullScreen）：
  VC-B  viewWillDisappear:
  VC-A  viewWillAppear:
  VC-A  viewDidAppear:
  VC-B  viewDidDisappear:
  VC-B  dealloc

Dismiss VC-B（PageSheet）：
  VC-B  viewWillDisappear:
  VC-B  viewDidDisappear:
  ⚠️ VC-A 的 viewWillAppear / viewDidAppear 不会调用！
  VC-B  dealloc
```

### 7.2 Child ViewController

```objc
// UIKit 支持控制器嵌套（Container VC 模式）
// 常用于实现多 Tab 内容区、分屏布局等

// ===== 添加 Child VC =====
- (void)addContentViewController:(UIViewController *)childVC {
    [self addChildViewController:childVC];                // 1. 建立父子关系
    childVC.view.frame = self.containerView.bounds;
    [self.containerView addSubview:childVC.view];         // 2. 添加视图
    [childVC didMoveToParentViewController:self];         // 3. 通知 child
}

// ===== 移除 Child VC =====
- (void)removeContentViewController:(UIViewController *)childVC {
    [childVC willMoveToParentViewController:nil];         // 1. 通知即将移除
    [childVC.view removeFromSuperview];                   // 2. 移除视图
    [childVC removeFromParentViewController];             // 3. 解除父子关系
}


// ===== 在两个 Child 之间切换 =====
- (void)transitionFromChild:(UIViewController *)fromVC
                     toChild:(UIViewController *)toVC {
    [self addChildViewController:toVC];
    [fromVC willMoveToParentViewController:nil];

    toVC.view.frame = self.containerView.bounds;
    toVC.view.alpha = 0;

    [self transitionFromViewController:fromVC
                      toViewController:toVC
                              duration:0.3
                               options:UIViewAnimationOptionTransitionCrossDissolve
                            animations:^{
        toVC.view.alpha = 1;
        fromVC.view.alpha = 0;
    } completion:^(BOOL finished) {
        [fromVC.view removeFromSuperview];
        [fromVC removeFromParentViewController];
        [toVC didMoveToParentViewController:self];
    }];
}


// ⚠️ 必须按正确顺序调用：
// 添加：addChildViewController → addSubview → didMoveToParentViewController
// 移除：willMoveToParentViewController:nil → removeFromSuperview → removeFromParentViewController
// 顺序错误会导致生命周期回调不正确
```

## 九、路由与页面管理

### 9.1 URL Router 设计

```objc
// 大型 App 通常使用路由系统来解耦页面跳转

// ===== 简单的路由实现 =====

typedef UIViewController * (^RouterHandler)(NSDictionary *params);

@interface Router : NSObject
@property (nonatomic, strong) NSMutableDictionary<NSString *, RouterHandler> *routes;
+ (instancetype)shared;
- (void)registerRoute:(NSString *)pattern handler:(RouterHandler)handler;
- (UIViewController *)viewControllerForURL:(NSString *)urlString;
- (void)openURL:(NSString *)urlString;
@end

@implementation Router

+ (instancetype)shared {
    static Router *instance;
    static dispatch_once_t onceToken;
    dispatch_once(&onceToken, ^{ instance = [[Router alloc] init]; });
    return instance;
}

- (instancetype)init {
    self = [super init];
    if (self) { _routes = [NSMutableDictionary dictionary]; }
    return self;
}

// 注册路由
- (void)registerRoute:(NSString *)pattern handler:(RouterHandler)handler {
    self.routes[pattern] = handler;
}

// 根据 URL 获取 VC
- (UIViewController *)viewControllerForURL:(NSString *)urlString {
    NSURL *url = [NSURL URLWithString:urlString];
    NSString *path = url.path;

    // 解析参数
    NSMutableDictionary *params = [NSMutableDictionary dictionary];
    NSURLComponents *components = [NSURLComponents componentsWithString:urlString];
    for (NSURLQueryItem *item in components.queryItems) {
        params[item.name] = item.value;
    }

    RouterHandler handler = self.routes[path];
    return handler ? handler(params) : nil;
}

// 打开 URL
- (void)openURL:(NSString *)urlString {
    UIViewController *vc = [self viewControllerForURL:urlString];
    if (!vc) return;

    UIViewController *topVC = [self topViewController];
    if (topVC.navigationController) {
        [topVC.navigationController pushViewController:vc animated:YES];
    } else {
        [topVC presentViewController:vc animated:YES completion:nil];
    }
}

// 获取最顶层 VC
- (UIViewController *)topViewController {
    UIViewController *root = UIApplication.sharedApplication.keyWindow.rootViewController;
    return [self topViewControllerFrom:root];
}

- (UIViewController *)topViewControllerFrom:(UIViewController *)vc {
    if ([vc isKindOfClass:[UINavigationController class]]) {
        return [self topViewControllerFrom:((UINavigationController *)vc).topViewController];
    }
    if ([vc isKindOfClass:[UITabBarController class]]) {
        return [self topViewControllerFrom:((UITabBarController *)vc).selectedViewController];
    }
    if (vc.presentedViewController) {
        return [self topViewControllerFrom:vc.presentedViewController];
    }
    return vc;
}

@end


// ===== 注册路由 =====
[[Router shared] registerRoute:@"/detail" handler:^UIViewController *(NSDictionary *params) {
    DetailViewController *vc = [[DetailViewController alloc] init];
    vc.itemId = params[@"id"];
    return vc;
}];

[[Router shared] registerRoute:@"/profile" handler:^UIViewController *(NSDictionary *params) {
    return [[ProfileViewController alloc] init];
}];


// ===== 使用路由 =====
[[Router shared] openURL:@"myapp:///detail?id=12345"];
[[Router shared] openURL:@"myapp:///profile"];
```

```objc
- (void)scene:(UIScene *)scene
    continueUserActivity:(NSUserActivity *)userActivity {
    if ([userActivity.activityType isEqualToString:NSUserActivityTypeBrowsingWeb]) {
        NSURL *url = userActivity.webpageURL;
        // url: https://yourdomain.com/detail/12345
        [[Router shared] openURL:url.absoluteString];
    }
}


// Universal Link vs URL Scheme：
// URL Scheme：任何 App 都可以注册，可能冲突，被拦截后无法 fallback
// Universal Link：基于域名验证，安全，在 Safari 中自动打开 App
//                  如果 App 未安装则打开网页（优雅降级）
```

## 十一、常见面试题

### Q1：push 和 present 的区别？

```
Push：
  - 由 UINavigationController 管理
  - 基于栈结构，新 VC 入栈
  - 有系统返回按钮和右滑返回手势
  - 适合层级递进的页面（列表 → 详情）

Present：
  - 任何 VC 都可以 present
  - 形成 presenting/presented 链
  - 样式多样（全屏、半屏、自定义）
  - 适合独立流程（登录、编辑、设置）

技术差异：
  - push 时前后 VC 共享同一个 NavigationBar
  - present FullScreen 时，前 VC 会从视图层级中移除
  - present PageSheet 时，前 VC 保留在视图层级中
  - push 有右滑返回手势，present PageSheet 有下拉关闭
```

### Q2：iOS 13 模态默认样式变化带来了什么影响？

```
iOS 13 之前：默认 FullScreen，前 VC 的 viewWillDisappear 会调用
iOS 13 之后：默认 PageSheet，前 VC 的 viewWillDisappear 不会调用

影响：
  - 在 viewWillAppear 中刷新数据的逻辑失效（dismiss 后不触发）
  - 解决方案：
    1. 显式设置 modalPresentationStyle = .fullScreen
    2. 使用 delegate/closure 回调通知数据变化
    3. 使用 presentationControllerDidDismiss 监听关闭事件
```

### Q3：如何实现自定义转场动画？

```
三步走：
  1. 被 present 的 VC 设置 transitioningDelegate
     modalPresentationStyle 设为 .custom
  2. delegate 返回 AnimationController（实现动画逻辑）
  3. 可选：返回 InteractionController（手势驱动）
     和 PresentationController（自定义背景、尺寸）

关键协议：
  - UIViewControllerTransitioningDelegate：提供动画/交互/展示控制器
  - UIViewControllerAnimatedTransitioning：执行动画
  - UIPercentDrivenInteractiveTransition：手势驱动进度
  - UIPresentationController：管理展示过程
```

### Q4：如何获取当前最顶层的 ViewController？

```
递归查找：
  1. 从 keyWindow.rootViewController 开始
  2. 如果是 NavigationController → 取 topViewController
  3. 如果是 TabBarController → 取 selectedViewController
  4. 如果有 presentedViewController → 递归查找
  5. 否则就是最顶层 VC

⚠️ iOS 13+ 多 Scene 场景下需要注意 keyWindow 的获取方式：
  UIWindowScene → windows → filter(keyWindow)
```

### Q5：ViewController 的 loadView、viewDidLoad 的区别？什么时候用 loadView？

```
loadView：
  - 创建 VC 的根 view
  - 默认实现：如果有 Storyboard/Xib 则从中加载，否则创建空白 UIView
  - 重写时不要调用 [super loadView]
  - 适合完全代码创建视图，不使用 Storyboard 时使用

viewDidLoad：
  - view 已经加载完成后调用
  - 在这里做子视图配置、数据绑定等
  - 只调用一次（除非 view 被卸载后重新加载，iOS 6+ 不再卸载）

调用顺序：
  init → loadView → viewDidLoad → viewWillAppear → viewDidAppear
```

### Q6：如何实现页面间传值？

```
正向传值（A → B）：
  - 属性赋值：B.property = value
  - 初始化方法传参

反向传值（B → A）：
  - Delegate 代理
  - Block/Closure 回调
  - NotificationCenter 通知
  - KVO

跨层级传值：
  - 路由参数
  - 全局状态管理（单例/NotificationCenter）
  - SwiftUI：@EnvironmentObject、@Environment
```

### Q7：hidesBottomBarWhenPushed 的注意事项？

```
1. 必须在 push 之前设置，push 之后设置无效
2. 设置为 YES 的 VC 及之后 push 的所有 VC 都会隐藏 TabBar
3. 通常只在第二级页面（从 Tab 页 push 出去的第一个页面）设置
4. pop 回到根 VC 时 TabBar 自动恢复
5. 不要在 viewWillAppear 中动态设置，可能导致动画异常
```

### Q8：SwiftUI NavigationStack 和 NavigationView 的区别？

```
NavigationView（iOS 13-15，已 deprecated）：
  - NavigationLink 与目标视图耦合
  - 不支持编程式导航
  - iPad 默认为 SplitView 样式
  - 无法方便地 popToRoot

NavigationStack（iOS 16+）：
  - 基于值的导航：NavigationLink(value:) + .navigationDestination
  - 支持 NavigationPath 编程式管理路径
  - 可以 append/removeLast 精确控制栈
  - 支持 Codable 状态保存与恢复
  - 统一的 stack 导航模型
```

### Q9：UIModalPresentationStyle 所有模式？

```
fullScreen（默认，iOS 13前）：
  - 模态页面覆盖整个屏幕
  - 遮挡系统 UI（状态栏等）
  - iOS 13+ 在 iPhone 上等同于 fullScreenCover

pageSheet：
  - iPad 上宽度略小于全屏，底部留有缝隙
  - iPhone 上等同于 fullScreen
  - 配合 UIModalTransitionStyle.coverVertical 使用

formSheet：
  - iPad 上显示为居中弹窗，尺寸小于 pageSheet
  - iPhone 12及以上（小屏）等同 fullScreen
  - iPhone SE 等大屏等同 pageSheet
  - iOS 15+：通过 sheetPresentationDetents 精确控制高度

overFullScreen / overSheetSheet：
  - 不遮挡底层 VC，底层仍可交互
  - 常用于弹窗、HUD、半透明遮罩
  - iOS 15+ overSheetSheet 为 iPad 专属

popover：
  - 仅 iPad 有效，iPhone 退化为 fullScreen
  - 必须设置 sourceRect 或 barButtonItem
  - 自动显示箭头指向触发位置
  - 点击外部自动dismiss（passthroughViews 可阻止）

automatic（iOS 15+）：
  - 系统根据设备和上下文自动选择最合适的样式
  - 推荐在新项目中使用
```

```swift
// iOS 15+ 精确控制 formSheet 高度
let vc = DetailViewController()
vc.modalPresentationStyle = .formSheet
if let sheet = vc.sheetPresentationController {
    sheet.detents = [.medium(), .large()]  // 支持拖拽切换高度
    sheet.prefersGrabberVisible = true      // 显示拖动手柄
}
present(vc, animated: true)
```

⚠️ Pitfalls: iPad 上 formSheet 必须配合 UISheetPresentationController 使用，否则可能崩溃；popover 在 iPhone 上会退化，需做设备判断。
✅ Best Practice: 新项目优先使用 iOS 15+ 的 Sheet/FullscreenCover API，替代传统的 modalPresentationStyle 枚举。

### Q10：UIModalTransitionStyle 所有类型？

```
coverVertical（默认）：
  - 从底部向上滑入，最常用
  - 配合 fullScreen/pageSheet 使用
  - iOS 15+ Sheet API 默认过渡效果

flipHorizontal：
  - 3D 翻页翻转效果，类似打开书本
  - iOS 6+ 已标记为过时，但在 Xcode 中仍可编译
  - 现代开发中基本不再使用

crossDissolve：
  - 淡入淡出效果，无位移
  - 适合 fullscreen over 类样式
  - 常用于 overFullScreen/overSheetSheet

partialCurl（iOS 15 移除）：
  - 翻页卷曲效果，类似 iPhone 4 书页动画
  - iOS 15 彻底移除，使用会 crash
  - 老项目迁移时需替换为 coverVertical 或 crossDissolve
```

```swift
let vc = DetailViewController()
vc.modalPresentationStyle = .formSheet
vc.modalTransitionStyle = .coverVertical  // 默认值，可省略
present(vc, animated: true)
```

⚠️ Pitfalls: iOS 15+ 使用 partialCurl 会直接崩溃，老项目迁移时必须检查并替换；crossDissolve 仅对 overFullScreen/overSheetSheet 有效，其他样式会退化为 coverVertical。
✅ Best Practice: 新项目统一使用 .coverVertical，特殊场景用 .crossDissolve；通过 iOS 15+ Sheet API 替代对 presentationStyle 的手动配置。

### iOS 13+ 基于 Scene 的呈现（UIScene、UISceneSession）

iOS 13 引入多场景（Scene）架构后，模态呈现不再局限于单一窗口。每个 UIScene 拥有独立的 UIWindowScene 和根控制器，跨 Scene 呈现需要通过 Scene Session 协调。

```swift
// 在同一 app 内不同 Scene 间传递数据
let config = UISceneConfiguration(
    name: nil,
    sessionRole: .applicationSceneConfiguration
)
config.delegateClass = NewSceneDelegate.self
// 通过 sceneSession 的 userInfo 传递上下文
config.sessionStateRestorationIdentifier = "detail"

// 在 SceneDelegate 中创建新的模态层级
func scene(_ scene: UIScene, willConnectTo session: UISceneSession, options: UIScene.ConnectionOptions) {
    guard let windowScene = scene as? UIWindowScene else { return }
    let window = UIWindow(windowScene: windowScene)
    window.rootViewController = DetailViewController()
    self.window = window
    window.makeKeyAndVisible()
}
```

⚠️ Pitfalls: 直接跨 Scene 调用 present() 会崩溃，每个 Scene 只能操作自己窗口层级内的视图控制器；iPad 分屏/Slide Over 场景下，每个实例独立管理 Scene，不要共享导航栈引用。
✅ Best Practice: 多 Scene 架构下用通知中心或共享 ViewModel 协调状态，而非直接操作其他 Scene 的视图层级；新 Scene 通过 UISceneConfiguration 配置而非硬编码窗口。

### 1.3.1 导航栈管理（viewControllers 数组操作）

直接操作 `viewControllers` 属性可以批量替换整个导航栈，是深链接、登录流程、中间页面注入等场景的核心手段。

```swift
// 场景1：登录后替换栈（不允许返回登录页）
let homeVC = HomeViewController()
let listVC = ListViewController()
let detailVC = DetailViewController()
navigationController?.setViewControllers([homeVC, listVC, detailVC], animated: true)

// 场景2：在栈中插入中间页面（如从详情页 pop 时经过列表页）
var stack = navigationController?.viewControllers ?? []
stack.insert(ListViewController(), at: stack.count - 1)
navigationController?.setViewControllers(stack, animated: true)

// 场景3：移除栈中某一级（如跳过某个中间步骤）
if let idx = stack.firstIndex(where: { $0 is UnwantedVC }) {
    stack.remove(at: idx)
    navigationController?.setViewControllers(stack, animated: true)
}
```

⚠️ Pitfalls: setViewControllers 会丢弃原栈中不存在的新控制器对应的 navigationItem 状态；不要直接修改返回的数组后不通过 setViewControllers 赋值——那样无效且不会触发导航动画。
✅ Best Practice: 始终通过 setViewControllers() 批量操作导航栈，animated: true 时系统自动对 topViewController 执行 push/pop 动画；栈根控制器（root）需保留以确保 popToRoot 正常工作。

### iOS 16+ showDetail / showAsModal 细节

iOS 16 为 `UIViewController` 新增 `showDetail()` 和 `showAsModal()` 方法，适配了分栏与模态呈现的现代 API，替代手动判断设备类型和场景逻辑。

```swift
// 1. showDetail — iPad 自动进入分栏副窗格，iPhone 模态呈现
let detailVC = DetailViewController(item: item)
detailVC.modalPresentationStyle = .automatic  // 系统自动选择
showDetail(detailVC)  // 无需手动判断 UITraitCollection

// 2. showAsModal — 强制模态呈现，不进入分栏
let settingsVC = SettingsViewController()
showAsModal(settingsVC)  // 等效于 show 但语义更明确

// 3. 在 NavigationStack 中使用（iOS 16+ SwiftUI 互操作）
// UIKit 侧 showDetail 会自动与 UIScene 的 splitViewController 协同
```

⚠️ Pitfalls: `showDetail()` 仅在主分栏场景下才走 `UISplitViewController` 的 `show(_:sender:)`；若无有效的 splitVC 层级（如从非根控制器调用），会退化为普通 present，导致 iPhone 和 iPad 行为不一致。
✅ Best Practice: 优先使用 `showDetail()` 替代手动的 `present()` + 设备判断，确保 iPad 分栏和 iPhone 模态行为统一；配合 `.automatic` presentationStyle 让系统决定最佳样式。

### Coordinator 模式 vs VIPER 导航

Coordinator 与 VIPER 是 iOS 架构中处理导航的两种主流方案。Coordinator 将导航逻辑提取到独立协调器中，解耦 ViewController 间的依赖；VIPER 则将导航分散在 Presenter 层，每个模块拥有独立的导航路径。两者核心区别在于**导航权归属**：Coordinator 集中管理路由表，VIPER 将导航嵌入模块内部。

```swift
// Coordinator 核心实现
protocol Coordinatable {
    var coordinator: MainCoordinator? { get set }
    func handleNavigation(action: NavigationAction)
}

class MainCoordinator {
    private var childCoordinators: [ChildCoordinator] = []
    private let nav: UINavigationController

    func start() {
        let loginVC = LoginViewController()
        loginVC.coordinator = self
        nav.setViewControllers([loginVC], animated: false)
    }

    func navigate(to destination: Destination) {
        switch destination {
        case .home:
            let homeVC = HomeViewController()
            homeVC.coordinator = self
            nav.pushViewController(homeVC, animated: true)
        case .settings:
            let coordinator = SettingsCoordinator(nav: nav)
            childCoordinators.append(coordinator)
            coordinator.start()
        }
    }
}
```

⚠️ Pitfalls: Coordinator 链过长时（3+层嵌套），导航回退路径难以追踪；VIPER 的 Presenter 耦合 NavigationComponent 导致模块边界模糊，单元测试需 Mock 整个导航栈。
✅ Best Practice: 小项目用 Coordinator 集中路由表（switch 枚举即可）；大型项目 VIPER + ModuleCoordinator 混合，每个模块自管内部导航、外部入口统一由主 Coordinator 调度。

### 深度链接 / Universal Links 实现

Universal Links 是 iOS 9+ 推荐的深度链接方案，通过 HTTPS 配置即可直接打开 App，无需 URL Scheme 的弹窗确认。核心流程：Apple 从 `apple-app-site-association` (AAASA) 文件获取 `path` 匹配规则，后台预取缓存，点击链接后静默打开 App。

```swift
// 1. AAASA 文件（托管在域名根目录，无扩展名，application/json）
// { "applinks": { "paths": { "exact /launch": { "appIDs": ["TEAM.com.app"] } } } }

// 2. App 侧处理 — SwiftUI 场景
import SwiftUI

@main struct App: App {
    @UIApplicationDelegateAdaptor(AppDelegate.self) var delegate
    var body: some Scene { WindowGroup { ContentView() } }
}

class AppDelegate: NSObject, UIApplicationDelegate {
    func application(_ app: UIApplication,
                     open url: URL,
                     options: [UIApplication.OpenURLOptionsKey: Any] = [:]) -> Bool {
        // Universal Links 走 openURL 回调
        handleDeepLink(url)
        return true
    }
}

// 3. iOS 13+ 推荐方案 — SceneDelegate
func scene(_ scene: UIScene, openURLContexts urls: Set<URLContext>) {
    guard let url = urls.first?.url else { return }
    handleDeepLink(url)  // url.userActivity.webpageURL 含原始链接
}
```

⚠️ Pitfalls: AAASA 必须通过 HTTPS 提供且不含 HTTP Redirect；`paths` 使用通配符 `*` 时优先级低于 `exact` 和 `begins`；Hotlink 模式下缓存失效需重新下载 AAASA。
✅ Best Practice: 开发期用 `xcrun simctl devices get_app_application_states <device> <bundleID>` 验证链接状态为 `Ready`；生产环境配合 Associated Domains 配置 `applinks:yourdomain.com`。

### 基于 AssociatedObject 的导航状态

在复杂导航场景中，ViewController 的状态往往分散在多个地方（视图层级、外部数据源、Coordinator）。利用 `objc_setAssociatedObject` 可以将导航相关状态以键值对形式挂载到 vc 实例上，实现轻量级的状态追踪与回溯。

```swift
import ObjectiveC

private var NavigationStateKey: UInt8 = 0

enum NavigationState: String {
    case initial, loading, error, detail
}

extension UIViewController {
    var navState: NavigationState {
        get {
            return objc_getAssociatedObject(self, &NavigationStateKey) as? NavigationState
                ?? .initial
        }
        set {
            objc_setAssociatedObject(self, &NavigationStateKey, newValue,
                                     .OBJC_ASSOCIATION_COPY_NONATOMIC)
            onUpdateState?(newValue)  // 可选回调通知
        }
    }

    var onUpdateState: ((NavigationState) -> Void)?
}

// 使用：在任何 vc 上直接读写状态
vc.navState = .loading
print(vc.navState.rawValue)  // "loading"
```

⚠️ Pitfalls: AssociatedObject 在 vc 从导航栈 pop 后不会自动清除，内存泄漏风险需手动在 deinit 中解除；多线程访问同一 vc 的 associated object 可能引发竞态条件。
✅ Best Practice: 仅用于临时导航状态标记（loading/refresh 标记），持久化数据仍存 ViewModel；结合 `navigationBarItems` 的 isHidden 属性实现状态驱动的 UI 切换。

### NavigationBar / TabBar 外观自定义（UINavigationBarAppearance）

iOS 13+ 引入 `UINavigationBarAppearance` 和 `UITabBarAppearance`，取代了旧的 tintColor/barTintColor 方案。通过配置 `Appearance` 对象并应用到 `standardAppearance`、`scrollEdgeAppearance` 和 `compactAppearance`，可实现不同场景下的精细化外观控制，包括背景色、毛玻璃效果、阴影线、大标题样式等。

```swift
// 1. 统一导航栏外观
let navAppearance = UINavigationBarAppearance()
navAppearance.configureWithOpaqueBackground()  // 不透明背景
navAppearance.backgroundColor = .systemBlue
navAppearance.titleTextAttributes = [.foregroundColor: UIColor.white]
navAppearance.shadowImage = UIImage()  // 移除底部阴影线

navigationController?.navigationBar.standardAppearance = navAppearance
navigationController?.navigationBar.scrollEdgeAppearance = navAppearance

// 2. TabBar 毛玻璃效果
let tabAppearance = UITabBarAppearance()
tabAppearance.configureWithBlurEffect(.systemUltraThinMaterial)
tabBar.standardAppearance = tabAppearance
tabBar.scrollEdgeAppearance = tabAppearance
```

⚠️ Pitfalls: `scrollEdgeAppearance` 必须在 `viewWillAppear` 中设置才生效，在 `viewDidLoad` 中设置可能导致滚动后样式丢失；`compactAppearance` 仅在 iPhone 分屏或 iPad 紧凑模式下触发，模拟器调试需手动调整窗口大小。
✅ Best Practice: 封装为 NavigationController 的 extension 统一配置，在设置 rootViewController 后立即应用；用 `traitCollection.userInterfaceStyle` 区分 Light/Dark 模式的 Appearance 对象实现自动适配。

### iPad 上的模态呈现（UIPopoverPresentationController、自适应布局）

在 iPad 上，`UIPresentationStyle` 的行为与 iPhone 截然不同。`fullScreen` 在 iPad 上退化为 `pageSheet`，而默认的 `.overFullScreen` 不会自动处理小屏布局。开发 iPad 应用时必须显式配置 `UIPopoverPresentationController`，否则模态窗口可能以不期望的大小出现在屏幕中心。

关键 API：`sourceView`/`sourceRect` 定义弹出箭头的锚点；`permittedArrowDirections` 控制箭头方向；`adaptivePresentationStyle` 决定在 iPhone 上的降级行为。对于自适应布局，结合 `UIContentSizeCategory` 和 `sizeClass` 实现响应式模态尺寸。

```swift
let vc = YourModalViewController()
vc.modalPresentationStyle = .popover

// iPad：指定弹出源视图和箭头
vc.popoverPresentationController?.sourceView = triggerButton
vc.popoverPresentationController?.sourceRect = triggerButton.bounds
vc.popoverPresentationController?.permittedArrowDirections = [.up, .down]
vc.popoverPresentationController?.adaptivePresentationStyle = .fullScreen  // iPhone 降级为全屏

// 监听尺寸变化，更新箭头位置
vc.popoverPresentationController?.delegate = self

// 设置最大尺寸，避免遮挡关键内容
vc.preferredContentSize = CGSize(width: 320, height: 400)

present(vc, animated: true)
```

⚠️ Pitfalls: `sourceView` 必须存在于视图中且不能是 nil，否则 popover 崩溃；在 popover 内 `present` 新 vc 会直接替换整个 popover 而非嵌套，需用 `show(_:sender:)` 实现层级导航。
✅ Best Practice: 封装 `presentPopover(from:contentSize:)` 辅助方法统一处理 sourceView/Rect 配置；使用 `UIAdaptivePresentationControllerDelegate` 的 `adaptivePresentationStyle(for:)` 按设备精确控制降级策略。

### Presentation Controller 自定义（自适应呈现）

`UIPresentationController` 是连接 presenting 和 presented ViewController 的桥梁，负责协调模态展示时的布局、过渡动画和交互行为。通过继承 `UIPresentationController` 并重写 `presentedViewInContainerView()` 和 `containerViewWillLayoutSubviews()`，可实现完全自定义的模态效果（如底部弹窗、遮罩层、侧边抽屉等），替代第三方弹窗库。

```swift
class BottomSheetPresentationController: UIPresentationController {
    private let blurView = UIVisualEffectView(effect: UIBlurEffect(style: .dark))
    
    override func presentedViewInContainerView() -> UIView {
        let frame = self.sheetFrame  // 底部半屏
        self.presentedView.frame = frame
        self.presentedView.layer.cornerRadius = 12
        return self.presentedView
    }
    
    override func presentationTransitionWillBegin() {
        containerView?.insertSubview(blurView, at: 0)
        containerView?.insertSubview(presentedView, aboveSubview: blurView)
        blurView.alpha = 0
        presentedView.alpha = 0
        
        presentedViewController.transitionPresenter?.animate(alongsideTransition: { _ in
            self.blurView.alpha = 0.6
            self.presentedView.alpha = 1
        })
        
        // 点击 blurView 关闭
        blurView.addGestureRecognizer(UITapGestureRecognizer(
            target: self, action: #selector(dismissTapped)))
    }
    
    @objc func dismissTapped() {
        presentedViewController.dismiss(animated: true)
    }
}
```

⚠️ Pitfalls: 自定义 PresentationController 必须在 presenting ViewController 的 `presentationController(forPresented:presenting:source:)` 中返回其实例；`containerView` 在 `init` 时为 nil，布局代码必须放在 `presentationTransitionWillBegin()` 中；`dismiss` 时若未处理反向动画，弹窗会直接消失而无退出过渡。
✅ Best Practice: 封装 `providePresentationController()` 方法统一返回自定义实例；配合 `UIAdaptivePresentationControllerDelegate` 的 `adaptivePresentationStyle(for:)` 按设备/尺寸类精确控制降级策略；简单场景优先用 `UISheetPresentationController`（iOS 15+），仅在需要特殊交互时选择继承。

### NavigationItem 大标题配置

`UINavigationItem` 的 `largeTitleDisplayMode` 控制导航栏大标题的显示行为，配合 `UINavigationController` 的 `navigationBar.prefersLargeTitles = true` 实现 iOS 11+ 的大标题效果。支持 `.automatic`（首现时显示，滚动缩小）、`.always`（始终显示）、`.never`（禁用）和 `.inline`（始终紧凑）四种模式。

```swift
// 开启全局大标题
navigationController?.navigationBar.prefersLargeTitles = true
navigationController?.navigationBar.largeTitleTextInsets = UIEdgeInsets(top: 8, left: 16, bottom: 8, right: 16)

// 单页面控制
navigationItem.largeTitleDisplayMode = .always     // 固定大标题
navigationItem.largeTitleDisplayMode = .automatic   // 滚动时折叠为小标题
navigationItem.largeTitleDisplayMode = .never       // 禁用大标题

// 自定义大标题样式
UINavigationBar.appearance().largeTitleTextAttributes = [
    .foregroundColor: UIColor.label,
    .font: UIFont.systemFont(ofSize: 34, weight: .bold)
]
```

⚠️ Pitfalls: `prefersLargeTitles` 必须在 `viewDidLoad` 之前设置才生效；`pushViewController` 动画过程中修改 `largeTitleDisplayMode` 会导致标题跳动；大标题模式下 `navigationItem.titleView` 会被忽略，需用 `navigationBar.titleViewController` 自定义。
✅ Best Practice: 根页面用 `.automatic` 实现标题折叠效果，子页面用 `.never` 保持紧凑导航栏；大标题样式统一在 `AppDelegate` 或主 NavigationController 初始化时配置，避免逐页面设置导致风格不一致。

### UISheetPresentationController / UIPopupPresentationController（iOS 16+）

`UISheetPresentationController` 是 iOS 15 引入、iOS 16 大幅增强的 sheet 展示控制器，支持 `.medium`、`.large` 等预定义尺寸及自定义 `detents`。iOS 16 新增 `UISheetDragHandleVisibility` 控制拖拽手柄显隐，`prefersGrabberVisible` 迁移至 `dragHandleVisibility`。`UIPopupPresentationController`（iOS 16+）则在 iPad 上提供类似 popup 的紧凑展示样式，自动适配分割视图。

```swift
// Sheet 预设尺寸 + 自定义 detents
let vc = MyViewController()
vc.modalPresentationStyle = .pageSheet
if let sheet = vc.sheet {
    sheet.detents = [.medium(), .large()]
    sheet.dragHandleVisibility = .always          // iOS 16+
    sheet.prefersGrabberVisible = false           // 隐藏拖拽手柄
    sheet.overrideTraitCollectionForDetents = UITraitCollection(horizontalSizeClass: .regular)
}
present(vc, animated: true)

// Popup 展示（iPad 专用）
let popupVC = PopupContentViewController()
popupVC.modalPresentationStyle = .popover
if let popup = popupVC.popoverPresentationController {
    popup.sourceView = sender as? UIView
    popup.sourceRect = (sender as? UIButton)?.bounds ?? .zero
    popup.permittedArrowDirections = [.up, .down]
}
present(popupVC, animated: true)
```

⚠️ Pitfalls: `sheet` 属性在 `present()` 之前配置才生效；`detents` 数组必须从小到大排序，否则崩溃；iPad 上 `.pageSheet` 自动降级为 `.formSheet`，需用 `UIAdaptivePresentationControllerDelegate` 拦截；`dragHandleVisibility` 在紧凑尺寸下 `.hidden` 会导致用户无法拖动。
✅ Best Practice: 统一封装 `presentSheet()` 扩展方法集中配置 detents 和手柄样式；多页面嵌套 sheet 时注意每个层级独立配置 `detents`，避免父子 sheet 尺寸冲突。
