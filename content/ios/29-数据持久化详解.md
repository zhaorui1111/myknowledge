# iOS 数据持久化详解

## 一、持久化方案总览

iOS 提供了多种数据持久化方案，各有适用场景：

| 方案 | 适用场景 | 数据格式 | 线程安全 | 性能 |
|------|---------|---------|---------|------|
| UserDefaults | 少量配置项 | plist（XML/二进制） | 是 | 中 |
| Keychain | 密码/Token等敏感信息 | 加密二进制 | 是 | 中 |
| 文件沙盒 | 图片/音频/缓存文件 | 任意 | 需自行保证 | 高 |
| Codable + JSON | 结构化数据序列化 | JSON | 是 | 中 |
| SQLite / WCDB | 大量结构化数据 | 关系型数据库 | 需自行保证 | 高 |
| Core Data | 复杂对象图、关系模型 | SQLite/XML/内存 | 是 | 中高 |

选择原则：配置用 UserDefaults，敏感信息用 Keychain，文件用沙盒，结构化数据用 Codable+文件或 SQLite，复杂关系用 Core Data。

## 二、UserDefaults

### 2.1 基本用法

```swift
let defaults = UserDefaults.standard

// 写入
defaults.set("张三", forKey: "userName")
defaults.set(25, forKey: "userAge")
defaults.set(true, forKey: "isFirstLaunch")
defaults.set(Date(), forKey: "lastOpenDate")

// 读取
let name = defaults.string(forKey: "userName") ?? ""
let age = defaults.integer(forKey: "userAge")
let isFirst = defaults.bool(forKey: "isFirstLaunch")

// 删除
defaults.removeObject(forKey: "userName")
```

### 2.2 存储自定义对象

UserDefaults 只能存储 plist 兼容类型（String、Int、Double、Bool、Data、Date、Array、Dictionary）。自定义对象需要先编码为 Data：

```swift
struct User: Codable {
    let id: String
    let name: String
    let avatar: String?
}

let user = User(id: "001", name: "张三", avatar: nil)

// 存
if let data = try? JSONEncoder().encode(user) {
    defaults.set(data, forKey: "currentUser")
}

// 取
if let data = defaults.data(forKey: "currentUser") {
    let user = try? JSONDecoder().decode(User.self, from: data)
}
```

### 2.3 App Groups

App Extension 和主 App 之间共享数据需要使用 App Groups：

```swift
let groupDefaults = UserDefaults(suiteName: "group.com.example.myapp")
```

### 2.4 注意事项

UserDefaults 会在合适时机将内存中的数据同步到磁盘，不保证即时持久化。如果需要在退出前确保写入，调用 `defaults.synchronize()`（虽然 Apple 文档说不再需要手动调用，但关键数据场景下仍然推荐）。

UserDefaults 的读取是在内存中进行的（首次加载时从磁盘读取到内存），所以读取速度很快，但不适合存储大量数据。一般建议不超过几十 KB。

## 三、Keychain

### 3.1 Keychain 基础

Keychain 是 iOS 提供的加密存储方案，数据被硬件级别的安全芯片加密保护，即使设备越狱也难以提取。适合存储密码、Token、证书等敏感信息。

Keychain Services API 是 C 语言接口，使用起来比较繁琐。通常封装为 Swift 工具类：

```swift
import Security

class KeychainHelper {
    static let shared = KeychainHelper()
    private init() {}

    func save(_ data: Data, service: String, account: String) {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account
        ]

        // 先删除旧值
        SecItemDelete(query as CFDictionary)

        // 添加新值
        var attributes = query
        attributes[kSecValueData as String] = data
        attributes[kSecAttrAccessible as String] = kSecAttrAccessibleAfterFirstUnlock
        SecItemAdd(attributes as CFDictionary, nil)
    }

    func read(service: String, account: String) -> Data? {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account,
            kSecReturnData as String: true,
            kSecMatchLimit as String: kSecMatchLimitOne
        ]

        var result: AnyObject?
        SecItemCopyMatching(query as CFDictionary, &result)
        return result as? Data
    }

    func delete(service: String, account: String) {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account
        ]
        SecItemDelete(query as CFDictionary)
    }
}
```

### 3.2 访问控制

`kSecAttrAccessible` 控制何时可以访问 Keychain 数据：

- `kSecAttrAccessibleWhenUnlocked`：设备解锁后可访问（默认）
- `kSecAttrAccessibleAfterFirstUnlock`：首次解锁后可访问，之后设备锁定也能访问（适合后台任务）
- `kSecAttrAccessibleWhenPasscodeSetThisDeviceOnly`：仅在设置了密码的设备上可访问，不随 iCloud 备份迁移
- `kSecAttrAccessibleAlways`：始终可访问（已废弃，不推荐）

### 3.3 Keychain 与 App 重装

Keychain 数据在 App 卸载后仍然保留（除非用户选择"清除所有内容和设置"）。可以利用这个特性存储设备唯一标识符等需要跨安装持久化的信息。但需注意 iOS 10.3+ 开始，卸载 App 后系统会弹出是否清除 Keychain 数据的选项。

### 3.4 第三方库推荐

直接使用 Security 框架比较繁琐，推荐使用第三方封装库：

- **KeychainAccess**：Swift 封装，API 简洁
- **SAMKeychain**：Objective-C 封装，成熟稳定

## 四、文件沙盒

### 4.1 沙盒目录结构

```
App Bundle/
├── Documents/      # 用户文档，iCloud 备份
├── Library/
│   ├── Caches/     # 缓存，可被系统清理，不备份
│   └── Preferences/ # UserDefaults plist 文件
├── tmp/            # 临时文件，随时可能被清理
└── App.app/        # 应用包，只读
```

### 4.2 获取目录路径

```swift
let documentsURL = FileManager.default.urls(for: .documentDirectory, in: .userDomainMask).first!
let cachesURL = FileManager.default.urls(for: .cachesDirectory, in: .userDomainMask).first!
let tmpURL = URL(fileURLWithPath: NSTemporaryDirectory())
```

### 4.3 文件读写

```swift
// 写入
let fileURL = documentsURL.appendingPathComponent("data.json")
try data.write(to: fileURL)

// 读取
let data = try Data(contentsOf: fileURL)

// 删除
try FileManager.default.removeItem(at: fileURL)
```

### 4.4 文件保护级别

通过 `NSFileProtectionComplete` 等属性设置文件加密级别：

```swift
try data.write(to: fileURL, options: [.atomic, .completeFileProtection])
```

`.completeFileProtection`：设备锁定后文件不可读，最高安全级别。`.completeFileProtectionUnlessOpen`：文件打开后可以继续访问，适合需要后台访问的文件。

## 五、Codable 与 JSON 持久化

### 5.1 Codable 协议

```swift
struct AppConfiguration: Codable {
    var theme: String
    var fontSize: Double
    var bookmarks: [Bookmark]
    var lastSyncDate: Date?
}

struct Bookmark: Codable {
    var title: String
    var url: String
    var createdAt: Date
}
```

### 5.2 自定义编解码

```swift
struct User: Codable {
    let id: Int
    let name: String
    let avatarURL: URL?

    enum CodingKeys: String, CodingKey {
        case id
        case name = "display_name"  // JSON key 映射
        case avatarURL = "avatar_url"
    }

    init(from decoder: Decoder) throws {
        let container = try decoder.container(keyedBy: CodingKeys.self)
        id = try container.decode(Int.self, forKey: .id)
        name = try container.decode(String.self, forKey: .name)
        avatarURL = try container.decodeIfPresent(URL.self, forKey: .avatarURL)
    }
}
```

### 5.3 持久化模式

将 Codable 对象序列化为 JSON 写入文件：

```swift
class JSONStore<T: Codable> {
    private let fileURL: URL

    init(filename: String) {
        let dir = FileManager.default.urls(for: .applicationSupportDirectory, in: .userDomainMask).first!
        try? FileManager.default.createDirectory(at: dir, withIntermediateDirectories: true)
        fileURL = dir.appendingPathComponent("\(filename).json")
    }

    func save(_ object: T) throws {
        let data = try JSONEncoder().encode(object)
        try data.write(to: fileURL, options: [.atomic, .completeFileProtectionUnlessOpen])
    }

    func load() -> T? {
        guard let data = try? Data(contentsOf: fileURL) else { return nil }
        return try? JSONDecoder().decode(T.self, from: data)
    }
}
```

## 六、SQLite 与 WCDB

### 6.1 SQLite3 C API

iOS 自带 SQLite3 库，但 C API 使用繁琐：

```swift
import SQLite3

var db: OpaquePointer?
sqlite3_open(dbPath, &db)

var statement: OpaquePointer?
let sql = "CREATE TABLE IF NOT EXISTS users (id INTEGER PRIMARY KEY, name TEXT, age INTEGER)"
sqlite3_prepare_v2(db, sql, -1, &statement, nil)
sqlite3_step(statement)
sqlite3_finalize(statement)
```

### 6.2 WCDB

WCDB（微信数据库）是腾讯开源的 SQLite 封装库，提供了 Swift/Objective-C 接口，性能优秀且接口友好：

```swift
import WCDBSwift

class User: TableCodable {
    var id: Int = 0
    var name: String = ""
    var age: Int = 0

    enum CodingKeys: String, CodingKey {
        case id
        case name
        case age
    }
    isAutoIncrement: true  // id 自增
}

// 创建数据库
let database = Database(at: dbPath)

// 创建表
try database.create(table: "users", of: User.self)

// 插入
let user = User(id: 1, name: "张三", age: 25)
try database.insert(user, intoTable: "users")

// 查询
let users: [User] = try database.getObjects(fromTable: "users", where: User.Properties.age > 20)

// 更新
try database.update(table: "users", on: User.Properties.name, with: "李四", where: User.Properties.id == 1)

// 删除
try database.delete(fromTable: "users", where: User.Properties.id == 1)
```

WCDB 的优势：基于 SQLite 但性能更好（批量操作优化），线程安全，支持加密（WCDBRepair），内置修复机制，Swift API 遵循 Codable。

### 6.3 索引优化

```swift
// 创建索引
try database.create(index: "idx_user_name", on: User.Properties.name, in: "users")

// 复合索引
try database.create(index: "idx_name_age", on: [User.Properties.name, User.Properties.age], in: "users")
```

索引选择原则：经常用于 WHERE 条件、ORDER BY、JOIN 的列考虑加索引。索引增加写入开销，不要盲目加索引。复合索引遵循最左前缀原则。

## 七、Core Data

### 7.1 Core Data 架构

Core Data 不是数据库，而是一个对象图管理框架。它的核心组件：

- **NSManagedObjectModel**：数据模型描述（.xcdatamodeld 文件）
- **NSPersistentStoreCoordinator**：协调模型与底层存储（SQLite/内存/XML）
- **NSManagedObjectContext**：工作区，所有增删改查在此进行
- **NSManagedObject**：实体对象

### 7.2 基本使用

```swift
// 创建 Core Data Stack（iOS 10+ 简化版）
lazy var persistentContainer: NSPersistentContainer = {
    let container = NSPersistentContainer(name: "MyApp")
    container.loadPersistentStores { _, error in
        if let error = error { fatalError("Core Data error: \(error)") }
    }
    return container
}()

var viewContext: NSManagedObjectContext {
    persistentContainer.viewContext
}

// 插入
let user = User(context: viewContext)
user.name = "张三"
user.age = 25
try viewContext.save()

// 查询
let request: NSFetchRequest<User> = User.fetchRequest()
request.predicate = NSPredicate(format: "age > %d", 20)
request.sortDescriptors = [NSSortDescriptor(key: "name", ascending: true)]
let users = try viewContext.fetch(request)

// 删除
viewContext.delete(user)
try viewContext.save()
```

### 7.3 关系建模

Core Data 支持一对多、多对多关系：

```swift
// 在 .xcdatamodeld 中定义关系
// Department <<-->> Employee（一对多）
// Employee --> Department（多对一）

// 使用关系
let dept = Department(context: viewContext)
dept.name = "Engineering"

let emp = Employee(context: viewContext)
emp.name = "张三"
emp.department = dept  // 设置关系
```

### 7.4 后台上下文

耗时操作应在后台上下文中执行：

```swift
persistentContainer.performBackgroundTask { backgroundContext in
    // 在后台线程操作 Core Data
    let user = User(context: backgroundContext)
    user.name = "李四"
    try backgroundContext.save()

    DispatchQueue.main.async {
        // 更新 UI
    }
}
```

### 7.5 Core Data vs SQLite

Core Data 适合：复杂对象关系图、需要撤销/重做、需要自动缓存管理的场景。

SQLite/WCDB 适合：简单表结构、高性能批量操作、跨平台数据格式、已有 SQL 经验的团队。

Core Data 的学习成本较高，但功能强大。对于简单需求，SQLite 或 Codable+JSON 往往更实用。

## 八、持久化方案选择决策

1. 几个配置项 → UserDefaults
2. 密码/Token → Keychain
3. 用户下载的文件/图片 → 文件沙盒 Documents 或 Caches
4. 中等量结构化数据（几十到几百个对象）→ Codable + JSON 文件
5. 大量结构化数据（上千条记录）→ SQLite / WCDB
6. 复杂对象关系、需要撤销重做 → Core Data
7. App Extension 共享数据 → App Groups + 对应存储方案
