# 无障碍开发详解

## 一、无障碍的意义

无障碍（Accessibility，缩写 a11y）是指让残障人士也能使用 App。iOS 提供了系统级无障碍支持，开发者只需正确配置即可让 App 被 VoiceOver、Voice Control、Switch Control 等辅助技术使用。

无障碍不仅是社会责任，在部分市场也是法律要求。美国 ADA 法案、欧盟 European Accessibility Act 都要求 App 满足无障碍标准。此外，无障碍也惠及临时性障碍用户（如单手操作、强光下看不清屏幕）和老年人。

## 二、VoiceOver 基础

### 2.1 VoiceOver 工作原理

VoiceOver 是 iOS 内置的屏幕阅读器。它通过语音朗读屏幕上的元素，盲人用户通过触摸浏览界面。VoiceOver 将 UI 元素转换为无障碍元素（Accessibility Element），逐个朗读。

开启 VoiceOver：设置 → 辅助功能 → VoiceOver。或在 Xcode 中打开 Accessibility Inspector。

### 2.2 VoiceOver 手势

- 单指触摸：朗读手指下方的元素
- 单指左右滑动：切换到上一个/下一个元素
- 单指双击：激活当前元素（等同点击）
- 双指双击：开始/停止朗读
- 三指上滑：朗读从顶部开始
- 三指下滑：滚动到下一页

## 三、无障碍属性配置

### 3.1 核心属性

```swift
// UILabel 通常不需要额外配置，text 自动作为无障碍标签
let label = UILabel()
label.text = "用户名"  // VoiceOver 朗读 "用户名"

// UIImageView 需要配置
let imageView = UIImageView()
imageView.image = UIImage(named: "profile")
imageView.isAccessibilityElement = true
imageView.accessibilityLabel = "用户头像"
imageView.accessibilityTraits = .image

// UIButton 通常自动读取 title 作为标签
let button = UIButton()
button.setTitle("登录", for: .normal)
// VoiceOver 朗读 "登录，按钮"
```

### 3.2 accessibilityLabel

无障碍标签是 VoiceOver 朗读的主要内容。对于纯视觉元素（图标、图片），必须提供：

```swift
// 图标按钮
let settingsButton = UIButton()
settingsButton.setImage(UIImage(systemName: "gear"), for: .normal)
settingsButton.isAccessibilityElement = true
settingsButton.accessibilityLabel = "设置"

// 不朗读图片的装饰性元素
decorativeImageView.isAccessibilityElement = false
```

### 3.3 accessibilityValue

表示元素的当前值（如滑块位置、开关状态）：

```swift
let slider = UISlider()
slider.accessibilityLabel = "音量"
slider.accessibilityValue = "50%"  // 自定义朗读值

let toggle = UISwitch()
toggle.accessibilityLabel = "推送通知"
// Switch 自动朗读 "开" 或 "关"
```

### 3.4 accessibilityTraits

描述元素的类型和行为，VoiceOver 据此决定朗读方式和交互：

```swift
element.accessibilityTraits = [.button, .selected]
```

常用 traits：`.button`（按钮）、`.image`（图片）、`.header`（标题）、`.selected`（选中状态）、`.notEnabled`（禁用）、`.adjustable`（可调节，如滑块）、`.link`（链接）、`.searchField`（搜索框）、`.keyboardKey`（键盘按键）。

### 3.5 accessibilityHint

提供额外的操作说明，当 label 不够明确时使用：

```swift
button.accessibilityLabel = "收藏"
button.accessibilityHint = "双击以收藏或取消收藏此文章"
```

Hint 应简洁，只在必要时使用。如果 label 已经清楚表达了功能，就不需要 hint。

## 四、组合与分组

### 4.1 将多个元素组合为一个无障碍元素

```swift
// 将头像、用户名、描述组合为一个元素
let containerView = UIView()
containerView.isAccessibilityElement = true
containerView.accessibilityLabel = "张三，软件工程师，查看个人资料"
containerView.accessibilityTraits = .button

// 子元素标记为非无障碍元素
avatarImageView.isAccessibilityElement = false
nameLabel.isAccessibilityElement = false
descLabel.isAccessibilityElement = false
```

### 4.2 自定义无障碍元素

当视图内容不是标准的 UI 元素时，可以通过实现 `UIAccessibilityContainer` 协议提供自定义无障碍元素：

```swift
class CustomChartView: UIView {
    var dataPoints: [(label: String, value: Double)] = []

    override var accessibilityElements: [Any]? {
        get {
            dataPoints.map { point in
                let element = UIAccessibilityElement(accessibilityContainer: self)
                element.accessibilityLabel = "\(point.label): \(Int(point.value * 100))%"
                element.accessibilityFrameInContainerSpace = frameForDataPoint(point)
                return element
            }
        }
        set {}
    }
}
```

## 五、无障碍通知

### 5.1 通知 VoiceOver 状态变化

当 UI 异步更新时，通知 VoiceOver：

```swift
// 通知布局变化
UIAccessibility.post(notification: .layoutChanged, argument: newElement)

// 通知屏幕变化（如 push/present）
UIAccessibility.post(notification: .screenChanged, argument: nil)

// 朗读自定义文本
UIAccessibility.post(notification: .announcement, argument: "加载完成")

// 页面切换时聚焦到新页面的第一个元素
UIAccessibility.post(notification: .screenChanged, argument: firstElementOnNewPage)
```

### 5.2 路由通知

引导 VoiceOver 聚焦到特定元素：

```swift
// 焦点跳转到搜索框
UIAccessibility.post(notification: .layoutChanged, argument: searchTextField)
```

## 六、Dynamic Type（动态字体）

### 6.1 支持动态字体

```swift
// 使用系统字体方法（自动响应 Dynamic Type）
label.font = UIFont.preferredFont(forTextStyle: .body)
label.adjustsFontForContentSizeCategory = true  // 必须设置

// SwiftUI 中
Text("Hello")
    .font(.body)  // 自动响应 Dynamic Type
```

### 6.2 文本样式

```swift
UIFont.TextStyle.case: Large Title (标题)
UIFont.TextStyle.title1, title2, title3: 标题层级
UIFont.TextStyle.headline: 粗体标题
UIFont.TextStyle.body: 正文（默认）
UIFont.TextStyle.callout: 标注
UIFont.TextStyle.subheadline: 副标题
UIFont.TextStyle.footnote: 脚注
UIFont.TextStyle.caption1, caption2: 说明文字
```

### 6.3 监听字体大小变化

```swift
// UIKit
override func traitCollectionDidChange(_ previousTraitCollection: UITraitCollection?) {
    super.traitCollectionDidChange(previousTraitCollection)
    if traitCollection.preferredContentSizeCategory != previousTraitCollection?.preferredContentSizeCategory {
        updateLayoutForFontChange()
    }
}

// SwiftUI 自动响应
```

### 6.4 自定义字体与 Dynamic Type

```swift
// 使用 UIFontMetrics 缩放自定义字体
if let customFont = UIFont(name: "CustomFont", size: 17) {
    let metrics = UIFontMetrics(forTextStyle: .body)
    label.font = metrics.scaledFont(for: customFont)
    label.adjustsFontForContentSizeCategory = true
}
```

## 七、降低运动

### 7.1 检测设置

```swift
// UIKit
if UIAccessibility.isReduceMotionEnabled {
    // 减少动画
    UIView.animate(withDuration: 0) {
        view.alpha = 1.0
    }
} else {
    UIView.animate(withDuration: 0.3) {
        view.alpha = 1.0
    }
}

// SwiftUI
@Environment(\.accessibilityReduceMotion) var reduceMotion

if reduceMotion {
    // 无动画
} else {
    // 有动画
}
```

### 7.2 减少透明度

```swift
if UIAccessibility.isReduceTransparencyEnabled {
    // 使用不透明背景
    view.backgroundColor = .white
} else {
    // 使用模糊效果
    view.backgroundColor = .clear
    let blurEffect = UIBlurEffect(style: .systemMaterial)
    let blurView = UIVisualEffectView(effect: blurEffect)
    view.insertSubview(blurView, at: 0)
}
```

## 八、无障碍测试

### 8.1 Accessibility Inspector

Xcode 自带的 Accessibility Inspector 工具可以检查无障碍配置：

- 打开 Xcode → Open Developer Tool → Accessibility Inspector
- 选择目标设备/模拟器
- 鼠标悬停 UI 元素，检查 label、traits、hint 是否正确
- 运行 Audit 自动检测常见问题

### 8.2 VoiceOver 测试清单

每个界面应完成的 VoiceOver 测试：

1. 关闭屏幕看（或闭上眼），只用 VoiceOver 完成核心操作
2. 检查每个可交互元素是否有清晰的 label
3. 检查元素朗读顺序是否符合视觉顺序
4. 检查表单输入是否可完成（包括错误提示）
5. 检查动态内容更新是否有通知
6. 检查模态/弹窗的聚焦管理
7. 检查回到上一页后的焦点恢复

### 8.3 自动化检测

```swift
// 在 UI 测试中验证无障碍属性
func testLoginButtonAccessibility() {
    let app = XCUIApplication()
    app.launch()

    let loginButton = app.buttons["loginButton"]
    XCTAssertTrue(loginButton.exists)
    XCTAssertEqual(loginButton.label, "登录")
}
```

## 九、SwiftUI 无障碍

SwiftUI 自动为大部分组件提供无障碍支持，但仍需要配置：

```swift
Image("profile")
    .accessibilityLabel("用户头像")

HStack {
    Image(systemName: "star.fill")
    Text("4.5")
    Text("(128 reviews)")
}
.accessibilityElement(children: .combine)  // 合并为一个元素
.accessibilityLabel("评分 4.5，128 条评价")

Button(action: {}) {
    Image(systemName: "heart")
}
.accessibilityLabel("收藏")
.accessibilityHint("双击以收藏或取消收藏")
.accessibilityAddTraits(.isButton)

// 隐藏装饰性元素
Image("background_pattern")
    .accessibilityHidden(true)
```

SwiftUI 的 `.accessibilityElement(children:)` 模式：`.combine` 合并子元素朗读，`.contain` 容器模式，`.ignore` 忽略子元素。
