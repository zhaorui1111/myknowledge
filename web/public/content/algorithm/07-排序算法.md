# 排序算法（Sorting Algorithms）详解

> **排序**（Sorting）是将一组记录按某个关键字（key）的非递减或非递增顺序重新排列的过程，是计算机科学中研究最透彻、应用最广泛的基础算法群。从冒泡、插入这样的「初级排序」，到快排、归并、堆排这样的「高级排序」，再到计数、基数、桶排这样的「非比较排序」，排序算法浓缩了分治、递归、贪心、数据结构设计、缓存友好性、稳定性权衡等几乎所有算法核心思想。本文从第一性原理出发，把十大经典排序逐一讲透、写对、比清楚，并给出 Swift 与 Objective-C 双语实现、复杂度推导、工程实践（含 iOS `sort` 底层）、典型面试题与易错点。

## 1. 排序问题的定义与衡量维度

### 1.1 形式化定义

给定一个含 `n` 个元素的序列 `a[0..n-1]`，以及一个定义在元素上的**全序关系**（total order）`≤`（满足自反性、反对称性、传递性、完全性），**排序**要求输出一个原序列的排列（permutation）`a[p0], a[p1], ..., a[p(n-1)]`，使得：

```
a[p0] ≤ a[p1] ≤ ... ≤ a[p(n-1)]
```

即结果**单调非递减**（升序）。降序则要求单调非递增。这里强调「排列」二字：排序不能增删元素，输出必须是输入的重排。

### 1.2 衡量一个排序算法的六个维度

评价排序算法绝不只看「快不快」，工程上要综合权衡：

| 维度 | 含义 | 为什么重要 |
| --- | --- | --- |
| **时间复杂度** | 关键字比较 + 元素移动的次数随 n 的增长阶 | 决定大数据量下能否跑得动 |
| **空间复杂度** | 除输入外额外占用的内存 | 内存受限场景（嵌入式、移动端）关键 |
| **稳定性**（stability） | 相等元素的相对次序是否保持 | 多关键字排序、增量更新场景必需 |
| **原地性**（in-place） | 是否只用 O(1) 额外空间 | 大数组排序避免复制开销 |
| **自适应性**（adaptive） | 对「近乎有序」的输入是否更快 | 现实数据常部分有序 |
| **缓存友好性** | 访问内存是否连续、可预测 | 现代 CPU 上常数因子的决定性因素 |

> **关键洞察**：算法导论里的复杂度只刻画「增长阶」，但在真实机器上，**常数因子**和**缓存命中率**往往决定胜负。这就是为什么 O(n²) 的插入排序在小数组上能打败 O(n log n) 的快排——这也是工业级排序「混合策略」的根源。

### 1.3 内部排序 vs 外部排序

- **内部排序**（internal sort）：所有数据都能一次性装入内存。本文讨论的均为内部排序。
- **外部排序**（external sort）：数据量超过内存，需借助磁盘，典型方法是「多路归并」（k-way merge）。大数据 `sort` 命令、数据库的 `ORDER BY` 在数据超内存时都走外部排序。

## 2. 稳定性、原地性与比较模型

### 2.1 稳定性的精确定义

设排序前两个相等元素 `a[i] == a[j]` 且 `i < j`。若排序后 `a[i]` 仍排在 `a[j]` 之前，则该算法是**稳定的**（stable）。

**为什么稳定性重要？** 看一个真实例子：先按「价格」排序商品，再按「销量」排序。如果第二次排序是稳定的，那么「销量相同的商品」会保持「价格有序」——这就实现了「主关键字销量、次关键字价格」的多级排序。若不稳定，第二次排序会打乱第一次的结果。

```
原始（已按价格升序）：
  (销量=5, 价格=10), (销量=3, 价格=20), (销量=5, 价格=30)
按销量升序稳定排序后：
  (销量=3, 价格=20), (销量=5, 价格=10), (销量=5, 价格=30)
                       └─ 销量相同的两个，价格仍是 10 在 30 前（保持） ✓
```

**记忆口诀**：稳定的排序有「插冒归计基桶」（插入、冒泡、归并、计数、基数、桶）；不稳定的有「选快希堆」（选择、快速、希尔、堆）。

### 2.2 原地排序

**原地排序**（in-place sort）指除输入数组外，只使用 O(1) 或 O(log n)（递归栈）的额外空间。

- 原地：冒泡、选择、插入、希尔、快速（递归栈 O(log n)）、堆排序。
- 非原地：归并（O(n) 辅助数组）、计数、基数、桶。

> **注意辨析**：快速排序常被说成「原地」，但其递归调用栈占用 O(log n)（最坏 O(n)）空间，严格说是「准原地」。归并排序虽然有 O(n) 空间的「数组归并」版本，但也存在 O(1) 空间的「原地归并」变体（实现复杂、常数大，工程少用）。

### 2.3 比较排序 vs 非比较排序

- **比较排序**（comparison sort）：只通过元素间的「两两比较」获取顺序信息，不假设元素的具体取值。冒泡、选择、插入、希尔、归并、快速、堆都是比较排序。其时间下界为 **Ω(n log n)**（见第 13 章证明）。
- **非比较排序**（non-comparison sort）：利用元素的「数值特征」（如整数的位、值域），不做元素间比较。计数、基数、桶属于此类，可突破 O(n log n) 达到线性，但对数据有额外假设（值域有限、可分桶等）。

### 2.4 排序在真实系统中的应用全景

排序不是孤立的算法练习，它是无数上层算法与系统的**基石**：

- **数据库**：`ORDER BY` 直接是排序；`GROUP BY`、`DISTINCT`、排序合并连接（Sort-Merge Join）都依赖排序；B+ 树索引本质是「持续维护有序」。
- **搜索引擎**：检索结果按相关度（如 TF-IDF、BM25、机器学习排序 LTR）排序；倒排索引归并多个 posting list 时用到多路归并。
- **图形渲染**：画家算法按深度排序图元；半透明物体必须从后往前排序绘制。
- **操作系统**：进程调度（按优先级/时间片）、磁盘 I/O 调度（电梯算法）本质是排序/优先队列。
- **算法预处理**：区间合并、贪心选择、最近点对、扫描线、双指针——大量算法第一步就是「先排序」，把无序问题转化为有序问题后线性扫描即可。
- **去重与统计**：排序后相等元素相邻，去重、求众数、求第 K 大都变得简单。
- **iOS / 客户端**：列表展示排序（按时间/距离/价格）、`NSFetchRequest` 的 `sortDescriptors`、Collection 数据源排序等，几乎每个 App 都在用。

正因排序如此基础且高频，理解其原理与权衡，是写出高性能、正确程序的必修课。下面逐一展开十大经典排序。

## 3. 冒泡排序（Bubble Sort）

### 3.1 是什么

**冒泡排序**（Bubble Sort）：重复遍历待排序序列，每次比较相邻两个元素，若顺序错误就交换，使得每一趟都把当前未排定部分的「最大值」像气泡一样「冒」到末尾。

```
一趟冒泡（升序）：把最大的 9 冒到最右
[5 2 9 1 7]
 ^^^ 5>2? 是 → 交换 → [2 5 9 1 7]
   ^^^ 5>9? 否 → 不动 → [2 5 9 1 7]
     ^^^ 9>1? 是 → 交换 → [2 5 1 9 7]
       ^^^ 9>7? 是 → 交换 → [2 5 1 7 9]   ← 9 已就位
```

### 3.2 为什么 / 原理

每趟遍历相邻比较，逆序对会被逐步消除。第 `k` 趟结束后，最后 `k` 个元素已是全局有序的最大几个。共需最多 `n-1` 趟。

### 3.3 怎么用（Swift 实现）

```swift
/// 冒泡排序（升序，稳定，原地）
/// - 复杂度：时间 O(n²)，最好 O(n)（已优化），空间 O(1)
func bubbleSort(_ nums: inout [Int]) {
    let n = nums.count
    guard n > 1 else { return }
    for i in 0..<(n - 1) {
        // swapped 优化：若某趟没有发生交换，说明已有序，提前结束 → 最好 O(n)
        var swapped = false
        // 每趟后末尾 i 个已就位，无需再比较
        for j in 0..<(n - 1 - i) {
            if nums[j] > nums[j + 1] {
                nums.swapAt(j, j + 1)
                swapped = true
            }
        }
        if !swapped { break }
    }
}
```

### 3.4 Objective-C 实现

```objc
// 冒泡排序（升序，稳定，原地）
// 传入 NSMutableArray<NSNumber *>，行为与 Swift 版一致
- (void)bubbleSort:(NSMutableArray<NSNumber *> *)nums {
    NSUInteger n = nums.count;
    if (n <= 1) return;
    for (NSUInteger i = 0; i + 1 < n; i++) {
        BOOL swapped = NO;
        for (NSUInteger j = 0; j + 1 < n - i; j++) {
            if ([nums[j] integerValue] > [nums[j + 1] integerValue]) {
                [nums exchangeObjectAtIndex:j withObjectAtIndex:j + 1];
                swapped = YES;
            }
        }
        if (!swapped) break;
    }
}
```

> **OC 注意**：循环条件用 `j + 1 < n - i` 而非 `j < n - 1 - i`，因为 `NSUInteger` 是无符号类型，当 `n - 1 - i` 在 `i` 较大时若写成 `n - 1 - i` 本身没问题（这里 i 最大为 n-2，n-1-i ≥ 1），但养成「移项避免下溢」的习惯更安全。

### 3.5 复杂度与特性

- **时间**：最坏/平均 O(n²)（完全逆序时比较 `n(n-1)/2` 次）；最好 O(n)（已有序 + swapped 优化，一趟即退出）。
- **空间**：O(1)，原地。
- **稳定性**：稳定（`>` 而非 `>=`，相等不交换）。
- **自适应**：是（swapped 优化后对近乎有序数据更快）。

### 3.6 常见坑

- 把 `if nums[j] > nums[j+1]` 写成 `>=` → **破坏稳定性**。
- 忘记 `swapped` 优化 → 对已有序数据仍跑满 O(n²)。
- 内层边界 `n - 1 - i` 写错导致越界或漏比。

## 4. 选择排序（Selection Sort）

### 4.1 是什么

**选择排序**（Selection Sort）：每趟从未排序区间中选出最小元素，放到已排序区间的末尾。与冒泡的区别是——选择排序每趟只交换一次（找到最小值后才交换），交换次数为 O(n)，但比较次数恒为 O(n²)。

### 4.2 原理与图示

```
已排序 | 未排序
[ ]    | [5 2 9 1 7]   选最小 1 → 与首位 5 交换
[1]    | [2 9 5 7]     选最小 2 → 已在首位，不动
[1 2]  | [9 5 7]       选最小 5 → 与 9 交换
...
```

### 4.3 Swift 实现

```swift
/// 选择排序（升序，不稳定，原地）
/// - 复杂度：时间恒 O(n²)，空间 O(1)，交换次数 O(n)
func selectionSort(_ nums: inout [Int]) {
    let n = nums.count
    guard n > 1 else { return }
    for i in 0..<(n - 1) {
        var minIdx = i
        for j in (i + 1)..<n {
            if nums[j] < nums[minIdx] {
                minIdx = j
            }
        }
        if minIdx != i {
            nums.swapAt(i, minIdx)
        }
    }
}
```

### 4.4 Objective-C 实现

```objc
// 选择排序（升序，不稳定，原地）
- (void)selectionSort:(NSMutableArray<NSNumber *> *)nums {
    NSUInteger n = nums.count;
    if (n <= 1) return;
    for (NSUInteger i = 0; i + 1 < n; i++) {
        NSUInteger minIdx = i;
        for (NSUInteger j = i + 1; j < n; j++) {
            if ([nums[j] integerValue] < [nums[minIdx] integerValue]) {
                minIdx = j;
            }
        }
        if (minIdx != i) {
            [nums exchangeObjectAtIndex:i withObjectAtIndex:minIdx];
        }
    }
}
```

### 4.5 为什么选择排序不稳定？

经典反例：`[5a, 5b, 2]`（5a、5b 是两个相等的 5，下标区分）。第一趟选最小 2，与首位 5a 交换 → `[2, 5b, 5a]`。原本 5a 在 5b 前，交换后 5a 跑到了 5b 后 → **相对次序被破坏**，故不稳定。

> 用「链表 + 不交换而是移动节点」可以实现稳定的选择排序，但数组上的标准实现不稳定。

### 4.6 复杂度与特性

- **时间**：恒 O(n²)（即使已有序也要找最小，比较次数不变）。
- **空间**：O(1)。
- **稳定性**：不稳定。
- **交换次数**：O(n)（每趟至多一次）——当「交换代价远大于比较代价」（如交换大对象）时，选择排序比冒泡有优势。
- **自适应**：否。

## 5. 插入排序（Insertion Sort）

### 5.1 是什么

**插入排序**（Insertion Sort）：将序列分为「已排序」和「未排序」两部分，每次取未排序的第一个元素，在已排序部分从后往前扫描，找到合适位置插入。类似打扑克时整理手牌。

```
[ 5 | 2 9 1 7 ]   取 2，插到 5 前 → [2 5 | 9 1 7]
[2 5 | 9 1 7]     取 9，9>5 不动   → [2 5 9 | 1 7]
[2 5 9 | 1 7]     取 1，一路前移   → [1 2 5 9 | 7]
[1 2 5 9 | 7]     取 7，插到 5 9 间 → [1 2 5 7 9]
```

### 5.2 为什么插入排序在小数组上极快

插入排序对「近乎有序」的数据**自适应**：若元素已在正确位置附近，内层循环几乎不移动。其常数因子极小、缓存友好（只访问相邻内存），所以工业级排序在**子数组足够小**时切换到插入排序。

### 5.3 Swift 实现

```swift
/// 插入排序（升序，稳定，原地，自适应）
/// - 复杂度：最坏/平均 O(n²)，最好 O(n)，空间 O(1)
func insertionSort(_ nums: inout [Int]) {
    let n = nums.count
    guard n > 1 else { return }
    for i in 1..<n {
        let key = nums[i]
        var j = i - 1
        // 把所有 > key 的元素后移一位；用 > 保证稳定
        while j >= 0 && nums[j] > key {
            nums[j + 1] = nums[j]
            j -= 1
        }
        nums[j + 1] = key
    }
}
```

### 5.4 Objective-C 实现

```objc
// 插入排序（升序，稳定，原地，自适应）
- (void)insertionSort:(NSMutableArray<NSNumber *> *)nums {
    NSUInteger n = nums.count;
    if (n <= 1) return;
    for (NSUInteger i = 1; i < n; i++) {
        NSNumber *key = nums[i];
        NSInteger j = (NSInteger)i - 1;   // 用有符号 NSInteger 防止 j 下溢
        while (j >= 0 && [nums[j] integerValue] > [key integerValue]) {
            nums[j + 1] = nums[j];
            j -= 1;
        }
        nums[j + 1] = key;
    }
}
```

> **OC 关键坑**：`j` 必须用有符号 `NSInteger`！若用 `NSUInteger`，当 `j` 减到 0 再减 1 会回绕成巨大正数 → 死循环或越界。这是无符号类型最经典的陷阱。

### 5.5 二分插入排序（优化比较次数）

用二分查找定位插入点，把「比较次数」从 O(n²) 降到 O(n log n)，但「移动次数」仍是 O(n²)（数组要挪位），整体时间仍 O(n²)。仅当「比较代价远大于移动代价」时有意义。

```swift
/// 二分插入排序：比较 O(n log n)，移动 O(n²)
func binaryInsertionSort(_ nums: inout [Int]) {
    let n = nums.count
    for i in 1..<n {
        let key = nums[i]
        // 在 [0, i) 中二分找第一个 > key 的位置（lower_bound for upper）
        var lo = 0, hi = i
        while lo < hi {
            let mid = lo + (hi - lo) / 2
            if nums[mid] <= key { lo = mid + 1 } else { hi = mid }
        }
        // 把 [lo, i) 整体后移一位，key 放到 lo
        var j = i
        while j > lo { nums[j] = nums[j - 1]; j -= 1 }
        nums[lo] = key
    }
}
```

> 注意这里用 `nums[mid] <= key` 才能保证「相等元素插到已存在相等元素之后」→ 稳定。

### 5.6 复杂度与特性

- **时间**：最坏/平均 O(n²)，最好 O(n)（已有序）。
- **空间**：O(1)。
- **稳定性**：稳定。
- **自适应**：是（强，逆序对越少越快；时间约为 O(n + d)，d 为逆序对数）。

## 6. 希尔排序（Shell Sort）

### 6.1 是什么

**希尔排序**（Shell Sort，1959 年 Donald Shell 提出），又称「缩小增量排序」（diminishing increment sort）。它是**插入排序的改进版**：先按一个较大的「增量 gap」把数组分成若干子序列分别做插入排序，然后逐步缩小 gap，最后 gap=1 时做一次普通插入排序。

### 6.2 为什么希尔排序更快

普通插入排序的瓶颈是「元素每次只能移动一位」，对远离最终位置的元素很慢。希尔排序用大 gap 让元素「大步跳跃」靠近最终位置，使数组快速变得「近乎有序」，最后 gap=1 时插入排序几乎不用移动。**核心思想：先粗调、再精调**。

```
原始：[8 9 1 7 2 3 5 4 6 0]，gap = 5
分组（步长 5）：(8,3) (9,5) (1,4) (7,6) (2,0) → 各组内插入排序
gap = 2 再分组 → gap = 1 最终一遍插入排序
```

### 6.3 Swift 实现

```swift
/// 希尔排序（升序，不稳定，原地）
/// - 使用经典 gap = n/2 递减序列
/// - 复杂度：与 gap 序列相关，n²/2 序列约 O(n²)；Hibbard 序列 O(n^1.5)
func shellSort(_ nums: inout [Int]) {
    let n = nums.count
    guard n > 1 else { return }
    var gap = n / 2
    while gap > 0 {
        // 对每个「以 gap 为步长」的子序列做插入排序
        for i in gap..<n {
            let key = nums[i]
            var j = i - gap
            while j >= 0 && nums[j] > key {
                nums[j + gap] = nums[j]
                j -= gap
            }
            nums[j + gap] = key
        }
        gap /= 2
    }
}
```

### 6.4 Objective-C 实现

```objc
// 希尔排序（升序，不稳定，原地）
- (void)shellSort:(NSMutableArray<NSNumber *> *)nums {
    NSInteger n = (NSInteger)nums.count;
    if (n <= 1) return;
    for (NSInteger gap = n / 2; gap > 0; gap /= 2) {
        for (NSInteger i = gap; i < n; i++) {
            NSNumber *key = nums[i];
            NSInteger j = i - gap;
            while (j >= 0 && [nums[j] integerValue] > [key integerValue]) {
                nums[j + gap] = nums[j];
                j -= gap;
            }
            nums[j + gap] = key;
        }
    }
}
```

### 6.5 增量序列与复杂度

希尔排序的复杂度**强依赖于 gap 序列**，这也是它在算法分析史上著名的「未完全解决的问题」：

| 增量序列 | 时间复杂度（最坏） |
| --- | --- |
| Shell 原始 `n/2, n/4, ..., 1` | O(n²) |
| Hibbard `2^k - 1` | O(n^1.5) |
| Sedgewick | O(n^4/3) |
| Knuth `(3^k - 1)/2` | 实测较好 |

- **空间**：O(1)。
- **稳定性**：不稳定（跨 gap 的交换会打乱相等元素相对位置）。
- **自适应**：部分。

> 希尔排序是「第一个突破 O(n²) 的排序算法」，历史意义重大；工程上已被快排/归并取代，但在嵌入式等场景因其原地、代码短小、无递归栈而仍有应用价值。

## 7. 归并排序（Merge Sort）

### 7.1 是什么

**归并排序**（Merge Sort，冯·诺依曼 1945 年提出）是**分治**（Divide and Conquer）思想的典范：把数组从中间一分为二，递归地分别排序两个子数组，再把两个**有序**子数组「归并」（merge）成一个有序数组。

```
分（divide）：              合（merge）：
[8 3 5 1 9 2]              [1 2 3 5 8 9]
   /      \                   /      \
[8 3 5]  [1 9 2]         [3 5 8]   [1 2 9]
 / \      / \             / \       / \
[8][3 5] [1][9 2]   →   [8][3 5]  [1][2 9]  →  ...
```

### 7.2 原理：归并两个有序数组

归并是归并排序的核心。给定两个有序数组 L、R，用双指针各指向头部，每次取较小者放入结果，直到一方耗尽，再把另一方剩余追加。**比较时用 `<=` 保证稳定**（相等时优先取左半部分，即原数组中靠前的元素）。

### 7.3 Swift 实现（自顶向下，递归）

```swift
/// 归并排序（升序，稳定，非原地 O(n)）
/// - 复杂度：时间恒 O(n log n)（最好=最坏=平均），空间 O(n)
func mergeSort(_ nums: [Int]) -> [Int] {
    let n = nums.count
    guard n > 1 else { return nums }
    let mid = n / 2
    let left = mergeSort(Array(nums[0..<mid]))
    let right = mergeSort(Array(nums[mid..<n]))
    return merge(left, right)
}

/// 归并两个有序数组（稳定：相等取左）
private func merge(_ left: [Int], _ right: [Int]) -> [Int] {
    var result = [Int]()
    result.reserveCapacity(left.count + right.count)
    var i = 0, j = 0
    while i < left.count && j < right.count {
        if left[i] <= right[j] {   // <= 保证稳定
            result.append(left[i]); i += 1
        } else {
            result.append(right[j]); j += 1
        }
    }
    // 追加剩余（最多只有一边有剩）
    if i < left.count { result.append(contentsOf: left[i...]) }
    if j < right.count { result.append(contentsOf: right[j...]) }
    return result
}
```

### 7.4 Swift 实现（原地下标版，避免频繁建数组）

上面的版本每层都 `Array(...)` 复制，常数大。工程上用「单块辅助数组 + 下标传递」更高效：

```swift
/// 归并排序（原地下标版，单块辅助数组）
func mergeSortInPlace(_ nums: inout [Int]) {
    let n = nums.count
    guard n > 1 else { return }
    var aux = nums   // 一次性分配辅助数组
    mergeSortHelper(&nums, &aux, 0, n - 1)
}

private func mergeSortHelper(_ a: inout [Int], _ aux: inout [Int], _ lo: Int, _ hi: Int) {
    guard lo < hi else { return }
    let mid = lo + (hi - lo) / 2
    mergeSortHelper(&a, &aux, lo, mid)
    mergeSortHelper(&a, &aux, mid + 1, hi)
    // 小优化：若已有序则跳过归并
    if a[mid] <= a[mid + 1] { return }
    mergeRange(&a, &aux, lo, mid, hi)
}

private func mergeRange(_ a: inout [Int], _ aux: inout [Int], _ lo: Int, _ mid: Int, _ hi: Int) {
    for k in lo...hi { aux[k] = a[k] }   // 复制到辅助区
    var i = lo, j = mid + 1
    for k in lo...hi {
        if i > mid {                 // 左耗尽
            a[k] = aux[j]; j += 1
        } else if j > hi {           // 右耗尽
            a[k] = aux[i]; i += 1
        } else if aux[i] <= aux[j] { // 稳定
            a[k] = aux[i]; i += 1
        } else {
            a[k] = aux[j]; j += 1
        }
    }
}
```

### 7.5 Objective-C 实现

```objc
// 归并排序（升序，稳定）。返回新的有序数组。
- (NSArray<NSNumber *> *)mergeSort:(NSArray<NSNumber *> *)nums {
    if (nums.count <= 1) return nums;
    NSUInteger mid = nums.count / 2;
    NSArray *left  = [self mergeSort:[nums subarrayWithRange:NSMakeRange(0, mid)]];
    NSArray *right = [self mergeSort:[nums subarrayWithRange:NSMakeRange(mid, nums.count - mid)]];
    return [self mergeLeft:left right:right];
}

- (NSArray<NSNumber *> *)mergeLeft:(NSArray<NSNumber *> *)left
                             right:(NSArray<NSNumber *> *)right {
    NSMutableArray *result = [NSMutableArray arrayWithCapacity:left.count + right.count];
    NSUInteger i = 0, j = 0;
    while (i < left.count && j < right.count) {
        if ([left[i] integerValue] <= [right[j] integerValue]) { // <= 稳定
            [result addObject:left[i++]];
        } else {
            [result addObject:right[j++]];
        }
    }
    while (i < left.count)  { [result addObject:left[i++]]; }
    while (j < right.count) { [result addObject:right[j++]]; }
    return result;
}
```

### 7.6 自底向上的非递归归并

避免递归栈，从宽度 1 开始两两归并，宽度翻倍直到覆盖整个数组：

```swift
/// 自底向上归并排序（非递归）
func mergeSortBottomUp(_ nums: inout [Int]) {
    let n = nums.count
    guard n > 1 else { return }
    var aux = nums
    var width = 1
    while width < n {
        var lo = 0
        while lo < n {
            let mid = min(lo + width - 1, n - 1)
            let hi = min(lo + 2 * width - 1, n - 1)
            if mid < hi { mergeRange(&nums, &aux, lo, mid, hi) }
            lo += 2 * width
        }
        width *= 2
    }
}
```

### 7.7 复杂度推导

递归式：`T(n) = 2T(n/2) + O(n)`。由主定理（Master Theorem）`a=2, b=2, f(n)=n`，`n^(log_b a) = n^1 = n`，属 Case 2，故 `T(n) = O(n log n)`。

**直观推导**：递归树共 `log₂n` 层，每层归并总工作量为 O(n)（所有子区间元素之和），故总计 `O(n log n)`。无论输入如何，层数与每层工作量都固定 → 归并排序时间**恒定** O(n log n)，最好=最坏=平均。

- **空间**：O(n)（辅助数组）+ O(log n)（递归栈）= O(n)。
- **稳定性**：稳定（`<=` 取左）。
- **自适应**：基础版否；加「`a[mid] <= a[mid+1]` 跳过」后对有序数据可达近 O(n)。

### 7.8 归并排序的杀手锏：外部排序与链表排序

- **外部排序**：数据超内存时，归并的「顺序访问」特性极其友好——可分块读入、排序、写回，再多路归并。`sort` 命令、数据库都靠它。
- **链表排序**：归并是**唯一**能在 O(1) 额外空间（不含栈）下稳定排序链表的算法（快排在链表上难以原地、堆排需随机访问）。LeetCode 148「排序链表」标准解法即归并。

## 8. 快速排序（Quick Sort）

### 8.1 是什么

**快速排序**（Quick Sort，C.A.R. Hoare 1960 年提出）：选一个「基准」（pivot），通过一趟「划分」（partition）把数组重排成「左边都 ≤ pivot、右边都 ≥ pivot」，pivot 落到最终位置，然后递归排序左右两部分。它是**分治 + 原地**的典范，平均性能最优，是大多数语言标准库的底层实现之一。

### 8.2 原理：划分（Partition）

划分是快排的灵魂，常见两种方案：

**Lomuto 划分**（以末尾为 pivot，易理解）：

```
[3 7 8 5 2 1 9 5 4]  pivot = 4（末尾）
用 i 指向「小于区」末尾，j 扫描；nums[j] < pivot 就交换到小于区
最终把 pivot 换到 i+1，pivot 就位
```

**Hoare 划分**（双指针对撞，交换次数更少，原版）：左右指针向中间逼近，左找 ≥ pivot、右找 ≤ pivot，交换。

### 8.3 Swift 实现（Lomuto + 三数取中 + 随机化）

```swift
/// 快速排序（升序，不稳定，原地）
/// - 复杂度：平均 O(n log n)，最坏 O(n²)（已有序+固定 pivot），空间 O(log n) 栈
func quickSort(_ nums: inout [Int]) {
    guard nums.count > 1 else { return }
    quickSortHelper(&nums, 0, nums.count - 1)
}

private func quickSortHelper(_ a: inout [Int], _ lo: Int, _ hi: Int) {
    // 小区间切插入排序（工程优化，阈值经验值 ~16）
    if hi - lo < 16 {
        insertionSortRange(&a, lo, hi)
        return
    }
    guard lo < hi else { return }
    let p = partition(&a, lo, hi)
    quickSortHelper(&a, lo, p - 1)
    quickSortHelper(&a, p + 1, hi)
}

/// Lomuto 划分，返回 pivot 最终位置
private func partition(_ a: inout [Int], _ lo: Int, _ hi: Int) -> Int {
    // 三数取中 + 随机化，避免对有序/逆序输入退化成 O(n²)
    let mid = lo + (hi - lo) / 2
    let medianIdx = medianOfThree(a, lo, mid, hi)
    a.swapAt(medianIdx, hi)   // 把中位数换到末尾作 pivot
    let pivot = a[hi]
    var i = lo               // i：小于区的下一个位置
    for j in lo..<hi {
        if a[j] < pivot {
            a.swapAt(i, j)
            i += 1
        }
    }
    a.swapAt(i, hi)          // pivot 归位
    return i
}

private func medianOfThree(_ a: [Int], _ x: Int, _ y: Int, _ z: Int) -> Int {
    let (ax, ay, az) = (a[x], a[y], a[z])
    if (ax <= ay && ay <= az) || (az <= ay && ay <= ax) { return y }
    if (ay <= ax && ax <= az) || (az <= ax && ax <= ay) { return x }
    return z
}

private func insertionSortRange(_ a: inout [Int], _ lo: Int, _ hi: Int) {
    guard lo < hi else { return }
    for i in (lo + 1)...hi {
        let key = a[i]
        var j = i - 1
        while j >= lo && a[j] > key { a[j + 1] = a[j]; j -= 1 }
        a[j + 1] = key
    }
}
```

### 8.4 三路划分（处理大量重复元素）

当数组含大量重复值时，普通快排会退化。**三路快排**（Dutch National Flag，荷兰国旗问题）把数组分成 `< pivot`、`== pivot`、`> pivot` 三段，相等区直接跳过，对重复多的数据极快：

```swift
/// 三路快速排序（应对大量重复元素）
func quickSort3Way(_ nums: inout [Int]) {
    guard nums.count > 1 else { return }
    sort3Way(&nums, 0, nums.count - 1)
}

private func sort3Way(_ a: inout [Int], _ lo: Int, _ hi: Int) {
    guard lo < hi else { return }
    let pivot = a[lo + Int.random(in: 0...(hi - lo))]  // 随机 pivot
    var lt = lo      // a[lo..<lt] < pivot
    var gt = hi      // a[gt+1...hi] > pivot
    var i = lo       // a[lt..<i] == pivot，a[i..gt] 待定
    while i <= gt {
        if a[i] < pivot {
            a.swapAt(lt, i); lt += 1; i += 1
        } else if a[i] > pivot {
            a.swapAt(i, gt); gt -= 1   // 注意 i 不前进，换来的元素未检查
        } else {
            i += 1
        }
    }
    sort3Way(&a, lo, lt - 1)
    sort3Way(&a, gt + 1, hi)
}
```

### 8.5 Objective-C 实现（Lomuto + 随机 pivot）

```objc
// 快速排序（升序，不稳定，原地）
- (void)quickSort:(NSMutableArray<NSNumber *> *)nums {
    if (nums.count <= 1) return;
    [self quickSort:nums lo:0 hi:(NSInteger)nums.count - 1];
}

- (void)quickSort:(NSMutableArray<NSNumber *> *)a lo:(NSInteger)lo hi:(NSInteger)hi {
    if (lo >= hi) return;
    NSInteger p = [self partition:a lo:lo hi:hi];
    [self quickSort:a lo:lo hi:p - 1];
    [self quickSort:a lo:p + 1 hi:hi];
}

- (NSInteger)partition:(NSMutableArray<NSNumber *> *)a lo:(NSInteger)lo hi:(NSInteger)hi {
    // 随机选 pivot，换到末尾，防止有序输入退化
    NSInteger r = lo + arc4random_uniform((uint32_t)(hi - lo + 1));
    [a exchangeObjectAtIndex:r withObjectAtIndex:hi];
    NSInteger pivot = [a[hi] integerValue];
    NSInteger i = lo;
    for (NSInteger j = lo; j < hi; j++) {
        if ([a[j] integerValue] < pivot) {
            [a exchangeObjectAtIndex:i withObjectAtIndex:j];
            i++;
        }
    }
    [a exchangeObjectAtIndex:i withObjectAtIndex:hi];
    return i;
}
```

### 8.6 复杂度与最坏情况

- **平均**：`T(n) = 2T(n/2) + O(n) = O(n log n)`，且常数因子小、缓存友好，实测最快。
- **最坏**：每次 pivot 都是最值（如对有序数组用固定首/尾 pivot），划分极不均衡 `T(n) = T(n-1) + O(n) = O(n²)`。**随机化 / 三数取中**可使最坏情况几乎不出现。
- **空间**：递归栈平均 O(log n)，最坏 O(n)（可用「先递归小区间、尾递归大区间」优化到 O(log n)）。
- **稳定性**：不稳定（划分时的远距离交换破坏相对次序）。

### 8.7 快排为什么实践中比归并快

虽然两者平均都是 O(n log n)，但快排：① **原地**，无 O(n) 辅助数组的分配/复制开销；② **缓存友好**，划分是顺序扫描；③ 常数因子更小。代价是不稳定、最坏 O(n²)。这是「平均最优 vs 最坏保证」的经典权衡。

### 8.8 Hoare 划分（双指针对撞，原版更优）

Hoare 在 1960 年的原始划分用两个指针从两端向中间逼近，相比 Lomuto 划分**交换次数约为其 1/3**，对重复元素也更友好（重复元素会被均匀分到两侧，避免极度不平衡）：

```swift
/// Hoare 划分：返回分界点 j，使 a[lo...j] ≤ pivot ≤ a[j+1...hi]
/// 注意：Hoare 划分返回的不是 pivot 的最终位置，递归时用 [lo, j] 与 [j+1, hi]
private func hoarePartition(_ a: inout [Int], _ lo: Int, _ hi: Int) -> Int {
    let pivot = a[lo + (hi - lo) / 2]   // 取中间值作 pivot，避免有序退化
    var i = lo - 1
    var j = hi + 1
    while true {
        repeat { i += 1 } while a[i] < pivot   // 从左找 ≥ pivot
        repeat { j -= 1 } while a[j] > pivot   // 从右找 ≤ pivot
        if i >= j { return j }                 // 指针相遇，返回分界
        a.swapAt(i, j)
    }
}

func quickSortHoare(_ a: inout [Int], _ lo: Int, _ hi: Int) {
    guard lo < hi else { return }
    let p = hoarePartition(&a, lo, hi)
    quickSortHoare(&a, lo, p)        // 注意是 p 而非 p-1
    quickSortHoare(&a, p + 1, hi)
}
```

> **关键差异**：Lomuto 返回 pivot 的最终位置，递归 `[lo, p-1]` 与 `[p+1, hi]`；Hoare 返回分界点 `j`，递归 `[lo, j]` 与 `[j+1, hi]`（pivot 可能仍在待排区间内）。写错边界会死循环或漏排，这是 Hoare 划分最大的坑。

### 8.9 快速选择（Quick Select）—— 不完全排序求第 K 小

快排的「划分」可以单独拿来解「找第 K 小/大」问题：每次划分后，若 pivot 正好落在第 K 位则返回，否则只递归 K 所在的那一侧，平均 O(n)（递归式 `T(n) = T(n/2) + O(n) = O(n)`，等比级数收敛）：

```swift
/// 快速选择：返回升序意义下第 k 小（k 从 0 计）
func quickSelect(_ nums: [Int], _ k: Int) -> Int {
    var a = nums
    var lo = 0, hi = a.count - 1
    while lo < hi {
        let p = partition(&a, lo, hi)   // 复用 Lomuto partition
        if p == k { return a[p] }
        else if p < k { lo = p + 1 }
        else { hi = p - 1 }
    }
    return a[lo]
}
```

> 进阶：**中位数的中位数**（Median of Medians，BFPRT 算法）可把快速选择的**最坏**复杂度也降到 O(n)，代价是常数较大，工程少用但理论重要。

## 9. 堆排序（Heap Sort）

### 9.1 是什么

**堆排序**（Heap Sort）：利用**二叉堆**（binary heap）这种数据结构排序。先把数组「建堆」成一个大顶堆（max-heap，父 ≥ 子），堆顶即最大值；然后反复「把堆顶与末尾交换、堆大小减一、对新堆顶下沉（sift down）恢复堆序」，逐个把最大值放到数组末尾。

### 9.2 二叉堆的数组表示

完全二叉树用数组隐式存储（0-indexed）：

```
节点 i 的左孩子：2i + 1
节点 i 的右孩子：2i + 2
节点 i 的父亲：  (i - 1) / 2
```

### 9.3 Swift 实现

```swift
/// 堆排序（升序，不稳定，原地）
/// - 复杂度：时间恒 O(n log n)，空间 O(1)
func heapSort(_ nums: inout [Int]) {
    let n = nums.count
    guard n > 1 else { return }
    // 1) 建大顶堆：从最后一个非叶子节点开始，自底向上下沉。O(n)
    for i in stride(from: n / 2 - 1, through: 0, by: -1) {
        siftDown(&nums, i, n)
    }
    // 2) 反复取堆顶（最大）放到末尾，缩小堆并恢复堆序。O(n log n)
    for end in stride(from: n - 1, through: 1, by: -1) {
        nums.swapAt(0, end)        // 最大值放到 end
        siftDown(&nums, 0, end)    // 对 [0, end) 恢复堆
    }
}

/// 下沉：把 i 处元素下沉到正确位置，堆有效范围 [0, size)
private func siftDown(_ a: inout [Int], _ start: Int, _ size: Int) {
    var parent = start
    while true {
        let left = 2 * parent + 1
        let right = 2 * parent + 2
        var largest = parent
        if left < size && a[left] > a[largest] { largest = left }
        if right < size && a[right] > a[largest] { largest = right }
        if largest == parent { break }   // 已满足堆序
        a.swapAt(parent, largest)
        parent = largest
    }
}
```

### 9.4 Objective-C 实现

```objc
// 堆排序（升序，不稳定，原地）
- (void)heapSort:(NSMutableArray<NSNumber *> *)nums {
    NSInteger n = (NSInteger)nums.count;
    if (n <= 1) return;
    for (NSInteger i = n / 2 - 1; i >= 0; i--) {
        [self siftDown:nums start:i size:n];
    }
    for (NSInteger end = n - 1; end >= 1; end--) {
        [nums exchangeObjectAtIndex:0 withObjectAtIndex:end];
        [self siftDown:nums start:0 size:end];
    }
}

- (void)siftDown:(NSMutableArray<NSNumber *> *)a start:(NSInteger)start size:(NSInteger)size {
    NSInteger parent = start;
    while (YES) {
        NSInteger left = 2 * parent + 1;
        NSInteger right = 2 * parent + 2;
        NSInteger largest = parent;
        if (left  < size && [a[left]  integerValue] > [a[largest] integerValue]) largest = left;
        if (right < size && [a[right] integerValue] > [a[largest] integerValue]) largest = right;
        if (largest == parent) break;
        [a exchangeObjectAtIndex:parent withObjectAtIndex:largest];
        parent = largest;
    }
}
```

### 9.5 为什么「建堆」是 O(n) 而非 O(n log n)

直觉上建堆要对 n/2 个节点各下沉 O(log n)，似乎是 O(n log n)。但精确分析：高度为 h 的节点最多 `n/2^(h+1)` 个，每个下沉代价 O(h)，求和：

```
Σ (h=0→log n) [ n / 2^(h+1) · h ]  =  (n/2) · Σ h/2^h
Σ h/2^h 收敛于 2  ⟹  建堆总代价 = O(n)
```

这是堆排序一个反直觉但重要的结论。

- **时间**：建堆 O(n) + n 次下沉 O(log n) = O(n log n)，最好=最坏=平均。
- **空间**：O(1)，真正原地（迭代下沉无递归栈）。
- **稳定性**：不稳定（下沉与堆顶交换会跨越相等元素）。
- **优点**：最坏也是 O(n log n)（无快排的 O(n²) 风险）且 O(1) 空间，适合「最坏保证 + 内存紧张」场景；缺点：缓存不友好（跳跃访问），实测常慢于快排。

> 堆结构还用于 **Top-K**（用大小为 K 的堆，O(n log K)）、**优先队列**、**Dijkstra 最短路** 等，详见「10-堆与优先队列」。

## 10. 计数排序（Counting Sort）

### 10.1 是什么

**计数排序**（Counting Sort）：非比较排序。统计每个值出现的次数，再按值域顺序依次输出。适用于**值域 k 不太大的整数**，可达线性 O(n + k)。

### 10.2 原理与稳定版实现

朴素版（统计次数后直接按值输出）简单但**不稳定**、且无法携带卫星数据（排序对象除 key 外的字段）。**稳定版**用「前缀和」确定每个元素的最终位置，从后往前填充：

```swift
/// 计数排序（稳定版，升序）
/// - 复杂度：时间 O(n + k)，空间 O(n + k)，k 为值域大小
/// - 要求：元素为整数且值域有限
func countingSort(_ nums: [Int]) -> [Int] {
    guard let mn = nums.min(), let mx = nums.max() else { return nums }
    let k = mx - mn + 1
    var count = [Int](repeating: 0, count: k)
    // 1) 统计频次
    for v in nums { count[v - mn] += 1 }
    // 2) 前缀和：count[i] 表示 ≤ (mn+i) 的元素个数 → 即最终位置上界
    for i in 1..<k { count[i] += count[i - 1] }
    // 3) 从后往前填充，保证稳定
    var result = [Int](repeating: 0, count: nums.count)
    for v in nums.reversed() {
        count[v - mn] -= 1
        result[count[v - mn]] = v
    }
    return result
}
```

### 10.3 Objective-C 实现

```objc
// 计数排序（稳定版，升序）。仅适用于整数且值域有限。
- (NSArray<NSNumber *> *)countingSort:(NSArray<NSNumber *> *)nums {
    if (nums.count == 0) return nums;
    NSInteger mn = NSIntegerMax, mx = NSIntegerMin;
    for (NSNumber *x in nums) {
        NSInteger v = [x integerValue];
        mn = MIN(mn, v); mx = MAX(mx, v);
    }
    NSInteger k = mx - mn + 1;
    NSInteger *count = calloc(k, sizeof(NSInteger));   // 记得 free
    for (NSNumber *x in nums) count[[x integerValue] - mn]++;
    for (NSInteger i = 1; i < k; i++) count[i] += count[i - 1];
    NSUInteger n = nums.count;
    NSMutableArray *result = [NSMutableArray arrayWithCapacity:n];
    for (NSUInteger i = 0; i < n; i++) result[i] = @0; // 占位
    for (NSInteger i = (NSInteger)n - 1; i >= 0; i--) {
        NSInteger v = [nums[i] integerValue];
        count[v - mn]--;
        result[count[v - mn]] = @(v);
    }
    free(count);
    return result;
}
```

### 10.4 复杂度与特性

- **时间**：O(n + k)，当 k = O(n) 时为线性。
- **空间**：O(n + k)。
- **稳定性**：稳定（从后往前 + 前缀和）。
- **适用**：整数、值域有限（如年龄 0~150、分数 0~100）；值域极大（如 32 位整数全域）则 k 爆炸，不适用。
- **常见坑**：负数要用 `v - mn` 做偏移；浮点数无法直接计数。

## 11. 基数排序（Radix Sort）

### 11.1 是什么

**基数排序**（Radix Sort）：非比较排序。按数字的「位」（个位、十位、百位……）从低位到高位（LSD，Least Significant Digit）依次做**稳定**的计数排序。利用「低位排好后，高位相同的会保持低位顺序」实现整体有序。

### 11.2 原理（LSD，十进制）

```
原始： 329  457  657  839  436  720  355
按个位排：720 355 436 457 657 329 839
按十位排：720 329 436 839 355 457 657
按百位排：329 355 436 457 657 720 839  ← 有序
```

每一轮必须用**稳定**排序（计数排序），否则会破坏前面已排好的低位顺序。

### 11.3 Swift 实现（非负整数 LSD）

```swift
/// 基数排序（LSD，仅非负整数，升序，稳定）
/// - 复杂度：时间 O(d·(n + base))，d 为最大位数，base 进制（这里 10）
func radixSort(_ nums: [Int]) -> [Int] {
    guard let mx = nums.max(), mx > 0 else { return nums }
    var arr = nums
    let base = 10
    var exp = 1                         // 当前处理的位：1, 10, 100, ...
    while mx / exp > 0 {
        arr = countingSortByDigit(arr, exp, base)
        exp *= base
    }
    return arr
}

/// 按 (num / exp) % base 这一位做稳定计数排序
private func countingSortByDigit(_ nums: [Int], _ exp: Int, _ base: Int) -> [Int] {
    var count = [Int](repeating: 0, count: base)
    for v in nums { count[(v / exp) % base] += 1 }
    for i in 1..<base { count[i] += count[i - 1] }
    var result = [Int](repeating: 0, count: nums.count)
    for v in nums.reversed() {          // 从后往前保证稳定
        let d = (v / exp) % base
        count[d] -= 1
        result[count[d]] = v
    }
    return result
}
```

### 11.4 Objective-C 实现

```objc
// 基数排序（LSD，仅非负整数，升序，稳定）
- (NSArray<NSNumber *> *)radixSort:(NSArray<NSNumber *> *)nums {
    if (nums.count == 0) return nums;
    NSInteger mx = 0;
    for (NSNumber *x in nums) mx = MAX(mx, [x integerValue]);
    if (mx == 0) return nums;
    NSMutableArray *arr = [nums mutableCopy];
    NSInteger base = 10;
    for (NSInteger exp = 1; mx / exp > 0; exp *= base) {
        arr = [[self countingSortByDigit:arr exp:exp base:base] mutableCopy];
    }
    return arr;
}

- (NSArray<NSNumber *> *)countingSortByDigit:(NSArray<NSNumber *> *)nums
                                         exp:(NSInteger)exp base:(NSInteger)base {
    NSInteger *count = calloc(base, sizeof(NSInteger));
    for (NSNumber *x in nums) count[([x integerValue] / exp) % base]++;
    for (NSInteger i = 1; i < base; i++) count[i] += count[i - 1];
    NSUInteger n = nums.count;
    NSMutableArray *result = [NSMutableArray arrayWithCapacity:n];
    for (NSUInteger i = 0; i < n; i++) result[i] = @0;
    for (NSInteger i = (NSInteger)n - 1; i >= 0; i--) {
        NSInteger d = ([nums[i] integerValue] / exp) % base;
        count[d]--;
        result[count[d]] = nums[i];
    }
    free(count);
    return result;
}
```

### 11.5 复杂度与特性

- **时间**：O(d·(n + base))，d = 最大数字位数。当 d 为常数（如固定 32 位整数、按字节分 4 轮），近似 O(n)。
- **空间**：O(n + base)。
- **稳定性**：稳定（依赖底层计数排序稳定）。
- **适用**：定长整数、字符串、定长 key；处理负数需偏移或分正负两轮。

## 12. 桶排序（Bucket Sort）

### 12.1 是什么

**桶排序**（Bucket Sort）：把数据按值域均匀分到若干「桶」中，每个桶内部用其他排序（如插入排序）排好，再按桶顺序拼接。当数据**均匀分布**时平均可达 O(n)。

### 12.2 Swift 实现（[0,1) 浮点数为例）

```swift
/// 桶排序（适合均匀分布数据，这里以 [0,1) 浮点为例，升序，稳定取决于桶内排序）
/// - 复杂度：平均 O(n + k)（均匀分布），最坏 O(n²)（全挤一桶）
func bucketSort(_ nums: [Double]) -> [Double] {
    let n = nums.count
    guard n > 1 else { return nums }
    var buckets = Array(repeating: [Double](), count: n)
    // 1) 分桶：值 v ∈ [0,1) 映射到桶 floor(v * n)
    for v in nums {
        let idx = min(Int(v * Double(n)), n - 1)
        buckets[idx].append(v)
    }
    // 2) 桶内排序（小数据用插入排序），再按桶序拼接
    var result = [Double]()
    for i in 0..<n {
        let sorted = buckets[i].sorted()   // 桶内排序
        result.append(contentsOf: sorted)
    }
    return result
}
```

### 12.3 复杂度与特性

- **时间**：平均 O(n + k)（n 个元素 + k 个桶，均匀分布下每桶 O(1) 元素）；最坏 O(n²)（所有元素落入同一桶且桶内用 O(n²) 排序）。
- **空间**：O(n + k)。
- **稳定性**：取决于桶内排序与分桶是否保序。
- **适用**：数据在值域内**均匀分布**；分布倾斜时退化严重。计数排序可视为「桶大小为 1」的特例。

## 13. 比较排序的下界证明

### 13.1 结论

**任何基于比较的排序算法，最坏情况下至少需要 Ω(n log n) 次比较。**

### 13.2 决策树（Decision Tree）证明

把任意比较排序的执行过程建模为**决策树**（comparison decision tree）：每个内部节点是一次比较 `a[i] : a[j]`，两条出边对应 `≤` 和 `>` 两种结果，每个叶子对应一种最终排列。

- n 个不同元素共有 **n!** 种排列，每种都必须能被算法区分（对应不同叶子），故决策树至少有 `n!` 个叶子。
- 一棵高度为 h 的二叉树最多有 `2^h` 个叶子，故 `2^h ≥ n!`，即 `h ≥ log₂(n!)`。
- 由斯特林公式（Stirling's approximation）`log₂(n!) = Θ(n log n)`。
- 树高 h 即最坏比较次数，故任何比较排序最坏 ≥ `Ω(n log n)`。 ∎

### 13.3 推论

归并排序、堆排序达到了这个下界（O(n log n)），是**渐近最优**的比较排序。计数/基数/桶排序之所以能突破 O(n log n)，正是因为它们**不通过比较**获取顺序信息，绕开了决策树模型的限制。

## 15. 工程实践：标准库与 iOS 的排序

### 15.1 主流语言标准库的排序策略

现代标准库几乎都用**混合排序**（hybrid sort），博采众长：

- **C++ `std::sort`**：Introsort（内省排序）= 快排为主 + 递归深度超 `2·log n` 时切堆排序（防 O(n²)）+ 小区间切插入排序。不稳定。`std::stable_sort` 用归并，稳定。
- **Java**：基本类型 `Arrays.sort` 用 Dual-Pivot Quicksort（双轴快排，不稳定）；对象数组用 **TimSort**（稳定）。
- **Python**：`sorted` / `list.sort` 用 **TimSort**（稳定）。
- **Rust**：`sort` 用 TimSort 变体（稳定）；`sort_unstable` 用 pattern-defeating quicksort（pdqsort，不稳定，更快）。

### 15.2 TimSort 简介

**TimSort**（Tim Peters, 2002）是归并 + 插入的混合体，专为「真实世界数据常部分有序」设计：先扫描出已有序的「run」（升/降序段），短 run 用插入排序补齐到最小长度，再用精心设计的「归并策略」（保持栈上 run 长度满足特定不变式）合并。最好 O(n)（已有序），平均/最坏 O(n log n)，稳定。Java/Python/Android 的对象排序都用它。

### 15.3 iOS / Swift 的排序

Swift 标准库 `Array.sort()` / `sorted()`：

```swift
var a = [3, 1, 4, 1, 5, 9, 2, 6]
a.sort()                       // 升序，原地
let b = a.sorted()             // 升序，返回新数组
let c = a.sorted(by: >)        // 降序
// 自定义比较：按字符串长度
let words = ["bb", "a", "ccc"]
let byLen = words.sorted { $0.count < $1.count }
```

> **稳定性说明**：Swift 标准库 `sort` 在历史早期**不保证稳定**。从 **Swift 5（2019）** 起，官方明确 `sort`/`sorted` 提供**稳定排序**语义。但跨版本、跨平台依赖稳定性时，仍建议在比较闭包里加入 tie-breaker（如原始下标）以确保确定性。

底层实现：Swift 标准库 `sort` 采用 **Introsort**（快排 + 堆排兜底 + 小数组插入排序），并通过实现细节保证稳定语义。

### 15.4 Objective-C 的排序 API

```objc
// NSArray：sortedArrayUsingComparator / descriptors
NSArray *a = @[@3, @1, @2];
NSArray *sorted = [a sortedArrayUsingComparator:^NSComparisonResult(NSNumber *x, NSNumber *y) {
    return [x compare:y];                 // 升序
}];
// NSMutableArray 原地排序
[mutableArr sortUsingComparator:^NSComparisonResult(id x, id y) { return [x compare:y]; }];
// 多级排序：NSSortDescriptor（稳定，天然依赖底层稳定性）
NSSortDescriptor *byAge   = [NSSortDescriptor sortDescriptorWithKey:@"age" ascending:YES];
NSSortDescriptor *byName  = [NSSortDescriptor sortDescriptorWithKey:@"name" ascending:YES];
NSArray *result = [people sortedArrayUsingDescriptors:@[byAge, byName]];
```

> Foundation 的 `sortedArrayUsingComparator:` 在不同 OS 版本实现略有差异，多数情况下表现稳定；`NSSortDescriptor` 多级排序依赖稳定语义。

### 15.5 实战选型决策树

```
要稳定吗？
├─ 是 → 数据量小/近有序 → 插入排序
│        数据量大 → 归并 / TimSort（标准库 sorted）
└─ 否 → 内存紧张/要最坏保证 → 堆排序
         追求平均最快 → 快排（三数取中+随机化+小区间插排）
         整数值域小 → 计数排序
         定长整数/字符串 → 基数排序
```

## 16. 经典算法题精讲

### 16.1 LeetCode 912 · 排序数组

> 给定数组，返回升序结果（不能用库 sort）。考察手写 O(n log n) 排序。标准答案：随机化快排 / 堆排 / 归并。注意快排要随机化，否则 LeetCode 的有序+重复用例会卡 TLE。三路快排是最稳妥的 AC 方案。

### 16.2 LeetCode 215 · 数组中第 K 个最大元素

> **快速选择**（Quick Select）：基于快排划分思想，只递归 pivot 所在的那一侧，平均 O(n)。

```swift
func findKthLargest(_ nums: [Int], _ k: Int) -> Int {
    var a = nums
    let target = a.count - k         // 第 k 大 = 升序第 (n-k) 小
    var lo = 0, hi = a.count - 1
    while true {
        let p = partition(&a, lo, hi)   // 复用第 8 章 partition
        if p == target { return a[p] }
        else if p < target { lo = p + 1 }
        else { hi = p - 1 }
    }
}
```

> 平均 O(n)，最坏 O(n²)（随机化可避免）。也可用大小为 k 的最小堆 O(n log k)。

### 16.3 LeetCode 88 · 合并两个有序数组

> 归并的 merge 步骤直接应用。**从后往前**填充避免覆盖：

```swift
func merge(_ nums1: inout [Int], _ m: Int, _ nums2: [Int], _ n: Int) {
    var i = m - 1, j = n - 1, k = m + n - 1
    while j >= 0 {
        if i >= 0 && nums1[i] > nums2[j] {
            nums1[k] = nums1[i]; i -= 1
        } else {
            nums1[k] = nums2[j]; j -= 1
        }
        k -= 1
    }
}
```

> 三路划分的原型：把 0/1/2 三色原地分成三段，一趟 O(n)、O(1) 空间。即第 8.4 节三路快排的 partition。

### 16.5 LeetCode 148 · 排序链表

> 要求 O(n log n) 且 O(1) 空间 → **自底向上归并排序链表**（递归版是 O(log n) 栈空间）。链表归并不需要随机访问，是归并的主场。

### 16.6 LeetCode 56 · 合并区间

> 先按区间左端点**排序**（O(n log n)），再线性扫描合并重叠区间。「排序 + 贪心」的经典套路，体现排序作为预处理手段的威力。

### 16.7 LeetCode 315 · 计算右侧小于当前元素的个数（归并求逆序对）

> **归并排序求逆序对数**：在 merge 时，当右半元素先于左半被取出，说明它与左半剩余所有元素构成逆序对，累加即可。剑指 Offer 51、LC 315 都是此思想，O(n log n)。

完整实现（剑指 Offer 51 · 数组中的逆序对）：

```swift
/// 归并排序统计逆序对总数，O(n log n)
func reversePairs(_ nums: [Int]) -> Int {
    var a = nums
    var tmp = nums
    return mergeCount(&a, &tmp, 0, a.count - 1)
}

private func mergeCount(_ a: inout [Int], _ tmp: inout [Int], _ lo: Int, _ hi: Int) -> Int {
    guard lo < hi else { return 0 }
    let mid = lo + (hi - lo) / 2
    var count = mergeCount(&a, &tmp, lo, mid) + mergeCount(&a, &tmp, mid + 1, hi)
    var i = lo, j = mid + 1, k = lo
    while i <= mid && j <= hi {
        if a[i] <= a[j] {
            tmp[k] = a[i]; i += 1
        } else {
            // a[i] > a[j]：左半 [i, mid] 都比 a[j] 大，构成 (mid - i + 1) 个逆序对
            count += mid - i + 1
            tmp[k] = a[j]; j += 1
        }
        k += 1
    }
    while i <= mid { tmp[k] = a[i]; i += 1; k += 1 }
    while j <= hi  { tmp[k] = a[j]; j += 1; k += 1 }
    for x in lo...hi { a[x] = tmp[x] }
    return count
}
```

> 先用哈希表统计频次，再按频次降序排序字符，最后拼接。「计数 + 排序」组合，也可用桶排序（桶下标=频次）做到 O(n)。

### 16.9 LeetCode 179 · 最大数

> 把数字拼成最大数：自定义比较器，比较 `a+b` 与 `b+a` 的字典序（如 `"3"` 与 `"30"`，比较 `"330"` vs `"303"`）。排序的「比较器设计」典型题。

```swift
func largestNumber(_ nums: [Int]) -> String {
    let strs = nums.map { String($0) }
    let sorted = strs.sorted { $0 + $1 > $1 + $0 }   // 自定义比较器
    if sorted.first == "0" { return "0" }            // 全 0 边界
    return sorted.joined()
}
```

### 16.10 自定义对象多级排序（iOS 工程实战）

> 实战中常需对模型对象按多个字段排序，且要稳定。Swift 用 tie-breaker 链式比较：

```swift
struct Order { let priority: Int; let time: Date; let id: Int }

let sorted = orders.sorted { a, b in
    if a.priority != b.priority { return a.priority > b.priority } // 主：优先级降序
    if a.time != b.time { return a.time < b.time }                 // 次：时间升序
    return a.id < b.id                                             // 末：id 升序（确定性 tie-breaker）
}
```

## 16.11 性能基准：各排序在不同数据形态下的表现

不同算法对「数据形态」高度敏感，下面用一段可运行的基准框架说明如何实测（结论随机器/编译优化而异，但相对趋势稳定）：

```swift
import Foundation

func benchmark(_ name: String, _ data: [Int], _ sort: (inout [Int]) -> Void) {
    var arr = data
    let start = Date()
    sort(&arr)
    let ms = Date().timeIntervalSince(start) * 1000
    assert(arr == data.sorted(), "\(name) 排序结果错误!")   // 正确性校验
    print(String(format: "%-12@ : %.2f ms", name as NSString, ms))
}

let n = 50_000
let random  = (0..<n).map { _ in Int.random(in: 0..<n) }     // 随机
let sortedA = Array(0..<n)                                    // 已升序
let reverse = Array((0..<n).reversed())                       // 完全逆序
let fewKeys = (0..<n).map { _ in Int.random(in: 0..<10) }    // 大量重复

// 典型趋势：
// - 已有序：插入排序 ≈ O(n) 极快；普通快排(固定pivot)退化 O(n²) 极慢
// - 大量重复：三路快排 ≫ 普通快排
// - 随机数据：快排 < 归并 < 堆排（时间），但归并稳定
```

> **基准启示**：① 永远在「随机 + 有序 + 逆序 + 大量重复」四种形态上测；② 加 `assert` 校验正确性，别只看速度；③ Release 模式（`-O`）下的数据才有参考价值，Debug 模式边界检查会严重失真。

### 16.12 外部归并排序示意（超内存数据）

```
阶段一（分块排序）：
  把 100GB 文件按 1GB 切成 100 块 → 每块读入内存排序 → 写回成 100 个有序临时文件
阶段二（多路归并）：
  用一个大小为 100 的最小堆，每次弹出最小元素写入输出，
  从该元素所属的临时文件再读一个补入堆，直到所有文件耗尽
复杂度：I/O 主导；CPU 为 O(N log K)，K = 路数
```

## 17. 面试题

**Q1：手写快速排序，并说明如何避免最坏 O(n²)？**
A：见第 8 章。避免退化的手段：① 随机选 pivot；② 三数取中；③ 三路划分应对大量重复；④ 小区间切插入排序。这些使「构造最坏输入」几乎不可能。

**Q2：归并排序和快速排序的区别？各自适用场景？**
A：归并稳定、最坏 O(n log n)、需 O(n) 空间、适合链表/外部排序；快排不稳定、最坏 O(n²) 但平均最快且原地、缓存友好。要稳定或要最坏保证用归并，追求平均速度用快排。

**Q3：哪些排序是稳定的？哪些不稳定？为什么选择排序不稳定？**
A：稳定——冒泡、插入、归并、计数、基数、桶；不稳定——选择、快速、希尔、堆。选择排序反例 `[5a,5b,2]`：选最小 2 与 5a 交换后 5a 跑到 5b 之后，破坏相对次序。

**Q4：为什么比较排序无法突破 O(n log n)？计数排序为什么能 O(n)？**
A：决策树模型——n! 种排列需 ≥ log₂(n!) = Θ(n log n) 次比较。计数排序不做元素比较，而是利用值域信息直接定位，绕开了决策树下界。

**Q5：堆排序建堆为什么是 O(n)？**
A：高度 h 的节点有 n/2^(h+1) 个，下沉代价 O(h)，求和 Σ h/2^h 收敛于 2，故 O(n)。

**Q6：从 10 亿个数里找最大的 100 个怎么做？**
A：维护一个大小为 100 的**最小堆**，遍历所有数，比堆顶大就替换并下沉，最终堆中即 Top 100，时间 O(n log k)、空间 O(k)，无需把全部数据载入内存。

**Q7：Swift 的 `sort` 稳定吗？**
A：Swift 5（2019）起官方保证稳定语义；更早版本不保证。依赖稳定性时建议加显式 tie-breaker。

**Q8：如何对超过内存的超大文件排序？**
A：**外部归并排序**——分块读入内存、各块内部排序后写回磁盘成有序段，再用多路（k-way）归并（用最小堆选下一个输出）合并所有段。`sort` 命令即如此。

**Q9：插入排序为什么在小数组上比快排快？**
A：插入排序常数因子极小、无递归/划分开销、缓存友好、对近有序自适应；小数组下 O(n²) 的 n² 很小，反而胜过快排的高常数。这是工业排序「小区间切插入」的原因。

**Q10：三路快排相比普通快排的优势？**
A：把数组分成 `<、=、>` 三段，相等区不再递归，对「大量重复元素」从 O(n²) 优化到接近 O(n)，是 LeetCode 912 防 TLE 的关键。

## 18. 易错点与最佳实践

### 18.1 高频易错点

1. **无符号下标下溢**：OC/C 中插入排序、堆排序的 `j -= 1` 若用 `NSUInteger` 会回绕成巨值 → 死循环/越界。**必须用有符号 `NSInteger`**。
2. **取中防溢出**：`mid = (lo + hi) / 2` 在大数组可能溢出，应写 `mid = lo + (hi - lo) / 2`。
3. **稳定性被 `>=` 破坏**：归并/冒泡里相等元素应「保持原序」，比较用 `<=`（取左）或 `>`（不交换），写成 `>=`/`<` 会破坏稳定。
4. **快排不随机化**：固定首/尾 pivot 遇有序输入退化 O(n²)，LeetCode 直接 TLE。
5. **计数排序忘记偏移负数**：用 `v - mn` 而非 `v` 做索引。
6. **基数排序每轮必须用稳定排序**：否则破坏低位顺序，结果错误。
7. **堆排序下沉边界**：`left < size` 的 `size` 是「当前堆有效大小」而非数组总长，排序阶段每次缩小。
8. **桶排序假设不成立**：数据倾斜时退化 O(n²)，别盲目用。

### 18.2 最佳实践

- **优先用标准库**：除非面试或特殊需求，生产代码直接用 `sorted`/`std::sort`，它们是经过千锤百炼的混合排序。
- **需要稳定且多级**：用 `NSSortDescriptor` / 多 key 比较闭包，或归并/TimSort。
- **手写快排黄金模板**：三数取中 + 随机化 + 三路划分 + 小区间（<16）切插入排序。
- **Top-K 用堆**，不要全排序后取前 K。
- **链表排序用归并**，不要转数组再排（除非可接受额外空间）。
- **超大数据用外部归并**。

## 19. 对比辨析

### 19.1 冒泡 vs 选择 vs 插入

三者都是 O(n²)，但：冒泡比较+交换都多、最稳定直观；选择交换最少（O(n)），适合交换代价高的场景；插入对近有序自适应、常数最小，是三者中实战价值最高的（小数组首选）。

### 19.2 快排 vs 归并 vs 堆排

三大 O(n log n) 排序的取舍：快排平均最快、原地、缓存友好但不稳定且最坏 O(n²)；归并稳定、最坏有保证但需 O(n) 空间；堆排原地、最坏有保证但不稳定且缓存不友好。「平均速度选快排，稳定选归并，省内存+最坏保证选堆排」。

### 19.3 比较排序 vs 非比较排序

比较排序通用（任何可比较类型）但下界 O(n log n)；非比较排序（计数/基数/桶）可线性但要求数据满足额外假设（整数、值域有限、均匀分布）。**没有银弹**：选型取决于数据特征。

### 19.4 计数 vs 基数 vs 桶

三者都是非比较线性排序：计数排序适合值域小的整数（桶大小=1）；基数排序适合位数固定的整数/字符串（多轮计数）；桶排序适合均匀分布的实数（每桶内再排序）。三者本质都是「用空间/分布信息换比较时间」。

## 20. 排序算法学习与记忆路线图

### 20.1 按「核心思想」归类记忆

与其孤立背诵十种算法，不如按思想串成网络：

- **逐步推进类**（O(n²)，相邻或局部操作）：冒泡（相邻交换冒最大）、选择（每趟选最值）、插入（有序区逐个插入）。三者是同一思想的三种变体，差别在「比较什么、何时交换」。
- **分治类**（O(n log n)，递归二分）：归并（先分后合，合是重点）、快排（先划分后递归，划分是重点）。一个「难在合」，一个「难在分」，互为镜像。
- **数据结构驱动类**：堆排序（借助二叉堆的堆序性质）、希尔（借助增量分组改良插入）。
- **空间换时间类**（线性，非比较）：计数（值域桶）、基数（按位多轮计数）、桶（区间分桶）。

### 20.2 手写优先级（面试高频度）

按面试出现频率与重要性，建议优先掌握顺序：

1. **快速排序**（必考，要能讲清划分 + 随机化 + 三路 + 快速选择）。
2. **归并排序**（必考，要能讲清 merge + 逆序对 + 链表归并）。
3. **堆排序 / 堆**（Top-K、优先队列高频）。
4. **插入排序**（小数组优化、TimSort 基础）。
5. **计数 / 基数**（特定场景加分项）。
6. 冒泡、选择、希尔、桶（理解即可，实战极少手写）。

## 总结

排序算法是算法学习的「试金石」，它把分治、递归、贪心、数据结构、复杂度分析、稳定性权衡、缓存优化等核心思想浓缩在一处。掌握排序的关键不在于背代码，而在于理解：① 每种算法的**核心思想**（冒泡的相邻交换、归并的分治合并、快排的划分、堆排的堆序、计数的值域映射）；② 复杂度从何而来（递归式、决策树下界、建堆均摊）；③ 稳定/原地/自适应的**取舍**；④ 工程上为何用**混合策略**（Introsort、TimSort）。能把快排手写对（随机化 + 三路 + 小区间插排）、能讲清归并求逆序对、能用堆解 Top-K、能说出 Swift `sort` 的稳定性边界，排序这一关就算真正过了。
