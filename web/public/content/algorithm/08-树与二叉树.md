# 树与二叉树（Tree & Binary Tree）详解

> **树**（Tree）是一种非线性的层次型数据结构，具有"一对多"的逻辑关系，广泛用于表达组织结构、文件系统、语法分析、数据库索引等几乎一切层级化场景。**二叉树**（Binary Tree）是树的最重要特例——每个节点至多两个子节点，其天然的递归定义使得许多算法可以优雅地递归实现，也是面试中出现频率最高的数据结构之一。本文从树的基本定义出发，逐层深入二叉树的存储、遍历（前/中/后/层+Morris）、构建与序列化、经典变体（BST/AVL/红黑树概览）、以及工程应用（iOS UIView 层级、DOM 树），给出 Swift 与 Objective-C 双语实现，并配套复杂度分析、典型题目、面试题与易错点。

## 1. 树的基本概念

### 1.1 定义

**树**（Tree）是 n（n >= 0）个节点的有限集合 T，满足：

- 当 n = 0 时为**空树**。
- 当 n > 0 时，有且仅有一个称为**根**（root）的特殊节点；其余节点可分为 m（m >= 0）个互不相交的有限集合 T1, T2, ..., Tm，每个集合本身又是一棵树，称为根的**子树**（subtree）。

这是一个**递归定义**，也正是树天然适合递归处理的根本原因。

### 1.2 关键术语

| 术语 | 英文 | 含义 |
|------|------|------|
| 节点的度 | degree | 该节点拥有的子树数目 |
| 树的度 | degree of tree | 树中节点度的最大值 |
| 叶节点/终端节点 | leaf / terminal node | 度为 0 的节点 |
| 内部节点 | internal node | 度 > 0 的非叶节点 |
| 父节点 | parent | 有子节点的节点 |
| 子节点 | child | 父节点的直接下属 |
| 兄弟节点 | sibling | 同一父节点的节点 |
| 祖先 | ancestor | 从根到该节点路径上的所有节点 |
| 后代 | descendant | 以该节点为根的子树中所有节点 |
| 层次/层级 | level | 根为第 1 层（或第 0 层），依次往下 +1 |
| 深度 | depth | 从根到该节点的路径长度（边数） |
| 高度 | height | 从该节点到最远叶节点的路径长度 |
| 树的高度 | height of tree | 根节点的高度 |
| 路径长度 | path length | 路径上经过的边数 |
| 森林 | forest | m（m >= 0）棵互不相交的树的集合 |

> **注意**：关于"深度从 0 还是 1 开始"，不同教材有不同约定。LeetCode 的"最大深度"一般指节点数。本文采用**深度 = 边数**的定义（根深度 = 0），高度同理，需要时会特别说明。

### 1.3 树的表示方法

- **双亲表示法**：每个节点存储指向父节点的指针/下标，适合快速查祖先，不利于查子节点。
- **孩子表示法**：每个节点存储子节点链表，适合遍历子节点。
- **孩子兄弟表示法**（左孩子右兄弟）：用二叉链表表示一般树——左指针指向第一个孩子，右指针指向下一个兄弟。这是**一般树与二叉树互转**的基础。

### 1.4 树与二叉树的互转

**任何一棵树都可以唯一对应一棵二叉树**（通过"左孩子右兄弟"表示法）。转换规则：

1. 树的根 → 二叉树的根（无右子树，因为根没有兄弟）。
2. 每个节点的第一个孩子 → 二叉树的左子节点。
3. 该孩子的下一个兄弟 → 二叉树的右子节点。

这一变换保证了：对一般树的前序遍历等价于对对应二叉树的前序遍历；对一般树的后序遍历等价于对对应二叉树的中序遍历。

## 2. 二叉树的定义与分类

### 2.1 定义

**二叉树**（Binary Tree）是 n（n >= 0）个节点的有限集合，满足：

- n = 0 时为**空二叉树**。
- n > 0 时由一个根节点和两棵互不相交的子二叉树（称为左子树、右子树）组成。

关键区别于度 <= 2 的树：**二叉树严格区分左右子树**，即使某节点只有一个子节点，也必须区分是左还是右。

### 2.2 五种基本形态

1. 空二叉树
2. 只有根节点
3. 根 + 只有左子树
4. 根 + 只有右子树
5. 根 + 左右子树都有

### 2.3 特殊二叉树

| 类型 | 定义 | 关键特性 |
|------|------|----------|
| **满二叉树**（Perfect Binary Tree）| 每层节点数都达到最大值 | n = 2^h - 1，h 为层数 |
| **完全二叉树**（Complete Binary Tree）| 除最后一层外每层都满，最后一层靠左紧排 | 适合数组存储，堆的底层结构 |
| **二叉搜索树**（BST）| 左 < 根 < 右，递归成立 | 中序遍历有序 |
| **平衡二叉树**（AVL）| BST + 左右高度差 <= 1 | h = O(log n) |
| **退化树/斜树**（Skewed）| 每个节点只有一个子节点 | 退化为链表，h = n-1 |
| **线索二叉树**（Threaded）| 空指针域存前驱/后继 | n 个节点有 n+1 个空指针 |

> **术语辨析**：国际上 "Full Binary Tree" 有两种含义——(1) 每个节点度为 0 或 2（无度为 1 的节点）；(2) 所有层都满（即 Perfect）。中文"满二叉树"通常指 (2)。面试时建议先确认定义。

### 2.4 二叉树的重要性质

**性质 1**：第 i 层最多有 2^(i-1) 个节点（i >= 1）。

**性质 2**：高度为 h 的二叉树最多有 2^(h+1) - 1 个节点（高度从 0 计），或 2^h - 1（层数从 1 计）。

**性质 3**：对任何非空二叉树，叶节点数 n0 与度为 2 的节点数 n2 满足：**n0 = n2 + 1**。

**证明**：设度为 0/1/2 的节点数分别为 n0, n1, n2。总节点数 n = n0 + n1 + n2。边数 = n - 1（树的基本性质）。又边数 = 各节点的出度之和 = 0*n0 + 1*n1 + 2*n2 = n1 + 2*n2。于是 n0 + n1 + n2 - 1 = n1 + 2*n2，化简得 **n0 = n2 + 1**。

**性质 4**：n 个节点的完全二叉树高度为 floor(log2(n))（高度从 0 计）。

**性质 5**：完全二叉树按层编号（从 1 开始）的下标关系：
- 父节点：floor(i/2)
- 左子：2*i
- 右子：2*i + 1

从 0 开始编号时：
- 父节点：floor((i-1)/2)
- 左子：2*i + 1
- 右子：2*i + 2

**性质 6**：n 个节点可以构成的不同二叉树的数量为 Catalan 数：C(n) = C(2n, n) / (n+1)。前几项：1, 1, 2, 5, 14, 42, 132...

## 3. 二叉树的存储结构

### 3.1 顺序存储（数组）

完全二叉树非常适合用数组存储：按层次遍历顺序将节点存入数组，利用性质 5 的下标关系 O(1) 找到父/子。

**优点**：cache-friendly、无指针开销、随机访问 O(1)。

**缺点**：非完全二叉树需要用特殊值填充空位，可能浪费大量空间。最坏情况（右斜树，高度 n-1）需要 2^n - 1 的数组空间。

**典型应用**：堆（Heap）——二叉堆本质上就是用数组存储的完全二叉树。

### 3.2 链式存储（二叉链表）

**Swift**：

```swift
/// 二叉树节点（泛型版）
class TreeNode<T> {
    var val: T
    var left: TreeNode?
    var right: TreeNode?
    
    init(_ val: T, left: TreeNode? = nil, right: TreeNode? = nil) {
        self.val = val
        self.left = left
        self.right = right
    }
}

/// LeetCode 常用整数版本
class TreeNode {
    var val: Int
    var left: TreeNode?
    var right: TreeNode?
    
    init(_ val: Int) {
        self.val = val
    }
}
```

**Objective-C**：

```objectivec
@interface SAKTreeNode : NSObject

@property (nonatomic, assign) NSInteger val;
@property (nonatomic, strong, nullable) SAKTreeNode *left;

```

### 3.3 三叉链表（含父指针）

在二叉链表基础上增加 parent 指针：


应用场景：LCA 的 naive 解法（向上追溯）、平衡树旋转、中序后继迭代器。代价是多一个指针的空间开销及维护成本。

## 4. 二叉树的遍历——递归篇

遍历（traversal）是树操作的基石——"遍历"指按某种规则不重复地访问树中所有节点恰好一次。

### 4.1 前序遍历（Preorder：根-左-右）

**Swift**：

```swift
func preorderTraversal(_ root: TreeNode?) -> [Int] {
    var result = [Int]()
    
    func dfs(_ node: TreeNode?) {
        guard let node = node else { return }
        result.append(node.val)   // 根
        dfs(node.left)            // 左
        dfs(node.right)           // 右
    }
    
    dfs(root)
    return result
}
```

**Objective-C**：

```objectivec
- (NSArray<NSNumber *> *)preorderTraversal:(SAKTreeNode *)root {
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    [self preorderHelper:root result:result];
    return [result copy];
}

- (void)preorderHelper:(SAKTreeNode *)node
                result:(NSMutableArray<NSNumber *> *)result {
    if (!node) return;
    [result addObject:@(node.val)];
    [self preorderHelper:node.left result:result];
    [self preorderHelper:node.right result:result];
}
```

**应用场景**：复制二叉树、序列化（前序+标记空节点可唯一确定一棵树）、目录结构打印。

### 4.2 中序遍历（Inorder：左-根-右）

**对 BST 而言，中序遍历输出恰好是有序序列**——这是 BST 最重要的性质。

**Swift**：

```swift
func inorderTraversal(_ root: TreeNode?) -> [Int] {
    var result = [Int]()
    
    func dfs(_ node: TreeNode?) {
        guard let node = node else { return }
        dfs(node.left)
        result.append(node.val)
        dfs(node.right)
    }
    
    dfs(root)
    return result
}
```

**Objective-C**：

```objectivec
- (NSArray<NSNumber *> *)inorderTraversal:(SAKTreeNode *)root {
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    [self inorderHelper:root result:result];
    return [result copy];
}

- (void)inorderHelper:(SAKTreeNode *)node
               result:(NSMutableArray<NSNumber *> *)result {
    if (!node) return;
    [self inorderHelper:node.left result:result];
    [result addObject:@(node.val)];
    [self inorderHelper:node.right result:result];
}
```

**应用场景**：BST 有序输出、验证 BST 合法性、BST 第 k 小元素。

### 4.3 后序遍历（Postorder：左-右-根）

**Swift**：

```swift
func postorderTraversal(_ root: TreeNode?) -> [Int] {
    var result = [Int]()
    
    func dfs(_ node: TreeNode?) {
        guard let node = node else { return }
        dfs(node.left)
        dfs(node.right)
        result.append(node.val)
    }
    
    dfs(root)
    return result
}
```

**Objective-C**：

```objectivec
- (NSArray<NSNumber *> *)postorderTraversal:(SAKTreeNode *)root {
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    [self postorderHelper:root result:result];
    return [result copy];
}

- (void)postorderHelper:(SAKTreeNode *)node
                 result:(NSMutableArray<NSNumber *> *)result {
    if (!node) return;
    [self postorderHelper:node.left result:result];
    [self postorderHelper:node.right result:result];
    [result addObject:@(node.val)];
}
```

**应用场景**：释放/销毁树（先释放子节点再释放父节点）、计算子树属性（高度、节点数、路径和）、表达式树求值。

### 4.4 递归遍历的复杂度

| 指标 | 值 | 说明 |
|------|-----|------|
| 时间 | O(n) | 每个节点访问恰好一次 |
| 空间（平衡） | O(log n) | 递归栈深度 = 树高 |
| 空间（最坏/斜树） | O(n) | 退化为链表 |

### 4.5 递归的本质与局限

递归遍历利用**函数调用栈**保存"待回溯"信息。优点是代码简洁，缺点是：

1. **栈溢出风险**：极深的树（如百万级链状结构）可能导致 Stack Overflow。
2. **性能开销**：函数调用有压栈/弹栈、参数传递等额外开销。
3. **不易暂停/恢复**：递归天然是"一口气跑完"，难以实现分步遍历（如迭代器模式）。

这些局限促使我们学习迭代遍历和 Morris 遍历。

## 5. 二叉树的遍历——迭代篇

### 5.1 迭代前序遍历

**思路**：用显式栈模拟。压根入栈；每次弹出栈顶并访问，然后先压右子再压左子（保证左先出栈）。

**Swift**：

```swift
func preorderIterative(_ root: TreeNode?) -> [Int] {
    guard let root = root else { return [] }
    var result = [Int]()
    var stack: [TreeNode] = [root]
    
    while !stack.isEmpty {
        let node = stack.removeLast()
        result.append(node.val)
        if let right = node.right { stack.append(right) }
        if let left = node.left { stack.append(left) }
    }
    
    return result
}
```

**Objective-C**：

```objectivec
- (NSArray<NSNumber *> *)preorderIterative:(SAKTreeNode *)root {
    if (!root) return @[];
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    NSMutableArray<SAKTreeNode *> *stack = [NSMutableArray arrayWithObject:root];
    
    while (stack.count > 0) {
        SAKTreeNode *node = stack.lastObject;
        [stack removeLastObject];
        [result addObject:@(node.val)];
        if (node.right) [stack addObject:node.right];
        if (node.left)  [stack addObject:node.left];
    }
    
    return [result copy];
}
```

### 5.2 迭代中序遍历

**思路**：利用栈 + 指针。当前指针不断左移入栈直到 nil；弹出栈顶访问；转向右子树。

**Swift**：

```swift
func inorderIterative(_ root: TreeNode?) -> [Int] {
    var result = [Int]()
    var stack = [TreeNode]()
    var current = root
    
    while current != nil || !stack.isEmpty {
        while let node = current {
            stack.append(node)
            current = node.left
        }
        let node = stack.removeLast()
        result.append(node.val)
        current = node.right
    }
    
    return result
}
```

**Objective-C**：

```objectivec
- (NSArray<NSNumber *> *)inorderIterative:(SAKTreeNode *)root {
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    NSMutableArray<SAKTreeNode *> *stack = [NSMutableArray array];
    SAKTreeNode *current = root;
    
    while (current || stack.count > 0) {
        while (current) {
            [stack addObject:current];
            current = current.left;
        }
        current = stack.lastObject;
        [stack removeLastObject];
        [result addObject:@(current.val)];
        current = current.right;
    }
    
    return [result copy];
}
```

### 5.3 迭代后序遍历

后序遍历的迭代实现最复杂，有多种经典方法：

**方法一：逆前序法**

后序 = 左-右-根。做一个"根-右-左"遍历（前序变体，先压左再压右），再逆序结果。

**Swift**：

```swift
func postorderIterativeReverse(_ root: TreeNode?) -> [Int] {
    guard let root = root else { return [] }
    var result = [Int]()
    var stack: [TreeNode] = [root]
    
    while !stack.isEmpty {
        let node = stack.removeLast()
        result.append(node.val)          // 根-右-左顺序收集
        if let left = node.left { stack.append(left) }
        if let right = node.right { stack.append(right) }
    }
    
    return result.reversed()  // 逆序得到 左-右-根
}
```

**方法二：单栈 + 标记法**（真正的后序模拟）

**Swift**：

```swift
func postorderIterativeMark(_ root: TreeNode?) -> [Int] {
    var result = [Int]()
    var stack = [TreeNode]()
    var current = root
    var lastVisited: TreeNode? = nil
    
    while current != nil || !stack.isEmpty {
        // 一路向左
        while let node = current {
            stack.append(node)
            current = node.left
        }
        // 查看栈顶（不弹出）
        let peek = stack.last!
        // 右子树存在且未访问过 -> 转向右子树
        if let right = peek.right, right !== lastVisited {
            current = right
        } else {
            // 弹出并访问
            stack.removeLast()
            result.append(peek.val)
            lastVisited = peek
        }
    }
    
    return result
}
```

**Objective-C（单栈+标记法）**：

```objectivec
- (NSArray<NSNumber *> *)postorderIterativeMark:(SAKTreeNode *)root {
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    NSMutableArray<SAKTreeNode *> *stack = [NSMutableArray array];
    SAKTreeNode *current = root;
    SAKTreeNode *lastVisited = nil;
    
    while (current || stack.count > 0) {
        while (current) {
            [stack addObject:current];
            current = current.left;
        }
        SAKTreeNode *peek = stack.lastObject;
        if (peek.right && peek.right != lastVisited) {
            current = peek.right;
        } else {
            [stack removeLastObject];
            [result addObject:@(peek.val)];
            lastVisited = peek;
        }
    }
    
    return [result copy];
}
```

### 5.4 统一迭代模板（标记法/颜色法）

为让前/中/后序迭代代码结构统一，引入"标记法"：每个入栈元素带标记（首次入栈/待输出）。未标记的按遍历逆序展开，已标记的直接输出。

**Swift**：

```swift
/// 统一迭代遍历模板
/// 修改 pushOrder 闭包即可切换前/中/后序
func unifiedTraversal(_ root: TreeNode?, order: String = "inorder") -> [Int] {
    guard let root = root else { return [] }
    var result = [Int]()
    // (node, shouldOutput)
    var stack: [(TreeNode, Bool)] = [(root, false)]
    
    while !stack.isEmpty {
        let (node, shouldOutput) = stack.removeLast()
        if shouldOutput {
            result.append(node.val)
        } else {
            switch order {
            case "preorder":
                // 逆序入栈：右-左-根(标记)
                if let r = node.right { stack.append((r, false)) }
                if let l = node.left  { stack.append((l, false)) }
                stack.append((node, true))
            case "inorder":
                // 逆序入栈：右-根(标记)-左
                if let r = node.right { stack.append((r, false)) }
                stack.append((node, true))
                if let l = node.left  { stack.append((l, false)) }
            case "postorder":
                // 逆序入栈：根(标记)-右-左
                stack.append((node, true))
                if let r = node.right { stack.append((r, false)) }
                if let l = node.left  { stack.append((l, false)) }
            default:
                break
            }
        }
    }
    
    return result
}
```

> **注意**：前序的逆序入栈应该是先入右、再入左、最后入根(标记)。这样出栈时先输出根，再展开左，再展开右。仔细推导即可验证正确性。

### 5.5 三种遍历复杂度对比

| 方式 | 时间 | 空间 | 特点 |
|------|------|------|------|
| 递归 | O(n) | O(h) | 简洁，有栈溢出风险 |
| 迭代（显式栈）| O(n) | O(h) | 无栈溢出风险，可控 |
| Morris | O(n) | O(1) | 无额外空间，会临时修改树 |

## 6. Morris 遍历（O(1) 空间）

### 6.1 核心思想

Morris 遍历的精髓：**利用叶节点大量空闲的 nil 指针（线索化）来保存回溯信息**，避免栈或递归，实现 O(1) 额外空间。

原理：在遍历过程中临时修改树的结构（建立"线索"指回父节点），遍历完再恢复。这种方法由 J.H. Morris 于 1979 年提出。

### 6.2 Morris 中序遍历

**算法**：

1. 初始化 current = root。
2. while current != nil：
   - 若 current.left == nil：访问 current，current = current.right。
   - 若 current.left != nil：找 current 左子树的**最右节点**（中序前驱）predecessor：
     - 若 predecessor.right == nil：建立线索 predecessor.right = current，current = current.left。
     - 若 predecessor.right == current：断开线索，访问 current，current = current.right。

**Swift**：

```swift
func morrisInorder(_ root: TreeNode?) -> [Int] {
    var result = [Int]()
    var current = root
    
    while current != nil {
        if current!.left == nil {
            result.append(current!.val)
            current = current!.right
        } else {
            // 找左子树的最右节点（前驱）
            var predecessor = current!.left!
            while predecessor.right != nil && predecessor.right !== current {
                predecessor = predecessor.right!
            }
            
            if predecessor.right == nil {
                // 第一次到达：建立线索
                predecessor.right = current
                current = current!.left
            } else {
                // 第二次到达：断开线索，访问
                predecessor.right = nil
                result.append(current!.val)
                current = current!.right
            }
        }
    }
    
    return result
}
```

**Objective-C**：

```objectivec
- (NSArray<NSNumber *> *)morrisInorder:(SAKTreeNode *)root {
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    SAKTreeNode *current = root;
    
    while (current) {
        if (!current.left) {
            [result addObject:@(current.val)];
            current = current.right;
        } else {
            SAKTreeNode *predecessor = current.left;
            while (predecessor.right && predecessor.right != current) {
                predecessor = predecessor.right;
            }
            
            if (!predecessor.right) {
                predecessor.right = current;
                current = current.left;
            } else {
                predecessor.right = nil;
                [result addObject:@(current.val)];
                current = current.right;
            }
        }
    }
    
    return [result copy];
}
```

### 6.3 Morris 前序遍历

与中序类似，唯一区别是**在第一次到达时就访问当前节点**（而非第二次）。

**Swift**：

```swift
func morrisPreorder(_ root: TreeNode?) -> [Int] {
    var result = [Int]()
    var current = root
    
    while current != nil {
        if current!.left == nil {
            result.append(current!.val)
            current = current!.right
        } else {
            var predecessor = current!.left!
            while predecessor.right != nil && predecessor.right !== current {
                predecessor = predecessor.right!
            }
            
            if predecessor.right == nil {
                result.append(current!.val)  // 前序：第一次到达时访问
                predecessor.right = current
                current = current!.left
            } else {
                predecessor.right = nil
                current = current!.right
            }
        }
    }
    
    return result
}
```

### 6.4 Morris 后序遍历

Morris 后序较复杂，核心思想：第二次到达某节点时，将其左子树的右边界（从左子到最右叶的路径）逆序输出。最后再逆序输出从根到最右叶的路径。实现复杂度较高，面试中一般不要求手写，知道思路即可。

### 6.5 Morris 遍历的复杂度

- **时间复杂度**：O(n)。虽然找前驱看似每次需要 O(h)，但整体而言每条边最多被遍历两次（建立线索一次、断开一次），总操作量 <= 2n = O(n)。
- **空间复杂度**：O(1)。不使用栈或递归，仅用几个指针变量。

### 6.6 Morris 遍历的注意事项

1. **会临时修改树结构**：如果遍历过程中有并发读取，可能看到不一致的状态。
2. **不适用于只读场景**：如 const 树或不允许修改的环境。
3. **遍历完后树结构完全恢复**：所有线索都会被断开。

## 7. 层次遍历（BFS）

### 7.1 基本层次遍历

利用队列（FIFO）实现逐层访问。

**Swift**：

```swift
func levelOrder(_ root: TreeNode?) -> [[Int]] {
    guard let root = root else { return [] }
    var result = [[Int]]()
    var queue: [TreeNode] = [root]
    
    while !queue.isEmpty {
        let levelSize = queue.count
        var currentLevel = [Int]()
        
        for _ in 0..<levelSize {
            let node = queue.removeFirst()
            currentLevel.append(node.val)
            if let left = node.left { queue.append(left) }
            if let right = node.right { queue.append(right) }
        }
        
        result.append(currentLevel)
    }
    
    return result
}
```

**Objective-C**：

```objectivec
- (NSArray<NSArray<NSNumber *> *> *)levelOrder:(SAKTreeNode *)root {
    if (!root) return @[];
    NSMutableArray<NSArray<NSNumber *> *> *result = [NSMutableArray array];
    NSMutableArray<SAKTreeNode *> *queue = [NSMutableArray arrayWithObject:root];
    
    while (queue.count > 0) {
        NSInteger levelSize = queue.count;
        NSMutableArray<NSNumber *> *currentLevel = [NSMutableArray array];
        
        for (NSInteger i = 0; i < levelSize; i++) {
            SAKTreeNode *node = queue.firstObject;
            [queue removeObjectAtIndex:0];
            [currentLevel addObject:@(node.val)];
            if (node.left)  [queue addObject:node.left];
            if (node.right) [queue addObject:node.right];
        }
        
        [result addObject:[currentLevel copy]];
    }
    
    return [result copy];
}
```

> **性能提示**：OC 中 `[queue removeObjectAtIndex:0]` 是 O(n) 操作。生产代码中应使用真正的队列实现（如环形缓冲区）或双端队列。Swift 的 `removeFirst()` 同理。面试中一般不考虑这一点，但工程中需注意。

### 7.2 层次遍历的变体

| 变体 | 做法 |
|------|------|
| 自底向上层次遍历（LC 107）| 将结果 reverse |
| 锯齿形层次遍历（LC 103）| 奇数层正序、偶数层逆序（或翻转该层结果）|
| 右视图（LC 199）| 每层最后一个节点 |
| 左视图 | 每层第一个节点 |
| 最大层宽（LC 662）| 用编号计算宽度 |
| 每层平均值（LC 637）| 累加后除以层大小 |

### 7.3 层次遍历复杂度

- **时间**：O(n)
- **空间**：O(w)，w 为最宽层的节点数。完全二叉树最宽层约 n/2 个节点，所以最坏 O(n)。

## 8. 二叉树的构建与序列化

### 8.1 由遍历序列重建二叉树

**定理**：以下组合可以唯一确定一棵二叉树：
- 前序 + 中序
- 后序 + 中序
- 层序 + 中序

**前序 + 中序不能唯一确定**需要中序的原因：前序确定根，中序区分左右子树。

**仅前序 + 后序不能唯一确定**（除非树中每个节点度为 0 或 2）。

### 8.2 从前序与中序构建（LC 105）

**思路**：前序第一个元素是根；在中序中找到根的位置，左边是左子树的中序，右边是右子树的中序；递归构建。

**Swift**：

```swift
func buildTree(_ preorder: [Int], _ inorder: [Int]) -> TreeNode? {
    // 用哈希表加速中序中查找根的位置
    var inorderMap = [Int: Int]()
    for (i, val) in inorder.enumerated() {
        inorderMap[val] = i
    }
    
    var preIdx = 0
    
    func build(_ inLeft: Int, _ inRight: Int) -> TreeNode? {
        if inLeft > inRight { return nil }
        
        let rootVal = preorder[preIdx]
        preIdx += 1
        let root = TreeNode(rootVal)
        
        let inRootIdx = inorderMap[rootVal]!
        root.left = build(inLeft, inRootIdx - 1)
        root.right = build(inRootIdx + 1, inRight)
        
        return root
    }
    
    return build(0, inorder.count - 1)
}
```

**Objective-C**：

```objectivec
- (SAKTreeNode *)buildTreeWithPreorder:(NSArray<NSNumber *> *)preorder
                               inorder:(NSArray<NSNumber *> *)inorder {
    NSMutableDictionary<NSNumber *, NSNumber *> *inorderMap = [NSMutableDictionary dictionary];
    for (NSInteger i = 0; i < inorder.count; i++) {
        inorderMap[inorder[i]] = @(i);
    }
    
    __block NSInteger preIdx = 0;
    
    SAKTreeNode *(^build)(NSInteger, NSInteger) = ^SAKTreeNode *(NSInteger inLeft, NSInteger inRight) {
        if (inLeft > inRight) return nil;
        
        NSInteger rootVal = preorder[preIdx].integerValue;
        preIdx++;
        SAKTreeNode *root = [[SAKTreeNode alloc] initWithVal:rootVal];
        
        NSInteger inRootIdx = inorderMap[@(rootVal)].integerValue;
        root.left = build(inLeft, inRootIdx - 1);
        root.right = build(inRootIdx + 1, inRight);
        
        return root;
    };
    
    // Block 递归需要 __block 修饰
    __block SAKTreeNode *(^buildBlock)(NSInteger, NSInteger);
    buildBlock = build;
    
    return buildBlock(0, (NSInteger)inorder.count - 1);
}
```

> **注意**：OC 中 Block 递归调用自身需要先声明 __block 变量再赋值，否则捕获的是 nil。上面的简化写法在实际中需要调整为：

```objectivec
- (SAKTreeNode *)buildTreeWithPreorder:(NSArray<NSNumber *> *)preorder
                               inorder:(NSArray<NSNumber *> *)inorder {
    NSMutableDictionary<NSNumber *, NSNumber *> *inorderMap = [NSMutableDictionary dictionary];
    for (NSInteger i = 0; i < inorder.count; i++) {
        inorderMap[inorder[i]] = @(i);
    }
    
    self.preIdx = 0;
    self.preorderArray = preorder;
    self.inorderMap = inorderMap;
    
    return [self buildHelper:0 inRight:(NSInteger)inorder.count - 1];
}

- (SAKTreeNode *)buildHelper:(NSInteger)inLeft inRight:(NSInteger)inRight {
    if (inLeft > inRight) return nil;
    
    NSInteger rootVal = self.preorderArray[self.preIdx].integerValue;
    self.preIdx++;
    SAKTreeNode *root = [[SAKTreeNode alloc] initWithVal:rootVal];
    
    NSInteger inRootIdx = self.inorderMap[@(rootVal)].integerValue;
    root.left = [self buildHelper:inLeft inRight:inRootIdx - 1];
    root.right = [self buildHelper:inRootIdx + 1 inRight:inRight];
    
    return root;
}
```

**复杂度**：时间 O(n)，空间 O(n)（哈希表 + 递归栈）。

### 8.3 从后序与中序构建（LC 106）

与前序类似，但后序的最后一个元素是根，且构建顺序应**先右子树再左子树**（因为后序逆序 = 根-右-左）。

**Swift**：

```swift
func buildTreeFromPostAndIn(_ inorder: [Int], _ postorder: [Int]) -> TreeNode? {
    var inorderMap = [Int: Int]()
    for (i, val) in inorder.enumerated() {
        inorderMap[val] = i
    }
    
    var postIdx = postorder.count - 1
    
    func build(_ inLeft: Int, _ inRight: Int) -> TreeNode? {
        if inLeft > inRight { return nil }
        
        let rootVal = postorder[postIdx]
        postIdx -= 1
        let root = TreeNode(rootVal)
        
        let inRootIdx = inorderMap[rootVal]!
        // 注意：先构建右子树，再构建左子树
        root.right = build(inRootIdx + 1, inRight)
        root.left = build(inLeft, inRootIdx - 1)
        
        return root
    }
    
    return build(0, inorder.count - 1)
}
```

### 8.4 序列化与反序列化（LC 297）

将二叉树编码为字符串，并能从字符串解码回原树。

**方法：前序遍历 + null 标记**

**Swift**：

```swift
class Codec {
    // 序列化
    func serialize(_ root: TreeNode?) -> String {
        var parts = [String]()
        
        func dfs(_ node: TreeNode?) {
            guard let node = node else {
                parts.append("null")
                return
            }
            parts.append(String(node.val))
            dfs(node.left)
            dfs(node.right)
        }
        
        dfs(root)
        return parts.joined(separator: ",")
    }
    
    // 反序列化
    func deserialize(_ data: String) -> TreeNode? {
        var tokens = data.split(separator: ",").map(String.init)
        var index = 0
        
        func dfs() -> TreeNode? {
            if index >= tokens.count { return nil }
            let token = tokens[index]
            index += 1
            
            if token == "null" { return nil }
            
            let node = TreeNode(Int(token)!)
            node.left = dfs()
            node.right = dfs()
            return node
        }
        
        return dfs()
    }
}
```

```objectivec
    node.left = [self deserializeHelper];
```

### 8.5 由层次遍历序列构建（LeetCode 格式）

LeetCode 的树表示格式是层次遍历（null 表示空）。构建方法：BFS + 队列。

**Swift**：

```swift
func buildFromLevelOrder(_ values: [Int?]) -> TreeNode? {
    guard !values.isEmpty, let firstVal = values[0] else { return nil }
    
    let root = TreeNode(firstVal)
    var queue: [TreeNode] = [root]
    var i = 1
    
    while i < values.count && !queue.isEmpty {
        let node = queue.removeFirst()
        
        // 左子
        if i < values.count {
            if let val = values[i] {
                let left = TreeNode(val)
                node.left = left
                queue.append(left)
            }
            i += 1
        }
        
        // 右子
        if i < values.count {
            if let val = values[i] {
                let right = TreeNode(val)
                node.right = right
                queue.append(right)
            }
            i += 1
        }
    }
    
    return root
}
```

## 9. 二叉树的经典性质与定理

### 9.1 节点数与高度的关系

| 树类型 | 节点数 n 与高度 h 的关系 |
|--------|--------------------------|
| 任意二叉树 | h+1 <= n <= 2^(h+1) - 1 |
| 完全二叉树 | 2^h <= n <= 2^(h+1) - 1，h = floor(log2(n)) |
| 满/完美二叉树 | n = 2^(h+1) - 1 |
| AVL 树 | n >= F(h+3) - 1（F 为 Fibonacci），h < 1.44*log2(n) |
| 红黑树 | h <= 2*log2(n+1) |

### 9.2 空指针域定理

n 个节点的二叉链表中，共有 2n 个指针域，其中 n-1 个非空（指向子节点的边），**n+1 个为空**（nil）。这些空指针域正是线索二叉树和 Morris 遍历利用的资源。

### 9.3 Catalan 数与二叉树计数

n 个节点可以构成的**结构不同**的二叉树数量为第 n 个 Catalan 数：

```
C(n) = C(2n, n) / (n+1) = (2n)! / ((n+1)! * n!)
```

递推公式：C(n) = sum(C(i) * C(n-1-i), i=0..n-1)

等价问题：n 个节点的 BST 数量（给定 1..n 的键）、n 对合法括号数量、n+1 个叶节点的满二叉树数量。

### 9.4 遍历序列与树的唯一确定

| 给定 | 能否唯一确定 | 原因 |
|------|-------------|------|
| 前序 + 中序 | 能 | 前序定根，中序分左右 |
| 后序 + 中序 | 能 | 后序定根，中序分左右 |
| 前序 + 后序 | 不能（一般情况）| 无法区分单子节点在左还是右 |
| 层序 + 中序 | 能 | 层序最前者为根 |
| 仅前序/中序/后序/层序 | 不能 | 信息不足 |

## 10. 最近公共祖先（LCA）

### 10.1 问题定义

给定二叉树中两个节点 p 和 q，找到它们的**最近公共祖先**（Lowest Common Ancestor）：即既是 p 的祖先也是 q 的祖先的最深节点（节点本身可以是自己的祖先）。

### 10.2 递归解法（LC 236）

**核心思路**：后序遍历（自底向上），对每个节点判断其左右子树是否分别包含 p 和 q。

**Swift**：

```swift
func lowestCommonAncestor(_ root: TreeNode?, _ p: TreeNode?, _ q: TreeNode?) -> TreeNode? {
    // base case
    if root == nil || root === p || root === q {
        return root
    }
    
    let left = lowestCommonAncestor(root?.left, p, q)
    let right = lowestCommonAncestor(root?.right, p, q)
    
    // p 和 q 分别在左右子树中 -> 当前节点就是 LCA
    if left != nil && right != nil {
        return root
    }
    
    // 都在同一侧
    return left ?? right
}
```

**Objective-C**：

```objectivec
- (SAKTreeNode *)lowestCommonAncestor:(SAKTreeNode *)root
                                    p:(SAKTreeNode *)p
                                    q:(SAKTreeNode *)q {
    if (!root || root == p || root == q) {
        return root;
    }
    
    SAKTreeNode *left = [self lowestCommonAncestor:root.left p:p q:q];
    SAKTreeNode *right = [self lowestCommonAncestor:root.right p:p q:q];
    
    if (left && right) return root;
    return left ? left : right;
}
```

**复杂度**：时间 O(n)，空间 O(h)。

### 10.3 BST 的 LCA（LC 235）

BST 中可以利用有序性：若 p、q 都小于 root 则 LCA 在左子树；都大于则在右子树；否则 root 就是 LCA。

**Swift**：

```swift
func lowestCommonAncestorBST(_ root: TreeNode?, _ p: TreeNode?, _ q: TreeNode?) -> TreeNode? {
    var node = root
    while let current = node {
        if p!.val < current.val && q!.val < current.val {
            node = current.left
        } else if p!.val > current.val && q!.val > current.val {
            node = current.right
        } else {
            return current
        }
    }
    return nil
}
```

**复杂度**：时间 O(h)，空间 O(1)（迭代版）。

### 10.4 带父指针的 LCA

若节点有 parent 指针，问题退化为"两个链表找交点"——分别从 p 和 q 向上走到根，计算路径长度差，长的先走差值步，然后同步走直到相遇。

或用集合：p 向上走把所有祖先加入集合，q 向上走遇到的第一个在集合中的就是 LCA。

## 11. 路径问题

### 11.1 根到叶路径（LC 257）

**Swift**：

```swift
func binaryTreePaths(_ root: TreeNode?) -> [String] {
    var result = [String]()
    
    func dfs(_ node: TreeNode?, _ path: String) {
        guard let node = node else { return }
        let currentPath = path.isEmpty ? "\(node.val)" : "\(path)->\(node.val)"
        
        if node.left == nil && node.right == nil {
            result.append(currentPath)
            return
        }
        
        dfs(node.left, currentPath)
        dfs(node.right, currentPath)
    }
    
    dfs(root, "")
    return result
}
```

### 11.2 路径总和（LC 112）

判断是否存在根到叶的路径，其节点值之和等于目标值。

**Swift**：

```swift
func hasPathSum(_ root: TreeNode?, _ targetSum: Int) -> Bool {
    guard let root = root else { return false }
    
    // 到达叶节点，检查剩余值
    if root.left == nil && root.right == nil {
        return root.val == targetSum
    }
    
    let remaining = targetSum - root.val
    return hasPathSum(root.left, remaining) || hasPathSum(root.right, remaining)
}
```

**Objective-C**：

```objectivec
- (BOOL)hasPathSum:(SAKTreeNode *)root targetSum:(NSInteger)targetSum {
    if (!root) return NO;
    
    if (!root.left && !root.right) {
        return root.val == targetSum;
    }
    
    NSInteger remaining = targetSum - root.val;
    return [self hasPathSum:root.left targetSum:remaining] ||
           [self hasPathSum:root.right targetSum:remaining];
}
```

### 11.3 路径总和 II（LC 113）——收集所有满足条件的路径

**Swift**：

```swift
func pathSum(_ root: TreeNode?, _ targetSum: Int) -> [[Int]] {
    var result = [[Int]]()
    var path = [Int]()
    
    func dfs(_ node: TreeNode?, _ remaining: Int) {
        guard let node = node else { return }
        path.append(node.val)
        
        if node.left == nil && node.right == nil && node.val == remaining {
            result.append(path)
        } else {
            dfs(node.left, remaining - node.val)
            dfs(node.right, remaining - node.val)
        }
        
        path.removeLast()  // 回溯
    }
    
    dfs(root, targetSum)
    return result
}
```

### 11.4 二叉树中的最大路径和（LC 124）

路径可以从任意节点到任意节点（不一定经过根）。这是一道 Hard 经典题。

**思路**：后序遍历，每个节点计算"经过该节点的最大路径和"，同时维护全局最大值。对每个节点：经过它的最大路径 = node.val + max(0, 左贡献) + max(0, 右贡献)。向上返回时只能选一条分支：node.val + max(0, max(左贡献, 右贡献))。

**Swift**：

```swift
func maxPathSum(_ root: TreeNode?) -> Int {
    var globalMax = Int.min
    
    @discardableResult
    func maxGain(_ node: TreeNode?) -> Int {
        guard let node = node else { return 0 }
        
        // 负贡献的子树不选（取 max 0）
        let leftGain = max(0, maxGain(node.left))
        let rightGain = max(0, maxGain(node.right))
        
        // 经过当前节点的最大路径和
        let pathSum = node.val + leftGain + rightGain
        globalMax = max(globalMax, pathSum)
        
        // 向上返回：只能选一条分支
        return node.val + max(leftGain, rightGain)
    }
    
    maxGain(root)
    return globalMax
}
```

**Objective-C**：

```objectivec
- (NSInteger)maxPathSum:(SAKTreeNode *)root {
    __block NSInteger globalMax = NSIntegerMin;
    [self maxGain:root globalMax:&globalMax];
    return globalMax;
}

- (NSInteger)maxGain:(SAKTreeNode *)node globalMax:(NSInteger *)globalMax {
    if (!node) return 0;
    
    NSInteger leftGain = MAX(0, [self maxGain:node.left globalMax:globalMax]);
    NSInteger rightGain = MAX(0, [self maxGain:node.right globalMax:globalMax]);
    
    NSInteger pathSum = node.val + leftGain + rightGain;
    *globalMax = MAX(*globalMax, pathSum);
    
    return node.val + MAX(leftGain, rightGain);
}
```

**复杂度**：时间 O(n)，空间 O(h)。

### 11.5 二叉树的直径（LC 543）

直径 = 任意两节点间路径的最大边数。等价于找一个节点使得 leftHeight + rightHeight 最大。

**Swift**：

```swift
func diameterOfBinaryTree(_ root: TreeNode?) -> Int {
    var diameter = 0
    
    @discardableResult
    func height(_ node: TreeNode?) -> Int {
        guard let node = node else { return 0 }
        let left = height(node.left)
        let right = height(node.right)
        diameter = max(diameter, left + right)
        return 1 + max(left, right)
    }
    
    height(root)
    return diameter
}
```

### 12.3 判断平衡二叉树（LC 110）

**自底向上法**（O(n)，比自顶向下的 O(n log n) 更优）：

**Swift**：

```swift
func isBalanced(_ root: TreeNode?) -> Bool {
    return checkHeight(root) != -1
}

func checkHeight(_ node: TreeNode?) -> Int {
    guard let node = node else { return 0 }
    
    let left = checkHeight(node.left)
    if left == -1 { return -1 }  // 左子树不平衡，提前终止
    
    let right = checkHeight(node.right)
    if right == -1 { return -1 }  // 右子树不平衡
    
    if abs(left - right) > 1 { return -1 }  // 当前不平衡
    
    return 1 + max(left, right)
}
```

**Objective-C**：

```objectivec
- (BOOL)isBalanced:(SAKTreeNode *)root {
    return [self checkHeight:root] != -1;
}

- (NSInteger)checkHeight:(SAKTreeNode *)node {
    if (!node) return 0;
    
    NSInteger left = [self checkHeight:node.left];
    if (left == -1) return -1;
    
    NSInteger right = [self checkHeight:node.right];
    if (right == -1) return -1;
    
    if (labs(left - right) > 1) return -1;
    
    return 1 + MAX(left, right);
}
```

## 14. 二叉树的变体概览

### 14.1 二叉搜索树（BST）

- **性质**：左 < 根 < 右（递归成立），中序遍历有序。
- **操作复杂度**：查找/插入/删除均为 O(h)。平衡时 h = O(log n)，最坏退化为 O(n)。
- **详见**：下一篇 09-二叉搜索树与平衡树。

### 14.2 AVL 树

- 最早的自平衡 BST（1962 年 Adelson-Velsky & Landis）。
- **平衡条件**：每个节点的左右子树高度差（平衡因子 Balance Factor）绝对值 <= 1。
- **维护**：插入/删除后通过单旋（LL/RR）或双旋（LR/RL）恢复平衡。
- **高度严格**：h < 1.44 * log2(n + 2) - 0.328，比红黑树更矮，查找更快，但插入/删除旋转更频繁。

### 14.3 红黑树（Red-Black Tree）

- 每个节点带颜色（红/黑），通过五条规则维持近似平衡。
- **高度保证**：h <= 2 * log2(n + 1)。
- **优势**：相比 AVL，插入/删除时旋转次数更少（最多 2-3 次），适合写多读少的场景。
- **应用**：C++ std::map/set、Java TreeMap/TreeSet、Linux 进程调度 CFS。

### 14.4 B 树 / B+ 树

- **多路搜索树**：每个节点可有多个子节点和多个键。
- **应用**：数据库索引（MySQL InnoDB 用 B+ 树）、文件系统。
- **优势**：减少磁盘 I/O（节点大小对齐磁盘页，树高极低）。

### 14.5 Trie（前缀树）

- 用于字符串的快速查找/前缀匹配。
- 每条边代表一个字符，从根到节点的路径表示一个前缀。
- 详见 17-字符串匹配篇。

### 14.6 线段树 / 树状数组

- 用于区间查询/修改问题。
- 线段树：完全二叉树结构，每个节点存储一个区间的聚合信息。
- 树状数组（Binary Indexed Tree, BIT/Fenwick Tree）：更紧凑的实现，适合前缀和与单点修改。

### 15.2 hitTest 的实现原理

`hitTest` 是一个经典的**逆后序 DFS**——从最上层（最后添加的）子视图开始向下递归查找点击目标：

```swift
// UIView hitTest 的简化逻辑
override func hitTest(_ point: CGPoint, with event: UIEvent?) -> UIView? {
    // 不可交互/隐藏/透明度<0.01 的跳过
    guard isUserInteractionEnabled, !isHidden, alpha > 0.01 else { return nil }
    guard self.point(inside: point, with: event) else { return nil }
    
    // 逆序遍历子视图（后添加的在上层，优先响应）
    for subview in subviews.reversed() {
        let convertedPoint = subview.convert(point, from: self)
        if let hitView = subview.hitTest(convertedPoint, with: event) {
            return hitView
        }
    }
    
    return self
}
```

### 15.3 SAKFlexboxLibrary 中的树遍历

在美团 Keeta 的动态布局引擎 SAKFlexboxLibrary 中，XML 模板被解析为 `SAKXMLElement` 树。核心方法 `traverseElementForCreateView:` 使用**迭代栈遍历**（非递归前序）来创建/复用 UIView 树——正是本章迭代遍历知识的工程应用。

```objectivec
// 简化的遍历逻辑（实际代码使用 _SAKDLNodeInfo 对象避免野指针）
- (void)traverseElementForCreateView:(SAKXMLElement *)root {
    NSMutableArray<_SAKDLNodeInfo *> *stack = [NSMutableArray array];
    _SAKDLNodeInfo *rootInfo = [[_SAKDLNodeInfo alloc] init];
    rootInfo.element = root;
    [stack addObject:rootInfo];
    
    while (stack.count > 0) {
        _SAKDLNodeInfo *info = stack.lastObject;
        [stack removeLastObject];
        
        // 处理当前节点：applyToView
        [info.element.renderObject applyToView:info.element.sak_view force:NO];
        
        // 子节点逆序入栈（保证正序遍历）
        NSArray *children = info.element.subElements;
        for (NSInteger i = children.count - 1; i >= 0; i--) {
            _SAKDLNodeInfo *childInfo = [[_SAKDLNodeInfo alloc] init];
            childInfo.element = children[i];
            childInfo.parent = info.element;
            [stack addObject:childInfo];
        }
    }
}
```

### 16.4 LC 114 - 二叉树展开为链表

将二叉树按前序展开为单链表（都用 right 指针连接，left 置 nil）。

**Swift（迭代法，O(1) 空间）**：

```swift
func flatten(_ root: TreeNode?) {
    var current = root
    
    while let cur = current {
        if cur.left != nil {
            // 找左子树的最右节点
            var rightmost = cur.left!
            while rightmost.right != nil {
                rightmost = rightmost.right!
            }
            // 将当前节点的右子树接到左子树最右节点后面
            rightmost.right = cur.right
            // 将左子树移到右边
            cur.right = cur.left
            cur.left = nil
        }
        current = cur.right
    }
}
```

**Objective-C**：

```objectivec
- (void)flatten:(SAKTreeNode *)root {
    SAKTreeNode *current = root;
    
    while (current) {
        if (current.left) {
            SAKTreeNode *rightmost = current.left;
            while (rightmost.right) {
                rightmost = rightmost.right;
            }
            rightmost.right = current.right;
            current.right = current.left;
            current.left = nil;
        }
        current = current.right;
    }
}
```

**思路**：对每个有左子树的节点，将右子树挂到左子树最右节点后面，然后把左子树移到右边。这本质上是 Morris 思想的应用。

```swift
func isValidBST(_ root: TreeNode?) -> Bool {

    func inorder(_ node: TreeNode?) -> Bool {
        guard let node = node else { return true }
        
        if !inorder(node.left) { return false }
```

```swift
func isValidBST2(_ root: TreeNode?) -> Bool {
    func validate(_ node: TreeNode?, _ low: Int, _ high: Int) -> Bool {
        guard let node = node else { return true }
        if node.val <= low || node.val >= high { return false }
        return validate(node.left, low, node.val) &&
               validate(node.right, node.val, high)
    }
    return validate(root, Int.min, Int.max)
}
```

### 16.6 LC 230 - BST 中第 K 小的元素

中序遍历到第 k 个即可停止。

**Swift（迭代中序，提前终止）**：

```swift
func kthSmallest(_ root: TreeNode?, _ k: Int) -> Int {
    var stack = [TreeNode]()
    var current = root
    var count = 0
    
    while current != nil || !stack.isEmpty {
        while let node = current {
            stack.append(node)
            current = node.left
        }
        let node = stack.removeLast()
        count += 1
        if count == k { return node.val }
        current = node.right
    }
    
    return -1  // 不应到达
}
```

```swift
func rightSideView(_ root: TreeNode?) -> [Int] {
```

### 16.8 LC 437 - 路径总和 III

统计树中路径和等于目标值的路径数量（路径不需要从根开始也不需要到叶结束，但方向必须向下）。

**前缀和 + 哈希表解法**（O(n) 时间）：

**Swift**：

```swift
func pathSumIII(_ root: TreeNode?, _ targetSum: Int) -> Int {
    var prefixSumCount: [Int: Int] = [0: 1]
    var count = 0
    
    func dfs(_ node: TreeNode?, _ currentSum: Int) {
        guard let node = node else { return }
        let newSum = currentSum + node.val
        
        // 检查是否存在前缀和使得 newSum - prefixSum == targetSum
        count += prefixSumCount[newSum - targetSum, default: 0]
        
        prefixSumCount[newSum, default: 0] += 1
        dfs(node.left, newSum)
        dfs(node.right, newSum)
        prefixSumCount[newSum, default: 0] -= 1  // 回溯
    }
    
    dfs(root, 0)
    return count
}
```

**Objective-C**：

```objectivec
- (NSInteger)pathSumIII:(SAKTreeNode *)root targetSum:(NSInteger)targetSum {
    NSMutableDictionary<NSNumber *, NSNumber *> *prefixSumCount = [NSMutableDictionary dictionary];
    prefixSumCount[@0] = @1;
    __block NSInteger count = 0;
    
    [self dfsPathSum:root currentSum:0 targetSum:targetSum
      prefixSumCount:prefixSumCount count:&count];
    
    return count;
}

- (void)dfsPathSum:(SAKTreeNode *)node
        currentSum:(NSInteger)currentSum
         targetSum:(NSInteger)targetSum
    prefixSumCount:(NSMutableDictionary<NSNumber *, NSNumber *> *)map
             count:(NSInteger *)count {
    if (!node) return;
    
    NSInteger newSum = currentSum + node.val;
    NSNumber *key = @(newSum - targetSum);
    *count += map[key].integerValue;
    
    NSNumber *sumKey = @(newSum);
    map[sumKey] = @(map[sumKey].integerValue + 1);
    
    [self dfsPathSum:node.left currentSum:newSum targetSum:targetSum
      prefixSumCount:map count:count];
    [self dfsPathSum:node.right currentSum:newSum targetSum:targetSum
      prefixSumCount:map count:count];
    
    map[sumKey] = @(map[sumKey].integerValue - 1);
}
```

### 16.9 LC 662 - 二叉树最大宽度

使用节点编号：根编号 1，左子 2*i，右子 2*i+1。每层宽度 = 该层最右编号 - 最左编号 + 1。

**Swift**：

```swift
func widthOfBinaryTree(_ root: TreeNode?) -> Int {
    guard let root = root else { return 0 }
    var maxWidth = 0
    var queue: [(TreeNode, Int)] = [(root, 1)]
    
    while !queue.isEmpty {
        let size = queue.count
        let leftMost = queue[0].1
        var rightMost = leftMost
        
        for _ in 0..<size {
            let (node, idx) = queue.removeFirst()
            rightMost = idx
            // 防止编号溢出：减去当前层最左编号做归一化
            let normalizedIdx = idx - leftMost
            if let l = node.left {
                queue.append((l, 2 * normalizedIdx + 1))
            }
            if let r = node.right {
                queue.append((r, 2 * normalizedIdx + 2))
            }
        }
        
        maxWidth = max(maxWidth, rightMost - leftMost + 1)
    }
    
    return maxWidth
}
```

> **注意**：编号可能溢出（稀疏树深度大时 2^h 很大），需要做归一化或用任意精度整数。

## 17. 面试题（附答案要点）

### Q1：前序遍历和中序遍历能否唯一确定一棵二叉树？后序和前序呢？

**答**：前序+中序可以唯一确定。前序第一个是根，在中序中定位根把序列分为左右两部分，递归即可重建。后序+前序一般不能唯一确定——当某节点只有一个子节点时，无法区分它是左子还是右子（除非额外规定所有节点度为 0 或 2）。

### Q2：二叉树的最大深度和最小深度有什么区别？为什么最小深度不能简单用 min 递归？

**答**：最大深度 = 根到最深叶的节点数。最小深度 = 根到最浅**叶**节点的节点数。如果一侧子树为空，该侧不是叶（因为叶要求度为 0），所以不能取 0。必须特判：若左为空则只算右侧，反之亦然。

### Q3：Morris 遍历为什么是 O(n) 时间而不是 O(n log n)？

**答**：虽然每个有左子树的节点需要沿右指针找到前驱（看似需 O(h) 步），但每条右边界上的每条边在整个算法中最多被遍历两次（一次建线索，一次断线索）。所有右边界的边数总和 = 树的总边数 = n-1。所以总步数 <= 2(n-1) + n = O(n)。

### Q4：什么场景下选择迭代而不是递归遍历？

**答**：(1) 树非常深（可能栈溢出），如链状树超百万节点；(2) 需要暂停/恢复遍历（实现迭代器/生成器模式）；(3) 性能敏感场景（避免函数调用开销）；(4) 需要灵活控制遍历状态（如后序遍历中需要判断"右子树是否已访问"）。

### Q5：如何在 O(1) 时间内判断一个节点是否为另一个节点的祖先？

**答**：若预先做一次 DFS，记录每个节点的进入时间 `in[v]` 和离开时间 `out[v]`（即 DFS 时间戳），则 u 是 v 的祖先当且仅当 `in[u] <= in[v] && out[v] <= out[u]`。预处理 O(n)，查询 O(1)。

### Q6：LCA 有哪些解法？各自复杂度？

**答**：
- 递归法（LC 236 解法）：O(n) 时间，O(h) 空间。单次查询，无需预处理。
- 倍增法（Binary Lifting）：预处理 O(n log n)，每次查询 O(log n)。适合多次查询。
- Tarjan 离线算法：O(n + q)（q 为查询数），利用并查集，离线处理所有查询。
- Euler Tour + RMQ（稀疏表）：预处理 O(n log n)，查询 O(1)。在线最优。

### Q7：二叉树的直径和最大路径和有何联系与区别？

**答**：直径计算的是边数（或节点数-1），路径和计算的是节点值之和。两者思路类似——都是后序遍历，在每个节点计算"经过该节点的最长路径/最大和"，同时维护全局最优。区别在于：直径只考虑长度，总是非负（取 max(0, ...)）；路径和可能为负，需要用 max(0, gain) 来决定是否选取某子树的贡献。

### Q8：如何将二叉树序列化使得空间最优？

**答**：(1) 前序 + null 标记：需要 O(n) 个标记（n+1 个 null）。(2) 如果是完全二叉树：用层次遍历数组，无需 null 标记。(3) 对于一般二叉树，可以用位图标记结构（1 位表示有/无子），再存值序列，接近信息论下界。(4) Succinct 表示：用 2n+1 位的括号序列表示结构，值另存。

### Q9：iOS 中 hitTest 为什么是逆序遍历 subviews？

**答**：因为后添加的 subview 显示在上层（z-order 更高），用户的点击应该先被上层视图响应。所以 hitTest 从 subviews 数组的末尾（即最上层）开始逆序遍历，找到第一个包含触摸点的最深子视图。

### Q10：红黑树相比 AVL 树在实际工程中更常用的原因是什么？

**答**：(1) 红黑树插入/删除最多旋转 2-3 次，AVL 可能需要 O(log n) 次旋转（尽管常数小）——在写多场景下红黑树维护成本更低。(2) 红黑树的平衡条件更宽松（高度最多 2*log n vs AVL 的 1.44*log n），稍微牺牲查找性能换取更低的维护开销。(3) 工业实现（如 Linux 内核）验证充分，bug 少。(4) 读多写少的场景（如内存数据库索引）AVL 反而更优。

## 18. 易错点与最佳实践

### 18.1 常见错误

| 错误 | 正确做法 |
|------|----------|
| 忘记处理空树（root == nil） | 所有递归函数第一行检查 nil |
| 最小深度对单侧为空的处理 | 单侧为空时不取 0，只递归非空侧 |
| 后序迭代遍历中未正确判断"右子树已访问" | 用 lastVisited 指针标记 |
| 构建树时前序/后序索引更新错误 | 前序递增、后序递减，注意先左后右 vs 先右后左 |
| BST 验证只比较父子而不是整个子树 | 必须用上下界法或中序遍历法 |
| LCA 递归中忘记"节点可以是自己的祖先" | base case 包含 root == p 或 root == q |
| 路径和题目中将负贡献的子树也加入 | 用 max(0, gain) 截断负贡献 |
| 层次遍历中未按层分组（缺少 levelSize 循环）| 每轮循环前记录当前队列长度 |
| Morris 遍历后忘记断开线索导致树结构损坏 | 第二次到达前驱时必须置 nil |
| OC 中 Block 递归捕获 nil | 用 __block 变量 + 实例方法替代 |

### 18.2 最佳实践

1. **优先递归，深度大时切迭代**：递归代码简洁、不易错；只在树可能很深（>1000）时才用迭代。

2. **DFS vs BFS 的选择**：
   - 求深度/路径/子树属性 → DFS（前/中/后序）
   - 逐层处理/最短路径/宽度 → BFS

3. **善用返回值**：递归函数返回有意义的值（高度、是否平衡、最大和等），避免全局变量过多。

4. **哈希表优化查找**：构建树时在中序中找根，务必用哈希表预存下标，将 O(n) 降为 O(1)。

5. **注意引用类型**：Swift 的 TreeNode 是 class（引用类型），修改会影响原树。如需不可变操作，需要深拷贝。

6. **OC 内存管理注意**：
   - TreeNode 用 strong 持有子节点，parent 用 weak/assign 避免循环引用。
   - C 结构体中不能存 strong OC 对象（ARC 限制），需用 OC 对象包装（参见 SAKFlexboxLibrary 的教训）。

7. **测试用例覆盖**：
   - 空树
   - 单节点
   - 只有左子树（左斜）
   - 只有右子树（右斜）
   - 完全二叉树
   - 满二叉树
   - 含负值节点
