# 贪心算法（Greedy Algorithm）

> 贪心算法是一种在每一步都选择当前最优解（局部最优），期望最终达到全局最优的算法策略。它不回退、不枚举所有可能，而是凭借问题本身的「贪心选择性质」与「最优子结构」，以极高的效率解决一类特定问题。

---

## 一、是什么：贪心算法的定义与直觉

### 1.1 核心定义

贪心算法（Greedy Algorithm）是一种逐步构建解的策略：在每一个决策点，都做出当前看起来最优的选择，且一旦做出决策便不再回退。形式化定义：

- 给定一个优化问题（最大化或最小化某个目标函数）
- 贪心算法将问题分解为一系列子决策
- 每次子决策独立选取局部最优
- 所有子决策组合后得到的解即为最终解

### 1.2 直觉理解

用生活类比：假设你要找零 36 分钱（美制硬币：25¢、10¢、5¢、1¢），贪心策略是每次尽量选面值最大的硬币——先选 25¢，再选 10¢，再选 1¢，共 3 枚。这恰好是最少硬币数。

但注意：并非所有面值系统都能用贪心得到最优解。例如面值为 {1, 3, 4} 时，找零 6 元：贪心选 4+1+1=3 枚，但最优是 3+3=2 枚。这说明贪心算法**只在满足特定条件**时才正确。

### 1.3 贪心 vs 动态规划 vs 回溯

| 特征 | 贪心 | 动态规划 | 回溯 |
|------|------|----------|------|
| 决策方式 | 每步取局部最优，不回退 | 枚举所有子问题，取全局最优 | 穷举所有路径，剪枝 |
| 时间复杂度 | 通常 O(n log n) 或 O(n) | 通常 O(n²) 或 O(n·W) | 指数级 |
| 适用条件 | 贪心选择性质 + 最优子结构 | 最优子结构 + 重叠子问题 | 通用（无特殊结构） |
| 正确性保证 | 需要证明 | 保证最优 | 保证最优（但慢） |
| 典型问题 | 活动选择、Huffman、最小生成树 | 背包、LCS、编辑距离 | N皇后、全排列 |

## 二、为什么：贪心算法的理论基础

### 2.1 贪心选择性质（Greedy Choice Property）

**定义**：一个全局最优解可以通过局部最优选择来达到。即存在一个最优解包含贪心策略在第一步所做的选择。

**证明思路**（交换论证法 / Exchange Argument）：
1. 假设存在一个最优解 OPT 不包含贪心选择 g
2. 证明可以将 OPT 中的某个元素替换为 g，得到的解 OPT' 不比 OPT 差
3. 因此存在包含 g 的最优解，贪心选择安全

### 2.2 最优子结构（Optimal Substructure）

**定义**：做出贪心选择后，剩余子问题的最优解与原问题的最优解一致。即原问题的最优解包含子问题的最优解。

### 2.3 拟阵理论（Matroid Theory）简述

在组合优化中，满足拟阵（Matroid）结构的问题都能用贪心算法得到最优解。拟阵 M = (S, I) 满足：
- 遗传性：若 A ∈ I 且 B ⊆ A，则 B ∈ I
- 交换性：若 A, B ∈ I 且 |A| < |B|，则存在 x ∈ B\A 使得 A ∪ {x} ∈ I

经典例子：图的森林（无向无环子图集合）构成拟阵，因此 Kruskal 最小生成树算法（贪心选最小权边）正确。

### 2.4 如何证明贪心正确性

常用三种方法：

**方法一：交换论证法（Exchange Argument）**
- 假设最优解与贪心解不同
- 找到第一个不同的位置
- 证明可以「交换」使解不变差
- 递推证明贪心解即最优解

**方法二：归纳法**
- 归纳假设：前 k 步贪心选择后，存在最优解包含这 k 步
- 归纳步骤：证明第 k+1 步贪心选择仍可被最优解包含

**方法三：反证法**
- 假设贪心解不是最优解
- 推导出矛盾

## 三、怎么用：贪心算法的通用模板

### 3.1 算法框架

```
贪心算法通用步骤：
1. 将问题分解为一系列选择
2. 确定贪心策略（排序准则 / 选择规则）
3. 按策略排序或建堆
4. 依次做出选择，更新状态
5. 返回结果
```

### 3.2 Swift 通用模板

```swift
/// 贪心算法通用模板
/// - items: 待选择的元素集合
/// - 返回: 贪心构建的解
func greedySolve<T>(items: [T], comparator: (T, T) -> Bool) -> [T] {
    // 1. 按贪心策略排序
    let sorted = items.sorted(by: comparator)
    
    // 2. 逐步选择
    var result: [T] = []
    var state = initialState() // 当前状态
    
    for item in sorted {
        if canSelect(item, currentState: state) {
            result.append(item)
            state = updateState(state, with: item)
        }
    }
    
    return result
}
```

### 3.3 Objective-C 通用模板

```objectivec
/// 贪心算法通用模板
- (NSArray *)greedySolveWithItems:(NSArray *)items {
    // 1. 按贪心策略排序
    NSArray *sorted = [items sortedArrayUsingComparator:^NSComparisonResult(id a, id b) {
        return [self greedyCompare:a with:b];
    }];
    
    // 2. 逐步选择
    NSMutableArray *result = [NSMutableArray array];
    id state = [self initialState];
    
    for (id item in sorted) {
        if ([self canSelect:item currentState:state]) {
            [result addObject:item];
            state = [self updateState:state withItem:item];
        }
    }
    
    return [result copy];
}
```

## 四、经典问题详解

### 4.1 活动选择问题（Activity Selection）

**问题描述**：给定 n 个活动，每个活动有开始时间 s[i] 和结束时间 f[i]，选择最多数量的互不重叠活动。

**贪心策略**：按结束时间从早到晚排序，依次选择与已选活动不冲突的活动。

**正确性证明**（交换论证）：
- 设 OPT 为最优解，a* 为结束时间最早的活动
- 若 OPT 不含 a*，设 OPT 第一个活动为 a1（f[a1] ≥ f[a*]）
- 用 a* 替换 a1，得 OPT'，因 f[a*] ≤ f[a1]，OPT' 中后续活动仍不冲突
- 故 |OPT'| = |OPT|，a* 可安全选入

**Swift 实现**：

```swift
struct Activity {
    let start: Int
    let end: Int
}

func activitySelection(_ activities: [Activity]) -> [Activity] {
    guard !activities.isEmpty else { return [] }
    
    // 按结束时间排序
    let sorted = activities.sorted { $0.end < $1.end }
    
    var selected: [Activity] = [sorted[0]]
    var lastEnd = sorted[0].end
    
    for i in 1..<sorted.count {
        if sorted[i].start >= lastEnd {
            selected.append(sorted[i])
            lastEnd = sorted[i].end
        }
    }
    
    return selected
}

// 示例
let activities = [
    Activity(start: 1, end: 4),
    Activity(start: 3, end: 5),
    Activity(start: 0, end: 6),
    Activity(start: 5, end: 7),
    Activity(start: 3, end: 9),
    Activity(start: 5, end: 9),
    Activity(start: 6, end: 10),
    Activity(start: 8, end: 11),
    Activity(start: 8, end: 12),
    Activity(start: 2, end: 14),
    Activity(start: 12, end: 16)
]
let result = activitySelection(activities)
// result: [(1,4), (5,7), (8,11), (12,16)]，共 4 个活动
```

**Objective-C 实现**：

```objectivec
@interface Activity : NSObject
@property (nonatomic, assign) NSInteger start;
@property (nonatomic, assign) NSInteger end;
+ (instancetype)activityWithStart:(NSInteger)s end:(NSInteger)e;
@end

@implementation Activity
+ (instancetype)activityWithStart:(NSInteger)s end:(NSInteger)e {
    Activity *a = [[Activity alloc] init];
    a.start = s;
    a.end = e;
    return a;
}
@end

- (NSArray<Activity *> *)activitySelection:(NSArray<Activity *> *)activities {
    if (activities.count == 0) return @[];
    
    // 按结束时间排序
    NSArray *sorted = [activities sortedArrayUsingComparator:^NSComparisonResult(Activity *a, Activity *b) {
        return a.end < b.end ? NSOrderedAscending : (a.end > b.end ? NSOrderedDescending : NSOrderedSame);
    }];
    
    NSMutableArray<Activity *> *selected = [NSMutableArray arrayWithObject:sorted[0]];
    NSInteger lastEnd = ((Activity *)sorted[0]).end;
    
    for (NSInteger i = 1; i < sorted.count; i++) {
        Activity *act = sorted[i];
        if (act.start >= lastEnd) {
            [selected addObject:act];
            lastEnd = act.end;
        }
    }
    
    return [selected copy];
}
```

**复杂度分析**：
- 时间：O(n log n)（排序主导）
- 空间：O(n)（排序额外空间 + 结果数组）

### 4.2 分数背包问题（Fractional Knapsack）

**问题描述**：给定 n 个物品，每个物品有重量 w[i] 和价值 v[i]，背包容量为 W。物品可以被分割（取一部分），求最大价值。

**贪心策略**：按单位重量价值（v[i]/w[i]）从大到小排序，优先装性价比高的物品。

**正确性**：分数背包满足贪心选择性质。若最优解中某个高性价比物品 x 未被完全选取，而低性价比物品 y 被选取，则可将 y 的部分替换为 x 的等量重量，价值只会更大。

**Swift 实现**：

```swift
struct Item {
    let weight: Double
    let value: Double
    var ratio: Double { value / weight }
}

func fractionalKnapsack(items: [Item], capacity: Double) -> Double {
    guard capacity > 0 && !items.isEmpty else { return 0 }
    
    // 按性价比降序排列
    let sorted = items.sorted { $0.ratio > $1.ratio }
    
    var remainCapacity = capacity
    var totalValue: Double = 0
    
    for item in sorted {
        if remainCapacity <= 0 { break }
        
        if item.weight <= remainCapacity {
            // 整个物品放入
            totalValue += item.value
            remainCapacity -= item.weight
        } else {
            // 放入部分物品
            totalValue += item.ratio * remainCapacity
            remainCapacity = 0
        }
    }
    
    return totalValue
}

// 示例
let items = [
    Item(weight: 10, value: 60),   // ratio = 6
    Item(weight: 20, value: 100),  // ratio = 5
    Item(weight: 30, value: 120)   // ratio = 4
]
let maxValue = fractionalKnapsack(items: items, capacity: 50)
// 取全部 item1(60) + 全部 item2(100) + item3 的 20/30 = 80
// 总价值 = 60 + 100 + 80 = 240
```

**Objective-C 实现**：

```objectivec
@interface KnapsackItem : NSObject
@property (nonatomic, assign) double weight;
@property (nonatomic, assign) double value;
@property (nonatomic, readonly) double ratio;
+ (instancetype)itemWithWeight:(double)w value:(double)v;
@end

@implementation KnapsackItem
+ (instancetype)itemWithWeight:(double)w value:(double)v {
    KnapsackItem *item = [[KnapsackItem alloc] init];
    item.weight = w;
    item.value = v;
    return item;
}
- (double)ratio {
    return self.weight > 0 ? self.value / self.weight : 0;
}
@end

- (double)fractionalKnapsack:(NSArray<KnapsackItem *> *)items capacity:(double)capacity {
    if (items.count == 0 || capacity <= 0) return 0;
    
    NSArray *sorted = [items sortedArrayUsingComparator:^NSComparisonResult(KnapsackItem *a, KnapsackItem *b) {
        double diff = b.ratio - a.ratio;
        return diff > 0 ? NSOrderedDescending : (diff < 0 ? NSOrderedAscending : NSOrderedSame);
    }];
    
    double remainCapacity = capacity;
    double totalValue = 0;
    
    for (KnapsackItem *item in sorted) {
        if (remainCapacity <= 0) break;
        
        if (item.weight <= remainCapacity) {
            totalValue += item.value;
            remainCapacity -= item.weight;
        } else {
            totalValue += item.ratio * remainCapacity;
            remainCapacity = 0;
        }
    }
    
    return totalValue;
}
```

**复杂度**：
- 时间：O(n log n)
- 空间：O(n)

**注意**：0-1 背包（物品不可分割）不能用贪心，必须用动态规划。

### 4.3 Huffman 编码（Huffman Coding）

**问题描述**：给定一组字符及其频率，构造前缀编码使得编码后总长度最小。

**贪心策略**：每次取频率最小的两个节点合并为新节点（频率相加），重复直到只剩一棵树。

**正确性**：频率最低的两个字符应在最深层（编码最长），且互为兄弟。交换论证可证明：若最优树中最深层两兄弟不是最低频的两个，交换后编码总长只会更短或不变。

**Swift 实现**：

```swift
class HuffmanNode: Comparable {
    let char: Character?
    let freq: Int
    var left: HuffmanNode?
    var right: HuffmanNode?
    
    init(char: Character?, freq: Int, left: HuffmanNode? = nil, right: HuffmanNode? = nil) {
        self.char = char
        self.freq = freq
        self.left = left
        self.right = right
    }
    
    static func < (lhs: HuffmanNode, rhs: HuffmanNode) -> Bool {
        return lhs.freq < rhs.freq
    }
    
    static func == (lhs: HuffmanNode, rhs: HuffmanNode) -> Bool {
        return lhs.freq == rhs.freq
    }
}

/// 简易最小堆实现（用于 Huffman）
struct MinHeap<T: Comparable> {
    private var elements: [T] = []
    
    var isEmpty: Bool { elements.isEmpty }
    var count: Int { elements.count }
    
    mutating func insert(_ element: T) {
        elements.append(element)
        siftUp(elements.count - 1)
    }
    
    mutating func extractMin() -> T? {
        guard !elements.isEmpty else { return nil }
        if elements.count == 1 { return elements.removeLast() }
        let min = elements[0]
        elements[0] = elements.removeLast()
        siftDown(0)
        return min
    }
    
    private mutating func siftUp(_ index: Int) {
        var i = index
        while i > 0 {
            let parent = (i - 1) / 2
            if elements[i] < elements[parent] {
                elements.swapAt(i, parent)
                i = parent
            } else { break }
        }
    }
    
    private mutating func siftDown(_ index: Int) {
        var i = index
        let count = elements.count
        while true {
            var smallest = i
            let left = 2 * i + 1, right = 2 * i + 2
            if left < count && elements[left] < elements[smallest] { smallest = left }
            if right < count && elements[right] < elements[smallest] { smallest = right }
            if smallest == i { break }
            elements.swapAt(i, smallest)
            i = smallest
        }
    }
}

func buildHuffmanTree(frequencies: [(Character, Int)]) -> HuffmanNode? {
    guard !frequencies.isEmpty else { return nil }
    
    var heap = MinHeap<HuffmanNode>()
    for (char, freq) in frequencies {
        heap.insert(HuffmanNode(char: char, freq: freq))
    }
    
    if heap.count == 1 {
        let node = heap.extractMin()!
        return HuffmanNode(char: nil, freq: node.freq, left: node)
    }
    
    while heap.count > 1 {
        let left = heap.extractMin()!
        let right = heap.extractMin()!
        let merged = HuffmanNode(char: nil, freq: left.freq + right.freq, left: left, right: right)
        heap.insert(merged)
    }
    
    return heap.extractMin()
}

func generateCodes(_ root: HuffmanNode?, prefix: String = "", codes: inout [Character: String]) {
    guard let node = root else { return }
    
    if let char = node.char {
        codes[char] = prefix.isEmpty ? "0" : prefix
        return
    }
    
    generateCodes(node.left, prefix: prefix + "0", codes: &codes)
    generateCodes(node.right, prefix: prefix + "1", codes: &codes)
}

// 示例
let freqs: [(Character, Int)] = [("a", 5), ("b", 9), ("c", 12), ("d", 13), ("e", 16), ("f", 45)]
if let root = buildHuffmanTree(frequencies: freqs) {
    var codes: [Character: String] = [:]
    generateCodes(root, codes: &codes)
    // 可能的编码: f=0, c=100, d=101, a=1100, b=1101, e=111
    // 加权路径长度 WPL = 45*1 + 12*3 + 13*3 + 5*4 + 9*4 + 16*3 = 224
}
```

**Objective-C 实现**：

```objectivec
@interface HuffmanNode : NSObject
@property (nonatomic, strong, nullable) NSString *character;
@property (nonatomic, assign) NSInteger freq;
@property (nonatomic, strong, nullable) HuffmanNode *left;
@property (nonatomic, strong, nullable) HuffmanNode *right;
+ (instancetype)nodeWithChar:(nullable NSString *)c freq:(NSInteger)f;
@end

@implementation HuffmanNode
+ (instancetype)nodeWithChar:(NSString *)c freq:(NSInteger)f {
    HuffmanNode *node = [[HuffmanNode alloc] init];
    node.character = c;
    node.freq = f;
    return node;
}
@end

- (HuffmanNode *)buildHuffmanTree:(NSArray<HuffmanNode *> *)nodes {
    if (nodes.count == 0) return nil;
    
    NSMutableArray<HuffmanNode *> *heap = [nodes mutableCopy];
    [heap sortUsingComparator:^NSComparisonResult(HuffmanNode *a, HuffmanNode *b) {
        return a.freq < b.freq ? NSOrderedAscending : (a.freq > b.freq ? NSOrderedDescending : NSOrderedSame);
    }];
    
    while (heap.count > 1) {
        HuffmanNode *left = heap[0];
        [heap removeObjectAtIndex:0];
        HuffmanNode *right = heap[0];
        [heap removeObjectAtIndex:0];
        
        HuffmanNode *merged = [HuffmanNode nodeWithChar:nil freq:left.freq + right.freq];
        merged.left = left;
        merged.right = right;
        
        // 二分插入保持有序
        NSInteger insertIdx = 0;
        for (NSInteger i = 0; i < heap.count; i++) {
            if (heap[i].freq >= merged.freq) { insertIdx = i; break; }
            insertIdx = i + 1;
        }
        [heap insertObject:merged atIndex:insertIdx];
    }
    
    return heap.firstObject;
}

- (void)generateCodes:(HuffmanNode *)node prefix:(NSString *)prefix result:(NSMutableDictionary<NSString *, NSString *> *)codes {
    if (!node) return;
    
    if (node.character) {
        codes[node.character] = prefix.length > 0 ? prefix : @"0";
        return;
    }
    
    [self generateCodes:node.left prefix:[prefix stringByAppendingString:@"0"] result:codes];
    [self generateCodes:node.right prefix:[prefix stringByAppendingString:@"1"] result:codes];
}
```

**复杂度**：
- 时间：O(n log n)（n 次堆操作，每次 O(log n)）
- 空间：O(n)（堆 + 树节点）

### 4.4 跳跃游戏（Jump Game）— LeetCode 55

**问题描述**：给定非负整数数组 nums，nums[i] 表示从位置 i 最远能跳 nums[i] 步。判断能否到达最后一个位置。

**贪心策略**：维护当前能到达的最远位置 maxReach，遍历数组时不断更新。若某个位置 i > maxReach，说明无法到达。

**Swift 实现**：

```swift
func canJump(_ nums: [Int]) -> Bool {
    var maxReach = 0
    
    for i in 0..<nums.count {
        if i > maxReach { return false }
        maxReach = max(maxReach, i + nums[i])
        if maxReach >= nums.count - 1 { return true }
    }
    
    return true
}

// 示例
canJump([2, 3, 1, 1, 4])  // true: 0→1→4 或 0→2→3→4
canJump([3, 2, 1, 0, 4])  // false: 最远只能到位置3
```

**Objective-C 实现**：

```objectivec
- (BOOL)canJump:(NSArray<NSNumber *> *)nums {
    NSInteger maxReach = 0;
    
    for (NSInteger i = 0; i < nums.count; i++) {
        if (i > maxReach) return NO;
        maxReach = MAX(maxReach, i + nums[i].integerValue);
        if (maxReach >= (NSInteger)nums.count - 1) return YES;
    }
    
    return YES;
}
```

**复杂度**：O(n) 时间，O(1) 空间。

### 4.5 跳跃游戏 II（Jump Game II）— LeetCode 45

**问题描述**：给定非负整数数组 nums，保证能到达最后位置，求最少跳跃次数。

**贪心策略**：BFS 层级思想——维护当前跳跃能到达的边界 curEnd，以及在该范围内下一步能到达的最远位置 farthest。到达 curEnd 时步数+1。

**Swift 实现**：

```swift
func jump(_ nums: [Int]) -> Int {
    guard nums.count > 1 else { return 0 }
    
    var jumps = 0
    var curEnd = 0       // 当前这一跳的最远边界
    var farthest = 0    // 从 [0, curEnd] 中下一步能到的最远
    
    for i in 0..<nums.count - 1 {
        farthest = max(farthest, i + nums[i])
        
        if i == curEnd {
            jumps += 1
            curEnd = farthest
            if curEnd >= nums.count - 1 { break }
        }
    }
    
    return jumps
}

// 示例
jump([2, 3, 1, 1, 4])  // 2: 0→1→4
jump([2, 3, 0, 1, 4])  // 2: 0→1→4
```

**Objective-C 实现**：

```objectivec
- (NSInteger)jump:(NSArray<NSNumber *> *)nums {
    if (nums.count <= 1) return 0;
    
    NSInteger jumps = 0;
    NSInteger curEnd = 0;
    NSInteger farthest = 0;
    
    for (NSInteger i = 0; i < (NSInteger)nums.count - 1; i++) {
        farthest = MAX(farthest, i + nums[i].integerValue);
        
        if (i == curEnd) {
            jumps++;
            curEnd = farthest;
            if (curEnd >= (NSInteger)nums.count - 1) break;
        }
    }
    
    return jumps;
}
```

### 4.6 分发糖果（Candy）— LeetCode 135

**问题描述**：n 个孩子站一排，每人有评分 ratings[i]。给每人至少 1 颗糖，相邻孩子中评分高的必须比评分低的拿更多糖。求最少需要多少糖。

**贪心策略**：两遍遍历——先从左到右保证右边比左边高分的多一颗；再从右到左保证左边比右边高分的多一颗。取两次结果的最大值。

**Swift 实现**：

```swift
func candy(_ ratings: [Int]) -> Int {
    let n = ratings.count
    guard n > 0 else { return 0 }
    
    var candies = Array(repeating: 1, count: n)
    
    // 从左到右：若右边比左边分高，右边 = 左边 + 1
    for i in 1..<n {
        if ratings[i] > ratings[i - 1] {
            candies[i] = candies[i - 1] + 1
        }
    }
    
    // 从右到左：若左边比右边分高，左边 = max(当前, 右边+1)
    for i in stride(from: n - 2, through: 0, by: -1) {
        if ratings[i] > ratings[i + 1] {
            candies[i] = max(candies[i], candies[i + 1] + 1)
        }
    }
    
    return candies.reduce(0, +)
}

// 示例
candy([1, 0, 2])      // 5: [2,1,2]
candy([1, 2, 2])      // 4: [1,2,1]
candy([1, 3, 2, 2, 1])  // 7: [1,2,1,2,1] -> 需验证 -> [1,3,1,2,1]
```

**Objective-C 实现**：

```objectivec
- (NSInteger)candy:(NSArray<NSNumber *> *)ratings {
    NSInteger n = ratings.count;
    if (n == 0) return 0;
    
    NSMutableArray<NSNumber *> *candies = [NSMutableArray arrayWithCapacity:n];
    for (NSInteger i = 0; i < n; i++) {
        [candies addObject:@1];
    }
    
    // 从左到右
    for (NSInteger i = 1; i < n; i++) {
        if (ratings[i].integerValue > ratings[i - 1].integerValue) {
            candies[i] = @(candies[i - 1].integerValue + 1);
        }
    }
    
    // 从右到左
    for (NSInteger i = n - 2; i >= 0; i--) {
        if (ratings[i].integerValue > ratings[i + 1].integerValue) {
            candies[i] = @(MAX(candies[i].integerValue, candies[i + 1].integerValue + 1));
        }
    }
    
    NSInteger total = 0;
    for (NSNumber *c in candies) {
        total += c.integerValue;
    }
    return total;
}
```

**复杂度**：O(n) 时间，O(n) 空间。

### 4.7 无重叠区间（Non-overlapping Intervals）— LeetCode 435

**问题描述**：给定区间集合 intervals，返回需要移除的最少区间数量，使剩余区间互不重叠。

**贪心策略**：本质是活动选择的逆问题——最多能保留多少个不重叠区间，用总数减去保留数即为需移除数。按结束时间排序，贪心选不重叠的。

**Swift 实现**：

```swift
func eraseOverlapIntervals(_ intervals: [[Int]]) -> Int {
    guard intervals.count > 1 else { return 0 }
    
    // 按结束时间排序
    let sorted = intervals.sorted { $0[1] < $1[1] }
    
    var kept = 1
    var lastEnd = sorted[0][1]
    
    for i in 1..<sorted.count {
        if sorted[i][0] >= lastEnd {
            kept += 1
            lastEnd = sorted[i][1]
        }
    }
    
    return intervals.count - kept
}

// 示例
eraseOverlapIntervals([[1,2],[2,3],[3,4],[1,3]])  // 1: 移除[1,3]
eraseOverlapIntervals([[1,2],[1,2],[1,2]])         // 2: 保留1个
eraseOverlapIntervals([[1,2],[2,3]])               // 0: 已不重叠
```

**Objective-C 实现**：

```objectivec

        }

}
```

```swift
            arrowPos = sorted[i][1]
        }
```

### 4.9 加油站问题（Gas Station）— LeetCode 134

**问题描述**：环路上有 n 个加油站，第 i 站有 gas[i] 升汽油。从第 i 站到 i+1 站消耗 cost[i] 升汽油。从某站出发（油箱初始为空），问能否绕环路一周？返回出发站编号，无解返回 -1。

**贪心策略**：
1. 如果总 gas < 总 cost，无解
2. 从头遍历，维护当前油量 tank。当 tank < 0 时，说明从上个起点到当前位置都不行，起点设为下一站，tank 重置为 0

**正确性**：若总 gas ≥ 总 cost 则一定有解。设出发点为 s，从 s 到 s 之前任何位置 j 开始都会在某处 tank < 0（否则 j 就是更早的出发点），所以 s 是唯一解。

**Swift 实现**：

```swift
func canCompleteCircuit(_ gas: [Int], _ cost: [Int]) -> Int {
    let n = gas.count
    var totalSurplus = 0
    var currentSurplus = 0
    var startStation = 0
    
    for i in 0..<n {
        let surplus = gas[i] - cost[i]
        totalSurplus += surplus
        currentSurplus += surplus
        
        if currentSurplus < 0 {
            // 从当前站出发不可行，尝试下一站
            startStation = i + 1
            currentSurplus = 0
        }
    }
    
    return totalSurplus >= 0 ? startStation : -1
}

// 示例
canCompleteCircuit([1,2,3,4,5], [3,4,5,1,2])  // 3
canCompleteCircuit([2,3,4], [3,4,3])           // -1
```

**Objective-C 实现**：

```objectivec
- (NSInteger)canCompleteCircuit:(NSArray<NSNumber *> *)gas cost:(NSArray<NSNumber *> *)cost {
    NSInteger n = gas.count;
    NSInteger totalSurplus = 0;
    NSInteger currentSurplus = 0;
    NSInteger startStation = 0;

        NSInteger surplus = gas[i].integerValue - cost[i].integerValue;
        totalSurplus += surplus;
        currentSurplus += surplus;

            startStation = i + 1;
            currentSurplus = 0;
        }
    }

}
```

**复杂度**：O(n) 时间，O(1) 空间。

### 4.10 合并区间（Merge Intervals）— LeetCode 56

**问题描述**：给出若干区间，合并所有重叠的区间。

**贪心策略**：按起点排序，依次尝试合并——若当前区间的起点 ≤ 上一个合并区间的终点，合并（终点取较大值）；否则开始新区间。

**Swift 实现**：

```swift
func merge(_ intervals: [[Int]]) -> [[Int]] {
    guard intervals.count > 1 else { return intervals }
    
    let sorted = intervals.sorted { $0[0] < $1[0] }
    var merged: [[Int]] = [sorted[0]]
    
    for i in 1..<sorted.count {
        if sorted[i][0] <= merged.last![1] {
            // 重叠，合并
            merged[merged.count - 1][1] = max(merged.last![1], sorted[i][1])
        } else {
            // 不重叠，新增
            merged.append(sorted[i])
        }
    }
    
    return merged
}

// 示例
merge([[1,3],[2,6],[8,10],[15,18]])  // [[1,6],[8,10],[15,18]]
merge([[1,4],[4,5]])                  // [[1,5]]
```

**Objective-C 实现**：

```objectivec
- (NSArray<NSArray<NSNumber *> *> *)merge:(NSArray<NSArray<NSNumber *> *> *)intervals {
    if (intervals.count <= 1) return intervals;
    
    NSArray *sorted = [intervals sortedArrayUsingComparator:^NSComparisonResult(NSArray *a, NSArray *b) {
        return [a[0] compare:b[0]];
    }];
    
    NSMutableArray *merged = [NSMutableArray arrayWithObject:[sorted[0] mutableCopy]];
    
    for (NSInteger i = 1; i < sorted.count; i++) {
        NSMutableArray *last = merged.lastObject;
        if ([sorted[i][0] integerValue] <= [last[1] integerValue]) {
            last[1] = @(MAX([last[1] integerValue], [sorted[i][1] integerValue]));
        } else {
            [merged addObject:[sorted[i] mutableCopy]];
        }
    }
    
    return [merged copy];
}
```

**复杂度**：O(n log n) 时间，O(n) 空间。

### 4.11 任务调度器（Task Scheduler）— LeetCode 621

**问题描述**：给定任务数组（字母表示任务类型）和冷却间隔 n。相同任务之间必须间隔至少 n 个时间单位。求完成所有任务的最短时间。

**贪心策略**：频率最高的任务决定框架。设最高频率为 maxFreq，有 maxCount 个任务都是最高频率。最短时间 = max(tasks.length, (maxFreq - 1) * (n + 1) + maxCount)。

**直觉**：将最高频任务排成 maxFreq 行，每行间隔 n+1 个位置，其他任务填入空位。若空位不够，则所有任务紧排。

**Swift 实现**：

```swift
func leastInterval(_ tasks: [Character], _ n: Int) -> Int {
    var freq = Array(repeating: 0, count: 26)
    for task in tasks {
        freq[Int(task.asciiValue! - Character("A").asciiValue!)] += 1
    }
    
    let maxFreq = freq.max()!
    let maxCount = freq.filter { $0 == maxFreq }.count
    
    // 框架长度 = (maxFreq-1) 个 "块"（每块 n+1 长）+ 最后一块放所有最高频任务
    let formulaResult = (maxFreq - 1) * (n + 1) + maxCount
    
    return max(tasks.count, formulaResult)
}

// 示例
leastInterval(Array("AAABBBCCC"), 2)  // 9: ABC|ABC|ABC（无空闲）
leastInterval(Array("AAABBB"), 2)     // 8: AB_|AB_|AB
leastInterval(Array("AAAABC"), 2)     // 10: A__|A__|A__|AB C → 实际 A B C A _ _ A _ _ A = 10
```

**Objective-C 实现**：

```objectivec
- (NSInteger)leastInterval:(NSArray<NSString *> *)tasks n:(NSInteger)n {
    NSInteger freq[26] = {0};
    for (NSString *task in tasks) {
        freq[[task characterAtIndex:0] - 'A']++;
    }
    
    NSInteger maxFreq = 0;
    for (int i = 0; i < 26; i++) {
        maxFreq = MAX(maxFreq, freq[i]);
    }
    
    NSInteger maxCount = 0;
    for (int i = 0; i < 26; i++) {
        if (freq[i] == maxFreq) maxCount++;
    }
    
    NSInteger formulaResult = (maxFreq - 1) * (n + 1) + maxCount;
    return MAX((NSInteger)tasks.count, formulaResult);
}
```

**复杂度**：O(n) 时间（n 为任务数），O(1) 空间（26 个字母）。

### 4.12 根据身高重建队列（Queue Reconstruction by Height）— LeetCode 406

**问题描述**：每人有属性 (h, k)，h 为身高，k 为前面有多少人身高 ≥ h。给出打乱的数组，重建队列。

**贪心策略**：
1. 按身高降序排列（身高相同按 k 升序）
2. 依次将每人插入结果数组的第 k 个位置

**正确性**：先插入高个子，他们的 k 值就是最终位置（因为后插入的矮个子不影响高个子的 k 计数）。

**Swift 实现**：

```swift
func reconstructQueue(_ people: [[Int]]) -> [[Int]] {
    // 按身高降序，身高相同按 k 升序
    let sorted = people.sorted { 
        $0[0] != $1[0] ? $0[0] > $1[0] : $0[1] < $1[1]
    }
    
    var queue: [[Int]] = []
    for person in sorted {
        queue.insert(person, at: person[1])
    }
    
    return queue
}

// 示例
reconstructQueue([[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]])
// 排序后: [7,0],[7,1],[6,1],[5,0],[5,2],[4,4]
// 插入过程:
//   [7,0]
//   [7,0],[7,1]
//   [7,0],[6,1],[7,1]
//   [5,0],[7,0],[6,1],[7,1]
//   [5,0],[7,0],[5,2],[6,1],[7,1]
//   [5,0],[7,0],[5,2],[6,1],[4,4],[7,1]
// 结果: [[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]]
```

**Objective-C 实现**：

```objectivec
- (NSArray<NSArray<NSNumber *> *> *)reconstructQueue:(NSArray<NSArray<NSNumber *> *> *)people {
    NSArray *sorted = [people sortedArrayUsingComparator:^NSComparisonResult(NSArray *a, NSArray *b) {
        if ([a[0] integerValue] != [b[0] integerValue]) {
            return [b[0] compare:a[0]]; // 身高降序
        }
        return [a[1] compare:b[1]]; // k 升序
    }];
    
    NSMutableArray *queue = [NSMutableArray array];
    for (NSArray *person in sorted) {
        [queue insertObject:person atIndex:[person[1] integerValue]];
    }
    
    return [queue copy];
}
```

**复杂度**：O(n²) 时间（n 次插入，每次 O(n)），O(n) 空间。

### 5.1 最优装载 / 会议室安排

**会议室 II（LeetCode 253）**：给定会议时间区间，求最少需要几个会议室。

**贪心 + 最小堆**：按开始时间排序，用堆维护各会议室最早结束时间。新会议来时，若堆顶结束时间 ≤ 新会议开始时间，复用该会议室；否则新开一间。

```swift
func minMeetingRooms(_ intervals: [[Int]]) -> Int {
    guard !intervals.isEmpty else { return 0 }
    
    let sorted = intervals.sorted { $0[0] < $1[0] }
    
    // 最小堆存储每个会议室的结束时间
    var endTimes: [Int] = []  // 简化用数组模拟
    
    func heapPush(_ val: Int) {
        endTimes.append(val)
        var i = endTimes.count - 1
        while i > 0 {
            let p = (i - 1) / 2
            if endTimes[i] < endTimes[p] {
                endTimes.swapAt(i, p)
                i = p
            } else { break }
        }
    }
    
    func heapPop() -> Int {
        let val = endTimes[0]
        endTimes[0] = endTimes.removeLast()
        var i = 0
        while true {
            var s = i
            let l = 2*i+1, r = 2*i+2
            if l < endTimes.count && endTimes[l] < endTimes[s] { s = l }
            if r < endTimes.count && endTimes[r] < endTimes[s] { s = r }
            if s == i { break }
            endTimes.swapAt(i, s)
            i = s
        }
        return val
    }
    
    heapPush(sorted[0][1])
    
    for i in 1..<sorted.count {
        if endTimes[0] <= sorted[i][0] {
            _ = heapPop()  // 复用会议室
        }
        heapPush(sorted[i][1])
    }
    
    return endTimes.count
}
```

**复杂度**：O(n log n) 时间，O(n) 空间。

### 5.2 字符串相关贪心

**去除重复字母使结果最小（LeetCode 316 / 1081）**：给定字符串 s，去除重复字母使每个字母只出现一次，且结果字典序最小。

**贪心 + 单调栈**：维护一个递增栈，遍历字符时：若栈顶字符 > 当前字符，且栈顶字符后面还会出现，则弹出栈顶。

```swift
func removeDuplicateLetters(_ s: String) -> String {
    let chars = Array(s)
    var lastIndex = [Character: Int]()
    for (i, c) in chars.enumerated() {
        lastIndex[c] = i
    }
    
    var stack: [Character] = []
    var inStack = Set<Character>()
    
    for (i, c) in chars.enumerated() {
        if inStack.contains(c) { continue }
        
        // 贪心：如果栈顶比当前大，且栈顶后面还会出现，弹出
        while let top = stack.last, top > c, lastIndex[top]! > i {
            inStack.remove(top)
            stack.removeLast()
        }
        
        stack.append(c)
        inStack.insert(c)
    }
    
    return String(stack)
}

// 示例
removeDuplicateLetters("bcabc")   // "abc"
removeDuplicateLetters("cbacdcbc") // "acdb"
```

**Objective-C 实现**：

```objectivec
- (NSString *)removeDuplicateLetters:(NSString *)s {
    NSInteger n = s.length;
    if (n <= 1) return s;
    
    // 记录每个字符最后出现的位置
    NSInteger lastIndex[26] = {0};
    for (NSInteger i = 0; i < n; i++) {
        lastIndex[[s characterAtIndex:i] - 'a'] = i;
    }
    
    NSMutableArray<NSNumber *> *stack = [NSMutableArray array];
    BOOL inStack[26] = {NO};
    
    for (NSInteger i = 0; i < n; i++) {
        unichar c = [s characterAtIndex:i];
        NSInteger idx = c - 'a';
        
        if (inStack[idx]) continue;
        
        while (stack.count > 0) {
            unichar top = stack.lastObject.unsignedShortValue;
            if (top > c && lastIndex[top - 'a'] > i) {
                inStack[top - 'a'] = NO;
                [stack removeLastObject];
            } else {
                break;
            }
        }
        
        [stack addObject:@(c)];
        inStack[idx] = YES;
    }
    
    NSMutableString *result = [NSMutableString string];
    for (NSNumber *num in stack) {
        [result appendFormat:@"%C", num.unsignedShortValue];
    }
    return [result copy];
}
```

**复杂度**：O(n) 时间，O(1) 空间（栈最多26个字符）。

```swift
func maxProfit(_ prices: [Int]) -> Int {
```

### 5.4 划分字母区间（Partition Labels）— LeetCode 763

```swift
func partitionLabels(_ s: String) -> [Int] {
    let chars = Array(s)
    var lastIndex = [Character: Int]()
    for (i, c) in chars.enumerated() {
        lastIndex[c] = i
    }
    
    var result: [Int] = []
    var start = 0
    var end = 0
    
    for (i, c) in chars.enumerated() {
        end = max(end, lastIndex[c]!)
        if i == end {
            result.append(end - start + 1)
            start = end + 1
        }
    }
    
    return result
}

// 示例
partitionLabels("ababcbacadefegdehijhklij")
// [9, 7, 8]: "ababcbaca" | "defegde" | "hijhklij"
```

**Objective-C 实现**：

```objectivec
- (NSArray<NSNumber *> *)partitionLabels:(NSString *)s {
    NSInteger n = s.length;
    NSInteger lastIndex[26] = {0};
    
    for (NSInteger i = 0; i < n; i++) {
        lastIndex[[s characterAtIndex:i] - 'a'] = i;
    }
    
    NSMutableArray<NSNumber *> *result = [NSMutableArray array];
    NSInteger start = 0, end = 0;
    
    for (NSInteger i = 0; i < n; i++) {
        end = MAX(end, lastIndex[[s characterAtIndex:i] - 'a']);
        if (i == end) {
            [result addObject:@(end - start + 1)];
            start = end + 1;
        }
    }
    
    return [result copy];
}
```

**复杂度**：O(n) 时间，O(1) 空间（26 个字母）。

### 5.5 单调递增的数字（Monotone Increasing Digits）— LeetCode 738

**问题描述**：给定整数 n，返回 ≤ n 的最大整数，使其各位数字单调不减。

**贪心策略**：从右到左扫描，找到第一个 digits[i-1] > digits[i] 的位置，将 digits[i-1] 减 1，后面全部填 9。

```swift
func monotoneIncreasingDigits(_ n: Int) -> Int {
    var digits = Array(String(n)).map { Int(String($0))! }
    let len = digits.count
    var mark = len  // 从 mark 位置开始全部变 9
    
    for i in stride(from: len - 1, through: 1, by: -1) {
        if digits[i - 1] > digits[i] {
            digits[i - 1] -= 1
            mark = i
        }
    }
    
    for i in mark..<len {
        digits[i] = 9
    }
    
    return digits.reduce(0) { $0 * 10 + $1 }
}

// 示例
monotoneIncreasingDigits(10)    // 9
monotoneIncreasingDigits(1234)  // 1234（已单调）
monotoneIncreasingDigits(332)   // 299
monotoneIncreasingDigits(1332)  // 1299
```

**Objective-C 实现**：

```objectivec
- (NSInteger)monotoneIncreasingDigits:(NSInteger)n {
    NSMutableString *str = [[NSString stringWithFormat:@"%ld", (long)n] mutableCopy];
    NSInteger len = str.length;
    NSInteger mark = len;
    
    for (NSInteger i = len - 1; i >= 1; i--) {
        if ([str characterAtIndex:i - 1] > [str characterAtIndex:i]) {
            unichar c = [str characterAtIndex:i - 1] - 1;
            [str replaceCharactersInRange:NSMakeRange(i - 1, 1) withString:[NSString stringWithCharacters:&c length:1]];
            mark = i;
        }
    }
    
    for (NSInteger i = mark; i < len; i++) {
        [str replaceCharactersInRange:NSMakeRange(i, 1) withString:@"9"];
    }
    
    return [str integerValue];
}
```

**复杂度**：O(d) 时间和空间，d 为数字位数（≤ 10）。

## 六、贪心在图论中的应用

### 6.1 Dijkstra 最短路径

Dijkstra 算法本质是贪心：每次从未确定的节点中选距离最小的节点 u，确定 u 的最短路径，然后松弛 u 的邻居。

贪心选择性质：对于非负权图，当前距离最小的未处理节点，其距离已经是最短的（因为通过其他未处理节点绕行只会更长）。

详细实现参见「第12篇 最短路径与最小生成树」。

### 6.2 Kruskal 最小生成树

Kruskal 算法是贪心：将所有边按权重排序，从最小开始，若加入该边不形成环则选取。

满足拟阵结构（图的森林拟阵），因此贪心最优。

### 6.3 Prim 最小生成树

Prim 算法是贪心：从一个节点出发，每次选连接已有树和未加入节点的最小权边。

与 Dijkstra 类似的优先队列贪心策略。

## 七、贪心算法的常见陷阱与反例

### 7.1 贪心失败的经典案例

| 问题 | 贪心策略 | 失败原因 | 正确方法 |
|------|----------|----------|----------|
| 0-1 背包 | 选性价比最高 | 不可分割，可能浪费容量 | DP |
| 最长递增子序列 | 每次选尽量小的 | 贪心选当前小的可能阻断后续 | DP / 贪心+二分 |
| 硬币找零（通用） | 先选大面值 | 非标准面值可能不最优 | DP |
| TSP 旅行商 | 最近邻居 | 可能走入死角 | DP / 近似算法 |
| 图着色 | 贪心染色 | 不保证最少颜色数 | NP-hard 精确算法 |

### 7.2 如何判断能否用贪心

**可以用贪心的信号**：
- 问题要求「最多」「最少」「最大」「最小」
- 排序后问题变简单
- 有明显的「局部最优 = 全局最优」直觉
- 问题具有区间调度 / 活动选择的形式
- 能证明交换论证成立

**不能用贪心的信号**：
- 存在「当前最优但未来亏损」的情况
- 需要考虑组合效应
- 问题有约束耦合（选 A 影响 B 的可选范围）
- 反例容易构造

### 7.3 贪心 vs DP 的抉择

```
问题 → 有贪心选择性质？
  ├─ 是 → 贪心（O(n log n) 级别）
  └─ 否 → 有最优子结构 + 重叠子问题？
        ├─ 是 → DP
        └─ 否 → 搜索/回溯
```

### 8.4 实战案例：iOS 瀑布流布局中的贪心

```swift
/// 瀑布流布局：贪心选最短列放置下一个 item
class WaterfallLayout: UICollectionViewLayout {
    var numberOfColumns = 2
    var cellPadding: CGFloat = 6
    private var cache: [UICollectionViewLayoutAttributes] = []
    private var contentHeight: CGFloat = 0
    
    override func prepare() {
        guard let collectionView = collectionView else { return }
        cache.removeAll()
        
        let columnWidth = (collectionView.bounds.width - cellPadding * CGFloat(numberOfColumns + 1)) / CGFloat(numberOfColumns)
        var columnHeights = Array(repeating: cellPadding, count: numberOfColumns)
        
        for item in 0..<collectionView.numberOfItems(inSection: 0) {
            // 贪心：选当前最短的列
            let shortestColumn = columnHeights.enumerated().min(by: { $0.element < $1.element })!.offset
            
            let xOffset = cellPadding + CGFloat(shortestColumn) * (columnWidth + cellPadding)
            let yOffset = columnHeights[shortestColumn]
            
            // 假设有获取 item 高度的方法
            let itemHeight = heightForItem(at: item, width: columnWidth)
            
            let frame = CGRect(x: xOffset, y: yOffset, width: columnWidth, height: itemHeight)
            let attributes = UICollectionViewLayoutAttributes(forCellWith: IndexPath(item: item, section: 0))
            attributes.frame = frame
            cache.append(attributes)
            
            columnHeights[shortestColumn] = frame.maxY + cellPadding
        }
        
        contentHeight = columnHeights.max() ?? 0
    }
    
    private func heightForItem(at index: Int, width: CGFloat) -> CGFloat {
        // 实际项目中根据内容计算高度
        return CGFloat(arc4random_uniform(100) + 100)
    }
    
    override var collectionViewContentSize: CGSize {
        return CGSize(width: collectionView?.bounds.width ?? 0, height: contentHeight)
    }
    
    override func layoutAttributesForElements(in rect: CGRect) -> [UICollectionViewLayoutAttributes]? {
        return cache.filter { $0.frame.intersects(rect) }
    }
    
    override func layoutAttributesForItem(at indexPath: IndexPath) -> UICollectionViewLayoutAttributes? {
        return cache[indexPath.item]
    }
}
```

## 十、面试高频题与答案要点

### 面试题 1：如何判断一个问题能不能用贪心？

**答案要点**：
- 看问题是否具有「贪心选择性质」——每步局部最优能导致全局最优
- 看问题是否具有「最优子结构」——子问题最优解组合成原问题最优解
- 尝试构造反例：若能找到贪心策略不最优的输入，则不能用
- 常见可贪心模式：区间调度（按结束时间）、分数背包（按性价比）、Huffman（最小优先合并）
- 需要严格证明时用交换论证法

### 面试题 2：活动选择为什么按结束时间排序而不是开始时间？

**答案要点**：
- 按结束时间排序保证每次选的活动「尽早结束」，给后续活动留出最多空间
- 按开始时间排序是错误的，反例：活动 A=[0,100], B=[1,2], C=[3,4]，按开始时间选 A 只得 1 个，但选 B+C 得 2 个
- 也不能按持续时间排序，反例：A=[0,5], B=[4,6], C=[5,10]，按持续时间选 B(2) 后 A 和 C 都冲突

### 面试题 3：0-1 背包为什么不能用贪心？

**答案要点**：
- 0-1 背包物品不可分割，贪心按性价比可能浪费剩余容量
- 反例：容量=10，物品 A(w=6,v=30,ratio=5)、B(w=5,v=24,ratio=4.8)、C(w=5,v=24,ratio=4.8)
- 贪心选 A(ratio最高) 后只剩容量 4，无法再放 B 或 C，总价值 30
- 最优选 B+C，总价值 48
- 分数背包可以贪心因为可以用「零头」填满背包

### 面试题 4：贪心算法的时间复杂度通常是多少？为什么比 DP 快？

**答案要点**：
- 贪心通常 O(n log n)（排序主导）或 O(n)（单次遍历），因为不枚举所有子问题
- DP 需要填表（O(n²) 或 O(n·W)），考虑所有可能的子问题组合
- 贪心之所以快，是因为利用了「贪心选择性质」跳过了大量不必要的计算
- 代价是适用范围更窄，且需要证明正确性

### 面试题 5：给出三个贪心失败的例子及其正确解法

**答案要点**：
1. **0-1 背包**：贪心按性价比选 → 可能浪费容量 → 正确用 DP
2. **硬币找零（非标准面值）**：贪心选最大面值 → 非标准面值不保证最优 → 正确用 DP
3. **最长递增子序列**：贪心选当前最小 → 可能堵住后续更长序列 → 正确用 DP 或贪心+二分（Patience Sorting）

### 面试题 6：Huffman 编码的贪心正确性如何证明？

**答案要点**：
- **贪心选择性质**：频率最低的两个字符在最优前缀码中一定是兄弟且在最深层。证明：假设最深层兄弟 a,b 不是最低频的两个 x,y，交换 a 与 x（或 b 与 y），编码总长只会减少或不变（因为低频字符放更深、高频字符放更浅更划算）
- **最优子结构**：合并 x,y 为新字符 z(freq=freq_x+freq_y) 后，新问题（n-1 个字符）的最优 Huffman 树对应原问题的最优树

### 面试题 7：实现「最少硬币找零」的贪心策略对美制硬币为什么有效？

**答案要点**：
- 美制硬币 {1, 5, 10, 25} 满足：每个大面值都是小面值的整数倍或接近整数倍
- 具体来说，用若干小面值拼出一个大面值的数量一定 > 1（如 5 个 1¢ > 1 个 5¢）
- 数学上可以证明：对任何金额，贪心选大面值得到的硬币数 = DP 求得的最优解
- 这是因为美制硬币系统是「规范系统」（canonical system）

### 面试题 8：跳跃游戏 II 为什么是贪心而不是 BFS？

**答案要点**：
- 本质上可以看作隐式图的 BFS，但用贪心优化到 O(n)
- BFS 的「层」对应一次跳跃能到达的范围 [curStart, curEnd]
- 贪心：不真正枚举每层的所有节点往外扩展，而是记录该层能到达的最远距离 farthest
- 当遍历到 curEnd 时意味着必须跳一步，此时 farthest 就是下一层的 curEnd
- 时间从 BFS 的 O(n²) 最坏优化到 O(n)

## 十一、易错点与最佳实践

### 11.1 常见错误

| 错误 | 说明 | 正确做法 |
|------|------|----------|
| 未证明贪心正确性就使用 | 贪心直觉正确不代表数学上正确 | 先尝试构造反例，再用交换论证证明 |
| 区间问题排序依据搞错 | 最多不重叠活动按结束排序，不是开始 | 明确目标：最多保留 → 按end；最多合并 → 按start |
| 忽略边界条件 | 数组为空、只有一个元素、所有元素相同 | 先处理特殊情况 |
| 贪心策略考虑不全面 | 分发糖果只做一遍遍历 | 需要两遍（左到右+右到左）保证双向约束 |
| 整数溢出 | 区间端点相乘或相加可能溢出 | 用 Int64/long long 或避免不必要的乘法 |
| 排序比较器不满足传递性 | 自定义排序可能导致未定义行为 | 确保比较器满足严格弱序 |

### 11.2 最佳实践

1. **先想贪心策略，再验证**：直觉想到贪心后，立即构造 2-3 个反例测试，通过再证明
2. **排序是核心**：大部分贪心题本质是「排序 + 线性扫描」
3. **贪心 + 堆 = 黄金组合**：当贪心需要动态取最优时，用优先队列
4. **写完验证**：用小规模数据手动模拟，确认贪心策略的每一步都合理
5. **与 DP 对照**：如果贪心过不了，考虑是否该用 DP；有时先写 DP 再观察规律简化为贪心

### 11.3 贪心算法设计 Checklist

```
□ 明确优化目标（最大化/最小化什么）
□ 确定贪心策略（排序依据/选择规则）
□ 构造反例验证（至少 3 个 case）
□ 证明正确性（交换论证/归纳/反证）
□ 处理边界条件（空输入/单元素/全相同）
□ 分析时间空间复杂度
□ 代码实现并测试
```

## 十三、延伸阅读与进阶主题

### 13.1 近似贪心

当贪心不能保证最优时，可作为近似算法使用：
- **集合覆盖问题**：贪心选覆盖最多未覆盖元素的集合，近似比 O(ln n)
- **TSP 最近邻**：贪心近似比 O(log n)
- **Vertex Cover**：贪心 2-近似

### 13.2 在线贪心 / 秘书问题

在线环境下（不知道未来输入），贪心策略需要调整：
- **秘书问题**：前 n/e 个候选人只观察不选，之后选第一个比之前都好的
- **竞争比分析**：衡量在线贪心与离线最优的差距

### 13.3 与其他算法的结合

- **贪心 + 二分**：最长递增子序列（Patience Sorting）
- **贪心 + 并查集**：Kruskal 算法
- **贪心 + 优先队列**：Dijkstra、Huffman、会议室
- **贪心 + DP**：某些问题先贪心确定局部结构，再 DP 求全局最优

## 十四、完整实战案例：任务调度优化器

综合运用贪心算法解决一个接近真实场景的问题：给定一批任务，每个任务有截止时间和收益，每个时间单位只能完成一个任务，求最大收益。

**问题**：Job Sequencing Problem

**Swift 完整实现**：

```swift
struct Job {
    let id: String
    let deadline: Int
    let profit: Int
}

/// 带截止时间的任务调度（贪心 + 并查集优化）
class JobScheduler {
    
    // 方法一：朴素贪心 O(n²)
    func scheduleJobsNaive(_ jobs: [Job]) -> (maxProfit: Int, scheduled: [String]) {
        // 按收益降序排序
        let sorted = jobs.sorted { $0.profit > $1.profit }
        let maxDeadline = sorted.map { $0.deadline }.max() ?? 0
        
        // slot[i] = true 表示时间槽 i 已被占用
        var slots = Array(repeating: false, count: maxDeadline + 1)
        var scheduledJobs: [String] = []
        var totalProfit = 0
        
        for job in sorted {
            // 贪心：尽量放在最晚的可用时间槽
            for t in stride(from: job.deadline, through: 1, by: -1) {
                if !slots[t] {
                    slots[t] = true
                    scheduledJobs.append(job.id)
                    totalProfit += job.profit
                    break
                }
            }
        }
        
        return (totalProfit, scheduledJobs)
    }
    
    // 方法二：并查集优化 O(n·α(n)) ≈ O(n)
    private var parent: [Int] = []
    
    private func find(_ x: Int) -> Int {
        if parent[x] != x {
            parent[x] = find(parent[x])
        }
        return parent[x]
    }
    
    func scheduleJobsOptimized(_ jobs: [Job]) -> (maxProfit: Int, scheduled: [String]) {
        let sorted = jobs.sorted { $0.profit > $1.profit }
        let maxDeadline = sorted.map { $0.deadline }.max() ?? 0
        
        // parent[i] 表示时间槽 i 可用的最近空位
        parent = Array(0...maxDeadline)
        
        var scheduledJobs: [String] = []
        var totalProfit = 0
        
        for job in sorted {
            let availableSlot = find(min(job.deadline, maxDeadline))
            if availableSlot > 0 {
                // 占用该时间槽
                parent[availableSlot] = availableSlot - 1
                scheduledJobs.append(job.id)
                totalProfit += job.profit
            }
        }
        
        return (totalProfit, scheduledJobs)
    }
}

// 示例
let jobs = [
    Job(id: "a", deadline: 2, profit: 100),
    Job(id: "b", deadline: 1, profit: 19),
    Job(id: "c", deadline: 2, profit: 27),
    Job(id: "d", deadline: 1, profit: 25),
    Job(id: "e", deadline: 3, profit: 15)

// 总利润 = 100 + 27 + 15 = 142
```

**Objective-C 完整实现**：

```objectivec
@interface Job : NSObject
@property (nonatomic, copy) NSString *jobId;
@property (nonatomic, assign) NSInteger deadline;
@property (nonatomic, assign) NSInteger profit;
+ (instancetype)jobWithId:(NSString *)jid deadline:(NSInteger)d profit:(NSInteger)p;
@end

@implementation Job
+ (instancetype)jobWithId:(NSString *)jid deadline:(NSInteger)d profit:(NSInteger)p {
    Job *job = [[Job alloc] init];
    job.jobId = jid;
    job.deadline = d;
    job.profit = p;
    return job;
}
@end

@interface JobScheduler : NSObject {
    NSInteger *_parent;
    NSInteger _parentSize;
}
- (NSDictionary *)scheduleJobs:(NSArray<Job *> *)jobs;
@end

@implementation JobScheduler

- (NSInteger)find:(NSInteger)x {
    if (_parent[x] != x) {
        _parent[x] = [self find:_parent[x]];
    }
    return _parent[x];
}

- (NSDictionary *)scheduleJobs:(NSArray<Job *> *)jobs {
    NSArray *sorted = [jobs sortedArrayUsingComparator:^NSComparisonResult(Job *a, Job *b) {
        return b.profit > a.profit ? NSOrderedDescending : (b.profit < a.profit ? NSOrderedAscending : NSOrderedSame);
    }];
    
    NSInteger maxDeadline = 0;
    for (Job *job in sorted) {
        maxDeadline = MAX(maxDeadline, job.deadline);
    }
    
    _parentSize = maxDeadline + 1;
    _parent = (NSInteger *)malloc(sizeof(NSInteger) * _parentSize);
    for (NSInteger i = 0; i < _parentSize; i++) {
        _parent[i] = i;
    }
    
    NSMutableArray<NSString *> *scheduled = [NSMutableArray array];
    NSInteger totalProfit = 0;
    
    for (Job *job in sorted) {
        NSInteger slot = [self find:MIN(job.deadline, maxDeadline)];
        if (slot > 0) {
            _parent[slot] = slot - 1;
            [scheduled addObject:job.jobId];
            totalProfit += job.profit;
        }
    }
    
    free(_parent);
    _parent = NULL;
    
    return @{@"profit": @(totalProfit), @"jobs": [scheduled copy]};
}

@end
```

**复杂度分析**：
- 朴素版：O(n² ) 时间（n 个任务 × 最多 n 个时间槽查找），O(n) 空间
- 并查集优化版：O(n · α(n)) ≈ O(n) 时间（α 为反 Ackermann 函数，几乎常数），O(n) 空间

## 十五、总结

贪心算法的核心哲学是「化繁为简」——在满足特定数学性质时，将指数级搜索空间压缩为线性或对数级的高效求解过程。

**记忆要点**：
1. 两个前提：贪心选择性质 + 最优子结构
2. 三种证明：交换论证、归纳法、反证法
3. 核心模式：排序 + 线性扫描（或 + 优先队列）
4. 常见陷阱：不证明就用、排序依据错误、忽略 0-1 约束
5. 与 DP 的关系：贪心是 DP 的特化——当 DP 的选择总是固定的那一个时，就退化为贪心

贪心不是万能的，但在它适用的场景下，效率无出其右。面试中遇到优化问题，优先思考能否贪心；若不能，再退回 DP 或搜索。
