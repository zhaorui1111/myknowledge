# 并查集（Union-Find / Disjoint Set Union）

> **并查集**（Union-Find，又称 Disjoint Set Union，DSU）是一种用于高效处理「动态连通性」问题的数据结构。它支持两种核心操作：合并（Union）两个集合、查询（Find）某个元素属于哪个集合。通过路径压缩（Path Compression）与按秩合并（Union by Rank）两大优化，单次操作均摊时间复杂度可降至接近 O(1) 的 O(α(n))，其中 α 是反阿克曼函数（Inverse Ackermann Function）——增长极其缓慢，对所有实际可能的输入规模（n < 2^65536）其值不超过 4，因此可视为常数。

## 1. 概念引入与直觉

### 1.1 现实世界类比

想象一个社交网络：最初每个人是独立的个体。当两个人成为朋友时，他们的「朋友圈」合并为一个。我们需要回答的问题是："A 和 B 是否在同一个朋友圈中？" 并查集就是为这类问题而生的数据结构。

再举一个例子：在一片地图上有若干城市，起初城市之间没有道路。我们逐步修建道路连接城市，任何时刻都可能被问到："城市 X 和城市 Y 之间是否连通（可以通过若干道路到达）？" 每次修路相当于 Union 操作，查询连通性相当于 Find 操作后比较根节点。

### 1.2 问题抽象

给定 n 个元素（编号 0 到 n-1），初始时每个元素各自属于一个独立集合。支持两种操作：

- **Union(x, y)**：将元素 x 所在集合与元素 y 所在集合合并为一个集合
- **Find(x)**：返回元素 x 所在集合的「代表元素」（Representative），通常是集合树结构的根节点

两个元素 x, y 在同一个集合中，当且仅当 Find(x) == Find(y)。

### 1.3 为什么不用其他方案

| 方案 | Union 复杂度 | Find 复杂度 | 缺点 |
|------|------------|-----------|------|
| 数组标记（每个元素记录所属集合 ID） | O(n)（需遍历修改一整个集合的标记） | O(1) | Union 太慢 |
| 链表（每个集合一条链表） | O(1) 尾接（但需维护尾指针） | O(n) 最坏 | Find 太慢 |
| 邻接表 + BFS/DFS | O(1) 加边 | O(V+E) | 查询太慢 |
| **并查集（优化版）** | **≈ O(1)** | **≈ O(1)** | 几乎无缺点 |

并查集通过森林结构 + 两大优化，在 Union 和 Find 之间取得了最佳平衡。

## 2. 基本原理与数据结构设计

### 2.1 森林表示法

并查集用一片**森林**（Forest）来表示所有不相交集合。每个集合是一棵树，树的根节点是该集合的代表元素。每个节点存储一个 `parent` 指针指向其父节点，根节点的 parent 指向自身。

```
初始状态（5个元素，各自一棵树）：
[0] [1] [2] [3] [4]

执行 Union(0, 1) 后：
  0        [2] [3] [4]
  |
  1

执行 Union(2, 3) 后：
  0    2       [4]
  |    |
  1    3

执行 Union(0, 2) 后：
    0           [4]
   / \
  1   2
      |
      3
```

### 2.2 核心数据

只需要一个数组 `parent[0..n-1]`：
- `parent[i] = i` 表示 i 是根节点（集合代表元素）
- `parent[i] = j (j ≠ i)` 表示 i 的父节点是 j

### 2.3 操作定义

**MakeSet(x)**：初始化元素 x，令 parent[x] = x

**Find(x)**：沿 parent 指针向上走直到找到根节点（parent[root] == root），返回 root

**Union(x, y)**：
1. 找到 x 的根 rootX = Find(x)
2. 找到 y 的根 rootY = Find(y)
3. 如果 rootX ≠ rootY，令 parent[rootX] = rootY（或反过来）

## 3. 朴素实现

### 3.1 Swift 实现

```swift
/// 朴素并查集 —— 无优化版本
/// 用于理解基本原理，实际使用请用优化版本
class NaiveUnionFind {
    private var parent: [Int]
    private(set) var count: Int  // 连通分量个数
    
    /// 初始化 n 个元素，每个元素自成一个集合
    /// - Parameter n: 元素个数
    init(_ n: Int) {
        parent = Array(0..<n)  // parent[i] = i，每个节点是自己的根
        count = n
    }
    
    /// 查找元素 x 所在集合的根节点（代表元素）
    /// 最坏 O(n) —— 退化为链表时
    /// - Parameter x: 要查找的元素
    /// - Returns: 该集合的根节点
    func find(_ x: Int) -> Int {
        var node = x
        while parent[node] != node {
            node = parent[node]
        }
        return node
    }
    
    /// 合并元素 x 和 y 所在的集合
    /// - Returns: 是否真正执行了合并（已在同一集合返回 false）
    @discardableResult
    func union(_ x: Int, _ y: Int) -> Bool {
        let rootX = find(x)
        let rootY = find(y)
        if rootX == rootY { return false }
        parent[rootX] = rootY  // 简单地将 x 的根挂到 y 的根下
        count -= 1
        return true
    }
    
    /// 判断 x 和 y 是否在同一集合中
    func connected(_ x: Int, _ y: Int) -> Bool {
        return find(x) == find(y)
    }
}
```

### 3.3 朴素版本的问题

考虑这样的 Union 序列：Union(0,1), Union(1,2), Union(2,3), ..., Union(n-2,n-1)

每次都将左边挂到右边根下，形成一条链：

```
0 → 1 → 2 → 3 → ... → n-1
```

此时 Find(0) 需要遍历整条链，时间复杂度 O(n)。如果有 m 次操作，最坏总时间 O(mn)，对大规模数据不可接受。

## 4. 优化一：路径压缩（Path Compression）

### 4.1 核心思想

Find 操作在向上寻找根节点的过程中，顺便将路径上所有节点直接挂到根节点下。这样下次再查找这些节点时就能直接到达根，大幅缩短了树的高度。

### 4.2 图解

```
Find(0) 前（链状树）：      Find(0) 后（路径压缩）：
    4                           4
    |                         / | \ \
    3                        0  1  2  3
    |
    2
    |
    1
    |
    0
```

### 4.3 两种实现方式

**方式一：递归路径压缩（完全压缩）**

在递归回溯时将每个节点直接连到根。代码优雅但有栈溢出风险（极深的树）。

```swift
/// 递归路径压缩
/// Find 时将路径上所有节点直接指向根
func find(_ x: Int) -> Int {
    if parent[x] != x {
        parent[x] = find(parent[x])  // 递归找根，回溯时压缩
    }
    return parent[x]
}
```

**方式二：迭代路径压缩（两遍法）**

第一遍找到根，第二遍将路径上所有节点的 parent 修改为根。无栈溢出风险。

```swift
/// 迭代路径压缩（两遍法）
func find(_ x: Int) -> Int {
    // 第一遍：找到根
    var root = x
    while parent[root] != root {
        root = parent[root]
    }
    // 第二遍：压缩路径
    var node = x
    while node != root {
        let next = parent[node]
        parent[node] = root
        node = next
    }
    return root
}
```

**方式三：路径分裂（Path Splitting）/ 路径对折（Path Halving）**

- 路径分裂：每个节点指向其祖父节点（parent[x] = parent[parent[x]]），单遍完成
- 路径对折：每隔一个节点做一次压缩

```swift
/// 路径对折（Path Halving）—— 实践中效果接近完全压缩
func find(_ x: Int) -> Int {
    var node = x
    while parent[node] != node {
        parent[node] = parent[parent[node]]  // 指向祖父
        node = parent[node]
    }
    return node
}
```

### 4.4 路径压缩效果分析

仅使用路径压缩（不用按秩合并），m 次操作 + n 个元素的总时间为 O(m · log n)（均摊 O(log n) 每次操作）。这已经比朴素版本好得多，但还不是最优的。

## 5. 优化二：按秩合并（Union by Rank）与按大小合并（Union by Size）

### 5.1 核心思想

在 Union 时，不再随意地将一棵树挂到另一棵下面，而是将**较矮**（或**较小**）的树挂到较高（或较大）的树下面，以避免树变得过高。

### 5.2 按秩合并（Union by Rank）

**Rank**（秩）≈ 树高的上界。初始每个节点 rank = 0。合并规则：
- 若 rank[rootX] < rank[rootY]：将 rootX 挂到 rootY 下（较矮挂到较高下）
- 若 rank[rootX] > rank[rootY]：将 rootY 挂到 rootX 下
- 若 rank[rootX] == rank[rootY]：任选一个挂，另一个的 rank + 1

注意：路径压缩后 rank 不再精确反映树高，但仍然作为启发式有效。

### 5.3 按大小合并（Union by Size）

用 `size[root]` 记录以 root 为根的集合元素个数。合并时将较小集合挂到较大集合下。

优势：size 的语义始终准确（路径压缩不影响集合大小），而且提供了额外信息（随时知道每个集合有多少元素）。

### 5.4 仅用按秩/按大小合并的效果

不用路径压缩，仅用按秩合并，树的高度上界为 O(log n)，因此 Find 复杂度 O(log n)。

证明思路：一棵 rank 为 k 的树至少有 2^k 个节点（归纳法），因此 n 个节点的树 rank ≤ log₂n。

### 5.5 两者结合：O(α(n))

路径压缩 + 按秩合并/按大小合并相结合，单次操作均摊 O(α(n))。这是 Tarjan 在 1975 年证明的经典结果，而 Fredman & Saks 在 1989 年证明了这是最优的（信息论下界）。

## 6. 完整优化实现

### 6.1 Swift 完整实现（按秩合并 + 路径压缩）

```swift
/// 并查集 —— 路径压缩 + 按秩合并
/// 均摊单次操作 O(α(n)) ≈ O(1)
struct UnionFind {
    private var parent: [Int]
    private var rank: [Int]
    private(set) var count: Int  // 当前连通分量数
    
    /// 初始化包含 n 个元素的并查集
    /// - Parameter n: 元素数量（编号 0..<n）
    /// - Complexity: O(n)
    init(_ n: Int) {
        parent = Array(0..<n)
        rank = Array(repeating: 0, count: n)
        count = n
    }
    
    /// 查找元素 x 所在集合的根节点（含路径压缩）
    /// - Parameter x: 要查找的元素（0..<n）
    /// - Returns: 集合代表元素（根节点）
    /// - Complexity: 均摊 O(α(n))
    mutating func find(_ x: Int) -> Int {
        if parent[x] != x {
            parent[x] = find(parent[x])  // 路径压缩：递归回溯时直连根
        }
        return parent[x]
    }
    
    /// 合并元素 x 和 y 所在的集合（按秩合并）
    /// - Parameters:
    ///   - x: 元素 x
    ///   - y: 元素 y
    /// - Returns: 是否真正执行了合并（同一集合返回 false）
    /// - Complexity: 均摊 O(α(n))
    @discardableResult
    mutating func union(_ x: Int, _ y: Int) -> Bool {
        let rootX = find(x)
        let rootY = find(y)
        if rootX == rootY { return false }
        
        // 按秩合并：较矮的树挂到较高的树下
        if rank[rootX] < rank[rootY] {
            parent[rootX] = rootY
        } else if rank[rootX] > rank[rootY] {
            parent[rootY] = rootX
        } else {
            parent[rootY] = rootX
            rank[rootX] += 1  // 等高时新树高度 +1
        }
        count -= 1
        return true
    }
    
    /// 判断 x 和 y 是否属于同一集合
    /// - Complexity: 均摊 O(α(n))
    mutating func connected(_ x: Int, _ y: Int) -> Bool {
        return find(x) == find(y)
    }
    
    /// 获取元素 x 所在集合的大小
    /// 注意：按秩合并版本不直接维护 size，如需此功能用 UnionFindBySize
    mutating func setSize(of x: Int) -> Int {
        let root = find(x)
        var size = 0
        for i in 0..<parent.count {
            var node = i
            // 无法直接获取，需遍历（O(n)），建议用 size 版本
            if find(node) == root { size += 1 }
        }
        return size
    }
}
```

### 6.2 Swift 实现（按大小合并 + 路径压缩）

```swift
/// 并查集 —— 路径压缩 + 按大小合并
/// 额外维护集合大小，适合需要查询集合元素数的场景
struct UnionFindBySize {
    private var parent: [Int]
    private var size: [Int]     // size[root] 表示以 root 为根的集合大小
    private(set) var count: Int // 当前连通分量数
    
    init(_ n: Int) {
        parent = Array(0..<n)
        size = Array(repeating: 1, count: n)
        count = n
    }
    
    mutating func find(_ x: Int) -> Int {
        if parent[x] != x {
            parent[x] = find(parent[x])
        }
        return parent[x]
    }
    
    @discardableResult
    mutating func union(_ x: Int, _ y: Int) -> Bool {
        let rootX = find(x)
        let rootY = find(y)
        if rootX == rootY { return false }
        
        // 按大小合并：小集合挂到大集合
        if size[rootX] < size[rootY] {
            parent[rootX] = rootY
            size[rootY] += size[rootX]
        } else {
            parent[rootY] = rootX
            size[rootX] += size[rootY]
        }
        count -= 1
        return true
    }
    
    mutating func connected(_ x: Int, _ y: Int) -> Bool {
        return find(x) == find(y)
    }
    
    /// O(1) 获取元素 x 所在集合的大小
    mutating func setSize(of x: Int) -> Int {
        let root = find(x)
        return size[root]
    }
    
    /// 获取最大集合的大小
    func maxSetSize() -> Int {
        return size.max() ?? 0
    }
}
```

### 6.3 Objective-C 完整实现（按秩合并 + 路径压缩）

```objectivec
/// 并查集 —— 路径压缩 + 按秩合并
/// 均摊单次操作 O(α(n))
@interface SAKUnionFind : NSObject

@property (nonatomic, readonly) NSInteger count; // 当前连通分量数

- (instancetype)initWithSize:(NSInteger)n;
- (NSInteger)find:(NSInteger)x;
- (BOOL)unionX:(NSInteger)x y:(NSInteger)y;
- (BOOL)connected:(NSInteger)x y:(NSInteger)y;

@end

@implementation SAKUnionFind {
    NSInteger *_parent;
    NSInteger *_rank;
    NSInteger _n;
}

- (instancetype)initWithSize:(NSInteger)n {
    self = [super init];
    if (self) {
        _n = n;
        _count = n;
        _parent = (NSInteger *)malloc(sizeof(NSInteger) * n);
        _rank = (NSInteger *)calloc(n, sizeof(NSInteger)); // 初始 rank 全 0
        for (NSInteger i = 0; i < n; i++) {
            _parent[i] = i;
        }
    }
    return self;
}

- (void)dealloc {
    free(_parent);
    free(_rank);
}

- (NSInteger)find:(NSInteger)x {
    // 迭代路径压缩（避免递归栈溢出）
    NSInteger root = x;
    while (_parent[root] != root) {
        root = _parent[root];
    }
    // 压缩路径
    NSInteger node = x;
    while (node != root) {
        NSInteger next = _parent[node];
        _parent[node] = root;
        node = next;
    }
    return root;
}

- (BOOL)unionX:(NSInteger)x y:(NSInteger)y {
    NSInteger rootX = [self find:x];
    NSInteger rootY = [self find:y];
    if (rootX == rootY) return NO;
    
    // 按秩合并
    if (_rank[rootX] < _rank[rootY]) {
        _parent[rootX] = rootY;
    } else if (_rank[rootX] > _rank[rootY]) {
        _parent[rootY] = rootX;
    } else {
        _parent[rootY] = rootX;
        _rank[rootX]++;
    }
    _count--;
    return YES;
}

- (BOOL)connected:(NSInteger)x y:(NSInteger)y {
    return [self find:x] == [self find:y];
}

@end
```

```objectivec
        _parent = (NSInteger *)malloc(sizeof(NSInteger) * n);
        _size = (NSInteger *)malloc(sizeof(NSInteger) * n);
        for (NSInteger i = 0; i < n; i++) {
            _parent[i] = i;
            _size[i] = 1;
        }
    }
    return self;
}

- (void)dealloc {
    free(_parent);
    free(_size);
}

- (NSInteger)find:(NSInteger)x {
    NSInteger root = x;
    while (_parent[root] != root) {
        root = _parent[root];
    }
    NSInteger node = x;
    while (node != root) {
        NSInteger next = _parent[node];
        _parent[node] = root;
        node = next;
    }
    return root;
}

- (BOOL)unionX:(NSInteger)x y:(NSInteger)y {
    NSInteger rootX = [self find:x];
    NSInteger rootY = [self find:y];
    if (rootX == rootY) return NO;
    
    // 小挂大
    if (_size[rootX] < _size[rootY]) {
        _parent[rootX] = rootY;
        _size[rootY] += _size[rootX];
    } else {
        _parent[rootY] = rootX;
        _size[rootX] += _size[rootY];
    }
    _count--;
    return YES;
}

- (BOOL)connected:(NSInteger)x y:(NSInteger)y {
    return [self find:x] == [self find:y];
}

- (NSInteger)setSizeOf:(NSInteger)x {
    NSInteger root = [self find:x];
    return _size[root];
}

@end
```

## 7. 复杂度分析与数学证明

### 7.1 时间复杂度总结

| 优化策略 | 单次 Find | 单次 Union | m 次操作总计 |
|---------|----------|-----------|------------|
| 无优化 | O(n) 最坏 | O(n) 最坏 | O(mn) |
| 仅路径压缩 | O(log n) 均摊 | O(log n) 均摊 | O(m · log n) |
| 仅按秩合并 | O(log n) 最坏 | O(log n) 最坏 | O(m · log n) |
| 路径压缩 + 按秩合并 | **O(α(n)) 均摊** | **O(α(n)) 均摊** | **O(m · α(n))** |

### 7.2 反阿克曼函数 α(n)

阿克曼函数 A(m, n) 是增长极快的函数（超指数增长）。其反函数 α(n) 增长极其缓慢：

| n | α(n) |
|---|------|
| 1-2 | 0 |
| 3 | 1 |
| 4-7 | 2 |
| 8-2047 | 3 |
| 2048 - 2^(2^(2^65536)) | 4 |

对于宇宙中所有原子数量级的 n（约 10^80），α(n) ≤ 4。因此在任何实际应用中，O(α(n)) 等价于 O(1)。

### 7.3 空间复杂度

- parent 数组：O(n)
- rank/size 数组：O(n)
- 总空间：**O(n)**

### 7.4 按秩合并的正确性证明

**定理**：使用按秩合并，任何 rank 为 k 的树至少有 2^k 个节点。

**证明（归纳法）**：
- 基础：rank = 0 的树有 1 = 2^0 个节点。✓
- 归纳步：rank 增加为 k+1，仅当两棵 rank = k 的树合并。由归纳假设，两棵树各有 ≥ 2^k 个节点，合并后有 ≥ 2^(k+1) 个节点。✓

**推论**：n 个节点的树，rank ≤ log₂n，因此 Find 最坏 O(log n)。

### 7.5 信息论下界

Fredman & Saks (1989) 证明：对于 n 个元素、m 次 Find/Union 操作的序列，任何数据结构至少需要 Ω(m · α(n)) 的时间。因此并查集（路径压缩 + 按秩合并）已经是渐近最优的。

## 8. 变体一：带权并查集（Weighted Union-Find）

### 8.1 概念

在普通并查集的基础上，每条边带有一个「权值」（weight），表示子节点到父节点的某种关系量。Find 时沿路径累加权值，可以获得任意节点到根的「距离」。从而回答"两个节点之间的关系量是多少？"

典型应用：
- 种类并查集（判断两个节点是否同类/异类）
- 除法关系（a/b = w，求 a/c）
- 区间偏移量

### 8.2 权值更新规则

设 `weight[x]` 表示 x 到 parent[x] 的权值。路径压缩时需要累加路径上所有权值：

```
Find(x):
  if parent[x] != x:
    root = Find(parent[x])
    weight[x] = weight[x] ⊕ weight[parent[x]]  // ⊕ 是权值的"结合"运算
    parent[x] = root
  return parent[x]
```

Union(x, y, w)（表示 x→y 的权值为 w）：
```
rootX = Find(x), rootY = Find(y)
if rootX == rootY: return (验证一致性)
parent[rootX] = rootY
weight[rootX] = weight[x]⁻¹ ⊕ w ⊕ weight[y]
  // 即：rootX→rootY = rootX→x + x→y + y→rootY
  //   = -weight[x] + w + weight[y]（对加法群）
```

### 8.3 Swift 实现（加法群带权并查集）

```swift
/// 带权并查集 —— 维护节点到根的权值（距离/偏移量）
/// 权值运算为加法：weight[x] 表示 x 相对于 root 的偏移
struct WeightedUnionFind {
    private var parent: [Int]
    private var rank: [Int]
    private var weight: [Double]  // weight[x]: x 到 parent[x] 的权值
    
    init(_ n: Int) {
        parent = Array(0..<n)
        rank = Array(repeating: 0, count: n)
        weight = Array(repeating: 0.0, count: n)  // 到自身的距离为 0
    }
    
    /// 查找根节点，同时进行路径压缩并累加权值
    /// 压缩后 weight[x] 表示 x 到 root 的直接权值
    mutating func find(_ x: Int) -> Int {
        if parent[x] != x {
            let root = find(parent[x])
            weight[x] += weight[parent[x]]  // 路径上的权值累加
            parent[x] = root
        }
        return parent[x]
    }
    
    /// 获取节点 x 到其根的权值
    mutating func weightToRoot(_ x: Int) -> Double {
        _ = find(x)  // 确保路径压缩完成
        return weight[x]
    }
    
    /// 合并：表示 x / y = w（x 到 y 的权值为 w）
    /// 对于除法关系：weight 表示乘法关系，权值运算用乘法
    /// 此处以加法为例：表示 value[x] - value[y] = w
    @discardableResult
    mutating func union(_ x: Int, _ y: Int, w: Double) -> Bool {
        let rootX = find(x)
        let rootY = find(y)
        if rootX == rootY {
            // 已在同一集合，可检查一致性
            // |weight[x] - weight[y] - w| < eps 则一致
            return false
        }
        
        if rank[rootX] < rank[rootY] {
            parent[rootX] = rootY
            // rootX → rootY 的权值 = x到rootX的逆 + x到y + y到rootY
            weight[rootX] = -weight[x] + w + weight[y]
        } else if rank[rootX] > rank[rootY] {
            parent[rootY] = rootX
            weight[rootY] = -weight[y] - w + weight[x]
        } else {
            parent[rootY] = rootX
            weight[rootY] = -weight[y] - w + weight[x]
            rank[rootX] += 1
        }
        return true
    }
    
    /// 查询 x 到 y 的权值（x 和 y 必须在同一集合）
    /// 返回 nil 表示不在同一集合
    mutating func query(_ x: Int, _ y: Int) -> Double? {
        guard find(x) == find(y) else { return nil }
        return weight[x] - weight[y]
    }
}
```

### 8.4 Objective-C 实现（带权并查集）

```objectivec
/// 带权并查集 —— 维护乘法关系（除法求值场景）

- (instancetype)initWithSize:(NSInteger)n;

- (BOOL)unionX:(NSInteger)x y:(NSInteger)y weight:(double)w;
- (NSNumber *)queryX:(NSInteger)x y:(NSInteger)y;  // nil 表示不连通

@end

@implementation SAKWeightedUnionFind {
    NSInteger *_parent;
    NSInteger *_rank;
    double *_weight;  // _weight[x]: x 到 parent[x] 的乘法权值
    NSInteger _n;
}

- (instancetype)initWithSize:(NSInteger)n {
    self = [super init];
    if (self) {
        _n = n;
        _parent = (NSInteger *)malloc(sizeof(NSInteger) * n);
        _rank = (NSInteger *)calloc(n, sizeof(NSInteger));
        _weight = (double *)malloc(sizeof(double) * n);
        for (NSInteger i = 0; i < n; i++) {
            _parent[i] = i;
            _weight[i] = 1.0;  // 乘法单位元
        }
    }
    return self;
}

- (void)dealloc {
    free(_parent);
    free(_rank);
    free(_weight);
}

- (NSInteger)find:(NSInteger)x {
    if (_parent[x] != x) {
        NSInteger root = [self find:_parent[x]];
        _weight[x] *= _weight[_parent[x]];  // 乘法累积
        _parent[x] = root;
    }
    return _parent[x];
}

- (BOOL)unionX:(NSInteger)x y:(NSInteger)y weight:(double)w {
    // w 表示 x / y = w，即 value[x] = w * value[y]
    NSInteger rootX = [self find:x];
    NSInteger rootY = [self find:y];
    if (rootX == rootY) return NO;
    
    if (_rank[rootX] < _rank[rootY]) {
        _parent[rootX] = rootY;
        // rootX 到 rootY 的权值
        _weight[rootX] = w * _weight[y] / _weight[x];
    } else if (_rank[rootX] > _rank[rootY]) {
        _parent[rootY] = rootX;
        _weight[rootY] = _weight[x] / (w * _weight[y]);
    } else {
        _parent[rootY] = rootX;
        _weight[rootY] = _weight[x] / (w * _weight[y]);
        _rank[rootX]++;
    }
    return YES;
}

- (NSNumber *)queryX:(NSInteger)x y:(NSInteger)y {
    NSInteger rootX = [self find:x];
    NSInteger rootY = [self find:y];
    if (rootX != rootY) return nil;
    return @(_weight[x] / _weight[y]);
}

@end
```

## 9. 变体二：可撤销并查集（Rollback DSU）

### 9.1 概念

普通并查集的 Union 是不可逆的（路径压缩使恢复困难）。可撤销并查集放弃路径压缩（仅用按秩合并），在每次 Union 前将修改操作记录到栈中，支持按时间顺序撤销 Union 操作。

应用场景：线段树分治、离线区间问题（需要在某些时间段内将边加入/移除）。

### 9.2 关键点

- **不能使用路径压缩**（否则无法恢复），仅用按秩合并
- 每次 Union 操作记录 `(rootX, rankX, rootY, rankY)` 到操作栈
- Rollback 时弹出栈顶，恢复 parent 和 rank

### 9.3 Swift 实现

```swift
/// 可撤销并查集 —— 仅按秩合并（不做路径压缩），支持 Undo
/// 适用于线段树分治等需要时间回退的场景
/// 单次操作 O(log n)
struct RollbackUnionFind {
    private var parent: [Int]
    private var rank: [Int]
    private var history: [(index: Int, oldParent: Int, oldRank: Int)]  // 操作历史栈
    private(set) var count: Int
    
    init(_ n: Int) {
        parent = Array(0..<n)
        rank = Array(repeating: 0, count: n)
        history = []
        count = n
    }
    
    /// 查找根（不做路径压缩，O(log n)）
    func find(_ x: Int) -> Int {
        var node = x
        while parent[node] != node {
            node = parent[node]
        }
        return node
    }
    
    /// 当前历史版本号（用于 rollback）
    var version: Int { history.count }
    
    /// 合并两个集合，记录操作到历史栈
    @discardableResult
    mutating func union(_ x: Int, _ y: Int) -> Bool {
        let rootX = find(x)
        let rootY = find(y)
        if rootX == rootY { return false }
        
        // 按秩合并：较矮挂较高
        if rank[rootX] < rank[rootY] {
            history.append((rootX, parent[rootX], rank[rootX]))
            parent[rootX] = rootY
        } else if rank[rootX] > rank[rootY] {
            history.append((rootY, parent[rootY], rank[rootY]))
            parent[rootY] = rootX
        } else {
            history.append((rootY, parent[rootY], rank[rootY]))
            parent[rootY] = rootX
            // 还需记录 rank 的变化
            history.append((rootX, parent[rootX], rank[rootX]))
            rank[rootX] += 1
        }
        count -= 1
        return true
    }
    
    /// 撤销到指定版本（version 对应 history.count）
    mutating func rollback(to targetVersion: Int) {
        while history.count > targetVersion {
            let record = history.removeLast()
            parent[record.index] = record.oldParent
            rank[record.index] = record.oldRank
            // 注意：如果当前 Union 记录了两条历史（rank 相等情况），rollback 会自动处理
            if parent[record.index] == record.index {
                count += 1  // 恢复为独立根
            }
        }
    }
    
    func connected(_ x: Int, _ y: Int) -> Bool {
        return find(x) == find(y)
    }
}
```

### 9.4 Objective-C 实现

```objectivec
    if (self) {
        _n = n;

    NSInteger rootY = [self find:y];
    if (rootX == rootY) return NO;
    
    if (_rank[rootX] < _rank[rootY]) {
        [_history addObject:@[@(rootX), @(_parent[rootX]), @(_rank[rootX])]];
        _parent[rootX] = rootY;
    } else if (_rank[rootX] > _rank[rootY]) {
        [_history addObject:@[@(rootY), @(_parent[rootY]), @(_rank[rootY])]];
        _parent[rootY] = rootX;
    } else {
        [_history addObject:@[@(rootY), @(_parent[rootY]), @(_rank[rootY])]];
        _parent[rootY] = rootX;
        [_history addObject:@[@(rootX), @(_parent[rootX]), @(_rank[rootX])]];
        _rank[rootX]++;
    }
    _count--;
    return YES;
}

- (void)rollbackToVersion:(NSInteger)targetVersion {
    while ((NSInteger)_history.count > targetVersion) {
        NSArray<NSNumber *> *record = _history.lastObject;
        [_history removeLastObject];
        NSInteger idx = record[0].integerValue;
        _parent[idx] = record[1].integerValue;
        _rank[idx] = record[2].integerValue;
        if (_parent[idx] == idx) _count++;
    }
}

    return [self find:x] == [self find:y];
}
```

## 10. 变体三：可持久化并查集

### 10.1 概念

可持久化并查集允许查询任意历史版本的状态，而不仅仅是撤销最近的操作。实现思路：将 parent 和 rank 数组替换为**可持久化数组**（基于路径拷贝的 Fat Node 或可持久化线段树/平衡树）。

### 10.2 适用场景

- 需要同时维护多个版本并支持「从版本 v 派生新操作」的场景
- 离线分治 + 并查集问题
- 竞赛中的"强制在线"连通性查询

## 11. 经典问题一：连通分量计数

### 11.1 问题描述

LeetCode 547：省份数量（Number of Provinces）

有 n 个城市，给定一个 n×n 的邻接矩阵 `isConnected`，`isConnected[i][j] = 1` 表示城市 i 和 j 直接相连。求省份总数（连通分量数）。

### 11.2 思路

遍历邻接矩阵的上三角，对每条边执行 Union。最终 `uf.count` 即为答案。

### 11.3 Swift 实现

```swift
/// LeetCode 547: 省份数量
/// 时间 O(n² · α(n))，空间 O(n)
func findCircleNum(_ isConnected: [[Int]]) -> Int {
    let n = isConnected.count
    var uf = UnionFind(n)
    
    for i in 0..<n {
        for j in (i + 1)..<n {
            if isConnected[i][j] == 1 {
                uf.union(i, j)
            }
        }
    }
    return uf.count
}
```

## 12. 经典问题二：冗余连接（检测环）

### 12.1 问题描述

LeetCode 684：冗余连接（Redundant Connection）

一棵 n 个节点的树多了一条边变成了图（含一个环）。给定所有 n 条边，返回可以删去的那条边使图变回树。如果有多个答案，返回最后出现的那条。

### 12.2 思路

并查集的经典应用——检测环。按顺序遍历边，对每条边 `(u, v)` 执行 Union：
- 如果 Find(u) == Find(v)，说明 u 和 v 已连通，再加这条边形成环，返回这条边
- 否则正常 Union

### 12.3 Swift 实现

```swift
/// LeetCode 684: 冗余连接
/// 时间 O(n · α(n))，空间 O(n)
func findRedundantConnection(_ edges: [[Int]]) -> [Int] {
    let n = edges.count
    var uf = UnionFind(n + 1)  // 节点编号 1..n
    
    for edge in edges {
        let u = edge[0], v = edge[1]
        if !uf.union(u, v) {
            return edge  // 已连通，此边冗余
        }
    }
    return []  // 不应到达
}
```

### 12.4 Objective-C 实现


## 13. 经典问题三：岛屿数量（二维网格并查集）

### 13.1 问题描述

LeetCode 200：岛屿数量（Number of Islands）

给定一个 m×n 的二维网格，'1' 表示陆地，'0' 表示水。计算岛屿数量。岛屿被水包围，由上下左右相邻的陆地连接形成。

### 13.2 思路

虽然 DFS/BFS 是经典解法，但并查集也能优雅解决：
1. 将二维坐标 (i, j) 映射为一维 id = i * cols + j
2. 初始时只有 '1' 格子加入并查集
3. 遍历每个 '1' 格子，与其右方和下方的 '1' 格子执行 Union
4. 最终连通分量数即为岛屿数

### 13.3 Swift 实现

```swift
/// LeetCode 200: 岛屿数量（并查集解法）
/// 时间 O(m·n·α(m·n))，空间 O(m·n)
func numIslands(_ grid: [[Character]]) -> Int {
    guard !grid.isEmpty else { return 0 }
    let m = grid.count, n = grid[0].count
    var uf = UnionFind(m * n)
    var waterCount = 0  // 水域格子数（不参与连通）
    
    for i in 0..<m {
        for j in 0..<n {
            if grid[i][j] == "0" {
                waterCount += 1
                continue
            }
            let id = i * n + j
            // 向右合并
            if j + 1 < n && grid[i][j + 1] == "1" {
                uf.union(id, id + 1)
            }
            // 向下合并
            if i + 1 < m && grid[i + 1][j] == "1" {
                uf.union(id, id + n)
            }
        }
    }
    // 岛屿数 = 总连通分量数 - 水域格子数（每个水域格子是独立分量但不算岛屿）
    return uf.count - waterCount
}
```

## 14. 经典问题四：账户合并

### 14.1 问题描述

LeetCode 721：账户合并（Accounts Merge）

给定一个列表 accounts，每个 account 是 `[name, email1, email2, ...]`。如果两个账户有共同的邮箱，则它们属于同一个人。合并属于同一人的所有账户，返回合并后的结果。

### 14.2 思路

1. 为每个邮箱分配一个唯一 ID
2. 同一账户内的所有邮箱执行 Union（它们属于同一人）
3. Union 完成后，按根节点分组，每组就是一个人的所有邮箱
4. 排序输出

### 14.3 Swift 实现

```swift
/// LeetCode 721: 账户合并
/// 时间 O(N·K·α(N·K) + N·K·log(N·K))，N 为账户数，K 为平均邮箱数
func accountsMerge(_ accounts: [[String]]) -> [[String]] {
    var emailToId: [String: Int] = [:]
    var emailToName: [String: String] = [:]
    var idCounter = 0
    
    // Step 1: 为每个邮箱分配 ID 并记录其所属姓名
    for account in accounts {
        let name = account[0]
        for i in 1..<account.count {
            let email = account[i]
            if emailToId[email] == nil {
                emailToId[email] = idCounter
                idCounter += 1
            }
            emailToName[email] = name
        }
    }
    
    // Step 2: 同一账户内的邮箱 Union
    var uf = UnionFind(idCounter)
    for account in accounts {
        guard account.count > 1 else { continue }
        let firstId = emailToId[account[1]]!
        for i in 2..<account.count {
            let curId = emailToId[account[i]]!
            uf.union(firstId, curId)
        }
    }
    
    // Step 3: 按根分组
    var rootToEmails: [Int: [String]] = [:]
    for (email, id) in emailToId {
        let root = uf.find(id)
        rootToEmails[root, default: []].append(email)
    }
    
    // Step 4: 组装结果
    var result: [[String]] = []
    for (_, emails) in rootToEmails {
        let sortedEmails = emails.sorted()
        let name = emailToName[sortedEmails[0]]!
        result.append([name] + sortedEmails)
    }
    return result
}
```

### 14.4 Objective-C 实现

```objectivec
        NSString *name = account[0];
        for (NSInteger i = 1; i < account.count; i++) {
```

## 15. 经典问题五：最小生成树 Kruskal 算法

### 15.1 Kruskal 算法与并查集

Kruskal 算法求最小生成树（MST）的核心思路：
1. 将所有边按权重排序
2. 从小到大遍历每条边 (u, v, w)
3. 如果 u 和 v 不在同一连通分量（Find(u) ≠ Find(v)），加入 MST 并 Union(u, v)
4. 重复直到 MST 有 n-1 条边

并查集在此用于快速判断"加入这条边是否形成环"。

### 15.2 Swift 实现

```swift
/// Kruskal 最小生成树算法
/// 时间 O(E·logE + E·α(V))，通常由排序主导为 O(E·logE)
/// - Parameter edges: 边列表 [(u, v, weight)]
/// - Parameter n: 节点数
/// - Returns: (MST总权重, MST边列表)
func kruskal(edges: [(Int, Int, Int)], n: Int) -> (Int, [(Int, Int, Int)]) {
    // 按权重排序
    let sorted = edges.sorted { $0.2 < $1.2 }
    var uf = UnionFind(n)
    var mstWeight = 0
    var mstEdges: [(Int, Int, Int)] = []
    
    for (u, v, w) in sorted {
        if uf.union(u, v) {
            // 成功合并 → 不形成环 → 加入 MST
            mstWeight += w
            mstEdges.append((u, v, w))
            if mstEdges.count == n - 1 { break }  // MST 完成
        }
    }
    return (mstWeight, mstEdges)
}

// 使用示例
// let edges = [(0,1,4), (0,2,3), (1,2,1), (1,3,2), (2,3,5)]
// let (weight, mst) = kruskal(edges: edges, n: 4)
// weight = 6, mst = [(1,2,1), (1,3,2), (0,2,3)]
```

### 15.3 Objective-C 实现

```objectivec
/// Kruskal 最小生成树
/// 输入：edges = @[@[@(u), @(v), @(w)], ...]，n = 节点数
/// 返回：MST 总权重
- (NSInteger)kruskal:(NSArray<NSArray<NSNumber *> *> *)edges nodeCount:(NSInteger)n {
    // 按权重排序
    NSArray *sorted = [edges sortedArrayUsingComparator:^NSComparisonResult(NSArray *a, NSArray *b) {
        return [a[2] compare:b[2]];
    }];
    
    SAKUnionFind *uf = [[SAKUnionFind alloc] initWithSize:n];
    NSInteger mstWeight = 0;
    NSInteger edgeCount = 0;
    
    for (NSArray<NSNumber *> *edge in sorted) {
        NSInteger u = edge[0].integerValue;
        NSInteger v = edge[1].integerValue;
        NSInteger w = edge[2].integerValue;
        
        if ([uf unionX:u y:v]) {
            mstWeight += w;
            edgeCount++;
            if (edgeCount == n - 1) break;
        }
    }
    return mstWeight;
}
```

## 16. 经典问题六：等式方程的可满足性

### 16.1 问题描述

LeetCode 990：等式方程的可满足性（Satisfiability of Equality Equations）

给定一组方程形如 "a==b" 或 "a!=b"（a, b 为小写字母），判断是否能同时满足所有方程。

### 16.2 思路

1. 先处理所有 `==` 方程：将相等的变量 Union
2. 再检查所有 `!=` 方程：如果不等的两个变量已在同一集合中，则矛盾

### 16.3 Swift 实现

```swift
/// LeetCode 990: 等式方程的可满足性
/// 时间 O(n · α(26)) = O(n)，空间 O(1)（固定 26 个字母）
func equationsPossible(_ equations: [String]) -> Bool {
    var uf = UnionFind(26)  // 26 个小写字母
    
    // 第一遍：处理等式，合并相等变量
    for eq in equations {
        let chars = Array(eq)
        if chars[1] == "=" {  // "x==y"
            let x = Int(chars[0].asciiValue! - Character("a").asciiValue!)
            let y = Int(chars[3].asciiValue! - Character("a").asciiValue!)
            uf.union(x, y)
        }
    }
    
    // 第二遍：检查不等式
    for eq in equations {
        let chars = Array(eq)
        if chars[1] == "!" {  // "x!=y"
            let x = Int(chars[0].asciiValue! - Character("a").asciiValue!)
            let y = Int(chars[3].asciiValue! - Character("a").asciiValue!)
            if uf.connected(x, y) {
                return false  // 矛盾：x 和 y 已被约束为相等
            }
        }
    }
    return true
}
```

### 16.4 Objective-C 实现

```objectivec
            }
        }
    }
    return YES;
}
```

## 17. 经典问题七：除法求值（带权并查集应用）

### 17.1 问题描述

LeetCode 399：除法求值（Evaluate Division）

给定一些已知的除法等式 a/b = k（k 为正实数），对于查询 x/y，求其值。如果无法确定，返回 -1.0。

### 17.2 思路

带权并查集，权值表示乘法关系：
- `weight[x]` 表示 x / root(x) 的值
- 已知 a/b = k → Union(a, b) with weight k
- 查询 x/y：如果 Find(x) == Find(y)，则 x/y = weight[x] / weight[y]

### 17.3 Swift 实现

```swift
/// LeetCode 399: 除法求值
/// 使用带权并查集，权值为乘法关系
func calcEquation(_ equations: [[String]], _ values: [Double], 
                  _ queries: [[String]]) -> [Double] {
    var varToId: [String: Int] = [:]
    var idCounter = 0
    
    // 为每个变量分配 ID
    for eq in equations {
        if varToId[eq[0]] == nil { varToId[eq[0]] = idCounter; idCounter += 1 }
        if varToId[eq[1]] == nil { varToId[eq[1]] = idCounter; idCounter += 1 }
    }
    
    // 带权并查集（乘法权值）
    var parent = Array(0..<idCounter)
    var weight = Array(repeating: 1.0, count: idCounter)  // weight[x] = x / parent[x]
    
    func find(_ x: Int) -> Int {
        if parent[x] != x {
            let root = find(parent[x])
            weight[x] *= weight[parent[x]]  // 路径上乘法累积
            parent[x] = root
        }
        return parent[x]
    }
    
    func union(_ x: Int, _ y: Int, _ w: Double) {
        let rootX = find(x)
        let rootY = find(y)
        if rootX == rootY { return }
        parent[rootX] = rootY
        // rootX / rootY = (x / rootX)⁻¹ * (x / y) * (y / rootY)
        //              = (1/weight[x]) * w * weight[y]
        weight[rootX] = w * weight[y] / weight[x]
    }
    
    // 建立关系
    for i in 0..<equations.count {
        let idA = varToId[equations[i][0]]!
        let idB = varToId[equations[i][1]]!
        union(idA, idB, values[i])  // a / b = values[i]
    }
    
    // 回答查询
    var results: [Double] = []
    for query in queries {
        guard let idX = varToId[query[0]], let idY = varToId[query[1]] else {
            results.append(-1.0)
            continue
        }
        let rootX = find(idX)
        let rootY = find(idY)
        if rootX != rootY {
            results.append(-1.0)  // 不连通
        } else {
            results.append(weight[idX] / weight[idY])  // x/y = (x/root) / (y/root)
        }
    }
    return results
}
```

### 17.4 Objective-C 实现

```objectivec
                               values:(NSArray<NSNumber *> *)values
```

## 18. 经典问题八：最长连续序列

### 18.1 问题描述

LeetCode 128：最长连续序列（Longest Consecutive Sequence）

给定一个未排序的整数数组 nums，找出数字连续的最长序列的长度。要求 O(n) 时间复杂度。

### 18.2 并查集解法思路

1. 用哈希表将每个数映射到并查集的 ID
2. 对每个数 num，如果 num+1 也在集合中，则 Union(num, num+1)
3. 最终最大集合的 size 即为答案

### 18.3 Swift 实现

```swift
/// LeetCode 128: 最长连续序列（并查集解法）
/// 时间 O(n · α(n)) ≈ O(n)，空间 O(n)
func longestConsecutive(_ nums: [Int]) -> Int {
    if nums.isEmpty { return 0 }
    
    // 去重并建立映射
    let numSet = Set(nums)
    var numToId: [Int: Int] = [:]
    var idCounter = 0
    for num in numSet {
        numToId[num] = idCounter
        idCounter += 1
    }
    
    var uf = UnionFindBySize(idCounter)
    
    // 对每个数，尝试与 num+1 合并
    for num in numSet {
        if let nextId = numToId[num + 1] {
            uf.union(numToId[num]!, nextId)
        }
    }
    
    return uf.maxSetSize()
}
```

### 18.4 Objective-C 实现

```objectivec
/// LeetCode 128: 最长连续序列（并查集解法）
- (NSInteger)longestConsecutive:(NSArray<NSNumber *> *)nums {
    if (nums.count == 0) return 0;
    
    NSMutableDictionary<NSNumber *, NSNumber *> *numToId = [NSMutableDictionary new];
    NSInteger idCounter = 0;
    
    // 去重 + 分配ID
    for (NSNumber *num in nums) {
        if (!numToId[num]) {
            numToId[num] = @(idCounter++);
        }
    }
    
    SAKUnionFindBySize *uf = [[SAKUnionFindBySize alloc] initWithSize:idCounter];
    
    for (NSNumber *num in numToId) {
        NSNumber *next = @(num.integerValue + 1);
        if (numToId[next]) {
            [uf unionX:numToId[num].integerValue y:numToId[next].integerValue];
        }
    }
    
    // 找最大集合
    NSInteger maxSize = 0;
    for (NSNumber *num in numToId) {
        NSInteger sz = [uf setSizeOf:numToId[num].integerValue];
        maxSize = MAX(maxSize, sz);
    }
    return maxSize;
}
```

## 20. 并查集 vs 其他数据结构对比

### 20.1 并查集 vs DFS/BFS

| 维度 | 并查集 | DFS/BFS |
|------|--------|---------|
| 适用场景 | 动态加边 + 连通性查询 | 静态图遍历、求路径 |
| 时间复杂度 | O(α(n)) 每次查询/合并 | O(V+E) 每次查询 |
| 支持求路径 | 否（只判连通） | 是 |
| 支持动态加边 | 是（核心优势） | 需重建/增量处理 |
| 支持删边 | 困难（需离线技巧） | 重新遍历即可 |
| 空间 | O(n) | O(V+E) |

### 20.2 并查集 vs 图的邻接表

| 维度 | 并查集 | 邻接表 |
|------|--------|--------|
| 信息量 | 只保留连通性（丢失边的具体信息） | 保留完整图结构 |
| 查询连通性 | O(α(n)) | O(V+E) |
| 适合 | 只关心"是否连通" | 需要遍历、求最短路等 |

### 20.3 并查集 vs 集合（Set/HashSet）

| 维度 | 并查集 | HashSet |
|------|--------|---------|
| 合并两集合 | O(α(n)) | O(min(|A|,|B|)) |
| 查询元素所属集合 | O(α(n)) | O(1)（但需额外映射） |
| 存储效率 | O(n) 一个数组 | 每个集合独立存储 |

### 20.4 何时选择并查集

选择并查集的信号：
- 问题本质是"动态连通性"
- 需要频繁合并 + 查询，但不需要拆分
- 边/关系是逐步添加的（增量式）
- 问题可以转化为等价类归并

不选择并查集的信号：
- 需要求具体路径（用 BFS）
- 需要删除边/断开连接（用可撤销并查集或离线技巧）
- 需要遍历某集合内所有元素（并查集不直接支持枚举）
- 图是静态的且只需做一次遍历

## 21. 面试高频题精选（附解答要点）

### 面试题 1：手写并查集模板

**题目**：请手写一个支持路径压缩和按秩合并的并查集，包含 init、find、union、connected 四个方法。

**考察点**：基本功、对两种优化的理解

**答案要点**：
- parent 数组初始化为自身
- find 递归/迭代 + 路径压缩
- union 先 find 两个根，按秩决定谁挂谁
- rank 相等时新根 rank+1
- connected = find(x) == find(y)

### 面试题 2：朋友圈问题

**题目**：给定 n 个人的关系矩阵，求朋友圈数量。

**解答要点**：遍历上三角矩阵的 1，Union 对应的人，最终 count 即为朋友圈数。时间 O(n²·α(n))。

### 面试题 3：判断图中是否有环

**题目**：给定无向图的边列表，判断是否存在环。

**解答要点**：逐条加边，若 Union 返回 false（两端点已连通），则检测到环。

### 面试题 4：最小生成树为什么用并查集

**题目**：解释 Kruskal 算法中并查集的作用。

**解答要点**：
- Kruskal 按权重从小到大贪心加边
- 需要快速判断"加入这条边是否形成环"
- 并查集 O(α(n)) 判断两端点连通性，比 DFS O(V+E) 快得多
- 整个算法由排序 O(E·logE) 主导

### 面试题 5：动态连通性 vs 静态连通性

**题目**：什么场景适合用并查集，什么场景用 BFS/DFS 更好？

**解答要点**：
- 动态加边 + 多次查询 → 并查集（增量维护）
- 静态图一次性查询 → BFS/DFS
- 需要路径信息 → BFS
- 需要删边 → 离线逆序处理 + 并查集，或重新 BFS

### 面试题 6：如何支持删除操作

**题目**：并查集如何支持删除节点/断开连接？

**解答要点**：
- 直接删除很难（路径压缩破坏了结构信息）
- 方案一：离线处理——将所有操作倒序，变"删除"为"添加"
- 方案二：惰性删除——为节点分配新 ID，旧 ID 废弃
- 方案三：可撤销并查集（不用路径压缩 + 操作栈）
- 方案四：Link-Cut Tree（支持动态断边，O(log n)）

### 面试题 7：种类并查集

**题目**：如何用并查集判断二分图/将节点分为两组，使同组无冲突边？

**解答要点**：
- 扩展域/种类并查集：每个节点拆成 node_same 和 node_diff
- "x 和 y 不同组" → Union(x_same, y_diff) 且 Union(x_diff, y_same)
- "x 和 y 同组" → Union(x_same, y_same) 且 Union(x_diff, y_diff)
- 矛盾检测：若 Find(x_same) == Find(x_diff) → 无解

### 面试题 8：岛屿数量的多种解法对比

**题目**：LeetCode 200 岛屿数量，DFS/BFS/并查集三种解法的时间空间复杂度对比。

**解答要点**：

| 方法 | 时间 | 空间 | 特点 |
|------|------|------|------|
| DFS | O(m·n) | O(m·n) 递归栈 | 最直观、代码最短 |
| BFS | O(m·n) | O(min(m,n)) 队列 | 避免栈溢出 |
| 并查集 | O(m·n·α(m·n)) | O(m·n) | 适合动态加陆地的变体 |

并查集的优势在 LeetCode 305（动态岛屿数量）中体现——逐个加入陆地并动态维护。

## 22. 易错点与最佳实践

### 22.1 常见易错点

**错误 1：Union 时忘记先 Find 根**

```swift
// ❌ 错误：直接用 x, y 作为 parent 赋值
func union(_ x: Int, _ y: Int) {
    parent[x] = y  // 错！x 可能不是根
}

// ✅ 正确：先找到各自的根再操作
func union(_ x: Int, _ y: Int) {
    let rootX = find(x)
    let rootY = find(y)
    if rootX != rootY { parent[rootX] = rootY }
}
```

**错误 2：路径压缩写错（忘记赋值 parent[x]）**

```swift
// ❌ 错误：只递归不赋值
func find(_ x: Int) -> Int {
    if parent[x] != x { return find(parent[x]) }  // 没有压缩！
    return parent[x]
}

// ✅ 正确：递归结果赋给 parent[x]
func find(_ x: Int) -> Int {
    if parent[x] != x { parent[x] = find(parent[x]) }
    return parent[x]
}
```

**错误 3：节点编号越界**

图问题中节点编号可能从 1 开始，但并查集从 0 开始。需要注意：
- 如果节点编号 1..n，初始化 UnionFind(n+1)
- 或者统一减 1 处理

**错误 4：使用 struct 时忘记 mutating**

Swift 中 struct 的方法如果修改了属性（parent, rank），必须标记 `mutating`。否则编译器报错。

**错误 5：带权并查集的权值更新顺序**

路径压缩时，必须**先递归**找到根并更新 parent[x]，再累加权值。顺序错误会导致权值计算错误。

```swift
// ❌ 错误顺序
func find(_ x: Int) -> Int {
    if parent[x] != x {
        weight[x] += weight[parent[x]]  // 此时 parent[x] 还没压缩完！
        parent[x] = find(parent[x])
    }
    return parent[x]
}

// ✅ 正确顺序
func find(_ x: Int) -> Int {
    if parent[x] != x {
        let root = find(parent[x])       // 先递归压缩 parent[x] 的路径
        weight[x] += weight[parent[x]]   // 此时 weight[parent[x]] 已是到 root 的值
        parent[x] = root
    }
    return parent[x]
}
```

**错误 6：Objective-C 中内存泄漏**

使用 malloc 分配的数组必须在 dealloc 中 free。忘记 free 会造成内存泄漏。建议使用 `calloc`（自动清零）或在 dealloc 中加断言检查。

### 22.2 最佳实践

**实践 1：优先使用按大小合并**

除非题目明确需要 rank，否则推荐用按大小合并（Union by Size）。原因：
- size 的语义始终准确（路径压缩不影响）
- 随时可以 O(1) 查询集合大小
- 实测性能与按秩合并几乎无差别

**实践 2：选择迭代式路径压缩**

在实际工程代码中（非竞赛），优先使用迭代两遍法而非递归。原因：
- 无栈溢出风险
- 对极深的树更安全
- 调试更方便

**实践 3：使用 struct（值类型）封装**

Swift 中推荐用 struct 而非 class 封装并查集：
- 值语义，避免意外的引用共享
- 可拷贝（保存快照）
- 性能更好（内存局部性）

**实践 4：预估容量**

如果元素数量已知，在初始化时一次性分配好数组，避免动态扩容的开销。

**实践 5：离线 + 逆序处理删边问题**

当问题涉及删除边时，常用技巧是将问题离线化：
1. 读取所有操作
2. 逆序处理：将"删边"变为"加边"
3. 用并查集正常处理
4. 逆序输出答案

**实践 6：二维坐标映射**

二维网格问题中，坐标 (i, j) → ID 的经典映射：`id = i * cols + j`。注意方向数组只需右和下（或四方向取其二），避免重复 Union。

### 22.3 调试技巧

```swift
/// 调试辅助：打印并查集内部状态
extension UnionFind {
    func debugPrint() {
        print("Parent: \(parent)")
        print("Rank:   \(rank)")
        print("Count:  \(count)")
        
        // 按根分组
        var groups: [Int: [Int]] = [:]
        var tempSelf = self
        for i in 0..<parent.count {
            let root = tempSelf.find(i)
            groups[root, default: []].append(i)
        }
        for (root, members) in groups.sorted(by: { $0.key < $1.key }) {
            print("  Group (root=\(root)): \(members)")
        }
    }
}
```
