# 位运算技巧（Bit Manipulation）

> 位运算是直接操作二进制位的底层运算，是算法竞赛与系统编程中的"瑞士军刀"。掌握位运算技巧不仅能写出极致高效的代码，更能深入理解计算机的本质运行方式。

---

## 一、基础概念与原理

### 1.1 什么是位运算（What is Bit Manipulation）

位运算（Bit Manipulation）是对整数在内存中的二进制表示逐位进行的运算操作。与算术运算不同，位运算直接在 ALU（算术逻辑单元）的门电路层面执行，通常只需一个 CPU 时钟周期，是所有运算中最快的。

计算机中所有数据最终都以二进制形式存储。一个 32 位整数占 4 字节（32 个 bit），每一位只能是 0 或 1。位运算就是对这些 0/1 位进行逻辑操作。

### 1.2 为什么需要位运算（Why）

- **极致性能**：位运算是 CPU 原生支持的最底层操作，一条指令完成，比乘除快一个数量级
- **空间压缩**：用一个整数的每一位表示一个布尔状态，32 个状态只需 4 字节
- **算法优化**：很多看似复杂的问题（如找唯一数、子集枚举）用位运算可以优雅解决
- **系统编程**：权限系统、网络协议、硬件寄存器、密码学中大量使用位运算
- **面试考察**：位运算是面试高频考点，考察对计算机底层的理解深度

### 1.3 六种基本位运算符

| 运算符 | 名称 | 规则 | 示例（8位） |
|--------|------|------|-------------|
| `&` | 按位与（AND） | 两位都为1结果才为1 | `0b1100 & 0b1010 = 0b1000` |
| `\|` | 按位或（OR） | 两位有一个为1结果就为1 | `0b1100 \| 0b1010 = 0b1110` |
| `^` | 按位异或（XOR） | 两位不同结果为1 | `0b1100 ^ 0b1010 = 0b0110` |
| `~` | 按位取反（NOT） | 0变1，1变0 | `~0b1100 = 0b0011`（简化） |
| `<<` | 左移（Left Shift） | 所有位向左移，右边补0 | `0b0011 << 2 = 0b1100` |
| `>>` | 右移（Right Shift） | 所有位向右移 | `0b1100 >> 2 = 0b0011` |

### 1.4 补码表示（Two's Complement）

现代计算机使用**补码**表示有符号整数：

- **正数**：补码 = 原码
- **负数**：补码 = 按位取反 + 1（即 `~x + 1 = -x`）
- **范围**：n 位补码表示范围为 `[-2^(n-1), 2^(n-1) - 1]`

```
 5 的 8 位补码：00000101
-5 的 8 位补码：11111011（= ~00000101 + 1 = 11111010 + 1）
```

关键恒等式：
- `-x = ~x + 1`（取负数等于取反加一）
- `~x = -x - 1`（取反等于取负减一）
- `x + (~x) = -1`（即全 1）

### 1.5 右移的两种模式

- **逻辑右移（Logical Right Shift）**：高位补 0，适用于无符号数
- **算术右移（Arithmetic Right Shift）**：高位补符号位，适用于有符号数

Swift 中 `>>` 对有符号数执行算术右移，对无符号数执行逻辑右移。
Objective-C 中 `>>` 的行为取决于操作数类型（`int` 算术右移，`unsigned int` 逻辑右移）。

## 二、核心位运算技巧（由浅入深）

### 2.1 基本操作集

#### 2.1.1 获取第 i 位

```swift
// Swift: 获取 n 的第 i 位（从右往左，0-indexed）
func getBit(_ n: Int, _ i: Int) -> Int {
    return (n >> i) & 1
}
```

```objectivec
// Objective-C
- (NSInteger)getBit:(NSInteger)n at:(NSInteger)i {
    return (n >> i) & 1;
}
```

**原理**：先将第 i 位右移到最低位，再与 1 做 AND 运算提取该位。

#### 2.1.2 设置第 i 位为 1

```swift
// Swift: 将第 i 位设为 1
func setBit(_ n: Int, _ i: Int) -> Int {
    return n | (1 << i)
}
```

```objectivec
// Objective-C
- (NSInteger)setBit:(NSInteger)n at:(NSInteger)i {
    return n | (1 << i);
}
```

**原理**：构造一个只有第 i 位为 1 的掩码 `(1 << i)`，与 n 做 OR 运算。

#### 2.1.3 清除第 i 位（设为 0）

```swift
// Swift: 将第 i 位设为 0
func clearBit(_ n: Int, _ i: Int) -> Int {
    return n & ~(1 << i)
}
```

```objectivec
// Objective-C
- (NSInteger)clearBit:(NSInteger)n at:(NSInteger)i {
    return n & ~(1 << i);
}
```

**原理**：构造一个只有第 i 位为 0、其余全为 1 的掩码 `~(1 << i)`，与 n 做 AND 运算。

#### 2.1.4 翻转第 i 位

```swift
// Swift: 翻转第 i 位（0→1，1→0）
func toggleBit(_ n: Int, _ i: Int) -> Int {
    return n ^ (1 << i)
}
```

```objectivec
// Objective-C
- (NSInteger)toggleBit:(NSInteger)n at:(NSInteger)i {
    return n ^ (1 << i);
}
```

**原理**：XOR 的自反性——与 1 异或翻转该位，与 0 异或保持不变。

#### 2.1.5 更新第 i 位为指定值

```swift
// Swift: 将第 i 位设为 val（0 或 1）
func updateBit(_ n: Int, _ i: Int, _ val: Int) -> Int {
    let cleared = n & ~(1 << i)  // 先清除
    return cleared | (val << i)   // 再设置
}
```

```objectivec
// Objective-C
- (NSInteger)updateBit:(NSInteger)n at:(NSInteger)i withValue:(NSInteger)val {
    NSInteger cleared = n & ~(1 << i);
    return cleared | (val << i);
}
```

### 2.2 经典位运算恒等式（必背）

以下恒等式是解题利器，需要熟练掌握：

| 表达式 | 含义 | 解释 |
|--------|------|------|
| `n & (n - 1)` | 消除最低位的 1 | `n-1` 会把最低的 1 变成 0，该位以下全变成 1 |
| `n & (-n)` | 提取最低位的 1（lowbit） | `-n = ~n + 1`，只有最低 1 处两者都是 1 |
| `n \| (n - 1)` | 将最低位 1 右侧全部置 1 | — |
| `n ^ (n - 1)` | 最低位 1 及其右侧全部变 1 | — |
| `n & (n + 1)` | 消除末尾连续的 1 | — |
| `n \| (n + 1)` | 将最低位 0 设为 1 | — |
| `x ^ y ^ y = x` | XOR 自消性 | 任何数与自身异或为 0 |
| `x ^ 0 = x` | XOR 恒等律 | — |
| `x ^ (-1) = ~x` | XOR 全 1 等于取反 | — |

#### `n & (n - 1)` 图解

```
n     = ...1 0 1 0 0 0   (最低的1在第3位)
n - 1 = ...1 0 0 1 1 1   (第3位变0，右侧全变1)
n & (n-1) = ...1 0 0 0 0 0   (最低的1被消除)
```

这是位运算中**最重要**的技巧之一，应用包括：
- 判断是否是 2 的幂：`n > 0 && (n & (n - 1)) == 0`
- 计算 1 的个数（Brian Kernighan 算法）：反复执行 `n &= (n - 1)` 直到 n 为 0
- 判断是否是 4 的幂：先判断是 2 的幂，再检查 1 在奇数位

#### `n & (-n)` 图解（lowbit）

```
n    = ...1 0 1 1 0 0   (最低的1在第2位)
-n   = ...0 1 0 1 0 0   (取反加1)
n&(-n) = ...0 0 0 1 0 0   (只保留最低的1)
```

lowbit 在树状数组（Binary Indexed Tree / Fenwick Tree）中是核心操作。

### 2.3 判断 2 的幂

```swift
// Swift: 判断 n 是否是 2 的幂
func isPowerOfTwo(_ n: Int) -> Bool {
    return n > 0 && (n & (n - 1)) == 0
}
```

```objectivec
// Objective-C
- (BOOL)isPowerOfTwo:(NSInteger)n {
    return n > 0 && (n & (n - 1)) == 0;
}
```

**原理**：2 的幂的二进制表示中只有一个 1。`n & (n-1)` 消除唯一的 1 后为 0。

**注意边界**：必须判断 `n > 0`，因为 `n = 0` 时 `0 & (-1) = 0` 也满足条件但 0 不是 2 的幂。

### 2.4 计数：统计二进制中 1 的个数（Population Count / Hamming Weight）

#### 方法一：Brian Kernighan 算法

```swift
// Swift: 计算 n 中 1 的个数
func hammingWeight(_ n: Int) -> Int {
    var count = 0
    var num = n
    while num != 0 {
        num &= (num - 1)  // 消除最低位的1
        count += 1
    }
    return count
}
```

```objectivec
// Objective-C
- (NSInteger)hammingWeight:(NSInteger)n {
    NSInteger count = 0;
    while (n != 0) {
        n &= (n - 1);
        count++;
    }
    return count;
}
```

**复杂度**：时间 O(k)，k 为 1 的个数；空间 O(1)。最坏 O(logN)。

#### 方法二：分治/并行计数（MIT HAKMEM / Divide and Conquer）

这是系统库（如 GCC 的 `__builtin_popcount`）使用的方法，O(1) 常数时间：

```swift
// Swift: O(1) popcount（32位版本）
func popcount32(_ x: UInt32) -> Int {
    var n = x
    n = (n & 0x55555555) + ((n >> 1) & 0x55555555)  // 每2位一组，统计1的个数
    n = (n & 0x33333333) + ((n >> 2) & 0x33333333)  // 每4位一组
    n = (n & 0x0F0F0F0F) + ((n >> 4) & 0x0F0F0F0F)  // 每8位一组
    n = (n & 0x00FF00FF) + ((n >> 8) & 0x00FF00FF)  // 每16位一组
    n = (n & 0x0000FFFF) + ((n >> 16) & 0x0000FFFF) // 最终32位
    return Int(n)
}
```

```objectivec
// Objective-C: O(1) popcount
- (NSInteger)popcount32:(uint32_t)x {
    x = (x & 0x55555555) + ((x >> 1) & 0x55555555);
    x = (x & 0x33333333) + ((x >> 2) & 0x33333333);
    x = (x & 0x0F0F0F0F) + ((x >> 4) & 0x0F0F0F0F);
    x = (x & 0x00FF00FF) + ((x >> 8) & 0x00FF00FF);
    x = (x & 0x0000FFFF) + ((x >> 16) & 0x0000FFFF);
    return (NSInteger)x;
}
```

**原理图解（以 8 位为例）**：

```
原始:  1 0 1 1 0 1 1 0
步骤1: [01][10][01][01]   每2位组内的1个数
步骤2: [0011]  [0010]     每4位组内的1个数
步骤3: [00000101]         最终结果 = 5
```

### 2.5 查找：最低位/最高位的位置

#### 最低有效位位置（Trailing Zeros / CTZ）

```swift
// Swift: 返回最低位1的位置（0-indexed），若 n=0 返回 -1
func trailingZeros(_ n: Int) -> Int {
    if n == 0 { return -1 }
    var count = 0
    var num = n
    while (num & 1) == 0 {
        count += 1
        num >>= 1
    }
    return count
}

// 更高效：利用 lowbit
func trailingZerosV2(_ n: Int) -> Int {
    if n == 0 { return -1 }
    // n & (-n) 提取最低位1，对其取 log2 即为位置
    let lowbit = n & (-n)
    return Int(log2(Double(lowbit)))
}
```

```objectivec
// Objective-C: 使用内置函数（推荐）
- (NSInteger)trailingZeros:(NSInteger)n {
    if (n == 0) return -1;
    return __builtin_ctzl((unsigned long)n);  // GCC/Clang 内置
}
```

#### 最高有效位位置（Leading Zeros / CLZ）

```swift
// Swift: 返回最高位1的位置（0-indexed from right）
func highestBit(_ n: Int) -> Int {
    if n == 0 { return -1 }
    var pos = 0
    var num = n
    // 二分法快速定位
    if num & 0xFFFF0000 != 0 { pos += 16; num >>= 16 }
    if num & 0x0000FF00 != 0 { pos += 8;  num >>= 8 }
    if num & 0x000000F0 != 0 { pos += 4;  num >>= 4 }
    if num & 0x0000000C != 0 { pos += 2;  num >>= 2 }
    if num & 0x00000002 != 0 { pos += 1 }
    return pos
}
```

```objectivec
// Objective-C: 使用内置函数
- (NSInteger)highestBit:(NSInteger)n {
    if (n == 0) return -1;
    return 63 - __builtin_clzl((unsigned long)n);  // 64位系统
}
```

### 2.6 位运算实现加法（不用 + 号）

```swift
// Swift: 不使用算术运算符实现加法
func add(_ a: Int, _ b: Int) -> Int {
    var x = a, y = b
    while y != 0 {
        let carry = x & y       // 进位：两位都为1时产生进位
        x = x ^ y               // 无进位求和：异或
        y = carry << 1           // 进位左移一位
    }
    return x
}
```

```objectivec
// Objective-C
- (NSInteger)add:(NSInteger)a b:(NSInteger)b {
    while (b != 0) {
        NSInteger carry = a & b;
        a = a ^ b;
        b = carry << 1;
    }
    return a;
}
```

**原理**：
- `a ^ b`：不考虑进位的加法（半加器）
- `a & b`：产生进位的位（两位都为 1）
- `carry << 1`：进位应作用于高一位
- 循环直到没有进位为止

**复杂度**：时间 O(n)，n 为位数（最多进位 n 次）；空间 O(1)。

### 2.7 位运算实现乘法

```swift
// Swift: 位运算实现乘法（俄罗斯农民乘法 / 快速乘）
func multiply(_ a: Int, _ b: Int) -> Int {
    var result = 0
    var x = a, y = b
    // 处理负数
    let negative = (x < 0) != (y < 0)
    x = abs(x); y = abs(y)
    
    while y > 0 {
        if y & 1 == 1 {
            result += x      // 当前位为1，累加
        }
        x <<= 1             // 被乘数左移（×2）
        y >>= 1             // 乘数右移（÷2）
    }
    return negative ? -result : result
}
```

```objectivec
// Objective-C
- (NSInteger)multiply:(NSInteger)a b:(NSInteger)b {
    NSInteger result = 0;
    BOOL negative = (a < 0) != (b < 0);
    NSInteger x = labs(a), y = labs(b);
    while (y > 0) {
        if (y & 1) result += x;
        x <<= 1;
        y >>= 1;
    }
    return negative ? -result : result;
}
```

**原理**：将乘法分解为"移位+加法"。例如 `13 × 11 = 13 × (1011)₂ = 13×8 + 13×2 + 13×1`。

### 2.8 交换两个数（不用临时变量）

```swift
// Swift: XOR 交换
func swapXOR(_ a: inout Int, _ b: inout Int) {
    guard a != b else { return }  // 重要：同一地址会归零！
    a ^= b   // a = a⊕b
    b ^= a   // b = b⊕(a⊕b) = a
    a ^= b   // a = (a⊕b)⊕a = b
}
```

```objectivec
// Objective-C
- (void)swap:(NSInteger *)a with:(NSInteger *)b {
    if (*a == *b) return;  // 防止同一地址
    *a ^= *b;
    *b ^= *a;
    *a ^= *b;
}
```

**注意**：如果 a 和 b 指向同一内存地址，XOR 交换会导致值变为 0！实践中建议用临时变量，更安全、编译器也能更好优化。

### 2.9 绝对值（不用分支）

```swift
// Swift: 位运算求绝对值（仅适用于固定位宽有符号整数）
func bitwiseAbs(_ n: Int32) -> Int32 {
    let mask = n >> 31        // 正数: 0x00000000，负数: 0xFFFFFFFF
    return (n ^ mask) - mask  // 正数不变；负数取反+1
}
```

```objectivec
// Objective-C
- (int32_t)bitwiseAbs:(int32_t)n {
    int32_t mask = n >> 31;
    return (n ^ mask) - mask;
}
```

**原理**：
- 正数：`mask = 0`，`(n ^ 0) - 0 = n`
- 负数：`mask = -1`（全 1），`(n ^ (-1)) - (-1) = ~n + 1 = -n`

**注意**：`INT_MIN` 的绝对值溢出（`|-2^31| = 2^31` 超出 `Int32` 范围），需要特殊处理。

## 三、进阶技巧与算法

### 3.1 子集枚举（Subset Enumeration）

位运算最经典的应用之一：用一个 n 位整数表示 n 个元素的子集，第 i 位为 1 表示选择第 i 个元素。

#### 枚举全部子集

```swift
// Swift: 枚举 n 个元素的所有子集
func enumerateAllSubsets(_ n: Int) -> [[Int]] {
    var result: [[Int]] = []
    let total = 1 << n  // 2^n 个子集
    
    for mask in 0..<total {
        var subset: [Int] = []
        for i in 0..<n {
            if mask & (1 << i) != 0 {
                subset.append(i)
            }
        }
        result.append(subset)
    }
    return result
}
```

```objectivec
// Objective-C
- (NSArray<NSArray<NSNumber *> *> *)enumerateAllSubsets:(NSInteger)n {
    NSMutableArray *result = [NSMutableArray array];
    NSInteger total = 1 << n;
    for (NSInteger mask = 0; mask < total; mask++) {
        NSMutableArray *subset = [NSMutableArray array];
        for (NSInteger i = 0; i < n; i++) {
            if (mask & (1 << i)) {
                [subset addObject:@(i)];
            }
        }
        [result addObject:subset];
    }
    return result;
}
```

**复杂度**：时间 O(2^n × n)；空间 O(n)（不计结果）。

#### 枚举某个集合的所有子集

```swift
// Swift: 枚举 mask 的所有子集（包含自身和空集）
func enumerateSubsetsOf(_ mask: Int) -> [Int] {
    var subsets: [Int] = []
    var sub = mask
    while sub > 0 {
        subsets.append(sub)
        sub = (sub - 1) & mask  // 关键：减1后与mask取交，得到下一个子集
    }
    subsets.append(0)  // 空集
    return subsets
}
```

```objectivec
// Objective-C
- (NSArray<NSNumber *> *)enumerateSubsetsOf:(NSInteger)mask {
    NSMutableArray *subsets = [NSMutableArray array];
    NSInteger sub = mask;
    while (sub > 0) {
        [subsets addObject:@(sub)];
        sub = (sub - 1) & mask;
    }
    [subsets addObject:@0];
    return subsets;
}
```

**复杂度**：时间 O(2^k)，k 为 mask 中 1 的个数。这比枚举全部 2^n 后过滤高效得多。

**原理**：`(sub - 1) & mask` 的妙处：
1. `sub - 1` 将最低的 1 变成 0，右侧全变 1
2. `& mask` 保证结果仍是 mask 的子集

### 3.2 状态压缩动态规划（Bitmask DP）

用位掩码表示状态，是解决"集合选择"类 DP 的标准手段。

#### 经典例题：旅行商问题（TSP）

```swift
// Swift: TSP - 给定 n 个城市的距离矩阵，求经过所有城市恰好一次的最短路径
func tsp(_ dist: [[Int]]) -> Int {
    let n = dist.count
    let full = (1 << n) - 1  // 所有城市都访问过的状态
    // dp[mask][i] = 从城市0出发，访问了mask中的城市，最后在城市i的最短距离
    var dp = Array(repeating: Array(repeating: Int.max / 2, count: n), count: 1 << n)
    dp[1][0] = 0  // 起点在城市0，只访问了城市0
    
    for mask in 1..<(1 << n) {
        for last in 0..<n {
            guard mask & (1 << last) != 0 else { continue }  // last必须在mask中
            guard dp[mask][last] < Int.max / 2 else { continue }
            
            for next in 0..<n {
                guard mask & (1 << next) == 0 else { continue }  // next不在mask中
                let newMask = mask | (1 << next)
                dp[newMask][next] = min(dp[newMask][next], dp[mask][last] + dist[last][next])
            }
        }
    }
    
    // 从所有可能的终点返回起点
    var ans = Int.max
    for last in 1..<n {
        ans = min(ans, dp[full][last] + dist[last][0])
    }
    return ans
}
```

```objectivec
// Objective-C
- (NSInteger)tsp:(NSArray<NSArray<NSNumber *> *> *)dist {
    NSInteger n = dist.count;
    NSInteger full = (1 << n) - 1;
    NSInteger INF = NSIntegerMax / 2;
    
    // dp[mask][i]
    NSInteger **dp = (NSInteger **)malloc((1 << n) * sizeof(NSInteger *));
    for (NSInteger i = 0; i < (1 << n); i++) {
        dp[i] = (NSInteger *)malloc(n * sizeof(NSInteger));
        for (NSInteger j = 0; j < n; j++) dp[i][j] = INF;
    }
    dp[1][0] = 0;
    
    for (NSInteger mask = 1; mask <= full; mask++) {
        for (NSInteger last = 0; last < n; last++) {
            if (!(mask & (1 << last))) continue;
            if (dp[mask][last] >= INF) continue;
            for (NSInteger next = 0; next < n; next++) {
                if (mask & (1 << next)) continue;
                NSInteger newMask = mask | (1 << next);
                NSInteger cost = dp[mask][last] + dist[last][next].integerValue;
                if (cost < dp[newMask][next]) dp[newMask][next] = cost;
            }
        }
    }
    
    NSInteger ans = INF;
    for (NSInteger last = 1; last < n; last++) {
        NSInteger cost = dp[full][last] + dist[last][0].integerValue;
        if (cost < ans) ans = cost;
    }
    
    for (NSInteger i = 0; i < (1 << n); i++) free(dp[i]);
    free(dp);
    return ans;
}
```

**复杂度**：时间 O(2^n × n^2)；空间 O(2^n × n)。适用于 n ≤ 20。

### 3.3 位运算与异或的高级应用

#### 3.3.1 找出唯一出现一次的数（Single Number）

数组中每个数出现两次，只有一个出现一次：

```swift
// Swift: XOR 所有元素，成对的互相抵消，剩下唯一的
func singleNumber(_ nums: [Int]) -> Int {
    return nums.reduce(0, ^)
}
```

```objectivec
// Objective-C
- (NSInteger)singleNumber:(NSArray<NSNumber *> *)nums {
    NSInteger result = 0;
    for (NSNumber *num in nums) {
        result ^= num.integerValue;
    }
    return result;
}
```

**原理**：XOR 的性质：`a ^ a = 0`，`a ^ 0 = a`，满足交换律和结合律。

#### 3.3.2 找出两个只出现一次的数（Single Number III）

数组中有两个数各出现一次，其余出现两次：

```swift
// Swift
func singleNumberIII(_ nums: [Int]) -> [Int] {
    // 步骤1：全部异或，得到 a^b
    let xorAll = nums.reduce(0, ^)
    
    // 步骤2：找到 a^b 中任意一个为1的位（a和b在该位不同）
    let diffBit = xorAll & (-xorAll)  // 取最低位的1
    
    // 步骤3：按该位分组，分别异或
    var a = 0, b = 0
    for num in nums {
        if num & diffBit != 0 {
            a ^= num
        } else {
            b ^= num
        }
    }
    return [a, b]
}
```

```objectivec
// Objective-C
- (NSArray<NSNumber *> *)singleNumberIII:(NSArray<NSNumber *> *)nums {
    NSInteger xorAll = 0;
    for (NSNumber *num in nums) xorAll ^= num.integerValue;
    
    NSInteger diffBit = xorAll & (-xorAll);
    
    NSInteger a = 0, b = 0;
    for (NSNumber *num in nums) {
        if (num.integerValue & diffBit) {
            a ^= num.integerValue;
        } else {
            b ^= num.integerValue;
        }
    }
    return @[@(a), @(b)];
}
```

**复杂度**：时间 O(n)；空间 O(1)。

#### 3.3.3 找出出现一次的数（其余出现三次）

```swift
// Swift: 统计每一位上1的个数，模3后剩下的就是唯一数
func singleNumberII(_ nums: [Int]) -> Int {
    var result = 0
    for i in 0..<64 {
        var count = 0
        for num in nums {
            count += (num >> i) & 1
        }
        if count % 3 != 0 {
            result |= (1 << i)
        }
    }
    return result
}

// 更优雅的状态机方法（使用两个变量模拟三进制计数器）
func singleNumberII_StateMachine(_ nums: [Int]) -> Int {
    var ones = 0, twos = 0
    for num in nums {
        ones = (ones ^ num) & ~twos
        twos = (twos ^ num) & ~ones
    }
    return ones
}
```

```objectivec
// Objective-C: 状态机方法
- (NSInteger)singleNumberII:(NSArray<NSNumber *> *)nums {
    NSInteger ones = 0, twos = 0;
    for (NSNumber *num in nums) {
        NSInteger n = num.integerValue;
        ones = (ones ^ n) & ~twos;
        twos = (twos ^ n) & ~ones;
    }
    return ones;
}
```

**状态机原理**：
- `ones`/`twos` 模拟每一位的三进制计数器
- 对于每一位：00 → 01 → 10 → 00（出现 0/1/2/3 次）
- `ones` 保存出现 1 次(mod 3) 的位，`twos` 保存出现 2 次(mod 3) 的位
- 最终 `ones` 就是只出现 1 次的数

### 3.4 位运算实现有限状态自动机

上面 Single Number II 的状态机方法可以推广。对于"其余出现 k 次，一个出现 p 次"的一般情况：

```swift
// Swift: 一般化 - 数组中一个数出现 p 次，其余出现 k 次，找出该数
// 要求: p % k != 0
func singleNumberGeneral(_ nums: [Int], k: Int, p: Int) -> Int {
    var result = 0
    for i in 0..<64 {
        var count = 0
        for num in nums {
            count += (num >> i) & 1
        }
        if count % k != 0 {
            result |= (1 << i)
        }
    }
    return result
}
```

### 3.5 格雷码（Gray Code）

格雷码是一种编码方式，相邻两个码字只有一位不同，广泛用于旋转编码器、纠错码等。

```swift
    var n = gray
    var mask = n >> 1
    while mask != 0 {
        n ^= mask
        mask >>= 1
    }
    return n
}

// 生成 n 位格雷码序列
func grayCode(_ n: Int) -> [Int] {
```

### 3.6 位反转（Reverse Bits）

```swift
// Swift: 反转 32 位整数的位
func reverseBits(_ n: UInt32) -> UInt32 {
    var x = n
    // 分治法：逐步交换相邻的 1位、2位、4位、8位、16位
    x = ((x & 0x55555555) << 1) | ((x >> 1) & 0x55555555)   // 交换相邻1位
    x = ((x & 0x33333333) << 2) | ((x >> 2) & 0x33333333)   // 交换相邻2位
    x = ((x & 0x0F0F0F0F) << 4) | ((x >> 4) & 0x0F0F0F0F)   // 交换相邻4位
    x = ((x & 0x00FF00FF) << 8) | ((x >> 8) & 0x00FF00FF)   // 交换相邻8位
    x = (x << 16) | (x >> 16)                                // 交换前后16位
    return x
}
```

### 3.7 下一个排列（Gosper's Hack）

给定一个有 k 个 1 的 n 位整数，找到下一个同样有 k 个 1 但数值更大的整数：

```swift
// Swift: Gosper's Hack - 找到下一个有相同数量1的更大整数
func nextPermutation(_ v: Int) -> Int {
    let t = v | (v - 1)        // 将最低的连续1的右侧填满1
    // 下一个排列 = t+1 加上原来需要移到最右边的1的数量
    let nextV = (t + 1) | (((~t & (t + 1)) - 1) >> (trailingZeros(v) + 1))
    return nextV
}

// 辅助函数
private func trailingZeros(_ n: Int) -> Int {
    if n == 0 { return 64 }
    var count = 0
    var x = n
    while x & 1 == 0 {
        count += 1
        x >>= 1
    }
    return count
}

// 更直观的版本
func nextSamePopcount(_ x: Int) -> Int {
    let lowest = x & (-x)         // 最低位的1
    let ripple = x + lowest       // 进位传播
    let ones = x ^ ripple         // 被进位消除的1
    let shifted = (ones >> 2) / lowest  // 右移到最低位
    return ripple | shifted
}
```

```objectivec
// Objective-C
- (NSInteger)nextSamePopcount:(NSInteger)x {
    NSInteger lowest = x & (-x);
    NSInteger ripple = x + lowest;
    NSInteger ones = x ^ ripple;
    NSInteger shifted = (ones >> 2) / lowest;
    return ripple | shifted;
}
```

**应用**：枚举所有 C(n, k) 的组合——从 `(1<<k)-1` 开始，反复调用 `nextSamePopcount` 直到越界。

### 3.8 位运算实现快速幂

```swift
// Swift: 快速幂 - 计算 base^exp % mod
func fastPow(_ base: Int, _ exp: Int, _ mod: Int) -> Int {
    var result = 1
    var b = base % mod
    var e = exp
    
    while e > 0 {
        if e & 1 == 1 {       // 当前位为1，乘入结果
            result = result * b % mod
        }
        b = b * b % mod       // 底数平方
        e >>= 1              // 右移一位
    }
    return result
}
```

```objectivec
// Objective-C
- (NSInteger)fastPow:(NSInteger)base exp:(NSInteger)exp mod:(NSInteger)mod {
    NSInteger result = 1;
    NSInteger b = base % mod;
    while (exp > 0) {
        if (exp & 1) result = result * b % mod;
        b = b * b % mod;
        exp >>= 1;
    }
    return result;
}
```

**复杂度**：时间 O(log n)；空间 O(1)。

## 四、树状数组（Binary Indexed Tree / Fenwick Tree）

树状数组是位运算（lowbit）最经典的数据结构应用。

### 4.1 原理

树状数组利用 `lowbit(i) = i & (-i)` 将数组分层管理：
- `tree[i]` 管理从 `i - lowbit(i) + 1` 到 `i` 的区间和
- 更新时沿 `i += lowbit(i)` 向上传播
- 查询时沿 `i -= lowbit(i)` 向下累加

```
索引:     1    2    3    4    5    6    7    8
lowbit:   1    2    1    4    1    2    1    8
管理区间: [1,1] [1,2] [3,3] [1,4] [5,5] [5,6] [7,7] [1,8]
```

### 4.2 完整实现

```swift
// Swift: 树状数组（1-indexed）
class BinaryIndexedTree {
    private var tree: [Int]
    private let n: Int
    
    init(_ size: Int) {
        n = size
        tree = Array(repeating: 0, count: size + 1)
    }
    
    // 单点更新：将位置 i 增加 delta
    func update(_ i: Int, _ delta: Int) {
        var idx = i
        while idx <= n {
            tree[idx] += delta
            idx += idx & (-idx)  // idx += lowbit(idx)
        }
    }
    
    // 前缀查询：返回 [1, i] 的区间和
    func query(_ i: Int) -> Int {
        var sum = 0
        var idx = i
        while idx > 0 {
            sum += tree[idx]
            idx -= idx & (-idx)  // idx -= lowbit(idx)
        }
        return sum
    }
    
    // 区间查询：返回 [l, r] 的区间和
    func rangeQuery(_ l: Int, _ r: Int) -> Int {
        return query(r) - query(l - 1)
    }
}
```

```objectivec
// Objective-C
@interface BinaryIndexedTree : NSObject
- (instancetype)initWithSize:(NSInteger)size;
- (void)update:(NSInteger)i delta:(NSInteger)delta;
- (NSInteger)query:(NSInteger)i;
- (NSInteger)rangeQueryFrom:(NSInteger)l to:(NSInteger)r;
@end

@implementation BinaryIndexedTree {
    NSInteger *_tree;
    NSInteger _n;
}

- (instancetype)initWithSize:(NSInteger)size {
    self = [super init];
    if (self) {
        _n = size;
        _tree = (NSInteger *)calloc(size + 1, sizeof(NSInteger));
    }
    return self;
}

- (void)dealloc {
    free(_tree);
}

- (void)update:(NSInteger)i delta:(NSInteger)delta {
    while (i <= _n) {
        _tree[i] += delta;
        i += i & (-i);
    }
}

- (NSInteger)query:(NSInteger)i {
    NSInteger sum = 0;
    while (i > 0) {
        sum += _tree[i];
        i -= i & (-i);
    }
    return sum;
}

- (NSInteger)rangeQueryFrom:(NSInteger)l to:(NSInteger)r {
    return [self query:r] - [self query:l - 1];
}

@end
```

**复杂度**：更新 O(logN)，查询 O(logN)，空间 O(N)。

### 4.3 应用：逆序对计数

```swift
// Swift: 用树状数组计算逆序对数量
func countInversions(_ nums: [Int]) -> Int {
    let n = nums.count
    // 离散化
    let sorted = Array(Set(nums)).sorted()
    var rank: [Int: Int] = [:]
    for (i, val) in sorted.enumerated() {
        rank[val] = i + 1
    }
    
    let bit = BinaryIndexedTree(sorted.count)
    var inversions = 0
    
    for num in nums {
        let r = rank[num]!
        // 已插入的数中，比当前数大的个数 = 已插入总数 - 不大于当前数的个数
        inversions += bit.query(sorted.count) - bit.query(r)
        bit.update(r, 1)
    }
    return inversions
}
```

## 六、位运算优化技巧

### 6.1 用移位替代乘除（编译器通常会自动做）

```swift
// 乘以2的幂 → 左移
let a = n << 1   // n * 2
let b = n << 3   // n * 8

// 除以2的幂 → 右移（仅限正数或无符号数）
let c = n >> 1   // n / 2
let d = n >> 4   // n / 16

// 取模2的幂 → 按位与
let e = n & 0x0F  // n % 16
let f = n & 0x3F  // n % 64

// 判断奇偶
let isOdd = (n & 1) == 1
```

**注意**：现代编译器会自动进行这些优化（strength reduction），手动写反而可能降低可读性。建议只在性能关键路径且明确编译器未优化时使用。

### 6.2 用位运算实现 min/max（无分支）

```swift
// Swift: 无分支 min（32位有符号）
func bitwiseMin(_ x: Int32, _ y: Int32) -> Int32 {
    let diff = x - y
    let sign = diff >> 31  // x < y 时为 -1（全1），否则为 0
    return y + (diff & sign)  // x < y ? x : y
}

// 无分支 max
func bitwiseMax(_ x: Int32, _ y: Int32) -> Int32 {
    let diff = x - y
    let sign = diff >> 31
    return x - (diff & sign)
}
```

**注意**：当 x - y 溢出时此方法不正确！实践中大多数情况下 `min()`/`max()` 即可。

### 6.3 条件赋值（无分支）

```swift
// Swift: 根据条件选择 a 或 b（condition 为 true/false 对应 -1/0）
func conditionalSelect(_ a: Int, _ b: Int, _ condition: Bool) -> Int {
    let mask = condition ? -1 : 0  // true: 全1；false: 全0
    return (a & mask) | (b & ~mask)
}
```

在 GPU 编程和 SIMD 中，无分支选择非常重要（分支预测失败代价高）。

### 6.4 位运算求平均值（避免溢出）

```swift
// Swift: 不溢出的平均值
```

### 7.3 LeetCode 338. Counting Bits（比特位计数）

**题意**：对 [0, n] 中的每个数，计算其二进制中 1 的个数。

```swift
// Swift: DP + 位运算，O(n) 时间
func countBits(_ n: Int) -> [Int] {
    var dp = Array(repeating: 0, count: n + 1)
    for i in 1...n {
        dp[i] = dp[i & (i - 1)] + 1  // i去掉最低位1后的结果 + 1
    }
    return dp
}
```

```objectivec
// Objective-C
- (NSArray<NSNumber *> *)countBits:(NSInteger)n {
    NSMutableArray *dp = [NSMutableArray arrayWithCapacity:n + 1];
    [dp addObject:@0];
    for (NSInteger i = 1; i <= n; i++) {
        NSInteger count = [dp[i & (i - 1)] integerValue] + 1;
        [dp addObject:@(count)];
    }
    return dp;
}
```

**DP 转移**：`dp[i] = dp[i & (i-1)] + 1`，因为 `i & (i-1)` 是 `i` 去掉最低位 1 后的数，其 popcount 已知。

### 7.4 LeetCode 268. Missing Number（丢失的数字）

**题意**：数组包含 0 到 n 中 n 个不同的数，找缺失的那个。

```swift
func missingNumber(_ nums: [Int]) -> Int {
    var result = nums.count  // 初始为 n（因为索引是 0 到 n-1）
    for i in 0..<nums.count {
        result ^= i ^ nums[i]
    }
    return result
}
```


### 7.7 LeetCode 78. Subsets（子集）

**题意**：给定无重复元素的数组，返回所有子集。

```swift
func subsets(_ nums: [Int]) -> [[Int]] {
    let n = nums.count
    var result: [[Int]] = []
    
    for mask in 0..<(1 << n) {
        var subset: [Int] = []
        for i in 0..<n {
            if mask & (1 << i) != 0 {
                subset.append(nums[i])
            }
        }
        result.append(subset)
    }
    return result
}
```

### 7.8 LeetCode 201. Bitwise AND of Numbers Range（数字范围按位与）

```swift

func rangeBitwiseAnd(_ m: Int, _ n: Int) -> Int {
    var left = m, right = n
    var shift = 0
    while left < right {
        left >>= 1
        right >>= 1
        shift += 1
    }
    return left << shift
}
```

```objectivec
// Objective-C

    NSInteger shift = 0;

        m >>= 1;
        n >>= 1;
        shift++;
    }
    return m << shift;
}
```

### 7.10 LeetCode 318. Maximum Product of Word Lengths（最大单词长度乘积）

**题意**：找出两个不含公共字母的单词，使它们长度之积最大。

```swift
// Swift: 用 26 位掩码表示每个单词包含的字母
func maxProduct(_ words: [String]) -> Int {
    // 预计算每个单词的位掩码
    let masks = words.map { word -> Int in
        var mask = 0
        for ch in word {
            mask |= 1 << (Int(ch.asciiValue! - Character("a").asciiValue!))
        }
        return mask
    }
    
    var maxProd = 0
    for i in 0..<words.count {
        for j in (i + 1)..<words.count {
            if masks[i] & masks[j] == 0 {  // 无公共字母
                maxProd = max(maxProd, words[i].count * words[j].count)
            }
        }
    }
    return maxProd
}
```

```objectivec
// Objective-C
- (NSInteger)maxProduct:(NSArray<NSString *> *)words {
    NSInteger n = words.count;
    NSInteger *masks = (NSInteger *)calloc(n, sizeof(NSInteger));
    
    for (NSInteger i = 0; i < n; i++) {
        NSString *word = words[i];
        for (NSInteger j = 0; j < word.length; j++) {
            unichar ch = [word characterAtIndex:j];
            masks[i] |= 1 << (ch - 'a');
        }
    }
    
    NSInteger maxProd = 0;
    for (NSInteger i = 0; i < n; i++) {
        for (NSInteger j = i + 1; j < n; j++) {
            if ((masks[i] & masks[j]) == 0) {
                NSInteger prod = words[i].length * words[j].length;
                if (prod > maxProd) maxProd = prod;
            }
        }
    }
    free(masks);
    return maxProd;
}
```

**复杂度**：预处理 O(L)（L 为所有字符总数），比较 O(n^2) 但每次比较是 O(1)。

### 8.1 快速求 log2（向下取整）

```swift
// Swift: 求 floor(log2(n))，即最高位1的位置
func floorLog2(_ n: Int) -> Int {
    guard n > 0 else { return -1 }
    var result = 0
    var x = n
    // 二分逼近
    if x >= (1 << 32) { result += 32; x >>= 32 }
    if x >= (1 << 16) { result += 16; x >>= 16 }
    if x >= (1 << 8)  { result += 8;  x >>= 8 }
    if x >= (1 << 4)  { result += 4;  x >>= 4 }
    if x >= (1 << 2)  { result += 2;  x >>= 2 }
    if x >= (1 << 1)  { result += 1 }
    return result
}
```

### 8.2 向上取到最近的 2 的幂

```swift
// Swift: 返回 >= n 的最小的 2 的幂
func nextPowerOf2(_ n: Int) -> Int {
    guard n > 0 else { return 1 }
    var x = n - 1
    x |= x >> 1
    x |= x >> 2
    x |= x >> 4
    x |= x >> 8
    x |= x >> 16
    x |= x >> 32
    return x + 1
}
```


### 8.3 判断是否是 4 的幂

```swift
// Swift
func isPowerOfFour(_ n: Int) -> Bool {

    // 条件2：这个1在奇数位（0, 2, 4, 6...位）

}
```

### 9.1 简易哈希函数（FNV-1a）

```swift
// Swift: FNV-1a 哈希（位运算核心）
func fnv1a(_ data: [UInt8]) -> UInt64 {
    var hash: UInt64 = 14695981039346656037  // FNV offset basis
    let prime: UInt64 = 1099511628211         // FNV prime
    
    for byte in data {
        hash ^= UInt64(byte)    // XOR
        hash &*= prime          // 乘以素数（允许溢出）
    }
    return hash
}
```

### 9.2 布隆过滤器（Bloom Filter）

布隆过滤器使用多个哈希函数和位数组实现高效集合判重：

```swift
// Swift: 简易布隆过滤器
class BloomFilter {
    private var bits: [UInt64]
    private let size: Int        // 位数
    private let hashCount: Int   // 哈希函数数量
    
    init(size: Int, hashCount: Int) {
        self.size = size
        self.hashCount = hashCount
        self.bits = Array(repeating: 0, count: (size + 63) / 64)
    }
    
    private func setBit(_ pos: Int) {
        bits[pos / 64] |= (1 << (pos % 64))
    }
    
    private func getBit(_ pos: Int) -> Bool {
        return bits[pos / 64] & (1 << (pos % 64)) != 0
    }
    
    private func hashes(_ element: String) -> [Int] {
        var result: [Int] = []
        for i in 0..<hashCount {
            var h = element.hashValue &+ i &* 0x9e3779b9
            h = ((h >> 16) ^ h) &* 0x45d9f3b
            h = ((h >> 16) ^ h)
            result.append(abs(h) % size)
        }
        return result
    }
    
    func insert(_ element: String) {
        for pos in hashes(element) {
            setBit(pos)
        }
    }
    
    func mightContain(_ element: String) -> Bool {
        for pos in hashes(element) {
            if !getBit(pos) { return false }
        }
        return true  // 可能存在（有假阳性）
    }
}
```

```objectivec
// Objective-C
@interface BloomFilter : NSObject
- (instancetype)initWithSize:(NSInteger)size hashCount:(NSInteger)hashCount;
- (void)insert:(NSString *)element;
- (BOOL)mightContain:(NSString *)element;
@end

@implementation BloomFilter {
    uint64_t *_bits;
    NSInteger _size;
    NSInteger _hashCount;
}

- (instancetype)initWithSize:(NSInteger)size hashCount:(NSInteger)hashCount {
    self = [super init];
    if (self) {
        _size = size;
        _hashCount = hashCount;
        NSInteger arraySize = (size + 63) / 64;
        _bits = (uint64_t *)calloc(arraySize, sizeof(uint64_t));
    }
    return self;
}

- (void)dealloc {
    free(_bits);
}

- (void)setBitAt:(NSInteger)pos {
    _bits[pos / 64] |= (1ULL << (pos % 64));
}

- (BOOL)getBitAt:(NSInteger)pos {
    return (_bits[pos / 64] & (1ULL << (pos % 64))) != 0;
}

- (NSArray<NSNumber *> *)hashesFor:(NSString *)element {
    NSMutableArray *result = [NSMutableArray array];
    NSUInteger base = element.hash;
    for (NSInteger i = 0; i < _hashCount; i++) {
        NSUInteger h = base + i * 0x9e3779b9;
        h = ((h >> 16) ^ h) * 0x45d9f3b;
        h = ((h >> 16) ^ h);
        [result addObject:@(h % _size)];
    }
    return result;
}

- (void)insert:(NSString *)element {
    for (NSNumber *pos in [self hashesFor:element]) {
        [self setBitAt:pos.integerValue];
    }
}

- (BOOL)mightContain:(NSString *)element {
    for (NSNumber *pos in [self hashesFor:element]) {
        if (![self getBitAt:pos.integerValue]) return NO;
    }
    return YES;
}

@end
```

```swift
func crc32(_ data: [UInt8]) -> UInt32 {
    var crc: UInt32 = 0xFFFFFFFF
    
    for byte in data {
        crc ^= UInt32(byte)
        for _ in 0..<8 {
            if crc & 1 != 0 {

        }
```

### 10.1 SIMD 基础概念

SIMD（Single Instruction, Multiple Data）指单条指令同时操作多个数据。位运算天然支持并行——对 64 位整数做一次 AND 操作，等效于同时对 64 个布尔值做 AND。

### 10.2 位并行算法：同时检查 8 字节中是否包含零字节

```swift
// Swift: 高效检测一个 UInt64 中是否包含零字节
// 用于 strlen 等函数的 SIMD 优化
func hasZeroByte(_ v: UInt64) -> Bool {
    // 经典的 SWAR (SIMD Within A Register) 技巧
    let mask: UInt64 = 0x7F7F7F7F7F7F7F7F
    return ((v &- 0x0101010101010101) & ~v & 0x8080808080808080) != 0
}
```

**原理**：libc 中的 `strlen` 并不是逐字节比较，而是每次加载 8 字节用上述技巧并行检测。

### 10.3 位板（Bitboard）在棋类游戏中的应用

国际象棋引擎使用 64 位整数表示棋盘（8×8=64 格），一个 bit 代表一格：

```swift
// Swift: 象棋位板基本操作
typealias Bitboard = UInt64

struct ChessBoard {
    var whitePawns: Bitboard = 0x000000000000FF00    // 第2行
    var blackPawns: Bitboard = 0x00FF000000000000    // 第7行
    
    // 白方兵向前推一步
    func whitePawnPush() -> Bitboard {
        let emptySquares: Bitboard = ~(whitePawns | blackPawns)
        return (whitePawns << 8) & emptySquares
    }
    
    // 检查某个位置是否被占据
    func isOccupied(_ square: Int) -> Bool {
        let mask: Bitboard = 1 << square
        return (whitePawns | blackPawns) & mask != 0
    }
    
    // 计算所有棋子数量
    func pieceCount(_ board: Bitboard) -> Int {
        var count = 0
        var b = board
        while b != 0 {
            b &= (b - 1)
            count += 1
        }
        return count
    }
}
```

## 十三、易错点与注意事项

### 13.1 运算符优先级陷阱

位运算符优先级**低于**比较运算符，这是最常见的 bug 来源：

```swift
// 错误！== 优先级高于 &
if n & 1 == 1 { ... }       // 实际执行: n & (1 == 1) = n & 1
// 正确
if (n & 1) == 1 { ... }

// 错误！
if mask & flag != 0 { ... }  // 实际执行: mask & (flag != 0)
// 正确
if (mask & flag) != 0 { ... }
```

**优先级顺序**（从高到低）：`~` > `<< >>` > `&` > `^` > `|` > `== !=` > `&&` > `||`

建议：位运算**总是加括号**，不依赖优先级记忆。

### 13.2 有符号数右移的陷阱

```swift
// Swift 中 Int 是有符号的，右移保留符号位
let x: Int = -1
let y = x >> 1   // 仍然是 -1（算术右移，高位补1）

// 如果需要逻辑右移，转为无符号数
let z = Int(bitPattern: UInt(bitPattern: x) >> 1)  // 为 Int.max / 2
```

### 13.3 左移溢出

```swift
// Swift 中左移超过位宽是未定义行为（C）或截断（Swift）
let a: Int32 = 1
let b = a << 32   // Swift: 结果为 0（shift amount >= type width）
                   // C: 未定义行为！

// 安全做法：确保移位量在 [0, bitWidth-1] 范围内
```

### 13.4 负数取模的位运算替代

```swift
// n % (2^k) 对负数结果不同于 n & ((1<<k) - 1)
let n = -7
```

## 十四、最佳实践

### 14.1 何时使用位运算

**适合使用**：
- 需要极致性能的热路径（如像素处理、网络包解析）
- 表示集合/状态（权限、标志位、子集）
- 算法竞赛中的巧妙技巧
- 系统级编程（内存管理、对齐、协议解析）
- 密码学、哈希、校验码

**不适合使用**：
- 普通业务逻辑（可读性 > 微优化）
- 编译器已能自动优化的场景（如 `n * 2` → `n << 1`）
- 涉及负数且不确定行为的场景

### 14.2 可读性优化建议

```swift
// 差：难以理解
```

```swift
func safeBitOp(_ n: Int, _ i: Int) -> Int? {
    guard i >= 0 && i < Int.bitWidth else { return nil }
    return (n >> i) & 1
}

// 避免整数溢出
func safeLeftShift(_ n: Int, _ i: Int) -> Int? {
    guard i >= 0 && i < Int.bitWidth else { return nil }
    guard n >= 0 else { return nil }  // 负数左移行为依赖实现
    guard n <= (Int.max >> i) else { return nil }  // 会溢出
    return n << i
}
```

## 十六、面试高频问答

### Q1: 为什么 `n & (n-1)` 能消除最低位的 1？

**答**：减 1 操作会向最低位的 1 借位，使该位变 0，右侧所有位变 1。与原数 AND 后，该位及右侧全为 0，左侧保持不变，从而精确消除了最低的 1。

### Q2: 如何不用额外空间找出数组中唯一出现奇数次的数？

**答**：全部异或。XOR 满足交换律、结合律，且 a⊕a=0、a⊕0=a。出现偶数次的数相互抵消为 0，剩下奇数次的数。

### Q3: 状态压缩DP中为什么 n 不能太大？

**答**：状态数为 2^n，当 n=20 时有约 100 万个状态，n=25 就超过 3000 万，内存和时间都不可接受。一般 n ≤ 20（配合 O(n^2) 转移）或 n ≤ 25（配合 O(n) 转移）是上限。

### Q4: popcount 有哪些实现方式？各自复杂度？

**答**：
- 逐位检测：O(w)，w 为位宽
- Brian Kernighan：O(k)，k 为 1 的个数
- 分治法：O(1)，固定步数
- 查表法：O(1)，预计算 256 项表
- CPU 指令：O(1)，如 x86 POPCNT、ARM CNT

### Q5: 树状数组为什么用 lowbit？

**答**：lowbit(i) = i & (-i) 决定了节点 i 管理的区间长度。更新时 i += lowbit(i) 跳到父节点，查询时 i -= lowbit(i) 跳到前一个不重叠区间。这种结构保证更新和查询都是 O(logN) 且常数极小。

### Q6: 位运算加法为什么一定会终止？

**答**：每次循环中，carry = a & b 的最低位 1 一定比原来高至少 1 位（因为左移了 1 位）。n 位整数最多产生 n 次进位，所以循环一定终止。

### Q7: 什么情况下 XOR 交换会出错？

**答**：当两个操作数是同一个内存地址的引用时（如 `swap(arr[i], arr[i])` 且 i==j），第一步 a ^= a 就会把值变为 0，后续无法恢复。

### Q8: 如何用位运算判断一个数是否是 2 的幂？为什么还要判断 > 0？

**答**：`n & (n-1) == 0` 说明只有一个 1，但 n=0 时 `0 & (-1) = 0` 也满足条件。0 不是任何正整数的幂，所以必须加 `n > 0`。

```swift
// Swift: 位图操作 - 对每个像素应用阈值二值化
func binarize(_ pixels: inout [UInt32], threshold: UInt8) {
    for i in 0..<pixels.count {
        let pixel = pixels[i]
        // 提取 RGB 分量并计算亮度
        let r = (pixel >> 16) & 0xFF
        let g = (pixel >> 8) & 0xFF
        let b = pixel & 0xFF
        let luminance = (r * 77 + g * 150 + b * 29) >> 8  // 近似 0.3R+0.59G+0.11B
        
        // 根据阈值设为黑/白
        let value: UInt32 = luminance >= UInt32(threshold) ? 0xFFFFFFFF : 0xFF000000
        pixels[i] = value
    }
}
```

### 17.5 内存池中的伙伴系统（Buddy System）

```swift
// Swift: 伙伴系统使用位运算确定块的伙伴地址和块大小
class BuddyAllocator {
    private let totalSize: Int
    private let minBlockSize: Int
    private var freeList: [Set<Int>]  // freeList[order] = 该层级的空闲块起始地址集合
    
    init(totalSize: Int, minBlockSize: Int) {
        self.totalSize = totalSize
        self.minBlockSize = minBlockSize
        let levels = floorLog2(totalSize / minBlockSize) + 1
        self.freeList = Array(repeating: Set<Int>(), count: levels)
        freeList[levels - 1].insert(0)  // 初始一个最大块
    }
    
    // 找伙伴地址：翻转对应层级的位
    func buddyOf(address: Int, order: Int) -> Int {
        return address ^ (minBlockSize << order)
    }
    
    // 块大小
    func blockSize(order: Int) -> Int {
        return minBlockSize << order
    }
    
    private func floorLog2(_ n: Int) -> Int {
        var result = 0
        var x = n
        while x > 1 { x >>= 1; result += 1 }
        return result
    }
}
```
