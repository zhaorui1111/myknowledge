# 国际化与本地化详解

## 一、国际化与本地化的概念

国际化（Internationalization，缩写 i18n）是指设计和开发应用时，使其能够适应不同语言和地区而不需要修改代码。本地化（Localization，缩写 l10n）是指针对特定语言和地区，将应用的内容翻译和适配为本地用户习惯的版本。

iOS 中国际化的核心机制是：根据用户的语言和区域设置，系统自动加载对应的资源文件。开发者只需要提供多语言资源，框架负责选择正确的版本。

## 二、字符串本地化

### 2.1 Localizable Strings 文件

在 Xcode 中创建 `.strings` 文件，或使用 String Catalog（Xcode 15+）。传统 `.strings` 文件格式为键值对：

```
// Localizable.strings (English)
"welcome_message" = "Welcome to MyApp";
"login_button" = "Log In";
"greeting" = "Hello, %@!";

// Localizable.strings (Simplified Chinese)
"welcome_message" = "欢迎来到 MyApp";
"login_button" = "登录";
"greeting" = "你好，%@！";
```

### 2.2 代码中使用本地化字符串

```swift
// 基本用法
let title = NSLocalizedString("welcome_message", comment: "Welcome screen title")

// 带格式参数
let name = "张三"
let greeting = String(format: NSLocalizedString("greeting", comment: ""), name)

// Swift 5.1+ 推荐用法（更安全）
let title = String(localized: "welcome_message")
let greeting = String(localized: "Hello, \(name)!")  // 插值本地化
```

### 2.3 String Catalog（Xcode 15+）

Xcode 15 引入了 String Catalog（`.xcstrings`），取代传统的 `.strings` 文件。它是一个 JSON 格式的文件，Xcode 提供可视化编辑界面：

- 自动扫描代码中的字符串并提取到 Catalog
- 支持复数、变体（设备、布局方向）的图形化配置
- 自动检测缺失的翻译
- 支持导出/导入翻译文件（XLIFF）

### 2.4 Storyboard/XIB 本地化

在 Xcode 的 File Inspector 中，将 Storyboard 的本地化勾选多种语言。Xcode 会生成 Storyboard 对应的 `.strings` 文件，翻译其中的文本属性。

## 三、复数处理

不同语言对复数的规则不同。英语区分单数（one）和复数（other），阿拉伯语有六种复数形式，中文不区分复数。

### 3.1 stringsdict 文件

传统方式使用 `.stringsdict` 文件处理复数：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>%d items</key>
    <dict>
        <key>NSStringLocalizedFormatKey</key>
        <string>%#@items@</string>
        <key>items</key>
        <dict>
            <key>NSStringFormatSpecTypeKey</key>
            <string>NSStringPluralRuleType</string>
            <key>NSStringFormatValueTypeKey</key>
            <string>d</string>
            <key>one</key>
            <string>%d item</string>
            <key>other</key>
            <string>%d items</string>
        </dict>
    </dict>
</dict>
</plist>
```

### 3.2 代码中使用复数

```swift
let count = 3
let text = String(format: NSLocalizedString("%d items", comment: ""), count)
// 英语: "3 items"
// 中文: "3 个项目"（如果配置了中文翻译）
```

### 3.3 String Catalog 中的复数

在 String Catalog 中，选中一个字符串，在右侧 Inspector 中勾选 "Plural"，即可图形化编辑各复数形式的翻译，无需手动编写 stringsdict XML。

## 四、RTL（从右到左）布局

### 4.1 RTL 概述

阿拉伯语、希伯来语等语言从右到左书写。iOS 系统在 RTL 语言下会自动镜像大部分 UI 元素的布局方向。

### 4.2 自动镜像

以下元素在 RTL 下自动镜像：`leading`/`trailing` 约束（等同 `left`/`right` 但方向自适应）、`UIButton` 的 image 和 title 位置、`UISwitch` 方向、导航返回箭头方向、`UIImageView` 的 `left`/`right` 内容模式。

```swift
// 使用 leading/trailing 而非 left/right
NSLayoutConstraint(item: label, attribute: .leading, relatedBy: .equal, toItem: container, attribute: .leadingMargin, multiplier: 1, constant: 16)
```

### 4.3 代码中判断 RTL

```swift
if UIView.userInterfaceLayoutDirection(for: view.semanticContentAttribute) == .rightToLeft {
    // RTL 布局
}

// 或
let isRTL = UIApplication.shared.userInterfaceLayoutDirection == .rightToLeft
```

### 4.4 RTL 布局注意事项

`left`/`right` 约束不会自动镜像。如果必须使用 `left`/`right`，需要根据 `userInterfaceLayoutDirection` 手动翻转。

图片方向：带有方向性的图标（如返回箭头）需要提供镜像版本。通过 `UIImage(flippedForRightToLeftLayoutDirection:)` 自动翻转：

```swift
let arrowImage = UIImage(systemName: "chevron.right")?.imageFlippedForRightToLeftLayoutDirection()
```

文本对齐：使用 `NSTextAlignment.natural` 让系统根据语言自动选择对齐方向，而非 `.left` 或 `.right`。

数字和日期保持 LTR：即使 RTL 布局，数字、日期、时间等应保持从左到右显示。使用 `UILabel` 时设置 `semanticContentAttribute = .forceLeftToRight`。

## 五、日期、数字、货币格式化

### 5.1 日期格式化

```swift
let formatter = DateFormatter()
formatter.dateStyle = .medium
formatter.timeStyle = .short
formatter.locale = Locale.current

let dateString = formatter.string(from: Date())
// 英语 (US): "Jan 15, 2024 at 3:30 PM"
// 中文 (CN): "2024年1月15日 下午3:30"
// 阿拉伯语: "١٥ يناير ٢٠٢٤ ٣:٣٠ م"
```

### 5.2 相对日期

```swift
let relativeFormatter = RelativeDateTimeFormatter()
relativeFormatter.unitsStyle = .short
let relativeString = relativeFormatter.localizedString(for: someDate, relativeTo: Date())
// "3 hours ago" / "3小时前"
```

### 5.3 数字与货币

```swift
let numberFormatter = NumberFormatter()
numberFormatter.numberStyle = .decimal
numberFormatter.locale = Locale.current
let numberString = numberFormatter.string(from: 1234567)!
// 英语: "1,234,567"
// 德语: "1.234.567"
// 阿拉伯语: "١٬٢٣٤٬٥٦٧"

let currencyFormatter = NumberFormatter()
currencyFormatter.numberStyle = .currency
currencyFormatter.locale = Locale(identifier: "zh_CN")
let currencyString = currencyFormatter.string(from: 99.99)!
// "¥99.99"
```

### 5.4 测量单位

```swift
let measurementFormatter = MeasurementFormatter()
let distance = Measurement(value: 5, unit: UnitLength.kilometers)
let distanceString = measurementFormatter.string(from: distance)
// 英语 (US): "3.107 mi"
// 中文: "5 km"
```

## 六、资源本地化

### 6.1 图片资源

对含有文字的图片，需要为每种语言提供版本。在 Xcode 的 Asset Catalog 中勾选本地化语言，然后提供各语言的图片文件。

不含文字的图片不需要本地化。

### 6.2 音频与视频

音频文件中的语音需要录制多语言版本。通过命名约定（如 `intro_en.mp4`、`intro_zh.mp4`）或在 Asset Catalog 中本地化。

## 七、伪本地化测试

### 7.1 什么是伪本地化

伪本地化（Pseudo-Localization）是将源字符串自动变换为"假翻译"——添加重音字符、加长字符串、翻转方向——以在不实际翻译的情况下检测 UI 的国际化问题。

### 7.2 Scheme 配置

在 Xcode 的 Scheme 中，勾选 "Show non-localized strings" 和 "Pseudolocalization"：

- **Accented Pseudolanguage**：字符替换为带重音的变体
- **Bounded Pseudolanguage**：用 `#` 包围字符串检测截断
- **Right-to-Left Pseudolanguage**：模拟 RTL 布局

### 7.3 常见问题检测

文本截断：德语翻译通常比英语长 30-50%，伪本地化可以提前发现布局空间不足。

硬编码字符串：未使用 NSLocalizedString 的字符串在伪本地化模式下不会被变换，方便检测。

拼接问题：字符串拼接在多语言下可能语法不通。应使用格式化占位符而非字符串拼接。

## 八、App 语言切换

### 8.1 系统级语言切换

iOS 13+ 支持在设置中为每个 App 单独设置语言。Xcode 会自动在项目的 Localizations 中配置支持的语言。

### 8.2 应用内语言切换

如果需要在 App 内部切换语言（不依赖系统设置），需要手动管理：

```swift
class LanguageManager {
    static let shared = LanguageManager()

    func setLanguage(_ language: String) {
        UserDefaults.standard.set([language], forKey: "AppleLanguages")

        // 重新设置 Bundle，使其加载指定语言的资源
        let path = Bundle.main.path(forResource: language, ofType: "lproj")!
        self.bundle = Bundle(path: path)

        // 发送语言变更通知，刷新 UI
        NotificationCenter.default.post(name: .languageDidChange, object: nil)
    }

    func localizedString(_ key: String) -> String {
        return bundle.localizedString(forKey: key, value: nil, table: nil)
    }
}
```

### 8.3 SwiftUI 语言切换

SwiftUI 中可以利用 `@Environment(\.locale)` 响应语言变化：

```swift
struct ContentView: View {
    @Environment(\.locale) var locale

    var body: some View {
        Text("welcome_message")
            .environment(\.locale, .init(identifier: "zh-Hans"))
    }
}
```

## 九、出海项目注意事项

### 9.1 RTL 布局测试清单

- 所有 `left`/`right` 约束是否改为 `leading`/`trailing`
- 带方向性图标是否提供镜像版本
- 文本对齐是否使用 `.natural` 而非 `.left`
- 数字和日期区域是否强制 LTR
- `UIImageView` 的内容模式是否适配 RTL
- 自定义绘制（`draw(rect:)`）中的坐标是否需要翻转

### 9.2 内容审查

不同地区有不同的内容审查要求。出海中东地区需要注意宗教敏感内容，出海欧盟需要 GDPR 合规。确保应用内容（用户生成内容、营销文案）符合目标市场的法规。

### 9.3 应用商店元数据本地化

App Store Connect 中需要为每种支持的语言提供：应用名称、副标题、描述、关键词、截图。这些元数据的本地化质量直接影响 ASO（App Store Optimization）效果。
