# 推送通知详解

## 一、推送通知概述

iOS 推送通知分为两大类：本地通知（Local Notification）和远程通知（Remote Notification，即 Push Notification）。

本地通知由 App 在设备端调度，不需要服务器参与，适合闹钟、提醒等场景。远程通知由开发者服务器通过 Apple Push Notification service（APNs）发送到用户设备，适合消息、活动提醒等场景。

## 二、本地通知

### 2.1 请求权限

```swift
import UserNotifications

UNUserNotificationCenter.current().requestAuthorization(options: [.alert, .badge, .sound]) { granted, error in
    if granted {
        print("Notification permission granted")
    } else {
        print("Notification permission denied")
    }
}
```

iOS 12+ 新增权限选项：`.providesAppNotificationSettings`（在通知中显示 App 内通知设置入口）、`.provisional`（静默授权，只发静默通知）、`.criticalAlert`（紧急警报，可绕过勿扰模式，需要特殊权限）。

### 2.2 创建本地通知

```swift
let content = UNMutableNotificationContent()
content.title = "提醒"
content.body = "您有一个待办事项"
content.sound = .default
content.badge = 1
content.userInfo = ["taskId": "12345"]  // 自定义数据

// 触发器
let trigger = UNTimeIntervalNotificationTrigger(timeInterval: 60, repeats: false)

// 也可以用日历触发
// let trigger = UNCalendarNotificationTrigger(
//     dateMatching: DateComponents(hour: 9, minute: 0),
//     repeats: true
// )

// 或位置触发
// let trigger = UNLocationNotificationTrigger(
//     region: CLCircularRegion(...),
//     repeats: false
// )

let request = UNNotificationRequest(
    identifier: "task-12345",
    content: content,
    trigger: trigger
)

UNUserNotificationCenter.current().add(request) { error in
    if let error = error {
        print("Error: \(error)")
    }
}
```

### 2.3 管理已调度通知

```swift
// 获取所有待触发的通知
UNUserNotificationCenter.current().getPendingNotificationRequests { requests in
    for request in requests {
        print(request.identifier)
    }
}

// 取消特定通知
UNUserNotificationCenter.current().removePendingNotificationRequests(withIdentifiers: ["task-12345"])

// 取消所有通知
UNUserNotificationCenter.current().removeAllPendingNotificationRequests()
```

### 2.4 通知交互

```swift
// 定义 Action Category
let acceptAction = UNNotificationAction(identifier: "ACCEPT", title: "接受", options: .foreground)
let declineAction = UNNotificationAction(identifier: "DECLINE", title: "拒绝", options: .destructive)

let category = UNNotificationCategory(
    identifier: "INVITE",
    actions: [acceptAction, declineAction],
    intentIdentifiers: [],
    options: []
)

UNUserNotificationCenter.current().setNotificationCategories([category])

// 在通知中使用
content.categoryIdentifier = "INVITE"
```

### 2.5 通知代理

```swift
class NotificationDelegate: NSObject, UNUserNotificationCenterDelegate {
    // App 在前台时收到通知
    func userNotificationCenter(_ center: UNUserNotificationCenter,
                                willPresent notification: UNNotification,
                                withCompletionHandler completionHandler: @escaping (UNNotificationPresentationOptions) -> Void) {
        // 前台也显示通知
        completionHandler([.banner, .sound, .badge])
    }

    // 用户点击通知
    func userNotificationCenter(_ center: UNUserNotificationCenter,
                                didReceive response: UNNotificationResponse,
                                withCompletionHandler completionHandler: @escaping () -> Void) {
        let action = response.actionIdentifier
        let userInfo = response.notification.request.content.userInfo

        switch action {
        case "ACCEPT":
            handleAccept(userInfo)
        case "DECLINE":
            handleDecline(userInfo)
        case UNNotificationDefaultActionIdentifier:
            // 用户直接点击通知打开 App
            handleOpen(userInfo)
        default:
            break
        }
        completionHandler()
    }
}
```

## 三、远程推送

### 3.1 APNs 流程

```
开发者服务器 → APNs (Apple Push Notification service) → iOS 设备 → App
```

1. App 向系统注册远程通知，获取 Device Token
2. App 将 Device Token 发送给开发者服务器
3. 开发者服务器携带 Token 和推送内容向 APNs 发送请求
4. APNs 将通知推送到对应设备
5. 设备收到通知后由系统显示，或唤醒 App 处理

### 3.2 注册远程推送

```swift
import UserNotifications

// 1. 请求权限
UNUserNotificationCenter.current().requestAuthorization(options: [.alert, .sound, .badge]) { granted, _ in
    if granted {
        DispatchQueue.main.async {
            // 2. 注册远程通知
            UIApplication.shared.registerForRemoteNotifications()
        }
    }
}

// 3. 获取 Token
func application(_ application: UIApplication, didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data) {
    let tokenString = deviceToken.map { String(format: "%02.2hhx", $0) }.joined()
    print("Device Token: \(tokenString)")
    // 发送到服务器
}

func application(_ application: UIApplication, didFailToRegisterForRemoteNotificationsWithError error: Error) {
    print("Failed to register: \(error)")
}
```

### 3.3 APNs 请求格式

开发者服务器通过 HTTP/2 向 APNs 发送推送：

```
POST /3/device/<deviceToken>
Host: api.sandbox.push.apple.com  (开发环境)
Host: api.push.apple.com          (生产环境)

Headers:
  apns-topic: com.example.MyApp
  apns-push-type: alert  (或 background)
  apns-priority: 10  (立即发送) 或 5 (省电模式)
  authorization: bearer <JWT>

Body:
{
    "aps": {
        "alert": {
            "title": "新消息",
            "body": "你有一条新消息"
        },
        "badge": 1,
        "sound": "default",
        "mutable-content": 1  // 允许 Notification Extension 修改
    },
    "customKey": "customValue"
}
```

`aps` 字典是 APNs 预定义的载荷，`mutable-content: 1` 允许 App 在收到通知前通过 Notification Service Extension 修改内容（如下载图片附件、解密内容）。

### 3.4 认证方式

**Token-based（推荐）**：使用 JWT（JSON Web Token），一个 token 可用于所有 App，不需要管理证书。

```
JWT Header: { "alg": "ES256", "kid": "<Key ID>" }
JWT Payload: { "iss": "<Team ID>", "iat": <timestamp> }
```

**Certificate-based**：使用 APNs 证书（.p8 文件），每个 App 一套证书。已逐渐被 Token 方式取代。

### 3.5 静默推送

静默推送不显示通知 UI，而是在后台唤醒 App 执行任务（如同步数据、预加载内容）：

```
Body:
{
    "aps": {
        "content-available": 1
    }
}
```

```swift
func application(_ application: UIApplication,
                didReceiveRemoteNotification userInfo: [AnyHashable: Any],
                fetchCompletionHandler completionHandler: @escaping (UIBackgroundFetchResult) -> Void) {
    // 处理静默推送
    syncData { result in
        switch result {
        case .newData: completionHandler(.newData)
        case .noData: completionHandler(.noData)
        case .failed: completionHandler(.failed)
        }
    }
}
```

静默推送的限制：系统会根据电量、网络、使用频率等因素决定是否唤醒 App。不能保证可靠到达。`apns-priority` 应设为 5（省电模式）。`apns-push-type` 设为 `background`。

## 四、Notification Service Extension

Notification Service Extension 在通知显示前拦截并修改内容：

```swift
import UserNotifications

class NotificationService: UNNotificationServiceExtension {
    var contentHandler: ((UNNotificationContent) -> Void)?
    var bestAttemptContent: UNMutableNotificationContent?

    override func didReceive(_ request: UNNotificationRequest,
                            withContentHandler contentHandler: @escaping (UNNotificationContent) -> Void) {
        self.contentHandler = contentHandler
        bestAttemptContent = request.content.mutableCopy() as? UNMutableNotificationContent

        guard let bestAttempt = bestAttemptContent else { return }

        // 修改标题或内容
        bestAttempt.title = "修改后的标题: \(bestAttempt.title)"

        // 下载图片附件
        if let imageURL = URL(string: userInfo["image_url"] as? String ?? "") {
            downloadImage(url: imageURL) { localURL in
                if let localURL = localURL {
                    let attachment = try? UNNotificationAttachment(
                        identifier: "image",
                        url: localURL,
                        options: nil
                    )
                    bestAttempt.attachments = [attachment].compactMap { $0 }
                }
                contentHandler(bestAttempt)
            }
        } else {
            contentHandler(bestAttempt)
        }
    }

    override func serviceExtensionTimeWillExpire() {
        // 系统给的时间（约 30 秒）即将到期
        if let contentHandler = contentHandler, let bestAttemptContent = bestAttemptContent {
            contentHandler(bestAttemptContent)
        }
    }
}
```

## 五、Notification Content Extension

Notification Content Extension 自定义通知展开后的 UI：

```swift
class NotificationViewController: UIViewController, UNNotificationContentExtension {
    @IBOutlet var titleLabel: UILabel!
    @IBOutlet var bodyLabel: UILabel!

    override func viewDidLoad() {
        super.viewDidLoad()
    }

    func didReceive(_ notification: UNNotification) {
        titleLabel.text = notification.request.content.title
        bodyLabel.text = notification.request.content.body
    }
}
```

在 Info.plist 中配置 category identifier，当通知的 categoryIdentifier 匹配时，展开通知会显示自定义 UI。

## 六、推送调试

### 6.1 推送不到的常见原因

**权限未授予**：用户关闭了通知权限。检查 `UNUserNotificationCenter.current().getNotificationSettings()`。

**Token 过期**：Device Token 可能在系统更新、App 重装等情况下变化。每次 App 启动都应注册并更新 Token。

**证书/Token 错误**：开发环境用 sandbox APNs，生产环境用 production APNs。证书与 Bundle ID 不匹配。

**静默推送被系统限制**：低电量、后台 App 过多等情况下系统可能不唤醒 App。

### 6.2 调试工具

**Xcode Push Notification Console**（Xcode 14+）：直接在 Xcode 中发送测试推送，无需服务器。

**Pusher**（第三方 Mac App）：可视化发送 APNs 推送，支持证书和 Token 认证。

**Apollo**：Apple 的 APNs 调试工具，可以查看推送发送状态和错误码。

## 七、通知分组与摘要

iOS 12+ 支持通知分组：

```swift
content.threadIdentifier = "group-chat-123"  // 同一 thread 的通知归为一组
content.summaryArgument = "张三"
content.summaryArgumentCount = 3
```

分组后的摘要可以在 `UNNotificationCategory` 中配置：

```swift
let category = UNNotificationCategory(
    identifier: "MESSAGE",
    actions: [],
    intentIdentifiers: [],
    options: [.hiddenPreviewsSummaryArgumentShowsArg]  // 在摘要中显示参数
)
```
