# 通信协议数据分包与重组机制

## 概述

当应用层的一帧数据超过传输层单次可传输的最大长度（如 BLE MTU 限制的 244 字节，或 UDP 的 1472 字节）时，必须将大帧拆分为多个子包发送，接收端再按序重组。数据分包与重组机制是通信协议设计的核心环节，直接关系到大数据传输的可靠性和效率。

## 问题场景

### BLE 场景

BLE 4.2 协商 MTU=247 后，单次 Write 的有效载荷为 244 字节。一次 OTA 固件升级可能需要传输 512KB 的固件，需要拆分为约 2150 个子包。

### WiFi/UDP 场景

UDP 数据报在以太网上的 MTU 为 1500 字节，减去 IP 头（20 字节）和 UDP 头（8 字节），有效载荷为 1472 字节。超过此大小的数据报会在 IP 层分片，而 IP 分片是不可靠的——任一分片丢失导致整个数据报丢弃。

### 核心挑战

**顺序保证**：子包可能乱序到达（尤其 UDP 场景），需要正确排序。

**完整性检测**：如何判断所有子包已接收完毕。

**丢包处理**：子包在传输中丢失，需要检测和重传。

**效率**：分包头开销尽量小，不影响传输吞吐。

## 分包协议设计

### 分包帧格式

在应用层协议帧的基础上，扩展分包头部：

```
原始应用帧:
┌──────────┬─────────┬────────────────┐
│ FrameHdr │ Payload │ CRC            │
│ (9 bytes)│ (N bytes)│ (2 bytes)     │
└──────────┴─────────┴────────────────┘

分包后的子包:
┌──────────┬────────────┬──────────┬────────────┬─────────────┬────────┐
│ FrameHdr │ FragmentHdr│ SeqNum   │ FragOffset │ FragPayload │ FragCRC│
│ (9 bytes)│ (见下文)   │ (2 bytes)│ (见下文)   │ (N bytes)   │(2 bytes)│
└──────────┴────────────┴──────────┴────────────┴─────────────┴────────┘
```

### 分包头部设计

```
Fragment Header:
┌──────────────┬──────────────┬───────────────┬─────────────────┐
│ TotalLength  │ FragIndex    │ FragTotal     │ Flags           │
│ 4 bytes      │ 2 bytes      │ 2 bytes       │ 1 byte          │
└──────────────┴──────────────┴───────────────┴─────────────────┘

TotalLength (4 bytes): 原始完整帧的总字节数（大端序）
FragIndex  (2 bytes): 当前子包序号，从 0 开始
FragTotal  (2 bytes): 子包总数
Flags      (1 byte):
  Bit 0:   1=本子包包含分包头，0=后续子包（可省略头部优化）
  Bit 1:   1=最后一个子包
  Bit 2:   1=请求 ACK，0=无需 ACK
  Bit 3-7: 保留
```

分包头开销：9 字节。对于 244 字节的 BLE 子包，开销约 3.7%，可接受。

### 子包 Payload 大小计算

```swift
func calculateFragmentPayloadSize(mtu: Int, headerSize: Int) -> Int {
    // MTU - ATT Header(3) - Frame Header(9) - Fragment Header(9) - CRC(2)
    let availablePayload = mtu - 3 - 9 - 9 - 2
    return max(1, availablePayload)  // 至少 1 字节
}

// 示例: MTU=247
// availablePayload = 247 - 3 - 9 - 9 - 2 = 224 bytes
```

## 分包发送实现

```swift
class FragmentSender {
    private let maxFragmentPayload: Int

    init(maxPayloadSize: Int) {
        // maxPayloadSize = 传输层单次最大可发送字节数
        // 减去帧头(9) + 分包头(9) + CRC(2) = 20
        self.maxFragmentPayload = maxPayloadSize - 20
    }

    func fragment(frame: ProtocolFrame) -> [Data] {
        let fullPayload = frame.encode()
        let totalLength = fullPayload.count
        let fragmentCount = Int(ceil(Double(totalLength) / Double(maxFragmentPayload)))

        var fragments: [Data] = []

        for index in 0..<fragmentCount {
            let offset = index * maxFragmentPayload
            let end = min(offset + maxFragmentPayload, totalLength)
            let fragPayload = fullPayload.subdata(in: offset..<end)

            // 构建分包帧
            var fragFrame = Data()

            // Frame Header (简化版，使用特殊 CmdID 标识分包帧)
            fragFrame.append(ProtocolFrame.preamble)      // 0x55
            fragFrame.append(contentsOf: [0xAA, 0x55])    // Magic
            fragFrame.append(0x10)                         // Version
            fragFrame.append(0xFE)                         // CmdID=0xFE 标识分包数据

            var seqNum = frame.seqNum.bigEndian
            fragFrame.append(Data(bytes: &seqNum, count: 2))

            var payloadLen = UInt16(fragPayload.count + 9).bigEndian  // 分包头 + 数据
            fragFrame.append(Data(bytes: &payloadLen, count: 2))

            // Fragment Header
            var totalLenBE = UInt32(totalLength).bigEndian
            fragFrame.append(Data(bytes: &totalLenBE, count: 4))

            var fragIndexBE = UInt16(index).bigEndian
            fragFrame.append(Data(bytes: &fragIndexBE, count: 2))

            var fragTotalBE = UInt16(fragmentCount).bigEndian
            fragFrame.append(Data(bytes: &fragTotalBE, count: 2))

            // Flags
            var flags: UInt8 = 0b00000001  // 包含分包头
            if index == fragmentCount - 1 {
                flags |= 0b00000010  // 最后一个子包
            }
            if index % 4 == 0 {  // 每 4 个子包请求一次 ACK
                flags |= 0b00000100
            }
            fragFrame.append(flags)

            // Fragment Payload
            fragFrame.append(fragPayload)

            // CRC
            let crcData = fragFrame.subdata(in: 1..<fragFrame.count)
            let crc = CRC16.ccitt(data: crcData)
            var crcBE = crc.bigEndian
            fragFrame.append(Data(bytes: &crcBE, count: 2))

            fragments.append(fragFrame)
        }

        return fragments
    }
}
```

## 重组接收实现

```swift
class FragmentReassembler {

    // 按帧序号缓存分片子包
    private struct ReassemblyBuffer {
        let totalLength: Int
        let fragmentCount: Int
        var receivedFragments: [Int: Data] = [:]
        var firstReceivedTime: Date

        var isComplete: Bool {
            receivedFragments.count == fragmentCount
        }

        var receivedBytes: Int {
            receivedFragments.values.reduce(0) { $0 + $1.count }
        }
    }

    private var buffers: [UInt16: ReassemblyBuffer] = [:]  // key: seqNum
    private let timeout: TimeInterval = 30  // 30 秒超时
    private var timer: DispatchSourceTimer?

    var onComplete: ((UInt16, Data) -> Void)?  // seqNum, reassembledData
    var onTimeout: ((UInt16) -> Void)?

    init() {
        startTimeoutChecker()
    }

    func processFragment(_ data: Data) {
        guard data.count >= ProtocolFrame.headerSize + 9 else { return }

        // 解析帧头
        let seqNum = data.subdata(in: 5..<7).withUnsafeBytes {
            $0.load(as: UInt16.self).byteSwapped
        }

        // 解析分包头
        let fragHeaderOffset = ProtocolFrame.headerSize
        let totalLength = Int(data.subdata(in: fragHeaderOffset..<fragHeaderOffset+4).withUnsafeBytes {
            $0.load(as: UInt32.self).byteSwapped
        })
        let fragIndex = Int(data.subdata(in: fragHeaderOffset+4..<fragHeaderOffset+6).withUnsafeBytes {
            $0.load(as: UInt16.self).byteSwapped
        })
        let fragTotal = Int(data.subdata(in: fragHeaderOffset+6..<fragHeaderOffset+8).withUnsafeBytes {
            $0.load(as: UInt16.self).byteSwapped
        })
        let flags = data[fragHeaderOffset + 8]

        // 提取分片数据
        let fragPayloadStart = fragHeaderOffset + 9
        let fragPayloadEnd = data.count - 2  // 减去 CRC
        guard fragPayloadStart < fragPayloadEnd else { return }
        let fragPayload = data.subdata(in: fragPayloadStart..<fragPayloadEnd)

        // 获取或创建重组缓冲区
        if buffers[seqNum] == nil {
            buffers[seqNum] = ReassemblyBuffer(
                totalLength: totalLength,
                fragmentCount: fragTotal,
                firstReceivedTime: Date()
            )
        }

        // 存储分片
        buffers[seqNum]?.receivedFragments[fragIndex] = fragPayload

        // 检查是否完整
        if let buffer = buffers[seqNum], buffer.isComplete {
            let reassembled = reassemble(buffer: buffer)
            buffers.removeValue(forKey: seqNum)
            onComplete?(seqNum, reassembled)
        }
    }

    private func reassemble(buffer: ReassemblyBuffer) -> Data {
        var data = Data()
        data.reserveCapacity(buffer.totalLength)

        for index in 0..<buffer.fragmentCount {
            if let frag = buffer.receivedFragments[index] {
                data.append(frag)
            }
        }

        return data
    }

    // 超时清理
    private func startTimeoutChecker() {
        let timer = DispatchSource.makeTimerSource(queue: .global(qos: .utility))
        timer.schedule(deadline: .now() + 5, repeating: 5)  // 每 5 秒检查一次
        timer.setEventHandler { [weak self] in
            self?.checkTimeouts()
        }
        timer.resume()
        self.timer = timer
    }

    private func checkTimeouts() {
        let now = Date()
        for (seqNum, buffer) in buffers {
            if now.timeIntervalSince(buffer.firstReceivedTime) > timeout {
                buffers.removeValue(forKey: seqNum)
                onTimeout?(seqNum)
            }
        }
    }

    // 请求重传缺失的分片
    func getMissingFragments(seqNum: UInt16) -> [Int] {
        guard let buffer = buffers[seqNum] else { return [] }
        var missing: [Int] = []
        for i in 0..<buffer.fragmentCount {
            if buffer.receivedFragments[i] == nil {
                missing.append(i)
            }
        }
        return missing
    }
}
```

## 选择性重传机制

### 丢包检测

```swift
class SelectiveRetransmit {
    private var sentFragments: [UInt16: [(index: Int, data: Data, sentTime: Date)]] = [:]
    private let ackTimeout: TimeInterval = 3.0  // 3 秒未收到 ACK 则重传

    // 记录已发送的分片
    func recordSent(seqNum: UInt16, fragments: [Data]) {
        let records = fragments.enumerated().map {
            (index: $0.offset, data: $0.element, sentTime: Date())
        }
        sentFragments[seqNum] = records
    }

    // 处理接收端的 ACK
    func handleACK(seqNum: UInt16, ackedIndexes: [Int]) {
        guard var records = sentFragments[seqNum] else { return }

        // 移除已确认的分片
        records.removeAll { ackedIndexes.contains($0.index) }

        if records.isEmpty {
            sentFragments.removeValue(forKey: seqNum)
        } else {
            sentFragments[seqNum] = records
        }
    }

    // 获取需要重传的分片
    func getRetransmitFragments() -> [(seqNum: UInt16, index: Int, data: Data)] {
        let now = Date()
        var retransmits: [(UInt16, Int, Data)] = []

        for (seqNum, records) in sentFragments {
            for record in records {
                if now.timeIntervalSince(record.sentTime) > ackTimeout {
                    retransmits.append((seqNum, record.index, record.data))
                }
            }
        }

        return retransmits
    }
}
```

### NACK（否定确认）机制

接收端主动报告缺失的分片，比超时重传更高效：

```swift
// 接收端发送 NACK
func sendNACK(seqNum: UInt16, missingIndexes: [Int]) {
    var payload = Data()
    // 原始帧序号
    var seqBE = seqNum.bigEndian
    payload.append(Data(bytes: &seqBE, count: 2))
    // 缺失分片数量
    payload.append(UInt8(missingIndexes.count))
    // 缺失分片序号
    for index in missingIndexes {
        var idxBE = UInt16(index).bigEndian
        payload.append(Data(bytes: &idxBE, count: 2))
    }

    let nackFrame = ProtocolFrame(
        cmdID: 0xFD,  // NACK 命令
        seqNum: seqNum,
        payload: payload
    )
    // 发送 NACK...
}

// 发送端处理 NACK
func handleNACK(_ frame: ProtocolFrame) {
    guard frame.payload.count >= 3 else { return }
    let seqNum = frame.payload.subdata(in: 0..<2).withUnsafeBytes {
        $0.load(as: UInt16.self).byteSwapped
    }
    let count = Int(frame.payload[2])

    var missingIndexes: [Int] = []
    for i in 0..<count {
        let offset = 3 + i * 2
        let idx = frame.payload.subdata(in: offset..<offset+2).withUnsafeBytes {
            $0.load(as: UInt16.self).byteSwapped
        }
        missingIndexes.append(Int(idx))
    }

    // 重传缺失的分片
    retransmitFragments(seqNum: seqNum, indexes: missingIndexes)
}
```

## 滑动窗口流控

```swift
class SlidingWindow {
    private var windowSize: Int
    private var sendBase = 0      // 最早未确认的序号
    private var nextSeqNum = 0    // 下一个待发送序号
    private var unacked: [(index: Int, data: Data, timer: DispatchSourceTimer?)] = []
    private let lock = NSLock()

    init(windowSize: Int = 8) {
        self.windowSize = windowSize
    }

    func canSend() -> Bool {
        lock.lock()
        defer { lock.unlock() }
        return nextSeqNum < sendBase + windowSize
    }

    func recordSend(data: Data) -> Int {
        lock.lock()
        defer { lock.unlock() }
        let seq = nextSeqNum
        nextSeqNum += 1
        unacked.append((seq, data, nil))
        return seq
    }

    func receiveACK(ackIndex: Int) {
        lock.lock()
        defer { lock.unlock() }

        // 累积确认：ACK n 表示 n 及之前的所有分片已收到
        sendBase = ackIndex + 1

        // 移除已确认的条目
        unacked.removeAll { $0.index <= ackIndex }
    }

    func getUnackedForRetransmit() -> [(Int, Data)] {
        lock.lock()
        defer { lock.unlock() }
        return unacked.map { ($0.index, $0.data) }
    }
}
```

## 内存管理

大数据传输时的内存管理至关重要：

```swift
class MemoryEfficientFragmenter {

    // 流式分包：不一次性加载全部数据到内存
    func streamFragment(filePath: String, chunkSize: Int,
                        handler: (Int, Data) -> Void) throws {
        let fileHandle = try FileHandle(forReadingFrom: URL(fileURLWithPath: filePath))
        defer { fileHandle.closeFile() }

        let totalSize = try fileHandle.seekToEnd()
        try fileHandle.seek(toOffset: 0)

        let fragmentCount = Int(ceil(Double(totalSize) / Double(chunkSize)))
        var index = 0

        while autoreleasepool(invoking: { () -> Bool in
            let remaining = totalSize - fileHandle.offsetInFile
            guard remaining > 0 else { return false }

            let readSize = min(UInt64(chunkSize), remaining)
            let chunk = fileHandle.readData(ofLength: Int(readSize))

            handler(index, chunk)
            index += 1
            return true
        }) { }
    }
}
```

## 常见问题

**1. 乱序到达**：UDP 传输中子包可能乱序。解决方案：每个子包携带 FragIndex，接收端按序号重组，不依赖到达顺序。

**2. 重复包**：网络重传可能导致重复子包。解决方案：接收端用 FragIndex 作为 key 存入字典，重复包自动覆盖。

**3. 内存爆炸**：大量并发的分包重组缓冲区可能导致内存暴涨。解决方案：限制同时进行的重组任务数量，超出的分包请求排队或拒绝。

**4. 分片风暴**：高频小帧分包导致大量小包。解决方案：设置最小分片阈值，小帧不分包直接发送。

## 总结

数据分包与重组是大数据传输的基础机制。核心设计包括：9 字节分包头携带总长度和序号信息、基于 FragIndex 的乱序重组、选择性重传（NACK）提高丢包恢复效率、滑动窗口控制发送速率。在 BLE 场景下需要特别关注 MTU 限制下的分片大小优化，在 UDP 场景下需要处理乱序和重复包问题。配合前文的数据帧格式设计，这套机制构成了完整的可靠传输协议栈。
