# 通信协议校验机制与加密策略

## 概述

IoT 设备通信面临两类风险：传输错误（噪声、干扰导致数据损坏）和安全攻击（窃听、篡改、伪造）。校验机制保证数据完整性，加密策略保证数据机密性和真实性。本文详细讲解通信协议中常用的校验算法、加密方案和密钥管理策略。

## 校验机制

### 校验算法对比

| 算法 | 检测能力 | 开销 | 适用场景 |
|------|---------|------|---------|
| 校验和 (Checksum) | 单字节错误 | 极低 | 简单场景，不推荐 |
| CRC8 | 1-2 字节错误 | 低 | 短帧校验 |
| CRC16-CCITT | 多字节错误，突发错误 | 中 | 通用帧校验 |
| CRC32 | 强突发错误检测 | 中高 | 大数据块、固件校验 |
| HMAC-SHA256 | 篡改 + 伪造 | 高 | 安全认证 |

### CRC16-CCITT 实现

CRC16-CCITT 是 BLE 通信中最常用的校验算法，能检测所有 1-2 位错误和 99.998% 的突发错误：

```swift
enum CRC16 {

    /// CRC16-CCITT (多项式 0x1021, 初始值 0xFFFF)
    /// 适用于大多数 IoT 通信帧校验
    static func ccitt(data: Data) -> UInt16 {
        var crc: UInt16 = 0xFFFF
        let polynomial: UInt16 = 0x1021

        for byte in data {
            crc ^= UInt16(byte) << 8
            for _ in 0..<8 {
                if crc & 0x8000 != 0 {
                    crc = (crc << 1) ^ polynomial
                } else {
                    crc <<= 1
                }
            }
        }
        return crc
    }

    /// CRC16-Modbus (多项式 0xA001, 初始值 0xFFFF)
    /// 适用于工业设备通信
    static func modbus(data: Data) -> UInt16 {
        var crc: UInt16 = 0xFFFF
        let polynomial: UInt16 = 0xA001

        for byte in data {
            crc ^= UInt16(byte)
            for _ in 0..<8 {
                if crc & 1 != 0 {
                    crc = (crc >> 1) ^ polynomial
                } else {
                    crc >>= 1
                }
            }
        }
        return crc
    }

    /// 查表法加速（牺牲空间换时间）
    private static let ccittTable: [UInt16] = {
        var table = [UInt16](repeating: 0, count: 256)
        let polynomial: UInt16 = 0x1021
        for i in 0..<256 {
            var crc = UInt16(i) << 8
            for _ in 0..<8 {
                if crc & 0x8000 != 0 {
                    crc = (crc << 1) ^ polynomial
                } else {
                    crc <<= 1
                }
            }
            table[i] = crc
        }
        return table
    }()

    static func ccittFast(data: Data) -> UInt16 {
        var crc: UInt16 = 0xFFFF
        for byte in data {
            let index = Int((crc >> 8) ^ UInt16(byte)) & 0xFF
            crc = (crc << 8) ^ ccittTable[index]
        }
        return crc
    }
}
```

### CRC32 实现（固件校验）

OTA 固件升级时需要校验整个固件文件的完整性：

```swift
enum CRC32 {

    private static let table: [UInt32] = {
        var table = [UInt32](repeating: 0, count: 256)
        let polynomial: UInt32 = 0xEDB88320
        for i in 0..<256 {
            var crc = UInt32(i)
            for _ in 0..<8 {
                if crc & 1 != 0 {
                    crc = (crc >> 1) ^ polynomial
                } else {
                    crc >>= 1
                }
            }
            table[i] = crc
        }
        return table
    }()

    /// 标准 CRC32（用于固件校验）
    static func calculate(_ data: Data) -> UInt32 {
        var crc: UInt32 = 0xFFFFFFFF
        for byte in data {
            let index = Int((crc ^ UInt32(byte)) & 0xFF)
            crc = (crc >> 8) ^ table[index]
        }
        return crc ^ 0xFFFFFFFF
    }

    /// 流式 CRC32（用于大文件分片校验）
    static func streamCalculate(_ data: Data, currentCRC: UInt32 = 0xFFFFFFFF) -> UInt32 {
        var crc = currentCRC
        for byte in data {
            let index = Int((crc ^ UInt32(byte)) & 0xFF)
            crc = (crc >> 8) ^ table[index]
        }
        return crc
    }
}
```

## 加密策略

### 加密层次

```
应用层加密    — 端到端加密，最高安全级别
  ↓
传输层加密    — BLE SMP 配对加密 / TLS
  ↓
链路层加密    — BLE LL 层 AES-CCM
```

BLE 协议栈在链路层（LL 层）提供了 AES-CCM 加密，但仅在配对后生效。对于安全要求高的场景，还需要应用层加密作为补充。

### BLE SMP 配对加密

BLE 安全管理协议（SMP）提供四种配对方式：

```
LE Legacy Pairing (Just Works)     — 无认证，无中间人保护
LE Legacy Pairing (Passkey)        — 6 位数字认证
LE Secure Connections (Just Works) — 无认证，ECDH 密钥交换
LE Secure Connections (Passkey)    — 6 位数字 + ECDH，最高安全级别
```

iOS 代码触发配对：

```swift
// 读取受保护的特征值时自动触发配对
peripheral.readValue(for: protectedCharacteristic)

// iOS 系统自动弹出配对请求 UI
// 用户在设备端输入配对码后完成配对
// 配对成功后连接自动加密
```

### 应用层 AES-CCM 加密

对于需要端到端加密的场景，在应用层实现 AES-CCM：

```swift
import CryptoKit

class AppLayerCrypto {

    /// AES-CCM 加密
    /// - Parameters:
    ///   - plaintext: 明文数据
    ///   - key: 128 位密钥
    ///   - nonce: 12 字节随机数（每次加密必须不同）
    ///   - authenticatedData: 需要认证但不需要加密的数据（如帧头）
    /// - Returns: 密文 + 认证标签（16 字节）
    static func encrypt(plaintext: Data,
                        key: SymmetricKey,
                        nonce: AES.GCM.Nonce,
                        authenticatedData: Data) throws -> Data {

        let sealedBox = try AES.GCM.seal(
            plaintext,
            using: key,
            nonce: nonce,
            authenticating: authenticatedData
        )

        // 密文 + 标签
        var result = Data()
        result.append(sealedBox.ciphertext)
        result.append(sealedBox.tag)
        return result
    }

    /// AES-CCM 解密
    static func decrypt(ciphertext: Data,
                        key: SymmetricKey,
                        nonce: AES.GCM.Nonce,
                        authenticatedData: Data) throws -> Data {

        // 分离密文和标签
        let tagSize = 16
        guard ciphertext.count > tagSize else {
            throw CryptoError.invalidCiphertext
        }

        let actualCiphertext = ciphertext.prefix(ciphertext.count - tagSize)
        let tag = ciphertext.suffix(tagSize)

        let sealedBox = try AES.GCM.SealedBox(
            nonce: nonce,
            ciphertext: actualCiphertext,
            tag: tag
        )

        return try AES.GCM.open(sealedBox, using: key, authenticating: authenticatedData)
    }

    enum CryptoError: Error {
        case invalidCiphertext
        case decryptionFailed
    }
}
```

### 加密帧格式

加密后的协议帧格式：

```
原始帧:
┌──────────┬─────────────────┬──────────┐
│ FrameHdr │ Payload(明文)    │ CRC      │
└──────────┴─────────────────┴──────────┘

加密帧:
┌──────────┬──────────┬───────────────┬──────────┬──────────┐
│ FrameHdr │ Nonce(12B)│ Payload(密文)  │ Tag(16B) │ CRC      │
│          │          │               │          │          │
│ 明文传输  │ 明文传输  │ 加密传输       │ 认证标签  │ 校验     │
└──────────┴──────────┴───────────────┴──────────┴──────────┘

FrameHdr 作为 AuthenticatedData 参与认证但不加密
```

        var counterBE = nonceCounter.bigEndian
        nonceData.append(Data(bytes: &counterBE, count: 4))  // 4 bytes 计数器

        // 3. 加密 Payload
        let encrypted = try AppLayerCrypto.encrypt(
            plaintext: frame.payload,
            key: cryptoKey,
            nonce: nonce,
            authenticatedData: headerData
        )

        var encryptedFrame = headerData
        encryptedFrame.append(nonceData)           // Nonce (12 bytes)
        var payloadLen = UInt16(encrypted.count).bigEndian
        encryptedFrame.append(Data(bytes: &payloadLen, count: 2))

## 密钥管理

### 密钥层次

```
Master Key (根密钥)
  ├── Session Key (会话密钥) — 每次连接协商，用于通信加密
  ├── OTA Sign Key (升级签名密钥) — 固件签名验证
  └── Provisioning Key (配网密钥) — 设备绑定认证
```

### 密钥协商流程

```swift
class KeyExchange {

    /// ECDH 密钥协商（P-256 曲线）
    static func performKeyExchange(
        devicePublicKey: Data,
        appPrivateKey: P256.KeyAgreement.PrivateKey
    ) -> SymmetricKey {

        // 1. 从设备公钥数据恢复公钥对象
        let devicePubKey = try! P256.KeyAgreement.PublicKey(
            rawRepresentation: devicePublicKey
        )

        // 2. ECDH 共享密钥计算
        let sharedSecret = try! appPrivateKey.sharedSecretFromKeyAgreement(
            with: devicePubKey
        )

        // 3. 派生对称密钥（HKDF）
        let symmetricKey = sharedSecret.hkdfDerivedSymmetricKey(
            using: SHA256.self,
            salt: Data("IoT-Salt".utf8),
            sharedInfo: Data("SessionKey".utf8),
            outputByteCount: 16  // AES-128
        )

        return symmetricKey
    }

    /// 生成 App 端密钥对
    static func generateKeyPair() -> (privateKey: P256.KeyAgreement.PrivateKey,
                                       publicKeyData: Data) {
        let privateKey = P256.KeyAgreement.PrivateKey()
        let publicKeyData = privateKey.publicKey.rawRepresentation
        return (privateKey, publicKeyData)
    }
}
```

### 密钥存储

```swift
import Security

class KeyStore {

    /// 使用 iOS Keychain 安全存储密钥
    static func storeKey(_ key: Data, account: String) -> Bool {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrAccount as String: account,
            kSecValueData as String: key,
            kSecAttrAccessible as String: kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly
        ]

        SecItemDelete(query as CFDictionary)  // 先删除旧值
        let status = SecItemAdd(query as CFDictionary, nil)
        return status == errSecSuccess
    }

    static func retrieveKey(account: String) -> Data? {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrAccount as String: account,
            kSecReturnData as String: true,
            kSecMatchLimit as String: kSecMatchLimitOne
        ]

        var result: AnyObject?
        let status = SecItemCopyMatching(query as CFDictionary, &result)
        return status == errSecSuccess ? result as? Data : nil
    }

    static func deleteKey(account: String) {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrAccount as String: account
        ]
        SecItemDelete(query as CFDictionary)
    }
}
```

### 密钥轮换

```swift
class KeyRotation {
    private var currentKey: SymmetricKey
    private var previousKey: SymmetricKey?  // 旧密钥，用于过渡期
    private let rotationInterval: TimeInterval = 3600  // 1 小时轮换一次
    private var lastRotationTime: Date

    init(initialKey: SymmetricKey) {
        self.currentKey = initialKey
        self.lastRotationTime = Date()
    }

    func checkAndRotate() -> SymmetricKey? {
        let elapsed = Date().timeIntervalSince(lastRotationTime)
        guard elapsed >= rotationInterval else { return nil }

        // 生成新密钥
        previousKey = currentKey
        currentKey = SymmetricKey(size: .bits128)
        lastRotationTime = Date()

        // 返回新密钥用于通知对端
        return currentKey
    }

    func decrypt(data: Data, nonce: AES.GCM.Nonce, auth: Data) -> Data? {
        // 先尝试当前密钥
        if let decrypted = try? AppLayerCrypto.decrypt(
            ciphertext: data, key: currentKey, nonce: nonce, authenticatedData: auth) {
            return decrypted
        }

        // 失败则尝试旧密钥（过渡期）
        if let prevKey = previousKey {
            if let decrypted = try? AppLayerCrypto.decrypt(
                ciphertext: data, key: prevKey, nonce: nonce, authenticatedData: auth) {
                return decrypted
            }
        }

        return nil
    }
}
```

## 固件签名验证

OTA 固件升级必须验证签名，防止恶意固件刷入：

```swift
class FirmwareSignatureVerifier {

    /// ECDSA-P256 签名验证
    static func verify(
        firmware: Data,
        signature: Data,
        publicKey: P256.Signing.PublicKey
    ) -> Bool {
        guard let ecdsaSignature = try? P256.Signing.ECDSASignature(
            rawRepresentation: signature
        ) else {
            return false
        }

        return publicKey.isValidSignature(ecdsaSignature, for: firmware)
    }

    /// 完整的固件包验证流程
    static func verifyFirmwarePackage(_ package: FirmwarePackage) -> Result<Data, FirmwareError> {
        // 1. 验证固件头
        guard package.magic == FirmwareMagic else {
            return .failure(.invalidMagic)
        }

        // 2. 验证 CRC32
        let calculatedCRC = CRC32.calculate(package.firmwareData)
        guard calculatedCRC == package.expectedCRC else {
            return .failure(.crcMismatch)
        }

        // 3. 验证签名
        guard verify(
            firmware: package.firmwareData,
            signature: package.signature,
            publicKey: package.signingPublicKey
        ) else {
            return .failure(.signatureInvalid)
        }

        // 4. 验证版本
        guard package.version > package.currentVersion else {
            return .failure(.versionTooOld)
        }

        return .success(package.firmwareData)
    }

    enum FirmwareError: Error {
        case invalidMagic
        case crcMismatch
        case signatureInvalid
        case versionTooOld
    }
}

struct FirmwarePackage {
    let magic: UInt32
    let version: UInt32
    let currentVersion: UInt32
    let firmwareData: Data
    let expectedCRC: UInt32
    let signature: Data
    let signingPublicKey: P256.Signing.PublicKey
}
```

## 防重放攻击

```swift
class AntiReplay {
    private var receivedSequenceNumbers: Set<UInt32> = []
    private var maxSeenSequence: UInt32 = 0
    private let windowSize: UInt32 = 64

    func validate(sequenceNumber: UInt32) -> Bool {
        // 如果序号在窗口内
        if sequenceNumber > maxSeenSequence - windowSize && sequenceNumber <= maxSeenSequence {
            // 检查是否已接收过
            if receivedSequenceNumbers.contains(sequenceNumber) {
                return false  // 重放攻击
            }
            receivedSequenceNumbers.insert(sequenceNumber)
            return true
        }

        // 新序号
        if sequenceNumber > maxSeenSequence {
            maxSeenSequence = sequenceNumber
            // 清理过旧的序号
            let minValid = maxSeenSequence > windowSize ? maxSeenSequence - windowSize : 0
            receivedSequenceNumbers = receivedSequenceNumbers.filter { $0 > minValid }
            receivedSequenceNumbers.insert(sequenceNumber)
            return true
        }

        // 序号过旧，拒绝
        return false
    }
}
```

## 安全实践清单

**密钥管理**：密钥存储在 Keychain（iOS）/ 安全元件中，不在代码中硬编码，不在日志中打印。会话密钥每次连接重新协商，长期密钥定期轮换。

**传输安全**：BLE 连接必须配对加密（LE Secure Connections 优先），应用层加密作为补充层。Nonce 不可重复使用，每次加密使用唯一计数器。

**固件安全**：OTA 固件必须签名验证，使用 ECDSA-P256 或 RSA-2048。固件包含版本号防降级，CRC32 校验数据完整性。

**防攻击**：实现防重放窗口，拒绝序号过旧或重复的帧。配网过程使用 ECDH 密钥协商，避免明文传输密钥。敏感操作（如解绑、恢复出厂设置）需要二次认证。

## 总结

校验和加密是通信协议安全性的两大支柱。CRC16-CCITT 适用于帧级校验，CRC32 适用于固件级校验。加密策略采用分层设计：BLE SMP 提供链路层加密，应用层 AES-GCM 提供端到端加密。密钥管理使用 ECDH 协商会话密钥，Keychain 安全存储长期密钥，定期轮换降低泄露风险。配合防重放攻击和固件签名验证，可以构建完整的 IoT 通信安全体系。
