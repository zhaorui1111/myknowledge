## 1. 概述与动机

### 1.1 什么是注意力机制

注意力机制（Attention Mechanism）是一种让模型在处理输入序列时，能够**动态地聚焦于最相关部分**的计算方法。它的核心思想源于人类视觉和认知系统——我们看一幅画时不会均匀地注视每个像素，而是把注意力集中在关键区域。

形式化地说，注意力机制是一个函数：

```
Attention: (Query, Keys, Values) → Weighted Sum of Values
```

其中权重由 Query 与各 Key 的相似度决定——越相似的 Key 对应的 Value 获得越大的权重。

### 1.2 为什么需要注意力机制

在注意力机制出现之前（2014 年之前），序列到序列（Seq2Seq）模型面临严重的**信息瓶颈问题**：

**问题 1：固定长度上下文向量**  
经典 Seq2Seq 的编码器（Encoder）将整个输入序列压缩为一个固定维度的上下文向量 c。当输入序列很长时（如翻译一段 500 词的文章），所有信息被强制塞进一个向量，必然丢失大量细节。

**问题 2：长距离依赖衰减**  
RNN/LSTM 在处理长序列时，早期 token 的信息随着时间步不断衰减。即使有门控机制（LSTM 的 forget gate），实际上超过 50~100 步后信息几乎丢失殆尽。

**问题 3：无法并行**  
RNN 必须逐步处理序列，无法利用 GPU 的大规模并行计算能力。

注意力机制优雅地解决了问题 1 和 2：**解码器在每个时间步可以「回头看」编码器的所有隐藏状态**，动态选择最相关的信息，而不再依赖单一压缩向量。后续的 Self-Attention 进一步解决了问题 3。

### 1.3 历史里程碑

| 年份 | 论文/事件 | 贡献 |
|------|-----------|------|
| 2014 | Bahdanau et al. "Neural Machine Translation by Jointly Learning to Align and Translate" | 首次提出加性注意力（Additive Attention）用于机器翻译 |
| 2015 | Luong et al. "Effective Approaches to Attention-based NMT" | 提出乘性注意力（Dot-Product Attention）及 Global/Local 变体 |
| 2017 | Vaswani et al. "Attention Is All You Need" | 提出 Self-Attention + Multi-Head Attention，构建纯注意力 Transformer |
| 2018 | BERT (Devlin et al.) | 双向 Self-Attention 预训练，席卷 NLP |
| 2018-2020 | GPT → GPT-2 → GPT-3 | 单向 Causal Self-Attention，规模化涌现 |
| 2022 | FlashAttention (Dao et al.) | IO-aware 的精确注意力加速算法 |
| 2023 | GQA (Ainslie et al.), Multi-Query Attention | 减少 KV Cache 的注意力变体 |

### 1.4 注意力机制的应用范围

注意力机制早已超越 NLP：

- **计算机视觉**：ViT (Vision Transformer)、DETR (目标检测)、Swin Transformer
- **语音处理**：Whisper (语音识别)、TTS (语音合成)
- **多模态**：CLIP、DALL-E、GPT-4V (图文对齐)
- **科学计算**：AlphaFold 2 (蛋白质结构预测中使用了 attention)
- **强化学习**：Decision Transformer

## 2. 从 Seq2Seq 到注意力：解决信息瓶颈

### 2.1 经典 Seq2Seq 的局限

经典 Encoder-Decoder 架构：

```
输入: x₁, x₂, ..., xₙ
      ↓ (Encoder RNN)
隐藏状态: h₁, h₂, ..., hₙ
      ↓ (取最后一步)
上下文向量: c = hₙ        ← 信息瓶颈！
      ↓ (Decoder RNN)
输出: y₁, y₂, ..., yₘ
```

问题：c 是固定维度的向量（如 256 或 512 维），无论输入有 5 个词还是 500 个词，所有信息都必须压缩到这同一个向量中。

实验证据：Cho et al. (2014) 发现，经典 Seq2Seq 在输入句子超过 20 个词后，BLEU 分数急剧下降。这直接证实了信息瓶颈的存在。

### 2.2 Bahdanau 注意力（Additive Attention）

**核心思想**：解码器在产生每个输出 yₜ 时，不再只用固定的 c，而是计算一个**动态上下文向量 cₜ**——编码器所有隐藏状态的加权和：

```
cₜ = Σᵢ₌₁ⁿ αₜᵢ · hᵢ
```

其中 αₜᵢ 是注意力权重（attention weight），表示解码第 t 步时对编码器第 i 个位置的关注程度。

**对齐分数计算（Additive/Concat Attention）**：

```
eₜᵢ = vᵀ · tanh(W_s · sₜ₋₁ + W_h · hᵢ)
αₜᵢ = softmax(eₜᵢ) = exp(eₜᵢ) / Σⱼ exp(eₜⱼ)
```

其中：
- sₜ₋₁ ∈ ℝ^d_dec 是解码器上一步的隐藏状态（充当 Query）
- hᵢ ∈ ℝ^d_enc 是编码器第 i 步的隐藏状态（充当 Key 和 Value）
- W_s ∈ ℝ^(d_attn × d_dec)、W_h ∈ ℝ^(d_attn × d_enc)、v ∈ ℝ^d_attn 是可学习参数
- tanh 引入非线性，使得对齐函数可以学习复杂的匹配模式

**直觉理解**：就像你翻译 "The cat sat on the mat" 到中文时——翻译 "猫" 时你的注意力自然聚焦在 "cat" 上，翻译 "垫子" 时聚焦在 "mat" 上。注意力机制让模型自动学会这种对齐。

**解码器更新**：

```
sₜ = GRU(sₜ₋₁, [yₜ₋₁; cₜ])    // 将上下文向量与前一个输出拼接
P(yₜ) = softmax(W_out · [sₜ; cₜ; yₜ₋₁])
```

### 2.3 Luong 注意力（Multiplicative Attention）

Luong et al. (2015) 提出了更简洁高效的打分方式：

```
Score 函数的三种变体:
  dot:     eₜᵢ = sₜᵀ · hᵢ              // 直接点积（无额外参数）
  general: eₜᵢ = sₜᵀ · W_a · hᵢ        // 带可学习矩阵
  concat:  eₜᵢ = vᵀ · tanh(W_a · [sₜ; hᵢ])  // 等价于 Bahdanau
```

**点积注意力的优势**：
- 无额外参数，计算量小
- 可用高效矩阵乘法实现
- 当维度不太大时（<512），效果与加性注意力相当

**Luong 的 Global vs Local Attention**：
- Global Attention：关注编码器的所有位置（与 Bahdanau 类似）
- Local Attention：只关注以预测的对齐位置为中心的一个窗口（减少计算，思想类似后来的 Local/Sliding Window Attention）

### 2.4 效果对比

在 WMT'14 English→German 翻译任务上：

```
经典 Seq2Seq (无 Attention):      BLEU ≈ 20
+ Bahdanau Attention:              BLEU ≈ 25  (+5)
+ Luong Attention (dot):           BLEU ≈ 25.9
Transformer (Self-Attention):      BLEU ≈ 28.4  (2017, 比上述更高)
```

## 3. 注意力机制的一般形式

### 3.1 Query-Key-Value 抽象框架

从更高层次抽象，**所有**注意力机制都可统一为 QKV 框架：

```
输入：Query (Q), Keys (K), Values (V)
输出：Attention(Q, K, V) = Σᵢ α(Q, Kᵢ) · Vᵢ

其中 α(Q, Kᵢ) = normalize(score(Q, Kᵢ))
```

**类比数据库检索**：
- Query（查询）：你的搜索条件 → 当前需要信息的位置
- Key（键）：数据库索引 → 每个候选位置的"标签"
- Value（值）：数据库记录内容 → 每个候选位置携带的真实信息
- 注意力权重：Query 和每个 Key 的匹配程度 → 决定从每条记录中取多少信息

注意：Key 和 Value 来自同一组数据（同一个序列），但通过不同的线性变换获得不同的角色。Key 负责"被匹配"，Value 负责"提供内容"。

### 3.2 打分函数的分类

| 类型 | 公式 | 参数量 | 计算复杂度 | 特点 |
|------|------|--------|-----------|------|
| 加性（Additive） | v^T tanh(W₁Q + W₂K) | O(d·d_attn) | O(d_attn) per pair | 灵活，Q/K 维度可不同 |
| 点积（Dot-Product） | Q^T K | 0 | O(d) per pair | 最高效，要求 Q/K 同维 |
| 缩放点积（Scaled Dot-Product） | Q^T K / √d_k | 0 | O(d) per pair | Transformer 标准，防梯度消失 |
| 双线性（General/Bilinear） | Q^T W K | O(d²) | O(d²) per pair | 灵活但参数多 |
| 余弦相似度 | cos(Q, K) = Q^T K / (‖Q‖·‖K‖) | 0 | O(d) per pair | 归一化，对向量长度不敏感 |

### 3.3 注意力的概率解释

注意力权重经过 softmax 后形成概率分布（所有权重 ≥ 0 且总和为 1）。因此注意力输出可理解为 Values 的**条件期望**：

```
Attention(Q, K, V) = E_{p(i|Q,K)}[Vᵢ] = Σᵢ P(i|Q, K) · Vᵢ
```

其中 P(i|Q, K) = softmax(score(Q, Kᵢ))

**贝叶斯视角**：给定查询 Q 和键集合 K，我们计算「每个位置 i 被选中的后验概率」P(i|Q, K)，然后对 Value 取期望。

**信息论视角**：注意力是一种"软检索"（soft retrieval）——不像 argmax 那样只选一个最相似的（hard attention），而是将信息按相似度加权混合。这使得梯度可以流向所有位置，便于端到端训练。

### 3.4 Hard Attention vs Soft Attention

- **Soft Attention**：使用所有位置的加权和，可微分，可以直接反向传播。这是 Transformer 使用的方式。
- **Hard Attention**：只选择一个位置（argmax 或采样），不可微分，需要 REINFORCE 等策略梯度方法训练。在早期图像 caption 工作中有使用。

实践中 Soft Attention 远比 Hard Attention 常用，因为训练更稳定、效果更好。

## 4. Scaled Dot-Product Attention

### 4.1 公式

这是 Transformer 中使用的核心注意力计算单元：

```
Attention(Q, K, V) = softmax(Q K^T / √d_k) · V
```

其中 d_k 是 Key 向量的维度。

### 4.2 为什么需要缩放（Scaling）——深入分析

**这是面试高频问题，也是 Transformer 最精巧的设计之一。**

假设 Q 和 K 的每个分量独立地从标准正态分布 N(0, 1) 中采样（这近似于初始化阶段或经过 LayerNorm 后的情况）。

点积 q^T k = Σᵢ₌₁^{d_k} qᵢ kᵢ 是 d_k 个独立随机变量的和，每个 qᵢkᵢ 的：
- E[qᵢkᵢ] = E[qᵢ]·E[kᵢ] = 0·0 = 0
- Var(qᵢkᵢ) = E[qᵢ²kᵢ²] - (E[qᵢkᵢ])² = E[qᵢ²]·E[kᵢ²] - 0 = 1·1 = 1

因此点积 q^T k 的：
- E[q^T k] = 0
- Var(q^T k) = d_k（d_k 个方差为 1 的独立项之和）
- 标准差 = √d_k

当 d_k = 64 时，点积的标准差为 8；d_k = 512 时标准差为约 22.6。

**问题**：将标准差很大的值输入 softmax：

```python
# d_k = 64 时，点积可能出现 |score| > 15 的极端值
softmax([15, 1, 2, -3, 0])  ≈ [0.9999, ε, ε, ε, ε]  # 几乎 one-hot

# 缩放后（除以 √64 = 8）
softmax([1.87, 0.12, 0.25, -0.37, 0])  ≈ [0.32, 0.16, 0.18, 0.10, 0.14]  # 分布更平滑
```

**后果 1 - 梯度消失**：softmax 在极端值处的梯度：
```
∂softmax(x)ᵢ/∂xⱼ = softmax(x)ᵢ · (δᵢⱼ - softmax(x)ⱼ)
```
当 softmax 输出接近 one-hot 时，大多数位置的梯度 ≈ 0，参数几乎无法更新。

**后果 2 - 表示坍缩**：注意力过度集中在单一位置，失去了多源信息聚合的能力，每个 query 的输出几乎就是某个单独的 value 向量，丢失了上下文融合的能力。

**缩放的效果**：除以 √d_k 后，点积的方差归一化为 1：
```
Var(q^T k / √d_k) = Var(q^T k) / d_k = d_k / d_k = 1
```
softmax 输入的量级与 d_k 无关，梯度和注意力分布都保持健康。

### 4.3 维度分析（Tensor Shape 追踪）

这是面试和工程调试中最容易出错的部分：

```
设 batch_size = B, 序列长度 = L, 模型维度 = d_model, 注意力头维度 = d_k = d_v

单头情况：
  Q: (B, L_q, d_k)     — L_q 是 query 序列长度
  K: (B, L_k, d_k)     — L_k 是 key/value 序列长度
  V: (B, L_k, d_v)     — V 的长度必须与 K 相同（一一对应）

计算过程：
  Step 1: Q @ K^T
    (B, L_q, d_k) × (B, d_k, L_k) → (B, L_q, L_k)   [注意力分数矩阵]

  Step 2: / √d_k
    (B, L_q, L_k)                                      [缩放后的分数]

  Step 3: + Mask（可选）
    (B, L_q, L_k)                                      [加 -∞ 遮蔽某些位置]

  Step 4: softmax(dim=-1)
    (B, L_q, L_k)                                      [注意力权重，每行和为 1]

  Step 5: @ V
    (B, L_q, L_k) × (B, L_k, d_v) → (B, L_q, d_v)   [加权聚合后的输出]

输出: (B, L_q, d_v)
```

**关键约束**：
- Q 和 K 的最后一维必须相同（都是 d_k），否则无法做点积
- K 和 V 的序列长度必须相同（L_k），因为它们是一一对应的
- Q 的序列长度 L_q 可以与 L_k 不同（cross-attention 时常见）

### 4.4 注意力分数矩阵的含义

Q @ K^T 得到的 (L_q, L_k) 矩阵中，第 i 行第 j 列的值表示"query 位置 i 对 key 位置 j 的原始注意力分数"。

对于 Self-Attention（L_q = L_k = L），这是一个 L×L 的方阵。可视化为热力图时：
- 对角线：每个 token 对自身的注意力
- 非对角线：token 间的交叉注意力
- 每一行经 softmax 后和为 1

### 4.5 PyTorch 实现

```python
import torch
import torch.nn.functional as F
import math
from typing import Optional, Tuple


def scaled_dot_product_attention(
    Q: torch.Tensor,           # (B, ..., L_q, d_k)
    K: torch.Tensor,           # (B, ..., L_k, d_k)
    V: torch.Tensor,           # (B, ..., L_k, d_v)
    mask: Optional[torch.Tensor] = None,  # broadcastable to (B, ..., L_q, L_k)
    dropout_p: float = 0.0,
    training: bool = True,
    scale: Optional[float] = None
) -> Tuple[torch.Tensor, torch.Tensor]:
    """
    Scaled Dot-Product Attention 的完整实现。
    
    支持任意 batch dimensions（适配多头注意力中 (B, h, L, d) 的形状）。
    
    Args:
        Q: Query 张量
        K: Key 张量
        V: Value 张量
        mask: 注意力掩码，True/1 表示允许注意，False/0 表示遮蔽
        dropout_p: Dropout 概率
        training: 是否在训练模式
        scale: 自定义缩放因子，默认为 1/√d_k
    
    Returns:
        output: 注意力输出 (B, ..., L_q, d_v)
        attn_weights: 注意力权重 (B, ..., L_q, L_k)
    """
    d_k = Q.size(-1)
    scale_factor = scale if scale is not None else 1.0 / math.sqrt(d_k)
    
    # 计算注意力分数
    # (..., L_q, d_k) @ (..., d_k, L_k) → (..., L_q, L_k)
    attn_scores = torch.matmul(Q, K.transpose(-2, -1)) * scale_factor
    
    # 应用 mask
    if mask is not None:
        # mask == 0 的位置设为 -inf，softmax 后变为 0
        attn_scores = attn_scores.masked_fill(mask == 0, float('-inf'))
    
    # Softmax 归一化（沿最后一个维度，即对所有 key 位置归一化）
    attn_weights = F.softmax(attn_scores, dim=-1)
    
    # 处理全 -inf 行（所有位置都被 mask 时，softmax 输出 nan）
    attn_weights = attn_weights.nan_to_num(0.0)
    
    # Dropout
    if dropout_p > 0.0 and training:
        attn_weights = F.dropout(attn_weights, p=dropout_p, training=True)
    
    # 加权求和
    # (..., L_q, L_k) @ (..., L_k, d_v) → (..., L_q, d_v)
    output = torch.matmul(attn_weights, V)
    
    return output, attn_weights


# ===== 验证 =====
if __name__ == "__main__":
    torch.manual_seed(42)
    
    B, L, d_k, d_v = 2, 5, 64, 64
    Q = torch.randn(B, L, d_k)
    K = torch.randn(B, L, d_k)
    V = torch.randn(B, L, d_v)
    
    output, weights = scaled_dot_product_attention(Q, K, V)
    print(f"Output shape: {output.shape}")       # (2, 5, 64)
    print(f"Weights shape: {weights.shape}")     # (2, 5, 5)
    print(f"Weights sum per query: {weights.sum(dim=-1)}")  # 每行和为 1
    
    # 验证缩放效果
    scores_no_scale = torch.matmul(Q, K.transpose(-2, -1))
    scores_scaled = scores_no_scale / math.sqrt(d_k)
    print(f"\n未缩放分数标准差: {scores_no_scale.std():.4f}")  # ≈ √64 = 8
    print(f"缩放后分数标准差: {scores_scaled.std():.4f}")      # ≈ 1
```

## 5. Multi-Head Attention

### 5.1 为什么需要多头

单头注意力只能学习一种"关注模式"。但自然语言中，一个词与其他词之间存在**多种并行关系**：

- **句法关系**：主语→谓语、定语→名词、介词→宾语
- **语义关系**：同义/反义、上下位、因果
- **共指关系**：代词→指代对象（"他"→"张三"）
- **位置关系**：相邻词的局部依赖、远距离照应
- **语用关系**：强调、话题-评论结构

如果只有一个注意力头，它只能学习一种关系的"混合体"，无法精确捕获每种关系。

**Multi-Head Attention 的思想**：让模型在**多个独立的子空间**中并行计算注意力，每个"头"（head）可以自由学习一种关注模式：

```
Head 1: 可能学会关注"主语在哪里"
Head 2: 可能学会关注"最近的形容词"
Head 3: 可能学会关注"与当前词语义最相似的词"
Head 4: 可能学会关注"句尾的标点/终止符"
...
```

### 5.2 数学公式

```
MultiHead(Q, K, V) = Concat(head₁, head₂, ..., head_h) · W^O

其中:
  head_i = Attention(Q · W_i^Q, K · W_i^K, V · W_i^V)
```

参数矩阵维度：
- W_i^Q ∈ ℝ^(d_model × d_k)  — 将 Q 投影到第 i 个头的 query 子空间
- W_i^K ∈ ℝ^(d_model × d_k)  — 将 K 投影到第 i 个头的 key 子空间
- W_i^V ∈ ℝ^(d_model × d_v)  — 将 V 投影到第 i 个头的 value 子空间
- W^O ∈ ℝ^(h·d_v × d_model)  — 将拼接后的多头输出映射回模型维度

标准设置：d_k = d_v = d_model / h。例如 d_model=512, h=8 时，d_k = d_v = 64。

### 5.3 维度追踪（完整流程）

```
输入: Q, K, V ∈ (B, L, d_model)    设 d_model=512, h=8, d_k=d_v=64

线性投影（所有头一起算，再 reshape）:
  Q·W^Q: (B, L, 512) @ (512, 512) → (B, L, 512) → reshape → (B, h, L, d_k) = (B, 8, L, 64)
  K·W^K: 同上 → (B, 8, L, 64)
  V·W^V: 同上 → (B, 8, L, 64)

每个头独立计算 Attention:
  scores: (B, 8, L, 64) @ (B, 8, 64, L) → (B, 8, L, L)
  weights: softmax(scores / √64) → (B, 8, L, L)
  output: (B, 8, L, L) @ (B, 8, L, 64) → (B, 8, L, 64)

拼接多头:
  (B, 8, L, 64) → transpose → (B, L, 8, 64) → reshape → (B, L, 512)

输出投影:
  (B, L, 512) @ W^O(512, 512) → (B, L, 512)
```

### 5.4 计算量分析

Multi-Head Attention 的计算量与单头全维度注意力**完全相同**：

```
单头 Attention（d_k = d_model = 512）:
  QK^T: O(L² · d_model)

Multi-Head（h=8, d_k=64）:
  8 个头各自 QK^T: 8 × O(L² · 64) = O(L² · 512) = O(L² · d_model)
```

但多头在**表达能力**上更强：h 个独立子空间各自学习不同模式，比 1 个高维空间学习单一模式更有效。这类似于卷积网络中多个 filter 各自检测不同特征。

额外的线性投影开销：
- Q/K/V 投影：3 × O(B·L·d_model²)
- 输出投影：O(B·L·d_model²)
- 总计：4 × O(B·L·d_model²)

### 5.5 完整 PyTorch 实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math


class MultiHeadAttention(nn.Module):
    """
    Multi-Head Attention (Transformer 原版实现)
    
    Args:
        d_model: 模型维度（输入/输出维度）
        n_heads: 注意力头数
        dropout: 注意力权重的 dropout 比率
        bias: 线性层是否使用偏置
    """
    
    def __init__(
        self,
        d_model: int = 512,
        n_heads: int = 8,
        dropout: float = 0.1,
        bias: bool = True
    ):
        super().__init__()
        assert d_model % n_heads == 0, "d_model must be divisible by n_heads"
        
        self.d_model = d_model
        self.n_heads = n_heads
        self.d_k = d_model // n_heads  # 每个头的维度
        self.scale = math.sqrt(self.d_k)
        
        # QKV 投影：分别用独立的线性层
        self.W_q = nn.Linear(d_model, d_model, bias=bias)
        self.W_k = nn.Linear(d_model, d_model, bias=bias)
        self.W_v = nn.Linear(d_model, d_model, bias=bias)
        
        # 输出投影
        self.W_o = nn.Linear(d_model, d_model, bias=bias)
        
        self.dropout = nn.Dropout(dropout)
        
        # 存储注意力权重（用于可视化）
        self.attn_weights = None
    
    def forward(
        self,
        query: torch.Tensor,   # (B, L_q, d_model)
        key: torch.Tensor,     # (B, L_k, d_model)
        value: torch.Tensor,   # (B, L_k, d_model)
        mask: torch.Tensor = None,  # (B, 1, L_q, L_k) 或可广播形状
        need_weights: bool = False
    ) -> torch.Tensor:
        B = query.size(0)
        L_q = query.size(1)
        L_k = key.size(1)
        
        # 1. 线性投影 + reshape 为多头
        # (B, L, d_model) → (B, L, d_model) → (B, L, h, d_k) → (B, h, L, d_k)
        Q = self.W_q(query).view(B, L_q, self.n_heads, self.d_k).transpose(1, 2)
        K = self.W_k(key).view(B, L_k, self.n_heads, self.d_k).transpose(1, 2)
        V = self.W_v(value).view(B, L_k, self.n_heads, self.d_k).transpose(1, 2)
        
        # 2. Scaled Dot-Product Attention
        # scores: (B, h, L_q, d_k) @ (B, h, d_k, L_k) → (B, h, L_q, L_k)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / self.scale
        
        # 3. 应用 mask
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float("-inf"))
        
        # 4. Softmax + Dropout
        attn_weights = F.softmax(scores, dim=-1)
        attn_weights = self.dropout(attn_weights)
        
        if need_weights:
            self.attn_weights = attn_weights.detach()
        
        # 5. 加权求和: (B, h, L_q, L_k) @ (B, h, L_k, d_k) → (B, h, L_q, d_k)
        context = torch.matmul(attn_weights, V)
        
        # 6. 拼接多头: (B, h, L_q, d_k) → (B, L_q, h, d_k) → (B, L_q, d_model)
        context = context.transpose(1, 2).contiguous().view(B, L_q, self.d_model)
        
        # 7. 输出投影
        output = self.W_o(context)
        
        return output


# 验证
if __name__ == "__main__":
    mha = MultiHeadAttention(d_model=512, n_heads=8)
    x = torch.randn(2, 10, 512)  # batch=2, seq_len=10, d_model=512
    
    # Self-Attention: Q=K=V=x
    output = mha(x, x, x, need_weights=True)
    print(f"Output shape: {output.shape}")  # (2, 10, 512)
    print(f"Attn weights shape: {mha.attn_weights.shape}")  # (2, 8, 10, 10)
    print(f"Total parameters: {sum(p.numel() for p in mha.parameters()):,}")
    # 4 × (512×512 + 512) = 1,050,624
```

## 6. Self-Attention（自注意力）

### 6.1 什么是 Self-Attention

Self-Attention 是注意力机制的一种特殊用法：**Query、Key、Value 都来自同一个序列**。即序列中的每个位置都和序列中所有位置（包括自身）计算注意力关系。

```
Cross-Attention:  Q 来自解码器，K/V 来自编码器（跨两个不同序列）
Self-Attention:   Q = K = V = 同一个序列的表示（序列自己关注自己）
```

### 6.2 Self-Attention 的核心优势

**优势一：O(1) 路径长度（全局直接连接）**

RNN 需要 O(L) 步才能让首尾两个 token 产生联系。Self-Attention 中任意两个位置之间只需 1 步计算就能直接交互：

```
序列: [The, cat, sat, on, the, mat]

RNN 中 "The" 到 "mat" 的信息流：
  The → cat → sat → on → the → mat （5步，信息逐步衰减）

Self-Attention 中 "The" 到 "mat"：
  The ←── 直接注意力连接 ──→ mat （1步，权重由相似度决定）
```

**优势二：完全可并行**

所有位置对 (i, j) 的注意力分数可以同时计算（一个矩阵乘法），而 RNN 必须串行处理。

**优势三：动态上下文感知**

每个 token 的表示会根据它在具体上下文中与其他 token 的关系动态调整：

```
"bank" 在不同上下文中的注意力模式：
  "I deposited money at the bank"  → bank 关注 money, deposited（金融义）
  "I sat by the river bank"        → bank 关注 river, sat（河岸义）
```

### 6.3 Self-Attention 的计算流程（图示）

```
输入序列 X = [x₁, x₂, x₃, x₄]  每个 xᵢ ∈ ℝ^d_model

  ┌─────────────────────────────────────────────┐
  │  Step 1: 线性投影得到 Q, K, V              │
  │                                             │
  │  Q = X · W_Q    K = X · W_K    V = X · W_V │
  │                                             │
  │  每个位置 i:                                │
  │    qᵢ = xᵢ·W_Q  ("我在找什么")            │
  │    kᵢ = xᵢ·W_K  ("我能提供什么")          │
  │    vᵢ = xᵢ·W_V  ("我的实际内容")          │
  └─────────────────────────────────────────────┘
           ↓
  ┌─────────────────────────────────────────────┐
  │  Step 2: 计算注意力分数矩阵                │
  │                                             │
  │       k₁   k₂   k₃   k₄                   │
  │  q₁ [ s₁₁  s₁₂  s₁₃  s₁₄ ]              │
  │  q₂ [ s₂₁  s₂₂  s₂₃  s₂₄ ]              │
  │  q₃ [ s₃₁  s₃₂  s₃₃  s₃₄ ]              │
  │  q₄ [ s₄₁  s₄₂  s₄₃  s₄₄ ]              │
  │                                             │
  │  sᵢⱼ = qᵢ · kⱼ / √d_k                    │
  └─────────────────────────────────────────────┘
           ↓
  ┌─────────────────────────────────────────────┐
  │  Step 3: Softmax → 注意力权重               │
  │                                             │
  │  每行做 softmax：αᵢⱼ = softmax(sᵢ)ⱼ       │
  │  每行和为 1，表示位置 i 对各位置的关注度   │
  └─────────────────────────────────────────────┘
           ↓
  ┌─────────────────────────────────────────────┐
  │  Step 4: 加权求和                           │
  │                                             │
  │  outputᵢ = Σⱼ αᵢⱼ · vⱼ                    │
  │                                             │
  │  每个位置的新表示 = 所有位置 value 的加权和 │
  └─────────────────────────────────────────────┘
```

### 6.4 Self-Attention vs Cross-Attention

| 维度 | Self-Attention | Cross-Attention |
|------|---------------|----------------|
| Q 来源 | 当前序列自身 | 解码器当前层 |
| K/V 来源 | 当前序列自身 | 编码器输出 |
| 用途 | 建模序列内部关系 | 建模序列间关系 |
| 典型场景 | Encoder 层、Decoder masked | Decoder 的 encoder-decoder attention |
| 矩阵形状 | (L, L) 方阵 | (L_q, L_k) 可能非方阵 |

### 6.5 Self-Attention 的复杂度

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|  
| QK^T 矩阵乘法 | O(L² · d) | O(L²) |
| Softmax | O(L²) | O(L²) |
| 注意力 × V | O(L² · d) | O(L · d) |
| **总计** | **O(L² · d)** | **O(L² + L·d)** |

其中 L 是序列长度，d 是模型维度。**L² 是 Self-Attention 的核心瓶颈**——当序列很长时（如 L=100K），注意力矩阵有 100K × 100K = 100亿个元素，显存和计算都难以承受。这催生了各种高效注意力变体（第 7 节详述）。

## 7. 注意力变体与改进

### 7.1 Multi-Query Attention (MQA)

**动机**：在自回归推理时，KV Cache 是显存的主要瓶颈。如果每个头都有独立的 K/V，缓存量 = 2 × h × L × d_k。

**做法**：所有注意力头共享同一组 K 和 V，只有 Q 是多头的：

```python
# 标准 Multi-Head: h 套 Q/K/V
Q: (B, h, L, d_k)   K: (B, h, L, d_k)   V: (B, h, L, d_k)

# Multi-Query: h 套 Q，但只有 1 套 K/V
Q: (B, h, L, d_k)   K: (B, 1, L, d_k)   V: (B, 1, L, d_k)
                     ↑ 广播到所有头
```

**效果**：KV Cache 减少 h 倍（8头→减少8倍），推理速度提升显著，质量损失很小。

**代表模型**：PaLM, StarCoder, Falcon

### 7.2 Grouped-Query Attention (GQA)

**动机**：MQA 只有 1 组 K/V 可能过于激进，GQA 是 MHA 和 MQA 的折中。

**做法**：将 h 个头分成 g 组，每组共享一套 K/V：

```python
# MHA:  h=8 组 K/V（每头独立）
# GQA:  g=2 组 K/V（每 4 个头共享 1 组）  
# MQA:  g=1 组 K/V（所有头共享）

# GQA 示例 (h=8, g=2):
Q: (B, 8, L, d_k)   # 8 个 query 头
K: (B, 2, L, d_k)   # 2 组 key（每组被 4 个 Q 头共享）
V: (B, 2, L, d_k)   # 2 组 value
```

**效果**：KV Cache 减少 h/g 倍，质量接近 MHA，推理接近 MQA 速度。

**代表模型**：LLaMA 2 (70B), Mistral, Gemma

### 7.3 FlashAttention

**动机**：标准注意力需要将完整的 L×L 注意力矩阵写入 HBM（GPU 高带宽显存），显存 IO 成为瓶颈。

**核心思想**：利用 GPU SRAM（片上内存）分块（tiling）计算注意力，**避免将完整 L×L 矩阵写入 HBM**。

```
标准 Attention 的 IO 复程：
  QK^T → 写入 HBM (L×L)  ← 瓶颈！
  softmax → 读/写 HBM
  × V → 写入 HBM

FlashAttention：
  将 Q/K/V 分成 blocks
  在 SRAM 中计算每个 block 的局部 softmax
  用 online softmax 技巧正确归并
  只将最终输出写入 HBM（L×d），跳过 L×L 中间结果
```

**效果**：
- 显存从 O(L²) 降为 O(L)
- 速度提升 2-4x（减少 HBM 读写）
- **精确计算**（不是近似），结果与标准注意力数学等价

**PyTorch 使用**：
```python
# PyTorch 2.0+ 内置 FlashAttention
from torch.nn.functional import scaled_dot_product_attention

# 自动选择最优 backend（FlashAttention / Memory-Efficient / Math）
output = scaled_dot_product_attention(Q, K, V, attn_mask=mask, is_causal=True)
```

### 7.4 Sparse Attention

**动机**：即使有 FlashAttention，O(L²) 的计算量在超长序列（>100K）时仍然很大。

**核心思想**：不计算所有 L² 个注意力对，只计算一个稀疏子集：

```
窗口注意力（Local/Window）：每个 token 只关注周围 w 个邻居
  复杂度: O(L·w)

全局+局部（Global+Local）：少数特殊 token 关注全局，其余只关注局部
  代表: Longformer, BigBird

块稀疏（Block Sparse）：将序列分块，只计算部分块之间的注意力
  代表: Sparse Transformer
```

### 7.5 Linear Attention

**动机**：能否将 O(L²) 降为 O(L)？

**核心思想**：利用核函数分解，改变计算顺序：

```
标准:  softmax(QK^T)V     先算 (L,L) 再乘 V → O(L²d)
线性:  φ(Q)(φ(K)^T V)    先算 (d,d) 再乘 Q → O(Ld²)

当 d << L 时（长序列场景），线性注意力显著更快
```

**代表**：Linear Transformer, RWKV, RetNet, Mamba（广义线性注意力/SSM）

### 7.6 各变体对比总结

| 变体 | 时间复杂度 | 空间复杂度 | KV Cache | 精确度 | 代表模型 |
|------|-----------|-----------|----------|--------|----------|
| MHA | O(L²d) | O(L²) | h×L×d_k | 精确 | GPT-2, BERT |
| MQA | O(L²d) | O(L²) | L×d_k | 精确 | PaLM |
| GQA | O(L²d) | O(L²) | g×L×d_k | 精确 | LLaMA 2 |
| FlashAttn | O(L²d) | O(L) | 不变 | 精确 | 通用加速 |
| Sparse | O(L·w·d) | O(L·w) | 不变 | 近似 | Longformer |
| Linear | O(L·d²) | O(d²) | O(d²) | 近似 | RWKV |

## 8. Mask 机制详解

### 8.1 为什么需要 Mask

注意力中的 Mask 有两个核心目的：

1. **Padding Mask**：处理 batch 中不等长序列的填充位置
2. **Causal Mask（因果掩码）**：在自回归生成中，防止当前位置看到未来信息

### 8.2 Padding Mask

批处理时，短序列会被 pad 到相同长度。我们不希望模型关注 padding token：

```python
# 序列: ["hello", "world", "<PAD>", "<PAD>"]
# padding_mask: [1, 1, 0, 0]   (1=有效, 0=padding)

# 扩展为注意力 mask:
# (B, 1, 1, L_k) — 广播到所有 query 位置和所有头
padding_mask = (input_ids != pad_token_id).unsqueeze(1).unsqueeze(2)
# shape: (B, 1, 1, L_k)

# 在 scores 中：padding 位置设为 -inf
scores = scores.masked_fill(padding_mask == 0, float("-inf"))
# softmax 后 padding 位置权重 = 0
```

### 8.3 Causal Mask（因果掩码/下三角掩码）

在 GPT 等自回归模型中，预测位置 t 的 token 时，只能看到位置 1~t（不能看未来）：

```python
# 因果掩码：下三角矩阵
# L=4 的例子:
#     pos1  pos2  pos3  pos4
# pos1 [1     0     0     0  ]   → 位置1只能看位置1
# pos2 [1     1     0     0  ]   → 位置2能看1,2
# pos3 [1     1     1     0  ]   → 位置3能看1,2,3
# pos4 [1     1     1     1  ]   → 位置4能看1,2,3,4

def create_causal_mask(L: int) -> torch.Tensor:
    """创建因果掩码（下三角为 1，上三角为 0）"""
    return torch.tril(torch.ones(L, L)).unsqueeze(0).unsqueeze(0)
    # shape: (1, 1, L, L) — 可广播到 (B, h, L, L)

# 使用
causal_mask = create_causal_mask(seq_len)  # (1, 1, L, L)
scores = scores.masked_fill(causal_mask == 0, float("-inf"))
```

### 8.4 组合 Mask

实际使用中通常需要同时应用 padding mask 和 causal mask：

```python
def create_combined_mask(
    seq_len: int,
    padding_mask: torch.Tensor,  # (B, L)  1=valid, 0=pad
    is_causal: bool = True
) -> torch.Tensor:
    """
    创建组合掩码
    Returns: (B, 1, L, L)  — 可广播到 (B, h, L, L)
    """
    B = padding_mask.size(0)
    
    # Padding mask: (B, L) → (B, 1, 1, L)
    pad_mask = padding_mask.unsqueeze(1).unsqueeze(2)  # (B, 1, 1, L)
    
    if is_causal:
        # Causal mask: (1, 1, L, L)
        causal = torch.tril(torch.ones(seq_len, seq_len, device=padding_mask.device))
        causal = causal.unsqueeze(0).unsqueeze(0)  # (1, 1, L, L)
        # 组合：两个条件都满足才允许注意
        combined = pad_mask * causal  # (B, 1, L, L) 广播乘法
    else:
        combined = pad_mask.expand(-1, -1, seq_len, -1)  # (B, 1, L, L)
    
    return combined
```

### 8.5 Mask 的数值技巧

为什么用 `-inf` 而不是 0 来遮蔽？

```python
softmax([-inf, 2.0, 3.0]) = [0.0, 0.269, 0.731]  # -inf 完美归零
softmax([0.0,  2.0, 3.0]) = [0.066, 0.245, 0.688]  # 0 仍有权重！
```

实践中用 `-1e9` 或 `float("-inf")`：
- `float("-inf")` 数学上完美，但某些混合精度下可能产生 NaN
- `-1e9` 或 `torch.finfo(dtype).min` 更安全

## 9. 数学推导与梯度分析

### 9.1 Softmax 的梯度

设 p = softmax(z)，即 pᵢ = exp(zᵢ) / Σⱼ exp(zⱼ)。

Softmax 的 Jacobian 矩阵：

```
∂pᵢ/∂zⱼ = pᵢ(δᵢⱼ - pⱼ)

其中 δᵢⱼ 是 Kronecker delta：
  i = j 时: ∂pᵢ/∂zᵢ = pᵢ(1 - pᵢ)
  i ≠ j 时: ∂pᵢ/∂zⱼ = -pᵢ·pⱼ
```

矩阵形式：∂p/∂z = diag(p) - ppᵀ

### 9.2 Scaled Dot-Product Attention 的反向传播

设前向计算为：
```
S = QK^T / √d_k          (scores)
P = softmax(S)            (attention weights)
O = PV                    (output)
```

给定上游梯度 dO（对 O 的梯度），我们需要计算 dQ, dK, dV：

**Step 1: dV 和 dP**
```
O = PV
dV = P^T · dO             # (L_k, L_q) × (L_q, d_v) → (L_k, d_v)
dP = dO · V^T             # (L_q, d_v) × (d_v, L_k) → (L_q, L_k)
```

**Step 2: dS（通过 softmax 反传）**
```
对每个 query 位置 i（即矩阵的每一行）:
  dSᵢ = Pᵢ ⊙ (dPᵢ - (dPᵢ · Pᵢ))

其中 ⊙ 是逐元素乘，dPᵢ·Pᵢ 是点积标量。

向量化写法：
  dS = P ⊙ (dP - (P ⊙ dP).sum(dim=-1, keepdim=True))
```

**Step 3: dQ 和 dK**
```
S = QK^T / √d_k
dQ = dS · K / √d_k        # (L_q, L_k) × (L_k, d_k) → (L_q, d_k)
dK = dS^T · Q / √d_k      # (L_k, L_q) × (L_q, d_k) → (L_k, d_k)
```

### 9.3 梯度消失/爆炸分析

**√d_k 缩放防止梯度消失**：

当 softmax 输入过大时（未缩放），输出接近 one-hot：
```
p ≈ [0, 0, ..., 1, ..., 0]
∂pᵢ/∂zᵢ = pᵢ(1-pᵢ) ≈ 1×0 = 0  （对峰值位置）
∂pᵢ/∂zⱼ = -pᵢpⱼ ≈ 0            （对其他位置）
```
梯度几乎全为 0，参数无法更新。缩放确保 softmax 输入在合理范围。

**残差连接保证梯度流**：

Transformer 中注意力层总是配合残差连接：
```
Output = X + Attention(X)     （残差连接）
∂Loss/∂X = ∂Loss/∂Output + ∂Loss/∂Output · ∂Attention/∂X
```
恒等映射的梯度为 1，保证梯度不会完全消失。

### 9.4 注意力的信息论视角

注意力权重 α = softmax(QK^T/√d_k) 形成概率分布。我们可以用熵来衡量注意力的「集中度」：

```
H(α) = -Σᵢ αᵢ log αᵢ

H(α) = 0        → 完全集中（one-hot），只看一个位置
H(α) = log(L)   → 完全均匀，等权看所有位置
```

实践中，深层网络的注意力熵通常经历：
- 早期训练：接近均匀分布（模型还没学会关注什么）
- 训练收敛后：低层头偏向局部模式（低熵），高层头更全局（较高熵）

### 10.1 从零实现完整的 Multi-Head Self-Attention（含所有细节）

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math
from typing import Optional, Tuple


class ScaledDotProductAttention(nn.Module):
    """
    Scaled Dot-Product Attention 的独立模块实现
    
    包含完整的 mask 支持、dropout、可选的注意力权重返回
    """
    
    def __init__(self, dropout: float = 0.0):
        super().__init__()
        self.dropout = nn.Dropout(dropout)
    
    def forward(
        self,
        Q: torch.Tensor,
        K: torch.Tensor,
        V: torch.Tensor,
        mask: Optional[torch.Tensor] = None,
        need_weights: bool = False
    ) -> Tuple[torch.Tensor, Optional[torch.Tensor]]:
        """
        Args:
            Q: (B, h, L_q, d_k)
            K: (B, h, L_k, d_k)  或 (B, 1, L_k, d_k) for MQA/GQA
            V: (B, h, L_k, d_v)  或 (B, 1, L_k, d_v) for MQA/GQA
            mask: (B, 1, L_q, L_k) 或 (1, 1, L_q, L_k)
        
        Returns:
            output: (B, h, L_q, d_v)
            attn_weights: (B, h, L_q, L_k) if need_weights else None
        """
        d_k = Q.size(-1)
        
        # (B, h, L_q, d_k) @ (B, h, d_k, L_k) → (B, h, L_q, L_k)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, torch.finfo(scores.dtype).min)
        
        attn_weights = F.softmax(scores, dim=-1)
        attn_weights = self.dropout(attn_weights)
        
        # (B, h, L_q, L_k) @ (B, h, L_k, d_v) → (B, h, L_q, d_v)
        output = torch.matmul(attn_weights, V)
        
        return output, attn_weights if need_weights else None


class MultiHeadSelfAttention(nn.Module):
    """
    完整的 Multi-Head Self-Attention 实现
    
    支持:
    - 标准 MHA (n_kv_heads = n_heads)
    - Grouped-Query Attention (1 < n_kv_heads < n_heads)
    - Multi-Query Attention (n_kv_heads = 1)
    - Pre-norm / Post-norm
    - Causal masking
    - KV Cache (推理加速)
    """
    
    def __init__(
        self,
        d_model: int = 512,
        n_heads: int = 8,
        n_kv_heads: Optional[int] = None,  # None = MHA, 1 = MQA, 其他 = GQA
        dropout: float = 0.1,
        bias: bool = False,
        max_seq_len: int = 2048
    ):
        super().__init__()
        
        self.d_model = d_model
        self.n_heads = n_heads
        self.n_kv_heads = n_kv_heads or n_heads  # 默认 MHA
        self.n_rep = self.n_heads // self.n_kv_heads  # 每组 KV 被多少个 Q 头共享
        self.d_k = d_model // n_heads
        
        assert d_model % n_heads == 0
        assert n_heads % self.n_kv_heads == 0
        
        # Q 投影：n_heads 个头
        self.W_q = nn.Linear(d_model, n_heads * self.d_k, bias=bias)
        # K/V 投影：n_kv_heads 个头（可能少于 Q）
        self.W_k = nn.Linear(d_model, self.n_kv_heads * self.d_k, bias=bias)
        self.W_v = nn.Linear(d_model, self.n_kv_heads * self.d_k, bias=bias)
        # 输出投影
        self.W_o = nn.Linear(n_heads * self.d_k, d_model, bias=bias)
        
        self.attention = ScaledDotProductAttention(dropout)
        self.resid_dropout = nn.Dropout(dropout)
        
        # 预计算 causal mask
        causal_mask = torch.tril(torch.ones(max_seq_len, max_seq_len))
        self.register_buffer("causal_mask", causal_mask.view(1, 1, max_seq_len, max_seq_len))
    
    def _repeat_kv(self, x: torch.Tensor) -> torch.Tensor:
        """将 KV 头重复以匹配 Q 头数（GQA/MQA 需要）"""
        if self.n_rep == 1:
            return x
        B, n_kv_heads, L, d_k = x.shape
        # (B, n_kv, L, d_k) → (B, n_kv, rep, L, d_k) → (B, n_heads, L, d_k)
        return (
            x[:, :, None, :, :]
            .expand(B, n_kv_heads, self.n_rep, L, d_k)
            .reshape(B, self.n_heads, L, d_k)
        )
    
    def forward(
        self,
        x: torch.Tensor,          # (B, L, d_model)
        is_causal: bool = False,
        mask: Optional[torch.Tensor] = None,
        kv_cache: Optional[Tuple[torch.Tensor, torch.Tensor]] = None,
        need_weights: bool = False
    ) -> Tuple[torch.Tensor, Optional[Tuple[torch.Tensor, torch.Tensor]]]:
        """
        Args:
            x: 输入 (B, L, d_model)
            is_causal: 是否使用因果掩码
            mask: 自定义 mask (B, 1, L_q, L_k)
            kv_cache: 之前缓存的 (K, V)，用于自回归推理
        
        Returns:
            output: (B, L, d_model)
            new_kv_cache: 更新后的 (K, V) cache
        """
        B, L, _ = x.shape
        
        # 1. 投影
        Q = self.W_q(x).view(B, L, self.n_heads, self.d_k).transpose(1, 2)
        K = self.W_k(x).view(B, L, self.n_kv_heads, self.d_k).transpose(1, 2)
        V = self.W_v(x).view(B, L, self.n_kv_heads, self.d_k).transpose(1, 2)
        
        # 2. KV Cache（推理时拼接历史）
        if kv_cache is not None:
            K_prev, V_prev = kv_cache
            K = torch.cat([K_prev, K], dim=2)  # 在序列维度拼接
            V = torch.cat([V_prev, V], dim=2)
        new_kv_cache = (K, V)
        
        # 3. GQA/MQA: 重复 K/V 以匹配 Q 头数
        K_expanded = self._repeat_kv(K)
        V_expanded = self._repeat_kv(V)
        
        # 4. 构建 mask
        L_k = K_expanded.size(2)
        if is_causal and mask is None:
            # 取 causal_mask 的对应子矩阵
            # 推理时 L_q=1（当前 token），L_k=历史长度+1
            start_pos = L_k - L
            mask = self.causal_mask[:, :, start_pos:start_pos+L, :L_k]
        
        # 5. Attention
        attn_output, attn_weights = self.attention(
            Q, K_expanded, V_expanded, mask=mask, need_weights=need_weights
        )
        
        # 6. 拼接 + 输出投影
        attn_output = attn_output.transpose(1, 2).contiguous().view(B, L, self.d_model)
        output = self.W_o(attn_output)
        output = self.resid_dropout(output)
        
        return output, new_kv_cache
```

```python
def attention_entropy(attn_weights: torch.Tensor) -> torch.Tensor:
    """
    计算注意力熵（衡量注意力集中/分散程度）
    
    Args:
        attn_weights: (B, h, L_q, L_k)
    
    Returns:
        entropy: (B, h, L_q)  每个 query 位置的注意力熵
    """
    # 避免 log(0)
    eps = 1e-10
    entropy = -(attn_weights * (attn_weights + eps).log()).sum(dim=-1)
    return entropy


def attention_distance(attn_weights: torch.Tensor) -> torch.Tensor:
    """
    计算平均注意力距离（衡量注意力的长距离/短距离偏好）
    
    Args:
        attn_weights: (B, h, L, L)  self-attention 权重
    
    Returns:
        avg_distance: (B, h)  每个头的平均注意力距离
    """
    L = attn_weights.size(-1)
    # 位置距离矩阵: |i - j|
    positions = torch.arange(L, device=attn_weights.device).float()
    distances = (positions.unsqueeze(0) - positions.unsqueeze(1)).abs()  # (L, L)
    
    # 加权平均距离
    avg_distance = (attn_weights * distances.unsqueeze(0).unsqueeze(0)).sum(dim=-1).mean(dim=-1)
    return avg_distance
```

### 10.3 高效 Attention 实现（利用 PyTorch 2.0+ SDPA）

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math


class EfficientMultiHeadAttention(nn.Module):
    """
    利用 PyTorch 2.0+ 的 scaled_dot_product_attention
    自动选择最优后端（FlashAttention-2 / Memory-Efficient / Math）
    """
    
    def __init__(
        self,
        d_model: int = 512,
        n_heads: int = 8,
        n_kv_heads: int = None,
        dropout: float = 0.1,
        bias: bool = False
    ):
        super().__init__()
        self.n_heads = n_heads
        self.n_kv_heads = n_kv_heads or n_heads
        self.d_k = d_model // n_heads
        self.n_rep = n_heads // self.n_kv_heads
        
        self.W_qkv = nn.Linear(
            d_model,
            (n_heads + 2 * self.n_kv_heads) * self.d_k,
            bias=bias
        )
        self.W_o = nn.Linear(d_model, d_model, bias=bias)
        self.dropout_p = dropout
    
    def forward(
        self,
        x: torch.Tensor,
        is_causal: bool = False,
        attn_mask: torch.Tensor = None
    ) -> torch.Tensor:
        B, L, _ = x.shape
        
        # 一次性投影 Q/K/V
        qkv = self.W_qkv(x)
        q_size = self.n_heads * self.d_k
        kv_size = self.n_kv_heads * self.d_k
        
        Q, K, V = qkv.split([q_size, kv_size, kv_size], dim=-1)
        
        Q = Q.view(B, L, self.n_heads, self.d_k).transpose(1, 2)
        K = K.view(B, L, self.n_kv_heads, self.d_k).transpose(1, 2)
        V = V.view(B, L, self.n_kv_heads, self.d_k).transpose(1, 2)
        
        # GQA: 重复 K/V
        if self.n_rep > 1:
            K = K.repeat_interleave(self.n_rep, dim=1)
            V = V.repeat_interleave(self.n_rep, dim=1)
        
        # 使用 PyTorch 内置 SDPA（自动选择 FlashAttention 等最优后端）
        output = F.scaled_dot_product_attention(
            Q, K, V,
            attn_mask=attn_mask,
            dropout_p=self.dropout_p if self.training else 0.0,
            is_causal=is_causal
        )
        
        output = output.transpose(1, 2).contiguous().view(B, L, -1)
        return self.W_o(output)
```

## 11. 可视化与直觉理解

### 11.1 注意力模式的直觉

研究发现，Transformer 不同层、不同头学会了截然不同的注意力模式：

```
┌─────────────────────────────────────────────────────────────┐
│  常见的注意力模式（通过分析真实模型发现）                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 对角线模式（关注自身）                                 │
│     ■ □ □ □ □       每个 token 最关注自己                  │
│     □ ■ □ □ □       常见于底层，保留原始信息               │
│     □ □ ■ □ □                                              │
│     □ □ □ ■ □                                              │
│     □ □ □ □ ■                                              │
│                                                             │
│  2. 前一个词模式（局部）                                   │
│     □ □ □ □ □       关注前一个位置                         │
│     ■ □ □ □ □       类似 bigram，捕获局部依赖             │
```

### 11.3 为什么 Self-Attention 效果如此好

从表示学习的角度，Self-Attention 让每个 token 的表示变成了**上下文感知的动态嵌入**：

```
静态嵌入（Word2Vec 时代）:
  "bank" → 同一个向量，无论上下文

Self-Attention 后的表示:
  "I went to the bank to deposit money"
    → bank 的表示 ≈ 0.3·went + 0.1·to + 0.4·deposit + 0.15·money + ...
    → 自动编码了金融含义

  "The river bank was covered with grass"
    → bank 的表示 ≈ 0.4·river + 0.1·covered + 0.2·grass + ...
    → 自动编码了河岸含义
```

这就是为什么 BERT/GPT 等模型能如此好地处理多义词——Self-Attention 天然实现了上下文化表示。

## 12. 工程实践与性能优化

### 12.1 KV Cache 详解

在自回归生成（如 GPT 推理）时，每生成一个 token 都需要对之前所有 token 做注意力。如果每次都重新计算所有 K/V，计算量是 O(L²) 总和。

**KV Cache 策略**：缓存已计算的 K/V，新 token 只需计算自己的 Q，然后与缓存中的所有 K/V 做注意力：

```
无 KV Cache（naive）:
  生成 token 1: 计算 Q₁K₁^T → O(1·d)
  生成 token 2: 计算 Q₂[K₁,K₂]^T → O(2·d)  重新算了 K₁
  生成 token 3: 计算 Q₃[K₁,K₂,K₃]^T → O(3·d)  重新算了 K₁,K₂
  ...
  生成 token L: O(L·d)
  总计: O(L²·d) 的 K/V 投影计算

有 KV Cache:
  生成 token 1: 计算 K₁,V₁ 并缓存 → O(d)
  生成 token 2: 计算 K₂,V₂ 并缓存, Q₂×[K₁,K₂]^T → O(d) + O(2d)
  生成 token 3: 计算 K₃,V₃ 并缓存, Q₃×[K₁,K₂,K₃]^T → O(d) + O(3d)
  ...
  K/V 投影总计: O(L·d)  （每个 token 只算一次）
  注意力计算总计: O(L²·d)  （不可避免）
```

**KV Cache 显存占用**：
```
每层每 token: 2 × n_kv_heads × d_k × dtype_size

LLaMA 2 7B (32层, 32头, d_k=128, fp16):
  每 token: 2 × 32 × 32 × 128 × 2 bytes = 512 KB
  4096 tokens: 2 GB
  
LLaMA 2 70B (80层, GQA 8 KV头, d_k=128, fp16):
  每 token: 2 × 80 × 8 × 128 × 2 bytes = 320 KB
  4096 tokens: 1.28 GB  （GQA 大幅减少！）
```

### 12.2 注意力中的数值稳定性

**问题 1：Softmax 溢出**

```python
# 直接计算 softmax 可能溢出
scores = torch.tensor([1000.0, 1001.0, 999.0])
torch.exp(scores)  # inf! 指数爆炸

# 安全做法：减去最大值（数学等价）
# softmax(x) = softmax(x - max(x))
scores_safe = scores - scores.max()  # [-1, 0, -2]
torch.exp(scores_safe)  # 安全范围内
```

PyTorch 的 `F.softmax` 内部已经做了这个优化。

**问题 2：全 -inf 行**

当 mask 导致某一行全部被遮蔽时：
```python
scores = torch.tensor([[float("-inf"), float("-inf"), float("-inf")]])
F.softmax(scores, dim=-1)  # [nan, nan, nan]!  0/0 = nan

# 解决：检测全 mask 行，将 nan 替换为 0
attn_weights = F.softmax(scores, dim=-1)
attn_weights = attn_weights.nan_to_num(0.0)
```

**问题 3：混合精度训练中的精度损失**

```python
# fp16 下大 softmax 输入可能损失精度
# 最佳实践：QK^T/√d_k 在 fp32 下计算 softmax
with torch.cuda.amp.autocast(enabled=False):
    scores = torch.matmul(Q.float(), K.float().transpose(-2, -1)) / math.sqrt(d_k)
    attn_weights = F.softmax(scores, dim=-1).to(V.dtype)
```

### 12.5 Attention 的常见 Bug 与调试

```python
# Bug 1: mask 维度不匹配
# 错误: mask (B, L) 直接用于 (B, h, L, L) 的 scores

# 正确: 对 key 维度做 softmax（每个 query 的权重分布）
attn = F.softmax(scores, dim=-1)  # 行和为1

# Bug 4: 忘记 contiguous()
# transpose 后直接 view 会报错
context = attn_output.transpose(1, 2).view(B, L, d_model)  # 可能报错
context = attn_output.transpose(1, 2).contiguous().view(B, L, d_model)  # 正确

# Bug 5: KV Cache 拼接维度错误
# 错误: 在头维度拼接

K = torch.cat([K_cache, K_new], dim=2)  # (B, h, L_old+L_new, d_k)
```

### 13.1 Attention vs RNN vs CNN

| 维度 | Self-Attention | RNN/LSTM | CNN |
|------|---------------|----------|-----|
| 最大路径长度 | O(1) | O(L) | O(L/k) |
| 每层计算复杂度 | O(L²·d) | O(L·d²) | O(k·L·d²) |
| 并行度 | 完全并行 | 串行 | 完全并行 |

| 归纳偏置 | 弱（需要位置编码） | 强（序列性） | 中（局部性） |

**关键洞察**：Self-Attention 在 L < d 时计算量比 RNN 小（短序列），在 L > d 时计算量更大（长序列）。但 Attention 的优势在于并行性和长距离建模能力。

| 小维度效果 | 更好 | 可能差（方差问题） |
| 大维度效果 | 类似 | 需要缩放（√d_k） |

## 14. 常见面试题（附答案要点）

### 面试题 1：为什么 Attention 要除以 √d_k？

**答案要点**：

假设 Q 和 K 的各维度元素独立同分布（均值 0，方差 1），那么 Q^T·K 的方差为 d_k。当 d_k 较大时，点积值的方差很大，部分值的绝对值会很大。这些大值输入 softmax 后，输出接近 one-hot 分布（一个位置接近 1，其余接近 0），导致梯度趋近于 0（因为 softmax 在极端值处的梯度 p(1-p) ≈ 0），模型难以训练。除以 √d_k 将点积的方差归一化为 1，使 softmax 保持在梯度健康的区域。

### 面试题 2：Self-Attention 的计算复杂度是多少？为什么是瓶颈？

**答案要点**：

时间复杂度 O(L²·d)，空间复杂度 O(L²)（存储注意力矩阵）。其中 L 是序列长度，d 是模型维度。当序列很长时（L=100K+），L² 项主导计算和显存消耗。例如 L=128K, d=4096 时，一个注意力矩阵需要 128K×128K×2B(fp16) ≈ 32GB，远超单卡显存。这催生了 FlashAttention（减少显存 IO）、稀疏注意力（减少计算量）和线性注意力（O(L)复杂度）等优化方案。

### 面试题 3：Multi-Head Attention 相比 Single-Head 有什么优势？计算量变了吗？

**答案要点**：

多头的优势是让模型在不同的表示子空间中学习不同的注意力模式（如语法、语义、位置等关系），增强了表达能力。计算量没有增加——单头用 d_model 维度计算一次 O(L²·d_model)，多头用 h 个 d_k 维度各计算一次 h×O(L²·d_k)，由于 d_k = d_model/h，总量相同。但多头用更少的参数（h 个小子空间）学到了更丰富的模式，实际效果显著优于单头。

### 面试题 4：解释 KV Cache 的原理和作用

**答案要点**：

在自回归生成中，每生成一个新 token 需要对之前所有 token 做注意力。如果每次重新计算所有位置的 K 和 V 投影，会有大量重复计算（已生成 token 的 K/V 不会变）。KV Cache 策略是将已计算的 K/V 向量缓存在显存中，新 token 只需计算自己的 Q/K/V，然后将新 K/V 追加到缓存中，Q 与完整缓存做注意力。这将每步的投影计算从 O(L·d²) 降为 O(d²)（只投影 1 个 token），代价是额外的显存占用（每层约 2×n_kv_heads×L×d_k×dtype_size）。

### 面试题 5：FlashAttention 是近似算法吗？它如何减少显存？

**答案要点**：

FlashAttention 不是近似算法，它与标准 Attention 结果完全相同（精确）。它的核心是利用 GPU 的内存层次结构（SRAM vs HBM）：标准实现需要将 L×L 的注意力矩阵整体写入 HBM（GPU 全局显存），IO 成为瓶颈。FlashAttention 通过 tiling（分块）策略，在 SRAM（片上高速缓存）中分块计算注意力，使用 online softmax 算法（逐块更新 softmax 的分母），只将最终 O(L×d) 的输出写入 HBM，完全避免了 O(L²) 中间矩阵的显存写入。这将空间复杂度从 O(L²) 降到 O(L)，速度提升 2-4x。

### 面试题 6：GQA 与 MQA 的区别是什么？为什么 LLaMA 2 选择 GQA？

**答案要点**：

MQA（Multi-Query Attention）让所有 Q 头共享同一组 K/V，GQA（Grouped-Query Attention）将 Q 头分成 g 组，每组共享一套 K/V。MQA 是 GQA 的极端情况（g=1）。LLaMA 2 70B 选择 GQA（h=64, g=8）的原因是：MQA 在大模型上质量损失不可忽略，而 GQA 在质量接近 MHA 的同时，将 KV Cache 减少到原来的 1/8（64/8=8 组 KV），推理速度大幅提升。实验表明 GQA 在质量-效率 trade-off 上是最优的折中。

### 面试题 7：Attention 机制有什么缺点或局限？

**答案要点**：

主要局限：(1) O(L²) 复杂度，限制了最大序列长度；(2) 缺乏位置信息——Attention 本身是排列不变的（permutation invariant），需要额外的位置编码来注入顺序信息；(3) 缺乏归纳偏置——不像 RNN 天然有序列性、CNN 有局部性，Attention 需要更多数据才能学到这些模式；(4) 注意力权重不可解释——虽然可以可视化，但研究表明注意力权重不一定等于"重要性"（attention ≠ explanation）。

### 面试题 8：Attention 中的 Dropout 应该加在哪里？为什么？

**答案要点**：

两处 Dropout：(1) **注意力权重 Dropout**（在 softmax 之后、乘 V 之前）：随机将部分注意力连接置零，迫使模型不过度依赖某些位置，增强泛化性。这相当于随机丢弃一些 "信息来源"，类似于 DropConnect。(2) **残差 Dropout**（在输出投影之后、加上残差之前）：标准的隐藏层 Dropout。注意：**不能在 softmax 之前加 Dropout**，因为 softmax 前的值代表 "分数"，dropout 会破坏相对大小关系并导致 softmax 输出不合理。

### 面试题 9：如何处理超长序列（100K+ tokens）的注意力计算？

**答案要点**：

多种策略组合使用：(1) FlashAttention-2 减少显存 IO，支持更长序列；(2) 滑动窗口注意力（Sliding Window Attention）如 Mistral/Longformer，每个 token 只关注最近 w 个位置，O(L·w)；(3) 稀疏注意力（全局 token + 局部窗口）如 BigBird；(4) Ring Attention 分布式跨设备切分序列维度；(5) 位置编码外推（RoPE + NTK-aware scaling / YaRN）让模型泛化到训练长度之外；(6) 线性注意力/SSM（如 Mamba）从根本上消除 L² 依赖。实践中通常是 RoPE + FlashAttention + GQA 的组合。

### 面试题 10：Softmax Attention 与 Linear Attention 有什么本质区别？

**答案要点**：

标准 softmax 注意力：Attn = softmax(QK^T)V，必须先算 L×L 矩阵再乘 V → O(L²d)。Linear Attention 用核函数 φ 替代 softmax：Attn = φ(Q)(φ(K)^T V)，利用结合律先算 φ(K)^T V（d×d 矩阵），再左乘 φ(Q) → O(Ld²)。当 L >> d 时线性注意力更快。但线性注意力的表达能力较弱——softmax 的非线性归一化赋予了标准注意力 "选择性聚焦" 的能力（尖锐分布），而线性注意力的权重分布相对平坦，在需要精确定位的任务上效果略逊。

## 15. 易错点与最佳实践

### 15.1 十大易错点

**1. 忘记缩放（÷√d_k）**
```python
# 错误：直接点积
scores = Q @ K.T  # 维度越大，值越大，梯度越消失
# 正确：
scores = Q @ K.T / math.sqrt(d_k)
```

**2. Softmax 维度搞错**
```python
# 错误：对 query 维度归一化（每列和为1）
weights = F.softmax(scores, dim=-2)
# 正确：对 key 维度归一化（每行和为1）
weights = F.softmax(scores, dim=-1)
```

**3. Mask 的布尔含义不统一**
```python
# 有的库 mask=True 表示保留（PyTorch nn.Transformer）
# 有的库 mask=True 表示屏蔽（torch.masked_fill）
# 务必查清当前框架的约定！

# PyTorch masked_fill: True 的位置被填充
scores.masked_fill(mask == True, -inf)  # mask=True → 屏蔽

# PyTorch nn.MultiheadAttention: key_padding_mask=True → 忽略该位置
```

**4. Causal mask 起始位置错误（推理时）**
```python
# 错误：始终从 position 0 开始
mask = torch.tril(torch.ones(L_new, L_new))  # 只看新 token 之间的关系

# 正确：KV cache 模式下，query 是新 token，key 包含所有历史
# mask 应为 (L_new, L_total)，允许看所有历史 + 自身因果
```

**5. transpose 后忘记 contiguous()**
```python
# view/reshape 要求内存连续
out = attn.transpose(1, 2)  # 非连续!
out.view(B, L, d_model)  # RuntimeError!

# 正确：
out = attn.transpose(1, 2).contiguous().view(B, L, d_model)
# 或用 reshape（自动处理，但可能有额外拷贝）
out = attn.transpose(1, 2).reshape(B, L, d_model)
```

**6. 混合精度下 Softmax 精度不足**
```python
# fp16 的 softmax 在大值时精度差
# 最佳实践：softmax 始终在 fp32 下计算
scores_fp32 = scores.float()
attn_weights = F.softmax(scores_fp32, dim=-1).to(scores.dtype)
```

**7. QKV 投影的 bias 选择**
```python
# 原版 Transformer: 有 bias
# LLaMA/现代模型: 无 bias（减少参数，配合 RMSNorm 更稳定）
# 注意：有 bias 时初始化要让 bias=0，否则初始注意力模式会有偏差
```

**8. Dropout 在推理时未关闭**
```python
# 错误：推理时仍然 dropout
model.train()  # 忘记切换模式
output = model(x)

# 正确：
model.eval()
with torch.no_grad():
    output = model(x)
```

**9. 多头数量不能整除模型维度**
```python
# d_model=768, n_heads=10 → 768/10=76.8 不整除！
# 必须保证 d_model % n_heads == 0
assert d_model % n_heads == 0, f"{d_model} not divisible by {n_heads}"
```

**10. Cross-Attention 中 K/V 长度与 Q 搞混**
```python
# Cross-Attention 中：
# Q 来自 decoder (长度 L_dec)
# K/V 来自 encoder (长度 L_enc)
# 注意力矩阵: (L_dec, L_enc) 不是方阵！

# 错误：mask 用了 (L_dec, L_dec)
mask = torch.ones(L_dec, L_dec)  # 维度不匹配
# 正确：
mask = padding_mask_encoder  # shape (B, 1, 1, L_enc)
```

### 15.2 最佳实践总结

**训练阶段**：
- 使用 Pre-LayerNorm（Norm 在 Attention 之前），训练更稳定
- Attention dropout + Residual dropout 各设 0.1（小模型可到 0.3）
- 初始化 W_O 时用较小的标准差（如 0.02/√(2·n_layers)），避免残差路径信号太大
- 使用 FlashAttention（PyTorch 2.0+ 的 SDPA 自动启用）
- 大模型用 GQA 而非 MHA，几乎无质量损失

**推理阶段**：
- 必须使用 KV Cache，否则生成速度 O(L²) 不可接受
- 对 KV Cache 做 int8 量化可减半显存，精度损失极小
- 使用 PagedAttention（vLLM）管理 KV Cache，避免显存碎片
- Batch 推理时用 Continuous Batching 提高 GPU 利用率

**调试与验证**：
- 可视化注意力权重，检查是否学到了合理的模式
- 监控注意力熵：太低（过度集中）或太高（完全均匀）都可能有问题
- 检查注意力权重每行是否和为 1（排除 nan/inf）
- 对比有无 √d_k 缩放的训练曲线

```
│                                                                     │
│  Q ∈ ℝ^(L_q × d_k)                                                │

├─────────────────────────────────────────────────────────────────────┤
│  Multi-Head Attention                                               │
│                                                                     │
│  MultiHead(Q, K, V) = Concat(head₁, ..., headₕ) · W^O             │
│  headᵢ = Attention(Q·Wᵢ^Q, K·Wᵢ^K, V·Wᵢ^V)                     │
│                                                                     │
│  Wᵢ^Q ∈ ℝ^(d_model × d_k)                                        │
│  Wᵢ^K ∈ ℝ^(d_model × d_k)                                        │
│  Wᵢ^V ∈ ℝ^(d_model × d_v)                                        │
│  W^O ∈ ℝ^(h·d_v × d_model)                                       │
├─────────────────────────────────────────────────────────────────────┤
│  复杂度                                                             │
│                                                                     │
│  Self-Attention: O(L² · d) 时间, O(L²) 空间                       │

│                                                                     │

```
