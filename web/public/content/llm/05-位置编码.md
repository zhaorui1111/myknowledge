## 1. 为什么 Transformer 需要位置编码

### 1.1 Self-Attention 的置换不变性

回顾标准 Self-Attention 的计算：

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
$$

其中 $Q = XW^Q$, $K = XW^K$, $V = XW^V$。如果我们对输入序列 $X = [x_1, x_2, \ldots, x_n]$ 施加任意排列 $\pi$，得到 $X' = [x_{\pi(1)}, x_{\pi(2)}, \ldots, x_{\pi(n)}]$，那么输出也仅仅是对应排列后的结果：

$$
\text{Attention}(\pi(X)) = \pi(\text{Attention}(X))
$$

这意味着 Self-Attention **天然不区分 token 的顺序**。对于语言任务，"The cat sat on the mat" 和 "mat the on sat cat The" 在没有位置信息时，对模型来说是等价的。

**直觉理解**：Self-Attention 本质是一个集合操作（set operation），而非序列操作（sequence operation）。要让它处理序列，必须额外注入位置信息。

### 1.2 位置编码的本质目标

位置编码需要满足以下数学性质：

1. **唯一性**：不同位置的编码必须可区分，即 $PE(i) \neq PE(j)$ 当 $i \neq j$
2. **有界性**：编码值应有界，不随位置增长而爆炸，避免与 token embedding 的量级失衡
3. **规律性**：相邻位置的编码应"相近"，体现序列的连续性
4. **可泛化性**（理想情况）：能处理训练时未见过的更长序列（外推能力）
5. **高效性**：计算代价低，不显著增加模型参数或 FLOPs

### 1.3 位置编码的三种注入方式

根据位置信息注入 Transformer 的位置，可分为三种方式：

| 注入方式 | 公式 | 代表方法 |
|---------|------|---------|
| 加到输入 Embedding | $h_i = x_i + PE(i)$ | Sinusoidal, Learned PE |
| 加到 Attention Score | $a_{ij} = q_i^\top k_j + b(i, j)$ | T5 Bias, ALiBi |
| 融入 Q/K 计算 | $q_i = f(x_i, i), k_j = f(x_j, j)$ | RoPE, CoPE |

每种方式各有优劣，后文将逐一详解。

## 2. 位置编码的分类体系

### 2.1 按编码范围分

- **绝对位置编码（Absolute PE）**：每个位置 $i$ 有一个独立的编码向量 $PE(i) \in \mathbb{R}^d$，编码的是位置本身
- **相对位置编码（Relative PE）**：编码位置之间的距离 $i - j$，关注的是 token 对之间的相对关系

### 2.2 按是否可学习分

- **固定编码（Fixed）**：通过确定性公式生成，无需训练（如 Sinusoidal）
- **可学习编码（Learnable）**：作为模型参数通过梯度下降学习（如 BERT/GPT 的 learned PE）
- **混合式**：固定公式 + 可学习缩放因子（如某些 RoPE 变体）

### 2.3 按外推能力分

- **无外推能力**：只能处理训练时见过的长度（如 Learned PE）
- **有限外推**：可处理略超训练长度的序列但性能递减（如 Sinusoidal）
- **强外推**：显式设计为支持长序列外推（如 ALiBi、RoPE + YaRN）

### 2.4 发展时间线

## 3. 绝对位置编码：正弦余弦（Sinusoidal）

### 3.1 核心公式

原始 Transformer 论文提出的正弦余弦位置编码，对位置 $pos$ 和维度 $i$：

$$
PE(pos, 2i) = \sin\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)
$$

$$
PE(pos, 2i+1) = \cos\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)
$$

其中：
- $pos \in \{0, 1, 2, \ldots, L-1\}$ 是位置索引
- $i \in \{0, 1, 2, \ldots, d_{\text{model}}/2 - 1\}$ 是维度索引
- $d_{\text{model}}$ 是模型隐藏层维度
- $10000$ 是基础频率（base frequency）

### 3.2 数学直觉：为什么是正弦/余弦

**频率视角**：每一对 (sin, cos) 对应一个不同频率的"时钟"。低维度的 (i 小) 对应高频振荡（变化快），高维度的 (i 大) 对应低频振荡（变化慢）。这类似于二进制编码中低位翻转快、高位翻转慢：

```
位置 0:  0 0 0 0
位置 1:  0 0 0 1    ← 最低位每步翻转
位置 2:  0 0 1 0
位置 3:  0 0 1 1
位置 4:  0 1 0 0    ← 第二位每2步翻转
位置 5:  0 1 0 1
位置 6:  0 1 1 0
位置 7:  0 1 1 1
位置 8:  1 0 0 0    ← 最高位每8步翻转
```

正弦编码是这种思想的**连续化版本**：用不同频率的正弦波代替离散的二进制位。

**波长范围**：频率从 $2\pi$（$i=0$ 时，波长为 $2\pi \approx 6.28$）到 $10000 \cdot 2\pi$（$i = d/2 - 1$ 时，波长约 62832）。这意味着：
- 最低维度：每约 6 个位置完成一个完整周期
- 最高维度：每约 62832 个位置完成一个周期

### 3.3 关键性质：线性变换可表达相对位置

正弦编码的核心设计意图是：存在一个**线性变换**矩阵 $M_k$，使得：

$$
PE(pos + k) = M_k \cdot PE(pos)
$$

具体来说，对于每一对维度 $(2i, 2i+1)$：

$$
\begin{bmatrix} \sin(\omega_i(pos+k)) \\ \cos(\omega_i(pos+k)) \end{bmatrix} = \begin{bmatrix} \cos(\omega_i k) & \sin(\omega_i k) \\ -\sin(\omega_i k) & \cos(\omega_i k) \end{bmatrix} \begin{bmatrix} \sin(\omega_i \cdot pos) \\ \cos(\omega_i \cdot pos) \end{bmatrix}
$$

其中 $\omega_i = 1/10000^{2i/d}$。

**意义**：这意味着模型理论上可以通过学习线性投影来捕获相对位置关系。不过实践中，这种能力是否被有效利用是有争议的。

### 3.4 点积与距离特性

两个位置编码之间的点积：

$$
PE(pos)^\top PE(pos + k) = \sum_{i=0}^{d/2-1} \cos(\omega_i k)
$$

这个点积**只依赖于相对距离 $k$**，不依赖于绝对位置 $pos$，这是正弦编码的一个优美性质。

**证明**：

$$
\begin{aligned}
&\sin(\omega_i \cdot pos)\sin(\omega_i(pos+k)) + \cos(\omega_i \cdot pos)\cos(\omega_i(pos+k)) \\
&= \cos(\omega_i(pos+k) - \omega_i \cdot pos) \quad \text{(和差化积)} \\
&= \cos(\omega_i k)
\end{aligned}
$$

对所有维度对求和即得上式。

### 3.5 PyTorch 实现

```python
import torch
import torch.nn as nn
import math

class SinusoidalPositionalEncoding(nn.Module):
    """
    原始 Transformer 的正弦余弦位置编码
    
    参数:
        d_model: 模型隐藏维度
        max_len: 支持的最大序列长度
        dropout: dropout 概率
    """
    
    def __init__(self, d_model: int, max_len: int = 5000, dropout: float = 0.1):
        super().__init__()
        self.dropout = nn.Dropout(p=dropout)
        
        # 创建位置编码矩阵 [max_len, d_model]
        pe = torch.zeros(max_len, d_model)
        
        # 位置索引 [max_len, 1]
        position = torch.arange(0, max_len, dtype=torch.float32).unsqueeze(1)
        
        # 频率分母项：10000^(2i/d_model)
        # 使用 exp(log) 技巧避免大数幂运算的数值问题
        div_term = torch.exp(
            torch.arange(0, d_model, 2, dtype=torch.float32) 
            * (-math.log(10000.0) / d_model)
        )
        # div_term shape: [d_model/2]
        # 等价于 1 / (10000^(2i/d_model))
        
        # 偶数维度用 sin，奇数维度用 cos
        pe[:, 0::2] = torch.sin(position * div_term)  # [max_len, d_model/2]
        pe[:, 1::2] = torch.cos(position * div_term)  # [max_len, d_model/2]
        
        # 添加 batch 维度 [1, max_len, d_model]
        pe = pe.unsqueeze(0)
        
        # 注册为 buffer（不参与梯度更新，但会随模型保存/加载/设备移动）
        self.register_buffer('pe', pe)
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        Args:
            x: 输入 tensor, shape [batch_size, seq_len, d_model]
        Returns:
            加上位置编码后的 tensor, shape 不变
        """
        # 取前 seq_len 个位置的编码，广播加到 x 上
        x = x + self.pe[:, :x.size(1), :]
        return self.dropout(x)


# 验证实现
def verify_sinusoidal_pe():
    """验证正弦位置编码的关键性质"""
    d_model = 512
    max_len = 1000
    
    pe_module = SinusoidalPositionalEncoding(d_model, max_len, dropout=0.0)
    pe = pe_module.pe.squeeze(0)  # [max_len, d_model]
    
    # 性质1: 不同位置编码不同
    assert not torch.allclose(pe[0], pe[1])
    assert not torch.allclose(pe[10], pe[20])
    print("✓ 不同位置编码互不相同")
    
    # 性质2: 点积只依赖相对距离
    # PE(3)·PE(8) 应约等于 PE(100)·PE(105)（相对距离都是5）
    dot_3_8 = torch.dot(pe[3], pe[8])
    dot_100_105 = torch.dot(pe[100], pe[105])
    assert torch.allclose(dot_3_8, dot_100_105, atol=1e-5)
    print(f"✓ PE(3)·PE(8) = {dot_3_8:.4f}, PE(100)·PE(105) = {dot_100_105:.4f}")
    
    # 性质3: 编码有界（值在 [-1, 1]）
    assert pe.min() >= -1.0 and pe.max() <= 1.0
    print("✓ 编码值有界 ∈ [-1, 1]")
    
    # 性质4: 编码范数大致恒定
    norms = pe.norm(dim=1)
    print(f"✓ 编码 L2 范数: mean={norms.mean():.2f}, std={norms.std():.4f}")


verify_sinusoidal_pe()
```

### 3.6 正弦编码的优缺点

**优点**：
- 无需训练参数，计算简单高效
- 编码有界，不会造成梯度问题
- 理论上通过线性变换可捕获相对位置
- 可以处理比训练长度更长的序列（有一定外推能力）

**缺点**：
- 实验表明其外推能力有限，超出训练长度后性能下降
- 位置信息与语义信息通过加法混合，可能互相干扰
- 在大规模预训练中，可学习位置编码通常表现更好
- 点积的相对距离性质在经过多层非线性变换后可能被稀释

## 4. 绝对位置编码：可学习位置嵌入（Learned Positional Embedding）

### 4.1 核心思想

最直接的方式：为每个位置创建一个可训练的向量：

$$
PE(pos) = E_{pos}, \quad E \in \mathbb{R}^{L_{\max} \times d_{\text{model}}}
$$

其中 $E$ 是一个参数矩阵，$L_{\max}$ 是预定义的最大序列长度。

使用时，将位置嵌入加到 token embedding 上：

$$
h_i = \text{TokenEmbed}(x_i) + E_i
$$

### 4.2 谁在用

- **BERT**：最大长度 512，使用 learned PE
- **GPT-1/2**：最大长度 512/1024，使用 learned PE
- **ViT（Vision Transformer）**：patch 位置使用 learned PE

### 4.3 PyTorch 实现

```python
class LearnedPositionalEmbedding(nn.Module):
    """
    可学习位置嵌入（BERT / GPT-2 风格）
    
    参数:
        max_len: 最大序列长度（决定参数量）
        d_model: 嵌入维度
    """
    
    def __init__(self, max_len: int, d_model: int, dropout: float = 0.1):
        super().__init__()
        self.position_embedding = nn.Embedding(max_len, d_model)
        self.dropout = nn.Dropout(p=dropout)
        
        self.register_buffer(
            'position_ids',
            torch.arange(max_len).unsqueeze(0)  # [1, max_len]
        )
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        Args:
            x: [batch_size, seq_len, d_model]
        """
        seq_len = x.size(1)
        
        if seq_len > self.position_embedding.num_embeddings:
            raise ValueError(
                f"序列长度 {seq_len} 超出最大位置编码长度 "
                f"{self.position_embedding.num_embeddings}"
            )
        
        position_ids = self.position_ids[:, :seq_len]  # [1, seq_len]
        pos_embed = self.position_embedding(position_ids)  # [1, seq_len, d_model]
        
        x = x + pos_embed  # 广播加法
        return self.dropout(x)
    
    @property
    def num_parameters(self) -> int:
        return self.position_embedding.num_embeddings * self.position_embedding.embedding_dim


# 使用示例
model = LearnedPositionalEmbedding(max_len=1024, d_model=768)
print(f"参数量: {model.num_parameters:,} = {model.num_parameters/1e6:.2f}M")
# 输出: 参数量: 786,432 = 0.79M
```

### 4.4 Learned PE vs Sinusoidal PE 实验对比

Vaswani et al. (2017) 在原始论文中指出两者性能几乎相同：

| 方法 | WMT En→De BLEU | 参数量 |
|------|----------------|--------|
| Sinusoidal PE | 25.8 | 0 |
| Learned PE | 25.7 | L×d |

但在后续大规模预训练中（如 GPT-2、BERT），learned PE 成为主流，原因：
- 大数据量下，可学习编码能适应数据分布的特殊模式
- 训练成本可以负担（参数量相对整个模型很小）
- 实验证明在 fine-tuning 场景下更灵活

### 4.5 优缺点

**优点**：
- 实现极其简单（一个 nn.Embedding）
- 模型可以自由学习最优的位置表示
- 在固定长度场景下通常略优于正弦编码

**缺点**：
- **零外推能力**：超过 $L_{\max}$ 的位置完全没有见过，无法处理
- 额外参数：$L_{\max} \times d$ 个参数（虽然通常很小）
- 学到的编码不易解释

## 5. 相对位置编码：核心思想与 Transformer-XL

### 5.1 从绝对到相对

绝对位置编码的核心假设是"位置本身重要"。但在语言中，更多时候是"位置之间的距离"更重要。例如：

- "I love you" 中 "love" 和 "you" 的关系主要取决于它们相邻，而不是它们分别在位置 1 和 2
- 同样的短语出现在句子中间或末尾，其内部关系应该一样

相对位置编码的核心思想：**在注意力计算时，让模型显式感知 query 和 key 之间的距离**。

### 5.2 标准 Attention 中位置信息的分解

假设使用绝对位置编码 $PE(i)$ 加到 embedding 上：

$$
q_i = (x_i + PE(i))W^Q, \quad k_j = (x_j + PE(j))W^K
$$

注意力分数为：

$$
\begin{aligned}
a_{ij} &= q_i^\top k_j \\
&= (x_i + PE(i))W^Q W^{K\top}(x_j + PE(j))^\top \\
&= \underbrace{x_i W^Q W^{K\top} x_j^\top}_{(a) \text{ 纯内容交互}} + \underbrace{x_i W^Q W^{K\top} PE(j)^\top}_{(b) \text{ 内容→位置}} \\
&\quad + \underbrace{PE(i) W^Q W^{K\top} x_j^\top}_{(c) \text{ 位置→内容}} + \underbrace{PE(i) W^Q W^{K\top} PE(j)^\top}_{(d) \text{ 位置→位置}}
\end{aligned}
$$

四个分项中，$(b)(c)(d)$ 都包含位置信息。相对位置编码的各种方法本质上都在修改这些项。

### 5.3 Transformer-XL 的相对位置编码

Dai et al. (2019) 的 Transformer-XL 提出了一种经典的相对位置编码方案。核心修改：

**将上述 (b)(c)(d) 项中的绝对位置替换为相对位置**：

$$
a_{ij} = \underbrace{x_i W^Q {W_E^K}^\top x_j^\top}_{(a)} + \underbrace{x_i W^Q {W_R^K}^\top R_{i-j}^\top}_{(b')} + \underbrace{u^\top {W_E^K}^\top x_j^\top}_{(c')} + \underbrace{v^\top {W_R^K}^\top R_{i-j}^\top}_{(d')}
$$

其中：
- $R_{i-j}$ 是相对位置 $i-j$ 对应的正弦编码向量
- $u, v \in \mathbb{R}^d$ 是可学习的全局偏置向量，替代了位置相关的 query
- $W_E^K$ 是内容的 Key 投影，$W_R^K$ 是位置的 Key 投影（分离了两个职责）

**四项的语义解读**：
- $(a)$：内容之间的关联（"what" to "what"）
- $(b')$：内容对相对位置的关注（"what" to "where"）——当前 token 的内容决定它应该关注多远的 token
- $(c')$：全局内容偏置（某些 token 天然更容易被关注）
- $(d')$：全局距离偏置（某些相对距离天然更重要，如相邻的 token）

### 5.4 Transformer-XL 实现

```python
class TransformerXLRelativeAttention(nn.Module):
    """
    Transformer-XL 风格的相对位置注意力
    
    核心改动：
    1. Key 分为内容 Key 和位置 Key 两个投影
    2. Query 的位置部分用全局可学习向量 u, v 替代
    3. 相对位置用正弦编码表示
    """
    
    def __init__(self, d_model: int, n_heads: int, d_head: int, max_rel_dist: int = 512):
        super().__init__()
        self.d_model = d_model
        self.n_heads = n_heads
        self.d_head = d_head
        self.scale = d_head ** -0.5
        
        # 内容相关的投影
        self.W_Q = nn.Linear(d_model, n_heads * d_head, bias=False)
        self.W_K_E = nn.Linear(d_model, n_heads * d_head, bias=False)  # 内容 Key
        self.W_V = nn.Linear(d_model, n_heads * d_head, bias=False)
        self.W_O = nn.Linear(n_heads * d_head, d_model, bias=False)
        
        # 位置相关的投影
        self.W_K_R = nn.Linear(d_model, n_heads * d_head, bias=False)  # 位置 Key
        
        # 全局可学习偏置（替代绝对位置的 query）
        self.u = nn.Parameter(torch.randn(n_heads, d_head))  # 内容偏置
        self.v = nn.Parameter(torch.randn(n_heads, d_head))  # 位置偏置
        
        # 预计算相对位置的正弦编码
        self.max_rel_dist = max_rel_dist
        self._build_sinusoidal_cache(max_rel_dist, d_model)
    
    def _build_sinusoidal_cache(self, max_dist: int, d_model: int):
        """预计算相对位置的正弦编码"""
        # 相对距离范围: [-(max_dist-1), ..., 0, ..., (max_dist-1)]
        positions = torch.arange(-max_dist + 1, max_dist, dtype=torch.float32)
        dim_pairs = torch.arange(0, d_model, 2, dtype=torch.float32)
        freqs = 1.0 / (10000 ** (dim_pairs / d_model))
        
        # [2*max_dist-1, d_model/2]
        angles = positions.unsqueeze(1) * freqs.unsqueeze(0)
        
        pe = torch.zeros(2 * max_dist - 1, d_model)
        pe[:, 0::2] = torch.sin(angles)
        pe[:, 1::2] = torch.cos(angles)
        
        self.register_buffer('rel_pe', pe)
    
    def _get_relative_pe(self, seq_len: int) -> torch.Tensor:
        """获取 [seq_len, seq_len, d_model] 的相对位置编码"""
        # 计算所有 (i, j) 对的相对距离 i - j
        positions = torch.arange(seq_len)
        rel_dist = positions.unsqueeze(0) - positions.unsqueeze(1)  # [seq_len, seq_len]
        
        # 映射到缓存索引（偏移 max_rel_dist - 1）
        indices = rel_dist + self.max_rel_dist - 1
        indices = indices.clamp(0, 2 * self.max_rel_dist - 2)
        
        return self.rel_pe[indices]  # [seq_len, seq_len, d_model]
    
    def forward(self, x: torch.Tensor, mask: torch.Tensor = None) -> torch.Tensor:
        """
        Args:
            x: [batch, seq_len, d_model]
            mask: [seq_len, seq_len] 注意力掩码
        Returns:
            [batch, seq_len, d_model]
        """
        B, L, D = x.shape
        H, d = self.n_heads, self.d_head
        
        # 内容投影
        q = self.W_Q(x).view(B, L, H, d)        # [B, L, H, d]
        k_e = self.W_K_E(x).view(B, L, H, d)    # [B, L, H, d]
        v = self.W_V(x).view(B, L, H, d)        # [B, L, H, d]
        
        # 相对位置编码
        R = self._get_relative_pe(L)              # [L, L, D]
        k_r = self.W_K_R(R).view(L, L, H, d)    # [L, L, H, d]
        
        # (a) 内容-内容: q_c @ k_e
        # q + u → 内容 query 加全局偏置
        q_with_u = q + self.u.unsqueeze(0).unsqueeze(0)  # [B, L, H, d]
        a_content = torch.einsum('bihd,bjhd->bijh', q_with_u, k_e)  # [B, L, L, H]
        
        # (b) 内容-位置: q @ k_r + 全局位置偏置
        q_with_v = q + self.v.unsqueeze(0).unsqueeze(0)  # [B, L, H, d]
        a_position = torch.einsum('bihd,ijhd->bijh', q_with_v, k_r)  # [B, L, L, H]
        
        # 合并
        attn_scores = (a_content + a_position) * self.scale  # [B, L, L, H]
        attn_scores = attn_scores.permute(0, 3, 1, 2)  # [B, H, L, L]
        
        if mask is not None:
            attn_scores = attn_scores.masked_fill(mask == 0, float('-inf'))
        
        attn_weights = torch.softmax(attn_scores, dim=-1)
        
        # 加权求和
        v = v.permute(0, 2, 1, 3)  # [B, H, L, d]
        output = torch.matmul(attn_weights, v)  # [B, H, L, d]
        output = output.permute(0, 2, 1, 3).contiguous().view(B, L, H * d)
        
        return self.W_O(output)
```

## 6. 相对位置编码：T5 Relative Bias

### 6.1 核心思想

T5（Raffel et al., 2020）采用了一种更简洁的相对位置编码方式：**直接在注意力分数上加一个可学习的偏置标量**：

$$
a_{ij} = q_i^\top k_j + b(i - j)
$$

其中 $b(i-j)$ 是一个根据相对距离查表得到的标量偏置。

### 6.2 距离分桶（Bucketing）

T5 不为每个精确距离都分配一个参数，而是将距离分桶：

- 近距离（|i-j| ≤ 某阈值）：每个距离一个桶（精确区分）
- 远距离：用对数分桶（log-spaced），越远的距离区分越粗

```python
def t5_relative_position_bucket(
    relative_position: torch.Tensor,
    bidirectional: bool = True,
    num_buckets: int = 32,
    max_distance: int = 128
) -> torch.Tensor:
    """
    T5 的相对位置分桶策略
    
    将相对位置映射到 [0, num_buckets) 的桶索引。
    近距离精确区分，远距离对数合并。
    
    Args:
        relative_position: 相对位置矩阵 (i - j)
        bidirectional: 是否区分正负方向（encoder=True, decoder=False）
        num_buckets: 桶的数量
        max_distance: 超过此距离的位置都映射到同一个桶
    
    Returns:
        桶索引 tensor，shape 与输入相同
    """
    ret = torch.zeros_like(relative_position, dtype=torch.long)
    
    if bidirectional:
        # 双向：一半桶给正方向，一半给负方向
        num_buckets //= 2
        # 正方向偏移 num_buckets
        ret += (relative_position > 0).long() * num_buckets
        n = torch.abs(relative_position)
    else:
        # 单向（causal）：所有负方向映射为 0
        n = torch.max(-relative_position, torch.zeros_like(relative_position))
    
    # 前半部分桶：精确映射（每个距离一个桶）
    max_exact = num_buckets // 2  # 例如 8
    is_small = n < max_exact
    
    # 后半部分桶：对数映射
    # 将 [max_exact, max_distance] 映射到 [max_exact, num_buckets)
    val_if_large = max_exact + (
        torch.log(n.float() / max_exact)
        / math.log(max_distance / max_exact)
        * (num_buckets - max_exact)
    ).long()
    val_if_large = torch.min(
        val_if_large, 
        torch.full_like(val_if_large, num_buckets - 1)
    )
    
    ret += torch.where(is_small, n, val_if_large)
    return ret
```

### 6.3 完整 T5 Relative Bias 实现

```python
class T5RelativePositionBias(nn.Module):
    """
    T5 风格的相对位置偏置
    
    每个注意力头有独立的偏置参数
    """
    
    def __init__(
        self, 
        n_heads: int, 
        num_buckets: int = 32, 
        max_distance: int = 128,
        bidirectional: bool = True
    ):
        super().__init__()
        self.n_heads = n_heads
        self.num_buckets = num_buckets
        self.max_distance = max_distance
        self.bidirectional = bidirectional
        
        # 可学习偏置：[num_buckets, n_heads]
        self.relative_attention_bias = nn.Embedding(num_buckets, n_heads)
    
    def forward(self, query_length: int, key_length: int) -> torch.Tensor:
        """
        计算相对位置偏置矩阵
        
        Returns:
            [1, n_heads, query_length, key_length] 偏置矩阵
        """
        # 构造相对位置矩阵
        context_position = torch.arange(query_length, dtype=torch.long)
        memory_position = torch.arange(key_length, dtype=torch.long)
        relative_position = memory_position.unsqueeze(0) - context_position.unsqueeze(1)
        # shape: [query_length, key_length]
        
        # 分桶
        rp_bucket = t5_relative_position_bucket(
            relative_position,
            bidirectional=self.bidirectional,
            num_buckets=self.num_buckets,
            max_distance=self.max_distance
        )
        
        # 查表获取偏置值
        values = self.relative_attention_bias(rp_bucket)  # [Q, K, n_heads]
        values = values.permute(2, 0, 1).unsqueeze(0)     # [1, n_heads, Q, K]
        
        return values


# 使用示例
bias_module = T5RelativePositionBias(n_heads=12, num_buckets=32, max_distance=128)
bias = bias_module(query_length=64, key_length=64)
print(f"偏置矩阵 shape: {bias.shape}")  # [1, 12, 64, 64]

# 在注意力计算中使用：
# attn_scores = q @ k.T / sqrt(d) + bias
```

### 6.4 T5 Bias 的设计哲学

T5 的设计体现了"简单但有效"的原则：

1. **参数高效**：只有 `num_buckets × n_heads` 个参数（如 32×12 = 384），远少于 learned PE
2. **共享偏置**：同一层的所有 token 对共享基于距离的偏置，不依赖具体内容
3. **每头独立**：不同注意力头学习不同的距离模式（有的关注近距离，有的关注远距离）
4. **对数分桶**：巧妙地在精度与泛化之间平衡——近处精确，远处模糊

### 6.5 优缺点

**优点**：
- 实现简单，计算开销极小（只是加法）
- 参数量很少
- 每个头可以学习不同的距离偏好
- 有一定的外推能力（超出 max_distance 的都映射到最后一个桶）

**缺点**：
- 偏置与内容无关，不能根据具体 token 调整位置感知
- 对数分桶可能丢失远距离的精确信息
- 外推能力仍然有限

## 7. 旋转位置编码 RoPE（Rotary Position Embedding）

### 7.1 核心思想

RoPE（Su et al., 2021, "RoFormer: Enhanced Transformer with Rotary Position Embedding"）是当前大语言模型中**最主流**的位置编码方案，被 LLaMA、Qwen、ChatGLM、Mistral、Gemma 等主流模型采用。

核心设计目标：**让位置 $m$ 的 query 和位置 $n$ 的 key 做点积时，结果天然包含相对位置信息 $m - n$**。

数学表达：找到一个函数 $f(x, m)$，使得：

$$
\langle f(q, m), f(k, n) \rangle = g(q, k, m - n)
$$

即：编码后的 query 和 key 的点积只依赖于原始内容 $q, k$ 和相对位置 $m-n$。

### 7.2 数学推导

**从二维情况开始**。考虑 $q, k \in \mathbb{R}^2$，我们希望找到 $f$ 使得：

$$
f(q, m)^\top f(k, n) = g(q, k, m-n)
$$

用复数表示：将 $(q_0, q_1)$ 看作复数 $q_0 + q_1 i$。那么**旋转**是最自然的选择：

$$
f(q, m) = q \cdot e^{im\theta}
$$

其中 $\theta$ 是预设的旋转角频率。验证：

$$
\begin{aligned}
\text{Re}[f(q, m) \cdot \overline{f(k, n)}] &= \text{Re}[q e^{im\theta} \cdot \overline{k e^{in\theta}}] \\
&= \text{Re}[q \bar{k} \cdot e^{i(m-n)\theta}]
\end{aligned}
$$

确实只依赖 $m - n$！

**推广到高维**：将 $d$ 维向量两两配对，形成 $d/2$ 个二维子空间，每个子空间独立旋转：

$$
f(x, m) = \begin{pmatrix} x_0 \\ x_1 \\ x_2 \\ x_3 \\ \vdots \\ x_{d-2} \\ x_{d-1} \end{pmatrix} \otimes \begin{pmatrix} \cos(m\theta_0) \\ \cos(m\theta_0) \\ \cos(m\theta_1) \\ \cos(m\theta_1) \\ \vdots \\ \cos(m\theta_{d/2-1}) \\ \cos(m\theta_{d/2-1}) \end{pmatrix} + \begin{pmatrix} -x_1 \\ x_0 \\ -x_3 \\ x_2 \\ \vdots \\ -x_{d-1} \\ x_{d-2} \end{pmatrix} \otimes \begin{pmatrix} \sin(m\theta_0) \\ \sin(m\theta_0) \\ \sin(m\theta_1) \\ \sin(m\theta_1) \\ \vdots \\ \sin(m\theta_{d/2-1}) \\ \sin(m\theta_{d/2-1}) \end{pmatrix}
$$

**等价的矩阵形式**（对每个二维子空间）：

$$
R_{\theta, m}^{(i)} = \begin{bmatrix} \cos(m\theta_i) & -\sin(m\theta_i) \\ \sin(m\theta_i) & \cos(m\theta_i) \end{bmatrix}
$$

完整的旋转矩阵是分块对角的：

$$
R_{\Theta, m} = \begin{bmatrix} R_{\theta_0, m} & & & \\ & R_{\theta_1, m} & & \\ & & \ddots & \\ & & & R_{\theta_{d/2-1}, m} \end{bmatrix}
$$

### 7.3 频率选择

RoPE 使用与 Sinusoidal PE 相同的频率公式：

$$
\theta_i = \frac{1}{10000^{2i/d}}, \quad i = 0, 1, \ldots, d/2 - 1
$$

**为什么沿用这个频率**：
- 低维度频率高（$\theta_0 = 1$）：对近距离位置差异敏感
- 高维度频率低（$\theta_{d/2-1} = 1/10000$）：对远距离提供粗粒度区分
- 这种多尺度频率确保了对各种距离的感知能力

### 7.4 RoPE 的核心性质

**性质 1：相对位置感知**

$$
\langle R_{\Theta,m} q, R_{\Theta,n} k \rangle = q^\top R_{\Theta, m-n}^\top k = q^\top R_{\Theta, n-m} k
$$

由旋转矩阵的正交性 $R_m^\top R_n = R_{n-m}$ 得证。

**性质 2：远程衰减**

RoPE 编码后的注意力分数具有长程衰减的趋势。直觉：当 $|m-n|$ 很大时，高频维度的 $\cos((m-n)\theta_i)$ 快速振荡，均值趋零，有效降低了远距离 token 的注意力得分。

**性质 3：不改变向量范数**

旋转是正交变换，不改变向量长度：$\|R_m q\| = \|q\|$。这意味着 RoPE 不会影响 attention score 的整体量级。

### 7.5 PyTorch 实现（生产级）

```python
class RotaryEmbedding(nn.Module):
    """
    旋转位置编码 RoPE
    
    LLaMA / Qwen / Mistral 等主流模型的标准实现
    """
    
    def __init__(self, dim: int, base: float = 10000.0, max_seq_len: int = 8192):
        super().__init__()
        self.dim = dim
        self.base = base
        self.max_seq_len = max_seq_len
        
        # 计算频率: theta_i = 1 / (base^(2i/dim))
        inv_freq = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
        self.register_buffer('inv_freq', inv_freq)  # [dim/2]
        
        # 预计算并缓存 cos/sin 值
        self._build_cache(max_seq_len)
    
    def _build_cache(self, seq_len: int):
        """预计算 cos/sin 缓存以避免重复计算"""
        t = torch.arange(seq_len, dtype=torch.float32)
        
        # 外积: [seq_len] x [dim/2] → [seq_len, dim/2]
        freqs = torch.outer(t, self.inv_freq)
        
        # 拼接成完整维度: [seq_len, dim]
        # 每个频率对应 (cos, sin) 两个维度
        emb = torch.cat([freqs, freqs], dim=-1)
        
        self.register_buffer('cos_cached', emb.cos())  # [seq_len, dim]
        self.register_buffer('sin_cached', emb.sin())  # [seq_len, dim]
    
    def forward(self, seq_len: int) -> tuple:
        """
        Returns:
            (cos, sin) 各自 shape [seq_len, dim]
        """
        if seq_len > self.max_seq_len:
            self._build_cache(seq_len)
        return self.cos_cached[:seq_len], self.sin_cached[:seq_len]


def rotate_half(x: torch.Tensor) -> torch.Tensor:
    """
    将向量的前半部分和后半部分交换并取负
    用于实现旋转操作: [-x1, x0, -x3, x2, ...]
    
    Args:
        x: [..., dim] 最后一维是偶数
    """
    x1 = x[..., :x.shape[-1] // 2]
    x2 = x[..., x.shape[-1] // 2:]
    return torch.cat([-x2, x1], dim=-1)


def apply_rotary_pos_emb(
    q: torch.Tensor, 
    k: torch.Tensor, 
    cos: torch.Tensor, 
    sin: torch.Tensor,
    position_ids: torch.Tensor = None
) -> tuple:
    """
    将 RoPE 应用到 query 和 key
    
    数学: f(x, m) = x * cos(m*theta) + rotate_half(x) * sin(m*theta)
    
    Args:
        q: [batch, n_heads, seq_len, head_dim]
        k: [batch, n_heads, seq_len, head_dim]  (或 [batch, n_kv_heads, seq_len, head_dim])
        cos: [seq_len, head_dim]
        sin: [seq_len, head_dim]
        position_ids: [batch, seq_len] 可选，用于非连续位置
    
    Returns:
        旋转后的 (q, k)
    """
    if position_ids is not None:
        # 非连续位置（如 KV cache 场景）
        cos = cos[position_ids].unsqueeze(1)  # [batch, 1, seq_len, dim]
        sin = sin[position_ids].unsqueeze(1)
    else:
        # 连续位置
        seq_len = q.shape[2]
        cos = cos[:seq_len].unsqueeze(0).unsqueeze(0)  # [1, 1, seq_len, dim]
        sin = sin[:seq_len].unsqueeze(0).unsqueeze(0)
    
    # 应用旋转
    q_embed = q * cos + rotate_half(q) * sin
    k_embed = k * cos + rotate_half(k) * sin
    
    return q_embed, k_embed


# 完整使用示例
def rope_attention_example():
    """演示 RoPE 在完整注意力计算中的使用"""
    batch_size = 2
    seq_len = 128
    n_heads = 32
    head_dim = 64  # RoPE 作用在 head_dim 维度
    d_model = n_heads * head_dim  # 2048
    
    # 初始化
    rope = RotaryEmbedding(dim=head_dim, base=10000.0)
    W_Q = nn.Linear(d_model, d_model, bias=False)
    W_K = nn.Linear(d_model, d_model, bias=False)
    W_V = nn.Linear(d_model, d_model, bias=False)
    
    # 输入
    x = torch.randn(batch_size, seq_len, d_model)
    
    # 投影
    q = W_Q(x).view(batch_size, seq_len, n_heads, head_dim).transpose(1, 2)
    k = W_K(x).view(batch_size, seq_len, n_heads, head_dim).transpose(1, 2)
    v = W_V(x).view(batch_size, seq_len, n_heads, head_dim).transpose(1, 2)
    # q, k, v: [batch, n_heads, seq_len, head_dim]
    
    # 应用 RoPE
    cos, sin = rope(seq_len)
    q_rotated, k_rotated = apply_rotary_pos_emb(q, k, cos, sin)
    
    # 注意力计算
    scale = head_dim ** -0.5
    attn_scores = torch.matmul(q_rotated, k_rotated.transpose(-2, -1)) * scale
    
    # Causal mask
    causal_mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1).bool()
    attn_scores.masked_fill_(causal_mask.unsqueeze(0).unsqueeze(0), float('-inf'))
    
    attn_weights = torch.softmax(attn_scores, dim=-1)
    output = torch.matmul(attn_weights, v)
    
    print(f"Q shape: {q.shape}")
    print(f"Q (rotated) shape: {q_rotated.shape}")
    print(f"Attention scores shape: {attn_scores.shape}")
    print(f"Output shape: {output.shape}")
    
    # 验证：旋转不改变范数
    q_norm_before = q.norm(dim=-1).mean()
    q_norm_after = q_rotated.norm(dim=-1).mean()
    print(f"\nQ 范数验证: before={q_norm_before:.4f}, after={q_norm_after:.4f}")
    print(f"范数变化: {abs(q_norm_before - q_norm_after):.6f} (应接近 0)")


rope_attention_example()
```

### 7.6 RoPE 的另一种等价实现（interleaved）

上面的实现使用 "half-half" 布局（前半 cos，后半 sin）。另一种常见布局是 "interleaved"（交替排列），在 LLaMA 原始代码中使用：

```python
def apply_rotary_emb_interleaved(xq: torch.Tensor, xk: torch.Tensor, 
                                  freqs_cis: torch.Tensor) -> tuple:
    """
    交替式 RoPE 实现（LLaMA 原始风格）
    使用复数乘法，更加简洁
    
    Args:
        xq: [batch, seq_len, n_heads, head_dim]
        xk: [batch, seq_len, n_heads, head_dim]
        freqs_cis: [seq_len, head_dim//2] 复数形式的频率
    """
    # 将实数向量视为复数: (x0, x1) → x0 + x1*i
    xq_complex = torch.view_as_complex(xq.float().reshape(*xq.shape[:-1], -1, 2))
    xk_complex = torch.view_as_complex(xk.float().reshape(*xk.shape[:-1], -1, 2))
    
    # 复数乘法 = 旋转
    # (a + bi)(cos θ + i sin θ) = (a cos θ - b sin θ) + (a sin θ + b cos θ)i
    freqs_cis = freqs_cis.unsqueeze(0).unsqueeze(2)  # [1, seq_len, 1, head_dim//2]
    
    xq_out = torch.view_as_real(xq_complex * freqs_cis).flatten(-2)
    xk_out = torch.view_as_real(xk_complex * freqs_cis).flatten(-2)
    
    return xq_out.type_as(xq), xk_out.type_as(xk)


def precompute_freqs_cis(dim: int, end: int, theta: float = 10000.0) -> torch.Tensor:
    """预计算复数形式的旋转频率"""
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2)[: (dim // 2)].float() / dim))
    t = torch.arange(end, dtype=torch.float32)
    freqs = torch.outer(t, freqs)  # [end, dim//2]
    freqs_cis = torch.polar(torch.ones_like(freqs), freqs)  # e^(i*freqs)
    return freqs_cis  # [end, dim//2] complex tensor


# 使用
freqs_cis = precompute_freqs_cis(dim=64, end=2048)
print(f"freqs_cis shape: {freqs_cis.shape}, dtype: {freqs_cis.dtype}")
# freqs_cis shape: torch.Size([2048, 32]), dtype: torch.complex64
```

### 7.7 RoPE 为什么能成为主流

1. **数学优美**：旋转变换天然保证点积编码相对位置，无需额外参数
2. **不增加参数**：所有频率由公式确定，零额外参数
3. **不改变范数**：正交变换不影响 attention score 的量级
4. **兼容 KV Cache**：每个位置的旋转独立计算，完美兼容自回归推理
5. **可扩展**：通过修改 base 或频率，可以进行长度外推（后文详述）
6. **局部性**：高频维度提供近距离辨别力，低频维度提供远距离感知

## 8. ALiBi（Attention with Linear Biases）

### 8.1 核心思想

ALiBi（Press et al., 2021, "Train Short, Test Long: Attention with Linear Biases Enables Input Length Extrapolation"）的设计极其简洁：

- **不添加任何位置编码到 embedding**
- **在 attention score 上加一个线性距离惩罚**

$$
a_{ij} = q_i^\top k_j - m \cdot |i - j|
$$

其中 $m$ 是每个头不同的固定斜率（不可学习），$|i - j|$ 是 query 和 key 之间的距离。

### 8.2 斜率选择

对于 $n$ 个注意力头，斜率按几何级数设置：

$$
m_h = \frac{1}{2^{8h/n}}, \quad h = 1, 2, \ldots, n
$$

例如，对于 8 个头：$m = [1/2, 1/4, 1/8, 1/16, 1/32, 1/64, 1/128, 1/256]$

**设计直觉**：
- 斜率大的头（如 $m=1/2$）：强烈惩罚远距离，主要关注局部上下文
- 斜率小的头（如 $m=1/256$）：轻微惩罚远距离，能捕获长程依赖
- 不同头自然形成多尺度的位置感知

### 8.3 PyTorch 实现

```python
class ALiBiPositionalBias(nn.Module):
    """
    ALiBi: Attention with Linear Biases
    
    特点：
    - 零额外参数
    - 不修改 embedding，只在 attention score 上加偏置
    - 不同头使用不同固定斜率
    - 强长度外推能力
    """
    
    def __init__(self, n_heads: int, max_seq_len: int = 8192):
        super().__init__()
        self.n_heads = n_heads
        
        # 计算每个头的斜率
        slopes = self._get_slopes(n_heads)
        self.register_buffer('slopes', torch.tensor(slopes, dtype=torch.float32))
        
        # 预计算偏置矩阵
        self._build_bias_cache(max_seq_len)
    
    @staticmethod
    def _get_slopes(n_heads: int) -> list:
        """
        计算 ALiBi 斜率
        
        当 n_heads 是 2 的幂时：直接几何级数
        否则：插值最近的 2 的幂的斜率
        """
        def get_slopes_power_of_2(n):
            start = 2 ** (-(2 ** -(math.log2(n) - 3)))
            ratio = start
            return [start * (ratio ** i) for i in range(n)]
        
        if math.log2(n_heads).is_integer():
            return get_slopes_power_of_2(n_heads)
        else:
            # 非 2 的幂：从最近的大 2 次幂中选取
            closest_power_of_2 = 2 ** math.floor(math.log2(n_heads))
            slopes = get_slopes_power_of_2(closest_power_of_2)
            
            # 从 2*closest_power_of_2 中补充额外的斜率
            extra_slopes = get_slopes_power_of_2(2 * closest_power_of_2)
            extra_slopes = extra_slopes[0::2][:n_heads - closest_power_of_2]
            
            return slopes + extra_slopes
    
    def _build_bias_cache(self, max_seq_len: int):
        """预计算距离偏置矩阵"""
        # 距离矩阵: |i - j|，对于 causal 模型是 i - j (i >= j)
        positions = torch.arange(max_seq_len)
        # 对于 causal 注意力，只有 i >= j 有效
        distance = positions.unsqueeze(0) - positions.unsqueeze(1)  # [L, L]
        distance = distance.float()
        
        # 乘以斜率: [n_heads, L, L]
        # slopes: [n_heads], distance: [L, L]
        bias = -torch.abs(distance).unsqueeze(0) * self.slopes.unsqueeze(1).unsqueeze(1)
        
        self.register_buffer('bias_cache', bias)  # [n_heads, max_L, max_L]
    
    def forward(self, query_length: int, key_length: int) -> torch.Tensor:
        """
        Returns:
            [1, n_heads, query_length, key_length] 偏置矩阵
        """
        # 从缓存中截取需要的部分
        bias = self.bias_cache[:, :query_length, :key_length]
        return bias.unsqueeze(0)  # [1, n_heads, Q, K]


# 使用示例 & 验证
alibi = ALiBiPositionalBias(n_heads=8)
print(f"8 头的斜率: {alibi.slopes.tolist()}")

bias = alibi(query_length=8, key_length=8)
print(f"\n第 1 个头（斜率最大，最关注局部）的偏置矩阵:")
print(bias[0, 0].numpy().round(2))
print(f"\n第 8 个头（斜率最小，关注更远）的偏置矩阵:")
print(bias[0, 7].numpy().round(2))
```

### 8.4 ALiBi 的外推能力

ALiBi 的核心优势是**强大的长度外推能力**。论文实验表明：

| 训练长度 | 测试长度 | Perplexity（Sinusoidal） | Perplexity（ALiBi） |
|---------|---------|------------------------|-------------------|
| 1024 | 1024 | 11.2 | 11.1 |
| 1024 | 2048 | 失败 (>100) | 11.3 |
| 1024 | 3072 | 失败 | 11.5 |
| 1024 | 6144 | 失败 | 12.1 |

**为什么 ALiBi 能外推**：线性惩罚是一个简单、规律的数学函数，模型在训练时学会了"远距离的 token 不太重要"这个先验，这个规律在更长序列上仍然成立。

### 8.5 ALiBi 的局限性

尽管外推能力强，ALiBi 也有不足：

1. **位置感知粗糙**：线性惩罚不能区分"第 5 个位置"和"第 6 个位置"的语义差异，只关注距离
2. **与 RoPE 相比的性能差距**：在需要精确位置感知的任务（如代码生成、数学推理）中，RoPE 通常更优
3. **不兼容 KV Cache 优化**：ALiBi 的偏置矩阵大小随序列长度平方增长
4. **固定斜率**：不能根据任务自适应调整

### 8.6 使用 ALiBi 的模型

- **BLOOM**（BigScience, 176B）
- **MPT**（MosaicML）
- **Falcon**（TII）的部分变体

## 9. 长度外推技术

### 9.1 问题背景

RoPE 虽然优秀，但有一个著名问题：**当推理长度超过训练长度时，困惑度（perplexity）急剧上升**。

例如，LLaMA-2 以 4096 长度训练，直接推理 8192 长度的文本时，perplexity 会从 ~5 飙升到 >100。

原因分析：训练时模型只见过 $m\theta_i$ 在 $[0, 4096 \cdot \theta_i]$ 范围内的值。推理时 $m > 4096$ 产生的角度超出了训练分布，注意力模式被破坏。

### 9.2 Position Interpolation（PI）

**Chen et al., 2023, "Extending Context Window of Large Language Models via Positional Interpolation"**

核心思想：**将超长序列的位置压缩到训练长度范围内**。

如果训练长度为 $L$，目标长度为 $L'$，则将位置索引线性缩放：

$$
m' = m \cdot \frac{L}{L'}
$$

即把 $[0, L')$ 的位置映射到 $[0, L)$ 范围内，确保所有角度都在训练时见过的范围内。

```python
class PositionInterpolationRoPE(nn.Module):
    """
    Position Interpolation: 线性缩放位置索引
    
    将超出训练长度的位置通过线性插值压缩到训练范围
    """
    
    def __init__(self, dim: int, base: float = 10000.0, 
                 original_max_len: int = 4096, target_max_len: int = 16384):
        super().__init__()
        self.dim = dim
        self.base = base
        self.original_max_len = original_max_len
        self.target_max_len = target_max_len
        
        # 缩放因子
        self.scale = original_max_len / target_max_len
        
        # 基础频率
        inv_freq = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
        self.register_buffer('inv_freq', inv_freq)
    
    def forward(self, seq_len: int):
        """返回缩放后的 cos/sin"""
        # 关键：位置索引乘以缩放因子
        t = torch.arange(seq_len, dtype=torch.float32) * self.scale
        
        freqs = torch.outer(t, self.inv_freq)
        emb = torch.cat([freqs, freqs], dim=-1)
        
        return emb.cos(), emb.sin()


# 对比：原始 vs PI
original_rope = RotaryEmbedding(dim=64, max_seq_len=4096)
pi_rope = PositionInterpolationRoPE(dim=64, original_max_len=4096, target_max_len=16384)

# 原始 RoPE 在位置 4096 处的频率
cos_orig, sin_orig = original_rope(4096)
print(f"原始 RoPE 最大角度 (dim 0): {4096 * 1.0:.1f}")
print(f"PI RoPE 在位置 16384 时等效角度 (dim 0): {16384 * (4096/16384):.1f}")
# 两者相同！PI 保证了角度不超出训练范围
```

**PI 的效果**：
- 需要**少量微调**（~1000 步）才能恢复性能
- 微调后可以稳定处理 4× 甚至更长的序列
- 代价：相邻位置的角度差变小，近距离的位置辨别能力下降

### 9.3 NTK-Aware Scaling

**bloc97 (Reddit), 2023**

PI 的问题：所有频率统一缩放，导致高频维度（本来区分近距离的）也被压缩，降低了近距离的分辨率。

NTK-Aware 的思想：**对不同频率使用不同的缩放策略**：
- 高频维度（低 $i$）：少缩放或不缩放（保持近距离辨别力）
- 低频维度（高 $i$）：多缩放（通过内插避免远距离角度外推）

实现方式：**增大 base**，而不是缩放位置：

$$
\theta_i^{\text{NTK}} = \frac{1}{(\alpha \cdot \text{base})^{2i/d}}
$$

其中 $\alpha$ 是缩放因子，与目标扩展比例相关。

```python
class NTKAwareRoPE(nn.Module):
    """
    NTK-Aware Scaled RoPE
    
    通过修改 base 实现高频外推 + 低频内插的效果
    
    原理：增大 base 等效于将所有频率降低，
    但高频降低后仍是"高频"，低频降低后有效实现了内插
    """
    
    def __init__(self, dim: int, base: float = 10000.0, 
                 original_max_len: int = 4096, target_max_len: int = 16384,
                 method: str = 'dynamic'):
        super().__init__()
        self.dim = dim
        self.original_base = base
        self.original_max_len = original_max_len
        self.target_max_len = target_max_len
        self.method = method
        
        # 计算缩放因子 alpha
        scale_ratio = target_max_len / original_max_len
        
        if method == 'fixed':
            # 固定 NTK: alpha = scale_ratio^(dim/(dim-2))
            alpha = scale_ratio ** (dim / (dim - 2))
            new_base = base * alpha
            inv_freq = 1.0 / (new_base ** (torch.arange(0, dim, 2).float() / dim))
            self.register_buffer('inv_freq', inv_freq)
            
        elif method == 'dynamic':
            # 动态 NTK: 根据当前序列长度动态调整
            self.base = base
            self.register_buffer(
                'base_inv_freq',
                1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
            )
    
    def _dynamic_inv_freq(self, seq_len: int) -> torch.Tensor:
        """动态计算频率（随序列长度自适应）"""
        if seq_len <= self.original_max_len:
            return self.base_inv_freq
        
        # 当序列超出训练长度时，动态增大 base
        scale = seq_len / self.original_max_len
        alpha = scale ** (self.dim / (self.dim - 2))
        new_base = self.original_base * alpha
        
        return 1.0 / (new_base ** (torch.arange(0, self.dim, 2).float() / self.dim))
    
    def forward(self, seq_len: int):
        if self.method == 'fixed':
            inv_freq = self.inv_freq
        else:
            inv_freq = self._dynamic_inv_freq(seq_len)
        
        t = torch.arange(seq_len, dtype=torch.float32)
        freqs = torch.outer(t, inv_freq)
        emb = torch.cat([freqs, freqs], dim=-1)
        
        return emb.cos(), emb.sin()


# 对比不同方法的频率分布

print()

print(f"低频缩放比: {ntk_freqs[-1]/original_freqs[-1]:.4f}  (明显缩小)")
```

### 9.4 YaRN（Yet another RoPE extensioN）

**Peng et al., 2023**

YaRN 是目前最完善的 RoPE 长度外推方法，综合了 PI 和 NTK-Aware 的优点：

**核心组件**：

1. **NTK-by-parts**：将维度分为三组，分别处理
   - 高频维度（$\lambda_i < 1$）：不缩放（外推）
   - 中频维度：线性插值比例渐变
   - 低频维度（$\lambda_i > 1$）：完全内插

2. **注意力缩放因子**：补偿因插值导致的 attention entropy 变化

$$
\text{scale} = \frac{1}{\sqrt{1 + \frac{\ln(s)}{d}}}
$$

其中 $s$ 是长度扩展比例。

```python
class YaRNRoPE(nn.Module):
    """
    YaRN: Yet another RoPE extensioN
    
    结合 NTK-by-parts + 注意力缩放的长度外推方法
    """
    
    def __init__(
        self,
        dim: int,
        base: float = 10000.0,
        original_max_len: int = 4096,
        target_max_len: int = 32768,
        beta_fast: float = 32.0,   # 高频边界
        beta_slow: float = 1.0,    # 低频边界
    ):
        super().__init__()
        self.dim = dim
        self.base = base
        self.original_max_len = original_max_len
        self.target_max_len = target_max_len
        self.scale_ratio = target_max_len / original_max_len
        
        # 注意力缩放因子
        self.attn_scale = 1.0 / math.sqrt(
            1 + math.log(self.scale_ratio) / dim
        )
        
        # 计算各维度的插值比例
        inv_freq = self._compute_yarn_inv_freq(dim, base, beta_fast, beta_slow)
        self.register_buffer('inv_freq', inv_freq)
    
    def _compute_yarn_inv_freq(self, dim, base, beta_fast, beta_slow):
        """
        YaRN 的核心：NTK-by-parts 频率计算
        
        将每个频率维度的 wavelength 与训练长度比较：
        - wavelength << train_len: 高频，外推（不缩放）
        - wavelength >> train_len: 低频，内插（完全缩放）
        - 中间：线性渐变
        """
        # 原始频率
        freq_base = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
        
        # 各频率的 wavelength (周期)
        wavelengths = 2 * math.pi / freq_base  # wavelength = 2π/θ
        
        # 归一化波长比
        # r_i = wavelength_i / original_max_len
        ratio = wavelengths / self.original_max_len
        
        # 计算每个维度的插值因子 ramp
        # 低于 beta_fast 的 → ramp=0（不内插，保持外推）
        # 高于 beta_slow 的 → ramp=1（完全内插）
        # 中间线性过渡
        low = beta_fast   # 注意：beta_fast 对应 "快速变化" = 高频 = 短波长
        high = beta_slow  # beta_slow 对应 "慢变化" = 低频 = 长波长
        
        # ramp 在 [0, 1] 之间渐变
        ramp = torch.clamp((ratio - low) / (high - low), 0.0, 1.0)
        
        # 对每个维度计算混合频率
        # ramp=0: 不缩放（外推），ramp=1: 完全缩放（内插）
        inv_freq_interpolated = freq_base / self.scale_ratio  # 内插频率
        
        # 混合: (1-ramp)*原始 + ramp*内插
        inv_freq = (1 - ramp) * freq_base + ramp * inv_freq_interpolated
        
        return inv_freq
    
    def forward(self, seq_len: int):
        t = torch.arange(seq_len, dtype=torch.float32)
        freqs = torch.outer(t, self.inv_freq)
        emb = torch.cat([freqs, freqs], dim=-1)
        
        # 返回时乘以注意力缩放因子
        return emb.cos() * self.attn_scale, emb.sin() * self.attn_scale


# YaRN 示例
yarn = YaRNRoPE(
    dim=128, 
    original_max_len=4096, 
    target_max_len=32768
)
cos, sin = yarn(32768)
print(f"YaRN 注意力缩放因子: {yarn.attn_scale:.4f}")
print(f"cos shape: {cos.shape}")
```

## 10. CoPE（Contextual Position Encoding）

### 10.1 核心动机

传统位置编码（包括 RoPE）的共同局限：**位置是固定的 token 索引**，不考虑语义内容。

但在实际语言中，"第 3 个句子"、"前一个动词"这类语义级的位置关系比 token 级索引更有意义。

CoPE（Golovneva et al., 2024, Meta）提出：**让位置编码依赖上下文内容**。

### 10.2 数学定义

CoPE 的位置不是固定的整数索引，而是由 attention 动态计算：

$$
\text{pos}_i = \sum_{j \leq i} g_{ij}, \quad g_{ij} = \sigma(q_i^\top k_j)
$$

其中 $\sigma$ 是 sigmoid 函数。这样，每个 token 的"位置"是由它和之前所有 token 的注意力交互决定的。

然后用这个 context-dependent position 去生成位置编码：

$$
\text{PE}(\text{pos}_i) = \text{interpolated from a learnable position embedding table}
$$

### 10.3 实现

```python
class CoPE(nn.Module):
    """
    CoPE: Contextual Position Encoding (Meta, 2024)
    
    位置不是固定索引，而是由上下文动态计算
    """
    
    def __init__(self, d_model: int, n_heads: int, max_positions: int = 2048):
        super().__init__()
        self.d_model = d_model
        self.n_heads = n_heads
        self.head_dim = d_model // n_heads
        
        # 可学习的位置嵌入表（用于插值）
        self.position_emb = nn.Embedding(max_positions, self.head_dim)
        
        # gates 的投影
        self.gate_proj = nn.Linear(self.head_dim, 1, bias=True)
    
    def forward(self, q: torch.Tensor, k: torch.Tensor) -> torch.Tensor:
        """
        计算 context-dependent 位置偏置
        
        Args:
            q: [batch, n_heads, seq_len, head_dim]
            k: [batch, n_heads, seq_len, head_dim]
        
        Returns:
            位置偏置 [batch, n_heads, seq_len, seq_len]
        """
        B, H, L, D = q.shape
        
        # 计算 gates: 哪些之前的 token 对当前位置有贡献
        # 使用简化版本：基于 Q-K 相似度的 sigmoid
        gates = torch.sigmoid(torch.matmul(q, k.transpose(-2, -1)))  # [B, H, L, L]
        
        # Causal mask: 只能看到之前的 token
        causal_mask = torch.triu(torch.ones(L, L, device=q.device), diagonal=1).bool()
        gates = gates.masked_fill(causal_mask.unsqueeze(0).unsqueeze(0), 0.0)
        
        # 计算 contextual position: 累积 gates
        # pos[i][j] = sum_{k=j}^{i} gates[i][k]  (j 到 i 之间的累积)
        # 简化：对每个 query i，计算它和每个 key j 的"语义距离"
        pos = gates.cumsum(dim=-1)  # [B, H, L, L]
        
        # 用位置查表（支持非整数位置的插值）
        pos_floor = pos.long().clamp(0, self.position_emb.num_embeddings - 2)
        pos_frac = pos - pos_floor.float()
        
        # 线性插值
        emb_floor = self.position_emb(pos_floor)      # [B, H, L, L, D]
        emb_ceil = self.position_emb(pos_floor + 1)   # [B, H, L, L, D]
        pos_emb = emb_floor + pos_frac.unsqueeze(-1) * (emb_ceil - emb_floor)
        
        # 计算位置偏置: q 和 pos_emb 的点积
        # q: [B, H, L, D] → [B, H, L, 1, D]
        pos_bias = torch.matmul(
            q.unsqueeze(3),  # [B, H, L, 1, D]
            pos_emb.transpose(-2, -1)  # [B, H, L, D, L]
        ).squeeze(3)  # [B, H, L, L]
        
        return pos_bias


# 测试

k = torch.randn(2, 8, 64, 64)

```

### 10.4 CoPE 的意义

CoPE 代表了位置编码的一个新方向：**从固定位置到动态位置**。它能够：
- 区分"语义上相近但物理上远离"的 token
- 对结构化文本（代码、列表）建模更好
- 理论上具有更强的外推能力（位置由内容决定，不受绝对索引限制）

不过目前 CoPE 仍处于研究阶段，尚未被主流大模型广泛采用。

```python
from enum import Enum
from typing import Optional, Tuple

class PositionEncodingType(Enum):
    SINUSOIDAL = "sinusoidal"
    LEARNED = "learned"
    ROPE = "rope"
    ALIBI = "alibi"
    T5_BIAS = "t5_bias"
    YARN = "yarn"


class UnifiedPositionEncoding(nn.Module):
    """
    统一的位置编码接口
    
    支持多种位置编码方案的切换对比
    """
    
    def __init__(
        self,
        method: PositionEncodingType,
        d_model: int,
        n_heads: int,
        max_len: int = 8192,
        rope_base: float = 10000.0,
        **kwargs
    ):
        super().__init__()
        self.method = method
        self.d_model = d_model
        self.n_heads = n_heads
        self.head_dim = d_model // n_heads
        
        if method == PositionEncodingType.SINUSOIDAL:
            self.encoder = SinusoidalPositionalEncoding(d_model, max_len)
            
        elif method == PositionEncodingType.LEARNED:
            self.encoder = LearnedPositionalEmbedding(max_len, d_model)
            
        elif method == PositionEncodingType.ROPE:
            self.encoder = RotaryEmbedding(self.head_dim, rope_base, max_len)
            
        elif method == PositionEncodingType.ALIBI:
            self.encoder = ALiBiPositionalBias(n_heads, max_len)
            
        elif method == PositionEncodingType.T5_BIAS:
            self.encoder = T5RelativePositionBias(n_heads)
            
        elif method == PositionEncodingType.YARN:
            original_max_len = kwargs.get('original_max_len', 4096)
            self.encoder = YaRNRoPE(
                self.head_dim, rope_base, 
                original_max_len, max_len
            )
    
    def apply_to_embedding(self, x: torch.Tensor) -> torch.Tensor:
        """对输入 embedding 应用位置编码（仅 Sinusoidal/Learned 使用）"""
        if self.method in (PositionEncodingType.SINUSOIDAL, PositionEncodingType.LEARNED):
            return self.encoder(x)
        return x  # RoPE/ALiBi/T5 不修改 embedding
    
    def apply_to_attention(
        self,
        q: torch.Tensor,
        k: torch.Tensor,
        attn_scores: torch.Tensor,
        seq_len: int
    ) -> Tuple[torch.Tensor, torch.Tensor, torch.Tensor]:
        """
        对注意力计算应用位置编码
        
        Returns:
            (q, k, attn_scores) 可能被修改的版本
        """
        if self.method == PositionEncodingType.ROPE:
            cos, sin = self.encoder(seq_len)
            q, k = apply_rotary_pos_emb(q, k, cos, sin)
            
        elif self.method == PositionEncodingType.YARN:
            cos, sin = self.encoder(seq_len)
            q, k = apply_rotary_pos_emb(q, k, cos, sin)
            
        elif self.method == PositionEncodingType.ALIBI:
            bias = self.encoder(seq_len, seq_len)
            attn_scores = attn_scores + bias
            
        elif self.method == PositionEncodingType.T5_BIAS:
            bias = self.encoder(seq_len, seq_len)
            attn_scores = attn_scores + bias
        
        return q, k, attn_scores
```

### 11.2 RoPE + GQA（Grouped Query Attention）兼容

现代模型常用 GQA，其中 KV heads < Q heads。RoPE 在此场景下的应用：

```python
def apply_rotary_pos_emb_gqa(
    q: torch.Tensor,
    k: torch.Tensor,
    cos: torch.Tensor,
    sin: torch.Tensor,
    n_kv_heads: int
) -> Tuple[torch.Tensor, torch.Tensor]:
    """
    GQA 兼容的 RoPE 应用
    
    Args:
        q: [batch, n_heads, seq_len, head_dim]
        k: [batch, n_kv_heads, seq_len, head_dim]  (n_kv_heads < n_heads)
        cos, sin: [seq_len, head_dim]
    """
    seq_len = q.shape[2]
    cos = cos[:seq_len].unsqueeze(0).unsqueeze(0)
    sin = sin[:seq_len].unsqueeze(0).unsqueeze(0)
    
    # Q 和 K 分别旋转（维度不同但公式相同）
    q_embed = q * cos + rotate_half(q) * sin
    k_embed = k * cos + rotate_half(k) * sin  # cos/sin 会自动广播
    
    return q_embed, k_embed
```

### 12.4 如何选择位置编码

**决策流程**：

1. **固定短序列**（≤2K）？→ Learned PE 或 Sinusoidal，简单够用
2. **需要长上下文**（>8K）？→ RoPE + YaRN/NTK
3. **极端外推需求**（train short test very long）？→ ALiBi
4. **精确位置感知**（代码/数学）？→ RoPE
5. **参数极度受限**？→ ALiBi（零参数）或 RoPE（零参数）
6. **研究/前沿**？→ CoPE、LongRoPE

## 14. 常见面试题（附答案要点）

### 面试题 1：为什么 Transformer 需要位置编码？不加会怎样？

**答案要点**：
- Self-Attention 是置换不变的（permutation invariant），对输入 token 的排列不敏感
- 没有位置编码，"猫吃鱼"和"鱼吃猫"对模型来说完全相同
- 位置编码是让 Transformer 从"集合操作"变成"序列操作"的关键组件
- 实验证明：去掉位置编码，NLP 任务性能灾难性下降（如 BLEU 降 10+）

### 面试题 2：解释正弦位置编码的设计意图，为什么用不同频率的正弦波？

**答案要点**：
- 类比二进制编码：不同位（频率）翻转速率不同，组合后可唯一标识每个位置
- 正弦波是二进制的连续化版本，低维高频区分近距离，高维低频区分远距离
- 核心性质：PE(pos+k) 可以由 PE(pos) 通过固定线性变换得到（旋转矩阵）
- 点积只依赖相对距离，理论上模型可以学到相对位置关系

### 面试题 3：RoPE 的核心思想是什么？为什么它能编码相对位置？

**答案要点**：
- 核心：在 Q/K 上施加位置相关的旋转变换，使得 Q·K 的点积天然包含相对位置信息
- 数学证明：$\langle R_m q, R_n k \rangle = q^\top R_{n-m} k$，只依赖 $m-n$
- 二维子空间上是复数乘法 $q \cdot e^{im\theta}$，高维是分块对角旋转矩阵
- 优势：零参数、不改变范数、兼容 KV Cache、可扩展

### 面试题 4：Position Interpolation 和 NTK-Aware 的区别？

**答案要点**：
- **PI**：统一缩放所有位置索引 $m' = m \cdot L/L'$，所有频率等比缩放
  - 缺点：高频维度也被压缩，近距离分辨率下降
- **NTK-Aware**：增大 base（$\text{base}' = \text{base} \cdot \alpha$），等效于对不同频率非均匀缩放
  - 高频基本不变（保持近距离辨别力），低频被缩放（避免远距离外推）
  - 更符合 NTK 理论：高频信息更难学习，应保持不变
- YaRN 是两者的综合改进，加入分维度混合策略

### 面试题 5：ALiBi 为什么能实现长度外推？

**答案要点**：
- ALiBi 不修改 embedding，只在 attention score 上加线性距离惩罚
- 线性惩罚 $-m|i-j|$ 是一个简单规律的数学函数
- 模型在训练时学会了"远距离 token 不太重要"的先验
- 这个先验在更长序列上仍然成立（外推就是这个简单规律的延伸）
- 不同头使用不同斜率，形成多尺度的位置感知

### 面试题 6：为什么 RoPE 的 base 选 10000？改成其他值会怎样？

**答案要点**：
- base 决定频率衰减速度：$\theta_i = 1/\text{base}^{2i/d}$
- base 越大，频率整体越低，最低频率对应的周期越长
- base=10000 时，最高维度的周期约 62832，足够覆盖几千 token 的训练长度
- **增大 base**（如 NTK-Aware）：等效于让低频维度覆盖更远的距离，实现长度外推
- **减小 base**：高频更集中在近距离，远距离感知变弱
- LLaMA 3 使用 base=500000 来原生支持更长上下文

### 面试题 7：Learned PE 和 Sinusoidal PE 实验效果差不多，为什么后来的模型都不用了？

**答案要点**：
- 两者都是绝对位置编码，核心问题是**无法外推**到训练长度之外
- Learned PE 完全没有外推能力（没见过的位置 = 随机噪声）
- 现代 LLM 需要处理长上下文（4K→128K+），绝对 PE 完全不适用
- RoPE/ALiBi 是相对位置编码，天然支持任意长度
- 此外，RoPE 的点积编码相对距离的性质，比绝对 PE 加法混合更优雅

### 面试题 8：如果让你设计一个支持 100 万 token 上下文的位置编码，你会怎么做？

**答案要点**（开放题，展示思考深度）：
- 基础选择：RoPE（已证明有效，工程成熟）
- 长度外推：YaRN 或 LongRoPE，通过搜索最优的维度缩放因子
- base 调整：使用大 base（如 500000），原生覆盖更长周期
- 分层策略：底层注意力用局部窗口（如 Sliding Window），高层用全局
- 配合架构：结合稀疏注意力（如 Ring Attention、Block-sparse）
- 考虑 CoPE：超长序列中 token 索引位置意义下降，语义位置更重要
- 工程约束：KV Cache 内存管理、分布式注意力计算

### 面试题 9：RoPE 和 Sinusoidal PE 都用了正弦/余弦函数，本质区别是什么？

**答案要点**：
- **注入方式不同**：Sinusoidal 加到 embedding 上（加法），RoPE 作用在 Q/K 上（乘法/旋转）
- **对点积的影响**：Sinusoidal 加法后经过 W_Q/W_K 投影，位置信息被混合稀释；RoPE 直接在投影后旋转，精确保证点积编码相对位置
- **数学保证**：RoPE 有严格证明 $\langle f(q,m), f(k,n) \rangle = g(q,k,m-n)$；Sinusoidal 只是"理论上可以"通过学习获得
- **外推能力**：RoPE 配合各种扩展方法可以大幅外推；Sinusoidal 的外推很有限

### 面试题 10：为什么 RoPE 作用在 Q/K 上而不作用在 V 上？

**答案要点**：
- 位置编码的目标是让模型**根据位置关系分配注意力权重**
- 注意力权重由 Q·K 的点积决定 → 位置信息需要编码在 Q 和 K 中
- V 是被聚合的"内容"，它不参与"决定看哪里"的计算
- 如果 V 也旋转，会改变输出的语义内容（不仅改变注意力分配，还改变了被注意的内容），这通常不是期望的行为
- 从信息论角度：位置影响"路由"（Q·K），不应影响"信息"（V）

## 15. 易错点与最佳实践

### 15.1 常见错误

**错误 1：RoPE 维度不匹配**

```python
# ❌ 错误：RoPE dim 应该等于 head_dim，不是 d_model
rope = RotaryEmbedding(dim=d_model)  # 错！

# ✓ 正确：
rope = RotaryEmbedding(dim=d_model // n_heads)  # head_dim
```

**错误 2：忘记 RoPE 只作用于部分维度**

某些实现（如 GPT-NeoX）只对 head_dim 的前 $r$ 个维度应用 RoPE，后面的维度不旋转：

```python
# 部分旋转（GPT-NeoX 风格）
def partial_rope(q, cos, sin, rotary_dim):
    q_rot = q[..., :rotary_dim]
    q_pass = q[..., rotary_dim:]
    q_rot = q_rot * cos + rotate_half(q_rot) * sin
    return torch.cat([q_rot, q_pass], dim=-1)
```

**错误 3：Position Interpolation 忘记微调**

```python
# ❌ 直接用 PI 推理，不微调 → 性能严重下降
# PI 改变了位置编码的分布，模型需要适应新的分布

# ✓ 正确流程：
# 1. 修改 RoPE 的缩放因子
# 2. 在目标长度的数据上微调 1000-2000 步
# 3. 然后推理
```

**错误 4：ALiBi 的 causal mask 处理**

```python
# ❌ ALiBi 的偏置不需要额外 mask（偏置本身就是负值，远距离自然被抑制）
# 但 causal attention 仍需要 mask future tokens

# ✓ 正确：
attn_scores = q @ k.T / sqrt(d) + alibi_bias  # 加偏置
attn_scores = attn_scores.masked_fill(causal_mask, float('-inf'))  # mask 未来
```

**错误 5：混淆 rotate_half 的两种实现**

```python
# 实现 A (half-half): 前半和后半交换
def rotate_half_A(x):
    x1 = x[..., :x.shape[-1]//2]
    x2 = x[..., x.shape[-1]//2:]
    return torch.cat([-x2, x1], dim=-1)

# 实现 B (interleaved): 相邻维度交换
def rotate_half_B(x):
    x = x.reshape(*x.shape[:-1], -1, 2)
    x = torch.stack([-x[..., 1], x[..., 0]], dim=-1)
    return x.reshape(*x.shape[:-2], -1)

# 两者数学上等价但布局不同！
# 必须与预训练模型的实现保持一致，否则加载权重后性能崩溃
```

### 15.2 最佳实践

1. **生产环境使用 RoPE**：当前最成熟、最主流、生态支持最好的方案

2. **长度外推首选 YaRN**：如果需要超出训练长度，YaRN 是综合最优的选择

3. **预计算并缓存**：cos/sin 值在模型初始化时就计算好，推理时只做查表

4. **注意精度**：RoPE 的三角函数计算建议用 float32，存储可以用 float16/bfloat16

5. **与 Flash Attention 配合**：现代推理框架（vLLM, TGI）已将 RoPE 融合到 attention kernel 中

6. **测试外推时务必验证 perplexity**：不要假设某种外推方法在所有场景都有效

7. **base 的选择**：
   - 标准场景：10000（原始 RoPE）
   - 长上下文预训练：500000（LLaMA 3）
   - 动态调整：seq_len > train_len 时自动增大（Dynamic NTK）

8. **GQA 兼容性**：确保 KV heads 的 RoPE 应用与 Q heads 一致

## 附录 B：数学证明补充

### B.1 RoPE 相对位置性质的完整证明

**定理**：对于 RoPE 编码函数 $f(x, m) = R_m x$（其中 $R_m$ 是位置 $m$ 的分块旋转矩阵），有：

$$
\langle f(q, m), f(k, n) \rangle = \langle q, R_{n-m} k \rangle
$$

**证明**：

对于每个二维子空间 $i$，旋转矩阵为：

$$
R_m^{(i)} = \begin{bmatrix} \cos(m\theta_i) & -\sin(m\theta_i) \\ \sin(m\theta_i) & \cos(m\theta_i) \end{bmatrix}
$$

旋转矩阵的性质：$R_m^\top = R_{-m}$，且 $R_m R_n = R_{m+n}$。

因此：

$$
\begin{aligned}
\langle R_m q, R_n k \rangle &= (R_m q)^\top (R_n k) \\
&= q^\top R_m^\top R_n k \\
&= q^\top R_{-m} R_n k \\
&= q^\top R_{n-m} k
\end{aligned}
$$

展开到具体计算：

$$
\begin{aligned}
&(R_m^{(i)} q^{(i)})^\top (R_n^{(i)} k^{(i)}) \\
=\;& (q_0\cos m\theta - q_1\sin m\theta)(k_0\cos n\theta - k_1\sin n\theta) \\
&+ (q_0\sin m\theta + q_1\cos m\theta)(k_0\sin n\theta + k_1\cos n\theta) \\
=\;& q_0 k_0(\cos m\theta\cos n\theta + \sin m\theta\sin n\theta) \\
&+ q_1 k_1(\sin m\theta\sin n\theta + \cos m\theta\cos n\theta) \\
&+ q_0 k_1(-\sin m\theta\cos n\theta + \cos m\theta\sin n\theta) \\
&+ q_1 k_0(-\cos m\theta\sin n\theta + \sin m\theta\cos n\theta) \\
=\;& q_0 k_0\cos(m-n)\theta + q_1 k_1\cos(m-n)\theta \\
&+ q_0 k_1\sin(n-m)\theta + q_1 k_0\sin(m-n)\theta \\
=\;& q^\top R_{n-m} k \quad \square
\end{aligned}
$$

### B.2 正弦编码的唯一性证明

**命题**：对于 $d_{\text{model}} \geq 2$，正弦编码的不同位置向量线性无关（因此两两不同）。

**证明思路**：考虑位置 $p$ 和 $q$（$p \neq q$）。第 $i$ 维的编码分别为 $\sin(p \cdot \omega_i)$ 和 $\sin(q \cdot \omega_i)$。由于 $\omega_i$ 是无理数的集合，且 $\sin$ 是非周期的无理频率组合，不同位置的编码不可能相同。

更严格地，可以通过构造性证明：取最高频率 $\omega_0 = 1$，$\sin(p)$ 和 $\sin(q)$ 在 $p \neq q$ 且两者不关于 $\pi/2$ 对称时不同。即使某一维度碰巧相同，由于使用了多个不同频率，其他维度几乎确定不同。
